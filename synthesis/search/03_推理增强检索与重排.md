## 一、范式演进

```
BM25 (统计) → DPR/E5 (对比学习双塔) → Cross-encoder (精确但慢)
     ↓ 推理能力注入 (2024-2026)
O1-Embedder (推理链增强 Embedding)
  → Rank-R1/REARANK/LimRank/DEAR (推理重排)
    → Qagent/Search-R1 (推理 Agent)
```

**核心转变**：从"语义相似性匹配"走向"推理驱动的相关性判断"。传统检索：query-doc 词汇/语义重叠即相关；推理检索：需要逻辑/知识推导才能判断相关性（如"比铝轻的金属" → 需要密度知识推理）。

## 二、核心技术维度

### 2.1 检索端：推理增强 Embedding

#### O1-Embedder — 先思考再编码

$$
e_q = \text{Encoder}(\underbrace{\text{Think}(q)}_{\text{推理链}} \oplus q)
$$

- 推理过程：分析隐含意图 → 扩展关键概念 → 消除歧义 → 增强编码
- 与 ANN 索引完全兼容（只改 query 编码，document 侧不变）
- 延迟增加 50-100ms

#### LREM — 电商搜索推理嵌入（淘宝/天猫部署）

- **Stage 1 SFT + InfoNCE**：教模型生成推理链 + 对齐 query-item embedding
- **Stage 2 RL**：优化推理轨迹质量
- 首个大规模工业验证的推理嵌入范式

#### ReasonEmbed — 训练时注入推理能力

- **ReMixer**：三阶段数据合成（条件 query 生成 → 源排除候选挖掘 → 推理增强标注），82K 高质量样本
- **Redapter**：动态调整样本权重，推理越难权重越高
- BRIGHT nDCG@10 = **38.1**（SOTA）

**两条路线对比**：

| 维度 | 推理时增强 (O1-Embedder/LREM) | 训练时增强 (ReasonEmbed/zELO) |
|------|------------------------------|------------------------------|
| 额外推理延迟 | 50-100ms | 0ms |
| 推理深度 | 灵活可调 | 固定在模型参数中 |
| 适用场景 | 低频精准检索（法律/医疗） | 高频通用检索 |

### 2.2 重排端：推理驱动的相关性判断

#### Rank-R1 — RL 推理重排

$$
r = \text{nDCG@K}(\hat{\pi}, \pi^*), \quad \mathcal{L}_{GRPO} = -\mathbb{E}\left[\frac{\pi_\theta}{\pi_\text{old}} \cdot \hat{r}\right], \quad \hat{r}_i = \frac{r_i - \bar{r}}{\text{std}(r)}
$$

- 奖励直接为 nDCG，无需人工标注推理链
- GRPO 无需 critic 网络（组内归一化），比 PPO 稳定、省内存 50%
- BRIGHT 上超越 GPT-4

#### LimRank — Less is More 选择性压缩

- 压缩掉低信息量段落 → 模型关注更集中 → 推理更准确
- Early Exit：当前候选集质量足够时提前终止
- 减少 40-60% token 消耗，精度持平

#### DEAR — 蒸馏让小模型获得推理能力

$$
\mathcal{L}_{distill} = \text{KL}(P_{student}(\text{rank}|q,D) \| P_{teacher}(\text{rank}|q,D))
$$

- 1.5B 小模型接近 GPT-4 重排效果
- 蒸馏传递的是推理结构（ranking rationale），不是参数
- 50K 蒸馏样本即可，极高数据效率

#### ERank — SFT + RL 两阶段高效方案

$$
R_{listwise}(\mathbf{S}, \pi^*) = \text{nDCG@10}(\text{argsort}(-\mathbf{S}), \pi^*)
$$

- SFT 提供初始化 + 细粒度判别力；RL 注入全局排序感知
- ERank-4B BRIGHT nDCG@10 = **38.7**，ERank-32B = **40.2**
- 目前最佳实践

#### REARANK — 极少标注的 RL 推理重排

- 仅 179 个标注样本即可 RL 训练
- RL 的"奖励驱动探索"比 SFT 更数据高效

#### Rank1 — Test-Time Compute

$$
\text{nDCG@10} \approx \alpha \cdot \log(T) + \beta
$$

- CoT 长度 T 可控：小 T = 快速低成本，大 T = 深度推理高准确率
- R1 蒸馏 60 万条推理轨迹到 7B 模型

**重排方案对比**：

| 方案 | 参数量 | 推理延迟 | 工业可行性 | 核心创新 |
|------|--------|----------|-----------|----------|
| DEAR | 1.5B | 20-40ms | ★★★★★ | 蒸馏 |
| LimRank | 7B | 50-80ms | ★★★★ | 压缩+Early Exit |
| Reasonrank | 7B | 100-200ms | ★★★ | CoT+RL |
| ERank-4B | 4B | ~60ms | ★★★★ | SFT+RL listwise |
| Rank1-7B | 7B | ~200ms | ★★★ | Test-time compute |

### 2.3 全流程：搜索 Agent

#### Qagent — 模块化搜索 Agent

$$
a_t = \text{Agent}(q, R_t, \text{History}_t), \quad a_t \in \{\text{refine}(q), \text{rerank}(R_t), \text{finalize}\}
$$

- 根据中间结果动态决策：改写/重排/终止
- 终止条件：相关性评分超阈值 OR 最大轮数

#### Search-R1 — 主动搜索智能体

$$
\pi_\theta: s_t \rightarrow a_t \in \{\text{think}(\cdot), \text{search}(q), \text{answer}(a)\}
$$

- 多跳推理：think → search → result → think → search → answer
- GRPO 以最终答案正确性为 reward

### 2.4 标签质量优化：zELO

```
LLM Ensemble → 稀疏 pairwise 偏好 → Thurstone 模型 → 绝对 zELO 分数 → Pointwise 微调
```

- **Thurstone > Bradley-Terry**：正态分布假设（中心极限定理）比 Gumbel 更符合排序数据
- 随机环采样 O(n)（非 O(n^2)），每 query 仅 ~400 对比较
- zerank-1 在金融/法律/代码/STEM 上超越所有闭源 reranker
- 无需人工标注，<10K H100-hours

### 2.5 文档切分：Evidence Units

- 本体论归一化（DoCO）+ 语义分配 + 图验证 + 跨 parser 验证
- **Recall@1：0.15 → 0.51（3.4x）**
- 核心启示：好的 chunk 不是固定长度切分，而是语义完整的证据单元

## 六、核心洞察

1. **推理能力注入 IR 是不可逆趋势**：Embedding 的"语义"定义不断升级——从词汇相似到语义相似到逻辑相关
2. **"Less is More" 是推理重排的工程关键**：给 LLM 喂更精炼的文档比喂更多文档效果好
3. **蒸馏是推理能力下沉的核心机制**：GPT-4 推理能力可通过 50K 样本蒸馏到 1.5B 模型
4. **GRPO 成为 IR RL 训练标配**：无需 critic 网络，直接在排序指标上做 RL
5. **训练时 vs 推理时注入是核心架构选择**：没有绝对优劣，取决于 SLA
6. **文档切分质量是 RAG 的隐藏瓶颈**：Evidence Units 3.4x Recall 提升证明本体驱动 > 启发式
7. **Embedding 训练的无监督革命**：zELO 证明无需人工标注也能训练 SOTA reranker

## 七、面试 Q&A（20 题）

### Q1: 为什么搜索需要推理能力？
许多查询的相关性基于逻辑推导而非语义相似度。"比铝轻的金属"与"镁材料应用"词汇/语义不相似，但逻辑上高度相关。

### Q2: Rank-R1 为什么不需要 CoT 标注数据？
RL 奖励来自排序是否正确（nDCG），模型通过探索自发学习推理链。

### Q3: DEAR 1.5B 为什么能接近 GPT-4？
蒸馏保留了推理模式（ranking rationale），小模型学会"判断相关性的思维方式"。

### Q4: LimRank 的 Less is More 原理？
LLM 有效注意力随 context 增加而分散（Lost in the Middle）。压缩低信息量段落后关注更集中。

### Q5: O1-Embedder 如何兼容现有 ANN 索引？
只改 query encoding（增加推理链前缀），document 侧不变。

### Q6: 推理延迟在搜索场景可接受吗？
电商需 50ms 以内 → DEAR（20-40ms）；法律/医疗可容忍 100-500ms → O1-Embedder/Reasonrank。

### Q7: zELO 为什么选 Thurstone 而非 Bradley-Terry？
排序偏好噪声是多因素叠加，中心极限定理趋近正态分布，Thurstone 假设正态更合理。

### Q8: ReasonEmbed ReMixer 如何解决合成数据平凡性？
三阶段：条件 query 生成 → 源排除候选挖掘 → 推理增强标注，保证训练数据真正需要推理。

### Q9: LREM 两阶段为什么比纯 SFT/RL 好？
SFT 暖启动（cold start），RL 探索更优轨迹（reward=检索质量）。

### Q10: Evidence Units 对 RAG chunking 的启示？
固定长度 chunking 拆散语义连贯内容。EU 基于文档本体将关联元素组装为语义完整单元。

### Q11: GRPO 和 PPO 在推理重排中的对比？
PPO 需额外 critic 网络（不稳定，内存高）；GRPO 组内对比归一化，无需 critic，内存 -50%。

### Q12: ERank 两阶段各解决什么问题？
仅 SFT 二分类 nDCG=29.3；SFT+RL listwise nDCG=38.7。SFT 提供初始化，RL 注入全局排序感知。

### Q13: REIC 用 RAG 做分类有什么优势？
新增 intent 只需更新索引无需重训；跨业务泛化；维护成本低。

### Q14: 搜索 Agent 和 RAG 的根本区别？
RAG 是"一次检索+生成"（被动）；Agent 是"检索+评估+改写+再检索"（主动，可迭代）。

### Q15: Test-Time Compute 在搜索重排中的意义？
允许模型推理阶段使用更多计算（更长 CoT），性能与 log(T) 近似线性。

### Q16: FollowIR 发现了什么问题？
现有 IR 模型只提取关键词，无法处理否定条件、长指令，p-MRR 测量指令响应能力。

### Q17: ThinkQE 的工程挑战？
每轮迭代需额外检索+LLM推理（+300-500ms），Jaccard 判断收敛，实践中限制 T=2。

### Q18: Listwise vs Pointwise 重排如何选？
Listwise 适合高质量可接受高延迟（>500ms）；Pointwise 适合实时搜索 <100ms。推荐 Hybrid。

### Q19: 推理链能作为搜索结果解释吗？
可以，在专业搜索（法律/学术）中透明度比速度更重要。

### Q20: 推理增强检索工业落地最大障碍？
(1) 延迟 50-200ms；(2) 成本（1000 查询 x 500 token = 50 万 token）；(3) 一致性。解法：蒸馏+缓存+确定性推理。
