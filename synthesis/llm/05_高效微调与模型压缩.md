## 一、LoRA 核心原理

### 低秩分解

$$
W' = W_0 + \Delta W = W_0 + BA, \quad B \in \mathbb{R}^{d \times r},\ A \in \mathbb{R}^{r \times k},\ r \ll \min(d, k)
$$

前向传播：

$$
h = W_0 x + \frac{\alpha}{r} B A x
$$

### 参数量对比

以 $d = k = 4096, r = 8$ 为例：
- 全参数：$d \times k = 16.8\text{M}$
- LoRA：$(d + k) \times r = 65\text{K}$
- **节省 99.6%**

### 理论基础：SVD 与低秩假设

预训练权重 $W_0 = U \Sigma V^\top$ 中，微调增量 $\Delta W$ 的大部分奇异值接近 0，低秩假设成立。Intrinsic Dimensionality 研究表明大多数 NLP 任务可在 <200 维子空间内微调。

### 初始化策略

$B = 0, A \sim \mathcal{N}(0, \sigma^2)$：训练开始时 $BA = 0$，模型从预训练权重直接出发，无随机扰动。缩放因子 $\alpha/r$ 控制初始学习步长。

### LoRA 部署

| 模式 | 推理开销 | 多任务 | 适用 |
|------|---------|-------|------|
| Merged（$W = W_0 + BA$） | 零 | 需重加载 | 单任务生产 |
| Adapter（分离） | 少量 | 动态切换 | 多任务服务 |

## 二、PEFT 技术演进

```
全参数微调(~2020) → Adapter(2019) → Prefix-Tuning(2021)
  → LoRA(2021) → QLoRA(2023) → DoRA/LoRA+(2024)
    → BOFT/AdaLoRA(2024) → Hybrid PEFT(2025) → LoRAFusion(2026)
```

### LoRA 家族变体

| 方法 | 年份 | 核心改进 | 特色 |
|------|------|---------|------|
| LoRA | 2021 | 低秩分解 ΔW=BA | PEFT 事实标准 |
| QLoRA | 2023 | NF4 量化 Base + LoRA | 单卡 A100 训练 65B |
| LoRA+ | 2024 | 更优初始化 | 收敛更快 |
| DoRA | 2024 | 方向+幅度分解 | 更细粒度控制 |
| AdaLoRA | 2024 | 自适应秩分配 | 不同层不同 rank |
| BOFT | 2024 | 蝶形正交微调 | 保持权重谱结构 |
| LoRA-GA | 2024 | 梯度对齐初始化 | 收敛最快 |
| LUNE | 2025 | LoRA + 负样本 | 精准知识遗忘 |
| Hybrid PEFT | 2025 | BOFT + LoRA-GA 自适应混合 | 综合最优 |
| LoRAFusion | 2026 | Kernel Fusion + 多任务调度 | 训练吞吐 +2.3x |

### QLoRA 三板斧

1. **NF4 量化**：Normal Float 4-bit，专为正态分布设计
2. **双重量化**：量化参数本身也被量化
3. **分页优化器**：优化器状态卸载到 CPU

峰值显存对比（7B 模型）：全参数微调 ~80GB → LoRA ~16GB → QLoRA ~6GB

### BOFT 正交微调

$$
W' = O^T W_0 O, \quad \text{s.t.} \quad OO^T = I
$$

蝶形矩阵结构的正交矩阵保持预训练权重的谱结构 $\sigma(O^T W O) = \sigma(W)$。

### Hybrid PEFT 自适应混合

$$
\Delta W_l^{\text{hybrid}} = \alpha_l(t) \cdot \Delta W_l^{\text{LoRA-GA}} + (1 - \alpha_l(t)) \cdot \Delta W_l^{\text{BOFT}}
$$

$$
\alpha_l(t) = \sigma\left(\frac{\|\nabla_l(t)\|_F - \tau}{\beta}\right)
$$

梯度范数大的层用 LoRA-GA（快速适应），梯度范数小的层用 BOFT（保持结构）。

## 三、模型量化技术演进

### 量化路线

```
第一阶段：权重量化 → PolarQuant, BitNet
第二阶段：KV Cache 量化 → KVQuant
第三阶段：弹性量化 → MF-QAT
第四阶段：自动化混合精度 → OneComp/AutoBit
```

### 量化方法对比

| 方法 | 量化对象 | 精度 | 核心操作 | 适用场景 |
|------|---------|------|---------|---------|
| GPTQ | 权重 | 4-bit | 后训练量化 | 通用部署 |
| AWQ | 权重 | 4-bit | 激活感知 | 通用部署 |
| PolarQuant | 权重 | 4-bit | Hadamard 旋转 + 高斯匹配量化 | 高质量压缩 |
| KVQuant | KV Cache | 3-bit | Per-channel + Pre-RoPE | 长上下文 |
| MF-QAT | 权重+激活 | 多格式 | Slice-and-Scale 格式转换 | 跨平台部署 |
| BitNet | 权重 | 1.58-bit | $W \in \{-1, 0, +1\}$ | CPU/边缘设备 |
| LCD | 权重 | 2-bit | 聚类量化 + 蒸馏辅助 | 极致压缩 |
| OneComp/AutoBit | 权重（混合精度） | 自适应 2-8bit | ILP 优化 per-layer bitwidth | 一键部署 |

### KVQuant 关键洞察

RoPE 位置编码会扭曲 Key 分布 → Pre-RoPE 量化效果更好。

KV Cache 内存：$\text{Mem}_{KV} = 2 \times L \times H \times d \times N \times \text{bits}/8$

### OneComp/AutoBit 自动混合精度

基于 ILP（整数线性规划）的 per-layer bitwidth 分配：

$$
\min \sum_l \epsilon_l(b_l) \quad \text{s.t.} \quad \sum_l m_l(b_l) \leq \text{VRAM}
$$

- 早期层和 attention output projection 更脆弱（高精度）
- 中间 MLP 层更耐压缩（低精度）

渐进量化流水线：Layer-wise PTQ → Block-wise PTQ → Global PTQ，每阶段产出可部署 checkpoint。

### 量化精度保留对比

| 方法 | 比特数 | 内存压缩 | 精度保留 | 工业可行性 |
|------|--------|---------|---------|-----------|
| FP16 | 16-bit | 1x | 100% | 高 |
| GPTQ/AWQ | 4-bit | 4x | ~98% | 高 |
| LCD | 2-bit | 8x | ~90% | 中 |
| BitNet | 1-bit | 16x | ~80% | 低 |

## 四、LUNE：LoRA 驱动的知识遗忘

$$
\mathcal{L}_{\text{total}} = \underbrace{-\mathbb{E}_{(x, y^-) \sim \mathcal{D}_{\text{forget}}} \log p_\theta(y^- | x)}_{\text{遗忘损失}} + \lambda \underbrace{\mathbb{E}_{(x, y) \sim \mathcal{D}_{\text{retain}}} [-\log p_\theta(y | x)]}_{\text{保留损失}}
$$

LoRA + 负样本微调：约 1/10 全量微调成本，可逆（移除 adapter），稳定。

## 五、微调 vs RAG vs Prompt Engineering 选型

### 决策框架

```
知识频繁更新（日/周级）？
├── 是 → RAG
└── 否 → 格式/风格问题？
    ├── 是 → Fine-tuning
    └── 否 → 数据量？
        ├── < 100条 → RAG + Few-shot
        ├── 100-10K → LoRA Fine-tuning
        └── > 10K → Full FT / Continue Pretraining
```

### 成本对比（7B 模型）

| 方案 | 一次性成本 | 运行成本 | 更新成本 |
|------|----------|---------|---------|
| Prompt Engineering | 低 | 高（长 prompt） | 低 |
| RAG | 中（向量库） | 中 | 低（更新文档） |
| LoRA FT | 中 | 低（短 prompt） | 中（重训） |
| Full FT | 高 | 低 | 高 |

### 关键准则

- 微调数据质量 >> 数据数量：1000 条高质量 > 10000 条低质量
- 用 Fine-tuning 教模型"怎么想"，用 RAG 告诉模型"知道什么"
- LoRA 学到的更多是**格式、风格、推理模式**，而非具体事实

## 六、工业实践要点

### LoRA 调参经验

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| rank r | 16 起步 | 数据多/任务复杂 r=32-64，简单任务 r=4-8 |
| 目标层 | Q/V 矩阵 | 全 linear 层也可，rank 设小些 |
| 学习率 | 1e-4 ~ 3e-4 | 比全参数微调高 |
| alpha | 通常 = 2r | 控制 LoRA 贡献比例 |

### 量化部署选型

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 通用推理服务 | GPTQ/AWQ 4-bit | 成熟、精度高 |
| 长上下文 | KVQuant 3-bit KV | 解决 KV Cache 瓶颈 |
| 跨平台部署 | MF-QAT | 一次训练多格式 |
| CPU/边缘设备 | BitNet 1-bit | 极致压缩 |
| 快速适配硬件 | OneComp | 自动混合精度 |
| 研究/快速验证 | LCD 2-bit | 8x 压缩，90% 精度 |

## 面试高频 Q&A

### Q1: LoRA 的原理？
**30秒**：冻结 $W_0$，训练低秩增量 $\Delta W = BA$（$r \ll d$）。推理时可合并 $W = W_0 + BA$，无额外延迟。rank 通常 16-64，参数节省 99%+。

### Q2: LoRA 应该加在哪些层？
**30秒**：Q/V 矩阵效果最好，K/O 收益递减。实践中可在所有 linear 层加 LoRA，rank 设小。

### Q3: QLoRA 的关键技术？
**30秒**：NF4 量化（专为正态分布设计）+ 双重量化（量化参数再量化）+ 分页优化器（卸载到 CPU）。65B 模型从 130GB 降至 35GB，单卡 A100 可训。

### Q4: 为什么不同层应该用不同 bitwidth？
**30秒**：Transformer 各层量化敏感度差异大——early layers 和 attn_o_proj 脆弱（高精度），mid MLP 层耐压（低精度）。OneComp/AutoBit 用 ILP 自动分配。

### Q5: KV Cache 量化为什么 Key 比 Value 更难？
**30秒**：RoPE 位置编码引入分布偏斜，Key 经 RoPE 后分布不均匀。Pre-RoPE 量化效果更好。

### Q6: 微调 vs RAG 怎么选？
**30秒**：知识常更新选 RAG，格式风格固定选 FT，两者常组合：FT 教模型理解领域 + RAG 注入实时知识。

### Q7: LoRA 微调后如何部署？
**30秒**：单任务用 Merged 模式（$W = W_0 + BA$，零延迟）。多任务服务用 Adapter 模式（分离存储，动态切换，如 S-LoRA 框架）。

### Q8: Hybrid PEFT 为什么优于单一 PEFT？
**30秒**：LoRA-GA 收敛快但可能破坏权重结构，BOFT 保持结构但收敛慢。Hybrid 根据梯度范数自适应混合——梯度大用 LoRA-GA（快速适应），梯度小用 BOFT（保持稳定）。

### Q9: 渐进量化 vs 一步到位的优势？
**30秒**：每阶段都有可部署 checkpoint，可根据质量-压缩 tradeoff 提前停止；后续阶段在前阶段基础上 refine，收敛更稳定。

### Q10: 机器遗忘如何实现？
**30秒**：LUNE 用 LoRA + 负样本微调——最大化遗忘数据上生成负样本的概率 + 保持保留集正常。约 1/10 全量微调成本，可逆（移除 adapter）。

## 记忆助手

- **LoRA = 贴便签**：不改原书（冻结 W0），在关键页面贴便签（BA），推理时可粘回原书（merge）
- **QLoRA = 压缩原书+贴便签**：口袋版原书（4bit） + 便签（LoRA），75% 省空间
- **量化四阶段**：权重量化 → KV Cache 量化 → 弹性量化 → 自动混合精度
- **LoRA 口诀**："W = W0 + (α/r) × BA，W0 冻结只训 BA"
- **选型口诀**："资源充足 LoRA，极度省钱 QLoRA，多任务 Adapter，自动压缩 OneComp"
