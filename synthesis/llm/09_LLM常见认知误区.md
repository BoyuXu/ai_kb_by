# LLM 常见认知误区：学界 vs 工业界

> 综合自 LLM常见认知误区 | 更新：2026-04-13 | 领域：LLM 工程实践
> 关联：[[01_LLM推理优化全景]] | [[03_RAG系统全景与决策框架]]

> 这些误区在面试中说出来会减分，在工作中踩到会浪费时间。

---

## 误区 1："更大的模型一定更好"

**学界看法**：Scaling Law 证明参数量越大，Loss 越低。

**工业界实践**：
- 7B LoRA fine-tuned 在特定任务上可以超过 GPT-4
- 70B 模型推理成本是 7B 的 10x，延迟是 3-5x
- **实际选型**：能用 7B 解决的任务，绝对不用 70B

**正确观点**：在同等质量下选最小的模型；只有当小模型不够用时才扩大。

---

## 误区 2："RAG 能解决所有知识问题"

**学界看法**：RAG 提供外部知识，减少幻觉。

**工业界实践**：
- RAG 的瓶颈在检索质量：chunk 切分不好或 query-document 语义差异大时，检索内容无关
- "检索到的内容对不对"本身也需要 LLM 判断，递归问题

**RAG 失效的典型场景**：
1. Query 需要跨多个文档综合（不是找"一个文档"而是"综合多个"）
2. Query 本身含有错误假设（检索内容会强化错误）
3. 文档更新但向量库没同步（陈旧 embedding）

**正确观点**：RAG 是工具，不是银弹。检索质量的上限决定了 RAG 效果的上限。

---

## 误区 3："CoT 让模型更聪明"

**学界看法**：Chain-of-Thought prompting 显著提升推理能力。

**工业界实践**：
- CoT 让模型"展示推理过程"，但本质是串联"看起来像对的推理步骤"，不是真正逻辑推理
- 对**有标准答案的任务**（数学、代码）效果好；对**主观任务**（建议、创意）提升有限
- 过长的 CoT 反而有害：中间步骤出错，后续步骤会强化错误

**正确观点**：CoT 是对模型能力的"召唤"，不是赋予。模型没有相关能力，CoT 也召唤不出来。

---

## 误区 4："Fine-tuning 会让模型记住所有训练数据"

**学界看法**：Fine-tuning 更新模型权重，新知识被记住。

**工业界实践**：
- Fine-tuning 容易导致**灾难性遗忘**（Catastrophic Forgetting）：学了新领域，忘了通用能力
- 模型不是数据库，无法精确"查到"训练数据中的某条信息
- LoRA Fine-tuning 学到的更多是**格式、风格、推理模式**，而不是具体事实

**正确观点**：用 Fine-tuning 教模型"怎么想"，用 RAG 告诉模型"知道什么"。

---

## 误区 5："LLM 在推理时不耗费显存"

**工业界实践**：
- KV Cache 随序列长度增长占据大量显存
- 长 context 推理时，KV Cache 可能超过模型权重本身（70B 模型权重 ~140GB，4096 token 的 KV Cache ~40GB）
- Batch 推理时，每个请求都有独立的 KV Cache，显存需求 $\propto$ batch_size $\times$ sequence_length

**正确观点**：推理时显存 = 模型权重 + 激活值 + KV Cache。KV Cache 是动态的，会随对话持续增长。

**KV Cache 内存公式**：

$$
\text{KV}_{Mem} = 2 \times L \times H_{KV} \times d_k \times N \times s
$$

详见 [[01_LLM推理优化全景]] 中 KV Cache 优化章节。

## 面试应用

这些误区在面试中常以"你怎么看 X"的形式出现。正确回答框架：

1. 承认学界观点有道理（体现知识面）
2. 指出工业实践中的 nuance（体现实战经验）
3. 给出"正确观点"的平衡表述（体现成熟度）

### 常见追问

| 误区 | 面试追问 | 答法 |
|------|---------|------|
| 模型越大越好 | 怎么决定模型大小？ | 先在目标任务上评估 7B→13B→70B，选满足需求的最小模型 |
| RAG 万能 | RAG 失效怎么办？ | 改善检索（chunk/embedding/reranker），或 FT 领域模型 |
| CoT 让模型更聪明 | CoT 什么时候无效？ | 主观任务、模型本身无该领域知识时 |
| FT 记住一切 | 怎么注入知识？ | FT 教格式/风格，RAG 注入事实，两者组合最优 |
| 推理不耗显存 | 长对话怎么处理？ | KV Cache 管理：PagedAttention/H2O 驱逐/量化 |
