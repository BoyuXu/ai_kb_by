## 2. 竞价机制（GSP/VCG）原理

### 2.1 机制设计基础

#### 关键概念
- **激励相容 (Incentive Compatible)**：广告主如实报价是最优策略
- **个体理性 (Individual Rationality)**：参与拍卖不会亏损
- **社会福利最大化 (Social Welfare)**：整体价值最大化

### 2.2 GSP (Generalized Second Price) 广义第二价格

```
┌────────────────────────────────────────────────────────────────┐
│                      GSP 竞价示例                               │
├────────────────────────────────────────────────────────────────┤
│                                                                 │
│  广告主    出价(Bid)    质量度(Q)    得分(RankScore)    排名    │
│  ─────────────────────────────────────────────────────────────  │
│    A        ¥10          0.9      10 × 0.9 = 9.0        #1      │
│    B        ¥8           0.8      8 × 0.8 = 6.4         #2      │
│    C        ¥5           0.9      5 × 0.9 = 4.5         #3      │
│    D        ¥4           0.7      4 × 0.7 = 2.8         #4      │
│                                                                 │
│  ═══════════════════════════════════════════════════════════    │
│                                                                 │
│  广告主A (排名#1) 实际扣费:                                       │
│  计费 = (下一名得分 / 自己质量度) + 0.01 = 6.4 / 0.9 + 0.01 ≈ ¥7.12 │
│                                                                 │
│  广告主B (排名#2) 实际扣费:                                       │
│  计费 = 4.5 / 0.8 + 0.01 ≈ ¥5.64                                 │
│                                                                 │
│  广告主C (排名#3) 实际扣费:                                       │
│  计费 = 2.8 / 0.9 + 0.01 ≈ ¥3.12                                 │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

**GSP 特点：**
| 特性 | 说明 |
|-----|------|
| 非激励相容 | 广告主可能虚报低价以获得更低扣费 |
| 计算高效 | O(n log n) 排序复杂度 |
| 实际广泛应用 | Google Ads、Meta Ads均采用变体 |
| 存在多重均衡 | 可能产生不稳定出价策略 |

### 2.3 VCG (Vickrey-Clarke-Groves) 机制

```
┌────────────────────────────────────────────────────────────────┐
│                      VCG 竞价示例                               │
├────────────────────────────────────────────────────────────────┤
│                                                                 │
│  场景: 3个广告位，4个广告主竞标                                   │
│                                                                 │
│  广告主    出价(真实价值)     质量度     得分                    │
│  ────────────────────────────────────────────────               │
│    A           ¥10            1.0       10.0                    │
│    B           ¥8             1.0       8.0                     │
│    C           ¥6             1.0       6.0                     │
│    D           ¥4             1.0       4.0                     │
│                                                                 │
│  分配: A→位置1, B→位置2, C→位置3 (按点击率加权价值)               │
│                                                                 │
│  ═══════════════════════════════════════════════════════════    │
│                                                                 │
│  广告主A的VCG价格计算:                                            │
│  ─────────────────────────────────                              │
│  没有A参与时: B→位置1, C→位置2, D→位置3                          │
│  社会福利 = 8×CTR₁ + 6×CTR₂ + 4×CTR₃                            │
│                                                                 │
│  有A参与时:   A→位置1, B→位置2, C→位置3                          │
│  其他人福利 = 8×CTR₂ + 6×CTR₃                                   │
│                                                                 │
│  A造成的外部性 = (8×CTR₁ + 6×CTR₂ + 4×CTR₃) - (8×CTR₂ + 6×CTR₃)  │
│                                                                 │
│  ∴ A的VCG价格 = 造成的外部性                                     │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

**VCG 特点：**
| 特性 | 说明 |
|-----|------|
| 激励相容 | 广告主真实报价是最优策略 |
| 社会福利最优 | 分配最大化总社会价值 |
| 计算复杂 | 需要计算n+1次社会福利 |
| 实际应用少 | Facebook曾实验，未大规模采用 |

### 2.4 GSP vs VCG 对比

| 维度 | GSP | VCG |
|-----|-----|-----|
| 激励相容 | ❌ 否 | ✅ 是 |
| 计算复杂度 | 低 O(n log n) | 高 O(n²) |
| 收益 | 可能更高 | 理论最优 |
| 稳定性 | 可能存在激烈竞价 | 稳定 |
| 实际采用 | Google、Meta主流 | 学术理想 |
| 透明度 | 易解释 | 难理解 |

## 3. pCTR × pCVR × Bid 出价公式

### 3.1 eCPM (effective Cost Per Mille) 公式

```
┌─────────────────────────────────────────────────────────────────────┐
│                      eCPM 计算公式体系                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│   ┌─────────────────────────────────────────────────────────┐       │
│   │  核心公式:                                               │       │
│   │                                                          │       │
│   │         eCPM = pCTR × pCVR × Bid × 1000                 │       │
│   │                                                          │       │
│   │  其中:                                                   │       │
│   │    • pCTR = P(click|impression)    预估点击率            │       │
│   │    • pCVR = P(conversion|click)    预估转化率            │       │
│   │    • Bid  = 广告主出价 (CPA/CPI目标成本)                 │       │
│   │                                                          │       │
│   └─────────────────────────────────────────────────────────┘       │
│                                                                      │
│   ┌─────────────────────────────────────────────────────────┐       │
│   │  展开形式:                                               │       │
│   │                                                          │       │
│   │   eCPM = P(click|impression) × P(conversion|click)       │       │
│   │          × Advertiser_Bid × 1000                        │       │
│   │                                                          │       │
│   │   = P(conversion|impression) × Advertiser_Bid × 1000    │       │
│   │                                                          │       │
│   └─────────────────────────────────────────────────────────┘       │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.2 多目标出价公式演进

#### 阶段1: CPC竞价 (早期)
```
eCPM_CPC = pCTR × CPC_Bid × 1000
```

#### 阶段2: oCPC (优化点击成本)
```
eCPM_oCPC = pCTR × Bid × Competition_Adjust × 1000
```

#### 阶段3: oCPM/oCPA (优化转化)
```
eCPM_oCPA = pCTR × pCVR × CPA_Bid × Calibration × 1000
```

#### 阶段4: 深度转化优化
```
eCPM_deep = pCTR × pCVR_shallow × pCVR_deep × Deep_Bid × 1000
```

### 3.3 实际系统中的出价公式

```python
# 伪代码：ADX出价排序计算
def calculate_ecpm(ad_candidate, user_context):
    """
    计算广告候选的eCPM得分
    """
    # 1. 预估CTR
    pctr = ctr_model.predict(
        user_features=user_context.features,
        ad_features=ad_candidate.features,
        context_features=user_context.context
    )
    
    # 2. 预估CVR (点击后转化)
    pcvr = cvr_model.predict(
        user_features=user_context.features,
        ad_features=ad_candidate.features,
        click_features=pctr.click_features
    )
    
    # 3. 获取广告主出价 (可能经过智能调整)
    bid = bid_optimizer.get_adjusted_bid(
        original_bid=ad_candidate.advertiser_bid,
        budget_status=ad_candidate.budget_status,
        pacing_rate=ad_candidate.pacing_rate
    )
    
    # 4. 校准因子 (解决预估偏差)
    calibration_factor = get_calibration_factor(
        pctr, pcvr, ad_candidate.ad_id
    )
    
    # 5. 质量度因子 (用户体验)
    quality_factor = calculate_quality_score(ad_candidate)
    
    # 6. 计算eCPM
    ecpm = pctr * pcvr * bid * calibration_factor * quality_factor * 1000
    
    return {
        'ecpm': ecpm,
        'pctr': pctr,
        'pcvr': pcvr,
        'adjusted_bid': bid,
        'quality': quality_factor
    }
```

### 3.4 出价公式中的关键挑战

| 挑战 | 问题描述 | 解决方案 |
|-----|---------|---------|
| **CTR预估校准** | 模型输出概率≠真实概率 | 保序回归、Platt Scaling |
| **CVR稀疏性** | 转化样本极少(1%→0.1%) | ES-DFM、多任务学习、延迟建模 |
| **冷启动** | 新广告无历史数据 | 跨广告迁移、元学习 |
| **位置偏差** | 高位CTR天然高 | Position-aware模型、因果推断 |
| **流量质量差异** | 不同流量转化差异大 | 流量分层预估、动态校准 |

## 4. CVR延迟反馈问题及解法

### 4.1 问题定义

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CVR延迟反馈问题                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│   时间线:                                                            │
│   ───────────────────────────────────────────────►                  │
│                                                                      │
│   Impression    Click         Conversion                            │
│       │           │                │                                 │
│       ▼           ▼                ▼                                 │
│      [展示]─────[点击]────────────[转化]                             │
│                   │                │                                 │
│                   │         ╔══════╧══════╗                         │
│                   │         ║  延迟窗口   ║  几分钟~几十天           │
│                   │         ╚═════════════╝                         │
│                   │                                                  │
│   问题：训练时部分点击的转化标签未知，导致样本标签噪声                 │
│                                                                      │
│   示例数据:                                                          │
│   ┌─────────┬────────┬──────────┬────────────┬──────────┐          │
│   │ User    │ Click  │ Label    │ Observed   │ Delay    │          │
│   ├─────────┼────────┼──────────┼────────────┼──────────┤          │
│   │ U1      │ Yes    │ Convert  │ Yes        │ 2h       │          │
│   │ U2      │ Yes    │ Convert  │ No(未观测) │ 3天      │          │
│   │ U3      │ Yes    │ No Conv  │ No         │ -        │          │
│   │ U4      │ Yes    │ Convert  │ Yes        │ 5min     │          │
│   └─────────┴────────┴──────────┴────────────┴──────────┘          │
│                                                                      │
│   U2在训练时显示为未转化，但实际上会转化 → 假阴性样本                  │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 4.2 解决方案对比

| 方案 | 核心思想 | 优点 | 缺点 |
|-----|---------|-----|------|
| **FSIW** | 样本重要性加权 | 理论扎实、可解释 | 需要延迟分布 |
| **ES-DFM** | 期望最大化去偏 | 端到端训练 | 计算复杂 |
| **延迟模型** | 显式建模延迟分布 | 物理意义清晰 | 需要假设分布 |
| **多任务** | 分解转化概率 | 可扩展性强 | 任务间关系复杂 |

### 4.3 FSIW (Feedback Shift Importance Weighting)

```
┌─────────────────────────────────────────────────────────────────────┐
│                      FSIW 原理                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│   核心思想: 通过重要性采样纠正延迟反馈带来的偏差                      │
│                                                                      │
│   定义:                                                              │
│   • P_obs(y=1|x): 观测到的转化率 (有偏)                              │
│   • P_true(y=1|x): 真实转化率 (目标)                                 │
│                                                                      │
│   关系:                                                              │
│   ╔═══════════════════════════════════════════════════════════════╗ │
│   ║  P_true(y=1|x) = P_obs(y=1|x) / P(observed|y=1,x)            ║ │
│   ╚═══════════════════════════════════════════════════════════════╝ │
│                                                                      │
│   其中 P(observed|y=1,x) 是延迟样本被观测到的概率                     │
│                                                                      │
│   ──────────────────────────────────────────────────────────────    │
│                                                                      │
│   实现步骤:                                                          │
│                                                                      │
│   1. 估计延迟分布 D(t) = P(转化发生时间 = t | 会转化)                │
│                                                                      │
│      常用分布:                                                       │
│      • 指数分布: D(t) = λ·exp(-λt)                                   │
│      • 对数正态: D(t) = lognormal(μ, σ²)                            │
│      • 非参数: 分段常数/核密度估计                                   │
│                                                                      │
│   2. 计算每个样本的权重                                              │
│                                                                      │
│      w_i = 1 / P(已观测到 | 会转化, 当前时间)                        │
│                                                                      │
│      = 1 / CDF_D(elapsed_time)  (已观测到的条件)                     │
│                                                                      │
│   3. 加权损失函数                                                    │
│                                                                      │
│      L = Σ w_i · [y_i·log(p_i) + (1-y_i)·log(1-p_i)]               │
│                                                                      │
│   4. 稳定性处理 (避免极端权重)                                        │
│                                                                      │
│      • 权重截断: w_i ← min(w_i, w_max)                              │
│      • 权重平滑: w_i ← w_i^α (0<α<1)                                │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 4.4 ES-DFM (Expectation-Maximization based Delay Feedback Model)

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ES-DFM 算法流程                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│   模型假设:                                                          │
│   ─────────────────────────────────                                 │
│                                                                      │
│   对于每个点击样本，定义两个隐变量:                                   │
│   • z ∈ {0,1}: 是否最终会转化                                        │
│   • d ∈ R+: 转化延迟时间                                             │
│                                                                      │
│   观测变量:                                                          │
│   • o ∈ {0,1}: 是否已观测到转化 (到当前时刻)                         │
│   • y ∈ {0,1}: 观测到的转化标签 (仅当o=1时有效)                      │
│                                                                      │
│   ═══════════════════════════════════════════════════════════════    │
│                                                                      │
│   概率分解:                                                          │
│                                                                      │
│   P(z=1, d | x) = P(z=1|x) · P(d|z=1,x)                             │
│                    │              │                                  │
│                    │              └── 延迟分布 (可参数化)            │
│                    └── CVR模型输出                                   │
│                                                                      │
│   P(o=1|z=1,d,t) = 𝟙[d ≤ t]  (在时间t前能否观测到)                   │
│                                                                      │
│   ═══════════════════════════════════════════════════════════════    │
│                                                                      │
│   EM算法流程:                                                        │
│                                                                      │
│   ┌─────────────────────────────────────────────────────────┐       │
│   │  E-step: 计算隐变量后验概率                              │       │
│   │                                                         │       │
│   │  对于观测到转化的样本 (o=1, y=1):                        │       │
│   │    P(z=1|x,o=1,y=1) = 1                                 │       │
│   │                                                         │       │
│   │  对于未观测到转化的样本 (o=0):                           │       │
│   │    P(z=1|x,o=0) = P(z=1|x)·P(d>t|z=1,x)                 │       │
│   │                   ─────────────────────────             │       │
│   │                   P(z=0|x) + P(z=1|x)·P(d>t|z=1,x)      │       │
│   │                                                         │       │
│   └─────────────────────────────────────────────────────────┘       │
│                                                                      │
│   ┌─────────────────────────────────────────────────────────┐       │
│   │  M-step: 最大化期望对数似然                              │       │
│   │                                                         │       │
│   │  更新CVR模型参数 θ:                                      │       │
│   │    θ_new = argmax_θ Σᵢ E[z_i|x_i] · log P(z=1|x_i;θ)   │       │
│   │                                                         │       │
│   │  更新延迟分布参数 φ:                                     │       │
│   │    φ_new = argmax_φ Σᵢ E[z_i] · log P(d_i|z=1,x_i;φ)   │       │
│   │                                                         │       │
│   └─────────────────────────────────────────────────────────┘       │
│                                                                      │
│   实际实现: 使用神经网络端到端训练，E-step和M-step交替进行            │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 4.5 延迟反馈解法代码示例

```python
# ES-DFM 简化实现示意
import torch
import torch.nn as nn

class DelayFeedbackModel(nn.Module):
    def __init__(self, feature_dim, hidden_dim=128):
        super().__init__()
        # CVR预测网络
        self.cvr_net = nn.Sequential(
            nn.Linear(feature_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, 1),
            nn.Sigmoid()
        )
        
        # 延迟分布参数 (假设对数正态分布)
        self.delay_mu = nn.Sequential(
            nn.Linear(feature_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, 1)
        )
        self.delay_sigma = nn.Sequential(
            nn.Linear(feature_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, 1),
            nn.Softplus()
        )
    
    def forward(self, x, elapsed_time, observed):
        """
        x: 特征
        elapsed_time: 距离点击的时间
        observed: 是否观测到转化
        """
        # 预测最终转化概率
        p_cvr = self.cvr_net(x).squeeze()
        
        # 延迟分布参数
        mu = self.delay_mu(x).squeeze()
        sigma = self.delay_sigma(x).squeeze()
        
        # 计算在时间t前被观测到的概率 (对数正态CDF)
        log_t = torch.log(elapsed_time + 1e-8)
        z_score = (log_t - mu) / (sigma + 1e-8)
        p_observed_given_convert = torch.sigmoid(z_score * 3)  # 近似CDF
        
        # ES-DFM损失
        # 对于已观测转化: 直接优化P(cvr)
        # 对于未观测: 考虑可能延迟的概率
        
        loss_observed = -torch.log(p_cvr + 1e-8) * observed
        
        # 未观测样本的期望
        p_not_observed_but_convert = p_cvr * (1 - p_observed_given_convert)
        p_no_convert = 1 - p_cvr
        
        loss_unobserved = -torch.log(
            p_no_convert + p_not_observed_but_convert + 1e-8
        ) * (1 - observed)
        
        return loss_observed + loss_unobserved
```

# 用户价值与广告耐受度分层
def user_ad_tolerance_segment(user):
    """
    根据用户价值和对广告的敏感度进行分层
    """
    ltv = predict_user_ltv(user)
    ad_sensitivity = estimate_ad_sensitivity(user)
    
    if ltv == 'High' and ad_sensitivity == 'Low':
        # 高价值低敏感：可增加广告密度
        return {
            'max_ads_per_session': 15,
            'quality_threshold': 0.6,
            'ecpm_weight': 0.7
        }
    elif ltv == 'High' and ad_sensitivity == 'High':
        # 高价值高敏感：保护体验，展示高质广告
        return {
            'max_ads_per_session': 5,
            'quality_threshold': 0.9,
            'ecpm_weight': 0.4
        }
    elif ltv == 'Low':
        # 低价值用户：标准策略
        return {
            'max_ads_per_session': 10,
            'quality_threshold': 0.7,
            'ecpm_weight': 0.6
        }
