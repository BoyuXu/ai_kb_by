## 一、出价调整系数（Bid Adjustments）

### 1.1 核心概念

出价调整系数（Bid Adjustments）是在基准出价之上，按维度叠加乘法系数，实现精细化出价控制。每个维度的调整系数独立设定，最终出价是所有系数的乘积。

**核心公式：**

```
final_bid = base_bid × Π(multipliers_i)
           = base_bid × m_device × m_timeslot × m_geo × m_audience × ...
```

其中：
- `base_bid`：基准出价（CPM / CPC / CPA 基础值）
- `m_i`：第 i 个维度的调整系数（通常以百分比偏移表示，如 +20% = 1.2，-30% = 0.7）
- `Π`：各维度系数连乘

### 1.2 各维度典型调整范围

#### 设备维度（Device）

| 设备类型 | 典型调整范围 | 使用场景 |
|---------|------------|---------|
| 移动端（Mobile） | -20% ~ +20% | 电商 App 推广偏好移动，B2B 偏好 PC |
| 桌面端（Desktop） | -20% ~ +30% | SaaS/企服类广告 PC 转化率更高 |
| 平板（Tablet） | -30% ~ +10% | 通常表现居中，电商稍好 |

**实战规则：**
- App 安装类广告：移动端系数 +30%~+50%，完全排除桌面
- 表单类广告（HR/SaaS）：桌面端系数 +20%~+40%
- 游戏广告：依据游戏平台而定，手游主攻移动端

#### 时段维度（Time of Day / Hour of Week）

| 时段 | 典型调整范围 | 原因 |
|-----|------------|-----|
| 早高峰（7:00-9:00） | +10% ~ +30% | 通勤时段，移动端流量激增 |
| 午休（12:00-14:00） | +20% ~ +50% | 休闲浏览，点击意愿强 |
| 晚高峰（20:00-23:00） | +30% ~ +60% | 购物决策时段，转化率最高 |
| 凌晨（0:00-6:00） | -50% ~ -80% | 流量低且质量差，大幅降价或暂停 |
| 工作日 vs 周末 | ±20% ~ ±30% | B2B 工作日 +，C2C 周末 + |

#### 地域维度（Geo）

| 地域类型 | 典型调整范围 | 逻辑 |
|---------|------------|-----|
| 一线城市（北上广深） | +20% ~ +50% | 消费能力强，LTV 高 |
| 二三线城市 | -10% ~ +10% | 竞争稍低，价格敏感 |
| 农村/偏远地区 | -30% ~ -50% | 转化率低，出价保守 |
| 境外地区 | 视产品而定 | 出海产品需单独评估 |

**地域出价的精细化实践：**

```python
# 地域出价系数示例（以全国均值=1.0为基准）
geo_multipliers = {
    "北京":  1.45,
    "上海":  1.42,
    "广州":  1.38,
    "深圳":  1.40,
    "成都":  1.15,
    "武汉":  1.10,
    "其他省会": 1.05,
    "地级市":  0.90,
    "县城":   0.70,
}
```

#### 受众维度（Audience）

| 受众类型 | 典型调整范围 | 备注 |
|---------|------------|-----|
| 已购用户（Existing Customers） | +50% ~ +200% | 复购ROI高 |
| 再营销受众（Remarketing） | +30% ~ +150% | 已有意向 |
| 相似受众（Lookalike 1%-3%） | +20% ~ +50% | 高精准 |
| 相似受众（Lookalike 5%-10%） | 0% ~ +20% | 精准度下降 |
| 兴趣受众（Interest） | -10% ~ +10% | 基准 |
| 宽泛受众 | -20% ~ -10% | 转化率低，降价保量 |

### 1.3 多维度调整的叠加方式

#### 乘法叠加（Multiplicative，推荐）

```
final_bid = base × m_device × m_timeslot × m_geo × m_audience
```

**示例：**
- base_bid = 10 元
- 移动端系数 = 1.2（+20%）
- 晚高峰系数 = 1.5（+50%）
- 北京系数 = 1.4
- 再营销受众系数 = 1.8

```
final_bid = 10 × 1.2 × 1.5 × 1.4 × 1.8 = 45.36 元
```

乘法叠加的优点：各维度相互独立，调整一个维度不影响其他维度的相对比例。

#### 加法叠加（Additive，不推荐）

```
final_bid = base × (1 + Σ adjustment_i)
           = 10 × (1 + 0.2 + 0.5 + 0.4 + 0.8) = 29 元
```

**加法叠加的问题：**
- 多个正向调整叠加后，总调整幅度可能偏小
- 无法体现维度间的乘法效应（高价值时段×高价值受众应是超线性叠加）
- Google Ads、Facebook Ads 等主流平台均采用乘法叠加

#### 实践建议

```python
def calculate_final_bid(base_bid, adjustments: dict) -> float:
    """
    adjustments: {'device': 0.2, 'timeslot': 0.5, 'geo': 0.4, 'audience': 0.8}
    每个值是百分比偏移，如 0.2 = +20%
    """
    multiplier = 1.0
    for dim, adj in adjustments.items():
        multiplier *= (1 + adj)
    return base_bid * multiplier

# 注意：需要设置出价上限，防止叠加过高
MAX_BID = base_bid * 5  # 通常不超过基础出价的5倍
final_bid = min(calculate_final_bid(base_bid, adjustments), MAX_BID)
```

## 二、流量分层出价

### 2.1 用户价值识别

流量分层的核心是将流量按预期商业价值分级，并为不同层级的流量设定差异化出价。

#### 高价值用户识别信号

**LTV（生命周期价值）预测信号：**
```
LTV_score = f(历史购买频次, 历史客单价, 最近购买时间, 品类偏好, 留存率)
```

| 信号类型 | 具体特征 | 权重说明 |
|---------|---------|---------|
| 历史行为 | 购买次数≥3、复购周期稳定 | 高权重（最直接） |
| 近期行为 | 7天内浏览/加购/收藏 | 高权重（时效性强） |
| 人口属性 | 25-35岁、一二线城市、高收入职业 | 中权重 |
| 设备信号 | iPhone新款、iPad Pro | 中权重（消费能力代理） |
| 社交信号 | 有分享/评价行为 | 低权重（口碑传播价值） |

#### 用户分层模型

```
┌─────────────────────────────────────────────┐
│              用户价值分层                     │
│                                              │
│  ┌────────────────────────────────────┐      │
│  │         高价值层（Top 10%）         │ ◄── 出价最高
│  │  LTV > 500元 / 月均消费 > 200元    │      │
│  └────────────────────────────────────┘      │
│  ┌────────────────────────────────────┐      │
│  │         中价值层（次30%）           │ ◄── 出价中等
│  │  LTV 100-500元 / 偶发消费          │      │
│  └────────────────────────────────────┘      │
│  ┌────────────────────────────────────┐      │
│  │         低价值层（后60%）           │ ◄── 出价最低或不竞价
│  │  LTV < 100元 / 纯浏览无消费         │      │
│  └────────────────────────────────────┘      │
└─────────────────────────────────────────────┘
```

### 2.2 Lookalike 受众梯度出价

Lookalike（相似受众）是以种子人群为基础，向外扩散相似用户的技术。相似度越高（百分比越小），与种子人群越接近，价值越高但量越小。

#### 梯度出价策略

```
出价倍率 = f(相似度百分比)
```

| 相似度区间 | 受众规模 | 出价系数（相对基准） | 特点 |
|-----------|---------|-----------------|-----|
| 1% | 最小 | 1.5x ~ 2.0x | 最精准，转化率最高 |
| 1%-3% | 小 | 1.3x ~ 1.5x | 较精准 |
| 3%-5% | 中 | 1.1x ~ 1.3x | 平衡精准与规模 |
| 5%-8% | 大 | 1.0x（基准） | 规模优先 |
| 8%-10% | 最大 | 0.8x ~ 1.0x | 相似度低，谨慎出价 |

**实战公式：**

```python
def lookalike_bid_multiplier(similarity_pct: float) -> float:
    """
    similarity_pct: 相似度百分比，范围 1-10
    返回出价倍率（相对于 5% 基准）
    """
    # 指数衰减函数：相似度每增加1%，倍率降低约10%
    base_pct = 5.0
    decay_rate = 0.10
    return 1.0 * (1 - decay_rate) ** (similarity_pct - base_pct)

# 示例
for pct in [1, 2, 3, 5, 7, 10]:
    print(f"Lookalike {pct}%: 出价倍率 = {lookalike_bid_multiplier(pct):.2f}x")

# 输出：
# Lookalike 1%: 出价倍率 = 1.46x
# Lookalike 2%: 出价倍率 = 1.31x
# Lookalike 3%: 出价倍率 = 1.19x
# Lookalike 5%: 出价倍率 = 1.00x
# Lookalike 7%: 出价倍率 = 0.81x
# Lookalike 10%: 出价倍率 = 0.59x
```

### 2.3 三类受众出价差异

```
核心受众 > 再营销受众 > 宽泛受众
（精准性）

再营销受众 > 核心受众 > 宽泛受众
（出价上限可以更高）
```

| 受众类型 | 定义 | 建议出价倍率 | 预算占比 |
|---------|-----|------------|---------|
| 再营销受众 | 访问/加购/未付款用户 | base × 2.0 ~ 3.0 | 30-40% |
| 核心受众 | 精准兴趣/人口属性定向 | base × 1.0 ~ 1.5 | 40-50% |
| Lookalike 受众 | 相似受众（1%-5%） | base × 1.2 ~ 1.8 | 15-20% |
| 宽泛受众 | 无精细定向 | base × 0.5 ~ 0.8 | 5-10% |

**宽泛受众的价值：** 虽然出价低，但用于触达新用户、发现新的高价值受众特征，是数据积累的重要来源。

|-----|---------|---------|-------|---------|

| 22:00-24:00 | 50-70 | 中 | 中 | 适度降价 |

### 3.2 高竞争时段策略

#### 动态提价公式

```python
def dynamic_bid_adjustment(base_bid: float, hour: int, traffic_index: float,
                            competition_index: float, expected_roi: float,
                            min_roi: float) -> float:
    """
    base_bid: 基础出价
    hour: 当前小时（0-23）
    traffic_index: 流量指数（0-100）
    competition_index: 竞争强度指数（0-100）
    expected_roi: 预期ROI
    min_roi: 最低可接受ROI
    """
    # 流量价值系数：流量越高，提价越积极
    traffic_multiplier = 1.0 + (traffic_index / 100) * 0.5
    
    # 竞争调整：竞争越激烈，需要更高出价保量
    competition_multiplier = 1.0 + (competition_index / 100) * 0.3
    
    # ROI保护：如果当前ROI已经接近下限，收紧出价
    roi_guard = min(1.0, expected_roi / min_roi) if min_roi > 0 else 1.0
    
    adjusted_bid = base_bid * traffic_multiplier * competition_multiplier * roi_guard
    
    return adjusted_bid
```

#### ROI判断：是否值得竞争高峰时段？

```
值得竞争的条件：
期望收益 = 流量×CTR×CVR×客单价×利润率 > 当前广告支出

ROI_threshold = 期望收益 / 预计广告支出 > 1.0

实际判断公式：
if (traffic_volume × estimated_CTR × estimated_CVR × AOV × margin_rate) 
   / (estimated_CPM × traffic_volume / 1000) > target_ROI:
    出价提高至竞争
else:
    维持原价或降价保守参与
```

**高峰时段通常值得竞争的品类：**
- 冲动型消费（零食/美妆/饰品）
- 时效性产品（外卖/活动票）
- 决策周期短的商品（书籍/低价数码配件）

**高峰时段不一定值得竞争的品类：**
- 高客单价/长决策周期产品（家电/汽车）
- B2B 产品（工作时段转化率高于娱乐时段）
- 预算极有限的中小广告主（被大品牌挤出）

### 3.3 低竞争时段：预算节省策略

#### 降价 vs 暂停

| 策略 | 降价 | 暂停 |
|-----|-----|-----|
| 适用场景 | 流量质量尚可，但竞争低 | 流量质量极差（凌晨0-6点） |
| 优点 | 保持曝光积累，持续获取低价流量 | 完全节省预算，集中用于高峰 |
| 缺点 | 可能积累低价值曝光，影响质量分 | 广告账号可能有学习期重置风险 |
| 建议降幅 | -30% ~ -60% | N/A |

**实践建议：**
- 凌晨 0:00-6:00：暂停（绝大多数品类）
- 工作日下午 14:00-18:00（B2C）：降价 -20%
- 周末上午（B2B）：降价 -40% 甚至暂停

### 3.4 Throttling（流量节流）

Throttling 是通过**降低竞价参与率**（而非降低出价）来控制预算消耗速度的机制。

```
正常模式：每次请求都参与竞价
Throttling模式：只有 X% 的请求参与竞价

例：预算消耗过快时，从参与100%竞价降至参与50%竞价
→ 保持出价不变，但只响应一半的广告请求
→ 平均有效CPM不变，但总消耗减半
```

#### Throttling vs 降价的区别

| 维度 | Throttling | 降价 |
|-----|-----------|-----|
| 胜出率 | 对参与的请求，胜率不变 | 胜率下降（出价低） |
| 流量质量 | 随机采样，质量分布不变 | 可能偏向低价值流量 |
| 计费方式 | 胜出时按正常价格付费 | 按降低后的出价付费 |
| 主要用途 | 预算均匀分配，防止提前耗尽 | 主动降低出价优先级 |
| 平台实现 | 平台预算分配算法内置 | 广告主手动设置 |

**Google Ads 的 Standard Delivery 本质上就是 Throttling：** 当预算不足以覆盖全天所有机会时，系统随机跳过部分竞价机会，而非降低出价。

```python
# Throttling 参与率控制示例（伪代码）
def should_participate_in_auction(
    remaining_budget: float,
    elapsed_time_ratio: float,  # 0.0 - 1.0，当天已过时间比例
    daily_budget: float
) -> bool:
    # 计划消耗率
    planned_spend_ratio = elapsed_time_ratio
    # 实际消耗率
    actual_spend_ratio = (daily_budget - remaining_budget) / daily_budget
    
    if actual_spend_ratio < planned_spend_ratio:
        # 消耗慢于计划，全力参与
        participation_rate = 1.0
    else:
        # 消耗快于计划，按比例节流
        overspend_ratio = actual_spend_ratio / planned_spend_ratio
        participation_rate = 1.0 / overspend_ratio
    
    return random.random() < participation_rate
```

## 四、出价上限 vs 目标出价

### 4.1 出价上限（Bid Cap）

出价上限是广告主设定的**每次竞价的最高出价**，平台绝不超过此值出价。

**特点：**
- 广告主完全掌控最高单价
- 系统无法自主提价抢量
- 可能因出价不足错过优质流量
- 学习期更长（系统优化空间受限）

**适用场景：**
- 预算敏感型广告主（中小企业、测试阶段）
- LTV 上限明确的品类（虚拟物品、固定利润率商品）
- 出价过高会严重亏损的高风险投放
- 有明确成本红线的 KPI 考核体系

### 4.2 目标出价（Target CPA / Target ROAS）

目标出价是让平台在**长期平均**达到目标转化成本（CPA）或目标广告支出回报率（ROAS）的前提下，自主决定每次竞价的出价。

**特点：**
- 单次出价可能大幅超过目标（但长期均值符合目标）
- 系统可以抓住高价值机会
- 需要足够的转化数据才能有效优化
- 规模优先，适合预算充足的广告主

**数据要求：**
```
Google Ads Target CPA 建议：≥ 30次/月转化
Meta Target CPA 建议：≥ 50次/周转化
否则：系统学习不足，出价不稳定
```

### 4.3 两种策略对比表

| 对比维度 | 出价上限（Bid Cap） | 目标出价（Target CPA） |
|---------|-----------------|-------------------|
| 控制方式 | 广告主设定硬上限 | 平台自主调价 |
| 单次出价 | ≤ Bid Cap | 可远超 Target CPA |
| 转化成本 | 平均低于 Bid Cap | 长期接近 Target CPA |
| 规模（流量量） | 受限，可能欠量 | 规模更大 |
| 稳定性 | 高（可预测） | 初期波动大 |
| 数据要求 | 无特殊要求 | 需要足够转化数据 |
| 学习期 | 较短 | 较长（1-2周） |
| 适合预算 | 有限预算 | 充足预算 |
| 适合阶段 | 测试期/预算敏感 | 规模增长期 |
| ROI 保障 | 强（成本可控） | 弱（短期波动大） |
| 自动化程度 | 低 | 高 |

**选择建议：**

```
新账户 / 测试期 → 先用 Bid Cap，保证不亏损
↓
积累了足够转化数据（30次/月以上）
↓
切换 Target CPA，扩大规模
↓
ROI 稳定后，进一步切换 Max Conversions / Max Conversion Value
```

## 五、再营销出价（Remarketing）

### 5.1 漏斗层级设计

再营销（Remarketing）基于用户在转化漏斗中的位置，进行精准出价。越接近付款的用户，出价越高。

```
转化漏斗（从上到下，人数递减，价值递增）：

┌─────────────────────────────────────────────────┐
│  曝光层（100,000人）                              │
│  用户看到过广告但未点击                           │
│  出价：base × 1.0（或不单独出价）                 │
├─────────────────────────────────────────────────┤
│  点击层（5,000人）                               │
│  点击过广告或访问过落地页                         │
│  出价：base × 1.5                               │
├─────────────────────────────────────────────────┤
│  加购层（1,000人）                               │
│  添加商品到购物车                                │
│  出价：base × 2.0                               │
├─────────────────────────────────────────────────┤
│  结算层（500人）                                 │
│  进入结算页但未完成付款                           │
│  出价：base × 2.5                               │
├─────────────────────────────────────────────────┤
│  付款层（300人）                                 │
│  已付款用户（用于交叉/向上销售）                   │
│  出价：base × 3.0                               │
├─────────────────────────────────────────────────┤
│  复购层（100人）                                 │
│  多次复购高忠诚用户                               │
│  出价：base × 3.5 ~ 5.0                         │
└─────────────────────────────────────────────────┘
```

### 5.2 各层级出价倍率设计

**完整出价倍率表：**

| 漏斗层级 | 用户行为定义 | 出价倍率 | 窗口期 | 目标 |
|---------|-----------|---------|-------|-----|
| 宽泛受众 | 无任何行为 | base × 0.7 | N/A | 品牌曝光 |
| 曝光未点击 | 看过广告，未点击 | base × 1.0 | 30天 | 低频提醒 |
| 点击层 | 访问落地页 | base × 1.5 | 14天 | 再吸引兴趣 |
| 加购层 | 加购/收藏 | base × 2.0 | 7天 | 推动购买决策 |
| 结算层 | 进入结算/填写地址 | base × 2.5 | 3天 | 强力召回 |
| 付款层 | 完成付款 | base × 3.0 | 90天 | 交叉销售 |
| 复购层 | ≥3次购买 | base × 4.0 | 180天 | 品牌大使 |

**计算示例（base = 10元CPM）：**

```python
remarketing_tiers = {
    "broad_audience":    {"multiplier": 0.7,  "window_days": None},
    "impression_no_click": {"multiplier": 1.0, "window_days": 30},
    "click":             {"multiplier": 1.5,  "window_days": 14},
    "add_to_cart":       {"multiplier": 2.0,  "window_days": 7},
    "checkout":          {"multiplier": 2.5,  "window_days": 3},
    "purchase":          {"multiplier": 3.0,  "window_days": 90},
    "repeat_buyer":      {"multiplier": 4.0,  "window_days": 180},
}

base_bid = 10.0

for tier, config in remarketing_tiers.items():
    bid = base_bid * config["multiplier"]
    window = config["window_days"]
    print(f"{tier}: 出价={bid:.1f}元, 窗口={window}天")
```

### 5.3 频次控制与出价联动

频次控制（Frequency Cap）和出价需要协同设计，避免高频次+高出价导致骚扰用户：

```
频次规则：
- 单日频次≤3次：正常出价
- 单日频次4-6次：出价降低30%（用户可能已经厌烦）
- 单日频次>6次：停止竞价（强制休息，防止用户屏蔽）

窗口期规则：
- 付款层用户：付款后7天内 → 出价降低50%（避免干扰新客）
- 加购层用户：超过窗口期（7天未转化） → 降低到点击层出价
- 点击层用户：超过14天未行动 → 降低到宽泛受众出价
```

**动态窗口期出价调整：**

```python
from datetime import datetime, timedelta

def get_remarketing_bid(user_behavior: dict, base_bid: float) -> float:
    """
    user_behavior: {
        'last_action': 'add_to_cart',
        'action_time': datetime对象,
        'daily_frequency': 4,  # 今天已展示次数
        'has_purchased': False
    }
    """
    tier = user_behavior['last_action']
    days_since_action = (datetime.now() - user_behavior['action_time']).days
    daily_freq = user_behavior['daily_frequency']
    
    # 基础倍率
    multiplier = remarketing_tiers[tier]['multiplier']
    window = remarketing_tiers[tier]['window_days']
    
    # 时间衰减：超过窗口期降级
    if window and days_since_action > window:
        multiplier *= 0.5  # 降级处理
    
    # 频次惩罚
    if daily_freq > 6:
        return 0  # 不参与竞价
    elif daily_freq > 3:
        multiplier *= 0.7
    
    return base_bid * multiplier
```

## 六、面试考点5条

### 考点 1：什么是出价调整系数？多维度出价调整如何叠加？

**参考答案：**

出价调整系数是在基准出价的基础上，按照设备、时段、地域、受众等维度设置乘法系数，精细化控制不同场景下的出价。

**叠加方式：** 主流平台（Google Ads、字节等）均采用**乘法叠加**：

```
final_bid = base_bid × m_device × m_timeslot × m_geo × m_audience
```

乘法叠加的优势在于各维度相互独立，高价值维度（如高峰时段+高价值受众）会产生超线性的出价提升，更符合业务直觉。加法叠加则会低估多维度高价值叠加的出价潜力。

**实践注意：** 需要设置出价上限，防止多维度正向叠加导致出价异常高（如：base=10元，各维度+50%叠加4个维度后 = 50.6元，需要人工审核合理性）。

---

### 考点 2：Lookalike 受众为什么要梯度出价？

**参考答案：**

Lookalike 受众的相似度（百分比）代表与种子人群的接近程度：1% 意味着从目标市场选取最相似的 1%，10% 则扩展到最相似的 10%。

**为什么梯度出价：**
1. **精准度递减**：1% Lookalike 的行为和价值观与种子人群高度吻合，转化率远高于 10% Lookalike
2. **成本效率**：高精准度受众的 CPM 通常更高（竞争更激烈），但转化效率足以支撑更高出价
3. **规模与精准的权衡**：1% 规模小，用更高出价保证充分曝光；10% 规模大，用较低出价控制成本

**实践策略：** 以 5% Lookalike 为基准出价，1% Lookalike 提价 40-50%，10% Lookalike 降价 20-30%，构成梯度出价体系。

---

### 考点 3：Bid Cap 和 Target CPA 如何选择？

**参考答案：**

**出价上限（Bid Cap）**：每次竞价不超过设定值，成本可预测但规模受限。适合：
- 新账户/测试阶段（缺乏足够转化数据）
- 预算敏感，成本控制严格的广告主
- 产品利润率固定，有明确成本红线

**目标出价（Target CPA）**：平台自主调价，长期均值趋近目标，短期单次出价可能超出。适合：
- 有足够转化数据（≥30次/月）的成熟账户
- 规模优先，愿意接受短期波动的广告主
- 需要系统自动抓住高价值流量机会

**选择路径**：测试期→Bid Cap，积累数据后→Target CPA，ROI稳定后→Max Conversions/Max Conversion Value。

---

### 考点 4：如何设计再营销漏斗的出价倍率？

**参考答案：**

再营销出价的核心逻辑是：**距离转化越近的用户，出价越高**，因为其转化概率更高，边际成本更低。

标准倍率设计：
- 付款层（已付款，交叉销售）：base × 3
- 结算层（进入结算未付款）：base × 2.5
- 加购层：base × 2.0
- 点击层：base × 1.5
- 曝光未点击：base × 1.0

**与频次控制联动：** 高出价的漏斗底层用户（如结算层），单日频次不超过3次，避免骚扰用户导致负面情绪，反而降低转化率。

**窗口期管理：** 结算层3天、加购层7天、点击层14天，超过窗口期自动降级出价，避免对失去兴趣的用户持续高价投放。

---

### 考点 5：什么是 Throttling？它和降价有何区别？

**参考答案：**

**Throttling（节流）** 是通过降低**竞价参与率**（不是出价）来控制预算消耗的机制。

例：日预算消耗速度过快，系统从参与100%的竞价请求降为50%，但单次出价不变。

**与降价的核心区别：**

| 维度 | Throttling | 降价 |
|-----|-----------|-----|
| 出价金额 | 不变 | 降低 |
| 对胜出率的影响 | 对参与的请求，胜率不变 | 胜率下降 |
| 流量质量分布 | 随机采样，质量不变 | 可能偏向低价值 |
| 适用场景 | 预算均匀分配 | 主动调整竞争力度 |

**Google Ads 的 Standard Delivery** 本质是 Throttling：预算不足时，系统随机跳过部分竞价，而非降低出价，以保证参与时的胜出率。

