# Project 2：多样性与 Feed 混排策略

> 项目类型：产品与算法设计 | 日期：20260324 | 领域：ads × feed

## 导言：为什么多样性很关键

**事实**：
- 用户浏览 Feed 时，如果连续看到同类广告，**点击率反而下降**
- 多样性优化能**同时提升 RPM 和用户留存**（帕累托改进）
- 大型平台（小红书、抖音）都有专门的"混排"团队优化这一块

**本项目核心**：从**技术约束**和**产品指标**两个维度理解混排

## Part A：多样性的多维度定义

### A1. 话题多样性（Topic Diversity）

```
定义：
不同话题/品类的广告在 Feed 中的分布均衡性

实现：
- 建立话题分类体系（电商、美妆、房产、汽车...）
- 每个广告分配主话题 + 副话题
- 计算 Shannon Entropy 或 Herfindahl Index

指标：
Topic_Diversity = - Σ p_i × log(p_i)  // Shannon Entropy
其中 p_i = i 话题在当前 Feed 中的占比

示例：
Feed = [电商, 电商, 美妆, 房产, 汽车]
p_电 = 2/5, p_美 = 1/5, p_房 = 1/5, p_车 = 1/5
Entropy = -[0.4×log(0.4) + 0.2×log(0.2) + 0.2×log(0.2) + 0.2×log(0.2)]
       = 1.28 bits  // 范围 0-log(n)，越高越多样

vs

均匀分布 [电商, 美妆, 房产, 汽车, 电商]
Entropy = 4 × (-0.2×log(0.2)) = 1.72 bits  // 更好
```

### A2. 来源多样性（Source/Advertiser Diversity）

```
定义：
不同广告主/品牌在 Feed 中的出现频率平衡

动机：
- 避免单一广告主"垄断" Feed
- 保证中小广告主有曝光机会

实现：
Source_Diversity = (不同广告主数) / (总广告数)
Range：[0, 1]，接近 1 表示每个广告主只出现 1 次

约束例子：
- 同一广告主最多 2-3 个广告
- Top-10 广告位不能有重复广告主
```

### A3. 创意多样性（Creative Diversity）

```
定义：
不同创意形式/风格的多样性

实现例子：
- 视频 vs 图文
- 长视频 vs 短视频
- 静态图 vs 动图
- 文案长度（短 vs 长）

约束例子：
Creative_Diversity = (视频广告) / (总广告) ≥ 0.3
```

### A4. 新颖性（Novelty / Freshness）

```
定义：
用户最近看过多少次同类内容

实现：
用户最近 K 次展示的话题分布
recent_freq = count(topic in last_K_shows) / K

novelty_score = 1 - recent_freq
范围：[0, 1]，0 表示都是重复内容，1 表示完全新内容
```

## Part B：多样性的产品权衡

### B1. 多样性 vs 相关性

```
【相关性】= 广告与用户兴趣的匹配度
【多样性】= 广告间的差异度

直觉上，两者似乎对立：
- 用户喜欢美妆广告 → 全推美妆（高相关）
- 但连续美妆广告导致用户疲劳 → 需要多样性

【解决方案】
1. 加权融合：Score = 0.6×relevance + 0.4×diversity
2. 约束优化：maximize relevance s.t. diversity ≥ threshold
3. 动态调整：根据用户浏览行为调整权重
```

### B2. 多样性 vs 收入（RPM）

```
【问题】
高多样性排序 → 展示更多"低 eCPM"的广告 → RPM 下降

【实测数据】（来自行业案例）
- 多样性提升 10% → 停留时间 +8%，再次打开 +3%
- RPM 短期下降 1-2%，但长期（1 个月）用户留存提升导致整体收入 +5%

【结论】
多样性是一种"长期投资"
- 短期：RPM 可能略降
- 长期：用户留存提升，整体收益更大
```

### B3. 广告与内容的混排比例（Ad Ratio）

```
【关键决策】：在一个 Feed 中，广告占多少比例最优？

【实测范围】
- 内容型平台（新闻、小红书）：10-30% 广告
- 视频平台（YouTube、抖音）：20-40% 广告
- 电商平台（Amazon、eBay）：30-60% 广告

【动态调整策略】
不同用户群体设不同比例：

用户粘性高 → ad_ratio = 0.4  （用户容忍度高）
用户粘性低 → ad_ratio = 0.2  （需要保护体验）
新用户 → ad_ratio = 0.1      （避免流失）

时间维度：
黄金时段（晚上）→ ad_ratio = 0.35  （用户活跃）
非黄金时段 → ad_ratio = 0.25
```

## Part C：混排系统架构

### C1. 典型混排 Pipeline

```
【精排输出】
Top-50 广告（已有初步排序）
- 广告 1: eCPM=5.0, 话题=电商, 广告主=A
- 广告 2: eCPM=4.8, 话题=美妆, 广告主=B
- ...

【混排层】
1. 多样性检查
   - 话题分布：电商 15%, 美妆 12%, 房产 10%, ...
   - 广告主分布：避免重复

2. 启发式调整
   - 交错广告与内容
   - 降低相邻广告的相似度

3. 用户个性化
   - 根据用户历史调整多样性权重
   - 新用户倾向保留低 eCPM、高多样性广告

【最终输出】
Top-20 广告，排列顺序优化

【约束检查】
- 多样性指标 ≥ 阈值
- 每个广告主最多 N 个
- RPM ≥ 目标值
- 高质量广告 ≥ 50%
```

### C2. 混排的关键决策点

```
【决策 1】：谁来决定混排规则？
- 中心化：平台制定全局规则，所有用户适用
- 分散化：不同用户群体不同规则（推荐）
- 个性化：为每个用户单独优化

【决策 2】：硬约束 vs 软约束？
示例：广告主频率限制

硬约束：
if count(advertiser_A) >= 3:
    skip this ad  // 拒绝展示

软约束：
score -= 0.2 × (count - 1)  // 每超过 1 次，分数 -0.2
```

## Part D：A/B 测试多样性

### D1. 关键指标体系

```
【排序质量指标】
- NDCG：排序与理想顺序的匹配度
- Topic Diversity Score：多样性得分
- Source Coverage：广告主覆盖率

【用户体验指标】
- CTR：点击率
- Dwell Time：停留时间
- Scroll Depth：向下滑动深度（越深越好）

【商业指标】
- RPM：千次展示收益
- CVR：转化率
- User Retention：用户留存率（日活、周活）
- User Fatigue Score：广告疲劳指数（低越好）
```

### D2. A/B 测试的陷阱

```
【陷阱 1】：短期数据误导
RPM 可能在测试初期下降，但长期上升
→ 建议运行 2+ 周，看用户留存指标

【陷阱 2】：辛普森悖论
整体用户指标下降，但某个细分用户群上升
→ 分层分析，不要只看平均值

【陷阱 3】：样本量不足
多样性改进通常是 2-5% 的提升，需要大样本
→ 计算所需样本量（统计功效分析）
```

### D3. 多目标 A/B 测试

```
【问题】：
多个目标（CTR、停留时间、留存）可能冲突
- 新排序提升了停留时间（+3%）
- 但 CTR 下降了（-2%）
- 这个排序好还是不好？

【解决方案】：
1. 定义关键指标（例如留存率最重要）
2. 设定保护指标（其他指标不能下降超过 1%）
3. 综合判断（整体是否改进）
```

## Part E：工业案例与最佳实践

### E1. 小红书的混排推测（公开信息）

```
【架构】
1. 召回层：多路召回，获得 1000+ 候选
2. 粗排：快速预估，Top-500
3. 精排：精确 CTR/CVR，Top-50
4. 混排（重排）：
   - 话题多样性约束（电商不超过 30%）
   - 广告与内容交错（1 广 + 2 内容）
   - 用户反馈融入（视频消费时长权重）

【效果】
- 日活留存：+8%（vs 无混排）
- 广告 CTR：+5%（因为多样性提升了用户体验）
- 平台 RPM：相对稳定（短期略降，长期上升）
```

### E2. YouTube 的广告多样性策略

```
【约束】
1. Ad Frequency Cap：同一广告最多 3 次/day
2. Advertiser Frequency：同一广告主最多 2 次/day
3. Category Diversity：前 10 个推荐中，广告不超过 4 个

【个性化】
不同用户群体的约束不同：
- Premium Member（付费用户）：广告比例最低
- Free User（免费用户）：广告比例正常
- New User（新用户）：广告比例最低
```
