## 一、计费模式定义与区别

### 1.1 CPM（Cost Per Mille / 每千次展示成本）

**定义：** 广告每被展示1000次所需支付的费用。

#### 适用场景
- **品牌曝光**：新产品上市、品牌知名度提升，重视曝光量而非点击
- **视频广告**：前贴片/中贴片，用户被动接受，点击率天然低
- **开屏广告**：强制展示1-3秒，适合品牌强推
- **大促预热**：提前造势，需要大量曝光但不一定立刻转化

#### 结算公式
```
广告主支付费用 = CPM × 展示次数 / 1000

示例：
  CPM = ¥20，投放100万次展示
  费用 = ¥20 × 1,000,000 / 1000 = ¥20,000
```

#### 广告主视角 vs 媒体视角

| 维度 | 广告主视角 | 媒体（Publisher）视角 |
|------|-----------|----------------------|
| 核心关注 | 每千次展示带来多少品牌价值/转化 | 填充率与CPM单价最大化 |
| 风险 | 展示后无人点击，预算浪费 | 流量质量差时CPM下跌 |
| 优化目标 | 提高ViewThroughRate（VTR） | 提高填充率（Fill Rate）|
| 定价权 | 固定CPM时广告主无议价空间 | 拍卖CPM中媒体设底价保护 |

#### CPM 与 CTR/CVR 的关联

```
eCPM（effective CPM）= CPM × 实际表现校正因子

对广告主来说：
  有效CPM = 总花费 / 总展示次数 × 1000
  ROI评估 = 品牌搜索提升率 / eCPM
```

### 1.2 CPC（Cost Per Click / 每次点击成本）

**定义：** 广告每被点击一次所需支付的费用，只有真实点击才计费。

#### 适用场景
- **效果类广告**：电商导流、应用下载，明确知道"点击≈意图"
- **搜索广告**：用户主动搜索关键词，点击质量高
- **信息流效果广告**：原生广告形式，用户可选择是否互动
- **中小广告主**：预算有限，希望"为真实兴趣付费"

#### 关键公式

```
CTR（点击率）= Clicks / Impressions × 100%

示例：
  展示 10,000 次，点击 150 次
  CTR = 150 / 10,000 = 1.5%

CPC与CPM的转换：
  CPM = CPC × CTR × 1000
  CPC = CPM / (CTR × 1000)

示例：
  CPC = ¥0.5，CTR = 2%
  等效CPM = 0.5 × 0.02 × 1000 = ¥10
```

#### 广告主可控制点

1. **出价（Bid）**：直接控制愿意为每次点击支付的最高价格
2. **关键词匹配方式**（搜索广告）：精确/词组/广泛匹配，影响流量质量
3. **广告创意**：高质量创意 → 更高CTR → 相同出价赢得更多竞拍
4. **落地页质量**：影响质量分（Quality Score），进而影响结算价
5. **受众定向**：精准人群 → 更高CTR → 更高自然胜出率
6. **出价时间调整**（Bid Adjustment）：按时段/设备/地区加价减价

#### 点击作弊（Click Fraud）问题

```
真实点击率检验方法：
  ┌─ 跳出率（Bounce Rate）< 30% 为健康信号
  ├─ 页面停留时长 > 30秒 为有效访问
  ├─ 会话深度（Pages/Session）> 1.5 说明有真实探索
  └─ 同IP重复点击检测（平台侧过滤）
```

### 1.3 CPA（Cost Per Action / 每次转化成本）

**定义：** 广告每带来一次有效转化（注册/购买/安装/表单填写等）所需支付的费用。

#### 适用场景
- **电商购买**：以"成交"为结算依据，ROI最可测量
- **游戏/App安装**：以"激活"或"付费"为转化目标
- **金融产品**：以"开户"或"首次充值"为转化
- **B2B表单**：以"询盘/预约"为转化

#### 归因窗口（Attribution Window）

归因窗口定义了"点击/展示后多少天内的转化算作该广告的贡献"：

```
常见归因窗口设置：
  ┌─ 点击后归因窗口：1天 / 7天 / 28天（Meta默认7天）
  ├─ 展示后归因窗口：1天（View-Through Conversion）
  └─ 多触点归因：Last Click / First Click / Linear / Data-Driven

示例（7天点击 + 1天展示归因）：
  Day 0：用户点击广告
  Day 5：用户购买
  → ✅ 算作该广告转化（在7天窗口内）

  Day 0：用户看到广告（未点击）
  Day 1：用户购买
  → ✅ 算作展示转化（在1天VT窗口内）

  Day 0：用户点击广告
  Day 9：用户购买
  → ❌ 不算（超出7天点击窗口）
```

#### 延迟转化问题（Delayed Conversion）

这是CPA广告最棘手的工程问题：

```
问题：
  用户在 T 时刻点击广告 → 竞拍发生在 T
  用户在 T+3天 才购买   → 转化信号回传在 T+3天+处理延迟

影响：
  1. 模型训练样本标签延迟（Label Delay）
     - 正样本比例被低估
     - 模型低估转化率，保守出价，流量损失

  2. Exploration-Exploitation 失衡
     - 新广告在24小时内看起来"效果差"，系统减少投放
     - 但实际上转化还没来得及回传

解决方案（工业界）：
  方案1 - Elapsed Time Delay 特征
    将"距点击的时间间隔"加入模型特征，校正延迟偏差

  方案2 - Importance Sampling
    对延迟样本进行重采样，恢复真实分布

  方案3 - 实时更新窗口
    以24h为快速标签，7天为最终标签，双阶段训练

  方案4 - 保守预估 + 后置校正
    先以已观测转化训练，定期用延迟到达的转化做模型修正
```

### 1.4 oCPM / oCPC / oCPA（优化型计费）

**核心区别：** 传统计费是"你告诉平台你的出价，平台帮你买量"；优化型计费是"你告诉平台你的目标，平台用AI替你优化出价买量"。

#### oCPM（Optimized CPM）

```
原理：
  广告主设置目标（如"更多点击"/"更多转化"）
  平台内部预测每次展示的预期价值
  自动调整实际CPM出价，让每次展示ROI最优

出价流程：
  广告主设置：目标CPA = ¥50，日预算 = ¥1000
  平台预测：某次展示的预期CVR = 2%，pCTR = 5%
  自动出价：等效CPM = 50 × 2% × 5% × 1000 = ¥50（太高，实际会压缩）
         → 系统用多次数据拟合校正系数K
  实际出价：CPM = K × 目标CPA × pCTR × pCVR × 1000
```

#### oCPC（Optimized CPC）

```
字节跳动oCPC两阶段模型：

阶段一（探索期）：
  - 按传统CPC计费，系统收集点击数据
  - 积累足够样本（一般100-200次转化）

阶段二（优化期）：
  - 系统建立CTR→CVR预测模型
  - 按预期转化出价，转化成本向目标CPA靠拢
  - 实际结算仍按CPC，但展示层按转化概率排序

与传统CPC的区别：
  传统CPC：谁出价高谁展示，结果转化率差异大
  oCPC：出价×预测CVR综合排序，转化率高的广告获更多流量
```

#### oCPA（Optimized CPA）

```
直接以转化为优化目标，平台全权负责出价：

广告主输入：
  - 目标CPA = ¥30（每次转化最多花¥30）
  - 转化定义：购买/注册/激活
  - 日预算上限

平台负责：
  - 预测每次展示的 pCTR × pCVR
  - 自动计算合理出价
  - 在保证转化成本不超标的前提下最大化转化量

平台"保量"机制：
  问题：广告主设了oCPA，但平台赢拍量太少怎么办？
  
  保量逻辑：
  ┌─ 控制成本线：在实际CPA ≤ 目标CPA时，尽力赢取竞拍
  ├─ 探索期豁免：新广告前期允许轻微超成本，用于学习
  ├─ 平滑控制：不是"超了就不出"而是动态调节出价
  └─ 最终结算保护：部分平台按实际CPA超出目标的差额补偿广告主
```

#### 三者对比

```
┌──────────┬──────────────┬──────────────┬─────────────────┐
│  模式    │  广告主控制  │  平台优化目标 │  适用阶段       │
├──────────┼──────────────┼──────────────┼─────────────────┤
│ 传统CPC  │ 关键词+出价  │ 最高出价赢拍 │ 品牌/关键词投放 │
│ oCPC     │ 目标CPA      │ 预期转化排序 │ 积累数据后优化  │
│ oCPM     │ 目标+预算    │ 展示层ROI    │ 品牌+效果兼顾   │
│ oCPA     │ 目标CPA      │ 转化量最大化 │ 纯效果投放      │
└──────────┴──────────────┴──────────────┴─────────────────┘
```

## 二、平台结算逻辑

### 2.1 二价拍卖下的实际CPC计算

二价拍卖（Second-Price Auction / Vickrey Auction）是搜索广告的基础定价机制：

```
规则：出价最高者获胜，但只需支付第二名出价 + 最小竞价增量（通常¥0.01）

示例：
  广告A出价 ¥2.00，质量分 8
  广告B出价 ¥1.50，质量分 9
  广告C出价 ¥1.20，质量分 7

  若纯按出价排序：A > B > C
  实际CPC（A）= ¥1.50 + ¥0.01 = ¥1.51

  引入质量分后（Ad Rank = Bid × QS）：
  Ad Rank A = 2.00 × 8 = 16
  Ad Rank B = 1.50 × 9 = 13.5
  Ad Rank C = 1.20 × 7 = 8.4

  排序不变（A > B > C）
  但实际CPC（A）= 第二名Ad Rank / 自身QS + ¥0.01
               = 13.5 / 8 + ¥0.01 = ¥1.69 + ¥0.01 = ¥1.70

  结论：质量分高的广告商，实际CPC可以比出价低得多！
```

### 2.2 eCPM 统一排序公式

不同计费模式（CPM/CPC/CPA）的广告需要统一比较，eCPM 是通用货币：

```
基础公式：
  eCPM = 广告主出价 × 预测点击率（pCTR） × 1000

完整公式（考虑转化）：
  eCPM = CPA出价 × pCTR × pCVR × 1000
       = CPC出价 × pCTR × 1000
       = CPM出价（直接）

示例（三种计费模式同台竞争）：
  广告1（CPM模式）：CPM = ¥10 → eCPM = ¥10
  广告2（CPC模式）：CPC = ¥1, pCTR = 1.5% → eCPM = 1 × 0.015 × 1000 = ¥15
  广告3（CPA模式）：CPA = ¥100, pCTR = 2%, pCVR = 0.5% → eCPM = 100 × 0.02 × 0.005 × 1000 = ¥10

  排序：广告2（¥15）> 广告1 = 广告3（¥10）
  广告2胜出展示
```

### 2.3 广告主出价 vs 平台内部 eCPM 的映射关系

```
广告主出价（User-Facing Bid）
         ↓
    [出价校正层]
    × 质量分系数
    × 账户历史表现系数
    × 竞争强度系数
         ↓
    内部等效出价（Internal Bid）
         ↓
    × 预测点击率（pCTR）          ← 由CTR预估模型输出
    × 预测转化率（pCVR）          ← 由CVR预估模型输出（若CPA模式）
         ↓
    eCPM（参与排序的最终分数）
         ↓
    [二价结算] → 实际CPC / CPM

关键：广告主看到的出价 ≠ 平台内部排序的分数
      平台通过这层转换，同时激励"好的创意"和"合理出价"
```

### 2.4 质量分（Quality Score）对结算价的影响

Google Ads 质量分（1-10分）由三部分组成：

```
Quality Score = f(预期CTR, 广告相关性, 落地页体验)

每个维度评级：
  - 高于平均水平（Above Average）
  - 平均水平（Average）
  - 低于平均水平（Below Average）

质量分对结算价的影响（以Google为例）：

  广告排名（Ad Rank）= 出价 × QS × 广告扩展预期影响

  实际CPC = 下一名Ad Rank / 自身QS + ¥0.01

  QS影响示例：
  ┌──────┬──────┬──────────┬─────────────────┬────────────┐
  │ 广告  │ 出价 │  QS分    │  Ad Rank        │ 实际CPC    │
  ├──────┼──────┼──────────┼─────────────────┼────────────┤
  │  A   │ ¥2.0 │  10      │  20             │ 最高位     │
  │  B   │ ¥4.0 │   4      │  16             │ 第二位     │
  │  C   │ ¥3.0 │   3      │   9             │ 第三位     │
  └──────┴──────┴──────────┴─────────────────┴────────────┘

  广告A（QS=10）以¥2出价赢过广告B（QS=4，出价¥4）！
  广告A实际CPC = B的Ad Rank / A的QS = 16/10 + 0.01 = ¥1.61
  
  结论：QS提升1分，实际CPC可降低10-20%
```

#### 提升质量分的工程方法

```
提升预期CTR：
  - 广告标题含关键词
  - 使用广告扩展（链接附加、电话附加等）
  - 历史表现好的账户加权

提升广告相关性：
  - 关键词-广告-落地页三者主题一致
  - 拆分细化广告组（每组10个以内关键词）

提升落地页体验：
  - 页面加载速度（< 3秒，移动端尤关键）
  - 落地页内容与广告承诺匹配
  - 清晰的转化路径（CTA按钮、表单）
  - 页面移动端适配
```

## 三、不同平台的出价体系对比

### 3.1 Google Ads

Google Ads 是全球最成熟的搜索广告出价体系，历经多年演进：

#### 手动出价 → 智能出价演进路径

```
Manual CPC
    ↓ 2013年
Enhanced CPC（ECPC）
    ↓ 2016年
Target CPA
    ↓ 2018年
Target ROAS / Maximize Conversions
    ↓ 2021年
Performance Max（全自动化，跨渠道）
```

#### Enhanced CPC（增强型CPC）

```
原理：在手动CPC基础上，系统自动±30%调节出价

条件：预测该次竞拍转化概率高于平均 → 自动加价
     预测该次竞拍转化概率低于平均 → 自动减价

适用：已有转化历史、希望保留出价控制权的广告主
限制：加价上限30%，保守，不适合转化信号稀疏的账户
```

#### 目标CPA（Target CPA）

```
工作原理：
  广告主设定：目标每次转化费用 = ¥50
  Google系统：
    ① 预测每次竞拍的转化概率
    ② 自动出价 = 目标CPA × 预测CVR
       如预测CVR=2%，则出价 = ¥50 × 2% = ¥1.00（CPC等效）
    ③ 跨竞拍平均CPA向¥50靠拢

注意：
  - 需要30天内至少50次转化才能启用
  - 前两周为"学习期"，CPA可能波动
  - 可设置出价上/下限（可选）
```

#### 目标ROAS（Target ROAS）

```
适用：电商场景，不同商品价值差异大

公式：
  ROAS = 广告收入 / 广告花费
  目标ROAS = 期望的广告回报率（如 500% = 每花¥1期望回收¥5）

系统出价：
  出价 = (预测转化价值 / 目标ROAS) × pCTR
  
示例：
  用户A：预测购买价值¥200，目标ROAS=400%，pCTR=3%
  出价 = (200 / 4) × 0.03 = ¥1.50
  
  用户B：预测购买价值¥50，目标ROAS=400%，pCTR=3%
  出价 = (50 / 4) × 0.03 = ¥0.375（远低于用户A）
```

### 3.2 Facebook / Meta

#### Meta 的三层出价体系

```
竞拍目标层（Campaign Objective）：
  品牌知名度 → 优化曝光和Reach
  流量       → 优化点击
  互动       → 优化点赞/评论
  应用安装   → 优化Install
  转化       → 优化Purchase/Lead

出价策略层（Bid Strategy）：
  最低费用（Lowest Cost）：在预算内最大化转化量，系统全权负责出价
  费用上限（Cost Cap）：控制平均CPA不超出设定值
  出价上限（Bid Cap）：硬性限制每次竞拍最高出价
  目标费用（Target Cost）：让CPA稳定在目标值附近（已于2023年下线）

计费层：
  CPM：按展示计费（品牌类）
  CPC：按点击计费（流量类）
  CPA：按转化计费（效果类）
```

#### Meta 拍卖公式

```
广告总价值 = 出价 × 预估行动率 + 用户价值（相关性/质量）

其中：
  预估行动率（Estimated Action Rate）：用户完成目标行动的概率
  用户价值：广告对用户的正负体验影响（负反馈会扣分）

这意味着：
  低相关性广告即使出价高，也可能输给高相关性低出价广告
  广告被隐藏/举报会显著降低"用户价值"分数
```

### 3.3 字节跳动（抖音 / 穿山甲）

#### 巨量引擎出价体系

```
┌─────────────────────────────────────────────────┐
│                巨量引擎出价体系                   │
├─────────────┬─────────────┬─────────────────────┤
│   CPM模式   │   CPC模式   │   oCPM/oCPC/oCPA    │
│  品牌展示   │  落地页点击  │   效果转化优化       │
└─────────────┴─────────────┴─────────────────────┘

oCPM 两种出价方式：
  1. 按展示付费（CPM）+ 目标优化（转化/点击/互动）
  2. 按转化付费（仅限白名单资质广告主）

oCPC 特色：
  - 广告主设目标CPC（如¥0.5/点击）
  - 系统优化"转化质量好的点击"，而非单纯最低CPC
```

## 四、面试考点

### 面试考点

---

**考点1：CPM / CPC / CPA / oCPM 的区别，以及什么场景用哪种？**

```
标准答题框架：
  CPM：品牌曝光，结算公式 = 花费/展示×1000，媒体收益稳定
  CPC：效果导流，结算按点击，广告主控点击意图
  CPA：深度效果，按转化付费，归因窗口和延迟转化是核心难点
  oCPM：平台托管出价，内部按eCPM排序，广告主设目标不管出价细节

场景选择：
  新品上市/品牌认知 → CPM
  搜索/导流 → CPC
  电商转化/应用安装 → CPA/oCPA
  预算有限希望AI托管 → oCPM/oCPC
```

---

**考点2：二价拍卖中为什么"诚实出价"是占优策略？（需要简单证明）**

```
证明（反证法）：
  假设真实价值为 v，竞争对手最高出价为 b2

  情形A：诚实出价 b = v
    若 v > b2 → 赢拍，获得 v - b2 > 0 的净收益 ✅
    若 v < b2 → 落败，净收益 = 0 ✅

  情形B：虚报高价 b > v（b > b2）
    若 b2 < v < b → 结果同A，无改变
    若 v < b2 < b → 赢拍但净收益 = v - b2 < 0 ❌（亏损！）

  情形C：虚报低价 b < v（b < b2）
    若 b < b2 < v → 落败，净收益=0；但诚实出价本可赢拍获正收益 ❌

  结论：任何偏离真实出价的策略，要么结果不变，要么更差
        → 诚实出价是弱占优策略（Weakly Dominant Strategy）
```

---

**考点3：eCPM 公式推导，以及不同计费模式如何统一排序？**

```
推导过程：
  平台视角：每次展示的期望收入
  
  CPM广告：期望收入 = CPM / 1000
  CPC广告：期望收入 = CPC × pCTR
  CPA广告：期望收入 = CPA × pCTR × pCVR

  转换为"每千次展示期望收入"（即eCPM）：
  eCPM(CPM) = CPM
  eCPM(CPC) = CPC × pCTR × 1000
  eCPM(CPA) = CPA × pCTR × pCVR × 1000

  统一排序：取 eCPM 最大者优先展示
```

---

**考点4：质量分（Quality Score）如何影响实际CPC？计算举例。**

```
公式：
  Ad Rank = 出价 × 质量分
  实际CPC = 下一位Ad Rank / 自身质量分 + ¥0.01

举例：
  广告A：出价¥3，QS=4，Ad Rank=12
  广告B：出价¥2，QS=8，Ad Rank=16
  广告C：出价¥1，QS=6，Ad Rank=6

  排序：B(16) > A(12) > C(6)

  广告B实际CPC = A的Ad Rank / B的QS + 0.01 = 12/8 + 0.01 = ¥1.51
  广告A实际CPC = C的Ad Rank / A的QS + 0.01 = 6/4 + 0.01 = ¥1.51

  广告B出价¥2，实际只花¥1.51；
  广告A出价¥3，实际也只花¥1.51，但质量分低所以排在B后面
```

---

**考点5：延迟转化（Delayed Conversion）对CTR/CVR预估模型有什么影响？如何解决？**

```
影响分析：
  训练数据构建：点击后N小时内有转化=正样本，否则=负样本
  延迟转化导致：在截止时间前正样本被低估（有些转化还没来）
  模型效果：CVR预估偏低 → 出价保守 → 流量损失 → 广告主ROI低估

解决方案：
  1. 延长等待窗口（Wait-then-Predict）：
     等待足够长时间收集完整标签，延迟上线但准确率高
  
  2. 时间折扣（Elapsed Time Features）：
     用"距点击的时间"作为特征，让模型学习延迟分布
  
  3. 重要性采样（Importance Sampling）：
     对不同延迟时间的样本重新加权
  
  4. 双阶段训练：
     即时标签（1h）训练快速响应模型
     延迟标签（7d）训练精准模型
     两个模型融合使用
  
  5. 阿里DIN/DIEN方案：
     将历史行为序列建模，用行为时间序列预测转化时机
```
