## 二、各平台深度解析

### 2.1 Google Ads：从二价到统一价，Smart Bidding 的进化

#### 拍卖机制演进：GSP → 统一价

Google Ads 的拍卖机制经历了一次重大转变：

**广义第二价格（GSP，Generalized Second-Price）时代（2000s-2019）：**
```
广告位1 赢家：付费 = 广告位2出价 / 广告位1质量分 + $0.01
广告位2 赢家：付费 = 广告位3出价 / 广告位2质量分 + $0.01
...
```
GSP 的核心：赢者付次高价，理论上促使广告主诚实出价。

**统一价（Unified First Price / Uniform Clearing Price）时代（2019+）：**
```
所有竞价者以同一清算价格成交（第二价格制的变体）
清算价格 = 使所有胜出广告主都以相同 eCPM 结算的价格
```

**为什么转换？** 随着程序化购买（Programmatic）的普及，多平台叠加的 GSP 被中间商利用（Bid Shading 技术），Google 为了简化系统并提高收益透明度，向统一价转型。

#### Smart Bidding：40+ 信号的机器学习出价

Smart Bidding 是 Google 的全自动出价系统，在每次竞价时实时考虑40+信号：

```
信号体系（部分）：
┌─────────────────────────────────────────────────────────┐
│  用户信号                  设备/时间信号                  │
│  ├─ 搜索意图（Query）      ├─ 设备类型                    │
│  ├─ 搜索历史              ├─ 位置/地域                    │
│  ├─ Gmail行为（隐匿）      ├─ 时段/日历                   │
│  ├─ Chrome Browse         ├─ 操作系统                    │
│  └─ Google账户属性         └─ 屏幕分辨率                  │
│                                                          │
│  广告主信号                竞价环境信号                   │
│  ├─ 再营销受众             ├─ 竞争对手出价水平             │
│  ├─ 转化历史              ├─ 展示位置（搜索位/内容网络）   │
│  ├─ 落地页质量            └─ 查询竞争强度                 │
│  └─ 广告相关性                                           │
└─────────────────────────────────────────────────────────┘
```

**Smart Bidding 产品演进：**

```
Enhanced CPC（eCPC，2010）
  └─ 手动出价 + 系统小幅自动调整（±30%）
  
Target CPA（2013）
  └─ 自动出价，长期均值趋近目标CPA
  
Target ROAS（2016）
  └─ 自动出价，以广告支出回报率为目标
  
Maximize Conversions（2017）
  └─ 在预算内最大化转化次数（不设成本约束）
  
Maximize Conversion Value（2018）
  └─ 在预算内最大化转化价值（GMV）
  
Performance Max（2021）
  └─ 跨所有Google渠道（搜索/购物/YouTube/展示/Discovery）
  └─ 全自动创意、定向、出价一体化
```

#### Privacy Sandbox 对归因的影响

2021年 Apple ATT（App Tracking Transparency）冲击 + 2024年浏览器 Cookie 逐步废弃，Google 推出 Privacy Sandbox 作为替代：

- **Topics API**：浏览器在本地计算用户的兴趣标签（~350个分类），直接提供给广告主，不暴露原始URL
- **FLEDGE（Protected Audience API）**：再营销不经过第三方服务器，竞价在用户浏览器本地完成（"设备端竞价"）
- **Attribution Reporting API**：转化归因在设备端完成，只向广告主发送聚合报告，不发送用户级数据

**影响：** 广告主的归因精度下降，Click-through Conversion 窗口缩短，需要更依赖 Google 的 Data-Driven Attribution（DDA）模型，减少自建归因系统的可能性。

### 2.2 字节跳动（巨量引擎）：oCPM 生态与短视频广告的特殊性

#### oCPM 生态统一框架

字节跳动的广告系统（抖音/今日头条/穿山甲联盟）以 **oCPM（optimized CPM）** 为核心计费模式，统一了之前碎片化的 CPC/CPA 计费。

```
oCPM 本质：
广告主设定目标转化成本（oCPX目标），平台自动优化出价
竞价层面：以 CPM 为单位竞价
计费层面：按 CPM 结算
优化目标：在 CPM 预算内，最大化满足广告主的转化目标

核心公式（平台侧）：
ecpm = pctr × pcvr × 目标oCPX × 1000

其中：
- pctr：预估点击率
- pcvr：预估转化率（下载/注册/付款等）
- 目标oCPX：广告主设定的目标转化成本
```

#### 短视频广告的特殊性

短视频广告引入了独特的优化目标，使出价维度更加丰富：

```
视频特有指标：
├─ 完播率（Video Completion Rate）
│   └─ 6秒完播 / 15秒完播 / 全部完播（VCR）
├─ 互动指标
│   ├─ 点赞（Like）
│   ├─ 评论（Comment）
│   ├─ 分享（Share）
│   └─ 关注（Follow）
└─ 深度行为
    ├─ 进入主页（Profile Visit）
    ├─ 加购（Add to Cart）：直播/橱窗场景
    └─ 直播间停留时长（Live Dwell Time）
```

广告主可以按**互动成本（CPE, Cost Per Engagement）** 或 **完播成本**出价，而不仅仅是转化成本。这是抖音区别于传统广告平台的核心差异。

#### 粗排→精排→竞价三阶段

```
请求处理流程（单次广告填充）：

[海量广告库：数百万活跃广告]
    │
    ▼
[粗排（Rough Ranking）]
    - 规则过滤（定向不符 / 预算耗尽 / 时段不对）
    - 轻量模型打分（DNN浅层版本，速度优先）
    - 候选集缩减：百万 → 数百个
    │
    ▼
[精排（Fine Ranking）]
    - 完整深度模型（DIEN / BERT-based CTR预估）
    - 多目标打分（CTR × CVR × VCR × 互动率）
    - 候选集缩减：数百 → 数十个
    │
    ▼
[竞价（Auction）]
    - eCPM = pCTR × pCVR × oCPX_target × 1000
    - 加入竞价调整系数
    - 二价竞价（广义第二价格变体）
    - 胜出者 → 展示
```

#### oCPX 自动出价

oCPX 是字节跳动的智能出价产品，广告主只需设定目标成本，系统自动决定每次竞价的出价：

- **优化目标**：点击/关注/表单/付款/ROAS等
- **出价保护**：短期单次出价可能偏高，但7天均值趋近目标
- **冷启动**：新广告需要至少50次转化才能退出学习期
- **超成本保护**：字节承诺7天内超成本部分平台补偿（"成本保障"机制）

### 2.3 腾讯广告（微信生态）：社交关系链与跨端统一出价

#### 社交关系链的定向优势

腾讯广告的核心差异化优势在于**社交关系链数据**，这是其他平台无法复制的：

```
关系链定向能力：
├─ 相似用户（类似 Lookalike，但基于社交关系而非行为）
│   └─ "与已转化用户有相同朋友"的用户
├─ 口碑传播定向
│   └─ 已有朋友关注某公众号 / 购买某商品的用户
└─ 关系链强度
    └─ 强关系（频繁互动朋友） vs 弱关系（普通联系人）

社交证明广告形式：
"你的好友 XXX 也在使用 [产品名]" — 朋友圈广告原生社交标注
效果：点击率可提升 20-40%（相比无社交标注）
```

#### 朋友圈广告的原生竞价规则

朋友圈广告是腾讯广告的核心资源，其竞价规则略有特殊性：

```
朋友圈广告竞价公式：
ecpm = 出价 × 质量度 × 匹配度 × 预估CTR × 场景系数

场景系数（朋友圈 vs 其他）：
- 朋友圈 FEED：场景系数最高（竞争最激烈，CPM最高）
- 公众号文中：场景系数中等
- 小程序内：场景系数中等
- 联盟（优量汇）：场景系数较低

广告形式溢价：
- 视频广告：比图片广告 CPM 高 30-50%（互动率更高）
- 卡片式（带链接跳转）：比纯展示高 20%
```

#### 公众号/小程序/视频号跨端统一出价

腾讯广告推出**全域投放（All-in-One）** 策略，将朋友圈、公众号、小程序、视频号、QQ等流量统一在一个广告活动中出价：

```
全域出价收益：
┌─────────────────────────────────────────────────┐
│  传统分渠道出价              全域统一出价          │
│  朋友圈：CPM=45元           系统自动分配预算到     │
│  公众号：CPM=28元    →      转化率最高的渠道        │
│  小程序：CPM=35元           综合CPM降低15-25%      │
│  视频号：CPM=30元           转化成本优化10-20%     │
└─────────────────────────────────────────────────┘
```

腾讯广告的**oCPA**（Optimized Cost Per Action）是类似字节oCPX的智能出价产品：

- 广告主设定目标行为（关注/注册/付款/小程序激活）
- 系统自动在多个流量源间动态分配预算和出价
- 关键约束：**微信数据不出域**——腾讯的社交数据不提供给第三方DMP，只能在腾讯广告平台内使用
- 对广告主影响：无法自建用户级再营销，只能依赖腾讯的黑盒优化

### 2.4 阿里妈妈（淘宝/天猫）：搜索竞价与GMV深度绑定

#### 搜索广告的关键词竞价特殊性

阿里妈妈的核心产品**直通车**是电商搜索广告，其竞价逻辑与 Google 搜索广告有相似性，但更深度整合了电商数据：

```
直通车排名公式：
广告得分 = 出价 × 质量分

质量分 = f(预估CTR, 关键词相关度, 创意质量, 商品转化历史, 店铺信誉)

关键差异（相比Google搜索）：
1. 质量分更看重"商品转化率"而非"落地页质量"
2. 同一关键词下，有历史销量的商品质量分天然更高（马太效应）
3. 支持"智能匹配"：系统自动扩展到语义相近关键词
```

**实际结算：**
```
CPC结算（仍是点击付费）：
实际扣费 = 下一名出价 × 下一名质量分 / 自身质量分 + 0.01元（最低扣费单位）

这与Google GSP类似：排名靠前，出价高，但质量分也高 → 实际扣费可能低于对手
```

#### 直通车 / 引力魔方 / 品牌广告的差异

| 产品 | 定位 | 出价方式 | 适用场景 |
|-----|-----|---------|---------|
| **直通车** | 精准搜索广告 | 关键词级CPC出价 | 有明确购买意图的用户 |
| **引力魔方** | 展示信息流广告 | 人群定向CPM/CPC | 品类教育/新品推广 |
| **品牌直播** | 品牌曝光 | CPT（时间买断）/ CPM | 品牌认知建设 |
| **万相台** | 全域智能投放 | 智能出价（oCPX） | 多渠道统一优化 |

#### 与GMV深度绑定的ROAS优化

阿里妈妈的核心差异：**广告系统与电商交易系统深度打通**，可以实现真正的 ROAS 优化而无需额外归因。

```
淘系 ROAS 优化的独特优势：
┌─────────────────────────────────────────────────────────┐
│  广告点击 → 加购 → 下单 → 付款 全程在淘系内完成          │
│                                                          │
│  无需第三方归因 SDK：                                     │
│  ├─ 无 ATT/iOS 隐私问题（App内全闭环）                   │
│  ├─ 无跨域归因争议（全在淘系ID体系内）                    │
│  └─ 可以精确计算每笔订单对应的广告花费                    │
│                                                          │
│  智能推广（oCPX）支持：                                   │
│  ├─ 以 ROI 为优化目标（目标ROAS）                        │
│  ├─ 支持全店ROI（而非单品ROI）                           │
│  └─ 支持"投产比"实时看板，与 GMV 挂钩                    │
└─────────────────────────────────────────────────────────┘
```

**阿里妈妈的局限性：**
- 流量高度集中在购物场景，难以覆盖购前心智培育
- 数据壁垒高，外部广告（Meta/字节）转化数据无法接入淘系归因
- 关键词竞价透明度高但竞争激烈，头部类目 CPC 虚高

## 三、关键差异深度分析

### 3.1 竞价机制差异：为什么字节/腾讯坚持 oCPM 而不学 Google 切换统一价？

**背景：** Google 在 2019 年将搜索广告从 GSP 切换到统一价（Uniform Clearing Price）。但字节、腾讯的主流广告产品仍以 oCPM + 二价为核心。

**原因一：流量质量差异**

```
Google 搜索广告：
- 用户有明确搜索意图
- 同一关键词下，竞争者数量相对有限
- 广告主对同一 Query 的价值判断相对接近
→ GSP 在此场景下的"赢者诅咒"问题小，转向统一价以提升透明度

字节/腾讯信息流广告：
- 用户无明确意图（被动曝光）
- 受众定向颗粒度细，每次竞价的受众组合可能独一无二
- 不同广告主对同一用户的价值判断差异巨大
→ oCPM 让平台统一管理定价，广告主只需设目标，降低定价复杂度
```

**原因二：oCPM 对平台更有利**

```
oCPM 的平台视角：
├─ 广告主只设目标CPA，实际出价由平台决定
├─ 平台在出价上有更大自由度（单次可能超出广告主目标）
├─ 平台可以整体优化收益，而非只看单次拍卖
└─ 广告主体验更简单，降低使用门槛（吸引中小广告主）

统一价对广告主更透明：
└─ 广告主可以精确知道为何胜出/失败，更适合精细化运营
→ 这与字节/腾讯"平台托管"的商业模式不符
```

**结论：** oCPM 本质上是平台代替广告主做出价决策，符合字节/腾讯"智能投放托管"的产品策略；而 Google 在搜索场景更强调透明度和广告主自主控制权。

---

### 3.2 归因机制差异：归因窗口对广告主 ROI 计算的影响

各平台的归因窗口设置直接影响广告主看到的转化数据量，进而影响出价决策：

| 平台 | 点击归因窗口 | 曝光归因窗口 | 多触点归因 | 跨平台归因 |
|-----|------------|------------|---------|---------|
| Google Ads | 30天（默认）/ 最长90天 | 1天（展示广告） | DDA数据驱动 | GA4跨渠道 |
| 字节跳动 | 7天（默认）/ 最长30天 | 1天 | 末次点击为主 | Oasis端侧增强 |
| 腾讯广告 | 7天（默认） | 1天 | 末次点击 | 腾讯生态内 |
| 阿里妈妈 | 15天（默认）/ 最长30天 | 1天 | 线性/末次 | 仅淘系内 |

**归因窗口对ROI的影响：**

```
示例：一个用户 Day1 点击广告，Day10 购买

Google（30天窗口）：✅ 归因到该广告
字节（7天窗口）：  ✅ 归因到该广告  
腾讯（7天窗口）：  ✅ 归因到该广告

同样用户 Day1 点击广告，Day20 购买：

Google（30天窗口）：✅ 归因到该广告（影响 Smart Bidding 优化）
字节（7天窗口）：  ❌ 不归因（广告主看到的CPA虚高）
腾讯（7天窗口）：  ❌ 不归因

结论：窗口越短，广告主看到的CPA越高（但实际效果可能一样）
→ 广告主在字节/腾讯可能因CPA虚高而降低出价，错失规模
→ Google 长窗口让 Smart Bidding 看到更多转化，优化更准确
```

**多触点归因的差异：**

```
用户路径：Google搜索广告(Day1) → 字节信息流(Day3) → 直接访问(Day5) → 购买

末次点击归因：购买 100% 归因给最后一次接触（直接访问）
首次点击归因：购买 100% 归因给 Google 搜索广告
线性归因：三次接触各归因 33%
数据驱动归因（DDA）：基于机器学习评估每个接触点的增量贡献

问题：每个平台只看自己的数据 → 各平台都声称自己贡献了这次转化
→ "重复计算问题"：广告主的归因总转化数 > 实际转化数（2-3倍很常见）
```

### 3.3 隐私政策下的应对差异：iOS ATT 冲击对各平台的不同影响

2021年 Apple ATT（App Tracking Transparency）要求 iOS App 必须弹窗征得用户同意才能获取 IDFA：

```
iOS ATT 冲击链路：
用户拒绝追踪 → IDFA 不可用 → 跨App行为追踪失效 → 转化归因失效 → 出价优化准确度下降
```

**各平台受冲击程度：**

| 平台 | iOS流量占比（估算） | ATT同意率 | 受冲击程度 | 应对方案 |
|-----|----------------|---------|---------|---------|
| Google | 中（Android为主） | ~30% | 中 | Privacy Sandbox（浏览器端解决方案） |
| 字节（抖音） | 高（iOS用户付费能力强） | ~25% | 高 | Oasis端侧归因框架 |
| 腾讯（微信） | 高 | ~25% | 中高 | 微信内全闭环，减少对IDFA依赖 |
| 阿里妈妈 | 中（淘宝App） | ~25% | 低（最轻） | 淘系内闭环，购买行为完全可追踪 |

**各平台应对方案详解：**

```
字节跳动 — Oasis（绿洲）端侧归因：
├─ 转化归因在用户设备本地完成
├─ 只上报聚合级别的转化报告，不上报用户ID
├─ 结合 SKAdNetwork（Apple官方API）和私有方案双轨并行
└─ 效果：iOS转化归因覆盖率从 <30% 提升至 >60%

腾讯广告 — 微信闭环优势：
├─ 小程序广告：点击 → 小程序 → 购买全程在微信内
├─ 不依赖 IDFA，使用微信OpenID
└─ 朋友圈广告仍受影响（点击跳转外链场景）

阿里妈妈 — 最小影响：
├─ 直通车点击 → 淘宝App（同一App内，无跨App追踪问题）
├─ 内嵌H5广告影响较大，但非主流
└─ 淘宝自有用户ID体系（淘宝账号）不受 IDFA 影响

Google — 最复杂应对：
├─ Android：Google拥有 GAID，受影响有限
├─ iOS：依赖 SKAdNetwork + Consent Mode
└─ 浏览器：Privacy Sandbox（2024-2025年落地）
```

**对广告主的实际影响：**
```
ATT之前：iOS广告ROI通常高于Android（用户付费能力强）
ATT之后：iOS广告ROI数据下降30-50%（归因不完整，并非真实效果下降）
          广告主感知到iOS ROI下降 → 降低iOS出价 → iOS CPM降低
          → 实际转化率未变，但出价降低，广告主实际 ROI 提升
          → 归因"消失"但效果仍在（未被追踪到的转化）
```

## 四、面试考点5条

### 考点1：字节跳动的出价机制和 Google Ads 的核心区别是什么？

**参考答案：**

**核心区别：**

1. **拍卖目标不同**：
   - Google（搜索）：用户有明确意图，广告与查询高度相关，竞价基于关键词
   - 字节（信息流/短视频）：用户无明确意图，系统主动预测用户兴趣，竞价基于人群

2. **出价机制不同**：
   - Google：广告主自主设置出价或Smart Bidding，平台以关键词/受众为出价单元
   - 字节：oCPM为主，广告主只设目标CPA，平台全权决定每次竞价出价（出价黑盒化）

3. **优化目标不同**：
   - Google：点击/转化/转化价值
   - 字节：新增了视频完播率/互动率/直播间停留等短视频特有指标

4. **数据优势不同**：
   - Google：搜索意图数据（最高价值）+ Chrome行为 + Gmail
   - 字节：内容消费偏好 + 兴趣图谱（通过短视频行为构建）

5. **归因窗口不同**：
   - Google支持最长90天归因，字节默认7天，影响广告主对长决策周期商品的ROI评估

---

### 考点2：阿里妈妈的关键词竞价和 Google 搜索广告有什么异同？

**参考答案：**

**相同点：**
- 都是用户搜索 → 广告竞价 → 展示 → 付费（CPC）的基本模式
- 都使用"质量分×出价"决定排名，赢者按次高价付费（类GSP）
- 都有关键词级别的精细化出价

**不同点：**

| 维度 | Google 搜索 | 阿里妈妈直通车 |
|-----|------------|------------|
| 用户场景 | 开放网络搜索（信息/购物混合） | 纯购物场景，强购买意图 |
| 质量分构成 | 落地页质量 + CTR + 相关性 | 商品转化率 + 点击率 + 店铺信誉 |
| 数据优势 | 跨域行为（Gmail/YouTube/Maps） | 深度购物行为（加购/收藏/购买历史） |
| 竞价结果透明度 | 高（提供质量分估算工具） | 中（质量分算法不公开） |
| 跨渠道整合 | Performance Max（Google全渠道） | 万相台（淘系全渠道） |
| 归因 | 跨网站归因（GA4） | 纯淘系内归因（最精准） |

关键差异：阿里妈妈的搜索广告与GMV深度绑定，质量分天然偏向有历史销量的商品，新品冷启动更难，但对成熟店铺的ROAS优化精度最高。

---

### 考点3：腾讯广告的社交关系链数据为何无法被复制？

**参考答案：**

腾讯的社交关系链优势来自微信/QQ的**强关系社交网络**，这在几个维度上是不可复制的：

1. **数据规模**：微信MAU超13亿，强关系（频繁互动好友）数据覆盖几乎全中国互联网用户

2. **数据性质**：强关系（真实朋友/同事/家人）vs 字节/小红书的弱关系（兴趣关注）。强关系的口碑传播效应更强，"你的朋友在用"这类社交证明对消费决策的影响远高于陌生人推荐

3. **数据不出域**：腾讯明确策略是微信数据不对外开放，广告主只能在腾讯广告平台使用这些数据，无法导出到其他平台或自建DMP

4. **护城河效应**：即使竞争对手有类似的社交产品（如微博/陌陌），其关系强度和使用频率都远低于微信，且用户不会轻易从微信迁移

5. **商业化节奏**：腾讯长期保守商业化（避免破坏微信体验），导致广告库存受限但单位价格高，是有意为之的策略

### 考点4：iOS ATT 对各平台影响为何有差异？哪个平台受冲击最小？

**参考答案：**

**受冲击最小：阿里妈妈**

原因：淘宝/天猫 App 内广告点击→购买的全程都在同一个 App 内完成，不涉及跨 App 追踪。iOS ATT 针对的是"跨 App/网站追踪"，单 App 内行为不受影响。淘系的 IDFA 只用于少量外部投放（如 DSP 跨平台展示广告），主流的搜索广告和信息流广告完全不受影响。

**受冲击最大：字节跳动（抖音）**

原因：
1. iOS 用户占抖音 DAU 约 40%，且 iOS 用户 ARPU 更高（付费能力强）
2. 抖音广告主的转化大量发生在外部 App（电商 App/注册表单），需要跨 App 追踪
3. ATT 后 iOS 转化归因覆盖率从 ~80% 骤降至 ~25%，广告主看到 CPA 暴涨
4. 字节被迫快速研发 Oasis 端侧归因框架补救

**腾讯广告：中等影响**

朋友圈广告中"点击跳出到外部网站/App"场景受冲击，但小程序广告（微信内闭环）不受影响。微信的 OpenID 体系部分缓解了 IDFA 失效的问题。

### 考点5：什么是 oCPM？它解决了传统 CPM/CPC 出价的什么问题？

**参考答案：**

**传统出价的问题：**

```
传统 CPM 出价：
广告主按每千次展示付费，但不保证这千次展示带来任何转化
→ 广告主承担流量质量风险，不适合效果类广告

传统 CPC 出价：
广告主按点击付费，但点击不等于转化
→ 点击刷量风险，且广告主仍需自己判断哪些流量值得高价
```

**oCPM 的解决方案：**

```
oCPM（optimized CPM）：
- 广告主设定目标行为（如：下载/注册/付款）和目标成本（oCPX目标）
- 平台将目标转化成本转换为 CPM 出价：
  竞价出价 = pCTR × pCVR × oCPX目标 × 1000
- 竞价仍以 CPM 为单位
- 结算仍以 CPM 为单位
- 但优化目标是"在 CPM 预算内最大化广告主的转化目标"

oCPM 的本质：广告主告诉平台"我愿意为一次转化付多少钱"
              平台负责找到最有可能转化的用户，并在合理的 CPM 内买到这些用户
              广告主无需关心中间的 CTR/CVR/CPM，只看最终 CPA 是否达标

优势：
1. 广告主体验简单（只设一个目标，其他交给平台）
2. 平台可以整体优化，跨广告主分配流量效率更高
3. 降低了广告主的出价专业门槛，吸引中小广告主

劣势：
1. 出价黑盒，广告主不知道为什么某次竞价赢了/输了
2. 冷启动需要足够转化数据（<50次/周效果不佳）
3. 平台有更大定价权，广告主对成本的掌控度降低
```

1. 新品冷启动（数据不足）：

   └─ 各平台数据仅用于同平台内的相对对比，不做跨平台绝对值对比
