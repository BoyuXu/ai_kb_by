## 第 1 部分：通用理论与数学基础

### 1.1 CTR/CVR 的定义与本质

#### CTR (Click-Through Rate) 定义
$$P(\text{click}|x) = \sigma(f(x; \theta))$$

- **广告场景**：用户看到广告后点击的概率
- **推荐场景**：用户看到推荐内容后点击的概率  
- **搜索场景**：用户看到搜索结果后点击的概率

#### CVR (Conversion Rate) 定义
$$P(\text{conversion}|click, x) = \sigma(g(x; \phi))$$

- **广告场景**：点击后转化（购买、咨询）的概率
- **推荐场景**：点击后转化（收藏、购买、关注）的概率
- **终极目标**：pCVR 乘以 pCTR 得到 pCTCVR，最小化 CPA（单个转化成本）

### 1.2 核心概念：校准性 vs 排序性

CTR 模型的两个独立指标：

#### 排序性（Ranking Ability）
- **评估指标**：AUC、Logloss
- **含义**：模型能否正确排序物品的点击概率
- **用途**：推荐、搜索的排序（只需相对顺序正确）
- **例子**：pCTR 预估为 [0.6, 0.3, 0.1] 和 [0.8, 0.5, 0.3] 排序顺序相同，都可用于排序

#### 校准性（Calibration）
- **评估指标**：ECE (Expected Calibration Error)、在线转化率
- **含义**：预估概率与实际发生率一致（pCTR=5% 的广告实际转化 5%）
- **用途**：广告出价、竞价策略、CPA 控制
- **例子**：如果出价 pCVR=5% 但实际 pCVR=10%，竞价会偏低，损失广告收入

**关键洞察**：
- 推荐系统：AUC 高就够了（排序能力强）
- 广告系统：必须校准（出价依赖绝对概率值）

### 1.3 多任务学习框架

在推荐和广告中，常需同时预估 CTR、CVR、转化率、时长等多个目标。

#### 多目标函数
$$L = \sum_{k=1}^{K} w_k \cdot L_k(y_k, \hat{y}_k) + \lambda R(\theta)$$

其中：
- $K$ = 任务数（如 3：CTR、CVR、收藏率）
- $w_k$ = 任务权重（动态调整平衡）
- $L_k$ = 第 k 个任务的损失函数（通常是交叉熵）
- $R(\theta)$ = 正则化项

#### 任务权重平衡的三种方法

**方法 1：固定权重**
$$w_{CTR} : w_{CVR} : w_{FAV} = 1 : 0.1 : 0.05$$
- 优点：简单、可控
- 缺点：不适应数据分布变化

**方法 2：不确定性加权 (Uncertainty Weighting)**
$$L = \sum_k \frac{1}{\sigma_k^2} L_k + \lambda \sum_k \log(\sigma_k)$$
- 自动调整权重，降低不确定性的任务权重更高
- Google 推荐

**方法 3：梯度归一化 (GradNorm)**
$$w_k^{new} = w_k \cdot \frac{\text{mean}(\nabla_k)}{\text{max}(\nabla_k)}$$
- 基于梯度幅度动态调整，保持各任务学习速率相近

## 第 3 部分：核心算法深潜

### 3.1 MMOE：多门控混合专家（多任务学习标配）

#### 解决问题
多任务学习中的**任务冲突**：
- CTR 任务偏好标题党，CVR 任务偏好质量内容
- 共享底层导致梯度冲突，一个任务提升另一个下降

#### 核心机制
```
输入特征
  ↓
Expert 网络群组 (N 个独立网络)
  ↓ 每个任务有独立的 Gating Network
  ↓ 加权融合 Expert 输出
  ↓
Task-specific Tower (CTR/CVR/FAV)
  ↓
输出预测
```

#### 关键公式

Expert 输出：
$$f_i(x) = \text{Expert}_i(x)$$

Gating Network（任务 k）：
$$g_k(x) = \text{Softmax}(W_{g_k} \cdot x)$$

融合表示：
$$h_k = \sum_{i=1}^{N} g_{ki}(x) \cdot f_i(x)$$

#### 工程落地
- Expert 数：4-8 个通常足够
- 适用：CTR + CVR、CTR + 收藏 + 评论
- 限制：任务相关性强时，MMOE 可能不如 Shared-Bottom

### 3.2 PLE：渐进分离多任务框架

#### 解决问题
MMOE 中所有任务共享所有 Expert。当任务差异大时：
- 一个任务的优化仍会损害另一个任务
- 跷跷板现象（A 提升导致 B 下降）

#### 核心创新：显式分离
```
Layer 1:
  Task A 私有 Expert + Shared Expert → Gating 选择
  Task B 私有 Expert + Shared Expert → Gating 选择
  
Layer 2:
  (逐层堆叠，渐进提取)
  
特点：
  - Task A 的私有 Expert 对 Task B 不可见
  - Shared Expert 学习通用知识
  - 多层逐层融合
```

#### 适用场景
- 任务高度不相关（如点击 + 评论 + 分享）
- 需要避免任务间干扰的场景

### 3.3 DIN：兴趣注意力机制（推荐和广告都在用）

#### 解决问题
不同用户对候选物品的兴趣程度不同，传统方法用平均兴趣表示不够精准。

#### 核心创新：加权平均
$$a_i = \frac{\exp(q_i \cdot v_i^T)}{\sum_j \exp(q_j \cdot v_j^T)}$$

$$\text{output} = \sum_i a_i \cdot v_i$$

其中：
- $q$ = 候选物品表示
- $v_i$ = 历史物品表示
- $a_i$ = 注意力权重

#### 工程落地
- 历史序列截断：通常 50-200
- 计算复杂度：O(序列长度 × 向量维度)

## 第 4 部分：领域对比（广告 vs 推荐 vs 搜索）

### 4.1 数据特性差异

| 维度 | 广告 | 推荐 | 搜索 |
|------|------|------|------|
| 正样本率 | ~0.1-0.5% | ~0.5-3% | ~5-15% |
| 样本构建 | 竞价日志 | 曝光日志 | 查询日志 |
| 特征 | 广告主、预算、出价 | 用户个性化 | Query 语义 |
| 更新频率 | 小时/天级 | 周级 | 实时 |

### 4.2 模型差异

#### 广告 CTR 特性
- **校准要求高**：出价依赖 pCVR 的绝对值
- **必须做 Platt Scaling 或 Isotonic Regression**
- **特征**：有广告主特征、预算特征
- **延迟转化处理**：关键，因为广告有 7-30 天的转化窗口

#### 推荐 CTR 特性
- **排序性要求高**：只需相对顺序正确
- **校准可以宽松**：主要用于排序
- **特征**：用户 Profile、物品内容、历史交互
- **多任务学习**：CVR、收藏、转发等

#### 搜索 CTR 特性
- **Query 理解**：Query 特征权重高
- **文档理解**：Title、Snippet、Anchor Text
- **排序多样性**：需要避免结果重复

### 4.3 线上模型架构对比

**广告系统**
```
特征 → Embedding → MMOE (CTR + CVR) → 精排
  → 出价模块 → 竞价
```

**推荐系统**
```
用户特征 → Embedding → DIN/Transformer → CTR + CVR + FAV
  → 混排与重排 → 个性化排序
```

**搜索系统**
```
Query 理解 → Document Embedding → 多头排序 (相关性 + 新鲜度 + 多样性)
  → Learning To Rank → 最终排序
```

## 第 5 部分：校准与在线学习

### 5.1 概率校准：为什么必要？

**问题示例**：
- 模型预估 pCVR = 3%，实际 CVR = 6%（过度自信）
- 出价算式：bid = target_CPA × pCVR
- 结果：实际出价 = target_CPA × 3%，亏损 50%

### 5.2 校准方法

#### Platt Scaling（简单、快速）
$$p_{calibrated} = \sigma(a \cdot \text{logit}(p) + b)$$

- 参数：只需学习 a 和 b
- 用途：全局校准，简单有效

#### Isotonic Regression（非参数、灵活）
$$p_{calibrated} = \text{IsotonicFit}(p)$$

- 使用分段常数函数拟合
- 更灵活，但需要更多数据
- 工业推荐：Platt Scaling for production

### 5.3 在线学习：FTRL 算法

在线学习对**快速变化的数据**（广告、天气相关内容）很关键。

#### FTRL 核心思想
每个新样本到达时立即更新参数：
$$w_{t+1} = \arg\min_w (\sum_{i=1}^t g_i \cdot w + \frac{1}{2}\sum_{i=1}^t \sigma_i \|w\|^2 + \lambda_1 \|w\|_1)$$

#### 与 SGD 对比
- FTRL：L1 正则化产生**稀疏解**，自动特征选择
- SGD：需要显式特征选择，否则维度爆炸
- 适用：十亿级特征维度场景（广告）

## 第 6 部分：最新趋势与案例

### 6.1 LLM 增强 CTR 预估（2024+）

#### 应用方向
1. **特征生成**：LLM 从文本/标题生成语义特征
2. **蒸馏**：用 LLM 生成弱标签，辅助训练小模型
3. **冷启动**：LLM 补充新物品/新用户的特征

#### 实战案例
```
广告创意理解：
  输入：创意标题、描述
  LLM：理解主题、情感、吸引力
  输出：语义向量，用于 CTR 预估
```

### 6.2 多目标排序（Pareto 前沿）

实际系统中不是单一优化 CTR，而是多目标：
$$\text{score} = \alpha \cdot \text{CTR} + \beta \cdot \text{转化率} + \gamma \cdot \text{用户满意度}$$

关键：**参数化 Pareto 前沿**，不同用户偏好不同权重。

### 6.3 因果推断处理偏差

#### 问题
- CTR 模型训练中：点击和不点击的样本分布不同
- 位置偏差：排名靠前的位置点击率自然高
- 展示偏差：系统倾向展示已有高点击的内容

#### 解决方向
- **IPW (Inverse Propensity Weighting)**：加权修正
- **因果森林**：估计因果效应
- **自适应重采样**：调整正负样本比例

| 阶段 | 核心算法 | 关键特性 | 适用场景 |
