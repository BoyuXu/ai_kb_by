## 1. Transformer 各组件参数量拆解

### 1.1 Embedding 层

Embedding 层负责把离散 token 映射到连续向量空间，是模型的"入口"。

#### Token Embedding

```
参数量 = V × d
```

- `V`：词表大小（vocabulary size）
- `d`：隐藏维度（hidden dimension）

示例：LLaMA-7B，V=32000，d=4096 → 32000 × 4096 = 131,072,000 ≈ 131M

#### Position Embedding（绝对位置编码）

```
参数量 = T × d
```

- `T`：最大序列长度（max sequence length）
- 仅 GPT-2 等使用可学习绝对位置编码的模型有此项

示例：GPT-2，T=1024，d=768 → 1024 × 768 = 786,432 ≈ 0.8M

#### RoPE / ALiBi（旋转/相对位置编码）

```
参数量 = 0
```

RoPE（Rotary Position Embedding）通过旋转矩阵注入位置信息，不引入任何可学习参数。LLaMA、Qwen、Mistral 系列均使用 RoPE。ALiBi 通过注意力偏置实现，同样无额外参数。

#### Segment Embedding（BERT 专属）

```
参数量 = 2 × d
```

BERT 的 segment embedding 区分句子 A/B，仅有 2 个 segment，因此是 2×d。在 decoder-only 的 GPT/LLaMA 系列中不存在。

#### Embedding 层小结

| 模型类型 | Embedding 参数量 |
|---------|----------------|
| GPT-2（绝对 PE） | V×d + T×d |
| LLaMA（RoPE，tied head） | V×d |
| BERT（绝对 PE + segment） | V×d + T×d + 2×d |

### 1.2 Multi-Head Attention（MHA）

注意力层是 Transformer 的核心，参数集中在 Q/K/V 投影矩阵和输出投影矩阵。

#### 标准 MHA（Multi-Head Attention）

每个注意力头的维度：`d_head = d / h`，其中 `h` 是头数。

Q/K/V 各自有一个 `d × d` 的投影矩阵（等价于 h 个 `d × d_head` 拼接）：

```
Q 投影: W_Q ∈ R^{d × d}，bias b_Q ∈ R^d
K 投影: W_K ∈ R^{d × d}，bias b_K ∈ R^d
V 投影: W_V ∈ R^{d × d}，bias b_V ∈ R^d
```

输出投影：

```
O 投影: W_O ∈ R^{d × d}，bias b_O ∈ R^d
```

有 bias 时每层注意力参数量：

```
P_attn = 3 × (d × d + d) + (d × d + d)
       = 3d² + 3d + d² + d
       = 4d² + 4d
```

无 bias 时（LLaMA、Mistral 等现代模型）：

```
P_attn = 4d²
```

示例：LLaMA-7B，d=4096，无 bias → 4 × 4096² = 67,108,864 ≈ 67.1M / 层

#### MQA（Multi-Query Attention）

MQA 的核心思想：所有头共享一组 K 和 V，只有 Q 保持多头。

```
Q 投影: d × d            （多头，不变）
K 投影: d × d_head        （单头）
V 投影: d × d_head        （单头）
O 投影: d × d            （不变）
```

参数量（无 bias）：

```
P_MQA = d² + d × d_head + d × d_head + d²
      = 2d² + 2d × d_head
      = 2d² + 2d²/h
```

与 MHA 对比（无 bias）：

```
MHA: 4d²
MQA: 2d² + 2d²/h = 2d²(1 + 1/h)
```

当 h=32 时，MQA 节省约 `(4d² - 2d² - 2d²/32) / 4d² ≈ 48.4%` 的注意力参数。

#### GQA（Grouped-Query Attention）

GQA 是 MHA 和 MQA 的折中：将 h 个查询头分为 g 个组，每组共享一组 K/V。

- g=h → 退化为 MHA
- g=1 → 退化为 MQA

```
Q 投影: d × d                 （不变）
K 投影: d × (g × d_head)      （g 组，每组一个 K 头）
V 投影: d × (g × d_head)      （g 组，每组一个 V 头）
O 投影: d × d                 （不变）
```

参数量（无 bias）：

```
P_GQA = d² + 2 × g × d_head × d + d²
      = 2d² + 2g × d_head × d
      = 2d² + 2gd²/h
```

示例：LLaMA-3-8B，d=4096，h=32，g=8，d_head=128

```
P_GQA = 2 × 4096² + 2 × 8 × 128 × 4096
      = 33,554,432 + 8,388,608
      = 41,943,040 ≈ 41.9M / 层
```

对比 MHA 的 4d²=67.1M，GQA 节省约 37.5% 注意力参数。

#### 注意力变体参数量对比

以 d=4096, h=32, d_head=128 为例（无 bias）：

| 变体 | 公式 | 参数量 | 相对 MHA |
|-----|------|-------|---------|
| MHA (g=h=32) | 4d² | 67.1M | 100% |
| GQA (g=8) | 2d²+2gd²/h | 41.9M | 62.5% |
| GQA (g=4) | 2d²+2gd²/h | 37.7M | 56.3% |
| GQA (g=2) | 2d²+2gd²/h | 35.7M | 53.1% |
| MQA (g=1) | 2d²+2d²/h | 34.6M | 51.6% |

### 1.3 FFN（Feed-Forward Network）

#### 标准 FFN

标准 Transformer FFN 由两个线性层组成：

```
FFN(x) = W_2 · ReLU(W_1 · x + b_1) + b_2
```

```
W_1 ∈ R^{d × d_ff}，b_1 ∈ R^{d_ff}
W_2 ∈ R^{d_ff × d}，b_2 ∈ R^{d}
```

有 bias 参数量：

```
P_FFN = d × d_ff + d_ff + d_ff × d + d
      = 2 × d × d_ff + d_ff + d
```

典型设置 d_ff = 4d（GPT-2/3, BERT）：

```
P_FFN = 2 × d × 4d + 4d + d = 8d² + 5d
```

无 bias（GPT-3 后续模型）：

```
P_FFN = 2 × d × d_ff = 2d × d_ff
```

d_ff = 4d 时：`P_FFN = 8d²`

#### SwiGLU FFN（LLaMA 系列）

SwiGLU 引入门控机制，使用三个线性层：

```
SwiGLU(x) = (W_1 · x ⊙ SiLU(W_gate · x)) · W_2
```

```
W_1    ∈ R^{d × d_ff}    （up projection）
W_gate ∈ R^{d × d_ff}    （gate projection）
W_2    ∈ R^{d_ff × d}    （down projection）
```

参数量（无 bias）：

```
P_SwiGLU = 3 × d × d_ff
```

为了与标准 FFN（8d²）保持参数量对等，SwiGLU 的 d_ff 通常设为：

```
3 × d × d_ff = 8d²
→ d_ff = 8d/3 ≈ 2.67d
```

实践中 d_ff 会进一步调整为 256 的整数倍（便于硬件对齐），例如：
- LLaMA-7B: d=4096, 理论 d_ff = 10923, 实际 d_ff = 11008
- LLaMA-3-8B: d=4096, d_ff = 14336（更大，约 3.5d）

示例：LLaMA-7B

```
P_SwiGLU = 3 × 4096 × 11008 = 135,266,304 ≈ 135.3M / 层
```

#### MoE FFN（Mixture of Experts）

MoE 用多个专家 FFN 替换单个 FFN，加一个路由网络：

```
路由器: W_router ∈ R^{d × E}
专家: E 个独立 FFN，每个参数量 P_expert
```

总参数量：

```
P_MoE = E × P_expert + d × E
```

注意：MoE 的"总参数量"很大，但每次前向传播只激活 top-k 个专家，实际计算量 ≈ k/E × 全参数计算量。

示例：Mixtral 8x7B，每层 8 个专家，每个专家是 SwiGLU FFN

```
P_expert = 3 × 4096 × 14336 = 176,160,768
P_router = 4096 × 8 = 32,768
P_MoE_layer = 8 × 176,160,768 + 32,768 ≈ 1.409B / 层
```

但每次只激活 2 个专家，等效计算量约 2/8 = 25% 的总参数。

#### FFN 参数量对比

以 d=4096 为例：

| FFN 变体 | d_ff | 公式 | 参数量 |
|---------|------|------|-------|
| 标准（无 bias） | 4d=16384 | 2d×d_ff | 134.2M |
| SwiGLU | 8d/3≈11008 | 3d×d_ff | 135.3M |
| SwiGLU（LLaMA-3） | 14336 | 3d×d_ff | 176.2M |
| MoE 8 experts（SwiGLU） | 14336 | 8×3d×d_ff | 1,409M |

### 1.4 LayerNorm / RMSNorm

#### LayerNorm

```
LayerNorm(x) = γ ⊙ (x - μ) / √(σ² + ε) + β
```

每层 LayerNorm 有两组参数：
- γ（scale）∈ R^d
- β（shift）∈ R^d

```
P_LN = 2d / 个
```

标准 Transformer 每个 block 有 2 个 LayerNorm（attention 后 + FFN 后），总计：

```
P_LN_per_block = 2 × 2d = 4d
```

#### RMSNorm

```
RMSNorm(x) = γ ⊙ x / RMS(x)
```

RMSNorm 省去了均值中心化和 β 偏置，只有 γ 参数：

```
P_RMS = d / 个
```

LLaMA 系列每个 block 有 2 个 RMSNorm（attention 前 + FFN 前），最终输出还有 1 个：

```
P_RMS_per_block = 2d
P_RMS_final = d
总 RMSNorm 参数 = L × 2d + d = (2L+1) × d
```

#### 对比

| 归一化方式 | 参数/个 | 每 block | 常见模型 |
|-----------|---------|---------|---------|
| LayerNorm | 2d | 4d | GPT-2, BERT |
| RMSNorm | d | 2d | LLaMA, Qwen, Mistral |

注意：pre-norm（先归一化再 attention/FFN）和 post-norm（先 attention/FFN 再归一化）的参数量相同，区别在于计算图。现代模型几乎都用 pre-norm。

### 1.5 LM Head（语言模型输出头）

LM Head 将最后一层的隐藏状态映射回词表空间，得到 logits：

```
logits = W_head · h + b_head
```

```
W_head ∈ R^{d × V}
```

#### Tied（权重共享）

大多数现代模型将 LM Head 权重与 Token Embedding 共享（weight tying）：

```
W_head = W_embed^T
额外参数 = 0
```

使用 weight tying 的模型：GPT-2、LLaMA-1/2/3、Qwen-2、Mistral 等。

#### Untied（独立权重）

```
额外参数 = V × d（无 bias）
额外参数 = V × d + V（有 bias）
```

GPT-3 使用独立 LM Head。某些大词表模型（V=128256 如 LLaMA-3）虽然 tied，但 embedding 本身已很大。

## 2. 总参数量公式

### 2.1 通用公式

```
P_total = P_embedding + L × P_block + P_norm_final + P_head
```

其中每个 Transformer block：

```
P_block = P_attention + P_FFN + P_norms
```

### 2.2 GPT-2 风格（标准 Transformer，有 bias，tied head）

```
P_embedding = V×d + T×d          （token + position embedding）
P_attention = 4d² + 4d           （每层，有 bias）
P_FFN = 8d² + 5d                 （每层，d_ff=4d，有 bias）
P_norms = 4d                     （每层，2 个 LayerNorm）
P_block = 12d² + 13d             （每层总计）
P_norm_final = 2d                （最终 LayerNorm）
P_head = 0                       （tied）
```

总参数量：

```
P = V×d + T×d + L×(12d² + 13d) + 2d
```

### 2.3 LLaMA 风格（无 bias，SwiGLU，RMSNorm，RoPE，tied head）

```
P_embedding = V×d                （token embedding，RoPE 无参数）
P_attention = 4d²                （MHA，无 bias）— 或 GQA: 2d² + 2gd²/h
P_FFN = 3×d×d_ff                 （SwiGLU，无 bias）
P_norms = 2d                     （每层 2 个 RMSNorm）
P_block = 4d² + 3d×d_ff + 2d    （MHA）或 2d²+2gd²/h + 3d×d_ff + 2d（GQA）
P_norm_final = d                 （最终 RMSNorm）
P_head = 0                       （tied）
```

总参数量（MHA 版本，d_ff ≈ 8d/3 → 3d×d_ff ≈ 8d²）：

```
P ≈ V×d + L×(4d² + 8d² + 2d) + d
  = V×d + L×(12d² + 2d) + d
  ≈ V×d + 12Ld²    （忽略低阶项）
```

### 2.4 快速估算法则

对于标准架构，参数量主要由 `12Ld²` 决定：

```
P ≈ 12Ld²
```

这个公式在 d >> 1 且 V×d 相对于 12Ld² 较小时非常准确。

直觉理解：
- 每层有 4d²（attention）+ 8d²（FFN）= 12d² 的参数
- FFN 贡献了 2/3 的参数，attention 贡献 1/3
- L 层叠加

## 3. 真实模型验证（手算 vs 官方）

### 3.1 GPT-2 (124M)

已知配置：
- V = 50257, d = 768, L = 12, h = 12, d_ff = 3072
- 最大序列长度 T = 1024
- 有 bias，使用 LayerNorm，tied LM Head
- 绝对位置编码（可学习）

逐项计算：

```
Token Embedding:    V × d = 50257 × 768 = 38,597,376
Position Embedding: T × d = 1024 × 768  = 786,432
```

每层 Attention（有 bias）：
```
Q/K/V: 3 × (768 × 768 + 768) = 3 × (589,824 + 768) = 3 × 590,592 = 1,771,776
O:     768 × 768 + 768 = 590,592
Attention 合计 = 1,771,776 + 590,592 = 2,362,368
```

每层 FFN（有 bias）：
```
W_1: 768 × 3072 + 3072 = 2,359,296 + 3,072 = 2,362,368
W_2: 3072 × 768 + 768 = 2,359,296 + 768 = 2,360,064
FFN 合计 = 2,362,368 + 2,360,064 = 4,722,432
```

每层 LayerNorm（2 个）：
```
2 × 2 × 768 = 3,072
```

每层总计：
```
P_block = 2,362,368 + 4,722,432 + 3,072 = 7,087,872
```

12 层合计：
```
12 × 7,087,872 = 85,054,464
```

最终 LayerNorm：
```
2 × 768 = 1,536
```

LM Head（tied，0 额外参数）。

总参数量：
```
P = 38,597,376 + 786,432 + 85,054,464 + 1,536
  = 124,439,808
```

官方：~124M ✅ 完美匹配

### 3.3 LLaMA-7B

已知配置：
- V = 32000, d = 4096, L = 32, h = 32, d_head = 128
- d_ff = 11008（SwiGLU）
- 无 bias，RMSNorm，RoPE，tied LM Head
- 标准 MHA（LLaMA-1-7B 不用 GQA）

```
Token Embedding: 32000 × 4096 = 131,072,000
```

```
Q/K/V + O: 4 × 4096² = 4 × 16,777,216 = 67,108,864
```

每层 FFN（SwiGLU，无 bias）：
```
W_gate + W_up + W_down: 3 × 4096 × 11008 = 3 × 45,088,768 = 135,266,304
```

每层 RMSNorm（2 个）：
```
2 × 4096 = 8,192
```

每层总计：
```
P_block = 67,108,864 + 135,266,304 + 8,192 = 202,383,360
```

32 层合计：
```
32 × 202,383,360 = 6,476,267,520
```

最终 RMSNorm：
```
4096
```

LM Head（tied，0）。

总参数量：
```
P = 131,072,000 + 6,476,267,520 + 4,096
  = 6,607,343,616 ≈ 6.61B
```

官方：6.74B。差异分析：Meta 公布的 6.74B 包含了部分实现细节差异（如 embedding 可能 untied 计算方式、padding token 等）。多数开源复现报告约 6.6-6.7B。核心公式验证通过 ✅

```
P = 6,607,343,616 + 131,072,000 = 6,738,415,616 ≈ 6.74B ✅
```

这说明 LLaMA-7B 官方参数量是按 untied head 计算的（虽然训练时 weight tying，但参数统计包含独立 head）。实际上 Meta 的实现中 LM Head 和 embedding 是独立参数。

### 3.6 Qwen2-7B

## 4. 参数量 vs 计算量(FLOPs) 的关系

### 4.1 前向传播 FLOPs

对于矩阵乘法 `Y = X × W`，其中 X ∈ R^{b×m}，W ∈ R^{m×n}：
- 每个输出元素需要 m 次乘法 + (m-1) 次加法 ≈ 2m 次浮点运算
- 总 FLOPs = 2 × b × m × n

因此，对于参数量为 P 的线性层，处理一个 token 的 FLOPs ≈ 2P。

整个模型的前向传播：

```
FLOPs_forward ≈ 2P（每 token）
```

这里 P 是模型的总参数量（忽略 embedding lookup、归一化等低成本操作）。

### 4.2 反向传播 FLOPs

反向传播需要计算两部分梯度：
1. 对输入的梯度（用于链式法则传递）：≈ 2P FLOPs
2. 对权重的梯度（用于参数更新）：≈ 2P FLOPs

```
FLOPs_backward ≈ 4P（每 token）
```

### 4.3 训练总 FLOPs

```
FLOPs_total = FLOPs_forward + FLOPs_backward ≈ 6P（每 token）
```

训练 D 个 token 的总 FLOPs：

```
C = 6PD
```

其中 C 是总计算量，P 是参数量，D 是训练 token 数。

示例：训练 LLaMA-7B（P=7B）on 1T tokens
```
C = 6 × 7×10⁹ × 10¹² = 4.2 × 10²² FLOPs
```

换算为 GPU 时间（A100 312 TFLOPS bf16）：
```
时间 = 4.2×10²² / (312×10¹²) = 1.35×10⁸ 秒 ≈ 1,560 GPU 天
```

用 1000 张 A100 约 1.56 天（考虑 MFU 约 40-50%，实际 3-4 天）。

### 4.4 Chinchilla Scaling Law

DeepMind 的 Chinchilla 论文（Hoffmann et al., 2022）发现：

```
最优训练 tokens D_opt ≈ 20P
```

即参数量和训练数据应等比增长。给定固定算力 C：

```
P_opt ∝ C^{0.5}
D_opt ∝ C^{0.5}
```

各模型规模的最优训练量：

| 模型规模 | Chinchilla 最优 tokens | 实际训练量 |
|---------|----------------------|-----------|
| 1B | 20B | — |
| 7B | 140B | LLaMA-1: 1T（过度训练 7× ← 数据充足时更高效） |
| 13B | 260B | LLaMA-1: 1T |
| 70B | 1.4T | LLaMA-2: 2T |
| 175B | 3.5T | GPT-3: 300B（严重欠训练） |

现代趋势：实践中倾向于"过度训练"小模型（训练远多于 20P 的数据），因为推理成本远大于训练成本。LLaMA 系列训练 1-2T tokens 远超 Chinchilla optimal，换取更好的推理性价比。

### 4.5 注意力计算的 FLOPs

注意力机制本身（QK^T 和 AV 矩阵乘法）不在 `2P` 估算中，需要单独计算：

```
FLOPs_attention = 2 × L × 2 × s² × d
```

其中 s 是序列长度。当 s 较大时，这部分不可忽略：

```
总 FLOPs ≈ 2P + 4Ls²d（每 token，注意力二次复杂度部分）
```

对于 s=2048, L=32, d=4096 的 7B 模型：
- 参数相关：2P = 14B FLOPs/token
- 注意力相关：4 × 32 × 2048 × 4096 = 1.07B FLOPs/token
- 注意力占比约 7%，较小

但当 s=128K 时，注意力 FLOPs 会急剧增长到主导地位。

### 4.6 KV Cache 显存

推理时，为避免重复计算，需要缓存所有层的 K 和 V：

```
KV Cache 大小 = 2 × L × 2 × d_kv × s × b × bytes_per_param
```

解释：
- 2：K 和 V 各一份
- L：层数
- d_kv：KV 维度。MHA 下 d_kv = d；GQA 下 d_kv = g × d_head
- s：当前序列长度
- b：batch size
- bytes_per_param：精度对应的字节数（FP16=2, INT8=1）

示例：LLaMA-3-8B，FP16 推理，batch=1，s=8192

```
MHA 下: KV = 2 × 32 × 4096 × 8192 × 2 bytes = 4,294,967,296 bytes = 4 GB
GQA(g=8) 下: KV = 2 × 32 × 1024 × 8192 × 2 bytes = 1,073,741,824 bytes = 1 GB
```

GQA 将 KV Cache 减少到 MHA 的 g/h = 8/32 = 1/4，这是 GQA 的核心优势——不仅节省参数，更关键的是大幅减少推理时的 KV Cache 内存。

## 5. 显存占用估算

### 5.1 训练显存

训练时显存由四部分组成：

```
总显存 = 模型参数 + 梯度 + 优化器状态 + 激活值
```

#### FP32 训练（纯精度）

```
参数:     4P bytes （FP32，每参数 4 字节）
梯度:     4P bytes
Adam 状态: 8P bytes （m: 4P + v: 4P，一阶/二阶矩）
合计:     16P bytes（不含激活值）
```

#### BF16 混合精度训练（主流方案）

```
参数(BF16):      2P bytes （前向/反向用 BF16）
梯度(BF16):      2P bytes
Master Weights:  4P bytes （FP32 主权重副本）
Adam m(FP32):    4P bytes
Adam v(FP32):    4P bytes
合计:           16P bytes（不含激活值）
```

注意：混合精度并不能节省优化器相关的显存！主要优势在于计算速度和激活值存储。

#### 激活值（Activation Memory）

激活值显存与 batch size、序列长度、层数相关，难以简单用 P 表达：

```
激活值 ≈ L × s × b × (34d + 5as²/s)  bytes  （BF16，近似）
```

其中 a 是注意力头数。通常使用激活重计算（activation checkpointing）将激活值显存降低为 √L 倍。

#### 训练显存估算示例

7B 模型 BF16 混合精度训练：

```
参数 + 梯度 + 优化器 = 16 × 7B = 112 GB
激活值（取决于 batch/seq）≈ 20-50 GB
总计 ≈ 130-160 GB
```

这就是为什么单卡 80GB A100 无法全量训练 7B 模型，需要多卡并行或 DeepSpeed ZeRO 等内存优化。

### 5.2 推理显存

推理只需加载模型参数 + KV Cache：

```
显存 = 模型参数 + KV Cache
```

各精度下的模型参数显存：

| 精度 | 每参数字节 | 7B 模型 | 13B 模型 | 70B 模型 |
|-----|----------|--------|---------|---------|
| FP32 | 4 | 28 GB | 52 GB | 280 GB |
| FP16/BF16 | 2 | 14 GB | 26 GB | 140 GB |
| INT8 | 1 | 7 GB | 13 GB | 70 GB |
| INT4 (GPTQ/AWQ) | 0.5 | 3.5 GB | 6.5 GB | 35 GB |
| GGUF Q4_K_M | ~0.56 | ~4 GB | ~7.3 GB | ~39 GB |

加上 KV Cache 和系统开销，实际显存需求约增加 10-30%。

### 5.3 推理显存实际案例

LLaMA-3-8B，INT4 量化，batch=1，seq=4096：

```
模型参数: 8B × 0.5 = 4 GB
KV Cache (GQA g=8): 2 × 32 × 1024 × 4096 × 2 bytes = 0.5 GB
系统开销: ~0.5 GB
总计: ~5 GB
```

可以在单张 RTX 4060 (8GB) 上运行。

LLaMA-2-70B，FP16，batch=1，seq=4096：

```
模型参数: 70B × 2 = 140 GB
KV Cache (GQA g=8): 2 × 80 × 1024 × 4096 × 2 bytes = 1.25 GB
总计: ~142 GB → 需要 2× A100 80GB
```

### 5.4 显存优化技术总结

| 技术 | 适用场景 | 节省量 |
|-----|---------|-------|
| 量化 (INT8/INT4) | 推理 | 模型参数 2-4× |
| GQA | 推理 | KV Cache h/g × |
| Flash Attention | 训练+推理 | 注意力激活值从 O(s²) 到 O(s) |
| Activation Checkpointing | 训练 | 激活值 ~√L × |
| ZeRO-1 | 训练 | 优化器状态分片 |
| ZeRO-2 | 训练 | 优化器+梯度分片 |
| ZeRO-3 | 训练 | 全分片 |
| LoRA | 微调 | 可训练参数 <1% |
| PagedAttention (vLLM) | 推理 | KV Cache 碎片利用 |

## 7. 面试高频问题

### Q1: 给定 d=4096, L=32，快速估算参数量？

答案：

```
P ≈ 12Ld² = 12 × 32 × 4096² = 12 × 32 × 16,777,216 = 6.44B
```

加上 embedding（取决于 V），约 6.5-8B。

要点：记住 `12Ld²` 这个核心公式。其中 4d² 来自注意力，8d² 来自 FFN（标准或 SwiGLU 均约 8d²）。

如果使用 GQA (g=8, h=32)：
```
Attention: 2d² + 2gd²/h = 2d² + d²/2 = 2.5d²
FFN: 8d²
每层 ≈ 10.5d²
P ≈ 10.5 × 32 × 4096² ≈ 5.6B + Embedding
```

---

### Q2: 为什么 LLaMA 用 SwiGLU 时 d_ff = 8d/3？

答案：

标准 FFN 参数量 = 2 × d × d_ff。当 d_ff = 4d 时，参数量 = 8d²。

SwiGLU 有三个权重矩阵（gate/up/down），参数量 = 3 × d × d_ff。

为了在相同参数预算下公平对比：
```
3 × d × d_ff = 8d²
d_ff = 8d/3
```

例如 d=4096 → d_ff = 10922.67，取整到 256 的倍数得 11008（LLaMA-7B 实际值）。

但 LLaMA-3 选择了更大的 d_ff=14336（约 3.5d），突破了参数对等约束，表明更宽的 FFN 能进一步提升性能，值得投入额外参数。

---

### Q3: MQA/GQA 省了多少参数和 KV Cache？

答案：

参数节省（以 d=4096, h=32 为例）：

| 方案 | 注意力参数 | 相对 MHA | KV Cache |
|-----|----------|---------|----------|
| MHA (g=32) | 4d² = 67.1M | 100% | 100% |
| GQA (g=8) | 2d²+d²/2 = 41.9M | 62.5% | 25% |
| GQA (g=4) | 2d²+d²/4 = 37.7M | 56.3% | 12.5% |
| MQA (g=1) | 2d²+d²/16 = 34.6M | 51.6% | 3.1% |

KV Cache 的节省比参数节省更为关键：

```
KV Cache 比例 = g/h
```

GQA(g=8, h=32) 的 KV Cache 只有 MHA 的 1/4，这对长序列推理至关重要。

实际意义：对于 batch serving，KV Cache 往往是显存瓶颈。GQA 能让同样显存下 batch size 增大 4 倍，直接提升吞吐量。

### Q5: Chinchilla optimal — 7B 模型最优训练 tokens 是多少？

答案：

```
D_opt ≈ 20 × P = 20 × 7B = 140B tokens
```

对应训练 FLOPs：
```
C = 6PD = 6 × 7B × 140B = 5.88 × 10²¹ FLOPs
```

但实际操作中，LLaMA 系列训练了 1-2T tokens（7-14× Chinchilla optimal），原因：
1. 训练成本是一次性的，推理成本是持续的
2. 过度训练的小模型在推理时比"Chinchilla optimal"的大模型更经济
3. 高质量训练数据充足时，过度训练仍然有收益（虽然收益递减）

LLaMA-3 训练了 15T tokens on 8B 模型，是 Chinchilla optimal 的 100+ 倍，但确实在同尺寸模型中效果出色。

### Q6: 参数量翻倍，效果能好多少？（Scaling Law）

答案：

Kaplan et al. (2020) 的 Neural Scaling Laws：

```
L(P) = (P_c / P)^{α_P}
```

其中 α_P ≈ 0.076（以 nats 为单位的 loss）。

这意味着：
- 参数量翻倍 → loss 降低约 `2^{0.076} - 1 ≈ 5.4%`
- 参数量增加 10× → loss 降低约 `10^{0.076} - 1 ≈ 19.1%`

更直观的经验法则：
- 7B → 13B（~2×）：基准测试提升 3-5 个百分点
- 7B → 70B（~10×）：基准测试提升 10-20 个百分点
- 70B → 405B（~6×）：基准测试提升 5-10 个百分点（收益递减）

重要注意：
- Scaling Law 描述的是 loss，不是下游任务准确率
- 某些"涌现能力"（如 CoT 推理）可能在特定规模出现跳跃
- 数据质量、训练方法的改进可以在不增加参数的情况下显著提升效果（GPT-4o mini vs GPT-3）

### Q7: 为什么 Transformer 的参数主要集中在 FFN？

答案：

以标准架构为例，每层参数分布：

```
Attention: 4d²  → 33.3%
FFN:       8d²  → 66.7%
```

FFN 参数是 Attention 的 2 倍，原因：
1. Attention 的 Q/K/V/O 四个 d×d 矩阵 = 4d²
2. FFN 的两层线性变换维度为 d→4d→d = 8d²（中间层 4 倍扩展）
3. FFN 的 4× 扩展倍率是经验选择，增大 FFN 容量有助于存储知识

研究发现（Geva et al., 2021）：FFN 的第一层可以看作"key memory"，第二层是"value memory"，整个 FFN 相当于一个巨大的键值记忆网络。这解释了为什么：
- 知识密集型任务更依赖 FFN（知识存储在 FFN 权重中）
- MoE 选择在 FFN 上做专家化（参数密集 + 知识存储）
- 模型编辑（knowledge editing）主要修改 FFN 层

### Q8: 如何估算一个新模型的参数量？

答案（系统化步骤）：

第一步：确认架构超参数
```
d（隐藏维度）、L（层数）、h（注意力头数）、V（词表大小）
d_ff（FFN 中间维度）、g（GQA 组数，若有）
是否有 bias、norm 类型、head 是否 tied
```

第二步：套公式
```
Embedding:  V × d
Attention:  L × (2d² + 2gd²/h)    （GQA，无 bias）
FFN:        L × 3d × d_ff          （SwiGLU，无 bias）
Norm:       L × 2d + d             （RMSNorm）
LM Head:    V × d                  （若 untied）
```

第三步：加总
```
P = V×d + L×(2d² + 2gd²/h + 3d×d_ff + 2d) + d + [V×d]
```

第四步：快速检验
```
主体 ≈ 12Ld²（或 GQA 下 ~10-11Ld²）
```

如果手算结果与 `12Ld²` 差距很大，检查是否遗漏了组件或 d_ff 取值不同。
