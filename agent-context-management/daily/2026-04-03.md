### 1. Memory in the LLM Era: Modular Architectures and Strategies in a Unified Framework
**★★★★★ 必读综述**

- **提交：** 2026-04-02
- **作者：** Yanchen Wu, Tenghui Lin, Yingli Zhou, Fangyuan Zhang et al.
- **性质：** 综述论文
- **核心：** 将 LLM 时代的 memory 作为核心模块，从模块化架构视角统一梳理

**框架概述：**
- Memory 是 LLM 系统的核心模块（而非附属品）
- 提出统一框架涵盖：模块化架构、存储策略、检索策略、更新策略
- 涵盖 In-Context Memory、Parametric Memory、External Memory 三大类

**面试要点：**
- 问"LLM memory 有哪些类型？" → 三分法：参数记忆、上下文记忆、外部记忆
- 问"如何设计 agent memory 架构？" → 模块化视角：写/索引/检索/使用/精炼
- 该综述可作为面试复习的 backbone

### 2. ContextBudget: Budget-Aware Context Management for Long-Horizon Search Agents
**★★★★★ 工程实战**

- **提交：** 2026-04-02
- **作者：** Yong Wu, YanZhao Zheng, TianZe Xu, ZhenTao Zhang et al. (多人团队)
- **场景：** 长 horizon 搜索型 agent（如 deep research agent）

**核心问题：**
- 长 horizon 任务中，搜索/工具调用积累的 context 越来越长
- 超出 context window → 截断导致信息丢失
- 朴素截断 = 随机丢弃重要信息

**ContextBudget 方案：**
- **预算感知（Budget-Aware）：** 预先分配 token 预算给不同信息类型
- **动态压缩：** 按重要性分级压缩已有上下文
- **优先级队列：** 高价值信息受保护，低价值自动摘要/丢弃
- 在长 horizon 搜索任务上显著优于朴素截断

**与推荐系统类比：**
- context budget ≈ 推荐系统的"曝光预算"
- 信息优先级 ≈ 候选排序中的相关性打分
- 动态压缩 ≈ 用户历史序列的时序衰减

**面试价值：**
- "如何处理长 context 超出 window 的问题？" → ContextBudget 方案

### 3. ByteRover: Agent-Native Memory Through LLM-Curated Hierarchical Context
**★★★★☆ 架构设计**

- **提交：** 2026-04-02
- **作者：** Andy Nguyen, Danh Doan, Hoang Pham et al.

**核心贡献：**
- **Memory-Augmented Generation (MAG)** 的新实现
- **LLM 主动策划 memory**：不是规则过滤，而是让 LLM 自己决定"什么值得记住"
- **分层 Context**：
  - Level 1: 原始交互历史（短期）
  - Level 2: LLM 萃取的结构化 memory（中期）
  - Level 3: 长期 knowledge base（持久化）

**架构要点：**
- Agent-Native：memory 是 agent 内建能力，非外挂模块
- Hierarchical Curation：LLM 在每层之间做 distillation
- 支持跨会话持久化

**面试价值：**
- "如何让 agent 具备长期记忆？" → MAG + 分层策划
- ByteRover 的 3 层架构是标准答案模板

### 4. Oblivion: Self-Adaptive Agentic Memory Control through Decay-Driven Activation
**★★★★☆ 机制创新**

- **提交：** 2026-03-31
- **作者：** Ashish Rana, Chia-Chien Hung, Qumeng Sun et al.

**核心洞察：**
- 人类记忆并非"存 or 删"二值，而是随时间衰减 + 可被激活重组
- 现有 agent memory 要么全保留（context 爆炸）、要么硬截断（信息丢失）

**Oblivion 机制：**
- **Decay-Driven**：记忆重要性随时间指数衰减（类 Ebbinghaus 遗忘曲线）
- **Contextual Reactivation**：相关上下文触发记忆重新激活（类人类联想）
- **Self-Adaptive**：无需人工设阈值，根据任务动态调整衰减率
- 实现"遗忘 = 压缩而非删除"

**关键术语（面试可直接用）：**
- Memory Decay: 时间衰减函数
- Reactivation: 上下文触发记忆回调
- Adaptive Decay Rate: 任务感知的衰减速率

**与推荐系统类比：**
- Decay ≈ 时序 attention 中的位置权重衰减
- Reactivation ≈ 序列推荐中的 target attention

### 5. AgentSwing: Adaptive Parallel Context Management Routing for Long-Horizon Web Agents
**★★★☆☆ 工程优化**

- **提交：** 2026-03-28
- **作者：** Zhaopeng Feng, Liangcai Su et al. (大团队)
- **场景：** 长 horizon Web Agent（网页操作/自动化）

**核心方案：**
- **并行 Context 路由**：不同子任务维护独立 context 分支
- **自适应合并**：根据子任务完成情况选择性合并 context
- 避免"context 污染"（不相关子任务的信息互相干扰）

**面试价值：**
- "Multi-agent 如何管理共享上下文？" → 并行分支 + 选择性合并

### 6. Reasoning Shift: How Context Silently Shortens LLM Reasoning
**★★★☆☆ 深刻洞察**

- **提交：** 2026-04-01

**核心发现：**
- LLM 的 extended reasoning（chain-of-thought）会因 context 内容而被"悄悄缩短"
- 更多的 context 不等于更好的推理 → 反直觉结论
- 验证了 context 质量 > context 数量

**面试价值：**
- "context 越长越好吗？" → Reasoning Shift 的反例
- 支持"精选 memory 优于堆砌 history"的论点

## 💡 今日核心洞见

### 洞见 1：Memory 三大范式的统一
```
参数记忆（权重）← 慢，需要训练，泛化强
上下文记忆（KV/prompt）← 快，无需训练，上下文有限
外部记忆（DB/vector store）← 持久，可扩展，检索有噪声
```
最优 agent = 三者有机结合，而非单一依赖

### 洞见 2：上下文预算 = 信息资源分配
ContextBudget 本质是把 context window 视为"有限资源"来做优化
→ 和推荐系统的"位置资源"（曝光位置）、"用户注意力"同属资源分配问题

### 洞见 3：遗忘即压缩
Oblivion 的核心哲学：**遗忘不是删除，而是压缩**
→ 近期高频信息保持高激活
→ 久远低频信息压缩为摘要
→ 相关时可重新展开

## 📌 面试准备 QA

**Q: LLM Agent 如何处理超长历史上下文？**
A: 三种方案：1) 预算感知压缩（ContextBudget） 2) 分层记忆（ByteRover） 3) 遗忘机制（Oblivion）。核心思路是"信息分级 + 动态压缩"。

**Q: 如何设计持久化 Agent Memory 系统？**
A: 参考 ByteRover 的三层架构：原始历史（短期）→ LLM 萃取结构化摘要（中期）→ 持久化知识库（长期）。关键是让 LLM 主动决定哪些信息值得长期保留。

**Q: Agent Memory 和人类记忆有什么类比？**
A: Oblivion 论文提供了最好的类比：Ebbinghaus 遗忘曲线 + 联想激活。记忆按时间衰减，但上下文相关时可重新激活——这比"全留/全删"更贴近认知科学。
