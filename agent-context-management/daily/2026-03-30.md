### 1. MemoryCD: Benchmarking Long-Context User Memory for Lifelong Cross-Domain Personalization
- **提交：** 2026-03-26 | arXiv
- **作者：** Weizhi Zhang, Xiaokai Wei, Wei-Chieh Huang, Zheng Hui, Chen Wang, Michelle Gong, Philip S. Yu
- **核心问题：** LLM 上下文窗口扩展到百万 token 级别后，如何评估 agent 在跨领域终身个性化任务中的长上下文用户记忆能力？
- **核心贡献：**
  - 提出 MemoryCD benchmark，专门评估 LLM agent 的**跨领域终身个性化**记忆
  - 覆盖多领域（购物、娱乐、健康等）用户历史行为的长上下文建模
  - 直接回应"百万 token 上下文窗口是否真的解决了用户记忆问题"
- **与推荐系统的交叉点：**
  - 跨域用户建模 → 跨域迁移推荐
  - 终身个性化 → long-term user interest modeling
  - 长上下文 user memory → user tower 的 sequence modeling 替代方案
- **面试价值：** ★★★★★ — "如何建模用户的长期兴趣"是推荐系统面试高频题

### 2. MemMA: Coordinating the Memory Cycle through Multi-Agent Reasoning and In-Situ Self-Evolution
- **提交：** 2026-03-19 | HuggingFace Daily 2026-03-27
- **作者：** Minhua Lin, Zhiwei Zhang, Hanqing Lu, Hui Liu, Xianfeng Tang, Qi He, Xiang Zhang, Suhang Wang
- **核心问题：** 当前 memory-augmented LLM agent 将记忆构建、检索、利用作为独立模块，导致"战略盲点"（strategic blindness）
- **核心贡献：**
  - 提出 MemMA 框架，通过多 Agent 协同推理打通 **memory cycle**（构建→检索→利用→演化）
  - **In-Situ Self-Evolution**：memory 在使用过程中自动精炼，无需人工标注
  - Contrastive Trajectory Distillation：通过对比 trajectory 蒸馏提升 memory 质量
- **关键概念：**
  - Memory Cycle = Write → Index → Retrieve → Use → Refine
  - Strategic Blindness = 各模块独立优化导致全局次优
- **面试价值：** ★★★★☆ — 适合回答"如何设计 agent 的记忆系统"

### 3. ClinicalAgents: Multi-Agent Orchestration for Clinical Decision Making with Dual-Memory
- **提交：** 2026-03-27 | arXiv
- **作者：** Zhuohan Ge, Haoyang Li, Yubo Wang, Nicole Hu, Chen Jason Zhang, Qing Li
- **核心问题：** 医疗 LLM 无法完成复杂的非线性假设驱动推理
- **核心贡献：**
  - 提出 **Dual-Memory 架构**：短期工作记忆（当前病例推理）+ 长期经验记忆（历史诊断知识）
  - 多 Agent 编排模拟人类临床医生的迭代假设验证
- **Dual-Memory 设计：**
  - Working Memory：当前上下文、症状、假设
  - Episodic Memory：历史成功/失败诊断案例
- **面试价值：** ★★★☆☆ — 双记忆系统设计思路可迁移到推荐/搜索 agent

---

### 4. BeliefShift: Benchmarking Temporal Belief Consistency and Opinion Drift in LLM Agents
- **提交：** 2026-03-24 | arXiv
- **作者：** Praveen Kumar Myakala, Manan Agrawal, Rahul Manche
- **核心问题：** LLM 作为长期对话 agent 运行时，信念/观点随时间是否保持一致？
- **核心贡献：**
  - BeliefShift benchmark：测量 LLM agent 在时间维度上的**信念一致性**和**意见漂移**
  - 暴露现有 LLM 在长期交互中的"记忆不一致"问题
- **关键洞察：**
  - 即使有外部 memory 支持，LLM 仍会产生意见漂移（opinion drift）
  - 时序记忆一致性是 agent 可信赖性的核心要素
- **面试价值：** ★★★☆☆ — 适合讨论 agent 可靠性与记忆一致性

---

### 5. Conversation Tree Architecture: A Structured Framework for Context-Aware Multi-Branch LLM Conversations
- **提交：** 2026-03-22 | arXiv
- **核心问题：** 当前对话界面的扁平 append-only 结构无法支持多话题并行对话
- **核心贡献：**
  - 提出"对话树"架构：将对话历史组织成树形结构，支持多分支、多话题并行
  - 每个分支独立维护上下文，避免跨话题的上下文污染
- **工程意义：**
  - Agent 系统中如何管理多任务并发上下文
  - 可应用于客服 bot、多步骤任务 agent 的上下文隔离
- **面试价值：** ★★★☆☆ — Context 隔离设计是 agent 工程面试常见问题

## 💡 今日洞察

### 趋势观察
1. **Memory Benchmark 爆发期**：本周出现 MemoryCD、BeliefShift、VehicleMemBench 三个独立 benchmark，说明社区认为"如何评估 memory"已成独立研究方向
2. **Dual-Memory 范式**：短期工作记忆 + 长期情景记忆的双层结构在多个领域被采用（医疗、车载），逐渐成为标准架构模式
3. **Memory Cycle 系统化**：MemMA 提出将 memory 的全生命周期（写入→检索→使用→演化）统一建模，打破各模块割裂的局面
4. **In-Situ Evolution**：memory 在使用中自动精炼的思路，类似推荐系统的在线学习（online learning），值得关注

### 与推荐系统的联系
- MemoryCD 的"跨域终身个性化"= 跨域推荐 + 用户终身兴趣建模
- Dual-Memory = 推荐系统的短期兴趣 + 长期兴趣分离建模（DIN/DIEN 思路的 agent 版本）
- Memory Cycle = 推荐系统特征工程 pipeline 的 agent 化
