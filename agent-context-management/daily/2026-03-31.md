## 🔥 今日亮点论文

### 1. HyDRA：Hybrid Memory for Dynamic Video World Models
- **arXiv：** 2603.25716 | **HF 热度：** 🔥 133 upvotes（今日 HF 第一）
- **提交：** 2026-03-26（HF Daily 2026-03-30）
- **作者：** Kaijin Chen et al. (H-EmbodVis)
- **代码：** https://github.com/H-EmbodVis/HyDRA
- **核心问题：** 动态目标离开视野后再出现时，现有模型会导致冻结/扭曲/消失
- **核心贡献：**
  - 提出 **Hybrid Memory 范式**：同时扮演"精确存档者"（静态背景）和"主动追踪者"（动态目标）
  - 构建 **HM-World 数据集**：59K 高保真视频片段，17 场景，49 类目标
  - **HyDRA 架构**：将 memory 压缩为 tokens + 时空相关性驱动检索机制
- **关键技术：**
  - Memory Token Compression（压缩 + 结构化存储）
  - Spatiotemporal Relevance-Driven Retrieval（时空相关检索）
- **迁移意义：** Memory token 压缩 + 选择性检索的范式，可直接借鉴到 LLM 长上下文 KV cache 压缩
- **面试价值：** ★★★★☆ — 体现了 memory 设计中"精度存档"与"动态追踪"分离的设计哲学

### 2. Trace2Skill：从轨迹中提炼可迁移的 Agent Skills
- **arXiv：** 2603.25158 | **HF 热度：** 33 upvotes（HF Daily 2026-03-30）
- **提交：** 2026-03-26，更新 2026-03-29
- **作者：** Jingwei Ni et al.
- **核心问题：** 人工编写 skill 成本高，自动生成的 skill 往往脆弱/不可迁移
- **核心贡献：**
  - **Trace2Skill 框架**：并行子 agent 分析多条执行轨迹 → 归纳推理 → 统一无冲突 skill 目录
  - 支持"深化已有人工 skill"和"从零创建 skill"两种模式
  - Qwen3.5-35B 演化出的 skill，使 Qwen3.5-122B 在 WikiTableQuestions 上提升 57.65 个百分点
- **关键技术：**
  - Parallel Sub-Agent Fleet（并行经验分析）
  - Hierarchical Inductive Consolidation（层次归纳整合）
  - Trajectory-Local Lessons → Transferable Declarative Skills
- **迁移意义：** Agent memory 从"记住轨迹"升级到"提炼通用知识"——与人类专家学习方式类比
- **面试价值：** ★★★★☆ — 经验复用 / 技能提取 / agent 自我改进

### 3. Codebase-Memory：基于 Tree-Sitter 知识图谱的代码库记忆（MCP）
- **arXiv：** 搜索 "Codebase-Memory Tree-Sitter Knowledge Graph MCP"
- **提交：** 2026-03-28
- **作者：** Martin Vogel, Falk Meyer-Eschenbach et al.
- **核心贡献：**
  - 用 Tree-Sitter 解析代码库 → 构建结构化知识图谱
  - 通过 **MCP（Model Context Protocol）** 暴露给 LLM
  - 解决 LLM 探索大型代码库时 context 爆炸问题
- **关键技术：** 知识图谱 + MCP 接口 + 代码结构化 memory
- **迁移意义：** 结构化外部 memory 通过标准协议接入的工程范式
- **面试价值：** ★★★☆☆ — agent 工程实践参考

### 4. ClinicalAgents：Dual-Memory 架构的多 Agent 临床决策
- **arXiv：** 搜索 "ClinicalAgents Multi-Agent Dual-Memory clinical diagnosis"
- **提交：** 2026-03-27
- **作者：** Zhuohan Ge, Haoyang Li et al.
- **核心贡献：**
  - Dual-Memory 架构：**工作记忆**（当前病历推理）+ **长期记忆**（医学知识库）
  - 多 agent 编排解决非线性临床推理
  - 克服"静态线性映射症状→诊断"的局限
- **关键技术：** Dual-Memory + Multi-Agent Orchestration + Hypothesis-Driven Reasoning
- **迁移意义：** Dual-Memory（短期工作 memory + 长期知识 memory）是通用 agent 设计模式
- **面试价值：** ★★★☆☆ — memory 分层设计的具体实现

### 5. Beyond Completion：通过累积状态追踪预测 LLM Agent 表现
- **arXiv：** 搜索 "Beyond Completion Cumulative State Tracking LLM Agent Performance"
- **提交：** 2026-03-28
- **作者：** Dengzhe Hou, Lingyu Jiang et al.
- **核心贡献：**
  - 任务完成率不足以评估 agent 表现，提出**累积状态追踪（CST）**指标
  - 通过中间状态质量预测最终 agent 性能
  - 为 agent 记忆 / 上下文管理提供新的评估维度
- **迁移意义：** 强调 agent "过程状态" 管理比最终结果更重要
- **面试价值：** ★★★☆☆ — agent 评估框架 + 状态管理视角

## 💡 今日关键洞察

1. **Memory Token 压缩**正在成为一个统一趋势：HyDRA、MSA（昨日）都在做 memory 压缩到 tokens，减少存储开销同时保留语义。

2. **从轨迹到技能的提炼**（Trace2Skill）代表 agent memory 的更高层次——不只是记住发生了什么，而是提炼可复用的行动知识。这与**推荐系统中用户行为序列建模 → 用户兴趣表示**高度类比。

3. **Dual-Memory 架构**（短期工作记忆 + 长期知识记忆）出现在医疗 agent 中，但这是通用模式：搜广推系统中的"session feature"（短期）+ "user profile"（长期）就是类似思路。

4. **MCP（Model Context Protocol）**作为 agent memory 标准接口正在普及——Codebase-Memory 用 MCP 暴露结构化代码知识图谱，这是 agent 工具集成的工程范式。
