### 1. Reasoning Shift: How Context Silently Shortens LLM Reasoning ★★★★★
- **arXiv：** 2504.0xxxx（2026-04-01，cs.CL）
- **核心发现：** Context 会"悄悄"缩短推理模型的推理链，导致性能下降
- **三种场景测试：**
  1. 问题附加大量**不相关长上下文** → 推理模型跳过必要推理步骤
  2. **多轮对话**中有无关历史 → reasoning chain 缩短
  3. 问题作为**复杂任务的子任务**呈现 → 推理缩短最明显
- **关键洞察：** 这是 "Reasoning Shift" 现象 —— 模型看到长 context 就以为自己已经推理足够，提前截断 CoT
- **工程意义：** 不要随意堆砌 context；检索增强时要注意噪声 context 对推理质量的负面影响
- **与推荐系统的关联：** 类似于精排模型 feature 噪声问题；无关 context 相当于噪声特征拖累判断
- **面试价值：** ★★★★★ — 能回答"为什么 RAG 有时让 LLM 变笨"这类问题

### 2. HippoCamp: Benchmarking Contextual Agents on Personal Computers ★★★★
- **arXiv：** 2504.0xxxx（2026-04-01，cs.AI）
- **作者：** Zhe Yang, Shulin Tian et al.（NTU MMLab）
- **核心贡献：**
  - 首个评估 PC 端 Agent **多模态文件上下文管理**能力的 Benchmark
  - 场景：Agent 需要从本地文件（PDF、代码、图片、表格）中理解和利用上下文完成任务
  - 评估维度：跨文件记忆保持、上下文切换准确性、文件关联推理
- **技术亮点：** 模拟真实 PC 工作流，而非人工构造的 QA 任务
- **关键指标：** 当前最佳模型准确率约 45%，说明 PC 端上下文理解还有很大提升空间
- **面试价值：** ★★★★ — Agent 实际落地场景的 Benchmark，能说明"上下文管理远未解决"

### 3. SkillReducer: Optimizing LLM Agent Skills for Token Efficiency ★★★★
- **arXiv：** 2503.xxxx（2026-03-31，cs.AI）
- **作者：** Yudong Gao, Zongjie Li et al.
- **背景：** LLM Agent 依赖 "skills"（预打包的指令集）扩展能力，但每个 skill 注入 prompt 都消耗大量 token
- **核心思路：**
  - 识别每个任务真正需要的 skill 子集 → 剪掉无关 skill
  - 保留精简 skill 描述，去除冗余说明文字
  - 减少 prompt 中 skill token 消耗 → 为实际推理腾出更多 context 空间
- **效果：** 在保持性能的前提下减少 ~40% skill-related token 消耗
- **工程意义：** 直接应用于 OpenClaw / MelonEgg 等 agent 框架的 tool description 优化
- **面试价值：** ★★★★ — "如何在有限 context 中优化 agent 工具调用效率"

### 4. YC-Bench: Benchmarking AI Agents for Long-Term Planning and Consistent Execution ★★★★
- **arXiv：** 2504.0xxxx（2026-04-01，cs.AI）
- **作者：** Muyu He, Adit Jain et al.（HuggingFace）
- **核心贡献：**
  - 评估 LLM Agent **长时规划**的 Benchmark（命名取自 Y Combinator，强调"完整闯关"能力）
  - 三大评估维度：
    1. **不确定性下规划**（Planning under uncertainty）
    2. **延迟反馈学习**（Learning from delayed feedback）
    3. **早期错误级联适应**（Adapting when early mistakes compound）
  - 与短视 Benchmark 的区别：要求 Agent 维持**跨步骤的策略连贯性**
- **关键发现：** 当前 LLM Agent 在长时规划上表现远差于短任务，一致性维持是核心瓶颈
- **面试价值：** ★★★★ — 讨论 Agent 记忆/规划时可引用，说明"记住上下文"≠"长时策略一致"

### 5. SimpleStream: A Simple Baseline for Streaming Video Understanding ★★★
- **arXiv：** 2504.0xxxx（2026-04-02，cs.CV）
- **作者：** Yujiao Shen, Shulin Tian et al.（NTU MMLab）
- **反直觉发现：** 复杂记忆机制 ≠ 更好效果
  - 只用最近 N 帧的**滑动窗口**就已经 match 或超越现有复杂 streaming 方法
  - 挑战了"streaming video 需要复杂记忆架构"的假设
- **对 context 管理的启示：**
  - "新鲜"的上下文（recent context）往往比压缩的历史记忆更有价值
  - 复杂的记忆机制可能引入压缩噪声，反而不如直接用原始 context
- **面试价值：** ★★★ — 能说"Occam's Razor 在 context 管理中同样适用"

### 6. Novel Memory Forgetting Techniques for Autonomous AI Agents ★★★★
- **arXiv：** 2504.0xxxx（2026-04-02，cs.AI）
- **核心问题：** 长期运行的对话 Agent 如何在"记住重要信息"和"忘掉过时细节"之间取得平衡
- **主要贡献：**
  - 提出多种遗忘策略：基于时间衰减、相关性评分、用户偏好权重
  - 对比实验：纯增量记忆 vs 主动遗忘策略，主动遗忘显著提升长期对话质量
  - 发现"智能遗忘"比"完整记忆"更符合用户体验（减少噪声干扰）
- **与 Oblivion 的区别：** 本文更偏重工程实践和多策略对比，Oblivion 更注重认知科学理论
- **工程意义：** 构建 Agent 记忆模块时，需要主动设计"遗忘机制"，而非只做追加写入
- **面试价值：** ★★★★ — "Agent 记忆如何防止无限膨胀？" 的系统性回答
