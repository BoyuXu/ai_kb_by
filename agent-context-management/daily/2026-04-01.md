## 🔥 核心论文（高度相关）

### 1. MuSEAgent: A Multimodal Reasoning Agent with Stateful Experiences
- **arXiv:** 2603.27813 | **提交：** 2026-03-29 | **来源：** HF Daily Papers
- **机构：** Southeast University, Monash University, Xiaohongshu, SJTU 等
- **核心贡献：**
  - 提出 **Stateful Experience Learning（状态化经验学习）** 范式
  - 将 agent 交互数据抽象为**原子决策经验**（atomic decision experiences）通过 hindsight reasoning
  - 构建 **quality-filtered experience bank（经验库）**，支持推理时 policy-driven 检索
  - 提出 **wide-search + deep-search** 互补策略，跨多语义视角动态检索多模态指导
- **与传统方法的区别：** 不依赖 trajectory-level retrieval，而是抽象到更细粒度的"原子决策"级别
- **实验结果：** 在 fine-grained visual perception 和 complex multimodal reasoning 上超过强 baseline
- **核心关键词：** stateful experience, experience bank, hindsight reasoning, atomic decisions
- **💡 面试价值：** ★★★★★ — 直接对应 "agent memory 如何组织和检索" 面试题
- **代码：** https://github.com/DeepExperience/MuSEAgent
- **备注：** 本文是今日最值得精读的论文，experience bank 设计思路可以直接用于工程实践

### 2. AdaptToken: Entropy-based Adaptive Token Selection for MLLM Long Video Understanding
- **arXiv:** 2603.28696 | **提交：** 2026-03-30 | **来源：** HF Daily Papers
- **机构：** ETH Zurich 等
- **核心贡献：**
  - **无需训练**的自适应 token 选择框架，解决 MLLM 长视频上下文管理问题
  - 用 **response entropy（响应熵）** 作为全局不确定性信号，跨视频组分配 token budget
  - 结合 **cross-modal attention** 做组内 token ranking
  - **AdaptToken-Lite** 变体支持 early-stopping，进一步降低计算成本
- **核心关键词：** token selection, context length, entropy, budget allocation, long context
- **💡 面试价值：** ★★★★☆ — 上下文窗口优化 / KV cache 管理的工程思路
- **迁移思考：** entropy-based token selection 的思路可以迁移到 agent context 的重要性评分上

### 4. Story2Proposal: A Scaffold for Structured Scientific Paper Writing
- **arXiv:** 2603.27065 | **提交：** 2026-03-29 | **来源：** HF Daily Papers
- **机构：** QuantaAlpha 等
- **核心贡献（上下文管理视角）：**
  - 提出 **persistent shared visual contract（持久化共享视觉契约）** 作为跨 agent 的上下文状态
  - 多 agent（architect/writer/refiner/renderer）围绕共享 contract state 协同
  - **generate–evaluate–adapt loop** 在生成过程中持续更新 contract
  - 解决跨 section 的上下文一致性问题（structural drift）
- **类比 agent memory：** visual contract ≈ 共享 working memory，跨 agent 状态同步
- **💡 面试价值：** ★★★☆☆ — 多 agent 共享状态管理的工程模式

## 🧠 今日关键洞察

### 洞察 1：原子化 vs 轨迹级记忆
MuSEAgent 的核心思路是将 trajectory 打碎成 **atomic decision experiences**，而不是整条轨迹存储。
这与人类记忆的"情节提炼"更接近——记住关键决策点，而不是每一步。
- **推荐系统类比：** 类似 session-based rec 中把用户行为序列提炼为关键 click 信号

### 洞察 2：熵作为上下文重要性信号
AdaptToken 用 response entropy 衡量视频片段的信息量，这个思路可以泛化：
- **高熵 = 不确定 = 重要** → 需要更多 context token
- **低熵 = 可预测 = 不重要** → 可以压缩/丢弃
- **应用场景：** agent 上下文压缩时，根据"不确定性"保留关键上下文

### 洞察 3：验证驱动的轨迹构建
Marco DeepResearch 把验证逻辑注入训练轨迹本身 → agent 学会"边做边验证"
而不是事后检查。这对 long-horizon task 的 context management 很关键：
减少错误在 context 中积累传播。
