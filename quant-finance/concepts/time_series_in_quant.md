# 金融时序分析 — 从ARIMA到深度学习

> 金融数据天然是时序数据。如何从价格、成交量的历史序列中提取预测信号，是量化投资的核心问题之一。本文梳理从经典统计到深度学习的完整演进。

## 演进时间线

```
1970s          1980s          2000s          2015+           2020+
ARIMA  ───→  GARCH  ───→  ML时序特征  ───→  LSTM/TCN  ───→  Transformer/PatchTST
│              │              │               │               │
线性趋势      波动率聚集     非线性捕获      长程依赖        注意力机制
均值预测      风险建模       特征工程驱动    端到端学习      全局模式
```

## 1. 经典统计方法

### 1.1 ARIMA（自回归积分移动平均）

**核心思想**：当前值 = 过去值的线性组合 + 噪声的线性组合。

$$ARIMA(p,d,q): \quad \Delta^d X_t = \sum_{i=1}^{p} \phi_i \Delta^d X_{t-i} + \sum_{j=1}^{q} \theta_j \epsilon_{t-j} + \epsilon_t$$

- p：自回归阶数（看多少步历史）
- d：差分阶数（消除趋势，让序列平稳）
- q：移动平均阶数（噪声的记忆长度）

**选参三步曲**：
1. ADF 检验 → 确定 d（需要差分几次才平稳）
2. ACF 图 → 确定 q（截尾点）
3. PACF 图 → 确定 p（截尾点）

**在量化中的地位**：直接用 ARIMA 预测价格效果很差（有效市场），但用于残差建模、信号提取仍有价值。

### 1.2 GARCH（广义自回归条件异方差）

**解决的问题**：金融序列的波动率不是恒定的——有"波动率聚集"现象（大波动后面跟大波动）。

$$\sigma_t^2 = \omega + \alpha \epsilon_{t-1}^2 + \beta \sigma_{t-1}^2$$

**直觉**：今天的波动率 = 常数底 + 昨天冲击的影响 + 昨天波动率的惯性。

**衍生模型**：
- EGARCH：允许非对称效应（暴跌对波动率的影响 > 暴涨）
- GJR-GARCH：同上，另一种建模方式
- GARCH-M：波动率本身影响收益（风险溢价）

**应用**：期权定价（隐含波动率建模）、VaR 计算、风险预算。

### 1.3 VAR（向量自回归）与协整

**VAR**：多个时序变量互相预测。
$$\mathbf{Y}_t = \mathbf{A}_1 \mathbf{Y}_{t-1} + \cdots + \mathbf{A}_p \mathbf{Y}_{t-p} + \mathbf{u}_t$$

**协整（Cointegration）**：两个非平稳序列的某个线性组合是平稳的。

**直觉**：两个醉汉分别随机游走，但他们之间用绳子绑着（长期均衡关系）。虽然各自的位置不可预测，但两人之间的距离是可预测的。

**应用**：配对交易的理论基础。找到协整关系 → 当价差偏离均衡 → 做空高估、做多低估 → 等回归。

## 2. 平稳性：为什么这么重要？

**非平稳序列的危害**：
1. **伪回归**：两个不相关的非平稳序列回归，R² 可能很高（都有趋势）
2. **模型假设违背**：大部分统计模型假设平稳性，不平稳时参数估计不一致
3. **预测失效**：在非平稳数据上训练的模型，未来的统计性质和过去不同

**怎么办**：
- 差分（d阶差分消除趋势）
- 对数变换（消除异方差）
- 去趋势（减去移动平均）
- ADF 检验确认平稳

## 3. ML 方法：特征工程驱动

### 3.1 XGBoost 时序特征

不直接让模型看原始序列，而是手工构造特征：

| 特征类 | 示例 | 含义 |
|--------|------|------|
| 滞后特征 | lag_1, lag_5, lag_20 | 过去的收益率 |
| 滚动统计 | rolling_mean_20, rolling_std_60 | 移动均线、波动率 |
| 技术指标 | RSI, MACD, Bollinger Bands | 技术分析信号 |
| 时间特征 | day_of_week, month, quarter | 日历效应 |
| 量价特征 | volume_ratio, OBV, VWAP | 成交量信号 |
| 截面特征 | rank_return, sector_momentum | 相对排名 |

**关键原则**：特征工程的质量决定了 ML 模型的上限。

### 3.2 Random Forest / LightGBM

- 天然处理非线性交互
- 特征重要性可解释
- 不需要标准化
- 缺点：不擅长捕获长程时序依赖

**实践技巧**：
- Purged K-Fold CV：时序交叉验证，防止信息泄露
- 样本权重：近期样本更重要
- 特征筛选：先用全量特征跑一遍，再根据重要性排序精选

## 4. DL 方法：端到端学习

### 4.1 LSTM（2015-2018 主流）

**优势**：门控机制解决梯度消失，能学长程依赖。

**在金融中的问题**：
- 金融时序信噪比极低，LSTM 容易过拟合噪声
- 训练不稳定，超参数敏感
- 可解释性差

### 4.2 TCN（时间卷积网络）

用因果（causal）膨胀卷积代替 RNN：
- 并行计算，训练速度快
- 感受野可精确控制
- 梯度稳定

### 4.3 Transformer 时序模型（2020+）

**Informer (2021)**：稀疏注意力，O(N log N) 复杂度，解决长序列预测。

**Autoformer (2021)**：分解 + 自相关注意力，更适合时序的周期性模式。

**PatchTST (2023)**：
- 将时序切成 patch（类似 ViT 对图像的操作）
- Channel-Independence：每个变量独立建模
- 在长期预测任务上 SOTA

**核心思想演进**：
```
逐点注意力（Transformer）→ 稀疏注意力（Informer）→ 自相关（Autoformer）→ Patch + CI（PatchTST）
```

### 4.4 最新趋势

- **Foundation Model for Time Series**：用大规模预训练（如 TimesFM、Chronos），小样本微调
- **State Space Models**：Mamba 等 SSM 在时序上的应用
- **LLM as Time Series Reasoner**：用 LLM 做时序推理（争议较大）

## 5. 🔄 与搜广推序列建模的异同

| 维度 | 金融时序 | 搜广推序列 |
|------|----------|-----------|
| **数据** | 价格/成交量（连续值） | 用户行为（离散事件） |
| **信噪比** | 极低（接近随机游走） | 相对较高（用户行为有规律） |
| **平稳性** | 非平稳是常态 | 相对平稳（用户兴趣缓慢漂移） |
| **预测目标** | 收益率（回归） | 点击/转化（分类） |
| **对抗性** | 极强（市场博弈） | 较弱（用户不会故意欺骗模型） |
| **特征工程** | 技术指标 + 基本面 | 行为序列 + 画像 |
| **序列长度** | 几百到几千（日频） | 几十到几百（行为序列） |
| **共同模型** | Transformer, LSTM | Transformer (SASRec), GRU (GRU4Rec) |
| **核心挑战** | 过拟合噪声 | 冷启动 + 实时性 |

**共同经验**：
- 两个领域都证明了 Transformer 的强大，但都需要针对性改造
- 特征工程在两个领域都仍然极其重要，"端到端"并没有完全取代手工特征
- 在线学习 / 增量更新在两个领域都是实践中的关键问题

## 6. 实践建议

1. **先做 baseline**：简单的动量 / 均值回归策略，确保信号存在
2. **特征工程 > 模型复杂度**：好的特征 + XGBoost 通常优于差特征 + Transformer
3. **注意时序泄露**：交叉验证必须用 Purged K-Fold，不能随机打乱
4. **信噪比决定模型选择**：高频数据信噪比更高 → DL 更有用；日频 → ML 可能就够了
5. **集成方法**：多个弱信号组合比单一强信号更稳健
