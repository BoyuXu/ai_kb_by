# 5 道策略设计面试题

> 策略设计题是量化面试中区分度最高的题型。面试官要的不是一个完美策略，而是**结构化的思考框架**和**对实际问题的敏感度**。

---

## 题 1：设计一个动量策略

**题目**：请设计一个基于价格动量的股票交易策略，并描述你会如何回测它。

### 面试官期望

不是听你说"买涨的卖跌的"，而是看你能不能把一个模糊的想法变成可执行的系统。

### 答题框架

**1. 投资宇宙与数据**
- A 股全市场（剔除 ST、上市不满 1 年、停牌）
- 日频收盘价、成交量、市值数据
- 数据源：Wind / 聚宽 / 通达信

**2. 信号构造**
- 计算过去 N 天（如 60/120/240）的累积收益率
- 跳过最近 5 天（避免短期反转）
- 横截面排名：按动量值从大到小排序

**3. 组合构建**
- 多头：买入动量前 10% 的股票
- 空头（如果允许）：卖出动量后 10%
- 等权 or 市值加权
- 月度调仓（平衡交易成本和信号时效性）

**4. 风控规则**
- 单股仓位上限 5%
- 行业暴露约束（防止押注单一行业）
- 最大回撤止损线：组合回撤 15% 减仓至半仓

**5. 回测框架**
```
for each rebalance_date:
    1. 计算所有股票的动量信号
    2. 排名、筛选、构建目标组合
    3. 计算交易成本（双边千三 + 滑点）
    4. 记录持仓和净值
```

**6. 评估指标**
- 年化收益、Sharpe、最大回撤、Calmar
- 分年度表现（动量策略在不同市场环境差异大）
- 换手率和交易成本敏感度分析

### 关键得分点
- 提到"跳过最近 N 天"避免短期反转
- 主动提到交易成本和滑点的影响
- 提到行业中性化处理
- 讨论动量策略在 A 股的特殊性（散户主导、涨跌停）

### 常见错误
- 忽略交易成本（回测很美，实盘亏损）
- 不做行业中性化（其实在做行业轮动而非动量）
- 用未来数据（用当天收盘价买入）

## 题 2：如何避免回测过拟合

**题目**：你的回测 Sharpe 达到 3.0，你担心过拟合。有哪些方法可以检验和避免过拟合？

### 面试官期望

对"过拟合"有深刻理解，不只是知道概念，而是有具体的方法论。

### 答题框架：5 种方法详解

**方法 1：样本外测试（Out-of-Sample Test）**
- 将数据分为训练集（如 2010-2018）和测试集（如 2019-2023）
- 训练集上开发策略，测试集上验证
- **关键**：测试集只能看一次。如果看完测试集又回去调参，等于没有样本外

**方法 2：交叉验证（Purged K-Fold CV）**
- 普通 K-Fold 不能用于时序数据（信息泄露）
- Purged CV：在训练集和测试集之间留 gap（如 5 天）
- Embargo：测试集之后也留 gap，防止标签泄露
- 参考 de Prado《Advances in Financial Machine Learning》

**方法 3：多重检验校正**
- 如果你测试了 100 组参数，最好的那组 Sharpe 高是必然的
- Deflated Sharpe Ratio：考虑试验次数后的真实 Sharpe
- $SR^* = SR - \sqrt{\frac{V[\hat{SR}]}{2}} \cdot \text{erfc}^{-1}\left(\frac{2 \cdot N_{trials}}{1}\right)$
- 经验法则：真实 Sharpe ≈ 回测 Sharpe × 0.5

**方法 4：参数稳健性检验**
- 在最优参数附近扰动（如回看期从 60 天改为 50/70 天）
- 如果小幅参数变化导致结果剧变 → 过拟合
- 好策略应该有一个"参数高原"（plateau），而非尖峰

**方法 5：经济逻辑检验**
- 问自己：这个策略赚钱的经济逻辑是什么？
- 谁在亏钱？风险溢价来自哪里？
- 如果解释不了为什么赚钱，大概率是过拟合
- 反例：高频做市赚 bid-ask spread → 逻辑清晰，不太可能是过拟合

### 关键得分点
- 提到 Purged CV（不是普通 CV）
- 提到多重检验校正
- 将统计方法和经济直觉结合
- 提到 de Prado 的工作

### 常见错误
- 只说"用测试集验证"但没提到只能看一次
- 不知道时序数据不能用随机 K-Fold
- 忽略经济逻辑，纯做统计检验

## 题 3：设计一个配对交易策略

**题目**：请设计一个统计套利的配对交易策略，包括选股、信号生成和风控。

### 面试官期望

理解协整的概念，并能把统计理论转化为可交易的策略。

### 答题框架

**1. 配对选择**

- **候选池**：同行业（如银行板块：招行 vs 工行）、同产业链（上下游）
- **相关性初筛**：相关系数 > 0.8 的股票对（必要条件，非充分）
- **协整检验（核心）**：
  - Engle-Granger 两步法：对价差序列做 ADF 检验
  - Johansen 检验：多变量协整检验
  - p-value < 0.05 才保留

```python
from statsmodels.tsa.stattools import coint
score, pvalue, _ = coint(price_A, price_B)
```

**2. 价差建模**

- 回归：$P_A = \beta \cdot P_B + \alpha + \epsilon$
- 价差序列：$\text{spread}_t = P_{A,t} - \hat{\beta} \cdot P_{B,t}$
- Z-Score 标准化：$z_t = \frac{\text{spread}_t - \mu}{\sigma}$（滚动窗口计算 μ 和 σ）

**3. 交易信号**

| Z-Score | 动作 |
|---------|------|
| z > 2.0 | 做空 A，做多 B（价差偏高，预期回归） |
| z < -2.0 | 做多 A，做空 B |
| |z| < 0.5 | 平仓（价差回归到均衡） |
| |z| > 4.0 | 止损平仓（协整关系可能已破裂） |

**4. 仓位管理**
- 市值中性：A 和 B 的持仓市值相等
- 使用 β 调整：做空 1 股 A → 做多 β 股 B
- 每对占总资金不超过 10%

**5. 风控**
- 协整关系定期重检（如每月）：如果 p-value > 0.1 → 停止该配对
- 价差偏离极端值（z > 4）→ 止损，而非加仓
- 结构性变化检测：regime change（并购、政策变化）

### 关键得分点
- 区分相关性和协整（相关性不代表协整）
- 提到协整关系会随时间变化
- 止损机制（z > 4 要止损而非加仓）
- 知道用 β 做市值中性

### 常见错误
- 混淆相关性和协整
- 用固定窗口的 μ 和 σ，不做滚动更新
- 没有止损（"反正会回归的"——不一定）
- 忽略交易成本（配对交易交易频繁）

## 题 4：多因子选股模型的完整流程

**题目**：请描述一个多因子选股模型从因子挖掘到实盘的完整流程。

### 面试官期望

端到端的系统性思维，从数据到交易的全链路理解。

### 答题框架

**Step 1：因子候选池构建**
- 价值类：PE、PB、PCF、股息率
- 成长类：营收增速、利润增速、ROE 变化
- 动量类：过去 N 日收益率（跳过近期）
- 质量类：ROE、毛利率、资产负债率
- 技术类：换手率、波动率、成交量变化
- 另类：分析师预期调整、供应链数据、NLP 情绪

**Step 2：因子预处理**
1. **缺失值处理**：行业中位数填充 / 剔除
2. **去极值**：MAD（中位数绝对偏差）法，3 倍 MAD 之外 Winsorize
3. **标准化**：Z-Score 标准化（横截面）
4. **行业市值中性化**：对行业哑变量和 ln(市值) 做回归，取残差

**Step 3：单因子检验**
- **IC 分析**：因子值与下期收益的 Rank IC（Spearman 相关系数）
  - |IC| > 0.03 且方向稳定 → 有效因子
  - IC_IR = mean(IC) / std(IC)，IC_IR > 0.5 为好因子
- **分层回测**：按因子值分 5 层（或 10 层），看各层收益单调性
  - 理想状态：第 1 层和第 5 层收益差距大，且单调

**Step 4：因子合成**
- **等权合成**：简单但稳健
- **IC 加权**：按过去 N 期的 IC 均值加权
- **最优化**：最大化合成因子的 IC_IR（注意过拟合风险）
- **机器学习**：XGBoost/LightGBM 以因子为特征，下期收益为标签

**Step 5：组合构建**
- 按合成因子排名，选前 N 只股票
- 权重方案：等权 / 因子值加权 / 风险预算
- 约束条件：行业偏离 < 5%，个股上限 3%，换手率约束

**Step 6：回测验证**
- Purged K-Fold 交叉验证
- 分年度、分市场环境分析
- 交易成本敏感度分析
- 参数稳健性检验

**Step 7：实盘部署**
- 每日收盘后更新因子
- T+1 开盘执行调仓（A 股 T+1）
- 执行算法：TWAP/VWAP 减少冲击成本
- 实盘归因：实际收益 vs 回测收益的差异分析

### 关键得分点
- 因子预处理的四步曲（缺失值 → 去极值 → 标准化 → 中性化）
- 知道 IC 和分层回测是标配检验手段
- 讨论实盘和回测的差异
- 提到行业中性化的必要性

### 常见错误
- 跳过预处理直接做回测
- 不做行业中性化（很多因子的效果其实是行业效应）
- 用未来信息（因子计算用了当期财报）
- 不考虑交易成本和执行

## 题 5：如何评估一个策略的好坏

**题目**：同事给你一个策略的回测报告，年化 35%，Sharpe 2.5。你如何判断这个策略是否真的好？

### 面试官期望

批判性思维，不被漂亮的数字蒙蔽。

### 答题框架：多维度评估

**维度 1：回测可信度**

- **数据是否干净**：前复权 vs 后复权？分红除权处理？
- **是否有前视偏差**：用了当天数据交易？用了未来公布的财报？
- **是否有幸存者偏差**：数据中是否包含已退市的股票？
- **交易成本**：是否包含佣金（双边千三）、印花税（千一）、滑点？
- **流动性**：策略买入的股票是否真的能买到那么多？小盘股尤其注意

**维度 2：统计显著性**

- **回测时间长度**：只有 2 年的回测不可信，至少需要覆盖一个完整牛熊周期
- **Deflated Sharpe Ratio**：考虑了试验次数后的真实 Sharpe
- **Monte Carlo 模拟**：对收益序列做 Bootstrap，看 Sharpe 的置信区间
- **参数稳健性**：参数小幅变动后表现是否剧变

**维度 3：风险特征**

| 指标 | 关注点 |
|------|--------|
| 最大回撤 | 年化 35% 但最大回撤 50%？不可接受 |
| 回撤恢复期 | 最长多久没创新高？ |
| 收益分布 | 是否有厚尾？偏度是正还是负？ |
| 尾部风险 | CVaR 是否极端？ |
| 收益来源 | 大部分收益是否来自少数几天？ |

**维度 4：收益归因**

- **因子暴露分析**：用 Fama-French 五因子回归，看 α 是否显著
- **行业归因**：收益是来自选股还是行业配置？
- **时间归因**：收益是否集中在特定时期（如 2015 年牛市）？
- **如果 α 不显著**：这个策略只是在承担因子风险，不是真的有 Alpha

**维度 5：实操可行性**

- **容量**：策略能容纳多少资金？小盘股策略可能只能容纳 1000 万
- **换手率**：年化换手率 50 倍？交易成本会吞噬大部分收益
- **执行难度**：需要多快的交易系统？延迟要求？
- **监管约束**：策略是否符合合规要求？（如 A 股不能裸卖空）

**维度 6：经济直觉**

- 策略赚的是谁的钱？风险溢价 or 行为偏差 or 信息优势？
- 这个优势是否可持续？竞争者进入后是否会消失？
- 市场结构变化（如注册制、做空机制完善）是否会影响策略？

### 关键得分点
- 第一反应不是"好厉害"而是"我需要验证"
- 多维度评估，不只看收益和 Sharpe
- 提到因子归因（可能不是 Alpha 而是因子暴露）
- 讨论策略容量和实操可行性

### 常见错误
- 只看收益和 Sharpe，不看风险
- 不考虑回测可信度（前视偏差、幸存者偏差）
- 不做因子归因（误把 Beta 当 Alpha）
- 不考虑策略容量（回测赚钱但实盘根本买不到）
