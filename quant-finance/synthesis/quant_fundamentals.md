## 1. 金融市场结构

### 1.1 股票市场

| 市场 | 交易所 | 交易时间 | 特点 |
|------|--------|---------|------|
| A股 | 上交所(SSE)、深交所(SZSE)、北交所(BSE) | 9:30-15:00 | T+1、涨跌停、散户占比高 |
| 港股 | 港交所(HKEX) | 9:30-16:00 | T+0、无涨跌停、可做空 |
| 美股 | NYSE、NASDAQ | 9:30-16:00 ET | T+0、盘前盘后、做空便利 |

**A股三大交易所分工**：
- **上交所**：大盘蓝筹（沪深300权重股集中）、科创板（688开头，注册制）
- **深交所**：中小盘成长股、创业板（300开头，注册制）
- **北交所**：创新型中小企业，流动性较差，量化策略慎入

> 面试常问：A股市场和美股市场的核心差异是什么？对量化策略有什么影响？→ 见第5节详细对比

### 1.2 期货市场

期货 = 标准化合约，约定未来某时以特定价格交割某资产。

**商品期货 vs 金融期货**：

| 类型 | 交易所 | 代表品种 | 保证金率 | 量化适用场景 |
|------|--------|---------|---------|------------|
| 商品期货 | 上期所/大商所/郑商所/广期所 | 螺纹钢、铜、原油 | 8%-15% | CTA趋势跟踪、套利 |
| 股指期货 | 中金所 | IF(沪深300)、IC(中证500)、IM(中证1000) | 12%-15% | 对冲、Alpha策略 |
| 国债期货 | 中金所 | T、TF、TS | 2%-3% | 利率套利、久期管理 |

**保证金交易的杠杆效应**：
- 保证金率 10% → 杠杆 10x
- 100万本金可控制 1000万名义价值的合约
- 这意味着：5% 的价格波动 = 50% 的盈亏，风控极其重要

### 1.3 期权市场

期权 = 权利（非义务）在未来以约定价格买入/卖出标的资产。

**核心概念四要素**：
- **认购(Call)** vs **认沽(Put)**：看涨买 Call，看跌买 Put
- **欧式** vs **美式**：欧式只能到期日行权，美式随时可行权（A股 ETF 期权为欧式）
- **行权价(Strike)**：合约约定的交易价格
- **到期日(Expiry)**：合约失效日期

**期权链(Option Chain)**：同一标的、同一到期日下不同行权价的全部合约。量化用途：波动率曲面建模、Delta对冲、波动率套利。A股主要品种：50ETF/300ETF/500ETF/创业板ETF期权。

### 1.4 其他交易工具

| 工具 | 特点 | 量化策略应用 |
|------|------|------------|
| **ETF** | 跟踪指数，T+1，可融资融券 | 套利（ETF vs 成分股）、资产配置 |
| **LOF** | 可场内场外交易 | 折溢价套利 |
| **可转债** | 债+期权，T+0，无涨跌停（2022年后设20%/30%限制） | 转股套利、低价策略 |

🔄 **与搜广推的联系**：金融市场的「价格发现」机制本质上是一个多方博弈的排序问题，类似于广告竞价中的 GSP（Generalized Second Price）拍卖。理解市场微观结构，有助于理解 [[concepts/multi_objective_optimization]] 中的博弈均衡思想。

## 2. A股交易机制

### 2.1 T+1 交割制度

**规则**：当日买入的股票，最早次一交易日才能卖出。

**对量化策略的深远影响**：
- **日内策略受限**：不能像美股那样日内反复买卖同一只股票
- **隔夜风险**：所有策略都承担至少一个隔夜的持仓风险
- **对冲方式**：需要用期货（T+0）或期权对冲日内风险
- **T+0 替代方案**：可转债 T+0、ETF 融券 T+0（成本高）、底仓 T+0（持有底仓后日内高抛低吸）

```python
# 底仓T+0：持有底仓，日内先卖后买回（卖的是昨日底仓，当日可卖）
def t0_with_base(base_shares, signals):
    """signals: [(timestamp, signal(+1/-1), price), ...]"""
    trades, pos = [], base_shares
    for ts, sig, px in signals:
        if sig == -1 and pos >= 100:
            trades.append(('sell', px, 100)); pos -= 100
        elif sig == 1:
            trades.append(('buy', px, 100)); pos += 100
    return trades  # 日终 pos 应等于 base_shares
```

> 面试常问：解释下A股的T+1制度对量化策略的影响？→ 答题要点：日内策略受限、隔夜风险、底仓T+0变通、期货对冲。

### 2.2 涨跌停制度

| 板块 | 涨跌停幅度 | 上市前5日 |
|------|----------|----------|
| 主板（沪深） | ±10% | 正常 |
| 科创板 | ±20% | 不设涨跌停 |
| 创业板 | ±20% | 不设涨跌停 |
| ST / *ST | ±5% | — |
| 北交所 | ±30% | 不设涨跌停 |

**量化策略注意事项**：
- 涨停 = 买不进，跌停 = 卖不出 → 流动性风险
- 一字涨停/跌停 = 完全无法交易
- 回测中如果不处理涨跌停，收益会严重高估（买入涨停股 = 实际不可能）

```python
def apply_price_limit(df, board='main'):
    """标注涨跌停，df 需含 close/pre_close 列"""
    limit = {'main': 0.10, 'gem': 0.20, 'star': 0.20, 'st': 0.05}[board]
    pct = df['close'] / df['pre_close'] - 1
    df['can_buy'] = pct < limit - 0.001   # 涨停不可买
    df['can_sell'] = pct > -limit + 0.001  # 跌停不可卖
    return df
```

### 2.3 集合竞价与连续竞价

**开盘集合竞价（9:15-9:25）**：
- 9:15-9:20：可申报、可撤单（试探阶段）
- 9:20-9:25：可申报、不可撤单（真实意图）
- 9:25 确定开盘价：使成交量最大的价格

**收盘集合竞价（14:57-15:00）**：
- 不可撤单，最后3分钟的订单统一撮合确定收盘价
- 量化策略常在 14:56 前完成下单，避免收盘集合竞价的不确定性

**连续竞价（9:30-11:30, 13:00-14:57）**：
- **价格优先**：买方出价高优先，卖方出价低优先
- **时间优先**：同价格按时间排序

🔄 **与搜索排序的类比**：集合竞价本质是一个批量排序问题——给定所有买卖订单，找到使成交量最大的价格。这类似于搜索引擎中的查询-文档匹配，都是在约束条件下寻找最优分配方案。

### 2.4 最小报价单位

- A股：0.01 元（1分钱）
- 期货：因品种而异（螺纹钢1元/吨，股指期货0.2点）
- 对高频策略而言，tick size 决定了最小利润空间

## 3. 订单类型

### 3.1 基本订单类型

| 订单类型 | 英文 | 特点 | A股支持 |
|---------|------|------|---------|
| 限价单 | Limit Order | 指定价格，不保证成交 | ✅ |
| 市价单 | Market Order | 立即成交，不保证价格 | ✅（上交所5种、深交所5种） |
| IOC | Immediate-or-Cancel | 立即成交否则撤单 | ✅（部分） |
| FOK | Fill-or-Kill | 全部成交否则全部撤单 | ✅（部分） |
| 冰山单 | Iceberg Order | 隐藏大部分数量 | ❌ A股不支持 |
| 隐藏单 | Hidden Order | 完全不显示在盘口 | ❌ A股不支持 |

**限价单 vs 市价单的取舍**：
- 限价单：控制成本，但可能不成交 → 适合不急的策略
- 市价单：保证成交，但可能滑点大 → 适合需要立即执行的信号

### 3.2 Python 模拟限价单撮合

```python
from dataclasses import dataclass
from typing import List, Tuple

@dataclass
class Order:
    side: str; price: float; quantity: int; timestamp: float

def match_limit_orders(buys: List[Order], sells: List[Order]) -> List[Tuple[float, int]]:
    """模拟限价单撮合引擎（价格优先+时间优先）"""
    buys = sorted(buys, key=lambda o: (-o.price, o.timestamp))
    sells = sorted(sells, key=lambda o: (o.price, o.timestamp))
    trades, bi, si = [], 0, 0
    while bi < len(buys) and si < len(sells):
        if buys[bi].price < sells[si].price:
            break
        tp = sells[si].price if sells[si].timestamp < buys[bi].timestamp else buys[bi].price
        tq = min(buys[bi].quantity, sells[si].quantity)
        trades.append((tp, tq))
        buys[bi].quantity -= tq; sells[si].quantity -= tq
        if buys[bi].quantity == 0: bi += 1
        if sells[si].quantity == 0: si += 1
    return trades
```

## 4. 交易费用与成本

### 4.1 A股交易费用明细

| 费用项 | 费率 | 方向 | 备注 |
|-------|------|------|------|
| 券商佣金 | 万2-万3（最低5元） | 双向 | 机构可谈到万1甚至更低 |
| 印花税 | 千1（0.1%） | 仅卖出 | 2023年8月降至千0.5 |
| 过户费 | 万0.1 | 双向 | 沪深统一 |
| 经手费 | 万0.341 | 双向 | 含在佣金中 |

**单次交易总成本估算**（万2.5佣金 + 千0.5印花税）：
- 买入成本 ≈ 0.025% + 0.001% = 0.026%
- 卖出成本 ≈ 0.025% + 0.05% + 0.001% = 0.076%
- 单次往返 ≈ 0.1%

### 4.2 滑点 (Slippage)

滑点 = 实际成交价 - 预期成交价

产生原因：行情延迟、订单簿深度不足、市价单直接吃对手方。经验值：大盘股 1-2 bp，小盘股 2-5 bp（1bp = 0.01%），涨跌停附近可能无穷大。

### 4.3 市场冲击成本 (Market Impact)

> 面试常问：滑点和冲击成本的区别？→ 滑点是单笔订单的价差，冲击成本是大额交易对市场价格的持续影响。滑点是微观的，冲击是宏观的。

**市场冲击模型（平方根模型）**：

$$Impact = \sigma \cdot \sqrt{\frac{V_{order}}{V_{daily}}}$$

- $\sigma$：标的日波动率
- $V_{order}$：订单量
- $V_{daily}$：日均成交量

```python
import numpy as np

def market_impact(order_val, daily_vol, daily_sigma, coeff=1.0):
    """平方根冲击模型：Impact = sigma * sqrt(V_order / V_daily)"""
    return coeff * daily_sigma * np.sqrt(order_val / daily_vol)

# 例：100万订单，日成交1亿，日波动率2% → 冲击 ≈ 0.20%
print(f"{market_impact(1e6, 1e8, 0.02)*100:.2f}%")
```

### 4.4 计算策略净收益

```python
def calculate_net_returns(gross_returns, turnover,
                          commission=0.00025, stamp_tax=0.0005,
                          transfer_fee=0.00001, slippage_bps=2.0):
    """扣除交易成本后的策略净收益（gross_returns/turnover 为 pd.Series）"""
    cost_rate = commission * 2 + stamp_tax + transfer_fee * 2 + slippage_bps / 10000 * 2
    trading_cost = turnover * cost_rate
    net = gross_returns - trading_cost
    print(f"往返成本: {cost_rate*100:.3f}% | 年化净收益: {net.mean()*252*100:.1f}%")
    return net
```

## 5. A股 vs 美股核心差异

### 5.1 交易规则对比

| 维度 | A股 | 美股 |
|------|-----|------|
| 交割制度 | T+1 | T+0（日内无限次） |
| 涨跌停 | 有（±10%/±20%） | 无（有熔断机制） |
| 做空机制 | 融券（门槛高、券源少） | 做空便利（券源充足） |
| 交易时间 | 4小时 | 6.5小时 + 盘前盘后 |
| 最小交易单位 | 100股（1手） | 1股 |
| 订单类型 | 较少 | 丰富（冰山单、止损单等） |
| 散户占比 | ~60%交易量 | ~20%交易量 |
| 量化占比 | ~30% | ~60-70% |

### 5.2 对量化策略选择的影响

**A股量化特点**：
- **以多头为主**：做空困难 → 量化多头（买入优质股票）是主流
- **日内策略受限**：T+1 → 纯日内策略需借助期货/可转债
- **散户博弈**：散户占比高 → 行为金融因子（反转、低波动）更有效
- **政策驱动**：监管政策对市场影响大 → 事件驱动策略有空间

> 面试常问：为什么A股量化策略以多头为主？→ 因为融券做空门槛高、券源有限、费率贵（年化8%-10%），且有融券限制（如2023年融券新规），导致多空策略难以实施。

**美股量化特点**：
- **多空均衡**：做空方便 → 经典多空策略可直接实施
- **竞争激烈**：量化占比高 → Alpha 衰减快
- **数据丰富**：另类数据（卫星、社交媒体）更成熟
- **高频可行**：T+0 + 丰富订单类型 → 高频/做市策略空间大

### 5.3 数据获取差异

- **A股**：万得(Wind)、聚宽、Tushare（行情）；万得、东方财富（财务）；Level-2 付费
- **美股**：Yahoo Finance、Polygon、Bloomberg（行情）；SEC EDGAR、Compustat（财务）；另类数据丰富（卫星图、信用卡消费等）

## 6. 面试高频考点汇总

> 面试常问：解释下A股的T+1制度对量化策略的影响？

答：(1) 日内策略受限 (2) 隔夜风险需期货对冲 (3) 变通：底仓T+0、可转债T+0 (4) A股量化以日频及以上为主。

> 面试常问：滑点和冲击成本的区别？

答：滑点是微观的单笔价差（延迟+订单簿深度），冲击成本是宏观的大额交易对价格的持续影响（含信息泄露成本）。冲击包含滑点。

> 面试常问：为什么A股量化策略以多头为主？

答：融券门槛高（50万起、券源少、年化8%+）+ 监管限制（2023融券T+1新规）→ 中性策略用股指期货对冲而非融券 → 多头端资金拥挤。

🔄 **跨领域思考**：金融市场中的「流动性」概念类似于推荐系统中的「曝光不足」问题——都是供需匹配不均衡导致的效率损失。理解 [[concepts/embedding_everywhere]] 中的向量空间匹配，有助于理解订单簿中买卖匹配的本质。
