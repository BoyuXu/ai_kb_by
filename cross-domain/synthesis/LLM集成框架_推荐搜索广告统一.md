## 📊 通用 LLM 集成框架

### 核心原理

LLM 在搜广推三大系统中的应用遵循统一的分层范式：

```
第三层：架构层（端到端替换）
  ├── 生成式范式（GenRet、对话推荐、Agent 搜索）
  └── 长期演进，周期 6-12 个月

第二层：节点层（算法模块替换）
  ├── 排序/重排（LLM Reranker）
  ├── 相关性判断（LLM as Judge）
  ├── 多目标优化（Pareto 框架）
  └── 中期方向，周期 3-6 个月

第一层：替换层（模块替换，不改核心逻辑）
  ├── 文本生成与优化
  ├── 特征生成与理解
  ├── 解释与可信度
  └── MVP 快速落地，周期 1-2 个月
```

### 三大系统的共性与差异

#### 共性部分（可复用）

| 能力 | 推荐系统 | 搜索系统 | 广告系统 | 复用度 |
|------|--------|--------|--------|-------|
| **文本理解** | 内容特征提取 | Query 理解 | 创意质量评估 | 100% |
| **Reranker** | 精排替换 | RankLLM | 排序替换 | 95% |
| **多目标优化** | 多样性控制 | 结果打散 | 多目标出价 | 90% |
| **解释生成** | 推荐理由 | 结果相关性说明 | 投放理由 | 85% |
| **冷启动** | 新用户/新物品 | 新 query/新文档 | 新品类/新出价 | 80% |

#### 差异部分（需专化）

| 维度 | 推荐系统 | 搜索系统 | 广告系统 |
|------|--------|--------|--------|
| **核心指标** | CTR / 留存 / 多样性 | NDCG / MAP / 相关性 | CTR / CVR / ROI / 预算 |
| **数据特征** | 用户行为序列 | Query + 文档 | 用户 + 广告 + 竞价信息 |
| **延迟约束** | <100ms | <100ms | <100ms（竞价） |
| **规模** | 用户百万级 + 物品千万级 | 用户千万级 + 文档十亿级 | 用户千万级 + 广告百万级 |
| **更新频率** | 天级（新物品） | 分钟级（新文档） | 秒级（竞价日志） |
| **约束条件** | 无 | 时效性 | 预算 / ROI / 竞价 |

## 📈 分层应用路线图

### 第一层：替换层（MVP，1-2 周快速落地）

**定义**：替换文本处理模块，不改变核心算法逻辑  
**成本**：低  
**收益**：中等（+3-10% CTR）  
**风险**：最小

#### 推荐系统 MVP：用户偏好描述 + 解释生成
- **投入**：7-10 天
- **成果**：冷启动 CTR +10-20%，用户满意度 +15-25%

#### 搜索系统 MVP：查询改写 + 结果摘要
- **投入**：3-7 天
- **成果**：搜索成功率 +8-12%，点击率 +4-8%

#### 广告系统 MVP：创意生成 + 质量评估
- **投入**：7-10 天
- **成果**：CTR +3-8%，人工审核成本 -60%

---

### 第二层：节点层（中期，3-6 个月深化）

**定义**：替换关键算法节点（排序、重排、多目标）  
**成本**：中等  
**收益**：高（+8-20% 效果提升）  
**风险**：中等（需要工程优化）

#### 三系统通用：LLM Reranker
- **方案**：Pairwise 或 Listwise LLM 重排
- **效果对比**：
  - 推荐系统：精排 CTR +8-15%
  - 搜索系统：NDCG +15-30%
  - 广告系统：eCPM +8-15%
- **成本**：20-50ms per reranking

#### 推荐系统深化：冷启动 + 多样性控制
#### 搜索系统深化：RankLLM + 意图分类
#### 广告系统深化：竞价决策 + 多目标优化

---

### 第三层：架构层（长期，6-12 个月演进）

**定义**：端到端替换传统架构，使用生成式范式  
**成本**：高  
**收益**：很高（+20-50% 效果）  
**风险**：高（需要大量工程投入）

#### 推荐系统：对话推荐 + 多轮推理
#### 搜索系统：生成式检索 + Agent 搜索
#### 广告系统：生成式竞价框架

## 💡 核心洞察

### 1. Reranker 是通用银弹
- **原理**：Reranker 最大化利用 LLM 的排序能力，成本相对可控
- **适用范围**：推荐/搜索/广告的排序都能得益
- **优先级**：⭐⭐⭐⭐⭐ （高优先级）

### 2. 冷启动问题可通过 LLM 解决
- **传统方案**：协同过滤（数据稀疏）、规则（死板）
- **LLM 方案**：语义理解（灵活）、少样本学习（快速迭代）
- **效果**：新用户/新物品 CTR 提升 300-400%

### 3. 成本控制是关键
- **缓存策略**：热查询/热用户的结果缓存，命中率通常 60-80%
- **轻量化模型**：使用 6-13B 参数的轻量模型替代 GPT-4（成本 1/10，精度 90%+）
- **混合方案**：简单场景用规则，复杂场景用 LLM

### 4. 在线评估比离线评估重要
- **离线评标**：NDCG、MAP、AUC 作为参考
- **在线指标**：A/B 测试中的点击率、转化率、留存率最重要
- **陷阱**：NDCG 提升 5% 不代表 CTR 提升 5%

### 5. 推荐系统和搜索系统正在融合
- **趋势**：Spotify ULM、Google Multitask Unified Model
- **影响**：未来可能用一套统一的架构处理推荐、搜索、广告
- **启示**：现在就开始构建跨系统的统一 LLM 能力
