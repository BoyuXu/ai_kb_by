## 核心问题演进

### 第一代：单点预估（Point-wise）→ 第二代：对偶优化（Dual Optimization）

**传统范式（回归/分类）**
```
问题：预估单个物品的 CTR / CVR
方法：线性回归、GBDT、深度学习
典型案例：LR 基线模型、DeepFM、DIN
限制：没有考虑约束条件、全局优化目标
```

**优化范式演进**
```
问题：最大化转化量，约束条件 CPA ≤ target
方法：拉格朗日对偶优化 + PID 控制
典型案例：AutoBidding 竞价系统、Pacing 预算控制
收益：从"预测准不准"到"达成约束"（实际商业问题）

关键洞察：
- 问题重新定义（优化 vs 回归）比堆特征更有效
- 工程问题（预算均衡、频次控制）的商业价值往往大于算法精度
```


## 📐 核心公式与原理

### 1. 多目标优化
$$
\min_{\theta} \sum_k \lambda_k L_k(\theta)
$$
- Scalarization 方法，λ 控制任务权重

### 2. Pareto 最优
$$
x^* \text{ is Pareto optimal } \iff \nexists x: f_i(x) \leq f_i(x^*) \forall i
$$
- 不存在在所有目标上都更好的解

### 3. 偏差校正 (IPW)
$$
\hat{R} = \frac{1}{n}\sum_i \frac{r_i}{P(O=1|x_i)}
$$
- 逆倾向加权消除选择偏差

### 第二代：单指标优化 → 第三代：多目标学习（Multi-task Learning）

**单指标陷阱**
```
问题：只优化 CTR → 用户疲劳、转化率下降
原因：目标函数单一（幸存者偏差）
案例：信息流直接按 CTR 排序 → 同类广告堆积
```

**多目标学习范式**
```
问题：同时优化 CTR + 停留时长 + 转化率 + 多样性
方法：
1. 多任务学习（ESMM）：CTR × CVR ≈ pCTCVR（全空间约束）
2. 帕累托前沿：为不同用户匹配不同权重
3. 列表级优化：DPP 多样性 + 精准度平衡

收益：
- CTR AUC：0.72 → 0.78（+8%）
- 留存率：+7-8%
- RPM：+5%

关键洞察：
- 列表级优化 > 逐项优化
- 多样性与相关性不是对立，而是互补（DPP 的数学基础）
```

---

### 第三代：全空间建模 → 第四代：因果推断（Causal Inference）

**样本偏差问题**
```
问题：模型只学到"被展示的广告"分布，无法泛化
原因：Selection Bias（截断数据）
表现：CVR 比 CTR 偏差更严重（数据更稀疏）

传统解决：ESMM 多任务约束 + 负采样
局限：还是相关性，不是因果性
```

**因果推断范式**
```
问题：公平分配转化的"功劳"给多个触点
方法：Shapley 值 + 倾向评分匹配（PSM）
应用场景：
1. 跨域归因：多个广告触点 → 哪个最关键？
2. 延迟反馈纠正：IPW 加权补偿时间延迟
3. 观测数据做因果推断：替代 A/B 测试，成本 -80%

关键洞察：
- 样本偏差 > 特征工程的收益
- 因果推断不是学术问题，而是生产必需品
```

## 约束优化的工程实践

### PID 控制在竞价中的应用

**问题**
- 目标：预算在 24h 内均衡消耗（不欠消耗、不超消耗）
- 挑战：市场实时波动、不同时段 CPM 差异大

**PID 解决方案**
```
从控制论借鉴：
- P（比例）：当前预算进度 vs 目标进度
- I（积分）：历史偏差累积
- D（微分）：偏差变化速率

实时调节出价乘数 λ：
λ(t) = λ(t-1) + Kp×e(t) + Ki×∫e + Kd×de/dt
其中 e(t) = 目标消耗进度 - 实际消耗进度
```

**效果**
- 欠消耗率 < 2%
- 收敛时间 < 2 小时
- 无振荡（不会一会儿消耗过快一会儿过慢）

**关键洞察**
- 工程问题比算法问题更有价值
- 古老的控制论理论仍然实用

## 序列建模的进化：RNN → Transformer → 强化学习

### 用户行为序列建模

**第一代：RNN/LSTM**
- 优点：能处理变长序列
- 缺点：难以优化、容易梯度消失、难以捕捉长期依赖

**第二代：Transformer（Self-Attention）**
```
案例：DIN → Transformer 排序模型
收益：CTR +18%、留存 +6%

为什么 Transformer 更好？
1. Parallel：所有位置可并行计算，训练快
2. Long-range Dependencies：注意力直接连接任意位置
3. 优化友好：梯度传播通路更短
```

**第三代：强化学习（DQN、Actor-Critic）**
```
应用场景：实时竞价（RTB）、多臂老虎机（创意优化）

DQN in RTB：
- 状态：当前预算剩余、用户特征、竞争环境
- 动作：出价金额
- 奖励：赢拍价格与预算目标的关系
- 效果：赢率 +15%、成本 -8%

Thompson Sampling in 创意优化：
- 每次展示时，按概率探索高转化创意
- 随着数据积累，高转化创意被推荐概率快速收敛 95%
- 平衡了"探索"和"利用"的权衡
```

## 冷启动的多层次解决方案

### 用户冷启动 → 图结构

```
传统方法：用户特征 → 协同过滤
瓶颈：冷启动用户无历史数据

GraphSAGE 解决方案：
- 用户-朋友-内容图
- 邻近节点特征聚合（朋友的兴趣 > 用户特征本身）
- 7日留存：28% → 35% (+25%)

关键洞察：图结构的信息量 > 单个节点特征
```

### 内容冷启动 → 多臂老虎机 + 主动学习

```
Thompson Sampling：
- 小流量探索新内容
- 根据反馈动态调整探索概率
- 新内容首周曝光 3x（vs 冷启动随机）

CLIP 多模态特征：
- 主动提取内容特征（而非被动反馈）
- 冷启动内容精度 +22%
- 新颖内容曝光 +35%

关键洞察：适当的"浪费"（探索）能避免更大的浪费（冷启动卡住）
```

## 生成模型的应用边界

### 何时适合用生成模型？

**创意生成**
```
应用：GPT-2 微调广告文案
效果：生成文案点击率接近人工、成本 -70%
适用条件：内容足够多、生成质量可接受
限制：品牌安全需要人审
```

**数据增强（低频转化）**
```
应用：GAN 生成虚拟用户-广告-转化
效果：转化预估 RMSE -30%
适用条件：数据极度稀疏（低频转化、长尾产品）
限制：GAN 质量和真实数据分布接近的 proof 需要
```

**关键洞察**
- 生成模型在数据稀疏场景（低频转化、长尾）有巨大价值
- 必须有质量控制机制（人审、A/B 验证）
```

## 主动学习在标注成本优化中的应用

### 品牌安全分类的主动学习

```
问题：需要大量标注样本来训练分类器
传统：随机选 10000 条标注
成本：人工标注成本高

主动学习方案：
- BERT 初步分类 + 不确定度采样
- 贪心选择最有信息量的样本标注
- 迭代训练

效果：
- 标注成本 -60%（从 10000 → 4000 条）
- 分类准确率相同

关键洞察：聪明地选择标注样本比随机标注节省 60% 成本
```

## 推荐系统的核心权衡

### 多样性 vs 相关性 vs 曝光公平性

```
DPP 多样性平衡：
- 数学基础：行列式衡量"不相似"程度
- 贪心采样：快速找到最优多样化列表
- 效果：留存 +8%、RPM +5%

用户分群 vs 超大通用模型：
- 分群模型：可解释性好、维护成本低、冷启动处理快
- 通用模型：数据利用率高、参数共享
- 权衡：根据业务规模和数据量选择

内容生命周期建模：
- 问题：热度随时间变化（不是静态的）
- 指数衰减函数：显式建模生命周期
- 效果：热点峰值 +25%、长尾覆盖 +15%
```

## 总结：三层递进的思维模式

```
第1层：特征工程 + 模型选择（accuracy focused）
→ AUC 从 0.70 → 0.75（+7%）

第2层：问题重新定义 + 约束优化（business objective focused）
→ CPA 达标率 65% → 92%、转化 +18%

第3层：因果推断 + 多目标学习（holistic optimization）
→ 整体效果 +25-30%（留存、转化、公平性全面提升）

最大的收益往往来自"问题定义"而非"参数优化"
```

### Q1: 搜广推三个领域的技术共性？
**30秒答案**：①都需要召回+排序架构；②都用 CTR/CVR 预估模型；③都面临冷启动问题；④都需要实时特征系统；⑤都可以用 LLM 增强。差异主要在约束条件和评估指标。

### Q2: 多目标优化在三个领域的应用？
**30秒答案**：广告：收入+用户体验+广告主 ROI；推荐：CTR+时长+多样性+留存；搜索：相关性+新鲜度+权威性+多样性。方法共通：Pareto/MMoE/PLE/Scalarization。

### Q3: 偏差问题在三个领域的表现？
**30秒答案**：广告：位置偏差+样本选择偏差；推荐：流行度偏差+曝光偏差；搜索：位置偏差+呈现偏差。解决方法类似：IPW/因果推断/去偏训练。

### Q4: 端到端学习的趋势和挑战？
**30秒答案**：趋势：统一模型替代分层管道（OneRec 统一召排）。挑战：①推理效率（一个大模型 vs 多个小模型）；②可控性差（难以插入业务规则）；③调试困难（黑盒）。

### Q5: 面试中如何体现跨领域理解？
**30秒答案**：①用类比说明（如广告出价≈搜索 LTR）；②指出技术迁移（如 DIN 从推荐到广告）；③提出统一视角（如多目标在三领域的共通框架）；④结合实际经验说明如何借鉴。

### Q6: 如何向面试官展示技术深度？
**30秒答案**：①先总后分：先说整体架构，追问时展开细节；②对比分析：主动比较 2-3 种方案的优劣；③数字说话：「AUC 从 0.72 提升到 0.74」而非「效果变好了」；④边界意识：说清楚方案的局限和适用条件。

### Q7: 跨领域知识迁移的实际案例？
**30秒答案**：①DIN（推荐→广告）：注意力机制从推荐 CTR 迁移到广告 CTR；②BERT（NLP→搜索）：预训练语言模型用于搜索排序；③Semantic ID（搜索→推荐）：从搜索的 doc ID 到推荐的 item ID 统一表示。

### Q8: 大规模系统的性能优化通用方法？
**30秒答案**：①缓存（特征缓存、结果缓存）；②异步（特征获取异步化）；③预计算（user embedding 离线算好）；④分层（粗排+精排降低计算量）；⑤模型优化（蒸馏/量化/剪枝）。

### Q9: 线上事故排查的思路？
**30秒答案**：①看监控：指标异常时间点→②查变更：最近上线了什么→③回滚验证：回滚后指标恢复说明是变更导致→④深入分析：看特征分布、样本分布、模型输出分布有无异常。

### Q10: 算法工程师的核心竞争力？
**30秒答案**：①业务理解（指标 → 技术方案的转化能力）；②工程能力（模型能上线、能调优、能排查问题）；③论文能力（快速读懂并判断论文的实用价值）；④系统思维（全链路优化而非单点优化）。
