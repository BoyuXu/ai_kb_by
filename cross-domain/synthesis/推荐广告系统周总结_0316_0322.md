## 📐 核心公式与原理

### 1. 多目标优化
$$
\min_{\theta} \sum_k \lambda_k L_k(\theta)
$$
- Scalarization 方法，λ 控制任务权重

### 2. Pareto 最优
$$
x^* \text{ is Pareto optimal } \iff \nexists x: f_i(x) \leq f_i(x^*) \forall i
$$
- 不存在在所有目标上都更好的解

### 3. 偏差校正 (IPW)
$$
\hat{R} = \frac{1}{n}\sum_i \frac{r_i}{P(O=1|x_i)}
$$
- 逆倾向加权消除选择偏差

## 一、推荐系统本周主题

### 主题1：生成式推荐（Generative Recommendation）— 本周最热趋势

**核心 insight：**
- 推荐正在从"判别式（给 user-item pair 打分）"转向"生成式（自回归生成下一物品的 token 序列）"，代表性工作：ActionPiece、DiffGRM、GEMs、TIGER、Web-scale LLM Rec
- **Semantic ID 是关键中间层**：物品 → RQ-VAE/BSQ 量化 → 多级 token → 语言模型处理，ActionPiece 的创新在于引入上下文感知 tokenization，同一物品在不同行为序列中有不同 token
- **DiffGRM 的反叛**：用扩散模型替代自回归，解决 AR 模型单向依赖问题；GEMs 用多流 Decoder 打破 O(n²) 长序列瓶颈（>1000 items）
- 工程落地挑战：生成式推荐延迟高（AR 逐 token 生成），需要 speculative decoding 或 parallel decoding 配合

**演进脉络：**
`CF/MF（2010s）→ DNN（DIN/DIEN 2018-2019）→ Transformer 序列建模（SASRec 2018）→ 对比学习（2021-2022）→ 生成式推荐（TIGER 2023→ActionPiece/DiffGRM 2025-2026）`

---

### 主题2：多任务学习 & 联合建模

**核心 insight：**
- MMoE → PLE → STAR（多场景）是工业界多任务架构三代演进
- **联合建模 Survey（2502.21195）** 给出完整分类体系：多任务联合（CTR+CVR）、多阶段联合（召回+排序端到端）、多模态联合（文本/图像/行为）
- 关键痛点：**目标不一致（Objective Misalignment）**——召回优化 recall@K，精排优化 AUC，重排优化多样性，三者 gradient 方向可能冲突
- 工业落地方向：共享 Transformer encoder + 任务特定轻量 head，减少重复算力

---

### 主题3：长序列用户行为建模

**核心 insight：**
- 用户行为序列从 50→500→5000+ items 的趋势不可阻挡，但 O(n²) attention 是性能杀手
- 解法体系：
  1. **稀疏 attention**（SIM 两阶段检索、ETA hash 检索）
  2. **低秩压缩**（HSTU Hourglass）
  3. **多流 Decoder**（GEMs：不同粒度的兴趣流并行处理，最后融合）
- SS4Rec 探索了 SSM（Mamba）用于序列推荐，O(n) 复杂度，但对随机访问模式的推荐效果还不及 Transformer

---

### 主题4：公平性 & 多目标重排

**核心 insight：**
- 准确性-公平性是 Pareto 前沿上的 trade-off，Elasticity-based 框架量化每篇文章"让步公平"的代价
- Jensen Gap 是理论工具：验证多目标优化时各子目标是否存在不可弥补的 gap
- 工程实践：重排阶段引入 Diversity、Fairness 约束，通常用 MMR 或 DPP（行列式点过程）

---

### 主题5：受众扩展 & 跨域推荐

**核心 insight：**
- 极端多分类（Extreme Multi-label Classification）是受众扩展的新范式：将"找相似用户"转化为对数亿用户标签的联合分类，精度显著超过双塔 cosine 方法
- VARK 解决 LLM 推荐在数据稀疏场景的问题，通过 LLM 生成的虚拟行为增广（Virtual Augmentation + Retrieval + Knowledge）

## 二、广告系统本周主题

### 主题1：一价拍卖（First-Price Auction）下的自动出价

**核心 insight：**
- 2019 年 Google AdX 切换到 FPA 是行业里程碑，出价策略从 truthful bidding 变成需要学习竞争对手分布的博弈问题
- **No-Regret Online Autobidding**：在预算约束+ROI 约束下，设计 Regret = O(√T) 的在线算法（Primal-Dual + 梯度下降），理论优雅但实际需要处理分布偏移
- **Joint Value Estimation and Bidding**：将价值估计和出价联合优化（只观测赢/输结果，利用 partial feedback 的贝叶斯更新），解决了单独学习 CVR 和出价的 inconsistency 问题
- **非平稳性是核心难点**：竞争对手策略变化、季节性流量波动，需要动态 bid shading + 快速适应机制

**演进脉络：**
`二价拍卖 Truthful Bidding → GSP（关键词广告）→ VCG 理论最优 → First-Price（2019+）→ No-regret Autobidding → RL/Diffusion Autobidding（2024-2026）`

---

### 主题2：预算节奏控制（Budget Pacing）

**核心 insight：**
- Budget Pacing 是每个广告系统必须有的基础设施，本质是**随机控制问题**：在全天各时段合理分配预算消耗
- 经典方法：Throttling（概率性跳过竞价）+ PID 控制器（根据消耗率调节出价系数）
- **AdaptableBudgetPlanner**：用 RL 预测预算消耗曲线，提前规划 pacing 策略，相比被动 PID 减少 under-delivery 20%+
- 大规模挑战：一个 DSP 管理数百万 campaign，需要轻量级分布式 pacing，不能每个 campaign 都跑复杂 RL

---

### 主题3：CTR/CVR 预估经典模型回顾 + 工业新进展

**核心 insight：**
- **DIN（2018，Alibaba KDD）**：用 target-aware attention 替代 fixed pooling，是兴趣建模的奠基之作；面试必考，核心是"动态兴趣"而非"静态画像"
- **ESMM（Entire Space Multi-task Model）**：解决 CVR 稀疏问题——在 impression 空间估计 pCTCVR，分解为 pCTR × pCVR，利用 CTR 大量样本辅助 CVR 训练
- **EGA v2 / MultimodalCTR**：多模态特征（图像 + 文本 + 行为）联合 CTR 预估，视觉特征对 native ads 提升显著（+1.5% AUC）
- **LLM-enhanced CTR**：用 LLM 生成语义特征 embedding 作为补充输入，解决稀疏 ID 特征的冷启动，但 serving latency 是瓶颈

---

### 主题4：生成式出价 & Diffusion 模型在广告中的应用

**核心 insight：**
- **DiffusionAutoBidding**：用扩散模型建模竞争对手出价分布，基于"完成-对齐"框架：先生成候选出价分布（Completer），再对齐广告主目标约束（Aligner）
- 创新点在于将**对抗性不确定性**（不知道竞争对手怎么出价）显式建模为扩散过程，而非简单假设高斯分布
- 工程可行性：diffusion 推理需要多步去噪，实时竞价毫秒级延迟下需要 DDIM 等加速采样（1-4 步）

---

### 主题5：多目标广告排序 & 反事实学习

**核心 insight：**
- 广告排序目标：平台收入 vs 用户体验（广告密度）vs 广告主 ROI，三者天然冲突
- **多目标帕累托**：用 Multi-gradient descent 或 Scalarization（加权和）在目标之间做 trade-off，需要业务方设定优先级
- **反事实学习（Counterfactual Learning for Unbiased Ranking）**：广告系统存在 position bias、selection bias，IPS（Inverse Propensity Scoring）是标准去偏方法，但 propensity 估计误差会放大方差
- LLM 用于广告创意生成（eBay Keyphrase + 淘宝商品文案）：用 LLM 生成多样化 ad copy，再用 relevance score 过滤，实测 CTR+3-8%

## 三、技术趋势：这些论文共同指向什么？

### 趋势1：生成式范式统一搜广推
推荐和广告都在向"生成"靠拢：推荐用 AR/Diffusion 生成物品 token，广告用 Diffusion 生成出价策略，搜索用 LLM 改写 query。**统一的生成式 backbone + 任务特定的约束层**是未来方向。

### 趋势2：LLM 从 feature encoder 升级为 reasoning engine
早期 LLM 在推荐/广告中只是替换 embedding 层（language feature），现在开始承担排序（Bing Web Rec）、创意生成（eBay/Taobao）、规则解释等 reasoning 任务。延迟和成本仍是最大障碍。

### 趋势3：强化学习 + 约束优化成为广告标配
RL 自动出价（预算约束 + ROI 约束 + 公平约束）相比人工规则提升明显，但需要仿真器（Simulator）解决样本效率问题；联合优化（联合学习价值估计 + 出价）比解耦学习更一致。

### 趋势4：长序列 + 多模态是推荐系统的双轮驱动
序列越来越长，物品模态越来越丰富（图、文、视频、行为），如何高效融合是工程核心问题。GEMs 的多流架构和 MultimodalCTR 的多模态融合分别代表这两个方向的前沿。

## 四、高频面试考点（直接可用）

**Q1: 请解释 oCPC/oCPA 的工作原理及其与传统 CPC 的区别**
A: 传统 CPC 由广告主手动设定出价，oCPC 由系统根据目标 CPA 自动调整出价，利用 CVR 预估动态计算 ecpm = pctr × pcvr × target_cpa。核心组件：CVR 预估模型（ESMM 解决稀疏问题）+ PID 控制器（根据实际 CPA 与目标偏差调整出价系数）。

**Q2: ESMM 为什么能解决 CVR 样本稀疏问题？**
A: ESMM 在全曝光空间（impression space）建模，分解 pCTCVR = pCTR × pCVR。CVR 子模型借用 CTR 的大量点击样本通过共享 embedding 学习，不再局限于稀少的转化样本。关键假设：CTR 和 CVR 共享用户行为表征。

**Q3: DIN 相比传统 DNN 的核心改进是什么？**
A: DIN 引入 target-aware attention：计算候选广告与历史行为 item 的相关性权重，加权 pooling 得到用户兴趣表征。传统 DNN 用固定 pooling，无法捕获与当前广告相关的局部兴趣。DIEN 进一步加入 GRU 建模兴趣演化，DSIN 建模会话级兴趣。

**Q4: 一价拍卖和二价拍卖的区别，以及对出价策略的影响？**
A: 二价拍卖（VCG/GSP）中最优策略是 truthful bidding（出自己真实估值），不需要考虑竞争对手。一价拍卖出价即支付，需要 bid shading（出低于真实估值的价格），最优出价取决于竞争对手出价分布。工业界 Autobidding 算法通过 RL 学习 shading 系数。

**Q5: MMoE 和 PLE 的区别？什么时候用 PLE？**
A: MMoE 所有任务共享相同的专家池，通过各任务门控软选择；PLE（Progressive Layered Extraction）引入任务专属专家 + 共享专家分层提取，解决 MMoE 在任务相关性低时门控塌陷（多个任务门控趋向相同）的问题。任务目标差异大（如 CTR 与时长）时用 PLE 效果更好。

**Q6: 生成式推荐中 Semantic ID 的作用是什么？**
A: Semantic ID 将物品 ID 离散化为多级 token 序列（如 RQ-VAE 量化），使物品的语义相似性反映在 token 的前缀共享上。同前缀 token 的物品语义更接近，使语言模型的 next-token prediction 天然学习到物品相似性，同时解决了传统 Item ID 的稀疏性问题。

**Q7: 预算 Pacing 的核心挑战是什么？PID 控制器如何应用？**
A: 核心挑战：预算全天均匀消耗（避免早耗尽或剩余），同时保证 ROI。PID 控制器：将"实际消耗率 vs 目标消耗率"的偏差作为输入，通过比例+积分+微分三项调节出价系数（bid multiplier）。P 项响应当前偏差，I 项修正累积误差，D 项预测趋势变化，工程中通常只用 PI 控制器。

**Q8: 反事实学习中 IPS 去偏的原理和局限？**
A: IPS 通过给样本加权（权重=1/propensity）消除 position bias：被高位展示的广告更容易被点击，IPS 给高位样本低权重使分布无偏。局限：propensity 估计不准时方差极大（high variance），需要 clipping 或 Doubly Robust 方法平衡偏差-方差。

### Q1: 搜广推三个领域的技术共性？
**30秒答案**：①都需要召回+排序架构；②都用 CTR/CVR 预估模型；③都面临冷启动问题；④都需要实时特征系统；⑤都可以用 LLM 增强。差异主要在约束条件和评估指标。

### Q2: 多目标优化在三个领域的应用？
**30秒答案**：广告：收入+用户体验+广告主 ROI；推荐：CTR+时长+多样性+留存；搜索：相关性+新鲜度+权威性+多样性。方法共通：Pareto/MMoE/PLE/Scalarization。

### Q3: 偏差问题在三个领域的表现？
**30秒答案**：广告：位置偏差+样本选择偏差；推荐：流行度偏差+曝光偏差；搜索：位置偏差+呈现偏差。解决方法类似：IPW/因果推断/去偏训练。

### Q4: 端到端学习的趋势和挑战？
**30秒答案**：趋势：统一模型替代分层管道（OneRec 统一召排）。挑战：①推理效率（一个大模型 vs 多个小模型）；②可控性差（难以插入业务规则）；③调试困难（黑盒）。

### Q5: 面试中如何体现跨领域理解？
**30秒答案**：①用类比说明（如广告出价≈搜索 LTR）；②指出技术迁移（如 DIN 从推荐到广告）；③提出统一视角（如多目标在三领域的共通框架）；④结合实际经验说明如何借鉴。

### Q6: 如何向面试官展示技术深度？
**30秒答案**：①先总后分：先说整体架构，追问时展开细节；②对比分析：主动比较 2-3 种方案的优劣；③数字说话：「AUC 从 0.72 提升到 0.74」而非「效果变好了」；④边界意识：说清楚方案的局限和适用条件。

### Q7: 跨领域知识迁移的实际案例？
**30秒答案**：①DIN（推荐→广告）：注意力机制从推荐 CTR 迁移到广告 CTR；②BERT（NLP→搜索）：预训练语言模型用于搜索排序；③Semantic ID（搜索→推荐）：从搜索的 doc ID 到推荐的 item ID 统一表示。

### Q8: 大规模系统的性能优化通用方法？
**30秒答案**：①缓存（特征缓存、结果缓存）；②异步（特征获取异步化）；③预计算（user embedding 离线算好）；④分层（粗排+精排降低计算量）；⑤模型优化（蒸馏/量化/剪枝）。

### Q9: 线上事故排查的思路？
**30秒答案**：①看监控：指标异常时间点→②查变更：最近上线了什么→③回滚验证：回滚后指标恢复说明是变更导致→④深入分析：看特征分布、样本分布、模型输出分布有无异常。

### Q10: 算法工程师的核心竞争力？
**30秒答案**：①业务理解（指标 → 技术方案的转化能力）；②工程能力（模型能上线、能调优、能排查问题）；③论文能力（快速读懂并判断论文的实用价值）；④系统思维（全链路优化而非单点优化）。
