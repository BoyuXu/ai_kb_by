## 📐 核心公式与原理

### 1. 多目标优化
$$\min_{\theta} \sum_k \lambda_k L_k(\theta)$$
- Scalarization 方法，λ 控制任务权重

### 2. Pareto 最优
$$x^* \text{ is Pareto optimal } \iff \nexists x: f_i(x) \leq f_i(x^*) \forall i$$
- 不存在在所有目标上都更好的解

### 3. 偏差校正 (IPW)
$$\hat{R} = \frac{1}{n}\sum_i \frac{r_i}{P(O=1|x_i)}$$
- 逆倾向加权消除选择偏差

## 🎯 核心洞察（4条）

1. **梯度下降家族是优化的核心**：SGD → Momentum → Adam → AdamW，理解每个优化器解决了什么问题是面试基础
2. **过拟合/欠拟合的诊断和治疗**：训练 loss 低+验证 loss 高=过拟合（加正则/dropout/更多数据）；两者都高=欠拟合（加模型容量/更好特征）
3. **损失函数的选择直接影响模型行为**：BCE 适合二分类（CTR）、BPR 适合排序、Focal Loss 适合类别不平衡、InfoNCE 适合对比学习
4. **Batch Normalization / Layer Normalization 是深度学习的基础设施**：BN 沿 batch 维度归一化（CV 常用），LN 沿特征维度归一化（NLP/推荐常用）

## 🎓 面试考点（6条）

### Q1: Adam 优化器的原理？
**30秒答案**：Adam 结合了 Momentum（梯度的指数移动平均，动量）和 RMSprop（梯度平方的指数移动平均，自适应学习率）。`m_t = β1×m_{t-1} + (1-β1)×g_t`（一阶矩），`v_t = β2×v_{t-1} + (1-β2)×g_t²`（二阶矩），`θ = θ - lr × m̂_t / (√v̂_t + ε)`。

### Q2: L1 vs L2 正则化的区别？
**30秒答案**：L1（Lasso）产生稀疏解（部分权重被压到0，自动特征选择）；L2（Ridge）使权重趋向小值但不为0（防止过大权重）。推荐/广告中 L1 常用于高维稀疏模型（FTRL），L2 常用于深度模型（weight decay）。

### Q3: Dropout 为什么能防止过拟合？
**30秒答案**：训练时随机丢弃 p% 的神经元，迫使网络学习冗余表示（不依赖任何单一神经元）。等效于训练 2^n 个子网络的 ensemble。推理时不 dropout，但权重乘以 (1-p) 补偿。

### Q4: Batch Normalization vs Layer Normalization？
**30秒答案**：BN 沿 batch 维度归一化（每个特征在 batch 内归一化），训练和推理不一致（推理用 running statistics）。LN 沿特征维度归一化（每个样本独立归一化），训练推理一致。Transformer 用 LN（batch 内序列长度不同）。

### Q5: 交叉熵损失 vs Focal Loss？
**30秒答案**：交叉熵 `L = -y×log(p) - (1-y)×log(1-p)`，对所有样本等权处理。Focal Loss `L = -α×(1-p)^γ×log(p)`，γ>0 时降低容易分类样本的权重，让模型聚焦难分样本。γ=2 是常用值。

### Q6: 特征工程中的数据泄露怎么识别？
**30秒答案**：①时间泄露：用未来信息预测过去（如用"是否转化"特征预测"是否点击"）；②目标泄露：特征中包含了目标变量的信息（如用户评分作为特征预测是否会评分）。识别：检查特征的信息增益是否异常高。

---


### Q7: 搜广推三个领域的技术共性？
**30秒答案**：①都需要召回+排序架构；②都用 CTR/CVR 预估模型；③都面临冷启动问题；④都需要实时特征系统；⑤都可以用 LLM 增强。差异主要在约束条件和评估指标。

### Q8: 多目标优化在三个领域的应用？
**30秒答案**：广告：收入+用户体验+广告主 ROI；推荐：CTR+时长+多样性+留存；搜索：相关性+新鲜度+权威性+多样性。方法共通：Pareto/MMoE/PLE/Scalarization。

### Q9: 偏差问题在三个领域的表现？
**30秒答案**：广告：位置偏差+样本选择偏差；推荐：流行度偏差+曝光偏差；搜索：位置偏差+呈现偏差。解决方法类似：IPW/因果推断/去偏训练。

### Q10: 端到端学习的趋势和挑战？
**30秒答案**：趋势：统一模型替代分层管道（OneRec 统一召排）。挑战：①推理效率（一个大模型 vs 多个小模型）；②可控性差（难以插入业务规则）；③调试困难（黑盒）。
