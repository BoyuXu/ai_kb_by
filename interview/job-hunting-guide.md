## 一、简历准备

### 1.1 简历核心结构

```
姓名 | 电话 | 邮箱 | GitHub | 学校
─────────────────────────────────────
教育背景（学校 专业 GPA 时间）
─────────────────────────────────────
实习/工作经历（STAR法则）
─────────────────────────────────────
项目经历（核心亮点+量化成果）
─────────────────────────────────────
技能清单（编程语言 框架 工具）
─────────────────────────────────────
论文/竞赛（可选，加分项）
```

### 1.2 关键要点

- **量化成果**：避免"负责了XX系统"，改为"优化XX使指标提升30%"
- **技术栈精准**：只写真正掌握的，否则面试会被追问
- **C++/Python二选一精通**：Python必须，C++加分
- **实习 > 论文 > 竞赛**（工程岗位）；顶会 > 实习（研究岗）

### 1.3 简历写作误区

❌ "熟悉机器学习算法" → ✅ "熟悉XGBoost/LightGBM，在XX数据集上AUC达到0.87"
❌ "参与了XX项目" → ✅ "独立设计并实现XX模块，TPS提升200%"
❌ 堆砌技术词汇 → ✅ 突出自己的贡献和结果

## 二、自我介绍模板

### 2.1 标准模板（1-2分钟）

```
面试官您好，我叫XX，来自[学校][专业]，研究方向是[方向]。

在学术方面，我[论文/项目成果简述]。

在实习方面，我曾在[公司]负责[核心职责]，期间[关键成果/数据指标]。

在技术方面，我熟练掌握[核心技术栈]，对[专业方向]有深入理解。

我对贵司[具体岗位/产品]非常感兴趣，期待能有机会为团队贡献价值。
```

### 2.2 应届生版本（无实习经历）

```
面试官您好，我叫XX，[学校][专业]研究生，专注[研究方向]。

在项目经历中，我独立完成了[项目名]，解决了[核心问题]，最终[结果/指标]。

我熟悉[框架/工具]，在[竞赛/开源项目]中有[成果]。

希望加入贵司的[岗位]团队，持续成长，谢谢。
```

## 三、STAR 项目介绍框架

STAR = **Situation（背景）+ Task（任务）+ Action（行动）+ Result（结果）**

### 3.1 模板

```
背景（Situation）：
"当时我们团队/项目面临的问题是..."

任务（Task）：
"我负责的部分是..."

行动（Action）：
"我具体做了以下工作：
  1. 首先分析了...，发现...
  2. 然后设计了...方案，因为...
  3. 实现时解决了...挑战，采用...方法"

结果（Result）：
"最终[指标A]提升了X%，[指标B]从X降到Y，
 项目上线后[业务影响]"
```

### 3.2 示例：推荐系统项目

```
背景：负责某电商App首页推荐CTR优化，原baseline为双塔模型，CTR为4.2%

任务：设计新的排序模型，提升CTR和GMV

行动：
1. 分析离线日志，发现用户行为序列特征利用不足
2. 引入DIN（Deep Interest Network），对历史行为加权注意力
3. 增加Item Side信息（类目/品牌Embedding）
4. A/B实验验证：先小流量灰度，确认稳定后全量

结果：CTR从4.2%提升到5.1%（+21%），GMV提升8%，
      上线后长期稳定运行
```

### 3.3 项目深挖问题应对

面试官常见追问：
- "你是怎么想到这个方案的？"→ 分析问题 + 查阅论文 + 对比方案
- "有没有遇到什么困难？如何解决？"→ 具体问题 + 解决思路
- "为什么不用X方案？"→ 对比优劣 + 结合业务场景
- "如果重来一次会怎么做？"→ 展示反思能力

## 四、技术面试准备清单

### 4.1 算法工程师核心考点

**基础算法（必考）：**
- 手写排序（快排、归并）
- 数据结构（链表、树、图）
- 动态规划（背包、路径、字符串）
- 刷题量建议：LeetCode 200+ 题（Easy全做，Medium重点）

**机器学习基础（必考）：**
- 决策树/GBDT/XGBoost原理
- 过拟合/欠拟合/正则化
- 特征工程方法
- 模型评估指标（AUC、F1、NDCG等）

**深度学习（算法岗必考）：**
- Transformer/Attention机制
- 反向传播原理
- 常见网络架构
- 训练技巧（BN/LN/Dropout/优化器）

### 4.2 搜广推方向重点

- CTR预估模型演进（LR → FM → DNN → 双塔 → 大模型）
- 召回/排序/重排系统架构
- 用户行为序列建模（DIN/DIEN）
- 多目标优化

## 五、HR面高频问题与策略

### 5.1 常见HR问题及应答策略

**Q：请做一下自我介绍？**
→ 参考STAR框架，1-2分钟，突出技术亮点和与岗位匹配度

**Q：你的优缺点是什么？**
→ 优点：具体化（不说"工作努力"）+"刻苦学习并快速应用"
→ 缺点：选择真实但不影响核心能力的点，并说明改进措施

**Q：为什么选择我们公司？**
→ 结合公司产品/技术/文化，说明与自己兴趣/规划的契合

**Q：职业规划是什么？**
→ 3年：在[方向]深耕，成为业务核心；5年：带领团队/技术专家

**Q：如何处理与上司意见不同的情况？**
→ 先理解对方立场 → 据理表达自己观点 → 尊重最终决策 → 记录以备复盘

**Q：期望薪资是多少？**
→ 提前调研行业水平；先让对方出价；说明范围而非固定数字

**Q：还在面其他公司吗？**
→ 诚实说有其他面试，但表明对该公司更感兴趣；切勿撒谎

## 六、薪资谈判技巧

### 6.1 了解市场行情

- 参考渠道：脉脉薪资、牛客讨论区、Glassdoor
- 互联网大厂算法岗参考（2025）：
  - 应届硕士：15k-35k × 15-16薪
  - 2-3年经验：25k-50k × 15-16薪

### 6.2 谈判策略

**时机：** 等收到口头Offer后再谈，主动权更大

**策略：**
1. **不要先报价**："您能告诉我这个岗位的薪资范围吗？"
2. **给出范围**：而非固定值，下限略高于预期
3. **锚定参考**："我目前有另一个Offer是X万，希望贵司能匹配"
4. **量化价值**："我在XX方向有XX年经验，相信能快速上手"

**面试官提问Checklist：**
- [ ] 薪资结构（月薪×几薪）
- [ ] 税前/税后
- [ ] 试用期薪资比例
- [ ] 五险一金缴纳基数
- [ ] 晋升周期和涨薪机制
- [ ] 股票/期权（上市/非上市）
- [ ] 绩效奖金比例

## 七、时间规划建议

| 时间节点 | 任务 |
|---------|------|
| 大三/研一 | 确定方向，打好基础（数学+编程+DL）|
| 大四/研二 | 实习（大厂优先）+ 参加比赛 |
| 秋招提前批（6-7月）| 优先投递，HC充足，要求低 |
| 秋招正式批（9-10月）| 批量投递，笔试面试密集期 |
| 冬招/春招 | 补漏，社招起点 |

**核心原则：**
1. **内推 > 官网投递**（免笔试+优先安排）
2. **提前批 > 正式批**（HC多，竞争少）
3. **大厂实习 > 小厂全职**（校招背书）
4. **60%准备好就投**（永远没有完全准备好的时候）

## 八、违约与多Offer处理

**收到多个Offer时：**
- 比较：薪资 + 方向匹配 + 公司前景 + 团队氛围 + 地点
- 不要因为"保底"而浪费时间在不感兴趣的公司
- 三段论：确定想去的A → 谈判 → 拿到后再放弃B

**违约话术（如需违约）：**
- 提前告知（不要拖到入职前）
- 态度诚恳：说明个人原因（非公司原因）
- 承担违约金（如有）
- 保持良好关系（行业很小）
