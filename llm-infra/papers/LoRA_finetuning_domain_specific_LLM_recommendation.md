# LoRA Fine-Tuning for Domain-Specific LLM in Recommendation
> 来源：arXiv (LoRA, DoRA) | 领域：LLM基础设施 | 学习日期：20260327

## 问题定义
将通用LLM适配到推荐场景（或其他垂直领域）面临：
1. **全量微调成本极高**：7B参数模型全量微调需要40+GB GPU显存
2. **灾难性遗忘**：全量微调后模型可能遗忘通用能力
3. **多任务适配需求**：推荐系统有多个子任务（召回/排序/解释），需要为每个任务微调

**目标**：通过LoRA/DoRA等参数高效微调方法，以<2%的效果损失实现-98%的训练成本。

## 核心方法与创新点

### 1. LoRA（Low-Rank Adaptation）
在预训练权重基础上加入低秩矩阵：
$$W' = W_0 + \Delta W = W_0 + BA$$

其中 $B \in \mathbb{R}^{d \times r}$，$A \in \mathbb{R}^{r \times k}$，秩 r << min(d, k)。
- 初始化：A用高斯随机初始化，B用零初始化（确保训练初始 ΔW = 0）
- 只训练A和B，参数量：$r(d+k)$ vs 原始 $d \times k$，减少比例 $= r/(d+k) \times (d+k)/(dk) = 1/r'$

实践中 r=8~64，参数量减少100~1000倍。

### 2. DoRA（Weight-Decomposed Low-Rank Adaptation）
将权重分解为方向和幅度两个组成部分：
$$W' = \frac{W_0 + BA}{||W_0 + BA||_c} \cdot M$$

- **方向部分**：$(W_0 + BA) / ||\cdot||_c$，用LoRA更新
- **幅度部分**：标量 $M \in \mathbb{R}^{1 \times k}$，可学习

DoRA更接近全量微调的更新模式，在相同秩r下比LoRA效果更好（+1~2%）。

### 3. 推荐场景的LoRA应用
**哪些模块加LoRA**（推荐）：
- Attention层的Q、V矩阵（必加）
- FFN的down_proj（可选）
- 不建议加K矩阵（实验证明效果增益小）

### 4. 多任务LoRA
每个推荐子任务有独立的LoRA权重，共享主干：
$$W'_{task_t} = W_0 + B_t A_t$$

推理时根据任务类型动态加载对应的LoRA权重（<1ms切换时间）。

## 实验结论
- **效果对比全量微调**：NDCG差距 <2%（LoRA r=64）
- **训练成本**：减少-98%（显存从80GB→10GB，训练时间减少90%）
- **DoRA vs LoRA**：相同r下，DoRA NDCG高+1.5%
- 多任务LoRA：3个子任务共用一个主干，推理成本比3个独立模型减少67%

## 工程落地要点
1. **秩的选择**：r越大效果越好但参数越多，通常r=16~64是效果/效率甜点
2. **α参数**：LoRA的缩放参数 α/r，建议α=2r（经验法则）
3. **目标模块**：推荐系统中Attention Q/V矩阵最重要，FFN次之
4. **合并推理**：部署时将LoRA权重合并到主干（W' = W_0 + BA），消除额外计算开销
5. **量化+LoRA（QLoRA）**：主干模型用4bit量化存储，只有LoRA权重用fp16，进一步减少显存

## 面试考点
Q1: LoRA为什么用低秩矩阵而不是其他参数化方式？
A: 直觉理解：神经网络的权重更新（ΔW）在训练时的内在秩往往很低——模型学习新任务时，实际上只改变了权重空间中几个主要方向。实验验证：全量微调的ΔW的奇异值分布非常集中（少数大奇异值，大量接近零），说明低秩近似是合理的。理论支撑：Aghajanyan等人证明了fine-tuning问题的"本征维度"（Intrinsic Dimensionality）远小于参数空间维度。

Q2: DoRA相比LoRA的核心改进是什么？
A: LoRA的问题：BA矩阵同时改变权重的方向和幅度，两者耦合，导致优化困难。DoRA将方向（归一化后）和幅度（标量M）分开优化：幅度可以精细调整（标量），方向用LoRA近似（低秩）。这更接近全量微调的行为（全量微调中，方向变化用低秩可以近似，但幅度变化需要独立控制）。代价：引入额外的幅度参数M，略增加参数量。

Q3: 推荐系统中使用LoRA fine-tune LLM时，如何防止灾难性遗忘？
A: (1)LoRA本身天然防遗忘：主干权重W_0完全冻结，只更新低秩ΔW，不可能遗忘原始知识；(2)正则化：在LoRA权重上加L2正则，防止ΔW过大（过大的ΔW等效于全量修改）；(3)回放（Replay）：在fine-tune数据中混入少量通用任务数据；(4)弹性权重整合（EWC）：对重要的参数（由Fisher信息矩阵度量）施加更强的正则化。推荐系统场景：因为不改变主干，LoRA本质上不存在遗忘问题。
