## 1. 内容理解在推荐中的应用（图文视频特征）

### 1.1 概述

多模态内容理解已成为现代推荐系统的核心技术，通过整合**图像、文本、视频、音频**等多模态信息，显著提升了推荐的准确性和用户体验。

### 1.2 关键技术方向

| 模态类型 | 应用场景 | 典型方法 |
|---------|---------|---------|
| **图像** | 电商商品推荐、时尚搭配 | VBPR、MMGCN、DualGNN |
| **文本** | 新闻推荐、商品描述理解 | BERT4Rec、UniSRec |
| **视频** | 短视频推荐、视频平台 | VideoMAE、CLAP、MLLM |
| **多模态融合** | 综合内容理解 | Recformer、P5 |

### 1.3 代表性工作

#### VBPR (Visual Bayesian Personalized Ranking)
- **核心思想**：使用CNN提取商品图像特征，结合传统BPR进行推荐
- **优势**：首次将视觉信息融入协同过滤框架
- **演进**：后续工作如ACF、DVBPR等进一步优化视觉特征提取

#### BERT4Rec
- **核心思想**：利用BERT预训练模型学习用户行为序列模式
- **创新点**：将推荐视为序列到序列的预测任务
- **影响**：开创了NLP预训练模型用于推荐的先河

#### Spotify MLLM视频推荐框架 (2025)
- **核心思想**：使用多模态大语言模型(MLLM)生成视频内容描述
- **流程**：
  1. 使用Qwen-VL等MLLM生成视频帧描述
  2. Whisper转录音频，Qwen-Audio生成音频摘要
  3. 文本编码器(BGE-large)生成特征向量
  4. 输入Two-Tower或SASRec进行推荐
- **效果**：音频转文本描述提升Two-Tower模型60%性能

### 1.4 技术挑战

1. **噪声问题**：图像背景干扰、文本描述夸张
2. **跨模态对齐**：不同模态特征空间的统一表示
3. **计算效率**：多模态特征提取的实时性要求
4. **领域适配**：通用预训练模型与推荐场景的gap

## 2. CLIP/BLIP在推荐中的特征提取

### 2.1 CLIP (Contrastive Language-Image Pre-training)

#### 基本原理
- **架构**：双塔结构（Vision Encoder + Text Encoder）
- **训练目标**：对比学习，最大化匹配图文对的相似度
- **优势**：强大的Zero-shot分类能力，跨模态对齐

#### 推荐系统应用

| 方法 | 核心创新 | 效果 |
|-----|---------|-----|
| **CPMM** | CLIP预训练+协同信息融合 | 支持序列推荐、协同过滤、CTR预测 |
| **VL-CLIP** | Visual Grounding + LLM增强文本 | CTR提升18.6%，GMV提升4.0% |
| **MoRec** | 使用CLIP作为Item Encoder | 替代传统ID嵌入 |

#### CPMM框架详解
```
核心组件：
1. 多模态语义增强层：将图像、文本、ID映射到统一对比空间
2. 三元组表示一致性：通过对比学习调节模态间距离
3. 协同信息注入：建模一阶相似性，融入协作信号
```

### 2.2 BLIP (Bootstrapping Language-Image Pre-training)

#### 特点
- **Q-Former架构**：桥接视觉和语言理解
- **能力**：图像描述、视觉问答、图文检索
- **优势**：生成的文本描述更丰富、准确

#### 推荐场景应用
1. **商品描述生成**：将商品图像转换为结构化文本特征
2. **跨模态检索**：文本搜图、图搜相似商品
3. **多模态融合**：与CLIP特征互补

### 2.3 领域适配挑战与解决方案

#### 挑战
| 问题 | 说明 |
|-----|-----|
| 语义偏差 | 通用预训练关注整体语义，推荐需细粒度属性 |
| 噪声干扰 | 图像含无关背景，文本含营销夸张 |
| 缺乏协同信号 | 预训练模型未考虑用户行为数据 |

#### 解决方案

**VL-CLIP (ACM RecSys 2025)**
- **Visual Grounding**：定位关键产品区域，提取细粒度视觉特征
- **LLM Agent增强**：用LLM消除产品描述歧义，生成 richer 文本嵌入
- **效果**：在千万级电商商品上显著提升检索准确性

**CPMM优化策略**
- 单模态净化：注入高斯噪声、归一化
- 跨模态对齐：对比学习对齐图文表示
- ID作为中间模态：连接多模态与协同信号

## 3. LLM作为推荐系统的几种范式

### 3.1 范式对比总览

```
┌─────────────────────────────────────────────────────────────────┐
│                    LLM for Recommendation                        │
├──────────────────┬──────────────────┬──────────────────────────┤
│   LLM as RS      │  LLM-enhanced RS │   LLM as Feature         │
│   (直接生成)      │   (特征增强)      │   Encoder (意图理解)      │
├──────────────────┼──────────────────┼──────────────────────────┤
│ • P5             │ • RLMRec         │ • UniSRec               │
│ • LLMRec         │ • LLM-ESR        │ • VQRec                 │
│ • ChatRec        │ • AlphaRec       │ • MoRec                 │
│ • TALLRec        │ • CoLLM          │ • Recformer             │
└──────────────────┴──────────────────┴──────────────────────────┘
```

### 3.2 范式一：直接生成 (LLM as RS)

#### 核心思想
将推荐任务转化为自然语言生成任务，直接输出推荐结果。

#### 代表性方法

**P5 (RecSys 2022)**
- **统一框架**：将多种推荐任务统一为NLP任务
- **多任务Prompt**：
  - 序列推荐："用户看过A、B、C，下一个可能看？"
  - 评分预测："用户会给这部电影打几分？"
  - 解释生成："为什么推荐这个商品？"
- **架构**：自回归解码，用户/物品用ID表示

**M6-Rec**
- **特点**：用纯文本表示用户/物品，无需ID
- **任务类型**：
  - 理解任务：CTR/CVR预测（打分）
  - 生成任务：个性化产品设计、解释生成

**ChatRec / TALLRec**
- **对话式推荐**：多轮对话理解用户意图
- **指令微调**：使用LoRA等技术微调开源LLM
- **优势**：冷启动场景表现优异

#### 优缺点
| 优势 | 劣势 |
|-----|-----|
| 强大的语义理解能力 | 推理成本高 |
| 优秀的冷启动泛化 | 难以捕捉协同信号 |
| 可解释性强 | 训练/部署资源需求大 |

### 3.3 范式二：特征增强 (LLM Embedding)

#### 核心思想
利用LLM生成高质量的文本/用户/物品嵌入，增强传统推荐模型。

#### 代表性方法

**RLMRec (2023)**
- **方法**：对齐LLM语义空间与推荐目标（对比学习）
- **架构**：LLM文本嵌入 + GNN表示
- **特点**：融合语义与结构信息

**LLM-ESR (NeurIPS 2024)**
- **目标**：解决长尾序列推荐
- **策略**：LLM嵌入作为传统模型初始化
- **优势**：缓解语义信息稀疏问题

**CoLLM (TKDE 2025)**
```
核心创新：显式整合协同信息
├── Prompt构建：增加User/Item ID字段
├── 混合编码：
│   ├── 文本信息：LLM Token嵌入
│   └── 协同信息：传统CF模型嵌入 → 映射到LLM空间
└── 两阶段训练：
    ├── 阶段1：仅语言信息学习推荐任务
    └── 阶段2：注入协同信息微调
```

**AlphaRec**
- 直接使用LLM嵌入增强协同过滤模型
- 保持传统模型的高效推理

### 3.4 范式三：意图理解 (Query改写/冷启动)

#### 应用场景

| 应用 | 方法 | 效果 |
|-----|-----|-----|
| **Query改写** | LLM理解搜索意图，扩展查询词 | 提升搜索召回 |
| **冷启动用户** | 利用LLM分析用户画像文本 | 缓解数据稀疏 |
| **物品内容增强** | LLM生成物品摘要、标签 | 丰富物品表示 |
| **推荐理由生成** | 个性化解释生成 | 提升用户信任 |

#### 技术实现

**Recformer (KDD 2023)**
- **理念**："Text Is All You Need"
- **方法**：完全用文本表示物品，无需ID
- **优势**：低资源场景、跨域推荐

**意图识别框架**
```python
# 典型Pipeline
用户Query → LLM意图分析 → 查询改写/扩展 → 检索 → 排序 → 推荐
```

### 3.5 新兴趋势：LLM Agent + 推荐

**E4SRec / LLaRA / RecInterpreter**
- 将协同信号编码到Tokenization中
- 解决LLM缺乏协同信息的根本问题

**DRDT (动态反射)**
- 基于检索的动态反思过程
- 利用上下文学习(In-context Learning)优化推荐

## 4. 知识图谱增强推荐 (KGCN/KGAT)

### 4.1 知识图谱在推荐中的作用

#### 价值
1. **丰富物品表示**：结构化属性信息
2. **可解释性**：通过关系路径解释推荐
3. **缓解稀疏性**：连接冷门物品
4. **跨域推荐**：共享实体知识

### 4.2 经典方法详解

#### KGCN (Knowledge Graph Convolutional Network)

**核心思想**
- 将GCN扩展到知识图谱
- 在用户-物品交互图上聚合KG信息

**关键步骤**
```
1. 实体嵌入：初始化KG中的实体表示
2. 关系感知聚合：考虑不同关系类型的信息传播
   e_agg = Σ α(e,r,e') · e'
   其中 α 是注意力权重
3. 用户表示：聚合历史交互物品的KG信息
4. 预测：用户与候选物品的交互概率
```

#### KGAT (Knowledge Graph Attention Network, KDD 2019)

**核心创新**
- **统一图结构**：整合用户-物品二部图与知识图谱
- **注意力机制**：区分邻居节点的重要性
- **高阶关系建模**：递归传播捕获多跳关系

**架构详解**
```
┌────────────────────────────────────────┐
│           KGAT Architecture            │
├────────────────────────────────────────┤
│ 1. Embedding Layer                     │
│    ├── User Embedding: u               │
│    ├── Item Embedding: i               │
│    └── Entity Embedding: e             │
│                                        │
│ 2. Attentive Embedding Propagation     │
│    ├── 递归传播：e ← Aggregate(Neighbors)│
│    ├── 知识感知注意力：π(h,r,t)         │
│    └── 关系建模：W_r · e               │
│                                        │
│ 3. Prediction Layer                    │
│    └── BPR Loss: 优化用户-物品交互      │
└────────────────────────────────────────┘
```

**注意力机制**
```
π(h,r,t) = (W_r · e_t)^T · tanh(W_r · e_h + e_r)
α(h,r,t) = softmax(π(h,r,t)) = exp(π) / Σ exp(π)
```

### 4.4 最新进展

#### AKDN (Attention-enhanced Knowledge-fused Dual Network)

**解决问题**
1. KG中的噪声三元组
2. 区分KG表示与协同过滤表示

**核心设计**
- **注意力聚合**：学习KG边的权重，过滤噪声
- **门控融合机制**：区分协同信号与知识信息
- **双重物品表示**：融合两种信息源

#### MetaKRec
- **元知识图**：利用先验元知识构建不同元图
- **优势**：减少噪声，简单模型即可达到最佳性能

## 5. 因果推断在推荐去偏中的应用

### 5.1 推荐系统中的偏差类型

| 偏差类型 | 描述 | 影响 |
|---------|-----|-----|
| **选择偏差** | 用户仅与部分物品交互，观测数据非随机 | 模型偏向热门物品 |
| **曝光偏差** | 用户只能看到系统曝光的物品 | 反馈循环放大偏差 |
| **位置偏差** | 排名靠前的物品获得更多点击 | 位置与相关性混淆 |
| **流行度偏差** | 热门物品被过度推荐 | 多样性降低，马太效应 |
| **一致性偏差** | 用户倾向于与群体行为一致 | 个性化被削弱 |

### 5.2 因果推断基础框架

#### 核心概念

```
因果图模型：
U(用户) ──→ I(物品曝光) ──→ Y(交互)
    ↑            ↑
    └──────── C(混杂因子) ────┘
              
目标：估计 P(Y|do(I)) 而非 P(Y|I)
```

#### 主要技术路线

| 路线 | 方法 | 代表工作 |
|-----|-----|---------|
| **混杂消除** | Backdoor Adjustment | PDA、DecRS |
| **反事实推断** | Counterfactual TIE | EliMRec |
| **IPS重加权** | Inverse Propensity Score | IPS-based MF |
| **双重稳健** | Doubly Robust | DR-JL |

### 5.3 去偏方法详解

#### 混杂消除 (Confounding Elimination)

**PDA (Popularity-bias Deconfounding Adjustment)**
- **识别**：物品流行度是曝光与交互之间的混杂因子
- **调整**：基于Backdoor Adjustment消除影响
- **公式**：P(Y|do(I)) = Σ_c P(Y|I,c)P(c)

**DecRS**
- **创新**：提出Backdoor Adjustment的近似算子
- **特点**：可无缝集成到大多数推荐模型
- **动态策略**：根据用户状态动态调节调整强度

#### 反事实推断 (Counterfactual Inference)

**核心思想**
```
总效应(TE) = 直接效应(NDE) + 间接效应(TIE)

去偏目标：消除TIE中的偏差分量
```

**EliMRec**
- **方法**：构建单模态变化的反事实场景
- **流程**：
  1. 保持其他模态不变，仅变化一个模态
  2. 计算真实世界与反事实世界的预测差
  3. 从总效应中减去偏差效应

**公平性模态去偏框架**
- **训练期**：引入单模态预测分支捕获模态偏差
- **推理期**：公平感知的反事实分析自适应消除偏差

#### IPS (Inverse Propensity Score)

**基本原理**
```
 unbiased_loss = Σ (observed_rating / propensity_score)
 
 倾向得分：P(observed | item, user, position)
```

**USR (User Side Representation)**
- **创新**：GRU通道估计基于历史序列的倾向得分
- **分解**：用户评分倾向 × 物品评分倾向 × 用户-物品相关倾向

**DEPS**
- **改进**：从用户视角估计倾向得分
- **优势**：更稳健的顺序去偏

#### 双重稳健方法 (Doubly Robust)

**AutoDebias**
- **结合**：IPS + 插补(Imputation)方法
- **通用框架**：适用于所有类型的偏差
- **优势**：降低IPS高方差问题

### 5.4 最新研究进展

#### 社会推荐中的因果推断 (2025)

**问题**：社会流行度偏差 + 物品流行度偏差

**解决方案**
1. **因果图建模**：理解物品/社会流行度如何影响用户表示
2. **Backdoor Adjustment**：学习无偏用户表示
3. **反事实推理**：获得准确排序分数

#### CausalRec / LCDR
- **表示解耦**：分离曝光、评分、混杂因子相关特征
- **MMD Loss**：最大均值差异约束
- **潜变量模型**：iVAE恢复未观测混杂因子

#### 因果推断+LLM (2025)
- **研究问题**：如何有效解耦和缓解LLM推荐中的流行度偏差
- **方法**：因果学习技术对齐LLM与推荐目标
