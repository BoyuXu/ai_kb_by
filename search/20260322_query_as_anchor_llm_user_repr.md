# Query as Anchor: Scenario-Adaptive User Representation via LLM for Search

> 来源：arxiv | 日期：20260322 | 领域：搜索系统

## 问题定义

搜索场景下，用户表示（User Representation）需要结合当前搜索 query 动态调整——同一用户搜索"跑步鞋"和"西装"时，应激活不同的历史行为和兴趣侧面。现有方法使用静态用户 Embedding，无法适应 query 的场景变化。

## 核心方法与创新点

- **Query as Anchor（QaA）框架**：以当前 query 为锚点，从用户历史中动态检索和激活相关记忆
- **LLM 增强的用户理解**：
  - 用 LLM 对用户历史行为序列生成自然语言摘要（"该用户近期关注轻奢女包，价格敏感度中等"）
  - LLM 摘要作为软提示（soft prompt）注入用户表示
- **场景自适应注意力**：
  - 用 query 表示作为 key/query，用户历史 item 表示作为 value
  - Cross-attention 动态权重激活与当前搜索场景相关的历史行为
- **多粒度 Query 理解**：字符级（关键词匹配）+ 语义级（嵌入相似度）+ 意图级（LLM 推断购买意图）
- **轻量级在线适配**：LLM 摘要离线生成缓存，在线只做 cross-attention 计算（毫秒级）

## 实验结论

- 电商搜索相关性评估（NDCG@10）提升 9.3%（vs 静态用户 Embedding）
- 多场景用户（横跨多品类）改善最明显：+14.7%（因为 query 锚点能精准激活对应场景兴趣）
- LLM 摘要增强 vs 无 LLM：+3.8% NDCG（LLM 能理解隐式偏好，如价格敏感度）
- 在线 A/B：搜索点击率 +5.1%，用户满意度（5分制）+0.3 分

## 工程落地要点

- **LLM 摘要更新频率**：建议每日离线更新（用户行为变化较慢），实时更新成本过高
- **摘要长度控制**：建议 100-200 字，避免 prompt 过长影响主模型的 attention 效率
- **冷启动**：新用户无历史行为，LLM 摘要退化为通用用户画像 + query 本身
- **隐私保护**：LLM 生成摘要时避免包含个人隐私信息，摘要本身存储需加密

## 面试考点

1. **Q：为什么要以 query 为锚点而不是用静态用户嵌入？**
   A：用户兴趣是多维的，不同搜索场景激活不同侧面；静态嵌入是所有历史的平均，会稀释场景特定信号

2. **Q：LLM 摘要在这里的作用是什么？能否替换？**
   A：LLM 能从行为序列中提炼隐式偏好（如"偏好高性价比"）；可以替换为更轻量的 seq2seq 模型，但语义理解深度下降

3. **Q：实时检索历史行为（类似 SIM）和 cross-attention 的区别？**
   A：SIM 用 ANN 检索（精确 topK 匹配），适合超长序列（1000+）；cross-attention 对所有历史加权，适合中等长度序列（<200），保留更多上下文
