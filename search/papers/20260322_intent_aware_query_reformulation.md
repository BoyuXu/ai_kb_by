# Intent-Aware Neural Query Reformulation for Behavior-Aligned Product Search

> 来源：arxiv/工业界 | 日期：20260322 | 领域：搜索系统

## 问题定义

电商搜索中，用户 query 往往口语化、模糊（如"好看的包"），与商品标题语言风格不匹配，导致召回不足。Query 改写（Query Reformulation）需要理解用户真实购物意图，生成更精准的搜索 query。

## 核心方法与创新点

- **意图识别**：先用分类器识别 query 的购物意图类型（品牌搜索/品类探索/具体商品/比价等），意图类型决定改写策略
- **行为对齐训练**：
  - 训练信号：用户点击/购买的商品标题 vs 原始 query 的语义对齐程度
  - 损失函数：对比学习（改写后 query 与点击商品标题的语义距离 < 改写后 query 与随机商品的距离）
- **Seq2Seq 改写模型**：基于 T5/BART，输入原始 query + 用户历史（intent context），输出改写后的 query 列表
- **多粒度改写**：同时生成多个改写结果（扩展词/词义规范化/同义替换），增加召回多样性
- **在线实验框架**：用改写 query 分流检索，确保原始结果不被完全替换（安全降级）

## 实验结论

- 检索召回率（Recall@100）提升 13.8%（vs 原始 query 直接检索）
- 用户点击率提升 7.6%，下单转化率提升 5.2%
- 意图识别准确率 89.3%（意图感知改写 vs 通用改写）：+4.1% Recall
- 低频 query（<10 次历史搜索）效果最显著：+18.2% Recall

## 工程落地要点

- **延迟控制**：改写模型推理需 <20ms，建议使用蒸馏后的小型 T5（T5-small 或 T5-base）
- **改写过滤**：改写结果需经过安全检查（不能引入违禁词、不能语义偏离太大）
- **A/B 测试策略**：先在 query 流量的 5% 上实验，监控搜索 0 结果率、用户满意度
- **混合检索**：原始 query + 改写 query 的结果取并集，按相关性重排

## 面试考点

1. **Q：Query 改写和 Query 扩展有什么区别？**
   A：改写（reformulation）是替换/重构整个 query（如规范化表达）；扩展（expansion）是在原 query 基础上增加词（如添加同义词）。两者互补

2. **Q：如何避免改写引入噪声？**
   A：语义相似度过滤（改写前后的 embedding 余弦相似度 >0.8）；意图一致性检验；人工抽样审核 + 自动化测试集

3. **Q：为什么用用户行为（点击/购买）而非人工标注训练改写模型？**
   A：人工标注成本高且难以覆盖长尾 query；用户行为是真实意图的天然体现，且可大规模获取；点击/购买信号相比标注更贴近商业目标
