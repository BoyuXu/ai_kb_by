# LegalMALR: Multi-Agent Query Understanding and LLM-Based Reranking for Chinese Statute Retrieval
> 来源：https://arxiv.org/abs/2501.18500 | 日期：20260319

## 问题定义
中文法律法规检索面临独特挑战：（1）法律查询语言专业，普通用户表达口语化；（2）法条文本密度高、语义相近但适用场景不同；（3）相关性判断需要法律专业知识。LegalMALR提出多Agent架构，结合查询理解和LLM重排，提升中文法条检索准确率。

## 核心方法与创新点
1. **多Agent查询理解**：
   - Agent1（意图识别）：判断查询类型（侵权/合同/刑事等）
   - Agent2（查询扩展）：补充法律术语、同义词、相关概念
   - Agent3（查询分解）：将复杂查询拆分为多个子问题
2. **层次检索**：先用BM25/Dense召回候选（Top-100），再用LLM精排（Top-10）
3. **LLM重排**：用法律领域微调的LLM对候选法条打分，输入格式：[查询][法条]→[相关性分数]
4. **结果融合**：多Agent查询结果用RRF（Reciprocal Rank Fusion）合并
5. **法律知识注入**：在LLM重排提示中加入法律体系结构信息，帮助模型理解法条层级关系

## 实验结论
- 在中文法律检索数据集上，MAP@10比BM25基线提升28%，比Dense单模型提升12%
- 多Agent查询理解相比直接检索提升约8%-15%（尤其对口语化查询）
- LLM重排比Cross-Encoder重排效果相当，但对复杂法律推理场景更优

## 工程落地要点
- 法律检索需要定期更新法条数据库（新法规发布、旧法废止）
- Multi-Agent增加延迟，建议并行执行多个Agent的查询扩展
- LLM重排成本高，建议只对Top-20候选进行，而非全量
- 可用判例数据微调LLM，提升领域相关性判断能力

## 面试考点
**Q: RRF（Reciprocal Rank Fusion）的原理？**
A: 将多个排序列表融合，每个文档的得分 = Σ 1/(k + rank_i)，k为常数（通常60），rank_i为文档在第i个列表中的排名。无需参数调优，对分布外数据鲁棒。

**Q: 法律检索与通用检索的主要区别？**
A: （1）专业术语密集，需领域词典；（2）法条结构化（章节条款），需层次索引；（3）相关性判断需理解法律逻辑，简单词汇匹配不足；（4）精确性要求极高，误检可能导致严重后果。

**Q: 大模型重排（LLM-as-Reranker）的优缺点？**
A: 优点：理解复杂语义、支持推理、zero-shot泛化；缺点：推理成本高（100候选需100次调用）、延迟大。常用优化：listwise批量打分、蒸馏到小模型。
