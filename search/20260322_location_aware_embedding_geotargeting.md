# Location Aware Embedding for Geotargeting in Sponsored Search Advertising

> 来源：arxiv/工业界 | 日期：20260322 | 领域：搜索系统

## 问题定义

搜索广告的地理定向（Geotargeting）需要广告主指定目标地区，但用户的实时地理位置和搜索意图存在复杂关联（如用户在北京搜索"上海旅游"）。如何学习位置感知的 Embedding 提升地理定向精准度？

## 核心方法与创新点

- **地理层次化 Embedding**：构建多粒度位置表示（国家→省→市→区县→POI），使用层次化嵌入表达地理关系
- **双位置编码**：
  - **用户当前位置嵌入**：基于 GPS 或 IP 解析的实时位置
  - **搜索意图位置嵌入**：从 query 中提取的目标地点实体
- **位置相关性矩阵**：预训练位置间的相关性（基于历史用户迁移数据、出行轨迹），将地理邻近关系注入嵌入空间
- **广告地理匹配**：用户位置嵌入 + 意图位置嵌入 × 广告地理定向范围嵌入，计算地理相关性分数
- **冷启动**：新地区（新开城市/新行政区划）通过层次化嵌入从父节点继承初始表示

## 实验结论

- 地理定向精准度：广告与用户真实需求的地理匹配率提升 14.7%
- 搜索广告 CTR：地理相关广告点击率提升 11.3%，无关广告降低 8.6%
- 广告主 ROI：地理定向 ROI 提升 9.4%（减少无效跨地区曝光）

## 工程落地要点

- **位置解析**：IP→城市精度约 70%，GPS（移动端）精度 >95%，优先使用 GPS
- **位置 Embedding 维度**：16-32 维足够，避免与主特征维度不匹配
- **隐私合规**：位置数据的采集和使用需符合 GDPR/个人信息保护法，建议使用粗粒度（城市级）位置
- **更新频率**：位置嵌入每季度更新（配合行政区划变化）

## 面试考点

1. **Q：如何处理"人在 A 地但搜索 B 地"的场景？**
   A：双位置编码分别表示用户当前位置和搜索意图位置，模型学习在不同场景下各位置的权重（如旅游 query 意图位置更重要）

2. **Q：地理层次化嵌入相比直接 one-hot 的优势？**
   A：层次化嵌入能表达地理父子关系（区属于市），使相邻区域有相似嵌入，解决低频地区（如偏远县城）的数据稀疏问题

3. **Q：地理定向广告和兴趣定向广告如何融合？**
   A：在精排阶段将地理匹配分和兴趣相关性分加权融合，权重可根据 query 类型自适应（本地服务类 query 地理权重更高）
