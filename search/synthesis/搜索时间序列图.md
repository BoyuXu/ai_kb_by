## 📐 核心公式与原理

### 1. BM25
$$BM25(q,d) = \sum_{t \in q} IDF(t) \cdot \frac{tf \cdot (k_1+1)}{tf + k_1(1-b+b\frac{|d|}{avgdl})}$$
- 经典稀疏检索评分

### 2. Dense Retrieval
$$score = E_q^T E_d$$
- 双塔编码器的向量内积

### 3. ColBERT MaxSim
$$score = \sum_i \max_j E_q^i \cdot E_d^j$$
- 每个 query token 找最相似的 doc token

## 🎯 核心洞察（4条）

1. **信息的时效性是搜索质量的关键维度**：新闻、社交媒体、电商促销等场景中，"最新"往往比"最相关"更重要，传统 BM25/Dense 检索缺乏时间感知
2. **时序图模型捕捉实体关系的动态演化**：RETE 用时序图网络建模实体（人/事件/地点）间的关系变化，在时效性查询上显著优于静态检索
3. **时间衰减权重是最简单有效的方案**：工程中最常用 `time_score = exp(-λ × age_days)` 与 relevance_score 加权融合，实现简单效果好
4. **图检索补充了向量检索的结构缺失**：GRIT 在知识图谱上做子图检索，捕捉实体间的关系路径（A→B→C），这是向量内积无法表达的

## 📈 技术演进脉络

```
BM25 + 时间排序因子（2000s）
  → TF-IDF + 时间衰减加权（2010s）
    → Dense Retrieval + 时间嵌入（2020-2022）
      → 时序图网络 RETE/GRIT（2023-2025）
        → LLM 时间感知检索（2025+）
```

**关键转折点**：
- **时间嵌入（2021）**：将时间戳编码为向量（类似位置编码），让 Dense 模型感知文档新鲜度
- **时序图网络（2023）**：RETE 首次在大规模搜索中验证时序图对时效性查询的有效性

## 🔗 跨文献共性规律

| 规律 | 体现 | 说明 |
|------|------|------|
| 时效性和相关性需要显式平衡 | 所有时序检索 | 太偏时效性会牺牲相关性，需要 query 意图判断 |
| 图结构补充向量表示的不足 | GRIT, RETE | 关系路径和演化模式是向量内积无法捕捉的 |
| 简单方法在工程中最可靠 | 时间衰减加权 | 复杂模型提升有限但工程成本高 |

## 🎓 面试考点（4条）

### Q1: 搜索引擎怎么处理时效性？
**30秒答案**：①query 意图分类（时效性 vs 常青）；②时效性 query 加大时间衰减权重 `score = rel_score × time_decay`；③文档爬取频率按更新频率分级（新闻站每小时，百科每天）。
**追问方向**：怎么判断 query 是否有时效性？答：关键词特征（"最新"、"2024"）+ 搜索趋势突增检测。

### Q2: 时序图在搜索中怎么用？
**30秒答案**：将文档/实体/事件构建为图，边带时间戳。查询时在图上做时序感知的子图匹配，找到与 query 在时间和语义上都相关的文档子集。
**追问方向**：图检索的延迟问题？答：离线预计算子图 embedding，在线近似匹配。

### Q3: 时间衰减函数怎么设计？
**30秒答案**：常用指数衰减 `decay = exp(-λ × age)`，λ 按场景调整：新闻 λ=0.1（日级衰减快），知识文档 λ=0.001（月级缓慢衰减）。也可用分段函数：<24h 不衰减，1-7d 线性衰减，>7d 指数衰减。

### Q4: 图检索（GRIT）vs 向量检索的区别？
**30秒答案**：向量检索用点对点相似度（cosine(q, d)），无法表达关系路径；图检索能发现"A 的合作者 B 最近发表了与 query 相关的论文"这种多跳关系。代价是图构建和维护成本高。

---


### Q5: 搜索系统的评估指标有哪些？
**30秒答案**：离线：NDCG、MRR、MAP、Recall@K。在线：点击率、放弃率、首页满意度、查询改写率。注意：离线和在线可能不一致。

### Q6: 稠密检索的训练数据构造？
**30秒答案**：正样本：人工标注/点击日志。负样本：①随机负样本；②BM25 Hard Negative；③In-batch Negative。Hard Negative 对效果至关重要。

### Q7: 搜索排序特征有哪些？
**30秒答案**：①Query-Doc 匹配（BM25/embedding 相似度/TF-IDF）；②Doc 质量（PageRank/内容长度/freshness）；③用户特征（搜索历史/偏好）；④Context（设备/地理/时间）。

### Q8: 向量检索的工程挑战？
**30秒答案**：①索引构建耗时（十亿级 HNSW 需要数小时）；②内存占用大（每个向量 128*4=512B，十亿=500GB）；③更新延迟（新文档需要重建索引）；④多指标权衡（召回率/延迟/内存）。

### Q9: RAG 系统的常见问题和解决方案？
**30秒答案**：①检索不相关：优化 embedding+重排序；②答案幻觉：加入引用验证；③知识过时：定期更新索引；④长文档处理：分块+层次检索。

### Q10: E5 和 BGE 嵌入模型的区别？
**30秒答案**：E5（微软）：通用文本嵌入，支持 instruct 前缀。BGE-M3（BAAI）：多语言+多粒度+多功能（dense+sparse+ColBERT 三合一）。BGE-M3 更全面但模型更大。
