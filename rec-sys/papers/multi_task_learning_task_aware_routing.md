# Multi-Task Learning with Task-Aware Dynamic Routing for Recommendation
> 来源：工业论文 | 领域：推荐系统 | 学习日期：20260327

## 问题定义
工业推荐系统需要同时优化多个目标：CTR（点击率）、CVR（转化率）、点赞、收藏、分享等。
现有多任务学习方案的挑战：
1. **跷跷板效应（Seesaw Phenomenon）**：优化一个任务会损害另一个任务
2. **专家共享不灵活**：MMoE等方案的专家门控是软的，实际上每个专家都参与了所有任务
3. **任务冲突梯度**：不同任务的梯度方向冲突，导致共享参数更新方向不确定
**目标**：设计task-aware动态路由机制，让每个任务动态选择最合适的专家组合。

## 核心方法与创新点

### 1. Task Embedding引导的Expert路由
每个任务有一个可学习的task embedding，用于动态决策expert选择：

$$g_k^{(t)} = \text{softmax}(\text{MLP}([e_{task}^{(t)}; x]))$$

其中 $e_{task}^{(t)}$ 是第t个任务的embedding，x是输入特征，$g_k^{(t)}$ 是第k个expert对任务t的权重。

### 2. 硬路由（Hard Routing）+ 软路由（Soft Routing）
- **软路由**：训练阶段，用Gumbel-Softmax实现可微的离散选择
- **硬路由**：推理阶段，Top-K选择，只激活K个专家（K通常为2-3）
推理时只计算被选中的专家，降低延迟。

### 3. 任务特定的负样本
不同任务的负样本构建策略不同：
- CTR：展示未点击 = 负样本
- CVR：点击未转化 = 负样本（不包含未点击）
- 点赞：曝光未点赞 = 负样本

### 4. 梯度手术（Gradient Surgery）
检测任务间梯度冲突，对冲突梯度进行投影：
$$g_t' = g_t - \sum_{i \neq t} \max(0, \frac{g_t \cdot g_i}{||g_i||^2}) g_i$$

## 实验结论
- **CTR提升**：+2.3%
- **CVR提升**：+3.1%
- **点赞提升**：+4.7%
- **收藏提升**：+2.8%
- 硬路由推理延迟比全量软路由减少40%
- 消融：任务embedding路由比Gate MLP路由提升+1%

## 工程落地要点
1. **Expert数量**：通常8-16个Expert，每次激活Top-2或Top-3
2. **任务采样权重**：训练时按任务难度动态调整批次采样比例
3. **负样本策略**：CVR必须用ESMM框架（只在点击样本上训练），否则样本选择偏差严重
4. **在线路由缓存**：用户的任务路由结果可以短时间缓存（5-10分钟），减少Expert推理次数
5. **新任务接入**：新增任务时，先冻结现有Expert，只训练新任务的task embedding和输出层

## 面试考点
Q1: MMoE和Task-Aware Routing的核心区别？
A: MMoE：门控网络的输入只有input feature，同一个输入的门控结果对所有任务通用（softmax(W·x)），Expert是"软选择"的。Task-Aware Routing：门控输入包含task embedding，不同任务对同一input可能选择不同的Expert组合，更个性化。推理时可以用硬路由，只激活被选中的Expert，效率更高。

Q2: 多任务学习中的跷跷板效应如何缓解？
A: (1)任务特定Expert：每个任务有私有Expert，减少直接竞争；(2)梯度手术：投影掉冲突梯度；(3)任务采样调度：动态调整各任务的学习率和批次权重；(4)不确定性加权损失：按任务学习难度自动平衡权重；(5)硬参数共享 + 软参数共享结合：低层共享高层分离。

Q3: CVR任务为什么要特殊处理？
A: CVR（转化率 = 购买/点击）面临样本选择偏差：只有被点击的样本才有转化标签，而点击本身是有偏的（热门物品更容易被点击）。直接训练CVR等于在有偏样本上优化，推理时应用到全部曝光样本上会产生严重偏差。ESMM等框架通过P(CVR) = P(CTR) × P(CVR|CTR)在全空间建模，消除偏差。
