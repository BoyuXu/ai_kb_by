# RecGPT: Next Generation Recommendation with Large Language Models
> 来源：工业论文 | 领域：推荐系统 | 学习日期：20260327

## 问题定义
传统推荐系统高度依赖ID特征（用户ID、物品ID），存在：
1. **冷启动问题**：新用户/新物品无历史ID，表现差
2. **泛化性差**：不同平台的用户行为无法迁移
3. **语义鸿沟**：ID特征不包含语义信息，无法理解物品内容
**目标**：将GPT架构直接应用于工业推荐，利用LLM的语义理解能力提升推荐质量。

## 核心方法与创新点

### 1. Next-Item Prediction with CLM Loss
将推荐建模为自回归语言模型任务：
- 输入：用户历史行为序列（物品标题/描述的自然语言表示）
- 输出：预测下一个最可能交互的物品

$$\mathcal{L}_{CLM} = -\sum_{t=1}^{T} \log P(item_t | item_1, ..., item_{t-1}; \theta)$$

与BERT4Rec的MLM不同，CLM是单向的，更适合在线推理（无需双向上下文）。

### 2. 物品文本化
将结构化物品特征转化为自然语言描述：
```
"[用户最近浏览] 轻薄笔记本电脑 MacBook Pro 16寸 M3 Pro芯片 → 
 [推荐] 苹果原装USB-C数据线 2米"
```
利用LLM对自然语言的理解能力，捕获物品间的语义关联。

### 3. 混合ID+文本表示
纯文本方案召回效率低，工业落地使用混合方案：
$$\text{ItemEmb} = \alpha \cdot \text{IDEmb} + (1-\alpha) \cdot \text{TextEmb}$$
α由物品的历史曝光量决定（热门物品权重偏向ID，冷门/新物品权重偏向文本）。

### 4. 指令微调（Instruction Tuning）
在推荐场景构建指令数据集：
- "给爱好户外运动的用户推荐商品"
- "根据以下购买记录推荐相似商品"
提升模型的指令跟随能力和场景适应性。

## 实验结论
- **Click提升**：+9%（在线A/B测试）
- **新用户CTR**：+23%（冷启动效果显著）
- **Recall@20**：+15%（冷门物品召回）
- 在MIND、Amazon等公开数据集上NDCG@10提升+0.05~+0.12

## 工程落地要点
1. **文本生成成本**：物品描述文本需要离线预处理，在线不能实时生成
2. **模型大小选择**：7B参数的LLM已经效果好，成本可控；不必上70B
3. **量化部署**：INT4量化后召回质量损失<1%，但推理速度提升4x
4. **混合比例α**：新物品上线前α=0（纯文本），随曝光积累逐渐增加α
5. **Prompt格式**：物品描述的格式对效果影响显著，需要仔细设计

## 面试考点
Q1: RecGPT为什么用CLM（因果语言模型）而不是BERT的MLM（掩码语言模型）？
A: CLM（自左向右生成）天然适合推荐场景：(1)在线推理时，用户历史是已知的左上文，需要预测未来行为，单向因果建模与此匹配；(2)推理速度更快，不需要双向编码；(3)更容易扩展到生成式推荐（直接生成物品）。MLM需要在序列中随机mask，不适合在线流式推荐。

Q2: 如何解决LLM推荐中的幻觉问题（生成不存在的物品）？
A: (1)约束解码：使用Trie树或有限状态机约束生成范围到合法物品ID/标题；(2)检索增强：生成的物品描述通过ANN检索到真实物品；(3)后处理过滤：过滤掉不在候选库中的生成结果；(4)训练约束：在训练数据中只包含真实存在的物品。

Q3: 物品文本化面临哪些挑战？
A: (1)文本质量不一：部分商品描述缺失、错误或过短；(2)语言多样：不同品类的表达方式差异大；(3)实时性：热门物品信息更新频繁，文本表示需要及时更新；(4)长尾物品：长尾物品缺乏丰富的文本描述，效果提升有限。
