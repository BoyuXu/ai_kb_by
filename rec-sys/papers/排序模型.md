## 核心知识体系

### 1. 经典排序模型演进

```
LR → FM → DeepFM → DCN → DCN-V2 → 
DIN → DIEN → BST → SIM → ETA → HiSAC
              ↑                    ↑
          行为序列建模         超长序列扩展
```

### 2. 特征交叉方法

| 方法 | 创新点 | 适用场景 |
|------|--------|---------|
| FM | 二阶特征交叉 | 特征稀疏场景 |
| Deep & Cross (DCN) | 显式高阶交叉 + DNN | 通用 |
| DCN-V2 | Matrix Kernel 替代 vector kernel | 大规模 |
| AutoInt | Attention-based 特征交叉 | 需要可解释性 |
| FiBiNet | SENET 特征重要性 + Bilinear 交叉 | 多场景 |

### 3. 用户行为序列建模

| 方法 | 序列长度 | 核心机制 | 延迟 |
|------|---------|---------|------|
| DIN | 50~200 | Target attention | 低 |
| DIEN | 50~200 | GRU + attention | 中 |
| SIM Hard | 200~1K | 类目 key 检索 | 低 |
| SIM Soft | 200~2K | ANN 检索 | 中 |
| ETA | 1K~5K | Hash-based 检索 | 中 |
| **HiSAC** | **10K+** | 层次稀疏激活压缩 | 中 |

## 2026-03-12 更新

### 本次新增内容

**1. HiSAC - 超长序列建模新突破**
- 来源：arXiv 2026-02-24
- 核心：层次化稀疏激活压缩（两级压缩：兴趣聚合 → 行为精选）
- 意义：序列长度从 5K 扩展到 10K+，AUC 持续提升
- 详细笔记：`../papers/20260312_HiSAC_超长序列推荐系统.md`

**2. CADET - Decoder-Only Transformer for Ads CTR**
- 来源：arXiv 2026-02-11
- 核心：用 GPT 式 causal attention 建模广告上下文，KV Cache 优化推理
- 意义：Decoder-Only 架构首次在工业广告 CTR 场景大规模验证
- 详细笔记：`../papers/20260312_CADET_广告CTR_Decoder_Only_Transformer.md`

**3. Farewell to Item IDs - 语义 Token 排序**
- 来源：arXiv 2026-01-30
- 核心：用 RQ 量化的语义 token 替代 item ID embedding，解决扩展性问题
- 意义：推荐系统"去 ID 化"的工业级验证
- 详细笔记：`../papers/20260312_FarewellItemIDs_语义Token扩展排序.md`
