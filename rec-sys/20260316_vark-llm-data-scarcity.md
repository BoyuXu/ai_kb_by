# Combating Data Scarcity in Recommendation: VARK Cognitive Types + LLM

> 来源：arxiv | 日期：20260316 | 领域：rec-sys

## 问题定义

推荐系统依赖大量用户交互数据，但在以下场景面临**数据稀缺**：
- 新用户（冷启动）
- 长尾 item
- 小众垂类平台（教育、医疗）

本文提出将 **VARK 认知模型**（Visual / Auditory / Reading / Kinesthetic 四种学习/认知风格）与 LLM 结合，通过认知类型推断补充稀疏交互数据。

## 核心方法与创新点

1. **VARK 认知类型建模**：
   - 从用户行为特征（浏览时长、内容类型偏好、交互方式）推断用户的 VARK 认知类型。
   - 认知类型作为**结构化先验**，约束用户偏好空间，减少数据需求。

2. **LLM 增强数据生成**：
   - 对数据稀缺用户，用 LLM（GPT/Llama）根据其认知类型 + 有限交互历史，**生成合成交互序列**。
   - 生成的序列作为弱监督数据，与真实数据混合训练。
   - LLM Prompt 模板：`用户类型: Visual, 历史交互: [item1, item2], 请推断该用户可能感兴趣的下10个内容`。

3. **认知感知协同过滤**：
   - 将 VARK 类型 embedding 拼接到用户 embedding，在矩阵分解和 CF 模型中使用。
   - 同认知类型用户之间的协同过滤信号更强，有效缓解数据稀缺。

4. **知识蒸馏**：将 LLM 生成的偏好知识蒸馏到轻量级推荐模型，生产环境不需要调用 LLM。

## 实验结论

- MovieLens-100K（极度稀疏设定）：NDCG@10 相比 CF 基线 +11.3%，相比 LLM 直接推荐 +4.2%。
- 教育推荐场景（K12 学习平台）：准确率 +8.7%，学习效果（后测成绩）+5.2%。
- 冷启动用户（<5 条交互）提升最显著：+19%。

## 工程落地要点

- VARK 类型推断可用简单问卷（4 道题）或行为推断，无需用户主动填写。
- LLM 生成数据质量参差不齐，需过滤：用 CF 模型打分，丢弃与真实分布偏差过大的生成样本。
- 教育/医疗场景的认知建模尤其有价值，这些领域数据天然稀缺且用户特征结构化。
- 隐私合规：LLM 生成是基于类型而非具体用户数据，隐私风险低。

## 面试考点

- Q: 推荐系统冷启动有哪些解决方案？
  A: (1) 基于内容的推荐（item 特征）；(2) 知识图谱补充关系；(3) 元学习（MAML）快速适应；(4) 跨域迁移（其他平台数据）；(5) LLM 生成/增强（本文）；(6) 问卷/引导流程收集初始偏好。

- Q: LLM 用于推荐系统的主要范式有哪些？
  A: (1) LLM as Ranker（直接排序候选集）；(2) LLM as Feature Extractor（提取语义特征）；(3) LLM as Data Augmentor（生成训练数据，本文）；(4) LLM as Conversational Rec（对话式推荐）；(5) LLM as Scoring Function（zero-shot 评分）。

- Q: 如何评估 LLM 生成的合成数据质量？
  A: (1) 分布一致性（KL 散度与真实分布比较）；(2) 下游任务指标（用合成数据训练后的推荐效果）；(3) 人工评测（采样审核）；(4) 过滤机制（用已有模型打分，去除低质量样本）。
