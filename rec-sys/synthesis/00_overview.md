## 二、各环节核心技术点详解

### 2.1 召回层 (Recall Layer)

#### 核心任务
从海量候选池(百万~亿级)中快速筛选出千~万级候选集

#### 代表技术

| 技术方向 | 核心模型/算法 | 特点 | 适用场景 |
|---------|--------------|------|---------|
| **协同过滤** | UserCF, ItemCF, SVD++ | 简单有效，解释性强 | 用户/物品行为丰富的场景 |
| **向量召回** | DSSM, Youtube DNN, MIND | 语义匹配能力强 | 内容推荐、语义搜索 |
| **图召回** | PinSAGE, GraphSAGE, LightGCN | 高阶关系建模 | 社交网络、知识图谱 |
| **序列召回** | SDM, SASRec, Bert4Rec | 序列兴趣建模 | 短视频、新闻推荐 |
| **树召回** | TDM, JTM | 全库高效检索 | 超大规模候选集 |

#### 关键技术点
- **Embedding学习**: 双塔模型、对比学习
- **向量检索**: FAISS、Milvus、ANN算法(HNSW/IVF)
- **多路融合**: 静态权重、 learned fusion

---

### 2.2 粗排层 (Coarse Ranking)

#### 核心任务
对召回候选集进行快速初筛，将数量级从千~万降至百级

#### 代表模型

| 模型类型 | 代表算法 | 特点 |
|---------|---------|------|
| **双塔模型** | DSSM, Sentence-BERT | 用户/物品向量预计算 |
| **轻量NN** | 浅层MLP, CIN-lite | 速度快，精度适中 |
| **蒸馏模型** | 知识蒸馏小模型 | 接近大模型效果 |
| **树模型** | LightGBM-fast | 可解释性强 |

#### 关键技术点
- **特征选择**: 仅使用高频、高区分度特征
- **模型压缩**: 量化、剪枝、蒸馏
- **延迟优化**: <10ms 响应要求

---

### 2.3 精排层 (Fine Ranking)

#### 核心任务
对粗排结果进行精准打分排序，输出CTR/CVR等预估概率

#### 经典模型演进

```
2016 ──── Wide&Deep ──────────────────────────────┐
         (LR + DNN, 记忆+泛化)                      │
                                                  │
2017 ──── DeepFM ────────────────────────────────┤
         (FM + DNN, 自动特征交叉)                   │
                                                  │
2018 ──── DIN ───────────────────────────────────┤
         (Attention机制, 用户兴趣动态建模)           │
                                                  ├── 特征交叉
2018 ──── xDeepFM ───────────────────────────────┤    + Attention
         (CIN, 显式高阶特征交互)                    │    + 序列建模
                                                  │
2019 ──── AutoInt ───────────────────────────────┤
         (Multi-head Self-Attention)               │
                                                  │
2020 ──── DCN-V2 ────────────────────────────────┘
         (Cross Network改进)

2018 ──── MMoE ──────────────────────────────────┐
         (Multi-gate Mixture-of-Experts)          │
                                                  ├── 多任务学习
2020 ──── PLE ───────────────────────────────────┘
         (Progressive Layered Extraction)

2018 ──── DIEN ──────────────────────────────────┐
         (GRU + Attention, 兴趣演化)               │
                                                  ├── 序列建模
2020 ──── SIM ───────────────────────────────────┘
         (长序列兴趣建模, 两阶段检索)
```

#### 关键技术点

| 技术维度 | 核心内容 |
|---------|---------|
| **特征工程** | 连续特征、离散特征、序列特征、交叉特征 |
| **特征交叉** | 显式交叉(DCN/CIN) vs 隐式交叉(DNN) |
| **序列建模** | Attention、GRU、Transformer、BERT |
| **多任务学习** | 样本权重、梯度均衡(PLE/GradNorm) |
| **损失函数** | Pointwise/Pairwise/Listwise |

---

### 2.4 重排层 (Re-ranking)

#### 核心任务
结合业务规则对精排结果进行多目标优化和多样性控制

#### 关键技术

| 优化目标 | 算法/策略 | 说明 |
|---------|----------|------|
| **多目标融合** | 加权融合、帕累托最优、MOO | CTR/CVR/GMV平衡 |
| **多样性** | MMR、DPP、Determinantal Point Process | 避免结果同质化 |
| **新颖性** | Exploration策略、Bandit算法 | 探索新物品 |
| **疲劳度** | 曝光去重、时间衰减 | 避免重复推荐 |
| **业务规则** | 品牌打散、价格区间、库存过滤 | 业务约束满足 |

#### 经典算法
- **MMR (Maximal Marginal Relevance)**: 相关性与多样性权衡
- **DPP**: 基于行列式点过程的多样性建模
- **Bandit**: EE平衡(Thompson Sampling, LinUCB)

## 三、推荐系统学习路径

### 阶段一：基础入门
1. 协同过滤 (UserCF/ItemCF)
2. 矩阵分解 (SVD/ALS)
3. 基础排序模型 (LR/GBDT)

### 阶段二：深度学习
1. 双塔模型 (DSSM)
2. Wide&Deep / DeepFM
3. Attention机制 (DIN)

### 阶段三：进阶提升
1. 序列建模 (DIEN/Transformer)
2. 多任务学习 (MMoE/PLE)
3. 图神经网络 (GraphSAGE/PinSAGE)

### 阶段四：工程实践
1. 实时特征工程
2. 模型Serving优化
3. A/B测试与效果评估

---

## 四、核心评估指标

| 指标类型 | 具体指标 | 说明 |
|---------|---------|------|
| **准确性** | AUC、GAUC、LogLoss | 排序能力评估 |
| **多样性** | ILS、Coverage | 结果多样性 |
| **新颖性** | Novelty、Surprise | 探索能力 |
| **业务指标** | CTR、CVR、GMV、停留时长 | 实际业务效果 |
