# LIMO: Less is More for Reasoning
> 来源：arXiv:2502.03387 | 领域：llm-infra | 学习日期：20260330

## 问题定义
主流观点认为训练复杂推理能力需要大量高质量数据（如 OpenAI o1 的内部数据规模）。LIMO 挑战这一假设：用极少量精心设计的训练样本（817条），显著提升 LLM 的数学推理能力，揭示推理能力的本质和高效训练机制。

## 核心方法与创新点
1. **少即是多假设**：基础模型已经通过预训练积累了大量数学知识（LIMO Hypothesis），SFT 只需要"激活"这些知识，而非"教会"新知识。因此极少量高质量样本就足够。
2. **样本质量 > 数量**：LIMO 的 817 条训练数据精选自难度高、解题步骤清晰、覆盖多种推理策略的数学题，平均每题 8+ 步骤解析。
3. **解题链质量标准**：每个训练样本要求：(a) 解题步骤完整（无跳步）；(b) 中间推理可验证；(c) 包含不同类型推理（代数/几何/组合）。
4. **对比实验证明**：用 817 条 LIMO 数据 vs 100K+ 标准数学数据 SFT，LIMO 在多个 benchmark 更优（数据效率提升 100x+）。
5. **泛化验证**：LIMO 训练的模型在非数学推理任务（逻辑/编程）上也有迁移改善，说明学到的是通用推理策略。

## 实验结论
- AIME 2024：LIMO-7B 得分 57.1%，对比数学专用 SFT 模型 43.2%（+14%）
- AMC 23：86.3%（接近 GPT-4 水平）
- 仅 817 条数据，训练 3 小时（单机 8×A100），极高效率

## 工程落地要点
- 数据质量 pipeline：人工审查 + 验证器（数学自动验证）+ 多样性采样（覆盖不同题型）
- 训练时 lr 建议比一般 SFT 低（1e-5 vs 2e-5），防止过拟合少量数据
- 基座模型选择关键：预训练阶段数学数据比例高的模型（Qwen2.5-Math、DeepSeek-Math）效果更好
- 可复用于其他推理领域（法律/逻辑）：同样少量高质量数据 SFT

## 常见考点
- Q: LIMO Hypothesis 的核心观点？
  - A: 复杂推理能力的"基础设施"在预训练中已建立，SFT 只需提供少量高质量示范来"激活"和"引导"这些能力，而非从零训练推理
- Q: 为什么数据质量比数量更重要（在 LIMO 场景）？
  - A: 低质量数据（跳步、错误推理）会引入噪声；高质量数据明确示范推理策略；基座模型已有知识，不需要大量重复确认
- Q: 如何评估推理数据的质量？
  - A: ① 解答正确性（数学验证器）；② 步骤完整性（无魔法跳步）；③ 多样性（题型/难度覆盖）；④ 解题策略清晰（人类可读）
