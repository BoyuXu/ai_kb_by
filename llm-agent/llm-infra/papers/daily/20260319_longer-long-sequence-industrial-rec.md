# LONGER: Scaling Up Long Sequence Modeling in Industrial Recommenders
> 来源：https://arxiv.org/abs/2505.04421 | 日期：20260319

## 问题定义
工业推荐系统中用户历史行为序列极长（数万条），Transformer的O(n²)注意力复杂度使长序列建模在工业规模下不可行。LONGER专门为工业推荐场景设计可扩展的超长序列建模方案，突破序列长度限制同时保证推理效率。

## 核心方法与创新点
1. **分层序列压缩**：将超长历史分段（如每100条）压缩为块级表征，再在块级别做注意力，将O(n²)降为O(n/k)²
2. **重要性感知采样**：用轻量评分网络对历史行为打重要性分，优先保留与当前候选相关的行为
3. **异步预计算**：将历史序列的压缩表征异步预计算并缓存，在线推理只需读缓存+少量实时计算
4. **稀疏注意力模式**：结合局部注意力（最近行为）+全局注意力（关键历史），近似全注意力效果
5. **流式更新**：用户新行为实时触发增量更新，避免全量重计算历史表征

## 实验结论
- 在千万级用户、平均历史长度10000+的工业场景，建模长度从200扩展到10000，AUC提升约1.2%
- 推理延迟从不可用（纯Transformer）降至<10ms（LONGER）
- 内存占用比全注意力降低约80%

## 工程落地要点
- 块级别缓存需要定期失效和更新，建议设置TTL（如1小时）
- 重要性评分网络需要轻量（<1ms），否则抵消压缩带来的收益
- 序列长度上限设置需权衡：更长历史收益递减，超过临界点边际收益可忽略
- 可与SIM（Search-based Interest Model）结合：先检索相关历史，再建模

## 常见考点
**Q: 超长序列推荐的主要工程挑战？**
A: （1）计算复杂度O(n²)随序列长度爆炸；（2）内存占用：万条历史的embedding占用GB级内存；（3）数据存储：历史行为的高效存储和检索；（4）实时更新：新行为实时融入历史。

**Q: SIM（Search-based Interest Model）的思路？**
A: 将超长用户历史存入索引，当推荐某候选item时，通过检索（category/item-based）从历史中召回最相关的top-K行为，用这K条行为做Transformer建模，避免全序列计算。

**Q: 线性注意力（Linear Attention）如何降低复杂度？**
A: 将softmax(QK^T)V的计算顺序改为Q(K^TV)，利用矩阵乘法结合律，将O(n²d)降为O(nd²)。代价是近似误差，需要核函数设计（如ELU+1）。
