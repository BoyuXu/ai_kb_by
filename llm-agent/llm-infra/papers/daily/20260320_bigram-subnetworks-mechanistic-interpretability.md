# Bigram Subnetworks: Minimal Circuits for Predicting Next Tokens

> 来源：https://arxiv.org/abs/2504.15471 | 日期：20260320 | 领域：llm-infra

## 问题定义

机制可解释性(Mechanistic Interpretability)旨在理解Transformer如何预测下一个token。当前研究面临的核心问题：
- 现有电路研究多关注复杂行为(induction heads, IOI等)，缺乏对**基础预测行为**的系统性理解
- 电路验证通常只评估"必要性"(necessity)，缺乏"充分性"(sufficiency)检验
- 需要一个**简单、定义明确、覆盖全输入空间**的起点来构建最小可解释电路

**Bigram预测**——仅基于当前token预测下一个token P(w_i|w_{i-1})——是满足上述条件的理想研究对象：
- 最简单非平凡的next token预测
- LLM在预训练早期就过拟合到bigram分布
- 可量化验证电路的充分性和必要性

## 核心方法与创新点

### 1. Bigram子网络的定义

Bigram子网络 = 模型参数的稀疏子集，满足：
- **充分性**：仅激活这些参数时，模型输出与bigram分布的surprisal相关性 r > 0.95
- **必要性**：消融这些参数会显著损害语言建模性能
- **最小性**：通过L1稀疏约束确保无冗余参数

### 2. 连续稀疏化发现方法 (Continuous Sparsification)

```
Loss(M, x) = CrossEntropy(P_bigram(x), MaskedModel_M(x)) + λ||M||₁/|M|
```

- 为每个参数学习mask值 m ∈ (-∞, +∞)，通过sigmoid映射到(0,1)
- 训练过程中逐渐降低temperature，使mask趋于二值化
- 冻结原始参数，仅优化mask，保证子网络是原模型的子集

### 3. 核心发现

#### 发现1：Bigram子网络规模与模型大小无关
- **惊人结果**：无论模型大小(Pythia 160M ~ 1B, GPT-2 small ~ large)，达到r>0.95仅需约**10M活跃参数**
- 在Pythia 1B中，仅占**0.17%**的非嵌入参数即可重建bigram预测

#### 发现2：高度集中于第一层MLP
- **82.1%**的bigram子网络参数位于第一层Transformer的MLP
- 即使在随机初始化模型中，第一层MLP也集中了大部分bigram相关参数
- 早期训练(checkpoint 0~4K)即形成这一结构，之后保持稳定

#### 发现3：重现残差流的本质特征
- 完整模型在第一层产生从"当前token"到"下一token"表示的剧烈旋转
- Bigram子网络**重现了这一旋转**：
  - 到输入表示的旋转：0° → 47-56°
  - 到输出表示的旋转：70° → 34-46°
- 随机同规模子网络无此特性，证明这是功能性结构非巧合

#### 发现4：与最优剪枝子网络高度重叠
- 以最小化原模型输出损失为目标训练的"最优子网络"，与bigram子网络有**15.3倍**于随机的参数重叠
- 38%的bigram子网络参数被包含在最优子网络中
- 消融实验：消融bigram子网络的性能损失 ≈ 消融最优子网络 >> 消融随机子网络

## 实验结论

### 模型覆盖
| 模型 | 参数规模 | Bigram相关性 | 子网络占比 |
|------|----------|-------------|-----------|
| Pythia 160M | 160M | r=0.96 | ~8.5% |
| Pythia 1B | 1B | r=0.959 | **0.17%** |
| GPT-2 small | 124M | r=0.95 | ~10M参数 |
| GPT-2 large | 774M | r=0.96 | ~0.10% |

### 预训练动态
- Bigram子网络在**模型过拟合bigram后仍持续存在**(checkpoint 4K+)
- 但随训练进行，维持相同bigram相关性需要更多参数(从500K→10M)
- 推测：bigram相关参数在后期训练中被"征用"参与更复杂预测

### 消融影响
```
Pythia-1B在OSCAR数据集上的loss变化：
- 基线: 2.45
- 消融随机子网络(同规模): 2.46 (+0.01)
- 消融bigram子网络: 4.82 (+2.37) 
- 消融最优子网络: 4.75 (+2.30)
```
消融bigram子网络导致生成文本退化为单一高频token重复。

## 工程落地要点

### 1. 模型压缩新视角
- Bigram子网络仅占0.2%参数却承担基础预测功能
- 提示**分层压缩策略**：第一层需高精度保留，上层可更激进剪枝
- 与结构化剪枝结合：优先保护第一层MLP

### 2. 预训练效率优化
- Bigram能力在极早期(4K steps)已充分学习
- 可考虑**课程学习**：早期聚焦bigram，逐步引入长程依赖
- 第一层MLP的"过参数化"可能是计算效率优化点

### 3. 可解释性工具链
```python
# 实践建议：提取bigram子网络作为基线
1. 对目标模型训练bigram子网络(λ∈[10,100])
2. 比较完整模型与bigram子网络的残差流相似性
3. 识别超出bigram能力的"增量电路"
4. 逐层叠加分析更复杂行为(induction heads等)
```

### 4. 与现有工作的关联
| 技术 | 关联 |
|------|------|
| **Tuned Lens** | 验证第一层到输出空间的映射 |
| **MLP as Key-Value Memories** | 第一层MLP存储bigram知识 |
| **Superposition** | 小参数实现bigram可能是超位置编码的结果 |

## 常见考点

**Q1: 什么是电路的"必要性"和"充分性"？为什么两者都需要验证？**

A: 必要性：消融该电路会损害目标行为；充分性：单独激活该电路可产生目标行为。仅验证必要性可能导致误判——电路可能只是完整行为的一个子操作(necessary but not sufficient)，或存在冗余路径(sufficient but not necessary)。两者结合才能准确界定电路功能边界。

**Q2: 为什么bigram子网络的规模(10M参数)与总模型大小无关？**

A: 可能的解释：(1) Bigram是离散有限映射(词汇表^2)，所需容量有上限；(2) 更大模型将更多参数分配给长程依赖和复杂推理，而非基础bigram统计；(3) 存在"超位置"(superposition)机制，小参数通过叠加编码bigram信息。

**Q3: 第一层MLP为何集中了bigram能力？**

A: 机制分析表明第一层MLP执行从"当前token嵌入"到"下一token预测"的映射：(1) 注意力在bigram任务中无需查询历史(Q/K作用有限)；(2) MLP的key-value记忆结构天然适合存储P(next|current)映射；(3) 残差流分析显示第一层即完成当前到下一token的表示旋转。

**Q4: 该研究对"大模型是否只是记忆bigram"的争论有何启示？**

A: 虽然bigram子网络性能强大且必要，但：(1) 完整模型远超bigram能力；(2) 最优子网络与bigram子网络仅38%重叠；(3) 消融bigram后模型虽退化但未崩溃。结论：bigram是必要基础而非全部，LLM在此基础上构建了更复杂能力。

**Q5: 如何应用该研究构建更大规模的可解释模型？**

A: 采用"积木式"方法：(1) 以bigram子网络为最小可解释基线；(2) 逐层添加induction heads等已知电路；(3) 每次添加验证性能增量和充分性；(4) 目标：构建由可解释电路组成的子网络，逼近完整模型性能。该研究已开源至1B模型的bigram子网络作为起点。
