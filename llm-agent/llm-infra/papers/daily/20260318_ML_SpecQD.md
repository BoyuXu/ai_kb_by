# ML-SpecQD: Multi-level Speculative Decoding with Quantized Drafts
> 来源：ML-SpecQD 论文 | 日期：20260318 | 领域：llm-infra

## 问题定义
大型语言模型推理延迟高，Autoregressive 解码每步需要完整前向传播。Speculative Decoding 通过小 draft 模型预生成 token、大模型并行验证来加速推理，但 draft 模型本身的推理开销和内存占用仍是瓶颈。ML-SpecQD 结合多层次投机解码与量化 draft 模型，进一步压缩延迟与显存开销。

## 核心方法与创新点
1. **多层次草稿生成（Multi-level Draft）**：构建层次化草稿树，不同层次使用不同精度的 draft 模型（4bit/8bit 量化），越浅层生成越多候选 token，越深层剪枝低概率分支。
2. **量化 Draft 模型**：对 draft 模型进行 INT4/INT8 量化，显存减少 50-75%，同时草稿生成吞吐大幅提升。
3. **自适应接受策略**：根据 draft token 与 target 模型分布的 KL 散度动态调整接受阈值，保证生成质量的同时最大化加速比。
4. **Tree Attention 优化**：利用树形注意力掩码并行验证多条候选序列，减少 target 模型调用次数。

## 实验结论
- 在 LLaMA-2-70B 上，相比普通 Autoregressive 解码加速 3.2-4.1x
- 相比标准 Speculative Decoding（非量化）进一步提升 1.3-1.6x
- 量化 draft 模型显存节省约 60%，几乎不影响 token 接受率（< 1% 下降）
- 在代码生成、摘要等任务上均有稳定提升

## 工程落地要点
- Draft 模型选择：通常选用 target 模型同系列的小模型（如 7B draft → 70B target）
- 量化精度权衡：INT4 量化加速最大但接受率略低，推荐 INT8 作为生产默认
- Tree Attention 实现需定制 CUDA kernel，可复用 vLLM / TensorRT-LLM 现有实现
- 批处理场景下加速比下降明显（批大小 > 8 时效果边际递减），适合在线低延迟推理

## 常见考点
**Q: 解释 Speculative Decoding 的基本原理和加速原理？**
- 要点：draft 模型快速生成 K 个候选 token → target 模型并行验证 → 接受/拒绝 → 平均每次 target 前向获得多个 token
- 关键：接受率（acceptance rate）是核心指标，取决于 draft 与 target 分布对齐程度

**Q: 为什么量化 draft 模型不会大幅降低最终输出质量？**
- draft 模型只负责生成候选，最终由 target 模型验证和纠正，draft 质量下降只影响接受率，不影响输出分布正确性
