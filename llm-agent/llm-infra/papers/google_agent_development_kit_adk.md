# Google Agent Development Kit (ADK)

> 来源：https://github.com/google/adk-docs | 领域：LLM基础设施 | 学习日期：20260331

## 问题定义

构建生产级LLM Agent需处理工具编排、状态管理、错误恢复、多Agent协作等复杂工程问题，缺乏标准化开发框架。

## 核心方法与创新点

1. **声明式Agent定义**：YAML/配置定义Agent角色、工具、规则
2. **工具注册中心**：统一管理和注册可调用工具
3. **状态机编排**：基于状态机管理执行流程
4. **多Agent协作协议**：定义通信、委托、结果聚合机制
5. **观测与调试**：内置trace、log、replay功能

## 实验结论

相比手写Agent代码，开发效率提升3倍，Agent可靠性（任务成功率）提升15%。

## 工程落地要点

- 声明式配置降低开发门槛
- 状态机提供可预测执行流程
- 内置重试和错误恢复
- 与Google Cloud深度集成
- 开源版本支持多种LLM后端

## 常见考点

1. **为什么需要Agent框架？** 减少重复工程、标准化接口、提升可靠性
2. **ADK vs LangChain/AutoGen？** ADK更偏声明式和生产级
3. **状态机在Agent中的作用？** 控制执行、断点恢复、限制无限循环
