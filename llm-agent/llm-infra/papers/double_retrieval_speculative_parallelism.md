# Double: Double Retrieval Speculative Parallelism

> 来源：https://arxiv.org/abs/2601.05524 | 领域：LLM基础设施 | 学习日期：20260331

## 问题定义

投机解码中draft模型准确率限制了加速比。如何提升draft质量以获得更高加速比？

## 核心方法与创新点

1. **双重检索投机**：同时使用小模型预测和检索缓存两个draft源
2. **并行验证**：两个draft源候选token并行提交target验证

$$\text{Accept}(t) = \min(1, \frac{P_{target}(t)}{\max(P_{draft_1}(t), P_{draft_2}(t))})$$

3. **检索缓存构建**：基于n-gram匹配和语义相似度从历史生成中检索
4. **自适应源选择**：动态调整两个draft源权重

## 实验结论

相比单一draft投机解码，平均接受长度提升40%，端到端加速比从2.5x提升到3.5x。

## 工程落地要点

- 检索缓存可基于现有KV Cache实现
- 并行验证充分利用GPU带宽
- 适合重复性高的任务（代码、模板文本）
- 缓存管理影响内存开销

## 面试考点

1. **投机解码基本原理？** 小模型快速draft，大模型一次verify
2. **双源比单源好？** 两个独立源互补，整体接受率更高
3. **检索缓存命中率？** 代码生成场景可达50%+
