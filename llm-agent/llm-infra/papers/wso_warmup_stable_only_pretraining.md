# Warmup-Stable-Only (WSO): Pre-training LLM without Learning Rate Decay

> 来源：arXiv 2603.16127 | 领域：llm-infra | 学习日期：20260402

## 问题定义

传统 LLM 预训练使用 cosine 学习率衰减调度，这要求预先确定训练步数，一旦需要延长训练或更换数据就必须重新开始。WSO 提出不需要学习率衰减的预训练方法，使训练过程可以随时暂停和恢复。

## 核心方法与创新点

1. **三阶段调度**：Warmup → Stable → Only 三阶段，其中 Stable 阶段使用恒定学习率
2. **无衰减训练**：省略传统的 cosine decay 阶段，恒定学习率直到训练结束
3. **可恢复训练**：因为没有学习率衰减的耦合，训练可以在任意步骤暂停并恢复
4. **数据灵活性**：支持训练过程中动态更换训练数据，无需从头开始

## 实验结论

- WSO 在相同训练步数下达到 cosine decay 95-100% 的性能
- 支持无缝恢复训练，节省大量重训练成本
- 允许在训练中期加入新数据源，最终模型质量提升 2-5%

## 工程落地要点

- **学习率选择**：恒定学习率的值需要精心选择，建议通过小规模实验确定
- **梯度噪声**：恒定学习率后期梯度噪声较大，可以通过增大 batch size 缓解
- **检查点管理**：可恢复训练需要完善的检查点保存和加载机制

## 常见考点

1. **Q：为什么传统方法需要学习率衰减？**
   A：衰减帮助模型在训练后期收敛到更精细的局部最小值。但 WSO 发现对于大规模 LLM，恒定学习率的探索性也有价值。
2. **Q：WSO 对大规模训练的实际意义？**
   A：大模型训练经常需要中断恢复（硬件故障、数据更新），WSO 省去了重新调度学习率的麻烦，节省大量时间和资源。
3. **Q：为什么叫 "Only"？**
   A：Only 指的是只有 Warmup 和 Stable 两个阶段，去掉了（Only without）传统的 Decay 阶段。
