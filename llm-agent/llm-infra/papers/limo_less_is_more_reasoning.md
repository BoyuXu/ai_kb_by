# LIMO: Less is More for Reasoning

> 来源：arxiv | 领域：llm-infra | 学习日期：20260328

## 问题定义

主流观点认为训练 LLM 推理能力需要海量数据（数百万条 CoT 样本）。LIMO 挑战这一假设：在预训练阶段已充分编码领域知识的基础模型上，是否可以用极少量数据（仅 817 条样本）激发复杂推理能力？

## 核心方法与创新点

### 1. LIMO 假说（Less-Is-More Reasoning Hypothesis）

> *在预训练阶段已全面编码领域知识的基础模型中，复杂推理可以通过极少量但精心设计的认知过程示范（cognitive templates）激发。*

两个关键因素决定推理激发效率：
- **(1) 预训练知识完整性**：模型预训练时是否充分覆盖了该领域知识
- **(2) 后训练示范质量**：少量样本是否有效展示了认知推理过程

### 2. 数据筛选策略
- 从大型数据集中筛选 817 条高质量数学推理样本（约是同类方法的 1/100）
- 选择标准：问题难度适中（不过难也不过易）、推理链完整展示认知过程、解题路径多样

### 3. 简单 SFT 训练
- 基础模型：Qwen2.5-32B-Instruct（大量预训练数学知识）
- 仅做标准 SFT，无 RL，无特殊技巧
- 关键洞察：基础模型中推理能力已潜在编码，SFT 仅起"解锁"作用

$$\mathcal{L}_{SFT} = -\sum_{t=1}^{T} \log P_\theta(y_t | x, y_{<t})$$

## 实验结论

| 基准 | LIMO (817条) | 对比方法 (83K条) | 提升 |
|------|------------|----------------|------|
| AIME24 | 63.3% | 6.5% | **+56.8%** |
| MATH500 | 95.6% | 59.2% | **+36.4%** |

- **OOD 泛化**：在多个 OOD 基准上平均提升 45.8%，超越用 100× 数据训练的模型
- 表明少量高质量示范可有效提升模型的泛化推理能力

## 工程落地要点

1. **数据质量 >> 数据量**：先有强预训练基座，再投高质量示范，1K条足够
2. **认知模板设计**：SFT 数据要展示完整推理过程，包括问题分解、中间验证、错误纠正
3. **基础模型选择**：LIMO 效果依赖预训练知识充分性，弱预训练模型效果不可复现
4. **快速验证路径**：小规模 SFT 实验先验证假设，再决定是否大规模训练
5. **去噪数据优先**：减少噪声样本比增加数据量更有效

## 面试考点

**Q1: LIMO Hypothesis 的核心观点是什么？**

A: 当基础模型在预训练中已充分掌握领域知识时，只需少量高质量的认知过程示范（cognitive templates）即可激发复杂推理能力，无需海量 CoT 数据。推理能力的门槛不由任务复杂度决定，而由预训练知识完整性和后训练示范质量决定。

**Q2: 为什么 LIMO 用 817 条数据就能超越用 83K 条数据训练的模型？**

A: 因为基础模型（Qwen2.5-32B）已在预训练中掌握了大量数学知识，推理能力潜在于模型中。LIMO 的高质量示范起到"认知模板"作用，引导模型正确使用已有知识，而非从头学习知识。

**Q3: LIMO 对工业界 SFT 数据策略有什么启示？**

A: 在强基座上，精心标注少量高质量 SFT 数据（展示完整推理过程）比批量生成大量低质量数据更有效。应重点关注数据的认知过程完整性，而非数量。

**Q4: LIMO 与 DeepSeek-R1 方法的主要区别？**

A: LIMO 是纯 SFT 方法，不使用 RL；DeepSeek-R1 使用 GRPO 强化学习。LIMO 证明了 SFT + 高质量数据的上限，DeepSeek-R1 通过 RL 探索进一步突破上限，两者互补。
