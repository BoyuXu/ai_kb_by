# A Framework for Formalizing LLM Agent Security

> 来源：https://arxiv.org/abs/2603.19469 | 领域：LLM基础设施 | 学习日期：20260331

## 问题定义

LLM Agent越来越多地被赋予工具调用和自主决策能力，但缺乏形式化安全框架来分析和防范安全风险。

## 核心方法与创新点

1. **威胁模型形式化**：定义Agent安全三个维度——机密性、完整性、可用性
2. **攻击面分类**：Prompt注入、工具滥用、记忆污染
3. **安全属性验证**：基于形式化方法验证Agent行为

$$
\forall s \in S_{reachable}: \text{SafetyPolicy}(s) = \text{True}
$$

4. **防御机制**：输入过滤、工具权限控制、输出审计

## 实验结论

应用防御机制使攻击成功率从45%降到8%，任务完成率仅下降3%。

## 工程落地要点

- 安全框架可作为中间件集成
- 工具权限控制类似操作系统ACL
- 审计日志支持事后分析
- 适合企业级Agent部署

## 面试考点

1. **Agent主要安全风险？** Prompt注入、工具滥用、数据泄露
2. **防范间接Prompt注入？** 输入过滤、上下文隔离、权限最小化
3. **Agent安全与传统安全区别？** Agent行为非确定性，需概率化分析
