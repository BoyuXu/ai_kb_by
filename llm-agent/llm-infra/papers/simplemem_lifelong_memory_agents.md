# SimpleMem: Efficient Lifelong Memory for LLM Agents

> 来源：arXiv 2601.02553 | 领域：llm-infra | 学习日期：20260402

## 问题定义

LLM Agent 缺乏跨会话的持久记忆能力——每次会话从零开始，无法积累用户偏好和历史交互知识。现有记忆方案（如 MemGPT）过于复杂。SimpleMem 提出高效的终身记忆系统，以最简设计实现持久化 Agent 记忆。

## 核心方法与创新点

1. **极简设计**：记忆存储为结构化的 key-value 对，key 是语义摘要，value 是原始交互内容
2. **自动记忆管理**：LLM 自动判断哪些信息值得记忆，自动归纳和压缩旧记忆
3. **分层检索**：先语义检索相关记忆条目，再按时间衰减加权，平衡相关性和新鲜度
4. **记忆合并**：相似的记忆条目自动合并，防止记忆爆炸

## 实验结论

- 在 Agent 长期对话任务上，记忆增强后任务完成率提升 35%
- 记忆检索准确率 87%，无关记忆干扰率 < 5%
- 10 万条记忆的检索延迟 < 20ms

## 工程落地要点

- **存储后端**：轻量场景用 SQLite + FAISS，大规模用 PostgreSQL + pgvector
- **隐私保护**：记忆中可能包含敏感信息，需要加密存储和访问控制
- **记忆过期**：设计合理的过期策略，避免过时信息误导 Agent

## 常见考点

1. **Q：Agent 记忆和 RAG 的区别？**
   A：RAG 检索外部知识库（静态文档），Agent 记忆存储交互历史（动态积累）。记忆是个人化的，RAG 是公共的。
2. **Q：记忆爆炸如何处理？**
   A：三策略——(1) 相似记忆合并；(2) 低重要性记忆淘汰；(3) 旧记忆压缩摘要化。
3. **Q：SimpleMem 的"Simple"体现在哪？**
   A：相比 MemGPT 的复杂内存管理层次，SimpleMem 用简单的 key-value 存储 + 语义检索就实现了主要功能，工程复杂度低得多。
