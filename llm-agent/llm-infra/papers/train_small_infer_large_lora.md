# Train Small, Infer Large: Memory-Efficient LoRA Training for Large Language Models

> 来源：arXiv 2502.13533 | 领域：llm-infra | 学习日期：20260402

## 问题定义

LoRA 微调虽然减少了可训练参数量，但仍需要在显存中加载完整的基础模型（数十到数百 GB），限制了在消费级 GPU 上微调大模型。本文提出在小模型上训练 LoRA 适配器，再无缝迁移到大模型推理的方案。

## 核心方法与创新点

1. **跨规模迁移**：在 7B 模型上训练的 LoRA 适配器可以直接应用到 70B 模型上
2. **层对齐**：设计层映射策略将小模型的 LoRA 层对齐到大模型的对应层
3. **维度适配**：通过投影矩阵将小模型的 LoRA 矩阵适配到大模型的维度
4. **渐进式微调**：先在小模型上粗调，再在大模型上轻量级精调，总显存需求降低 70%

## 实验结论

- 7B→70B 迁移后性能保持 90% 的直接在 70B 上训练的效果
- 显存需求从 160GB 降低到 48GB，可在单张 A100 上完成
- 渐进式微调（小模型粗调 + 大模型精调 1 epoch）达到直接大模型训练 97% 的效果

## 工程落地要点

- **模型家族一致性**：迁移要求大小模型属于同一架构家族（如 LLaMA 系列）
- **层映射策略**：均匀映射 vs 注意力层优先映射，需要根据任务选择
- **精调成本**：渐进式的大模型精调仍需要大显存，但训练步数大幅减少

## 常见考点

1. **Q：为什么 LoRA 可以跨规模迁移？**
   A：LoRA 学习的是任务特定的低秩更新方向，这些方向在同一架构家族的不同规模模型间有一定共享性。
2. **Q：跨规模迁移的局限性？**
   A：不同架构族之间无法迁移；维度差异大时信息损失增加；任务复杂度高时 7B 学到的知识可能不足以指导 70B。
