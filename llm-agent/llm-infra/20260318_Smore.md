# S'more: Structural Mixture of Residual Experts for LLM Finetuning
> 来源：S'more 论文 | 日期：20260318 | 领域：llm-infra

## 问题定义
LLM 微调面临两个核心矛盾：(1) 全参数微调（Full Fine-tuning）效果最好但显存/计算开销极大；(2) LoRA 等参数高效微调（PEFT）显著降低开销但表达能力有限，特别在多任务或领域差异大时性能不足。S'more 提出结构化残差专家混合机制，在参数高效的前提下提升模型容量。

## 核心方法与创新点
1. **残差专家（Residual Experts）**：在 Transformer 的 FFN 层旁并联多个轻量专家（类 LoRA 结构），每个专家学习残差适配，而非替换原始权重。
2. **结构化混合（Structural Mixture）**：不同 Transformer 层分配不同数量的专家（浅层少、深层多），匹配各层对任务的重要性，避免参数浪费。
3. **软路由（Soft Routing）**：使用 token 级门控网络为每个 token 动态分配专家权重（非 hard top-k），梯度更稳定，专家利用率更均衡。
4. **与 LoRA 正交**：S'more 可与 LoRA 叠加使用，进一步提升效果。

## 实验结论
- 在 LLaMA-3-8B/70B 上，同等参数量下 S'more 比标准 LoRA 在 instruction following 任务上提升 3-7%
- 多任务微调场景（8 个任务联合训练）提升最显著，专家分工明确
- 相比全参数微调，S'more 仅用约 5% 参数量达到 90%+ 效果
- 训练显存比全参数微调节省 60-70%

## 工程落地要点
- 专家数量建议：每层 4-8 个专家，过多专家路由开销增大收益递减
- 软路由实现注意负载均衡损失（Load Balancing Loss），防止专家坍缩
- 推理时若需加速，可将激活专家合并回主干（Expert Merging），实现零推理额外开销
- 适合企业多业务线共享基座模型、按业务线独立训练专家的部署模式

## 面试考点
**Q: LoRA 和 MoE 的区别？S'more 如何结合两者优势？**
- LoRA：低秩分解，参数高效但单一适配器容量有限
- MoE：条件激活，容量大但训练不稳定、路由开销大
- S'more：LoRA 式残差专家 + MoE 式路由，参数高效 + 多专家容量，训练稳定

**Q: 多任务微调为什么需要混合专家结构？**
- 不同任务所需的知识表示差异大，单一 LoRA 难以同时适配，专家混合允许模型对不同任务激活不同专家子集
