## 分类体系

```
知识蒸馏（Knowledge Distillation）
│
├── 经典 KD（Hinton 2015）
│   ├── Response-Based（软标签/logits 蒸馏）
│   ├── Feature-Based（中间层特征对齐）
│   └── Relation-Based（样本间关系对齐）
│
├── 结构压缩蒸馏
│   ├── 量化辅助蒸馏（GPTQ / AWQ / LCD）
│   ├── 剪枝辅助蒸馏（DistilBERT / TinyBERT）
│   └── 架构搜索蒸馏（NAS + KD）
│
├── LLM 蒸馏（2023-2026 主流）
│   ├── Off-Policy SFT 蒸馏（用大模型生成数据训小模型）
│   ├── On-Policy 蒸馏（学生生成→对齐教师 logits，KL 散度）
│   ├── 推理链蒸馏（CoT Distillation / DEAR / DeepSeek-R1）
│   └── Strong-to-Weak（Qwen3 两阶段蒸馏）
│
└── 搜广推场景蒸馏
    ├── 排序模型压缩（精排→粗排蒸馏）
    ├── LLM→小模型能力迁移（IDProxy / DEAR）
    └── 生成式推荐（MTGR INT8量化+蒸馏，压缩1/4）
```

## 核心方法演进

### 阶段一：经典 KD（2015-2020）

**Hinton 软标签蒸馏**（Response-Based）：

$$
\mathcal{L}_{KD} = (1-\alpha) \mathcal{L}_{CE}(y, \hat{p}_s) + \alpha T^2 \cdot \text{KL}\left(\sigma\!\left(\frac{z_t}{T}\right) \bigg\| \sigma\!\left(\frac{z_s}{T}\right)\right)
$$

- $T$：温度参数（T>1 软化分布，暴露更多类间关系）
- $\alpha$：平衡硬标签和软标签的权重
- $z_t, z_s$：教师/学生 logits

**关键直觉**：温度 $T=5$ 时，「马」预测中「斑马 0.1%、驴 0.3%」等暗知识（dark knowledge）被放大，学生模型学到类间相似关系，而非只盯住「正确答案」。

**FitNets / TinyBERT（Feature-Based）**：

$$
\mathcal{L}_{feat} = \sum_{l=1}^{L} \left\| \mathbf{h}_s^{(l)} W_l - \mathbf{h}_t^{(l')} \right\|_2^2
$$

中间层特征对齐（需要投影矩阵 $W_l$ 适配维度差异）。

**RKD（Relation-Based）**：

$$
\mathcal{L}_{RKD} = \sum_{(x_i, x_j)} \ell\!\left(d_s(f_s(x_i), f_s(x_j)),\ d_t(f_t(x_i), f_t(x_j))\right)
$$

学生模仿教师对样本对之间的距离关系（角度/距离 kernel）。

### 阶段二：BERT 时代 NLP 蒸馏（2019-2021）

| 模型 | 压缩比 | 方法 | 效果损失 |
|------|--------|------|---------|
| DistilBERT | 40% 参数 | 响应蒸馏 + cos 相似 | GLUE -3% |
| TinyBERT | 87% 参数↓ | 4层：Embedding+Attention+Hidden+Pred | GLUE -3.9% |
| MobileBERT | 4.3x 小 | 瓶颈层 + 渐进式蒸馏 | GLUE -2.4% |
| PKD | 6层 | 耐心蒸馏（多层对齐）| - |

**TinyBERT 的四层对齐**：

$$
\mathcal{L}_{total} = \mathcal{L}_{embd} + \sum_{l=1}^{M} (\mathcal{L}_{attn}^l + \mathcal{L}_{hidn}^l) + \mathcal{L}_{pred}
$$

### 阶段三：LLM 蒸馏范式（2023-2026）

#### 核心挑战

大模型蒸馏面临新问题：
1. **巨大能力差距**：70B → 7B，不是简单 logit 对齐
2. **推理能力蒸馏**：CoT 推理步骤比最终答案更难迁移
3. **数据效率**：数百万高质量样本的生成成本

#### DeepSeek-R1 蒸馏路径

```
DeepSeek-R1-671B (教师)
    ↓ Phase 1: Rejection Sampling
    生成高质量 CoT 数据（含推理链）
    ↓ Phase 2: SFT 蒸馏
R1-Distill-Qwen-7B / 14B / 32B（学生）
```

效果：R1-Distill-7B 在 AIME 上超越 GPT-4o，蒸馏出的推理能力接近大模型。

**关键发现**：蒸馏 CoT 数据时，「涌现行为」（自我反思/回溯）也能被迁移，而不仅仅是表面答案格式。

#### Qwen3 Strong-to-Weak 两阶段蒸馏

**Stage 1 – Off-Policy SFT**：

$$
\mathcal{L}_{SFT} = -\sum_t \log P_{student}(y_t | x, y_{<t})
$$

使用教师模型（Qwen3-32B/235B）生成的完整序列训练学生模型（静态数据集）。

**Stage 2 – On-Policy KL 蒸馏**：

$$
\mathcal{L}_{KD} = \mathbb{E}_{x \sim \mathcal{D}} \left[ \mathbb{E}_{y \sim P_{student}(\cdot|x)} \text{KL}(P_{teacher}(\cdot|x,y_{<t}) \| P_{student}(\cdot|x,y_{<t})) \right]
$$

学生自己生成序列（on-policy），实时对齐教师 logits。

**优势**：On-policy 蒸馏避免训练-推理分布偏移（Exposure Bias），效果比纯 SFT 强。仅需四阶段训练的 **1/10 GPU hours**。

#### LCD 量化蒸馏（2-bit LLM）

$$
\mathcal{L}_{LCD} = \alpha \mathcal{L}_{task} + (1-\alpha) \text{KL}(P_{student} \| P_{teacher})
$$

聚类量化（低比特 codebook）+ 蒸馏恢复精度，实现 2-bit LLM 量化，大幅减少 HBM 占用。

## 推荐搜索场景下的蒸馏

### 搜索重排：DEAR 蒸馏

```
GPT-4 / DeepSeek-R1（70B 教师）
    → 生成 ranking rationale（推理链）
    → SFT 蒸馏到 1.5B 小模型（DEAR）
    → 在线部署，延迟 20-40ms
```

**蒸馏损失**：

$$
\mathcal{L}_{DEAR} = \text{KL}(P_{student}(\text{rank}|q,D) \| P_{teacher}(\text{rank}|q,D))
$$

**效果**：1.5B 模型通过蒸馏接近 GPT-4 的重排效果。原因：蒸馏保留了「ranking rationale」推理模式，小模型学到「怎么判断相关性」而非记住答案。

**工程数据**：
- 蒸馏数据量：约 100K 高质量 (query, doc, rationale) 三元组
- 延迟：20-40ms（vs GPT-4 的 1-3s）
- 效果：NDCG 损失 < 5%

### 广告/推荐：IDProxy 蒸馏

```
多模态 LLM（大，离线）
    → 为冷启动物品生成语义 Embedding
    ↓ 蒸馏（特征对齐）
在线小模型（轻量 Encoder）
    → 实时推理，P99 < 5ms
```

**关键决策**：「离线蒸馏」而非「在线调用」，将 LLM 能力注入在线模型，兼顾效果与延迟。

### 推荐精排：MTGR（美团）

```
MTGR 大模型（双流架构，训练期）
    → INT8 量化 + 知识蒸馏
    → 模型体积压缩到 1/4
    → 线上部署，延迟满足 SLA
```

### 排序模型级联蒸馏（工业通用）

精排 → 粗排 蒸馏（最常见）：

```
精排模型（复杂，200ms）
    → 生成样本的 logit 分布
    → 蒸馏到粗排（简单，5ms）
    → 粗排学精排的判断能力
```

效果：通常 AUC 损失 < 0.5pp，延迟降低 20-40x。

## 大模型蒸馏实战选型

| 场景 | 方法 | 原因 |
|------|------|------|
| 通用对话能力迁移 | Off-policy SFT | 快速，数据一次性生成 |
| 推理能力迁移（数学/代码）| CoT SFT + On-policy KD | 需要涌现行为，on-policy 防偏移 |
| 极致压缩（<4B）| 量化 + KD 联合 | 单独量化精度损失大，KD 补偿 |
| 搜广推重排 | Reasoning 蒸馏（DEAR） | 保留推理链，延迟可控 |
| 冷启动 Embedding | 特征蒸馏（IDProxy） | 对齐语义空间，轻量在线推理 |

## 工程实践要点

### 1. 温度参数 T 的选择

- **T=1**：标准 softmax，分布尖锐，暗知识弱
- **T=2~5**：平衡暗知识与信息量，BERT 蒸馏常用 T=2
- **T=10+**：分布趋近均匀，有利于类间关系迁移，但信噪比下降
- **实践**：和 $\alpha$ 联合搜索，通常 T=4, $\alpha=0.5$ 是较好起点

### 2. On-policy vs Off-policy 蒸馏

| 维度 | Off-policy SFT | On-policy KL |
|------|----------------|--------------|
| 数据 | 教师生成（静态）| 学生生成（动态）|
| 分布偏移 | 有（Exposure Bias）| 无 |
| 计算成本 | 低（离线生成一次）| 高（需实时教师推理）|
| 效果 | 好 | 更好（Qwen3 实测）|
| 适用 | 大多数场景首选 | 对效果极致追求时 |

### 3. 能力差距过大时的课程学习

当教师 (70B) 和学生 (1.5B) 差距悬殊时：

```
Step 1: 中间教师（7B）蒸馏到 1.5B（减小跨度）
Step 2: 1.5B 直接对齐 70B（已有基础）
或:
Step 1: 先 SFT 强化学生基础能力
Step 2: 再 KD 对齐教师分布
```

### 4. 蒸馏数据质量 >> 数量

DEAR 的实践数据：
- 100K 高质量 (q, doc, rationale) >> 1M 低质量样本
- 教师生成的推理链质量直接决定蒸馏上限
- 过滤策略：保留教师「高置信度」的样本（教师自身 logit entropy 低）

### 5. 推荐系统蒸馏的独特挑战

1. **ID 特征不迁移**：精排有数十亿 ID Embedding，蒸馏到粗排时 ID 空间不同 → 解决：蒸馏行为层面的 logit，不蒸馏 Embedding 本身
2. **样本不均衡**：正负样本比例悬殊（1:100+）→ 解决：重采样 + focal 蒸馏损失
3. **延迟约束**：在线推理 P99 < 5ms，蒸馏模型不能太大 → 解决：分层蒸馏（精排→中排→粗排）

## 面试常见考点

**Q1: 为什么知识蒸馏用软标签比硬标签效果好？**

A: 软标签携带「暗知识」（dark knowledge）。以图像分类为例，「猫」的预测中「虎 0.2%、豹 0.1%」的概率不为零，反映了类间相似关系。学生模型不仅学到「这是猫」，还学到「猫和虎豹相似」。硬标签是 one-hot，丢弃了这些信息。数学上，软标签的交叉熵损失等价于最小化 KL 散度，让学生的概率分布更接近教师的知识分布。

**Q2: TinyBERT 和 DistilBERT 的核心区别？**

A: 两个维度：① **蒸馏层次**：DistilBERT 只做 logit（response-based）+ 隐层余弦对齐；TinyBERT 全面对齐 Embedding、每层 Attention 矩阵、Hidden States、最终 logit（四层对齐），更充分迁移中间知识。② **训练策略**：TinyBERT 分两阶段（通用蒸馏 + 任务蒸馏），在下游任务上进一步对齐，DistilBERT 只做通用蒸馏。结果：TinyBERT 在相同压缩比下效果更好，但训练成本更高。

**Q3: DeepSeek-R1 蒸馏为什么能保留推理能力？**

A: 核心是蒸馏了「推理链」而非只有最终答案。R1-671B 生成的 CoT 数据包含完整的推理过程（自我反思、回溯、错误校正），学生模型在 SFT 时学到了这些推理 pattern。实验表明，仅蒸馏最终答案的小模型推理能力远差于蒸馏 CoT 数据的版本。直觉上：与其让学生记答案，不如让学生学做题方法。

**Q4: On-policy 蒸馏比 Off-policy 好在哪里？Qwen3 怎么实现？**

A: Off-policy 蒸馏：学生在教师生成的数据上 SFT，推理时学生自己生成数据，存在分布偏移（Exposure Bias）。On-policy：学生先生成序列，再让教师打分并用 KL 散度对齐，消除了分布偏移。Qwen3 的实现：Phase 1 用教师数据做 Off-policy SFT 给学生打基础；Phase 2 学生自己采样序列，与教师 logits 做 token 级 KL 对齐，效率 1/10 vs 完整训练。

**Q5: 搜广推中精排蒸馏粗排，如何处理 ID 特征不一致的问题？**

A: 两种策略：① **只蒸馏行为 logit**：粗排只学精排的打分结果（pairwise 或 pointwise），不涉及具体 Embedding 层，天然回避 ID 空间不同问题。② **特征空间桥接**：将精排的高层表示（不含原始 ID 的语义 Embedding）作为蒸馏目标，粗排用更简单的特征学习逼近精排的语义空间。工业实践中方法①更常见，稳定可靠。

**Q6: 什么时候蒸馏比直接训练小模型更好？**

A: 三种典型场景：① **软标签有信息增益**：小数据集时硬标签不足，教师软标签有正则效果（类似数据增强）。② **能力差距大**：小模型独立训练无法拟合复杂模式，教师引导减小优化难度（如 1.5B → 70B 的 CoT 能力）。③ **中间知识有价值**：特征对齐类任务，教师的中间表示携带结构信息（如 BERT 的 attention 矩阵反映语言依存关系）。反例：若教师和学生分布差距极大（如自然语言 → 代码），蒸馏可能无效。
