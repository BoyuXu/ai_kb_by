# Agent 失败模式：踩坑记录与解法

## 失败概率的数学

单步成功率p时，n步任务成功率 = p的n次方。

p=95%时：5步=77%，10步=60%，20步=36%。

要让20步任务>95%，单步需>99.7%。Agent系统需要极其严格的错误处理。

## 失败模式1：工具调用级联失败

第5步API返回空，Agent未处理异常，后续8步基于错误数据全部失败。

解法：工具调用后校验返回格式，异常时显式重试（最多3次），关键步骤设置早停。

## 失败模式2：目标漂移

任务是"找便宜机票"，搜索过程中Agent开始推荐便宜酒店，忘了原始目标。

解法：每步prompt开头重复原始目标，明确列不能做的事，每5步做回顾检查。

## 失败模式3：幻觉工具调用

LLM调用了不存在的参数或工具（把训练数据里见过的API pattern幻觉出来）。

解法：JSON Schema校验，工具描述精简，用结构化Function Calling格式。

## 失败模式4：Context溢出

长任务到一半context满了，LLM开始重复已做过的步骤。

解法：每N步做进度总结压缩，中间结果写文件不放context，分解任务为独立短任务。

## 失败模式5：过度思考

CoT模型在简单任务上花大量token推理，延迟极高成本翻倍。

解法：简单任务用普通LLM，复杂推理才用o1/R1，设置max_thinking_tokens限制。
