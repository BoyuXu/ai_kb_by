# KIMI k1.5: Scaling Reinforcement Learning with LLMs
> 来源：https://arxiv.org/abs/2501.12599 | 领域：llm-infra | 日期：20260323

## 问题定义
月之暗面发布的KIMI k1.5，探索RL在LLM后训练中的Scaling规律：随着计算预算（RL训练FLOPs）增加，推理能力如何提升。k1.5展示了Long-CoT RL scaling的有效性。

## 核心方法与创新点
- Long-CoT训练：支持最长128K token推理链的RL训练
- RL Scaling：增加RL训练计算量，持续提升推理效果
- Short-CoT蒸馏：将Long-CoT模型蒸馏到快速推理的Short-CoT版本
- 多模态推理：视觉输入的推理能力（图表/几何题）

## 实验结论
KIMI k1.5在AIME达到77.5%（接近o1），MATH 96.2%；RL计算量增加10x对应推理能力线性提升；Short-CoT版本速度是Long-CoT的10x，保留约85%效果；视觉推理超越GPT-4V。

## 工程落地要点
- Long-CoT serving需要支持数千token的流式输出和进度展示
- 思维链长度与问题难度正相关，难题需要更多"思考时间"
- RL训练的Context长度需要128K+，对GPU内存要求极高

## 面试考点
1. **Q: Long-CoT的"Scaling"体现在哪里？** A: 推理时计算量（inference compute）可以按需扩展——难题思考更长
2. **Q: Long-CoT和Short-CoT蒸馏如何平衡速度与质量？** A: 用Long-CoT生成的推理链训练Short-CoT，让小模型"记住"解题步骤
3. **Q: 为什么视觉推理需要特殊训练？** A: 图像中的空间关系和数量关系需要专门的多模态推理能力训练
4. **Q: KIMI k1.5的RL reward如何设计？** A: 数学：答案正确性（RLVR）；代码：测试通过；感知推理：人工标注+规则
5. **Q: Long-CoT的"探索性思考"（self-reflection）如何涌现？** A: RL训练中，模型发现"验证答案"和"尝试不同方法"能提升奖励，自发学会这些策略
