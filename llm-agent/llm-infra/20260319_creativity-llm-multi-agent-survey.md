# Creativity in LLM-based Multi-Agent Systems: A Survey
> 来源：https://arxiv.org/abs/2505.21116 | 日期：20260319

## 问题定义
LLM的创造力（Creativity）在多Agent系统中如何涌现、如何量化、如何工程化是新兴研究方向。单个LLM的创造力受训练数据限制，但多Agent交互（辩论、协作、竞争）可能催生超越单体的创意输出。本文综述LLM创造力的定义、评测方法和多Agent创造力增强机制。

## 核心方法与创新点
1. **创造力维度定义**：
   - 新颖性（Novelty）：输出与训练数据/已知输出的距离
   - 有用性（Usefulness）：输出对特定目标的实用价值
   - 惊喜性（Surprise）：违反预期的程度
2. **多Agent创造力机制**：
   - 辩论（Debate）：Agent间相互挑战，促使观点演化
   - 角色扮演（Role Playing）：不同角色激活不同思维模式
   - 集体头脑风暴：并行生成多样想法后汇总
   - 进化算法：Agent生成→评估→选择→变异的迭代循环
3. **评测框架**：自动评测（embedding距离）+ 人工评测（专家评分）+ 任务特定指标（代码通过率/故事评分）
4. **创造力促进技术**：高温采样、提示多样化、思维链多样性、角色约束解除
5. **创造力与幻觉的边界**：区分创意输出（有价值的新颖内容）和幻觉（错误信息）

## 实验结论
- 多Agent辩论比单Agent在创意写作任务新颖性评分提升约20%
- 角色扮演能解锁LLM在直接提问时受限的创意输出
- 高温采样在创意任务收益显著，但事实性任务有害

## 工程落地要点
- 内容创作、营销文案、产品设计等创意场景适合多Agent架构
- 需要事实准确性的任务（法律/医疗）应限制创造力，使用低温度+RAG
- 创意输出的质量控制：增加Critic Agent进行后验审查
- 多样性-质量权衡：过度追求多样性可能降低实用性

## 面试考点
**Q: 如何量化LLM输出的多样性？**
A: （1）Self-BLEU：一组输出互相计算BLEU，分数越低越多样；（2）语义多样性：embedding聚类数量；（3）词汇多样性：distinct-n指标；（4）输出熵：概率分布的熵值。

**Q: 多Agent辩论（Debate）如何提升推理准确性？**
A: 多个Agent各自提出答案和论据，然后相互质疑，经过几轮辩论后汇总形成更鲁棒的答案。Du et al.(2023)证明辩论在数学和推理任务上优于单次CoT。

**Q: LLM的创造力是否真实存在？**
A: 有争议。支持方：LLM能生成训练集中未见过的组合；反对方：本质是统计模式的重新组合，缺乏理解。工程上关注"是否有用"而非"是否真正创意"。
