### 1. MemCollab: Cross-Agent Memory Collaboration via Contrastive Trajectory Distillation
**arXiv:** 2026-03-24 | **领域：** 多Agent记忆协作

**核心思想：**
- 提出跨Agent记忆共享框架——MemCollab
- 使用**对比轨迹蒸馏（Contrastive Trajectory Distillation）**将一个Agent的经验蒸馏到另一个Agent的记忆
- 解决多Agent系统中各自独立记忆无法互通的问题

**关键机制：**
- 对比学习：区分"有效轨迹"和"无效轨迹"
- 轨迹蒸馏：将经验压缩为可传输的记忆单元
- 跨Agent检索：共享记忆池中的按需检索

**面试价值：**
> Q: 多Agent系统如何实现记忆共享？
> A: MemCollab 提出对比轨迹蒸馏——将成功/失败轨迹对比学习，提炼为结构化记忆单元，通过共享记忆池跨Agent分发，避免重复试错。

**评分：** ⭐⭐⭐⭐⭐（强烈推荐精读）

### 3. BeliefShift: Benchmarking Temporal Belief Consistency and Opinion Drift in LLM Agents
**arXiv:** 2026-03-24 | **领域：** 记忆一致性评测

**核心思想：**
- 研究长期运行对话Agent的**信念漂移问题**：Agent在同一话题上随时间产生前后矛盾的观点
- 提出 BeliefShift 基准，量化 LLM 的时序信念一致性

**问题描述：**
```
用户第1天：你认为Python适合大数据处理吗？→ Agent: "适合，有PySpark"
用户第30天：同一问题 → Agent: "不适合，性能不足"
```

**核心指标：**
- Belief Consistency Score (BCS)：跨时间的答案一致性
- Opinion Drift Index (ODI)：观点漂移速率
- Context Sensitivity：对上下文的敏感程度

**与记忆的关联：** 信念漂移的根因是缺乏持久化的显式知识存储——记忆系统必须锚定关键信念。

**评分：** ⭐⭐⭐⭐

### 4. MSA: Memory Sparse Attention for Efficient End-to-End Memory Model Scaling to 100M Tokens
**arXiv:** 2026-03-05 | **领域：** 长上下文 / 记忆机制

**核心思想：**
- 提出 Memory Sparse Attention（MSA）：实现端到端记忆模型扩展至 **1亿 token**
- 突破 Full Attention 的 O(n²) 计算瓶颈
- 目标：让AI具备"终身规模"信息处理能力

**技术架构：**
```
MSA = Sparse Attention Patterns + Memory Compression
      ↓                            ↓
  局部窗口 + 全局稀疏采样      重要性加权的记忆压缩
```

**与 Flash Attention 的区别：**
- Flash Attention：优化全注意力的IO效率
- MSA：改变注意力结构本身，引入记忆压缩层

**规模对比：**
| 方法 | 有效上下文长度 |
|------|--------------|
| GPT-4 | ~128K |
| Claude | ~200K |
| MSA | ~100M |

**评分：** ⭐⭐⭐⭐⭐（长上下文技术突破，必读）

### 5. GSEM: Graph-based Self-Evolving Memory for Experience Augmented Clinical Reasoning
**arXiv:** 2026-03-23 | **领域：** 记忆组织 / 图结构记忆

**核心思想：**
- 提出 GSEM：基于图的自演化记忆系统
- 将历史决策经验组织为**动态更新的知识图**
- 应用场景：临床决策（但方法通用）

**图结构设计：**
```
节点：经验案例 / 决策结果
边：相似性 / 因果关系 / 时序关联
自演化：新经验触发图结构更新（节点合并/拆分/重链接）
```

**与向量存储的对比：**
- 向量存储：flat，检索靠相似度，难以表达关系
- GSEM：图结构，保留经验间的因果和时序关系

**推荐系统类比：** GSEM 的图演化机制类似推荐系统中的 User Interest Graph 演化——用户兴趣随时间动态更新图结构。

**评分：** ⭐⭐⭐⭐

### 6. Conversation Tree Architecture: A Structured Framework for Context-Aware Multi-Branch LLM Conversations
**arXiv:** 2026-03-22 | **领域：** 上下文管理 / 对话结构

**核心思想：**
- 批判当前对话界面的**平铺上下文（Flat Context）**问题
- 提出对话树（Conversation Tree）：多话题分支独立管理上下文
- 解决跨话题污染导致的响应质量下降

**现有问题：**
```
话题A: 机器学习 ──┐
话题B: 旅行计划 ──┤→ 单一线性上下文 → 话题干扰
话题C: 代码调试 ──┘
```

**对话树方案：**
```
根节点（会话）
├── 分支A: 机器学习 [独立上下文窗口]
├── 分支B: 旅行计划 [独立上下文窗口]  
└── 分支C: 代码调试 [独立上下文窗口]
```

**工程应用：** 长对话系统、多任务Agent的上下文隔离管理。

**评分：** ⭐⭐⭐⭐

### 7. User Preference Modeling for Conversational LLM Agents: Weak Rewards from Retrieval-Augmented Interaction
**arXiv:** 2026-03-21 | **领域：** 个性化记忆 / 用户偏好建模

**核心思想：**
- 用长短期偏好向量（Long-term + Short-term vectors）表示用户
- 偏好向量用于**偏置检索评分**，实现个性化RAG
- 用弱标量奖励（用户反馈）在线更新偏好向量，无需per-user fine-tuning

**技术框架：**
```
用户偏好 = α × 长期向量 + β × 短期向量
          ↓
检索分数 = 语义相似度 + γ × 偏好向量对齐度
          ↓
在线更新：用户点赞/踩 → 弱奖励 → 向量梯度更新
```

**推荐系统视角：**
这本质上是将推荐系统的**双塔用户建模**迁移到RAG Agent领域：
- 推荐：用户行为 → 用户Embedding → 物品检索
- Agent：用户反馈 → 偏好向量 → 知识检索

**面试价值：**
> Q: 如何实现无需重新训练的用户个性化？
> A: 维护长短期偏好向量，用弱奖励在线更新，注入检索评分——类似推荐系统双塔的实时更新策略。

**评分：** ⭐⭐⭐⭐⭐（与推荐系统高度相关，强烈推荐）

### 8. Memory Poisoning and Secure Multi-Agent Systems
**arXiv:** 2026-03-20 | **领域：** 记忆安全

**核心思想：**
- 研究多Agent系统的**记忆投毒攻击**（Memory Poisoning）
- 攻击者通过注入恶意记忆影响Agent决策
- 提出防御机制框架

**攻击类型：**
- 直接投毒：向记忆存储写入虚假经验
- 间接投毒：通过环境交互让Agent自主学习恶意记忆
- 级联投毒：在多Agent系统中扩散恶意记忆

**安全启示：** 记忆系统需要验证机制——类似数据库的事务完整性保证。

**评分：** ⭐⭐⭐

### 9. Memori: A Persistent Memory Layer for Efficient, Context-Aware LLM Agents
**arXiv:** 2026-03-20 | **领域：** 持久化记忆工程

**核心思想：**
- Memori 作为独立的记忆层插入 LLM Agent 架构
- 支持写入、检索、遗忘三类操作
- 强调**效率**：最小化记忆检索的延迟开销

**系统架构：**
```
LLM Agent
    ↕ (read/write API)
Memori Layer
    ├── 短期缓存（Session Memory）
    ├── 中期压缩（Episodic Memory）
    └── 长期持久（Semantic Memory）
```

**与 MemGPT 对比：**
- MemGPT：记忆操作由LLM自主决策（高灵活，高延迟）
- Memori：专用记忆层独立管理（低延迟，高确定性）

**评分：** ⭐⭐⭐⭐

### 10. All-Mem: Agentic Lifelong Memory via Dynamic Topology Evolution
**arXiv:** 2026-03-19 | **领域：** 终身记忆 / 动态拓扑

**核心思想：**
- 面向长期（月-年级别）的 Agent 终身记忆系统
- 记忆组织结构随时间**动态演化拓扑**（合并、分裂、重组）
- 类比人脑记忆巩固（Memory Consolidation）机制

**拓扑演化规则：**
- 频繁共同激活的记忆节点合并
- 长期未访问的记忆节点衰减/遗忘
- 新记忆根据内容相似性接入现有拓扑

**学术价值：** 将神经科学的记忆巩固理论工程化为可计算的图拓扑算法。

**评分：** ⭐⭐⭐⭐

### 11. MemArchitect: A Policy Driven Memory Governance Layer
**arXiv:** 2026-03-18 | **领域：** 记忆治理 / 策略驱动

**核心思想：**
- 记忆治理层（Memory Governance Layer）：对记忆的读写操作施加策略控制
- 支持：访问控制、保留策略、隐私保护、合规过滤

**治理维度：**
```
写入策略：什么信息可以存入记忆？
保留策略：记忆保留多久？如何老化？
读取策略：谁可以访问哪些记忆？
删除策略：用户要求遗忘时如何执行？
```

**企业场景价值：** 合规驱动的AI Agent（医疗、金融）必须有记忆治理层。

**评分：** ⭐⭐⭐

## 📋 常见题积累

**Q1: Agent 的记忆系统有哪几层？各自解决什么问题？**
> A: 三层架构：工作记忆（当前上下文窗口，处理即时任务）→ 情节记忆（Episodic，存储历史交互片段）→ 语义记忆（长期知识，向量或图结构）。各层对应不同时间尺度和存储成本。

**Q2: 如何解决 Agent 长期运行的信念一致性问题？**
> A: BeliefShift 指出信念漂移的根因是缺乏显式持久化知识锚点。解决方案：对关键信念显式存入持久化记忆（如Key-Value），每次响应前检索校验。

**Q3: 多 Agent 系统如何共享记忆？**
> A: MemCollab 提出对比轨迹蒸馏：成功/失败轨迹对比学习 → 蒸馏为结构化记忆单元 → 存入共享记忆池 → 其他Agent按需检索。避免每个Agent重复试错。

**Q4: 推荐系统的用户建模技术如何迁移到 Agent 记忆？**
> A: 类比双塔模型：用长短期偏好向量表示用户，注入RAG检索评分（偏好向量对齐度加权），用户交互弱奖励在线更新向量——实现无需fine-tuning的个性化。
