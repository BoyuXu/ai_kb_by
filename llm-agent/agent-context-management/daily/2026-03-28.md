### 1. Memory for Autonomous LLM Agents: Mechanisms, Evaluation, and Emerging Frontiers
**arXiv:** 2026-03-08 | **领域：** 综述 Survey | **评分：** ⭐⭐⭐⭐⭐

**核心思想：**
- 系统性综述 LLM Agent 记忆机制，从基础分类到前沿挑战
- 三层记忆分类：**工作记忆**（Context Window）、**情节记忆**（Episodic）、**语义记忆**（Semantic/Parametric）
- 评测维度：记忆保真度、检索效率、更新一致性、跨任务泛化

**关键框架：**
```
Agent Memory System
├── 工作记忆 (Working Memory)
│   └── Context Window + KV Cache
├── 情节记忆 (Episodic Memory)
│   └── 对话历史、任务轨迹、事件序列
├── 语义记忆 (Semantic Memory)
│   └── 外部知识库、向量数据库、结构化图谱
└── 程序记忆 (Procedural Memory)
    └── 技能/工具调用模式（微调参数）
```

**前沿挑战：**
1. **遗忘控制**：如何实现有选择性遗忘而非全部保留
2. **记忆冲突**：新旧知识更新时的一致性保障
3. **隐私边界**：多用户/多租户环境下的记忆隔离
4. **可解释性**：Agent 调用记忆时的推理透明度

**面试价值：**
> Q: LLM Agent 记忆系统如何分类？
> A: 四类——工作记忆（当前上下文窗口+KV Cache）、情节记忆（历史交互轨迹）、语义记忆（外部知识库/向量DB）、程序记忆（参数中的技能知识）。类比人类记忆心理学。

### 2. Neural Paging: Learning Context Management Policies for Turing-Complete Agents
**arXiv:** 2026-02-11/03 | **领域：** 上下文管理策略 | **评分：** ⭐⭐⭐⭐⭐

**核心思想：**
- 将 LLM + 外部读写存储的系统证明为**图灵完备**（理论基础）
- 类比 OS 的内存分页机制（Paging）：设计 **Neural Paging**——让 Agent 学习何时将信息"换页"到外部存储
- 核心贡献：将上下文管理变为**可学习策略**而非规则

**技术对比：**
| 方法 | 机制 | 局限 |
|------|------|------|
| 固定窗口截断 | 丢弃最旧 tokens | 信息损失无法控制 |
| RAG 检索 | 被动检索相关片段 | 无主动规划能力 |
| Neural Paging | 强化学习策略，主动决定换页时机 | 需要训练成本 |

**工程启示：** 未来的 Agent 运行时（Runtime）应内置"内存换页"能力，上下文管理不应是 Prompt 工程而是学习到的策略。

**面试价值：**
> Q: Agent 如何应对超长任务导致的上下文溢出？
> A: Neural Paging 提出用 RL 学习上下文换页策略——把不常用信息"换出"到外部存储，需要时再换回，类比 OS 虚拟内存，避免了硬截断的信息损失。

### 3. Orla: A Library for Serving LLM-Based Multi-Agent Systems
**arXiv:** 2026-03-13 | **领域：** 多Agent系统服务框架 | **评分：** ⭐⭐⭐⭐

**核心思想：**
- 面向生产部署的多 Agent LLM 服务库
- 关键设计：**请求路由**、**上下文共享**、**并发会话管理**
- 解决问题：多 Agent 并发推理时的资源竞争与上下文隔离

**架构亮点：**
- Session Manager：每个 Agent 独立上下文，跨 Agent 通信通过消息总线
- KV Cache 复用：相同 System Prompt 的 Agent 共享前缀 KV Cache（节省显存）
- 负载均衡：基于 Agent 任务类型的智能路由

**与推荐系统类比：** 类似推荐系统的请求路由层，Orla 是 Agent 系统的"实时服务框架"。

### 4. AgentRM: An OS-Inspired Resource Manager for LLM Agent Systems
**arXiv:** 2026-03-13 | **领域：** Agent 资源管理 | **评分：** ⭐⭐⭐⭐

**核心思想：**
- 将操作系统资源管理思想引入 LLM Agent：**CPU调度 → Token计算配额**，**内存管理 → 上下文分配**，**I/O调度 → 工具调用队列**
- 提出 AgentRM：统一的 Agent 资源调度层

**OS 类比映射：**
```
OS 概念          AgentRM 对应
-----------      ----------------
进程调度          Agent 任务优先级
内存分页          上下文 / KV Cache 分配
I/O 调度         工具调用 / API 请求队列
内核态/用户态     System Prompt / User Prompt 隔离
```

**关键洞察：** Agent OS 化是趋势——将 Agent 系统按 OS 分层设计，解耦推理层、调度层、存储层。

### 5. SuperLocalMemory V3: Information-Geometric Foundations for Zero-LLM Enterprise Agent Memory
**arXiv:** 2026-03-15 | **领域：** 企业级记忆系统 | **评分：** ⭐⭐⭐

**核心思想：**
- "Zero-LLM"：企业 Agent 记忆层**完全不依赖 LLM**，而是基于信息几何学（Information Geometry）
- 使用流形上的测地距离（Geodesic Distance）替代余弦相似度，实现更精确的记忆检索
- 优势：低延迟（无需 LLM 推理）、可解释（数学透明）、隐私友好（无需外部 API）

**信息几何核心：**
- 记忆表示为统计流形上的点（非欧空间）
- 检索 = 在流形上找测地最近邻
- Fisher 信息矩阵作为局部度量

**实际意义：** 不是所有记忆检索都需要 LLM，几何方法在某些企业场景（安全敏感、低延迟）更合适。

### 6. AMemGym: Interactive Memory Benchmarking for Assistants in Long-Horizon Conversations
**arXiv:** 2026-03-02 | **领域：** 记忆评测 | **评分：** ⭐⭐⭐⭐

**核心思想：**
- 针对**长期对话助手**的记忆能力基准——不是单轮测试，而是跨百轮对话的记忆保持评测
- "Interactive"：评测过程中允许 Agent 主动查询/更新记忆，而非被动回答
- 覆盖场景：个人助手、客服、医疗随访等长期交互

**评测维度：**
1. 事实记忆保留率（Factual Retention）
2. 个性化偏好追踪（Preference Tracking）
3. 时序事件理解（Temporal Reasoning over Memory）
4. 矛盾信息处理（Contradiction Handling）

**benchmark 设计亮点：** 引入"记忆注入"机制——在对话中植入需要被记住的信息，后续多轮后考察回忆。

### 7. SideQuest: Model-Driven KV Cache Management for Long-Horizon Agentic Reasoning
**arXiv:** 2026-02-25/27 | **领域：** KV Cache / 长程推理 | **评分：** ⭐⭐⭐⭐

**核心思想：**
- 针对**深度研究类 Agent**（multi-hop reasoning over multiple docs）的 KV Cache 管理
- 传统 KV Cache 管理是启发式的（按位置/注意力分数），SideQuest 是**模型驱动**的——让模型自己决定哪些 KV 值得保留
- 引入"侧任务"（Side Quest）：并行运行轻量评估器判断 KV 价值

**性能：** 在长程 Agentic Reasoning 任务上，在相同显存预算下比标准 KV 管理提升 15-25% 准确率。

**面试价值：**
> Q: 如何优化长程 Agent 任务中的 KV Cache？
> A: SideQuest 提出模型驱动的 KV 管理——用轻量"侧任务"评估器为每个 KV 打价值分，低分 KV 优先被驱逐，比启发式方法更准确反映任务相关性。

### 8. Structured Prompt Language (SPL): Declarative Context Management for LLMs
**arXiv:** 2026-02-23 | **领域：** 上下文管理语言 | **评分：** ⭐⭐⭐

**核心思想：**
- 提出 **SPL（Structured Prompt Language）**：SQL 启发的声明式语言，将 LLM 上下文窗口视为"受约束资源"
- 语法例子：`WITH BUDGET 4096 TOKENS SELECT context FROM memory WHERE relevance > 0.7`
- 显式控制 token 预算、优先级、信息来源

**意义：** 将上下文管理从隐式 Prompt 工程变为显式声明式编程，提高可控性和可维护性。

### 9. The Pensieve Paradigm: Stateful Language Models Mastering Their Own Context
**arXiv:** 2026-02-12 | **领域：** 有状态 LM 设计 | **评分：** ⭐⭐⭐⭐⭐

**核心思想：**
- 类比哈利·波特中邓布利多的"Pensieve"（思忆盆）：AI 已拥有外部存储，但**模型缺少主动控制外部记忆的能力**（"没有魔杖的邓布利多"）
- 提出 Pensieve Paradigm：训练模型**主动发出读写记忆指令**，而非被动接受检索结果

**两种 Agent 记忆模式对比：**
```
被动模式（当前主流）：
  User Prompt → RAG检索 → 注入Context → LLM回答
  问题：检索由外部系统控制，LLM 无法决定"该记什么"

主动模式（Pensieve）：
  LLM → 发出 WRITE/READ/FORGET 指令 → 操作外部记忆 → 更新Context
  优势：LLM 自主管理记忆，类似人类主动遗忘无用信息
```

**面试价值：**
> Q: RAG 和 Agent Memory 的核心区别是什么？
> A: RAG 是被动检索，外部系统决定给模型什么信息。Agent Memory（如 Pensieve Paradigm）让模型主动控制读写——自己决定存什么、忘什么、何时检索，更接近真正的认知主体。

### 10. HyMem: Hybrid Memory Architecture with Dynamic Retrieval Scheduling
**arXiv:** 2026-02-14 | **领域：** 混合记忆架构 | **评分：** ⭐⭐⭐⭐

**核心思想：**
- 混合三层记忆：**Hot Memory**（近期上下文，直接在Context Window）、**Warm Memory**（中期记忆，向量检索）、**Cold Memory**（长期归档，关键词索引）
- **动态调度**：根据访问频率和重要性自动在三层间迁移
- 类比计算机存储层次：L1/L2 Cache → RAM → 硬盘

**调度策略：**
- Hot → Warm：超过 N 轮未被访问
- Warm → Cold：超过 K 天未被访问
- Cold → Warm：检索到相关性 > 阈值时自动提升

**与推荐系统类比：** 类似推荐系统的 Feature Store 分层（在线/近线/离线），HyMem 是 Agent 记忆的分层存储方案。

### 11. Architecting AgentOS: From Token-Level Context to Emergent System-Level Intelligence
**arXiv:** 2026-02-24 | **领域：** Agent 操作系统理论 | **评分：** ⭐⭐⭐⭐

**核心思想：**
- 提出 AgentOS 架构理论：从 token 级别微观处理到宏观系统级涌现智能的完整理论桥梁
- 四层架构：**Token Layer**（基础推理）→ **Context Layer**（短期规划）→ **Memory Layer**（持久知识）→ **System Layer**（涌现能力）

**理论贡献：**
- 证明上下文窗口大小和记忆质量决定"有效推理深度"的上界
- 涌现能力（Emergent Capability）不仅依赖模型参数，还依赖**记忆系统架构质量**

## 💡 今日核心洞察

### 洞察1：Agent Memory 正在经历 OS 化转型
从今日多篇论文（AgentRM, AgentOS, Neural Paging）可以看出：LLM Agent 系统正在朝**操作系统化**演进——引入进程调度、内存分层、换页机制等经典 OS 概念。这意味着 Agent 工程师需要同时掌握 LLM 知识和系统设计知识。

### 洞察2：被动 RAG → 主动记忆控制
Pensieve Paradigm 代表了一个重要趋势：记忆不再是"给模型提供上下文"，而是"模型主动管理自己的记忆"。这是从工具增强（Tool-Augmented）到认知主体（Cognitive Agent）的范式转变。

### 洞察3：记忆分层与推荐系统 Feature Store 高度同构
HyMem 的三层架构（Hot/Warm/Cold）与推荐系统的特征存储层次几乎完全类比——这意味着搜广推工程师转型 Agent 工程师有天然优势，过往的分层存储设计经验可以直接复用。

### 洞察4：综述论文是面试备考的高效入口
"Memory for Autonomous LLM Agents: Mechanisms, Evaluation, and Emerging Frontiers" 是本周最值得精读的综述——系统覆盖四类记忆（工作/情节/语义/程序）的分类框架，面试中用这个框架回答"Agent 记忆系统如何设计"类问题非常有条理。
