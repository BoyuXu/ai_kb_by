## 🔥 今日重点论文

### 1. MemCollab: Cross-Agent Memory Collaboration via Contrastive Trajectory Distillation
- **arXiv:** 近期提交（2026-03-24）
- **作者：** Yurui Chang, Yiran Wu, Qingyun Wu, Lu Lin
- **核心贡献：**
  - 多 Agent 之间的记忆共享与协作框架
  - 通过对比轨迹蒸馏（Contrastive Trajectory Distillation）实现跨 Agent 记忆迁移
  - 解决多 Agent 场景下知识孤岛问题
- **关键洞察：** 单个 Agent 的记忆可以通过对比学习蒸馏到其他 Agent，实现"集体记忆"
- **适用场景：** 多 Agent 协作系统、推荐系统中多塔模型的知识共享

### 2. MSA: Memory Sparse Attention for Efficient End-to-End Memory Model Scaling to 100M Tokens
- **arXiv:** 2026-03-05（较早提交）
- **作者：** Yu Chen et al., Alibaba/腾讯团队
- **核心贡献：**
  - 提出 Memory Sparse Attention（MSA），将有效上下文扩展到 **1亿 token**
  - 突破全注意力架构的内存瓶颈
  - End-to-End 记忆模型，无需外部向量数据库
- **关键技术：**
  - 稀疏注意力机制（Sparse Attention）
  - 分层记忆压缩
  - 长期 vs 短期记忆的动态分配
- **面试考点：** 长上下文处理 vs RAG 的权衡；稀疏注意力如何降低 O(n²) 复杂度

### 4. Memori: A Persistent Memory Layer for Efficient, Context-Aware LLM Agents
- **arXiv:** 2026-03-20
- **作者：** Luiz C. Borro et al.
- **核心贡献：**
  - 为 LLM Agent 设计持久化记忆层（Persistent Memory Layer）
  - 上下文感知（Context-Aware）：根据当前对话动态检索相关记忆
  - 提升 Agent 的跨会话一致性
- **架构特点：**
  - 记忆写入：结构化抽取 + 向量化存储
  - 记忆读取：混合检索（语义 + 关键词）
  - 记忆更新：冲突检测与合并策略
- **工程价值：** 直接对应 OpenClaw memory-lancedb-pro 的设计思路

### 5. Conversation Tree Architecture: A Structured Framework for Context-Aware Multi-Branch LLM Conversations
- **arXiv:** 2026-03-22
- **核心问题：** 现有对话接口的平铺追加（flat append-only）结构导致上下文无限膨胀
- **解决方案：**
  - 树形对话结构：每个话题分支独立管理
  - 避免无关历史污染当前上下文
  - 支持多话题并行、回溯与合并
- **洞察：** 对话 = 树，而非线性序列；这是解决"上下文遗忘"的根本架构思路
- **实践意义：** 长会话 Agent 应使用树形记忆而非线性 buffer

### 6. The Library Theorem: How External Organization Governs Agentic Reasoning Capacity
- **arXiv:** 2026-03-22
- **作者：** Zachary F. Mainen
- **核心论点：**
  - Transformer Agent 已经利用 CoT 进行外化推理
  - 但**结构化检索**（对自身推理状态的索引）仍未充分探索
  - 提出"图书馆定理"：外部组织（索引结构）决定 Agent 的推理容量上限
- **关键结论：** Agent 的智能上限 = 记忆组织方式的上限
- **面试价值：** 解释为什么 RAG 比 long-context 更具扩展性

### 7. MemArchitect: A Policy Driven Memory Governance Layer
- **arXiv:** 2026-03-18
- **核心贡献：**
  - 持久化 LLM Agent 的记忆治理（Memory Governance）
  - 基于策略（Policy-Driven）的记忆权限控制
  - 解决"记忆泄露"和"敏感信息持久化"问题
- **治理维度：**
  - 记忆生命周期管理（TTL）
  - 访问控制（哪些 Agent 可读写哪些记忆）
  - 审计日志（Memory Audit Trail）
- **工程重要性：** 生产环境 Agent 必须考虑记忆安全

### 8. TA-Mem: Tool-Augmented Autonomous Memory Retrieval for LLM in Long-Term Conversational QA
- **arXiv:** 2026-03-10
- **核心贡献：**
  - 工具增强的自主记忆检索（Tool-Augmented Memory Retrieval）
  - Agent 自主决定何时调用记忆工具

### 9. BeliefShift: Benchmarking Temporal Belief Consistency and Opinion Drift in LLM Agents
- **arXiv:** 2026-03-24
- **核心问题：** LLM 作为长期对话 Agent 时，其"信念"（Belief）随时间漂移
- **Benchmark 设计：**
  - 测量意见一致性（Temporal Belief Consistency）
  - 检测观点漂移（Opinion Drift）
- **关联记忆：** 记忆系统应该"锚定"Agent 的核心信念，防止漂移
- **面试角度：** 如何在个性化推荐中维持用户兴趣的长期稳定性

### 10. KV Cache Optimization Strategies for Scalable and Efficient LLM Inference
- **arXiv:** 2026-03-20
- **作者：** Yichun Xu et al.
- **核心内容：**
  - KV Cache 是 Transformer 推理的核心优化
  - 内存占用随序列长度线性增长
  - 综述多种 KV Cache 压缩策略
- **主要策略：**
  - **Token 淘汰（Token Eviction）：** 丢弃低重要性 token 的 KV
  - **量化（Quantization）：** INT8/INT4 KV 压缩
  - **分页（Paging）：** vLLM 的 PagedAttention
  - **前缀缓存（Prefix Caching）：** 系统 prompt 复用
- **面试必考：** 画出 KV Cache 的内存结构；解释 PagedAttention 原理

## 💡 面试准备笔记

**Q: LLM Agent 的记忆有哪些类型？**
- **工作记忆（Working Memory）：** 当前对话上下文 / Context Window
- **短期记忆（Short-Term）：** 近期会话历史，可被 KV Cache 加速
- **长期记忆（Long-Term）：** 持久化存储，需要外部数据库（如 LanceDB）
- **语义记忆（Semantic）：** 关于世界的知识，通常通过 RAG 获取
- **情节记忆（Episodic）：** 特定事件的记忆，带时间戳

**Q: 如何处理长上下文中的"迷失中间"问题（Lost in the Middle）？**
- 重要信息放开头或结尾
- 使用 Memory Sparse Attention 压缩中间内容
- 结构化摘要：将中间历史压缩为语义向量

**Q: RAG vs Long-Context，如何选择？**
- RAG：知识库大、更新频繁、延迟要求低 → 适合推荐/搜索场景
- Long-Context：任务需要细粒度上下文、文档较短 → 适合代码理解、文档问答
