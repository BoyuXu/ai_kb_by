## 🏆 本周重点论文（Top 3）

### 1. IDProxy - 小红书多模态 LLM 解决广告冷启动
**arXiv 2026-03-02 | 重要程度: ⭐⭐⭐⭐⭐**

- **一句话**：用多模态 LLM 为新广告生成 proxy ID embedding，冷启动 CTR +5~8%，已在小红书线上部署
- **为什么重要**：冷启动是广告系统"永恒痛点"，这是首个大规模工业验证 LLM proxy embedding 方案的工作
- **Boyu 应重点关注**：proxy embedding 与 CTR 模型的对齐训练方法，可直接借鉴到现有系统
- 详细笔记：`papers/20260312_IDProxy_冷启动CTR_小红书多模态LLM.md`

### 2. Generative Recommendation for Large-Scale Advertising
**arXiv 2026-03-04 | 重要程度: ⭐⭐⭐⭐⭐**

- **一句话**：生成式召回（GR）首次在亿级广告库的工业系统落地，RPM +2.3%，P99 延迟 <30ms
- **为什么重要**：GR 从学术走向工业广告系统的里程碑，标志着召回架构正在经历范式转移
- **Boyu 应重点关注**：RQ 量化 pipeline + eCPM 约束 beam search 的工程实现
- 详细笔记：`papers/20260312_GenerativeRec_大规模广告生成式推荐.md`

### 3. HiSAC - 超长用户行为序列建模
**arXiv 2026-02-24 | 重要程度: ⭐⭐⭐⭐⭐**

- **一句话**：层次化稀疏激活压缩，将序列建模能力扩展到 10K+ 行为，AUC 持续提升
- **为什么重要**：用户行为序列长度是推荐/广告效果的核心 lever，HiSAC 提供了工程可行的扩展方案
- **Boyu 应重点关注**：两级压缩策略（兴趣聚合 + 行为精选）的工程实现和缓存设计

## 📈 技术趋势总结（200字以内）

2026 年 Q1 搜广推领域呈现三大趋势：

**① 生成式推荐工业化加速**：GR 不再只是学术 demo，已有头部广告平台在亿级广告库部署，RPM 增益明确。核心门槛是 RQ 量化工程和在线延迟控制。

**② LLM 深入广告工程链路**：不是简单的 LLM as Ranker，而是 LLM 生成 proxy embedding 解决冷启动、LLM 统一离线分析任务（AdNanny）、Decoder-Only 架构用于 CTR 上下文建模（CADET）。LLM 从"实验性工具"变为"工程组件"。

**③ 超长序列成新战场**：从 SIM 的 1K 到 HiSAC 的 10K+，用户行为序列长度的竞争持续，直接对应模型效果和广告收入。谁先工程化超长序列，谁就有竞争优势。
