## 今日 Top 3 技术点（资深工程师视角）

### 🔥 Top 1: Semantic ID 统一了生成式推荐的表示空间

今天 rec-sys 的主线是 Spotify 的生成式推荐三部曲：
- **Semantic ID** 将物品内容语义压缩为层次化 token 序列（RQ-VAE），使语言模型能直接"生成"物品
- **Variable-Length Semantic ID** 进一步让不同复杂度的物品自适应 ID 长度，序列总长压缩 30%
- **工程关键**：Constrained Beam Search（Trie 约束解码）是生成式检索线上可用的必要条件，没有约束 = 线上空结果

实际落地时，Semantic ID 的版本管理（embedding 更新导致 ID 变化）和冷启动（新物品即可语义检索）是核心价值，值得立即调研应用。

---

### 🔥 Top 2: GRPO 用组内相对排名消灭了 Critic 网络

DeepSeek R1 成功的核心技术之一，GRPO 的精妙在于：
- 对同一问题采样 G 个回答，**组内归一化**代替 Critic 的状态价值估计
- 本质上是把"绝对价值估计"转化为"相对排名"，消除了大量训练不稳定因素
- 可验证奖励（数学/代码正确性）= GRPO 的最佳适用场景，因为奖励客观、精确、无法被"破解"

面试高频考点：GRPO vs PPO 对比、RL 为什么能提升推理而 SFT 不能，必须烂熟于心。

---

### 🔥 Top 3: 广告系统的多目标优化是长期博弈，不是短期最优

今天 ads 的核心洞察：
- **短期收入最大化 ≠ 长期平台价值**：用户体验下降 → 留存率下降 → 未来收入损失（LTV 损失可能超过短期增益）
- **约束优化比加权求和更易调参**：将体验指标设为约束（≥ 阈值），只优化收入，更符合业务决策逻辑
- **IPS 纠偏是广告排序的必备基础设施**：位置偏差不纠偏会导致模型越训越偏，新广告永无出头之日

工业界经常犯的错：只看 7 天实验的 eCPM 提升，忽视用户体验指标的微小下降积累成长期的平台衰退。
