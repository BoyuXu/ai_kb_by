## 今日 Top 3 技术点（资深工程师视角）

### 1. Semantic ID 生态的成熟与工业落地（Spotify 双论文）
Spotify 同日发布 Semantic ID 播客发现和统一 LM 两篇工业论文，标志着生成式推荐的工业落地进入新阶段。**关键洞察**：RQ-VAE 的层次化离散 ID 将内容语义压缩为可生成的 token 序列，配合 Variable-Length 动态长度（冷启动用长 ID，热门 item 用短 ID），形成完整的工业化 Semantic ID 工程体系。这是 2025 年推荐系统最重要的范式转移之一。

### 2. GRPO 开创无 Critic 的高效 RLHF 新路线（DeepSeek Math 核心技术）
GRPO 用组内相对奖励替代 Critic 模型，显存减少 40%、吞吐提升 1.5×，同时在数学推理上超越 PPO。**工程价值**：中小团队终于可以在有限 GPU 资源上做高质量 RLHF 训练，不需要维护与 policy 等规模的 value network。配合规则奖励函数（数学/代码有明确正确性判断），GRPO 成为垂直领域对齐的首选方案。

### 3. Dense vs Sparse 不是 OR 而是 AND（混合检索成为工业标配）
今日综合多篇检索论文（SPLADE-v3、Dense vs Sparse 评估综述）的结论：精确查询 Sparse 胜，语义查询 Dense 胜，长尾 query 混合最优。**落地建议**：工业搜索系统应将 BM25/SPLADE 和 Dense ANN 作为并行两路召回，通过 RRF 融合，而非选边站。SPLADE-v3 的稀疏语义向量可直接复用倒排索引基础设施，是 BM25 的语义升级平替，迁移成本极低。
