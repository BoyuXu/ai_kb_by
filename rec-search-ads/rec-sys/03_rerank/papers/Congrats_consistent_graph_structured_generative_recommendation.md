# Congrats: Consistent Graph-structured Generative Recommendation

> 来源：https://arxiv.org/abs/2510.10127 | 领域：rec-sys | 学习日期：20260329

## 问题定义

生成式重排（Generative Reranking）将重排问题建模为序列生成任务，能隐式捕捉 item 间复杂依赖关系，但现有方法存在**似然陷阱（Likelihood Trap）**问题：

1. **高似然 ≠ 高质量**：似然最高的序列往往包含大量重复 item，被用户认为质量低劣
2. **多样性受损**：贪心搜索/beam search 倾向于选高概率但同质的 item，限制用户参与度
3. **列表整体效用未优化**：现有方法按 token 预测，难以显式建模列表级别的多样性和 item 依赖

**核心洞察**：重排需要**图结构**建模 item 间的依赖关系，并通过**一致性训练**直接对齐用户偏好评估。

## 核心方法与创新点

### CONGRATS：两大核心创新

**1. Graph-structured Model（图结构生成模型）**
- 将 item 间的关系建模为图（而非传统线性序列）
- 通过**多路径图遍历**扩展解码空间，自然引入多样性
- **图转移（Graph Transitions）**显式建模 item 依赖：
  - 节点：候选 item
  - 边：item 间的语义/协同关联
  - 生成序列时探索图的多条路径，避免陷入单一高概率路径
- 同时提升**预测准确性**（显式依赖建模）和**输出多样性**（多路径探索）

**2. Consistent Differentiable Training（一致性可微训练）**
- 引入**评估器（Evaluator）**作为额外监督信号
- 训练目标从 MLE（最大似然）转变为直接学习用户偏好
- 一致性设计确保生成的序列在用户偏好评估下保持一致
- 区别于标准的 RLHF：通过可微方式训练，不依赖 PPO 等 RL 算法
- 打破似然陷阱：用评估器分数而非序列概率引导生成

## 实验结论

**公开 Benchmark**（多个重排标准数据集）：
- 在准确性和多样性指标上均优于 SOTA 重排方法
- 显著缓解 Likelihood Trap 问题

**工业部署（快手 Kuaishou）**：
- 日活用户超 **3亿**
- 推荐质量（Quality）和多样性（Diversity）**双提升**
- 验证了图结构生成重排在超大规模场景的有效性

## 工程落地要点

1. **图的构建**：item 间边的定义需根据业务场景设计（语义相似度、协同点击、时序共现等），建议离线预计算
2. **图遍历解码**：可用 beam search on graph 实现多路径探索，控制 beam width 平衡延迟和多样性
3. **评估器训练**：需要有质量的列表级别标注数据（用户滑动行为/完播率等），或用代理信号
4. **Likelihood Trap 监控**：线上需监控候选重复率、类目多样性等指标，检测是否陷入高似然低质量
5. **快手经验**：超大规模系统中，图结构引入的额外复杂度（构图、图遍历）需要专门的工程优化

## 常见考点

**Q1：什么是 Likelihood Trap？为什么生成式重排容易陷入这个问题？**
> A：Likelihood Trap 指模型倾向于生成概率最高但质量低（重复、同质）的序列。原因：自回归模型中，热门 item 的 token 概率高，beam search 贪心地选高概率路径，导致列表充斥同类 item。本质是 MLE 训练目标（最大化 ground truth 序列似然）与用户偏好（多样有价值的列表）不对齐。

**Q2：Graph-structured Model 如何解决 Likelihood Trap？**
> A：图结构通过显式建模 item 间依赖（边连接关系），生成时必须遍历图的不同分支（路径），自然避免重复选择同一聚类的 item。图的多路径探索增大了解码空间，有效跳出高似然低多样性陷阱。

**Q3：Consistent Differentiable Training 与标准 RLHF 的区别？**
> A：RLHF 使用 PPO 等 RL 算法优化奖励模型输出，梯度传播链路长，训练不稳定。Congrats 的一致性可微训练直接将评估器（用户偏好）作为可微目标，通过 end-to-end 梯度反传，更稳定高效，避免了 RL 的 reward hacking 问题。

**Q4：生成式重排与传统排序（LTR）相比，核心优势是什么？**
> A：传统 LTR 对每个 item 独立打分后排序，无法建模 item 间依赖（如互补/替代关系）。生成式重排将 item list 整体作为输出序列，通过自回归建模隐式捕捉 item 间依赖，生成列表时能考虑已选 item 的影响，实现更优的列表整体效用。

**Q5：快手（Kuaishou）3亿 DAU 场景中，如何保证图结构重排的推理延迟？**
> A：（1）离线预计算 item 图，避免在线实时构图；（2）图剪枝：只保留 top-K 高置信边；（3）候选集控制：重排阶段候选通常几百个，图规模可控；（4）解码 beam width 调优：权衡延迟与质量；（5）模型蒸馏或量化压缩大模型。
