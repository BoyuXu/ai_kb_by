# CONGRATS: Consistent Graph-structured Generative Recommendation

> 来源：arxiv 2510.10127 | 领域：rec-sys | 学习日期：20260328 | 平台：快手 Kuaishou

## 问题定义

重排（Reranking）是推荐系统的最后一道关卡，直接决定用户看到的内容顺序。生成式重排（Generative Reranking）将重排建模为序列生成任务，能隐式建模item间复杂依赖，但面临核心问题：

**似然陷阱（Likelihood Trap）**：
- 生成式模型倾向于输出高似然序列，而高似然≠高质量
- 高似然序列往往**重复、同质化**（模型记住了训练集中的高频组合）
- 用户实际上偏好**多样而新颖**的推荐序列

现有生成式重排方法（PRM等）缺乏机制跳出似然陷阱，导致用户参与度受限。

## 核心方法与创新点

CONGRATS（**Con**sistent **Gra**ph-structured **G**enerative Recommenda**t**ion **S**ystem）提出两大创新：

### 1. Graph-structured Model（图结构模型）
将传统的**链式（Chain）**序列生成改为**图（DAG）**结构生成：

```
传统链式：item1 → item2 → item3 → item4（一条路径）

图结构：
       item_A
      /       \
item1           item3 → item5
      \       /
       item_B
```

优势：
- **扩展解码空间**：从指数级的排列空间中探索多条路径，不局限于单一高似然路径
- **显式建模item依赖**：图边权重表示item间的相关/互补/替代关系
- **促进多样性**：通过图上的多路径探索，自然产生多样化推荐序列

数学形式：

$$
P(seq | user) = \prod_{(i,j) \in path} P(j | i, \mathbf{h}_{user}) \cdot w_{ij}
$$

其中 $w_{ij}$ 为图中item i → item j的转移权重，由item特征和用户状态联合决定。

### 2. Consistent Differentiable Training（一致性可微训练）
引入**评估器（Evaluator）**，让模型直接从用户偏好中学习：

$$
\mathcal{L} = \mathcal{L}_{gen} + \lambda \cdot \mathcal{L}_{eval}
$$

- **生成器（Generator）**：生成候选重排序列
- **评估器（Evaluator）**：对序列质量打分（对齐用户真实反馈）
- **一致性训练**：评估器的梯度反传给生成器，让生成器学会生成评估器打高分的序列（而非最高似然序列）

这本质上是一种**Actor-Critic**框架在生成式重排中的应用。

## 实验结论

**离线实验**：
- 在多个公开重排数据集上（MIND, Zhihu-News等），CONGRATS在NDCG、ERR等指标上超过SOTA重排方法

**快手线上实验**（3亿+ DAU平台）：
- 推荐质量（用户互动指标）显著提升
- 推荐多样性（覆盖度）同时提升
- 实现了**质量与多样性的同步改善**（通常两者trade-off，此处同时提升）

## 工程落地要点

1. **图构建策略**：item间的图边不需要手工设计，可用协同过滤共现矩阵、内容相似度或用户session数据自动构建，T+1更新
2. **解码策略**：图上的束搜索（Graph Beam Search）：同时维护K条活跃路径，每步按路径得分剪枝，保留多样性的同时控制计算量
3. **评估器设计**：评估器输入为序列级特征（顺序、多样性分、新颖度等），避免退化为item-wise scoring
4. **Actor-Critic稳定性**：与RL训练类似，需要clip gradient、reward normalization，防止训练不稳定
5. **延迟控制**：图结构生成比链式生成更复杂，需要提前离线缓存item图，线上只做图上的路径搜索

## 常见考点

**Q1：什么是似然陷阱（Likelihood Trap）？为什么生成式重排容易陷入？**
A：生成式模型用MLE（最大似然估计）训练，倾向于给训练集中高频模式分配高概率。推荐场景中，热门item的共现是高频模式，模型会反复推荐相似的热门组合（虽然对数概率高，但用户体验差）。根本原因是MLE目标与用户感知质量不对齐。

**Q2：CONGRATS的图结构相比链式序列生成的本质区别？**
A：链式生成是有向路径（单一因果链），每个item只依赖前一个item；图结构是DAG，每个item可以有多个前驱，解码时可探索多条路径。这使得模型不会"锁定"到单条高似然路径，自然实现多样化推荐。

**Q3：Consistent Differentiable Training中评估器如何避免reward hacking？**
A：评估器的输入包含用户真实反馈（点击率、停留时长等），而非纯模型打分，降低了评估器被生成器"欺骗"的风险。同时可以用多个评估器（集成），防止单一评估器被over-optimize。训练时定期用真实用户反馈重新校准评估器。

**Q4：推荐多样性和相关性为什么通常trade-off？CONGRATS如何同时提升？**
A：高相关性 → 模型推荐最确定的item → 往往是热门相似item → 低多样性。CONGRATS通过图结构探索多路径，每条路径内部保持相关性，路径间通过图的不同分支实现多样性。本质是在更大的解码空间中找到既相关又多样的序列，而不是在单条路径内权衡。

**Q5：快手这类短视频推荐的重排有哪些特殊挑战？**
A：(1) 序列消费特性：用户按顺序消费，前几个视频的体验影响后续消费意愿（上下文效应）；(2) 多目标：点击、观看时长、互动、关注等多个指标；(3) 实时性：用户兴趣在单次session内快速变化，重排需要感知上下文；(4) 规模：3亿DAU，重排延迟要求极低（<10ms）。

```
         │  │                                                   │ │

         ┌─────────────────────────────────────────────────────────┐
         │              Consistency Evaluator                       │
         │  ┌─────────────────────────────────────────────────┐   │
         │  │  输入：序列级特征(顺序、多样性分、新颖度)        │   │

         │  │  训练信号：用户真实反馈(点击率、停留时长)         │   │
         │  └─────────────────────────┬───────────────────────┘   │
         │                            │ ∇Q → 反传给Generator      │
         └────────────────────────────┼───────────────────────────┘
                                      │
                                      ▼
                            最终重排序列输出
```

**训练流程（Actor-Critic 风格）**：

$$
\theta_{gen}^{t+1} = \theta_{gen}^{t} - \eta \nabla_{\theta_{gen}} \left[ \mathcal{L}_{gen} + \lambda \cdot \mathcal{L}_{eval} \right]
$$

$$
\theta_{eval}^{t+1} = \theta_{eval}^{t} - \eta \nabla_{\theta_{eval}} \left[ \text{MSE}(Q_{\theta_{eval}}(S), R_{user}) \right]
$$

其中 $R_{user}$ 为用户实际反馈信号的归一化值。

## 模型架构详解

### 输入处理
- **候选集编码**：精排 Top-K 结果（通常 K=50~200），每个候选带精排分数和特征
- **列表级特征**：候选间的相似度矩阵、类目分布、已有展示历史
- **上下文**：用户实时状态、页面位置、时间特征

### 核心模块
- **候选交互**：Self-Attention / Graph Attention 捕获候选间关系
- **序列解码**：Pointer Network / 自回归方式逐步选择最优排列
- **约束满足**：多样性/品类/广告密度等约束的可微松弛

### 优化目标
- **Listwise Loss**：NDCG/MAP 的可微近似（ApproxNDCG / LambdaLoss）
- **多目标加权**：CTR × 时长 × 多样性 × 新颖度的帕累托权衡
- **强化学习**：将重排建模为序列决策问题，用 RL 优化长期指标

## 面试深度追问

- **Q: 重排为什么不能直接用精排分数排序？**
  A: 精排是 Pointwise 打分，忽略候选间的交互效应。例如连续展示同类商品会降低用户兴趣（边际效用递减），重排需要考虑列表级效果。

- **Q: 多样性和相关性的平衡策略？**
  A: 1) MMR：λ × 相关性 - (1-λ) × 冗余度；2) DPP：行列式点过程建模子集多样性；3) 约束优化：多样性作为约束条件，相关性作为目标函数。

- **Q: 重排阶段的延迟预算如何分配？**
  A: 典型链路延迟预算：召回 10ms + 粗排 5ms + 精排 20ms + 重排 10ms = 总计 < 50ms。重排需在 10-20ms 内完成，通常限制候选集 ≤ 200。

- **Q: 生成式重排和传统重排的区别？**
  A: 生成式重排将排列视为序列生成问题（类似 Seq2Seq），可以直接优化全局排列指标。传统方法（Greedy/Beam Search）是启发式搜索，可能陷入局部最优。
