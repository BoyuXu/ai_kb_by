# 双塔召回模型（Two-Tower / Dual Encoder）

> 来源：Google DSSM 系列 | 年份：2016-2020 | 领域：rec-sys/01_recall（向量检索/召回）

## 问题定义

召回阶段需要从亿级物料库中快速找到用户可能感兴趣的 Top-K 候选：
- **效率**：每次请求 <10ms，不能做用户-物品交叉特征在线计算
- **质量**：召回结果要覆盖精排最终推荐的物品
- **可扩展性**：物品库动态增长（每天新增数万物品），系统需要适应

传统协同过滤无法融入 side information（图文特征、上下文）。双塔模型通过**离线建立物品索引 + 在线用户向量检索**解决这一问题。

## 核心方法与完整公式

### 公式1：Sampled Softmax Loss

$$\mathcal{L} = -\log \frac{e^{u \cdot v^+ / \tau}}{e^{u \cdot v^+ / \tau} + \sum_{j=1}^{K} e^{u \cdot v_j^- / \tau}}$$

**解释：**
- $u$：用户向量
- $v^+$：正样本物品向量
- $v_j^-$：负样本物品向量（in-batch 或采样）
- $\tau$：温度参数（通常 0.05-0.1），控制分布锐度
- 温度越小 → 分布越集中 → 模型对正负样本区分更严格

### 公式2：Sampling Bias Correction

$$s_{corrected}(u, v) = u \cdot v - \log p(v)$$

**解释：**
- $p(v)$：物品 $v$ 被采样为负样本的概率（热门物品 $p(v)$ 大）
- 减去 $\log p(v)$ 修正热门偏差：热门物品的得分被降低
- 不做修正 → 模型倾向推荐长尾物品（因为热门物品在负样本中过度出现被抑制）

### 公式3：Hard Negative Mining

$$\text{NegSet}(u) = \text{Random}(K_1) \cup \text{Hard}(K_2)$$

$$\text{Hard}(K_2) = \text{TopK}_{v \notin \text{Pos}(u)}(u \cdot v)$$

**解释：**
- 混合随机负样本 + 难负样本（被召回但未点击的物品）
- Hard negatives 帮助模型学习精细区分"相关"和"相似但不相关"
- 比例通常 $K_1 : K_2 = 4:1$ 或 $3:1$

### 公式4：多兴趣扩展（MIND）

$$[u_1, u_2, \ldots, u_K] = \text{CapsuleRouting}(e_1, e_2, \ldots, e_N)$$

$$\text{score}(u, v) = \max_k (u_k \cdot v)$$

**解释：**
- 用胶囊网络将用户行为序列路由为 K 个兴趣向量
- 每个兴趣向量独立做 ANN 检索
- 取 K 路检索结果的并集作为最终召回

## 与基线方法对比

| 方法 | 交叉特征 | 物品索引 | 适用规模 | 核心优势 |
|------|---------|---------|---------|---------|
| **矩阵分解（ALS）** | 无 | 可离线 | 中等 | 简单 |
| **双塔（Two-Tower）** | 无 | 可离线 | 亿级 | 融入 side info |
| **精排模型（DeepFM）** | 有 | 不可索引 | 千级 | 精确但慢 |
| **MIND** | 无 | 多向量索引 | 亿级 | 多兴趣 |
| **Graph 召回** | 有 | 可离线 | 亿级 | 关系建模 |

## 实验结论

- vs 矩阵分解：有 side information 场景 Recall@100 提升 20-30%
- Hard negative mining vs 纯随机：Recall@100 +5-10%
- 温度参数 $\tau$ 对训练稳定性影响显著：过小（<0.01）训练不稳定，过大（>0.5）区分度不够
- Sampling bias correction：长尾物品召回率 +15%

## 工程落地要点

1. **特征实时性**：用户行为序列需 near-real-time 更新（Flink 流式特征），冷热分离存储
2. **向量更新频率**：Item 向量每小时/每天全量更新，需与模型版本对齐
3. **负样本偏差修正**：In-batch negative 热门物品偏差需用 sampling bias correction 修正
4. **多路召回**：部署 3-5 路（双塔 + 协同过滤 + 内容 + 图），取并集送精排
5. **ANN 索引选择**：Faiss IVF+PQ 适合超大规模（10亿+），HNSW 适合中等规模但精度更高

## 面试考点

**Q1：双塔模型为什么不能做用户-物品交叉特征？**
> 因为需要离线预计算所有物品向量并建立索引。引入交叉特征则物品向量依赖用户，无法离线预算，在线需对所有候选做联合推理，计算量爆炸（从 O(K) 变成 O(N)，N 为全量物品数）。

**Q2：In-batch negative 的问题？**
> ① 热门 item 作为负样本概率更高 → 模型被"训练"去抑制热门物品 → 推荐偏向长尾。② 同批次内可能存在假负样本（用户实际感兴趣但无正样本标签）。解决：sampling bias correction + hard negative 混合。

**Q3：双塔和 DSSM 的关系？**
> DSSM（2013 Microsoft）是双塔先驱，用于搜索 Query-Document 匹配。后被推荐系统采用并扩展：加入行为序列建模（YoutubeDNN）、多兴趣（MIND）、对比学习（SimCSE 风格）等。

**Q4：温度参数 τ 的作用和调优？**
> τ 控制 softmax 分布锐度。τ 小 → 分布集中 → 模型对相似度差异更敏感 → 但训练不稳定。τ 大 → 分布平坦 → 区分度低。通常 0.05-0.1，可用 learnable temperature。

**Q5：双塔召回如何评估效果？**
> Recall@K（Top-K 中正样本比例）、Hit Rate@K（至少命中一个正样本的请求比例）、离线召回率（精排推荐结果在召回中的覆盖率）。Recall@100 是最常用指标。

**Q6：如何处理新物品（冷启动）？**
> Item Tower 可以用内容特征（标题/图片 embedding）生成向量，不依赖行为数据。新物品只需有内容特征即可入索引。但协同过滤信号缺失会影响初始排序。
