**一句话**：生成式推荐把「找商品」变成「写文章」——但写文章有两种方式：逐字写（自回归）和先画草稿再慢慢雕琢（扩散），各有各的优势。

**类比**：你要写一封推荐信。自回归方式：从「亲爱的」开始，逐字逐句写到结尾，写完一个字才能写下一个字。扩散方式：先随手涂鸦一封信（加噪），然后一遍遍修改（去噪），最终得到完整的信。扩散允许你改变任何位置，而自回归只能往后写。

## 📐 核心公式与原理

### 1. 矩阵分解

$$
\hat{r}_{ui} = p_u^T q_i
$$

- 用户和物品的隐向量内积

### 2. BPR 损失

$$
L_{BPR} = -\sum_{(u,i,j)} \ln \sigma(\hat{r}_{ui} - \hat{r}_{uj})
$$

- 正样本得分 > 负样本得分

### 3. 序列推荐

$$
P(i_{t+1} | i_1, ..., i_t) = \text{softmax}(h_t^T E)
$$

- 基于历史序列预测下一次交互

## 两种范式对比

| 维度 | 自回归（TIGER, SASRec） | 扩散（DiffGRM） |
|------|------------------------|----------------|
| 生成方式 | 逐 token 串行生成 | 并行迭代去噪 |
| 上下文 | 单向（只看已生成的） | 双向（全局感知） |
| 曝光偏差 | 有（训练用真实 token，推理用预测 token） | 无（去噪过程不依赖自身历史） |
| 多样性 | 确定性强，多样性依赖 Beam Search | 随机性内置，多样性自然更高 |
| 推理延迟 | O(L) 串行解码步骤 | O(T) 去噪步骤（T 通常 10-50） |
| 训练收敛 | 快（2× 更快） | 慢 |
| 目前精度 | 更高（工业主流） | 学术追赶中（+8.3% HR@10） |

## DiffGRM 的核心思路

```
传统 DDPM（图像生成）：
  图片 → 加高斯噪声（T步） → 纯噪声
  纯噪声 → 去噪（T步，每步用 UNet）→ 图片

DiffGRM（生成式推荐）：
  Semantic ID 嵌入 → 加高斯噪声 → 噪声向量
  噪声向量 → 条件去噪（条件：用户历史编码） → Semantic ID 嵌入
  嵌入 → ANN 检索最近邻 → 候选商品
```

**关键细节**：
- 在连续嵌入空间扩散（不是离散 token），更稳定
- 条件：用户历史序列的 Transformer 编码向量
- 双向 Attention：扩散 Transformer 可以看到所有噪声位置
- 课程扩散：训练时先学「大噪声去噪」（粗粒度），再学「小噪声去噪」（细粒度）

---

## 为什么扩散没有曝光偏差？

自回归的曝光偏差（Exposure Bias）：
```
训练时：input = [真实历史, 真实token_1, ...], predict token_2
推理时：input = [真实历史, 预测token_1, ...], predict token_2
                              ↑ 预测的，可能有误差
```
误差会随序列长度累积。

扩散模型没有这个问题：
- 训练时：直接预测「从噪声到原始嵌入的映射」
- 推理时：同样的映射，只是从随机噪声开始
- 没有「先预测的结果作为下一步输入」的链式依赖

---

## 什么场景用扩散，什么场景用自回归？

| 场景 | 推荐方案 | 理由 |
|------|---------|------|
| 推荐多样性要求高（音乐、视频） | 扩散 | 内置随机性 |
| 推荐精度优先（广告、高转化） | 自回归 | 精度更高，可控 |
| 超长历史序列（>1000 items） | 自回归 + GEMs（今日） | 扩散的双向 Attention 计算量更大 |
| 冷启动+内容丰富 | 扩散 | 连续嵌入空间更好利用内容特征 |

## 技术演进脉络

```
2020 SASRec：Transformer 自回归序列推荐
    ↓ 离散 item_id → 词汇量爆炸
2022 TIGER/P5：Semantic ID + 自回归，生成式推荐
    ↓ 曝光偏差 + 单向上下文限制
2024 DiffGRM：扩散模型替代自回归，消除曝光偏差
    ↓ 两条路线并行发展
2025 可变长度 Semantic ID（与自回归兼容）
     Long-sequence（GEMs）提升自回归效率
```

## 面试考点

**Q：生成式推荐和传统推荐的核心区别是什么？**  
答：传统推荐（双塔+ANN）是判别式的：给定 user 和 item，预测相关性分数。生成式推荐是生成式的：给定 user，直接生成目标 item 的标识（Semantic ID），通过解码而非检索得到候选。优势：能处理超大候选集（不需要向量化所有 item），更自然地融合内容语义；劣势：训练更复杂，延迟更高（自回归解码）。

**Q：DiffGRM 的连续空间扩散 vs 离散扩散有什么区别？**  
答：离散扩散（D3PM）在 token 空间加噪（随机替换 token），但 Semantic ID 的码字空间不像文字那样有丰富的替换语义，收敛不稳定。连续扩散在 embedding 向量上加高斯噪声，扩散过程平滑且数学性质好（DDPM 理论保证），最后通过 ANN 检索回离散 ID，两全其美。

### Q1: 推荐系统的实时性如何保证？
**30秒答案**：①用户特征实时更新（Flink 流处理）；②模型增量更新（FTRL/天级重训）；③索引实时更新（新物品上架）；④特征缓存+预计算降低延迟。

### Q2: 推荐系统的 position bias 怎么处理？
**30秒答案**：训练时：①加 position feature 推理时固定；②IPW 加权；③PAL 分解 P(click)=P(examine)×P(relevant)。推理时：设置固定 position 或用 PAL 只取 P(relevant)。

### Q3: 工业推荐系统和学术研究的差距？
**30秒答案**：①规模（亿级 vs 百万级）；②指标（商业指标 vs AUC/NDCG）；③延迟（<100ms vs 不关心）；④迭代（A/B 测试 vs 离线评估）；⑤工程（特征系统/模型服务 vs 单机实验）。

### Q4: 推荐系统面试中设计题怎么答？
**30秒答案**：按层回答：①明确场景和指标→②召回策略（多路）→③排序模型（DIN/多目标）→④重排（多样性）→⑤在线实验（A/B）→⑥工程架构（特征/模型/日志）。

### Q5: 2024-2025 推荐技术趋势？
**30秒答案**：①生成式推荐（Semantic ID+自回归）；②LLM 增强（特征/数据增广/蒸馏）；③Scaling Law（Wukong）；④端到端（OneRec 统一召排）；⑤多模态（视频/图文理解）。

### Q6: 推荐系统的 EE（Explore-Exploit）怎么做？
**30秒答案**：①ε-greedy：ε 概率随机推荐；②Thompson Sampling：从后验分布采样；③UCB：置信上界探索；④Boltzmann Exploration：按 softmax 温度控制探索度。工业实践：对新用户多探索，老用户少探索。

### Q7: 推荐系统的负反馈如何利用？
**30秒答案**：①隐式负反馈：曝光未点击（弱负样本）、快速划过（中等负样本）；②显式负反馈：「不喜欢」按钮（强负样本）。处理：加大显式负反馈的权重，用 skip 行为做弱负样本。

### Q8: 多场景推荐（Multi-Scenario）怎么做？
**30秒答案**：同一用户在首页/搜索/详情页/直播间有不同推荐需求。方法：①STAR：场景自适应 Tower；②共享底座+场景特定头；③Scenario-aware Attention。核心：共享知识避免数据孤岛，同时保留场景差异。

### Q9: 推荐系统的内容理解怎么做？
**30秒答案**：①文本理解（NLP/LLM 提取标题、标签语义）；②图片理解（CNN/ViT 提取视觉特征）；③视频理解（时序模型提取关键帧+音频）；④多模态融合（CLIP-style 对齐文本和视觉）。

### Q10: 推荐系统的公平性问题？
**30秒答案**：①供给侧公平（小创作者也有曝光机会）；②需求侧公平（不同用户群体获得同等服务质量）；③内容公平（避免信息茧房）。方法：公平约束重排、多样性保障、定期公平性审计。


## 📐 核心公式直观理解

### 判别式 vs 生成式推荐

$$
\text{判别式}: P(y|x; \theta), \quad \text{生成式}: P(x, y; \theta) = P(y|x; \theta) P(x; \theta)
$$

**直观理解**：判别式模型直接学"给定用户特征预测是否点击"（条件概率），生成式模型学"用户特征和行为的联合分布"（先理解用户再预测行为）。生成式更"理解"用户，但训练更难——就像"学会写作文"比"学会判断作文好坏"更难。

### 自回归 vs 非自回归生成

$$
\text{AR}: P(\mathbf{y}) = \prod_t P(y_t | y_{<t}), \quad \text{NAR}: P(\mathbf{y}) = \prod_t P(y_t | \mathbf{x})
$$

**直观理解**：自回归逐步生成（每步依赖前一步），质量高但慢（串行）。非自回归一次全部生成（并行），快但可能不一致（位置间缺乏协调）。推荐场景中自回归更适合生成排序列表（前面选了什么影响后面选什么），非自回归适合召回（独立打分）。

### 生成多样性 vs 相关性的 trade-off

$$
\text{score}(i) = \lambda \cdot \text{relevance}(i) + (1-\lambda) \cdot \text{novelty}(i)
$$

**直观理解**：只推最相关的→用户看到的都差不多（filter bubble）；太追求新奇→推一些用户根本不感兴趣的。$\lambda$ 控制这个平衡。生成式方法天然有一定随机性（采样温度），可以更优雅地实现多样性。
