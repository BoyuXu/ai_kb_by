# MaRI: Accelerating Ranking Model Inference via Structural Re-parameterization in Large Scale Recommendation System

> 来源：[https://arxiv.org/abs/2405.xxxxx] | 日期：20260313 | 领域：rec-sys

## 问题定义
大规模工业推荐系统中，精排模型越来越复杂（多塔、多任务、深层交叉网络），导致线上推理延迟飙升，在严格的SLA（如<10ms）约束下难以部署深层模型。传统量化和剪枝方法往往损伤精度。本文借鉴计算机视觉中RepVGG的结构重参数化思路，提出MaRI：训练时使用复杂结构（多分支并联），推理时将多分支合并为等价的单路结构，实现"训练复杂、推理简单"。

## 核心方法与创新点
- **结构重参数化（Structural Re-parameterization）**：训练时，每个全连接层替换为多条并联路径（如原始Linear + Shortcut + BN分支），利用多分支的隐式正则化效果提升训练精度；推理时，将多个分支的权重矩阵通过矩阵加法合并为单一等价权重矩阵，推理计算量与原始单层相同。
- **推荐系统适配**：将重参数化从CV的卷积核扩展到Embedding后的MLP层，并针对稀疏特征（Embedding lookup）的特殊结构设计了兼容的融合策略。
- **多任务扩展**：在多任务学习中，共享底层使用重参数化结构，任务塔独立保留，有效提升共享表征质量而不增加推理开销。
- **训练-推理无缝切换**：通过简单的`convert()`接口，训练完成后一键转换为推理模型，无需重新训练或Fine-tuning。

## 实验结论
- 在某大型电商精排模型上，MaRI在不损失AUC（<0.001）的前提下，推理延迟降低25%~35%。
- 对比知识蒸馏（KD）方法：MaRI在相同延迟约束下AUC高0.3%~0.5%，且无需训练独立的Teacher模型。
- 在多任务场景下，CTCVR联合指标提升0.2%，说明多分支结构对多任务学习有额外正则化增益。

## 工程落地要点
- **权重合并验证**：重参数化合并后需在验证集上逐层验证输出一致性（允许浮点误差<1e-5），防止合并过程存在数值精度问题。
- **与量化结合**：重参数化后的单路结构更适合INT8量化（分支归并后BN参数融合进权重，量化更稳定），推荐先重参数化再量化。
- **Embedding层不参与重参数化**：稀疏Embedding的结构重参数化意义有限，MaRI主要作用于Dense层（Post-Embedding MLP部分）。
- **线上A/B测试**：由于训练和推理等价性在理论上成立，A/B测试主要验证工程实现的正确性，通常离线AUC与线上CTR变化一致。

## 常见考点
**Q1: 结构重参数化的核心原理是什么？**
A: 核心是利用线性层的叠加性：多个线性变换（含BN）在推理时可以等价地合并为一个线性变换。例如 `y = W1*x + W2*x + b` 等价于 `y = (W1+W2)*x + b`，合并后计算量与单层相同，但训练时多分支提供了更好的梯度流和隐式正则化。

**Q2: 推荐系统和CV中使用重参数化的主要区别？**
A: CV（RepVGG）主要在卷积层上操作，推荐系统主要在Embedding后的全连接层上操作。推荐系统还面临稀疏特征和样本分布不均匀的挑战，需要特别处理Embedding层的BN行为（在线更新统计量时需注意batch size对稀疏特征的影响）。

**Q3: 为什么重参数化优于蒸馏？**
A: 蒸馏需要先训练一个大Teacher模型再训练Student模型，流程复杂且Teacher需要额外资源；重参数化只需一次训练，训练复杂度与Teacher相当但直接产出推理轻量的模型，工程实施更简洁。
