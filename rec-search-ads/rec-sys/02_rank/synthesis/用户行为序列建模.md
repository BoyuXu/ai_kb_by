## 📐 核心公式与原理

### 📐 1. DIN（Deep Interest Network）Target Attention 推导

DIN 的核心创新：用候选物品（target item）$\mathbf{e}_a$ 作为 Query，attend 用户历史行为序列 $\{\mathbf{e}_1, \ldots, \mathbf{e}_T\}$：

$$
\mathbf{v}_u = \sum_{t=1}^{T} \alpha(\mathbf{e}_t, \mathbf{e}_a) \cdot \mathbf{e}_t
$$

注意力权重（非归一化，不做 softmax）：

$$
\alpha(\mathbf{e}_t, \mathbf{e}_a) = \sigma\!\left(\text{MLP}\!\left([\mathbf{e}_t;\ \mathbf{e}_a;\ \mathbf{e}_t \odot \mathbf{e}_a;\ \mathbf{e}_t - \mathbf{e}_a]\right)\right)
$$

**推导步骤：**

1. **为何用 Target Attention 而非 Self-Attention**：NLP 的 self-attention 让序列内部相互 attend；DIN 的 target attention 让每个历史行为 $\mathbf{e}_t$ 与候选物品 $\mathbf{e}_a$ 计算相关度，提取"这次曝光最可能激活的历史兴趣"——更符合推荐的点击预测任务。

2. **输入四元组 $[\mathbf{e}_t; \mathbf{e}_a; \mathbf{e}_t \odot \mathbf{e}_a; \mathbf{e}_t - \mathbf{e}_a]$**：
   - $\mathbf{e}_t$：历史物品 $t$ 的 Embedding
   - $\mathbf{e}_a$：候选物品的 Embedding
   - $\mathbf{e}_t \odot \mathbf{e}_a$：逐元素乘积（特征交叉，捕捉维度级相关）
   - $\mathbf{e}_t - \mathbf{e}_a$：差分（捕捉距离信息）
   四元组拼接比单一内积能表达更丰富的"行为-候选"关系。

3. **不做 softmax 的原因**：softmax 会强制所有权重和为 1——当用户历史中没有相关行为时，softmax 仍会分配高权重给最相近的（噪声）行为。DIN 用 sigmoid 输出 $[0,1]$ 的绝对权重，无关行为可以获得接近 0 的权重，避免噪声污染。

4. **Dice 激活函数**（DIN 的另一个创新）：

$$
\text{Dice}(s) = p(s) \cdot s + (1 - p(s)) \cdot \alpha s, \quad p(s) = \sigma\!\left(\frac{s - \mathbb{E}[s]}{\sqrt{\text{Var}[s] + \epsilon}}\right)
$$

   数据自适应的激活函数，用 BN 统计量动态决定正负区间的分割点（而非 ReLU 的固定 0 点）。

**符号说明：**

| 符号 | 含义 |
|------|------|
| $\mathbf{e}_t \in \mathbb{R}^d$ | 用户历史行为序列中第 $t$ 个物品的 Embedding |
| $\mathbf{e}_a \in \mathbb{R}^d$ | 候选（Target）物品的 Embedding |
| $\alpha(\mathbf{e}_t, \mathbf{e}_a)$ | 历史行为 $t$ 与候选物品的相关性权重（sigmoid 输出）|
| $\mathbf{v}_u \in \mathbb{R}^d$ | 用户的兴趣向量（加权历史 Embedding 之和）|
| $\odot$ | 逐元素乘法（Hadamard 积）|
| $T$ | 用户历史序列长度（DIN 截断到 ~50）|

**直观理解：** DIN 就像"按需激活"——给你推荐运动鞋时，你买过的运动装备历史权重高；给你推荐美食时，你的餐厅浏览历史权重高。同一个用户，根据候选物品的不同，动态激活不同的历史片段。

### 📐 2. BPR 损失推导

$$
\mathcal{L}_{\text{BPR}} = -\sum_{(u,i,j) \in \mathcal{D}_S} \ln \sigma(\hat{r}_{ui} - \hat{r}_{uj})
$$

**推导步骤：**

1. **假设**：用户对正样本 $i$（交互过的）的偏好应高于负样本 $j$（未交互的），即 $r_{ui} > r_{uj}$
2. **概率建模**：$P(i \succ_u j) = \sigma(\hat{r}_{ui} - \hat{r}_{uj})$，最大化所有正负样本对的成对偏序概率
3. **等价于**：最大化 $\hat{r}_{ui} - \hat{r}_{uj}$ 的 sigmoid 值，迫使正负样本得分之差尽可能大

**符号说明：**
- $\mathcal{D}_S = \{(u, i, j) : i \in \mathcal{I}_u^+, j \notin \mathcal{I}_u^+\}$：训练三元组（用户、正样本、负样本）
- $\hat{r}_{ui}$：模型预测的用户 $u$ 对物品 $i$ 的评分/相关性

---

### 3. 序列推荐（SASRec/BERT4Rec）

$$
P(i_{t+1} \mid i_1, \ldots, i_t) = \text{softmax}(\mathbf{h}_t^\top \mathbf{E})
$$

其中 $\mathbf{h}_t = \text{Transformer}(\mathbf{e}_{i_1}, \ldots, \mathbf{e}_{i_t})$，$\mathbf{E}$ 是所有物品的 Embedding 矩阵。Self-Attention 的时间复杂度 $O(T^2 d)$，这也是长序列建模（$T > 1000$）需要特殊优化（SIM、ETA、Mamba）的根本原因。

## 🎯 核心洞察（4条）

1. **用户行为序列是推荐最有价值的信号**：比用户画像（年龄/性别）信息密度高 10x+，行为序列包含了用户的真实兴趣和意图变化
2. **Target Attention 是序列建模的核心技术**：不同于 NLP 的 self-attention，推荐中用候选物品（target item）作为 query 来 attend 用户历史行为，提取"与当前候选最相关的历史信号"
3. **兴趣多样性需要多向量表示**：用户不是"一个人"而是"多个兴趣的集合"——喜欢数码也喜欢美食，MIND/ComiRec 用多个向量分别表示不同兴趣
4. **行为序列长度从 50→5000+ 是不可逆趋势**：更长的序列 = 更全面的用户理解，但计算效率是核心挑战

## 📈 技术演进脉络

```
平均池化用户行为（~2016）
  → DIN target attention（2018，~50 行为）
    → DIEN GRU+兴趣演化（2019，~100 行为）
      → BST Transformer 序列建模（2019，~150 行为）
        → MIND 多兴趣向量（2019，召回场景）
          → SIM 两阶段超长序列（2020，5000+ 行为）
            → HSTU 沙漏 Transformer（2023，10000+ 行为）
              → LLM 序列理解（2025+）
```

## 🎓 常见考点（6条）

### Q1: DIN vs DIEN vs BST 的核心区别？
**30秒答案**：DIN——target attention 加权历史行为（无序列建模）；DIEN——GRU 建模兴趣演化序列（捕捉兴趣漂移）；BST——Transformer self-attention 捕捉行为间依赖关系（最强但最耗算力）。
**追问方向**：三者哪个工业最常用？答：DIN 变体最多（简单有效），BST 在算力允许时效果最好。

### Q2: 多兴趣模型（MIND/ComiRec）的设计思路？
**30秒答案**：用户兴趣不是单一向量——MIND 用 capsule network 将行为聚类为 K 个兴趣向量；ComiRec 用可控多兴趣提取，K 个向量分别检索后合并。召回阶段特别有效。
**追问方向**：K 怎么确定？答：通常 4-8，太少兴趣覆盖不全，太多冗余且检索成本线性增长。

### Q3: SIM 怎么解决超长序列的计算问题？
**30秒答案**：两阶段——①General Search Unit (GSU)：用简单规则（类目匹配+时间衰减）从 5000+ 行为中选 top-200 相关行为；②Exact Search Unit (ESU)：对选出的 200 条行为做标准 target attention。
**追问方向**：GSU 的筛选规则会丢失重要行为吗？答：可能。ETA 用 SimHash 近似最近邻代替规则筛选，减少信息丢失。

### Q4: 行为序列中的时间信息怎么建模？
**30秒答案**：①时间间隔编码：将行为间的时间间隔离散化为 Embedding（<1h, 1-6h, 6-24h, 1-7d, >7d）；②时间衰减：近期行为权重 = exp(-λ×time_gap)；③位置编码：用行为发生的绝对时间/相对顺序做位置编码。
**追问方向**：推荐中的位置编码和 NLP 有什么不同？答：推荐中时间间隔不均匀（不像文本 token 等距），所以时间间隔编码比固定位置编码更适合。

### Q5: 实时行为 vs 历史行为的融合？
**30秒答案**：实时行为（最近 10 分钟）通过 Flink 流式计算更新特征，历史行为从离线存储读取。融合方式：①拼接（实时特征 concat 历史 embedding）；②gate 机制（动态决定实时 vs 历史的权重）。

### Q6: LLM 在用户行为建模中的角色？
**30秒答案**：①行为序列理解：将用户行为转化为自然语言描述（"用户最近看了3部科幻电影"），LLM 提取深层意图；②行为生成：预测用户下一步可能的行为序列。
**追问方向**：LLM 处理行为序列的延迟问题？答：离线生成用户兴趣标签（语义特征），在线作为额外特征输入到传统模型。

---

### Q7: 推荐系统的实时性如何保证？
**30秒答案**：①用户特征实时更新（Flink 流处理）；②模型增量更新（FTRL/天级重训）；③索引实时更新（新物品上架）；④特征缓存+预计算降低延迟。

### Q8: 推荐系统的 position bias 怎么处理？
**30秒答案**：训练时：①加 position feature 推理时固定；②IPW 加权；③PAL 分解 P(click)=P(examine)×P(relevant)。推理时：设置固定 position 或用 PAL 只取 P(relevant)。

### Q9: 工业推荐系统和学术研究的差距？
**30秒答案**：①规模（亿级 vs 百万级）；②指标（商业指标 vs AUC/NDCG）；③延迟（<100ms vs 不关心）；④迭代（A/B 测试 vs 离线评估）；⑤工程（特征系统/模型服务 vs 单机实验）。

### Q10: 推荐系统面试中设计题怎么答？
**30秒答案**：按层回答：①明确场景和指标→②召回策略（多路）→③排序模型（DIN/多目标）→④重排（多样性）→⑤在线实验（A/B）→⑥工程架构（特征/模型/日志）。

## 记忆助手 💡

### 类比法

- **DIN = 聚光灯**：在一堆历史行为中，候选商品就是"聚光灯"，只照亮和自己相关的历史行为，其他行为暗掉
- **DIEN = 追剧**：用 GRU 建模你的"追剧进度"（兴趣演化），从喜欢动作片 → 科幻片 → 悬疑片的变化轨迹
- **BST = 全班同学讨论**：Transformer self-attention 让所有历史行为互相"讨论"，发现行为之间的关联（买了手机壳的人可能最近买了手机）
- **SIM = 先查目录再精读**：5000 本书不可能都翻，先按目录（类目匹配）找到 100 本相关的，再仔细读
- **MIND 多兴趣 = 多重人格**：用户不是一个兴趣，而是"数码控 + 美食家 + 运动达人"多个人格，每个人格用一个向量表示

### 对比表

| 模型 | 核心机制 | 序列长度 | 计算复杂度 | 适用场景 |
|------|---------|---------|-----------|---------|
| Avg Pooling | 平均池化 | 不限 | O(N) | 基线 |
| DIN | Target Attention | ~50 | O(Nd) | 有候选item的精排 |
| DIEN | GRU+AUGRU | ~100 | O(NH) | 需要兴趣演化的精排 |
| BST | Transformer | ~150 | O(N²d) | 算力充足的精排 |
| SIM | 两阶段检索+Attention | 5000+ | O(N)+O(md) | 超长序列 |
| HSTU | 沙漏Transformer | 10000+ | O(N) | 超大规模系统 |

### 口诀/助记

- **序列建模五代记**："均池→注意→演化→变换→检索" — Avg Pooling → DIN → DIEN → BST → SIM
- **DIN 三不做**："不做 softmax（兴趣非互斥），不做 self-attention（看候选就够），不做序列（无需捕捉顺序）"
- **长序列三字经**："先检索，再打分，控规模"

### 面试一句话

- **DIN vs DIEN**："DIN 用 target attention 激活相关行为但忽略顺序；DIEN 用 GRU 建模兴趣从 A 到 B 的演化过程，AUGRU 让演化方向由候选 item 引导"
- **BST vs DIN**："BST 用 Transformer self-attention 建模行为之间的依赖关系（买了手机→买壳），DIN 只看每个行为和候选的关系，不建模行为间关系"
- **SIM 核心**："两阶段范式，GSU 从万级行为中快速检索百级相关子序列，ESU 在可控规模上做精细 attention，兼顾长期兴趣和计算效率"
