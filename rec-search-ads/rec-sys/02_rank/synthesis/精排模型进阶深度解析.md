## 📐 核心公式与原理

### 1. 多目标融合

$$
score = w_1 \cdot pCTR + w_2 \cdot pCVR + w_3 \cdot duration\_pred
$$

- 加权融合多个预估目标

### 2. Attention 机制

$$
\alpha_i = \frac{\exp(f(e_i, e_{target}))}{\sum_j \exp(f(e_j, e_{target}))}
$$

- DIN-style 注意力加权

### 3. 多任务学习

$$
L = \sum_k \lambda_k L_k(\theta_{shared}, \theta_k)
$$

- 共享参数 + 任务独有参数

#### 1.1.2 SNR (Sub-Network Routing)

SNR的核心创新：**显式控制器决定子网络连接**，而非简单的加权融合。

```python
class SNR(nn.Module):
    """
    Sub-Network Routing for Multi-Task Learning
    参考: "SNR: Sub-Network Routing for Flexible Parameter Sharing in Multi-Task Learning"
    """
    def __init__(self, input_dim, subnetwork_num, experts_per_subnet, task_num):
        super().__init__()
        self.task_num = task_num
        self.subnetwork_num = subnetwork_num
        
        # 构建子网络（每个子网络内部是MMoE结构）
        self.subnetworks = nn.ModuleList([
            MMoELayer(input_dim, experts_per_subnet, input_dim)
            for _ in range(subnetwork_num)
        ])
        
        # Controller: 决定任务到子网络的路由
        # 输出维度: task_num × subnetwork_num 的连接权重
        self.controller = nn.Sequential(
            nn.Linear(input_dim, 128),
            nn.ReLU(),
            nn.Linear(128, task_num * subnetwork_num)
        )
        
        # 任务Tower
        self.towers = nn.ModuleList([
            nn.Sequential(
                nn.Linear(input_dim, 64),
                nn.ReLU(),
                nn.Linear(64, 1)
            ) for _ in range(task_num)
        ])
    
    def forward(self, x):
        # 1. Controller生成路由权重 [batch, task_num * subnetwork_num]
        routing_logits = self.controller(x)
        routing_weights = routing_logits.view(-1, self.task_num, self.subnetwork_num)
        routing_weights = F.softmax(routing_weights, dim=-1)  # [batch, task_num, subnetwork_num]
        
        # 2. 计算所有子网络的输出
        subnet_outputs = torch.stack([
            subnet(x) for subnet in self.subnetworks
        ], dim=1)  # [batch, subnetwork_num, input_dim]
        
        # 3. 根据路由权重融合
        task_outputs = []
        for task_id in range(self.task_num):
            # 该任务的路由权重 [batch, subnetwork_num]
            task_routing = routing_weights[:, task_id, :]  # [batch, subnetwork_num]
            
            # 加权融合子网络输出
            # [batch, subnetwork_num, 1] * [batch, subnetwork_num, input_dim]
            fused = torch.sum(
                task_routing.unsqueeze(-1) * subnet_outputs,
                dim=1  # 沿subnetwork_num维度求和
            )  # [batch, input_dim]
            
            # 输入Tower
            pred = self.towerstask_id
            task_outputs.append(pred)
        
        return task_outputs
```

**SNR vs MMOE关键区别**：
- MMOE：单层Expert加权，权重仅基于输入
- SNR：子网络级别路由，可以学习更复杂的连接模式
- SNR Controller可以学习"任务A主要使用子网络1和2，任务B主要使用子网络2和3"这样的结构化信息

#### 1.1.3 AITM (Adaptive Information Transfer Multi-task)

AITM的核心创新：**建模任务间显式依赖关系，自适应信息传递**。

```python
class AITM(nn.Module):
    """
    Adaptive Information Transfer Multi-task Framework
    适用于有显式依赖关系的任务，如: 曝光→点击→转化
    参考: "Modeling the Sequential Dependence among Audience Multi-step Conversions"
    """
    def __init__(self, input_dim, expert_num_per_task, task_num):
        super().__init__()
        self.task_num = task_num
        
        # 每个任务有一组Expert
        self.task_experts = nn.ModuleList([
            nn.ModuleList([
                Expert(input_dim, input_dim) 
                for _ in range(expert_num_per_task)
            ]) for _ in range(task_num)
        ])
        
        # AIT模块: 自适应信息传递
        # 任务i可以从任务0到i-1接收信息
        self.ait_modules = nn.ModuleList([
            AITModule(input_dim, i)  # i表示前序任务数
            for i in range(task_num)
        ])
        
        # 任务Tower
        self.towers = nn.ModuleList([
            TaskTower(input_dim) for _ in range(task_num)
        ])
    
    def forward(self, x):
        task_outputs = []
        prev_task_representations = []  # 存储前序任务的表示
        
        for task_id in range(self.task_num):
            # 1. 计算当前任务的Expert输出
            expert_outputs = torch.stack([
                expert(x) for expert in self.task_experts[task_id]
            ], dim=1)  # [batch, expert_num, input_dim]
            
            # 2. 平均聚合（或使用Gating）
            current_repr = expert_outputs.mean(dim=1)  # [batch, input_dim]
            
            # 3. AIT: 从前面所有任务自适应接收信息
            if task_id == 0:
                # 第一个任务，没有前序任务
                final_repr = current_repr
            else:
                # 后续任务，通过AIT融合前序任务信息
                final_repr = self.ait_modulestask_id
            
            # 4. 存储当前任务表示供后续任务使用
            prev_task_representations.append(final_repr)
            
            # 5. 预测
            pred = self.towerstask_id
            task_outputs.append(pred)
        
        return task_outputs

class AITModule(nn.Module):
    """
    Adaptive Information Transfer Module
    使用Attention机制从前面任务选择性地接收信息
    """
    def __init__(self, input_dim, prev_task_num):
        super().__init__()
        self.prev_task_num = prev_task_num
        
        # Query: 当前任务表示
        self.query_proj = nn.Linear(input_dim, input_dim)
        
        # Key/Value: 前序任务表示
        self.key_proj = nn.Linear(input_dim, input_dim)
        self.value_proj = nn.Linear(input_dim, input_dim)
        
        # 输出融合
        self.output_proj = nn.Linear(input_dim * 2, input_dim)
    
    def forward(self, current_repr, prev_representations):
        """
        current_repr: [batch, input_dim] - 当前任务表示
        prev_representations: list of [batch, input_dim] - 前序任务表示列表
        """
        # 堆叠前序任务表示 [batch, prev_task_num, input_dim]
        prev_stack = torch.stack(prev_representations, dim=1)
        
        # Query [batch, 1, input_dim]
        Q = self.query_proj(current_repr).unsqueeze(1)
        
        # Key/Value [batch, prev_task_num, input_dim]
        K = self.key_proj(prev_stack)
        V = self.value_proj(prev_stack)
        
        # Attention [batch, 1, prev_task_num]
        attn_scores = torch.matmul(Q, K.transpose(-2, -1)) / (current_repr.size(-1) ** 0.5)
        attn_weights = F.softmax(attn_scores, dim=-1)
        
        # 加权聚合 [batch, 1, input_dim]
        transferred = torch.matmul(attn_weights, V)
        transferred = transferred.squeeze(1)  # [batch, input_dim]
        
        # 融合当前表示和传递信息
        combined = torch.cat([current_repr, transferred], dim=-1)
        output = self.output_proj(combined)
        
        return output
```

#### 1.1.4 四者对比总结

| 维度 | MMOE | PLE | SNR | AITM |
|------|------|-----|-----|------|
| **核心思想** | 软共享Expert | 显式私有/共享分离 | 子网络路由 | 任务间信息传递 |
| **Expert可见性** | 全部共享 | 私有+共享 | 子网络内共享 | 任务独立+信息传递 |
| **任务关系** | 隐式学习 | 部分显式 | 通过Controller | 显式依赖链 |
| **适用场景** | 任务相关性中等 | 任务差异大 | 需要灵活路由 | 序列依赖任务 |
| **代表任务** | CTR+CVR | 点击+收藏+评论 | 复杂多任务场景 | 曝光→点击→转化 |
| **参数量** | O(K·N·d) | O(K·N·d + S·d) | O(N·E·d + C) | O(K·N·d + AIT) |
| **训练难度** | 中 | 较难 | 较难 | 中等 |

**选型建议**：
- 任务间相关性高（如CTR+CVR）：MMOE或AITM
- 任务间差异大（如点击+评论+分享）：PLE
- 任务关系复杂且动态：SNR
- 有明显序列依赖（曝光→点击→转化）：AITM

### 1.2 任务冲突时的梯度处理

#### 1.2.1 问题定义

多任务学习中，不同任务的梯度方向可能冲突：

```
           Task A 梯度方向
                ▲
                │  ╲
                │    ╲
                │      ╲
                │   θ*   ╲
                │─────────► Task B 梯度方向
                │
                │

当两个任务梯度方向相反或夹角>90°时，一个任务的更新会损害另一个任务
```

数学定义：
- 任务i的梯度: $g_i$
- 冲突条件: $g_i \cdot g_j < 0$ (两任务梯度内积为负)

#### 1.2.2 PCGrad (Project Conflicting Gradients)

核心思想：**投影冲突梯度到正交方向**。

```python
class PCGrad:
    """
    Project Conflicting Gradients
    参考: "PCGrad: Gradient Surgery for Multi-Task Learning"
    """
    def __init__(self, optimizer):
        self.optimizer = optimizer
    
    def pc_grad(self, gradients):
        """
        gradients: list of gradient tensors for each task
        """
        num_tasks = len(gradients)
        
        # 对每个任务的梯度
        for i in range(num_tasks):
            for j in range(num_tasks):
                if i == j:
                    continue
                
                # 计算两个任务梯度的内积
                dot_product = torch.dot(gradients[i], gradients[j])
                
                # 如果内积为负，说明存在冲突
                if dot_product < 0:
                    # 将gi投影到gj的法平面上
                    # proj = gj * (gi·gj) / ||gj||^2
                    proj = gradients[j] * dot_product / (gradients[j].norm() ** 2)
                    
                    # 从gi中减去冲突分量
                    gradients[i] = gradients[i] - proj
        
        # 合并处理后的梯度
        merged_gradient = sum(gradients) / num_tasks
        return merged_gradient
    
    def step(self, task_losses, shared_parameters):
        """
        task_losses: list of loss tensors for each task
        shared_parameters: shared model parameters
        """
        # 计算每个任务的梯度
        gradients = []
        for loss in task_losses:
            self.optimizer.zero_grad()
            loss.backward(retain_graph=True)
            
            # 收集共享参数的梯度
            grad = []
            for p in shared_parameters:
                if p.grad is not None:
                    grad.append(p.grad.view(-1))
            gradients.append(torch.cat(grad))
        
        # 应用PCGrad
        merged_grad = self.pc_grad(gradients)
        
        # 将合并后的梯度设置回参数
        offset = 0
        for p in shared_parameters:
            numel = p.numel()
            p.grad = merged_grad[offset:offset+numel].view_as(p)
            offset += numel
        
        self.optimizer.step()
```

**PCGrad原理图解**：

```
原始梯度: g1 (Task A), g2 (Task B)
冲突情况: g1·g2 < 0

PCGrad操作:
1. 计算g2在g1方向上的投影: proj = (g1·g2/||g2||²) * g2
2. 从g1中减去冲突分量: g1' = g1 - proj
3. 结果: g1' ⊥ g2，两个任务不再冲突

几何解释:
           g1
            │╲
            │ ╲ g1'
            │  ╲
            │   ╲
            │    ╲
            └─────► g2
           proj

修改后的g1'与g2正交，两者更新方向不再冲突
```

#### 1.2.3 GradNorm

核心思想：**自适应调整任务权重，使各任务以相似速度学习**。

```python
class GradNorm:
    """
    Gradient Normalization for Adaptive Loss Balancing
    参考: "GradNorm: Gradient Normalization for Adaptive Loss Balancing in Deep Multitask Networks"
    """
    def __init__(self, task_num, alpha=1.5, lr=0.025):
        self.task_num = task_num
        self.alpha = alpha  # 恢复力强度
        self.lr = lr        # 任务权重的学习率
        
        # 可学习的任务权重
        self.loss_weights = nn.Parameter(torch.ones(task_num))
        
        # 记录初始损失（用于计算相对训练速度）
        self.initial_losses = None
    
    def compute_grad_norm(self, losses, shared_weights):
        """
        计算每个任务梯度相对于共享权重的L2范数
        """
        grad_norms = []
        for loss in losses:
            # 计算梯度
            grads = torch.autograd.grad(loss, shared_weights, retain_graph=True)
            grad_norm = torch.norm(torch.cat([g.view(-1) for g in grads]))
            grad_norms.append(grad_norm)
        return torch.stack(grad_norms)
    
    def forward(self, losses, shared_weights):
        """
        返回加权后的总损失，并更新任务权重
        """
        # 1. 记录初始损失
        if self.initial_losses is None:
            self.initial_losses = [l.item() for l in losses]
        
        # 2. 计算当前各任务梯度范数
        grad_norms = self.compute_grad_norm(losses, shared_weights)
        
        # 3. 计算平均梯度范数
        mean_grad_norm = grad_norms.mean()
        
        # 4. 计算相对反训练速率
        # r_i(t) = (L_i(t) / L_i(0)) / E[L_j(t) / L_j(0)]
        relative_losses = torch.tensor([
            losses[i].item() / self.initial_losses[i]
            for i in range(self.task_num)
        ])
        mean_relative_loss = relative_losses.mean()
        
        # 5. 计算目标梯度范数
        # G̃_i(t) = Ḡ(t) × [r_i(t)]^α
        target_grad_norms = mean_grad_norm * (relative_losses ** self.alpha)
        
        # 6. 构造GradNorm损失来更新任务权重
        # 最小化: |G_i(t) - G̃_i(t)|
        gradnorm_loss = torch.abs(grad_norms - target_grad_norms).sum()
        
        # 7. 加权总损失
        weighted_loss = sum(
            self.loss_weights[i] * losses[i] 
            for i in range(self.task_num)
        )
        
        return weighted_loss, gradnorm_loss, self.loss_weights
```

**GradNorm核心公式**：

$$
\tilde{G}_{i}(t) = \bar{G}(t) \times \left[\frac{L_i(t)}{L_i(0)}\right]^\alpha
$$

- $\tilde{G}_{i}(t)$: 任务i的目标梯度范数
- $\bar{G}(t)$: 所有任务的平均梯度范数
- $L_i(t)/L_i(0)$: 任务i的相对损失（衡量训练速度）
- $\alpha$: 恢复力强度，控制调节力度

#### 1.2.4 Uncertainty Weighting

核心思想：**基于任务的不确定性自动加权**。

```python
class UncertaintyWeighting:
    """
    Multi-Task Learning Using Uncertainty to Weigh Losses
    参考: "Multi-Task Learning Using Uncertainty to Weigh Losses for Scene Geometry and Semantics"
    """
    def __init__(self, task_num):
        self.task_num = task_num
        
        # 可学习的不确定性参数（对数方差）
        self.log_vars = nn.Parameter(torch.zeros(task_num))
    
    def forward(self, losses):
        """
        losses: list of task losses
        """
        total_loss = 0
        
        for i, loss in enumerate(losses):
            # 精度 = 1 / 不确定性 = exp(-log_var)
            precision = torch.exp(-self.log_vars[i])
            
            # 加权损失: precision * loss + log_var
            # log_var作为正则项，防止不确定性无限增大
            weighted_loss = precision * loss + self.log_vars[i]
            total_loss += weighted_loss
        
        return total_loss, torch.exp(self.log_vars)  # 返回损失和不确定性
```

#### 1.2.5 梯度处理方法对比

| 方法 | 核心思想 | 适用场景 | 优点 | 缺点 |
|------|---------|---------|------|------|
| **PCGrad** | 投影冲突梯度 | 梯度方向明显冲突 | 直接解决梯度冲突 | 计算开销较大 |
| **GradNorm** | 平衡训练速度 | 任务学习速度差异大 | 自适应调整权重 | 需要维护初始损失 |
| **Uncertainty** | 不确定性加权 | 任务噪声水平不同 | 理论优雅，端到端 | 可能欠拟合高噪声任务 |
| **DWA** | 动态权重平均 | 任务收敛速度不同 | 简单有效 | 超参数敏感 |

```python
class SIM:
    """
    Search-based Interest Model
    参考: "Search-based User Interest Modeling with Lifelong Sequential Behavior Data"
    """
    def __init__(self, item_vocab_size, embedding_dim, search_mode='hard'):
        self.embedding_dim = embedding_dim
        self.search_mode = search_mode
        
        # Item Embedding
        self.item_embedding = nn.Embedding(item_vocab_size, embedding_dim)
        
        if search_mode == 'soft':
            # Soft Search需要构建ANN索引
            self.faiss_index = None  # 离线构建
        
        # ESU: 使用Transformer建模
        self.transformer = nn.TransformerEncoder(
            nn.TransformerEncoderLayer(
                d_model=embedding_dim,
                nhead=4,
                dim_feedforward=embedding_dim * 4
            ),
            num_layers=2
        )
        
        # 输出层
        self.output_layer = nn.Sequential(
            nn.Linear(embedding_dim * 2, 128),
            nn.ReLU(),
            nn.Linear(128, 1)
        )
    
    def general_search_unit_hard(self, candidate_item, user_long_sequence, 
                                  category_dict, brand_dict, top_k=100):
        """
        Hard Search: 基于属性匹配
        """
        candidate_id = candidate_item.item_id
        candidate_category = category_dict[candidate_id]
        candidate_brand = brand_dict[candidate_id]
        
        # 筛选同Category或同Brand的历史行为
        related_items = []
        for item in user_long_sequence:
            if (category_dict[item.item_id] == candidate_category or 
                brand_dict[item.item_id] == candidate_brand):
                related_items.append(item)
        
        # 按时间倒序取top_k
        related_items = sorted(related_items, 
                               key=lambda x: x.timestamp, 
                               reverse=True)[:top_k]
        
        return related_items
    
    def general_search_unit_soft(self, candidate_item, top_k=100):
        """
        Soft Search: 基于向量相似度
        """
        # 获取候选物品向量
        candidate_vec = self.item_embedding(candidate_item)
        candidate_vec = F.normalize(candidate_vec, p=2, dim=-1)
        
        # 使用FAISS进行ANN检索
        # 注意：FAISS索引需要离线构建和维护
        distances, indices = self.faiss_index.search(
            candidate_vec.cpu().numpy(), 
            top_k
        )
        
        return indices
    
    def exact_search_unit(self, candidate_item, related_items):
        """
        ESU: 使用Transformer建模相关子序列
        """
        # 获取Embedding
        candidate_emb = self.item_embedding(candidate_item)  # [batch, dim]
        
        # 相关子序列Embedding [batch, seq_len, dim]
        related_embs = self.item_embedding(related_items)
        
        # 拼接候选物品作为Query
        # [batch, seq_len + 1, dim]
        sequence = torch.cat([
            related_embs,
            candidate_emb.unsqueeze(1)
        ], dim=1)
        
        # Transformer建模
        # [batch, seq_len + 1, dim]
        transformed = self.transformer(sequence)
        
        # 取候选物品位置的表示作为用户兴趣
        user_interest = transformed[:, -1, :]  # [batch, dim]
        
        # 拼接候选物品向量和用户兴趣
        combined = torch.cat([candidate_emb, user_interest], dim=-1)
        
        # 预测
        prediction = self.output_layer(combined)
        return prediction
```

#### 2.2.2 SimHash-based Attention

```python
class ETA(nn.Module):
    """
    Efficient Target Attention for Lifelong Sequential Behavior Modeling
    参考: "ETA: Estimating Target Attention for Lifelong Sequential Behavior Modeling"
    """
    def __init__(self, embedding_dim, hash_bits=64, top_k=100):
        super().__init__()
        self.embedding_dim = embedding_dim
        self.hash_bits = hash_bits
        self.top_k = top_k
        
        # 可学习的Hash投影矩阵 [embedding_dim, hash_bits]
        self.hash_proj = nn.Parameter(torch.randn(embedding_dim, hash_bits))
        
        # Target Attention
        self.target_attention = TargetAttention(embedding_dim)
        
        # 输出层
        self.output_layer = nn.Sequential(
            nn.Linear(embedding_dim * 2, 128),
            nn.ReLU(),
            nn.Linear(128, 1)
        )
    
    def simhash(self, embeddings):
        """
        将Embedding转换为SimHash签名
        embeddings: [batch, seq_len, embedding_dim]
        return: [batch, seq_len, hash_bits] (binary)
        """
        # 投影到Hash空间
        projected = torch.matmul(embeddings, self.hash_proj)  # [batch, seq_len, hash_bits]
        
        # 二值化 (训练时用sigmoid软化，推理时用sign)
        if self.training:
            hash_codes = torch.sigmoid(projected)  # 连续值，可微
        else:
            hash_codes = torch.sign(projected)  # 二值 {-1, 1}
        
        return hash_codes
    
    def hamming_distance(self, query_hash, sequence_hashes):
        """
        计算Hamming距离
        query_hash: [batch, hash_bits]
        sequence_hashes: [batch, seq_len, hash_bits]
        return: [batch, seq_len] 距离
        """
        # 扩展维度以便广播
        query_expanded = query_hash.unsqueeze(1)  # [batch, 1, hash_bits]
        
        # Hamming距离 = 不同bit的数量
        if self.training:
            # 训练时使用连续近似
            distance = torch.abs(sequence_hashes - query_expanded).sum(dim=-1)
        else:
            # 推理时使用二值计算
            diff = sequence_hashes != query_expanded
            distance = diff.sum(dim=-1).float()
        
        return distance
    
    def forward(self, candidate_item, user_long_sequence):
        """
        candidate_item: [batch, embedding_dim]
        user_long_sequence: [batch, seq_len, embedding_dim]
        """
        batch_size, seq_len, _ = user_long_sequence.shape
        
        # 1. 计算SimHash
        candidate_hash = self.simhash(candidate_item.unsqueeze(1))  # [batch, 1, hash_bits]
        sequence_hashes = self.simhash(user_long_sequence)  # [batch, seq_len, hash_bits]
        
        # 2. 计算Hamming距离，选择Top-K
        distances = self.hamming_distance(candidate_hash.squeeze(1), sequence_hashes)
        
        # 选择距离最小的top_k个 (最相似的)
        _, top_indices = torch.topk(distances, k=min(self.top_k, seq_len), 
                                    dim=-1, largest=False)
        
        # 3. 提取Top-K序列
        # [batch, top_k, embedding_dim]
        topk_sequence = torch.gather(
            user_long_sequence, 
            dim=1, 
            index=top_indices.unsqueeze(-1).expand(-1, -1, self.embedding_dim)
        )
        
        # 4. Target Attention建模
        user_interest = self.target_attention(candidate_item, topk_sequence)
        
        # 5. 预测
        combined = torch.cat([candidate_item, user_interest], dim=-1)
        prediction = self.output_layer(combined)
        
        return prediction

class TargetAttention(nn.Module):
    """
    候选物品作为Query，历史序列作为Key/Value
    """
    def __init__(self, embedding_dim, num_heads=4):
        super().__init__()
        self.num_heads = num_heads
        self.head_dim = embedding_dim // num_heads
        
        self.query_proj = nn.Linear(embedding_dim, embedding_dim)
        self.key_proj = nn.Linear(embedding_dim, embedding_dim)
        self.value_proj = nn.Linear(embedding_dim, embedding_dim)
        self.output_proj = nn.Linear(embedding_dim, embedding_dim)
    
    def forward(self, query, sequence):
        """
        query: [batch, embedding_dim] - 候选物品
        sequence: [batch, seq_len, embedding_dim] - 历史序列
        """
        batch_size, seq_len, _ = sequence.shape
        
        # 投影
        Q = self.query_proj(query).view(batch_size, 1, self.num_heads, self.head_dim)
        K = self.key_proj(sequence).view(batch_size, seq_len, self.num_heads, self.head_dim)
        V = self.value_proj(sequence).view(batch_size, seq_len, self.num_heads, self.head_dim)
        
        # 转置以便计算 [batch, num_heads, seq_len, head_dim]
        Q = Q.transpose(1, 2)
        K = K.transpose(1, 2)
        V = V.transpose(1, 2)
        
        # Attention计算
        scores = torch.matmul(Q, K.transpose(-2, -1)) / (self.head_dim ** 0.5)
        attn_weights = F.softmax(scores, dim=-1)
        
        # 加权求和
        output = torch.matmul(attn_weights, V)  # [batch, num_heads, 1, head_dim]
        output = output.transpose(1, 2).contiguous().view(batch_size, self.embedding_dim)
        
        output = self.output_proj(output)
        return output
```

```python
class TWIN(nn.Module):
    """
    TWIN: TWo-view Interest Network for CTR Prediction
    双塔结构，支持粗排阶段的序列建模
    """
    def __init__(self, embedding_dim, short_term_len=50, long_term_len=1000):
        super().__init__()
        self.embedding_dim = embedding_dim
        self.short_term_len = short_term_len
        self.long_term_len = long_term_len
        
        # Item Embedding共享
        self.item_embedding = nn.Embedding(1000000, embedding_dim)  # vocab_size
        
        # 短期兴趣提取: DIN-style Attention
        self.short_term_extractor = DINAttention(embedding_dim)
        
        # 长期兴趣提取: Target Attention (类似ETA)
        self.long_term_extractor = TargetAttention(embedding_dim)
        
        # 兴趣融合
        self.fusion = nn.Sequential(
            nn.Linear(embedding_dim * 2, embedding_dim),
            nn.ReLU(),
            nn.Linear(embedding_dim, embedding_dim)
        )
        
        # 用户塔输出投影
        self.user_proj = nn.Linear(embedding_dim * 2, embedding_dim)
        
        # 物品塔
        self.item_tower = nn.Sequential(
            nn.Linear(embedding_dim, embedding_dim),
            nn.ReLU(),
            nn.Linear(embedding_dim, embedding_dim)
        )
    
    def user_tower(self, user_profile, short_sequence, long_sequence, candidate_item):
        """
        用户塔: 提取用户兴趣向量
        """
        # 1. 短期兴趣 (最近N个行为)
        short_embs = self.item_embedding(short_sequence)  # [batch, short_len, dim]
        short_interest = self.short_term_extractor(candidate_item, short_embs)
        
        # 2. 长期兴趣 (采样或全量)
        if long_sequence.size(1) > self.long_term_len:
            # 随机采样
            indices = torch.randperm(long_sequence.size(1))[:self.long_term_len]
            long_sequence = long_sequence[:, indices]
        
        long_embs = self.item_embedding(long_sequence)  # [batch, long_len, dim]
        long_interest = self.long_term_extractor(candidate_item, long_embs)
        
        # 3. 融合
        combined_interest = self.fusion(
            torch.cat([short_interest, long_interest], dim=-1)
        )
        
        # 4. 结合用户画像
        user_vector = self.user_proj(
            torch.cat([user_profile, combined_interest], dim=-1)
        )
        
        # L2归一化 (双塔通常需要)
        user_vector = F.normalize(user_vector, p=2, dim=-1)
        
        return user_vector
    
    def item_tower(self, item_features):
        """
        物品塔: 提取物品向量
        """
        item_emb = self.item_embedding(item_features)
        item_vector = self.item_tower(item_emb)
        item_vector = F.normalize(item_vector, p=2, dim=-1)
        return item_vector
    
    def forward(self, user_profile, short_sequence, long_sequence, candidate_item):
        """
        训练时: 计算User和Item向量，内积得分
        """
        user_vec = self.user_tower(user_profile, short_sequence, long_sequence, candidate_item)
        item_vec = self.item_tower(candidate_item)
        
        # 内积得分
        score = (user_vec * item_vec).sum(dim=-1, keepdim=True)
        
        return score, user_vec, item_vec
```

```python
class SENET(nn.Module):
    """
    Squeeze-and-Excitation Network for Feature Recalibration
    COLD使用SENET进行轻量级特征交叉
    """
    def __init__(self, num_fields, reduction_ratio=4):
        super().__init__()
        self.num_fields = num_fields
        
        # Squeeze后的特征维度
        reduced_dim = max(1, num_fields // reduction_ratio)
        
        # Excitation: FC层
        self.excitation = nn.Sequential(
            nn.Linear(num_fields, reduced_dim),
            nn.ReLU(),
            nn.Linear(reduced_dim, num_fields),
            nn.Sigmoid()  # 输出权重在0-1之间
        )
    
    def forward(self, feature_embeddings):
        """
        feature_embeddings: [batch, num_fields, embedding_dim]
        """
        # 1. Squeeze: 沿embedding维度求平均
        # [batch, num_fields]
        z = feature_embeddings.mean(dim=-1)
        
        # 2. Excitation: 学习每个field的重要性权重
        # [batch, num_fields]
        weights = self.excitation(z)
        
        # 3. Reweight: 对每个field的embedding进行加权
        # [batch, num_fields, 1] * [batch, num_fields, embedding_dim]
        reweighted = weights.unsqueeze(-1) * feature_embeddings
        
        return reweighted, weights

class COLD(nn.Module):
    """
    COLD: Computing power cost-aware Online and Lightweight Deep ranking
    参考: "COLD: Computing power cost-aware Online and Lightweight Deep ranking"
    """
    def __init__(self, feature_dims, embedding_dim, num_fields, hash_size=100000):
        super().__init__()
        self.embedding_dim = embedding_dim
        self.num_fields = num_fields
        
        # 共享Embedding表 (使用Hash Trick减少参数量)
        self.embedding = nn.Embedding(hash_size, embedding_dim)
        
        # SENET特征重标定
        self.senet = SENET(num_fields)
        
        # 轻量级DNN
        input_dim = num_fields * embedding_dim
        self.dnn = nn.Sequential(
            nn.Linear(input_dim, 512),
            nn.ReLU(),
            nn.Dropout(0.2),
            nn.Linear(512, 256),
            nn.ReLU(),
            nn.Dropout(0.2),
            nn.Linear(256, 128),
            nn.ReLU(),
            nn.Linear(128, 1),
            nn.Sigmoid()
        )
        
        # 可选: FiBiNET风格的组合
        self.use_fibinet = True
        if self.use_fibinet:
            # Bilinear interaction
            self.bilinear = nn.Parameter(torch.randn(num_fields, num_fields, embedding_dim))
    
    def hash_feature(self, feature_indices):
        """
        特征哈希: 将任意大小的特征ID映射到固定大小的Embedding表
        """
        return feature_indices % self.embedding.num_embeddings
    
    def forward(self, feature_indices_list):
        """
        feature_indices_list: list of [batch] tensors, length = num_fields
        """
        batch_size = feature_indices_list[0].size(0)
        
        # 1. Embedding Lookup with Hash
        embeddings = []
        for indices in feature_indices_list:
            hashed = self.hash_feature(indices)
            emb = self.embedding(hashed)  # [batch, embedding_dim]
            embeddings.append(emb)
        
        # 堆叠: [batch, num_fields, embedding_dim]
        embeddings = torch.stack(embeddings, dim=1)
        
        # 2. SENET特征重标定
        reweighted, field_weights = self.senet(embeddings)
        
        # 3. 展平输入DNN
        dnn_input = reweighted.view(batch_size, -1)
        
        # 4. DNN预测
        output = self.dnn(dnn_input)
        
        return output, field_weights
```

```python
class DistillationLoss(nn.Module):
    """
    知识蒸馏损失
    支持: Response-based (软标签)、Feature-based (中间层)
    """
    def __init__(self, temperature=4.0, alpha=0.5):
        super().__init__()
        self.temperature = temperature
        self.alpha = alpha  # 蒸馏损失权重
        self.kl_div = nn.KLDivLoss(reduction='batchmean')
        self.ce_loss = nn.BCELoss()
    
    def response_distillation(self, student_logits, teacher_logits, labels):
        """
        基于响应的蒸馏: 学习教师模型的输出分布
        """
        # 软标签 (使用temperature缩放)
        soft_teacher = F.softmax(teacher_logits / self.temperature, dim=-1)
        soft_student = F.log_softmax(student_logits / self.temperature, dim=-1)
        
        # KL散度损失
        distill_loss = self.kl_div(soft_student, soft_teacher) * (self.temperature ** 2)
        
        # 硬标签损失
        hard_loss = self.ce_loss(
            torch.sigmoid(student_logits), 
            labels.float()
        )
        
        # 组合损失
        total_loss = self.alpha * distill_loss + (1 - self.alpha) * hard_loss
        
        return total_loss, distill_loss, hard_loss
    
    def feature_distillation(self, student_features, teacher_features):
        """
        基于特征的蒸馏: 学习教师模型的中间表示
        """
        # 特征维度对齐（可能需要投影层）
        if student_features.size(-1) != teacher_features.size(-1):
            # 在实际实现中，需要添加投影层
            pass
        
        # MSE损失
        feat_loss = F.mse_loss(student_features, teacher_features)
        
        return feat_loss

class RankingDistillation:
    """
    精排到粗排的知识蒸馏训练流程
    """
    def __init__(self, teacher_model, student_model):
        self.teacher_model = teacher_model
        self.student_model = student_model
        self.distill_loss = DistillationLoss(temperature=4.0, alpha=0.7)
        
        # 冻结教师模型
        for param in self.teacher_model.parameters():
            param.requires_grad = False
        self.teacher_model.eval()
    
    def train_step(self, batch):
        """
        单步训练
        """
        features, labels = batch
        
        # 1. 教师模型前向 (无梯度)
        with torch.no_grad():
            teacher_logits = self.teacher_model(features)
        
        # 2. 学生模型前向
        student_logits = self.student_model(features)
        
        # 3. 计算蒸馏损失
        total_loss, distill_loss, hard_loss = self.distill_loss.response_distillation(
            student_logits, teacher_logits, labels
        )
        
        return total_loss, distill_loss, hard_loss
```

### 4.2 延迟反馈建模

#### 4.2.1 问题定义

**延迟反馈问题**: 用户点击后，转化行为可能有数小时甚至数天的延迟。

```
时间线:
────────────────────────────────────────────────────────────────────────►
│            │                    │                              │
│    曝光    │       点击         │         转化延迟             │   转化
│    (0s)    │      (10s)         │           (?)                │  (1h~7d)
│            │                    │                              │
└────────────┴────────────────────┴──────────────────────────────┘

问题:
1. 训练时不知道最终是否转化，只能看到当前已观察到的标签
2. 简单使用0作为标签会引入假阴性(False Negative)
3. 等待足够长时间再训练会损失实时性
```

#### 4.2.2 DEFER (Delay Feedback ER)

```python
class DEFER(nn.Module):
    """
    Delay Feedback Modeling with Revisit
    参考: "Addressing Delayed Feedback for Continuous Training"
    """
    def __init__(self, feature_dim, max_delay=24*7):  # 最大延迟7天(小时)
        super().__init__()
        
        # CVR预测模型
        self.cvr_model = nn.Sequential(
            nn.Linear(feature_dim, 128),
            nn.ReLU(),
            nn.Linear(128, 64),
            nn.ReLU(),
            nn.Linear(64, 1),
            nn.Sigmoid()
        )
        
        # 延迟时间预测模型 (输出延迟概率分布)
        self.delay_model = nn.Sequential(
            nn.Linear(feature_dim, 64),
            nn.ReLU(),
            nn.Linear(64, max_delay),
            nn.Softmax(dim=-1)
        )
        
        self.max_delay = max_delay
    
    def forward(self, features, elapsed_time=None):
        """
        features: 特征
        elapsed_time: 从点击到当前的时间(小时)，推理时可为None
        """
        # 1. 预测CVR
        cvr = self.cvr_model(features)  # [batch, 1]
        
        # 2. 预测延迟分布
        delay_dist = self.delay_model(features)  # [batch, max_delay]
        
        if elapsed_time is not None:
            # 训练时: 使用已观察时间调整
            # 计算在elapsed_time之前转化的概率
            # P(convert | elapsed_time) = P(convert) * P(delay <= elapsed_time | convert)
            
            # 创建时间掩码 (delay <= elapsed_time)
            time_mask = torch.arange(self.max_delay, device=features.device).unsqueeze(0)
            time_mask = (time_mask <= elapsed_time.unsqueeze(-1)).float()  # [batch, max_delay]
            
            # 在时间窗口内的转化概率
            p_convert_in_window = (delay_dist * time_mask).sum(dim=-1, keepdim=True)
            
            # 调整后的CVR
            adjusted_cvr = cvr * p_convert_in_window
            
            return adjusted_cvr, cvr, delay_dist
        else:
            # 推理时: 返回最终CVR
            return cvr
    
    def loss(self, features, labels, elapsed_time):
        """
        DEFER损失函数
        labels: 当前观察到的转化标签 (0或1)
        elapsed_time: 已等待的时间
        """
        adjusted_cvr, cvr, delay_dist = self.forward(features, elapsed_time)
        
        # 对于已转化的样本，使用adjusted_cvr
        # 对于未转化的样本，需要考虑两种可能:
        # 1. 真实负样本 (不会转化)
        # 2. 延迟正样本 (尚未观察到转化)
        
        # 使用EM算法处理未观察到的延迟转化
        # 简化版: 使用Importance Weighting
        
        bce_loss = F.binary_cross_entropy(adjusted_cvr, labels.float())
        
        return bce_loss
```

#### 4.2.3 ES-DFM (Elapsed Time Sampling DFM)

```python
class ESDFM(nn.Module):
    """
    Elapsed-Time Sampling Delay Feedback Model
    参考: "ESDFM: Explicitly-Separated Delayed Feedback Model"
    """
    def __init__(self, feature_dim, time_buckets=24):
        super().__init__()
        self.time_buckets = time_buckets
        
        # 基础CVR预测
        self.cvr_predictor = nn.Sequential(
            nn.Linear(feature_dim, 128),
            nn.ReLU(),
            nn.Linear(128, 1),
            nn.Sigmoid()
        )
        
        # 延迟时间分布预测 (分桶)
        self.delay_predictor = nn.Sequential(
            nn.Linear(feature_dim, 64),
            nn.ReLU(),
            nn.Linear(64, time_buckets),
            nn.Softmax(dim=-1)
        )
        
        # Fake Negative (假阴性) 识别
        self.fake_negative_detector = nn.Sequential(
            nn.Linear(feature_dim + 1, 64),  # 特征+已等待时间
            nn.ReLU(),
            nn.Linear(64, 1),
            nn.Sigmoid()
        )
    
    def forward(self, features, elapsed_hours=None):
        cvr = self.cvr_predictor(features)
        delay_probs = self.delay_predictor(features)
        
        if elapsed_hours is not None:
            # 假阴性检测
            elapsed_bucket = (elapsed_hours / (24 * 7) * self.time_buckets).long()
            elapsed_bucket = elapsed_bucket.clamp(0, self.time_buckets - 1)
            
            # 该样本是假阴性的概率
            fn_prob = self.fake_negative_detector(
                torch.cat([features, elapsed_hours.unsqueeze(-1) / 168.0], dim=-1)
            )
            
            return cvr, delay_probs, fn_prob
        
        return cvr
    
    def compute_loss(self, features, click_label, convert_label, 
                     elapsed_time, is_fake_negative_weight=0.5):
        """
        ES-DFM损失:
        - 已确定转化: 正常计算CVR损失
        - 已确定未转化: 正常计算
        - 等待中(未确定): 考虑假阴性可能
        """
        cvr, delay_probs, fn_prob = self.forward(features, elapsed_time)
        
        # 分类讨论
        # 1. 已转化样本 (convert_label=1)
        converted_mask = (convert_label == 1).float()
        converted_loss = F.binary_cross_entropy(
            cvr, convert_label.float(), reduction='none'
        ) * converted_mask
        
        # 2. 未转化但可能是假阴性
        not_converted_mask = (convert_label == 0).float()
        
        # 假阴性损失: 认为真实标签是1，但当前观察到0
        fake_negative_loss = fn_prob * F.binary_cross_entropy(
            cvr, torch.ones_like(convert_label).float(), reduction='none'
        )
        
        # 真阴性损失: 真实标签确实是0
        true_negative_loss = (1 - fn_prob) * F.binary_cross_entropy(
            cvr, convert_label.float(), reduction='none'
        )
        
        not_converted_loss = (fake_negative_loss + true_negative_loss) * not_converted_mask
        
        total_loss = (converted_loss + not_converted_loss).mean()
        
        return total_loss
```

```python
import torch

class CompleteMultiTaskModel(nn.Module):
    """
    完整多任务学习模型 (MMOE + GradNorm)
    """
    def __init__(self, 
                 feature_dims: dict,
                 embedding_dim: int = 64,
                 expert_num: int = 4,
                 expert_dim: int = 256,
                 task_num: int = 3,
                 use_gradnorm: bool = True):
        super().__init__()
        
        self.task_num = task_num
        self.use_gradnorm = use_gradnorm
        
        # Embedding层
        self.embeddings = nn.ModuleDict({
            name: nn.Embedding(dim, embedding_dim)
            for name, dim in feature_dims.items()
        })
        
        # 计算输入维度
        self.input_dim = len(feature_dims) * embedding_dim
        
        # MMOE层
        self.experts = nn.ModuleList([
            nn.Sequential(
                nn.Linear(self.input_dim, expert_dim),
                nn.ReLU(),
                nn.Dropout(0.2),
                nn.Linear(expert_dim, expert_dim)
            ) for _ in range(expert_num)
        ])
        
        # Gating Networks
        self.gates = nn.ModuleList([
            nn.Sequential(
                nn.Linear(self.input_dim, expert_num),
                nn.Softmax(dim=-1)
            ) for _ in range(task_num)
        ])
        
        # Task Towers
        self.towers = nn.ModuleList([
            nn.Sequential(
                nn.Linear(expert_dim, 128),
                nn.ReLU(),
                nn.Dropout(0.3),
                nn.Linear(128, 64),
                nn.ReLU(),
                nn.Linear(64, 1)
            ) for _ in range(task_num)
        ])
        
        # GradNorm参数
        if use_gradnorm:
            self.loss_weights = nn.Parameter(torch.ones(task_num))
            self.initial_losses = None
    
    def forward(self, features: dict) -> List[torch.Tensor]:
        """
        features: dict of {field_name: indices}
        """
        # Embedding lookup
        embs = `self.embeddings[name` 
                for name, indices in features.items()]
        x = torch.cat(embs, dim=-1)  # [batch, input_dim]
        
        # Expert outputs
        expert_outs = torch.stack([expert(x) for expert in self.experts], dim=1)
        # [batch, expert_num, expert_dim]
        
        # Task-specific aggregation
        task_outputs = []
        for i in range(self.task_num):
            gate_weights = self.gatesi  # [batch, expert_num]
            fused = torch.sum(
                gate_weights.unsqueeze(-1) * expert_outs,
                dim=1
            )  # [batch, expert_dim]
            output = self.towersi
            task_outputs.append(torch.sigmoid(output))
        
        return task_outputs
    
    def compute_loss(self, features, labels, task_names):
        """
        计算多任务损失
        """
        predictions = self.forward(features)
        
        losses = []
        for i, (pred, label) in enumerate(zip(predictions, labels)):
            task_loss = F.binary_cross_entropy(pred.squeeze(), label.float())
            
            if self.use_gradnorm:
                # GradNorm加权
                weighted_loss = self.loss_weights[i] * task_loss
                losses.append(weighted_loss)
            else:
                losses.append(task_loss)
        
        total_loss = sum(losses)
        return total_loss, losses, predictions
```

```python
class CompleteLongSequenceModel(nn.Module):
    """
    完整长序列模型 (SIM-style)
    """
    def __init__(self,
                 item_vocab_size: int,
                 embedding_dim: int = 64,
                 long_seq_len: int = 10000,
                 short_seq_len: int = 50,
                 retrieved_len: int = 100):
        super().__init__()
        
        self.embedding_dim = embedding_dim
        self.long_seq_len = long_seq_len
        self.short_seq_len = short_seq_len
        self.retrieved_len = retrieved_len
        
        # Embedding
        self.item_embedding = nn.Embedding(item_vocab_size, embedding_dim)
        
        # 短期序列: DIN Attention
        self.din_attention = DINLayer(embedding_dim)
        
        # 长期序列: 两阶段 (GSU + ESU)
        self.gsu_projection = nn.Linear(embedding_dim, embedding_dim)
        self.esu_transformer = nn.TransformerEncoder(
            nn.TransformerEncoderLayer(
                d_model=embedding_dim,
                nhead=4,
                dim_feedforward=embedding_dim * 4,
                batch_first=True
            ),
            num_layers=2
        )
        
        # 融合与预测
        self.fusion = nn.Sequential(
            nn.Linear(embedding_dim * 3, 256),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(256, 128),
            nn.ReLU(),
            nn.Linear(128, 1)
        )
    
    def general_search_unit(self, candidate_emb, long_sequence, top_k=100):
        """
        GSU: 基于向量相似度检索相关子序列
        """
        # 计算相似度
        candidate_norm = F.normalize(candidate_emb, p=2, dim=-1)
        sequence_norm = F.normalize(long_sequence, p=2, dim=-1)
        
        # 余弦相似度
        similarities = torch.matmul(sequence_norm, candidate_norm.unsqueeze(-1)).squeeze(-1)
        
        # Top-K检索
        topk_vals, topk_indices = torch.topk(similarities, k=min(top_k, similarities.size(1)))
        
        # 提取相关子序列
        retrieved = torch.gather(
            long_sequence, 
            dim=1,
            index=topk_indices.unsqueeze(-1).expand(-1, -1, self.embedding_dim)
        )
        
        return retrieved
    
    def exact_search_unit(self, candidate_emb, retrieved_sequence):
        """
        ESU: 使用Transformer精细化建模
        """
        # 拼接候选物品
        sequence = torch.cat([retrieved_sequence, candidate_emb.unsqueeze(1)], dim=1)
        
        # Transformer建模
        transformed = self.esu_transformer(sequence)
        
        # 取候选物品位置的表示
        user_interest = transformed[:, -1, :]
        
        return user_interest
    
    def forward(self, 
                candidate_item,
                short_sequence,
                long_sequence=None):
        """
        forward
        """
        # Embedding
        candidate_emb = self.item_embedding(candidate_item)
        
        # 短期兴趣
        short_embs = self.item_embedding(short_sequence)
        short_interest = self.din_attention(candidate_emb, short_embs)
        
        # 长期兴趣 (如果有)
        if long_sequence is not None:
            long_embs = self.item_embedding(long_sequence)
            
            # GSU检索
            retrieved = self.general_search_unit(candidate_emb, long_embs, self.retrieved_len)
            
            # ESU建模
            long_interest = self.exact_search_unit(candidate_emb, retrieved)
        else:
            long_interest = torch.zeros_like(candidate_emb)
        
        # 融合预测
        combined = torch.cat([candidate_emb, short_interest, long_interest], dim=-1)
        output = self.fusion(combined)
        
        return torch.sigmoid(output)

class DINLayer(nn.Module):
    """DIN Attention Layer"""
    def __init__(self, embedding_dim):
        super().__init__()
        self.attention = nn.Sequential(
            nn.Linear(embedding_dim * 4, 256),
            nn.ReLU(),
            nn.Linear(256, 128),
            nn.ReLU(),
            nn.Linear(128, 1)
        )
    
    def forward(self, query, sequence):
        # query: [batch, dim]
        # sequence: [batch, seq_len, dim]
        
        seq_len = sequence.size(1)
        
        # 扩展query
        query_expanded = query.unsqueeze(1).expand(-1, seq_len, -1)
        
        # 拼接特征: [query, item, query-item, query*item]
        diff = query_expanded - sequence
        prod = query_expanded * sequence
        
        concat = torch.cat([query_expanded, sequence, diff, prod], dim=-1)
        
        # 计算注意力权重
        attn_scores = self.attention(concat).squeeze(-1)  # [batch, seq_len]
        attn_weights = F.softmax(attn_scores, dim=-1)
        
        # 加权求和
        output = torch.sum(attn_weights.unsqueeze(-1) * sequence, dim=1)
        
        return output
```

```python
class Trainer:
    """
    完整训练Pipeline示例
    """
    def __init__(self, model, optimizer, device='cuda'):
        self.model = model.to(device)
        self.optimizer = optimizer
        self.device = device
    
    def train_epoch(self, dataloader):
        self.model.train()
        total_loss = 0
        
        for batch in dataloader:
            # 解包数据
            features, labels = batch
            features = {k: v.to(self.device) for k, v in features.items()}
            labels = labels.to(self.device)
            
            # 前向
            self.optimizer.zero_grad()
            predictions = self.model(features)
            
            # 计算损失
            if isinstance(predictions, list):  # 多任务
                loss = sum(
                    F.binary_cross_entropy(pred.squeeze(), labels[:, i].float())
                    for i, pred in enumerate(predictions)
                )
            else:  # 单任务
                loss = F.binary_cross_entropy(predictions.squeeze(), labels.float())
            
            # 反向
            loss.backward()
            torch.nn.utils.clip_grad_norm_(self.model.parameters(), 1.0)
            self.optimizer.step()
            
            total_loss += loss.item()
        
        return total_loss / len(dataloader)
    
    def evaluate(self, dataloader):
        self.model.eval()
        all_preds = []
        all_labels = []
        
        with torch.no_grad():
            for batch in dataloader:
                features, labels = batch
                features = {k: v.to(self.device) for k, v in features.items()}
                
                predictions = self.model(features)
                
                if isinstance(predictions, list):
                    predictions = predictions[0]  # 主任务
                
                all_preds.append(predictions.cpu())
                all_labels.append(labels)
        
        # 计算AUC
        from sklearn.metrics import roc_auc_score
        preds = torch.cat(all_preds).numpy()
        labels = torch.cat(all_labels).numpy()
        auc = roc_auc_score(labels, preds)
        
        return auc
```

## 📐 核心公式直观理解

### PLE（Progressive Layered Extraction）

$$
\mathbf{o}_k^{(l)} = \sum_i g_k^{(i)}(x) \cdot E_i^{(l)}(x) + g_k^{(\text{shared})}(x) \cdot E_{\text{shared}}^{(l)}(x)
$$

**直观理解**：PLE 在 MMoE 基础上加入"每个任务独有的 expert"——既有共享 expert 提供通用知识，又有私有 expert 学习任务特定的模式。多层堆叠让低层学通用交叉、高层学任务特定交叉，减少负迁移（CTR 和 CVR 方向冲突时互不干扰）。

### FiBiNET 的 SENET 特征重标定

$$
\mathbf{v}_i' = s_i \cdot \mathbf{v}_i, \quad \mathbf{s} = \sigma(W_2 \cdot \text{ReLU}(W_1 \cdot \mathbf{z}))
$$

- $\mathbf{z}$：所有特征 embedding 的 pooling 向量

**直观理解**：不是所有特征都同样重要——SENET 先"压缩"所有特征为一个全局摘要 $\mathbf{z}$，再根据摘要为每个特征分配重要性权重 $s_i$。相当于"全局看一眼后决定关注哪些特征"。原本用于图像通道选择，移植到推荐中选择重要特征域。

### Focal Loss 解决样本不平衡

$$
\text{FL}(p_t) = -(1-p_t)^\gamma \log(p_t)
$$

**直观理解**：CTR 数据中负样本（不点击）远多于正样本。标准交叉熵下，大量"简单"负样本贡献的梯度淹没了少数"困难"正样本。Focal Loss 给"模型已经很自信"的样本降权（$(1-p_t)^\gamma$ 接近 0），让训练聚焦于"模型不确定"的困难样本。$\gamma=2$ 是常用值。

### 延迟反馈建模（DEFER/ES-DFM）

**核心公式**：

$$
P(\text{convert}|\text{click}, t) = P(\text{convert}) \times P(\text{delay} \leq t | \text{convert})
$$

**ES-DFM 假阴性检测**：对未转化样本，用 fake negative detector 估计其为延迟正样本的概率，加权损失。

**工程要点**：
- 点击后7天内成交→确定正样本
- 7天后成交→按延迟概率加权
- 假阴性率随等待时间递减

### 面试高频追问

**Q: SIM的Hard Search和Soft Search如何选择？**
A: 两者结合：Hard Search速度快做初筛（Category/Brand匹配），Soft Search语义召回更全面（FAISS向量检索）。候选物品属性完整时Hard Search优先，属性缺失时依赖Soft Search。

**Q: TWIN和普通双塔的本质区别？**
A: 普通双塔用户侧是静态特征，TWIN在用户塔内部引入序列建模。关键：序列建模必须在输出用户向量之前完成，保持最终仍是向量内积形式以支持ANN检索。

**Q: 知识蒸馏中温度参数T的作用？**
A: T越高，Teacher的输出分布越平滑，Student能学到更多"软"信息（各类别间的相对关系）。T=1退化为标准softmax。CTR蒸馏中T=2-4效果较好。

## 记忆助手 💡

### 类比法

- **粗排 = 安检门**：快速过滤明显不合格的（简单模型，低延迟），不要求精确，只要不漏掉好的
- **精排 = 面试官**：仔细评估每个候选（复杂模型，DIN/DCN），给出精确分数
- **ESMM = 概率链式法则**："转化 = 先点击 × 再转化"，在全空间训练避免 CVR 的样本选择偏差
- **知识蒸馏 = 名师带徒弟**：大模型（老师）输出"软标签"（各类别概率分布），小模型（徒弟）学的不仅是对错，还有"为什么"
- **在线学习 = 边考试边复习**：模型看到用户实时反馈后立即更新参数（FTRL），不等下次大考（离线重训）

### 口诀/助记

- **推荐漏斗四步走**："召排粗精重" — 召回 → 粗排 → 精排 → 重排
- **精排进阶三件套**："序列建模 + 特征交叉 + 多任务"
- **蒸馏温度口诀**："温度高学软知识，温度低学硬标签，CTR 蒸馏 T=2-4 刚好"
- **在线学习记**："FTRL 在线更，稀疏正则化，实时追漂移"

### 面试一句话

- **粗排设计**："粗排是精排的轻量近似，用知识蒸馏从精排大模型学到小模型，在 10ms 内完成千级候选到百级的筛选"
- **ESMM**："利用 pCTCVR = pCTR × pCVR 的链式分解，在全曝光空间训练 CVR 子网络，避免只用点击样本训练导致的样本选择偏差"
- **在线学习**："FTRL 算法对每条样本实时更新模型参数，追踪数据分布漂移，配合 L1 正则保持模型稀疏性"
