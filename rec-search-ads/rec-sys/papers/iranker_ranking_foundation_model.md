# iRanker: Towards Ranking Foundation Model

> 来源：https://arxiv.org/abs/2506.21638 | 领域：推荐系统 | 学习日期：20260331

## 问题定义

推荐排序模型需针对每个场景重新训练，缺乏跨场景通用排序能力。能否构建排序基础模型？

## 核心方法与创新点

1. **统一排序表征**：不同场景排序信号统一到同一表征空间
2. **大规模预训练**：多个推荐场景联合预训练，学习通用排序能力
3. **Prompt-based微调**：场景Prompt快速适配新场景

$$h = \text{Encoder}([\text{Scene\_Prompt}; \text{User\_Feat}; \text{Item\_Feat}])$$

4. **对比学习目标**：正样本对与负样本对的对比学习

## 实验结论

iRanker在5个不同推荐场景上zero-shot达到场景专有模型90%水平，few-shot（100样本）超过专有模型2-3%。

## 工程落地要点

- 基础模型可部署为统一服务，降低多场景维护成本
- 新场景冷启动从零变为Prompt微调
- 模型体积较大需压缩
- 适合平台型企业多业务线场景

## 面试考点

1. **排序基础模型的核心挑战？** 特征空间异构、标签分布差异大
2. **如何统一不同场景特征？** 特征token化 + 统一Embedding空间
3. **与NLP基础模型的类比？** 类似BERT预训练后fine-tune范式
