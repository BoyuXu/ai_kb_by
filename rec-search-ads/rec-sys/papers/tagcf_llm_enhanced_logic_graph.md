# TagCF: LLM-Enhanced Tag-based Logic Graph for Collaborative Filtering

> 来源：arXiv 2505.11234 | 领域：rec-sys | 学习日期：20260402

## 问题定义

传统协同过滤仅基于用户-物品交互矩阵建模，忽略了用户的社会角色信息（如"新手妈妈"、"数码极客"）。TagCF 利用 LLM 从用户行为中提取角色标签，构建逻辑图谱辅助推荐。

## 核心方法与创新点

1. **LLM 角色标签提取**：用 LLM 分析用户历史行为序列，推断用户的角色标签（多值）
2. **逻辑图谱构建**：将用户、物品和角色标签构建为异构图，利用图神经网络传播信息
3. **逻辑推理**：在图上定义逻辑规则（如"新手妈妈 → 偏好母婴用品"），提升推荐的可解释性
4. **冷启动改善**：新用户即使行为稀少，通过角色标签匹配也能获得合理推荐

## 实验结论

- Recall@20 提升 4.2%，冷启动用户的推荐准确率提升 15%
- 可解释性评估中，84% 的推荐理由被用户认为合理

## 工程落地要点

- **标签维护**：角色标签需要定期用 LLM 更新，控制 API 成本
- **图规模**：大规模系统中图的节点和边数量巨大，需要分布式图计算框架

## 常见考点

1. **Q：角色标签和传统用户画像有什么区别？**
   A：传统画像基于统计特征（年龄、性别），角色标签基于行为语义（购物达人、健身爱好者），更细粒度且动态变化。
2. **Q：LLM 提取标签的成本如何控制？**
   A：批量处理 + 缓存 + 分层采样，只对活跃用户定期更新。
