# Agent0: Leveraging LLM Agents to Discover Multi-value Features from Text for Enhanced Recommendations

> 来源：arXiv 2507.18963 | 领域：rec-sys | 学习日期：20260402

## 问题定义

推荐系统中特征工程仍然高度依赖人工设计，尤其是从非结构化文本中提取多值特征（如商品属性、用户偏好标签）需要大量领域知识和人力。Agent0 利用 LLM Agent 自动从文本数据中发现并提取多值特征，减少人工干预。

## 核心方法与创新点

1. **LLM Agent 自动特征发现**：设计 Agent 工作流自动分析文本数据，识别有价值的多值特征维度（如"风格标签"、"使用场景"）
2. **迭代优化**：Agent 通过 downstream 推荐效果的反馈循环迭代优化特征提取策略
3. **多值特征编码**：提出变长多值特征的高效编码方案，支持不同物品拥有不同数量的特征值
4. **成本控制**：通过缓存和批处理策略控制 LLM API 调用成本

## 实验结论

- 在电商推荐场景，Agent0 自动发现的特征使 CTR 提升 1.5%，超过人工设计特征
- 特征发现过程从数周人工工作缩短到数小时自动化流程
- 发现的特征具有可解释性，与人工特征有 70% 重叠，另外 30% 是人工未曾考虑的新维度

## 工程落地要点

- **LLM 成本控制**：大规模物品库需要分层采样策略，先在样本上发现特征模式，再批量提取
- **特征质量验证**：自动发现的特征需要经过 A/B 测试验证，设计特征质量评分机制
- **增量更新**：新物品上线时自动触发特征提取，保持特征的时效性

## 常见考点

1. **Q：为什么用 LLM Agent 而不是直接用 LLM？**
   A：Agent 能通过工具调用和反馈循环迭代优化特征策略，而非一次性生成。Agent 可以评估特征效果并自动调整。
2. **Q：多值特征编码的挑战是什么？**
   A：不同物品的特征值数量不同，需要变长编码。常用方案包括 pooling、attention 聚合和 multi-hot 编码。
