# MaRI：结构重参数化加速大规模推荐模型推理

> 来源：MaRI: Accelerating Ranking Model Inference via Structural Re-parameterization in Large Scale RS | 日期：20260318 | 领域：rec-sys

## 问题定义

工业推荐系统精排模型（如 DNN、DeepFM、DCN）在训练时为了充分拟合，往往使用多分支结构（多个 MLP、多路交叉网络并行）。这些多分支在推理时带来显著的内存访问压力和计算冗余，成为在线服务延迟的瓶颈。如何在不损失精度的前提下降低推理时的模型复杂度，是工程关键问题。

## 核心方法与创新点

1. **结构重参数化（Structural Re-parameterization）**：灵感来自 CV 领域的 RepVGG。训练时保留多分支结构（丰富梯度流），推理时将多个并行分支的权重等价合并为单一分支，实现"训练时宽、推理时窄"。

2. **推荐场景适配**：
   - 针对 Embedding + MLP 结构，将多个并行 MLP 分支（具有相同输入/输出维度）的权重矩阵直接相加。
   - 针对包含 BatchNorm 的分支（推荐模型常用 LayerNorm），推导 LN 吸收到线性层的等价变换公式。
   - 对跨维度（bottleneck）结构设计降维对齐补丁，保证维度一致后再合并。

3. **训练时蒸馏辅助**：在多分支训练阶段加入自蒸馏损失，使各分支输出尽量一致，降低合并后的精度损失。

## 实验结论

- 在某大型电商平台精排模型上，推理 FLOPs 降低 40%，P99 延迟降低 35ms（绝对值）。
- AUC 损失 < 0.001（统计不显著），线上 A/B 收益持平于原始多分支模型。
- 内存占用降低约 30%，有效缓解 GPU 显存压力，为 batch size 提升提供空间。

## 工程落地要点

- **合并时机**：在模型导出（export to ONNX/TorchScript）前完成合并，不影响训练流程。
- **维度对齐检查**：合并前必须验证所有并行分支的输入输出维度完全一致，否则需补齐 padding 或放弃合并该层。
- **LN 吸收公式**：`W_merged = W / sqrt(var + eps) * gamma`，`b_merged = (b - mean) / sqrt(var + eps) * gamma + beta`，需要在导出脚本中实现。
- **回归测试**：合并后必须用线下样本验证 logit 数值一致性（max diff < 1e-5），防止数值误差累积。

## 面试考点

**Q: 什么是结构重参数化？推荐系统为什么能用？**
- 训练时多分支，推理时等价合并为单分支。推荐的 MLP 层是线性变换，满足加法等价的代数条件，因此可直接合并。

**Q: BatchNorm / LayerNorm 如何"吸收"到线性层？**
- LN 本质是对输入做仿射变换，可以预先计算统计量，将 scale/shift 折叠进前一层权重矩阵和偏置，推理时省去 LN 计算。

**Q: 如何保证合并后精度不下降？**
- 自蒸馏让各分支在训练期间输出对齐，合并后的单分支行为接近各分支均值；同时数值验证确保合并无计算错误。
