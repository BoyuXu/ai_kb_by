# Cold-Starts in Generative Recommendation

> 来源：arXiv 2025 | 领域：rec-sys | 学习日期：20260404

## 问题定义

冷启动是推荐系统的长期挑战：新用户/新物品缺乏历史交互数据，传统协同过滤完全失效。生成式推荐（Generative Recommendation）直接生成物品 ID 序列，冷启动场景下面临 **ID 生成偏差** 与 **语义-ID 映射断裂** 双重问题。

$$P(i_{t+1} | u, i_{1:t}) \rightarrow P(\text{id}_{t+1} | u, \text{id}_{1:t})$$

冷启动物品因从未出现在训练序列中，其 ID token 概率趋近于零。

## 核心方法与创新点

1. **语义 ID 初始化**：利用物品文本（标题、类目、描述）通过预训练语言模型生成初始 ID 向量，使冷启动物品拥有合理的先验概率分布。

2. **温暖提示（Warm-up Prompting）**：构造虚拟交互序列（基于物品相似用户的合成轨迹），在推理时注入为 few-shot context：

$$\text{context} = [\tilde{i}_1, \tilde{i}_2, \ldots, \tilde{i}_k, i_{\text{cold}}]$$

3. **双塔对齐损失**：在训练时引入对比损失对齐 ID 嵌入与语义嵌入：

$$\mathcal{L}_{\text{align}} = -\log \frac{\exp(\text{sim}(e_{\text{id}}, e_{\text{sem}}) / \tau)}{\sum_{j} \exp(\text{sim}(e_{\text{id}}, e_j) / \tau)}$$

4. **渐进 ID 分配**：冷启动物品先用语义聚类 ID，随交互积累逐步迁移至精确 ID。

## 实验结论

- 在工业数据集上，冷启动 Recall@10 相比基线提升 **+23.5%**
- 热门物品性能无损（±0.3% 以内）
- 语义初始化单独贡献约 60% 的提升
- Warm-up Prompting 在极端冷启动（0 交互）场景最有效

## 工程落地要点

- ID 分配需要在线服务中维护 **两套 Embedding**（语义 + 精确），增加存储约 2x
- 建议设置阈值：交互数 < 50 时使用语义 ID，≥ 50 时切换精确 ID
- 虚拟交互序列生成：离线批量，每日更新，避免实时计算开销
- 对齐损失权重 λ 需调参，过大会损害热门物品性能

## 面试考点

1. **Q**: 生成式推荐如何解决冷启动？  
   **A**: 通过语义 ID 初始化（LLM 编码物品文本），使冷启动物品获得先验概率；通过 Warm-up Prompting 构造虚拟序列。

2. **Q**: 语义 ID 与精确 ID 如何融合？  
   **A**: 对比损失对齐两空间，渐进式 ID 分配策略（按交互阈值切换）。

3. **Q**: 为什么传统生成式推荐对冷启动物品失效？  
   **A**: 模型只学过出现在训练序列中的 ID token，冷启动物品 ID 从未被预测过，生成概率趋零。
