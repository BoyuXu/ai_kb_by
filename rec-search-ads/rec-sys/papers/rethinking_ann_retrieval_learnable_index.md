# Rethinking ANN-based Retrieval: Multifaceted Learnable Index for Large-scale Rec

> 来源：https://arxiv.org/abs/2602.16124 | 领域：推荐系统 | 学习日期：20260331

## 问题定义

传统ANN检索（HNSW、IVF）的索引结构独立于推荐模型训练，存在「索引-模型」目标不一致问题。

## 核心方法与创新点

1. **端到端可学习索引**：索引参数参与模型反向传播
2. **层级残差量化**：多层残差编码提升索引精度

$$
c(v) = c_1(v) + c_2(v - c_1(v)) + ... + c_L(v - \sum_{l=1}^{L-1} c_l)
$$

3. **多面表征**：从内容/协同/上下文多个语义维度建立索引
4. **联合优化**：推荐Loss + 索引重建Loss + 负载均衡Loss

## 实验结论

在10亿级工业数据上，可学习索引相比HNSW在Recall@200提升18%，检索延迟仅增加5%。

## 工程落地要点

- 索引更新需重新训练，适合日更/周更频率
- 可与现有向量数据库插件化集成
- 支持增量索引更新减少计算开销
- 多面索引天然支持多路召回

## 常见考点

1. **传统ANN索引为什么次优？** 优化最近邻准确率而非推荐指标
2. **可学习索引训练开销？** 比标准训练增加约30%，离线可接受
3. **层级残差量化的好处？** 逐层细化表征，提升近似精度
