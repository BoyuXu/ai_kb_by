## 🧭 为什么要从"统一视角"看推荐和广告？

传统上，推荐系统和广告系统被视为不同领域：
- 推荐：内容平台驱动，目标是用户满意度和留存
- 广告：商业变现驱动，目标是广告主 ROI 和平台收入

但 2024-2026 年出现了一个令人惊讶的现象：**两个领域在技术路线上高度收敛**，都走向了"生成式范式"：

| 维度 | 推荐系统 | 广告系统 | 共同趋势 |
|------|---------|---------|---------|
| 核心范式 | HSTU → 生成式推荐 | EGA-v2/GPR → 生成式广告 | 判别式→生成式 |
| 物品表示 | Semantic ID (RQ-VAE) | Semantic ID (RQ codes) | 离散化token |
| 模型架构 | Transformer decoder | Transformer decoder | 自回归生成 |
| 多任务 | SMES 稀疏专家 | MGOE 宏图专家 | 专家混合+稀疏路由 |
| 序列建模 | HSTU pointwise attn | UniROM 序列感知CTR | 行为序列统一建模 |
| 扩展范式 | PROMISE test-time scaling | GRAD diffusion出价 | 生成+概率建模 |
| 未来方向 | Agentic RecSys | Agentic 广告投放 | 智能体自适应 |

## 📐 核心公式与原理

### 1. 生成式推荐的核心范式（HSTU）

HSTU 用无 softmax 的 Pointwise Aggregated Attention 实现高效序列建模：

$$
\mathbf{O} = \text{silu}(Q) \cdot \text{silu}(K)^\top \cdot V \cdot M_{\text{causal}}
$$

其中 $\text{silu}(x) = x \cdot \sigma(x)$，避免了 softmax 在长序列上的 $O(n^2)$ 瓶颈。推荐生成目标：

$$
\mathcal{L} = -\sum_{t=1}^{T} \log P(a_t \mid a_{<t}, \mathbf{u}; \theta)
$$

**与广告系统的直接对应**：GPR 的 Hierarchical Hybrid Decoder 采用完全相同的自回归框架，只是将"推荐物品"换成了"广告候选"：

$$
P(a \mid u, C) = \prod_{l=1}^{L} P_{\text{coarse}(g_l \mid u) \cdot P_{\text{fine}(a \mid g_l, u)
$$

### 2. Semantic ID 的跨域共用

推荐（Reasoning Semantic IDs）和广告（EGA-v2）都使用 RQ-VAE 将物品/广告量化为多层离散 token：

$$
\text{SemanticID}(x) = (c_1, c_2, \ldots, c_L), \quad c_l = \arg\min_{j \in [K]} \|\mathbf{r}_{l-1} - \mathbf{e}_j^{(l)}\|^2
$$

其中残差更新：$\mathbf{r}_l = \mathbf{r}_{l-1} - \mathbf{e}_{c_l}^{(l)}$

**差异**：推荐侧用 reasoning tokens 增强语义理解（$P(c_l \mid \mathbf{z}_l, c_{<l}, \mathbf{h}_u)$）；广告侧（EGA-v2）用 $256 \times 4$ 层码本适配百万级广告库，更注重工程可扩展性。

### 3. 多任务专家的两种宏观路由

**推荐侧 SMES（稀疏门控）**：每任务只激活 Top-K 专家：
$$
g_{t,m} = \begin{cases} s_{t,m} & \text{if } s_{t,m} \in \text{TopK}(\{s_{t,j}\}_{j=1}^{M}) \\ 0 & \text{otherwise} \end{cases}
$$

**广告侧 MGOE（宏观图路由）**：任务级全局路由权重通过 Gumbel-Softmax 可微分学习：
$$
w_{t,m}^{(l)} = \frac{\exp((\alpha_{t,m}^{(l)} + g_m) / \tau)}{\sum_{m'} \exp((\alpha_{t,m'}^{(l)} + g_{m'}) / \tau)}
$$

两者本质相同：**减少路由搜索空间，提升任务特定知识的纯度**。SMES 是样本级动态稀疏，MGOE 是任务级静态稀疏。

### 4. 生成式竞价的概率框架

GRAD 用 Diffusion 建模出价分布（vs. GRPO 生成式出价遮蔽）：

$$
q(b_t \mid b_{t-1}) = \mathcal{N}(b_t; \sqrt{1-\beta_t} b_{t-1}, \beta_t \mathbf{I})
$$

逆向去噪生成最优出价（带预算 guidance）：
$$
p_\theta(b_{t-1} \mid b_t, c) \propto p_\theta(b_{t-1} \mid b_t) \cdot p(c \mid b_{t-1})
$$

这与 PROMISE 在推荐侧的过程奖励思路殊途同归：**都在用生成+评估的两阶段范式替代直接预测**。

### 5. PROMISE 的测试时扩展（Test-Time Scaling）

$$
\hat{y} = \arg\max_{y^{(i)}, i \in [N]} \sum_{t=1}^{T} \text{PRM}(y_{<t}^{(i)}, y_t^{(i)}, c)
$$

**在广告侧的对应**：GRAD 的 Diffusion 多步采样 + 奖励引导正是同样的"生成多个候选，选最优"逻辑。两者共同指向：**推理时计算（Inference-time Compute）是下一个性能提升维度**。

## 🎯 核心洞察

1. **范式迁移的根本驱动力是信息无损性**。传统 pipeline（召回→精排→重排）在每个阶段都丢失信息，生成式模型通过统一建模实现全局最优。这在推荐（HSTU 统一召回+精排）和广告（GPR/EGA 统一全链路）中是完全一致的逻辑。

2. **Semantic ID 是桥梁技术**。它解决了一个核心矛盾：如何让离散的物品/广告 ID 能被自回归模型"生成"？RQ-VAE 编码的层次化语义结构不仅支持生成，还内嵌了粗到细的语义层次，使 beam search 更高效。

3. **专家稀疏化是规模扩展的必要条件**。无论 SMES（推荐，20+ 任务）还是 MGOE（广告，亿级参数），稀疏化解决的是同一个问题：任务数量 × 参数量的联合爆炸。关键洞察：**宏观路由（任务级）比微观路由（样本级）更稳定**。

4. **OneTrans 和 UniROM 证明特征交叉与序列建模可以统一**。前者（推荐）把所有特征 token 和行为 token 放入同一 Transformer；后者（广告）把 CTR/CVR/LTV 多目标预测放入同一模型。这是架构层面的"大一统"趋势。

5. **LLM 增强的落地路径已经清晰**。推荐侧（PROMISE/Reasoning）偏向在线推理增强；广告侧（ELEC/IDProxy/Foundation Model）偏向离线特征工厂模式。原因是广告系统对延迟要求更严苛（<5ms），必须把 LLM 的计算代价离线化。

6. **生成式竞价是广告侧独有的创新方向**。推荐系统没有"出价"概念，广告竞价（GRAD/生成式 Bid Shading/Causal Bidding）是广告系统特有的问题。但其技术路线（生成+RL/因果推断）可以反哺推荐侧的强化学习探索。

7. **Agentic 系统是共同终点**。Agentic RecSys 的四能力（感知/规划/行动/反思）和 LLM-based 广告投放优化共享同一基础设施：LLM 推理引擎 + 工具调用 + 强化学习对齐。未来的推荐和广告系统本质上是同一个"智能体"在不同约束下的优化。

8. **工程落地的核心矛盾是延迟 vs 质量**。生成式模型（自回归）天然比判别式模型（单次前向传播）慢。GR4AD 的毫秒级优化经验（动态 beam width + KV cache + 投机解码）是整个行业的参考，推荐侧同样需要。

## 🎓 面试考点（30个跨域Q&A）

### 基础概念

**Q1: 为什么说生成式范式优于判别式范式？**
A：判别式模型（如 DIN/DeepFM）对每个物品独立打分，无法建模物品间的联合分布和序列依赖。生成式模型（如 HSTU）通过自回归建模用户行为序列，天然捕捉"看了A再看B"的共现模式。更重要的是，生成式模型可以统一多阶段 pipeline（召回+排序），消除阶段间信息损失，实现全局最优。代价是推理延迟更高（需要多步解码）。

**Q2: Semantic ID 解决了什么问题？为什么不用原始 ID？**
A：原始 ID（随机整数）无语义，无法泛化到未见物品（冷启动难）；相似物品的 ID 完全不相关，模型无法利用物品相似性。Semantic ID（RQ-VAE 编码）：①层次化语义（第1层=大类，后续层=细粒度）；②相似物品的 Semantic ID 共享前缀；③支持自回归生成（可以 beam search）。代价是需要先跑 RQ-VAE 预训练，且码本需要定期重建。

**Q3: HSTU 的 Pointwise Aggregated Attention 和标准 Softmax Attention 区别？**
A：标准 Attention 用 softmax 归一化权重：$O = \text{softmax}(QK^\top/\sqrt{d}) \cdot V$，复杂度 $O(n^2 d)$，长序列时 softmax 分母需要全局求和。HSTU 用 silu（sigmoid linear unit）替代 softmax：$O = \text{silu}(Q) \cdot \text{silu}(K)^\top \cdot V \cdot M_\text{causal}$，避免全局归一化，GPU 上可以用矩阵乘法直接实现，长序列加速3-5倍。代价是无法归一化注意力权重（不是概率分布），表达能力略低。

**Q4: 多任务学习中，SMES（推荐）和 MGOE（广告）的核心区别？**
A：SMES 是**样本级稀疏**：每个样本独立选 Top-K 专家，不同样本的路由路径不同。MGOE 是**任务级稀疏**：每个任务有固定的专家路径（通过 Gumbel-Softmax 学习），同一任务的所有样本走相同路径。MGOE 更适合任务数量固定、规模极大（亿级）的场景，路由稳定性更好；SMES 更灵活，适合样本差异大的场景。

**Q5: 广告系统为什么不直接用推荐系统的方法（如 HSTU）？**
A：三个核心差异：①**拍卖机制**：广告必须参与实时竞价（RTB），推荐无需出价，因此广告需要竞价优化层；②**多目标冲突**：广告同时优化用户体验（CTR）、广告主 ROI（CVR/GMV）、平台收入（eCPM），目标冲突更严重；③**延迟要求**：广告投放 SLA 通常 <5ms（比推荐 <10ms 更严格），自回归解码需要大量工程优化（GR4AD 的毫秒级优化）。

### 架构设计

**Q6: GPR（华为）和 EGA-v2 同样是"生成式广告全链路"，有什么区别？**
A：GPR 用**层次混合解码器（HHD）**：Coarse Decoder 先生成粗粒度候选集，Fine Decoder 精细排序，Interaction Decoder 建模交互信号。是多个解码器的流水线式组合。EGA-v2 用**Semantic ID 自回归**：将广告 embedding RQ 量化为 token 序列，单一自回归模型直接生成广告 token 序列。GPR 更接近"层次判别式"，EGA-v2 更纯粹的"生成式"。GPR 工程落地更稳健，EGA-v2 理论更优雅。

**Q7: OneTrans（推荐侧）把特征交叉和序列建模统一的核心机制？**
A：**异构注意力掩码（Heterogeneous Attention Mask）**。把所有输入（静态特征 tokens + 行为序列 tokens）放入同一 Transformer，通过不同的掩码模式控制注意力：Feature-Feature（全连接，实现特征交叉）、Feature-Action（全连接，target-aware 序列聚合）、Action-Action（因果掩码，序列建模）、Action-Feature（全连接，上下文感知序列）。四种模式在一个 Transformer 前向传播中并行完成。

**Q8: UniROM（广告统一排序）和 OneTrans（推荐统一特征）的异同？**
A：都是"统一"，但统一的维度不同。OneTrans 统一的是**输入处理**（特征+序列共用一个 Transformer）；UniROM 统一的是**输出目标**（CTR+CVR+LTV+eCPM 用一个模型输出多个头）。前者关注架构统一，后者关注目标统一。理想的下一代系统应该既有 OneTrans 的输入统一，也有 UniROM 的输出统一。

**Q9: ELEC 的 LLM 离线特征工厂为什么是广告侧的主流路线，而非推荐侧？**
A：广告对延迟极其敏感（SLA <5ms），任何在线 LLM 推理（>100ms）都不可接受。因此广告侧选择"LLM 离线生成特征 → 在线轻量 CTR 模型消费"的模式，LLM 成本离线化。推荐侧（PROMISE）可以接受在推荐候选集上多次重排（<50ms 级别），因此可以在测试时引入更多计算。本质上是两个领域的延迟约束不同导致了技术路线分叉。

**Q10: 如何为一个全新广告场景快速启动（冷启动）？**
A：三层解决方案：①**内容特征替代 ID 特征**（IDProxy：用多模态 LLM 生成广告代理 embedding，对齐到 warm ID 空间）；②**基础模型迁移**（Foundation Model Ads CTR：大规模预训练后用 Adapter 快速 finetune，减少 70% 标注需求）；③**Bandit 标注优化**（ICL-Bandit：用多臂老虎机选最优 few-shot 示例，提升数据效率）。三者层次递进，可组合使用。

### 竞价与优化

**Q11: 什么是 Bid Shading？生成式 Bid Shading 比传统方法好在哪？**
A：Bid Shading 是首价拍卖（FPA）中广告主降低出价以避免过度支付的策略（出价 × shading factor → 最终出价）。传统两阶段方法：先预测市场价格分布，再优化折扣因子，存在级联误差。生成式 Bid Shading（自回归+GRPO）：将 shading factor 离散为 token，直接从竞价上下文生成最优 token 序列，端到端避免级联误差。GRPO 的组内相对优势估计比 PPO 更稳定，不需要 critic 网络。

**Q12: GRAD 用 Diffusion 做出价 vs. 传统 RL 出价有什么优势？**
A：RL 出价（如 RL-bid）：在线探索成本高，每个广告主独立训练；Diffusion 出价（GRAD）：离线学习市场分布，预训练共享跨广告主知识，新广告主通过 prompt conditioning 快速适配（无需重训练）。关键优势：①Diffusion 建模出价的多峰分布（同一情况可能有多个合理出价）；②预算约束通过 guided diffusion 自然嵌入（$p_\theta(b_{t-1}|b_t) \cdot p(c|b_{t-1})$）；③单模型服务1000+广告主，维护成本降低99%。

**Q13: 因果竞价（Causal Bidding）为什么比普通回归出价更好？**
A：普通回归出价在历史数据上训练，存在**选择偏差**——只观测到"用当时策略出价 $b$"的结果，无法知道"如果出价 $b'$ 会怎样"。这导致模型学到的是当前策略的局部最优而非全局最优。因果竞价引入反事实推理（IPW + Double-Robust 估计），校正选择偏差，估计"如果出价不同会怎样"。代价是需要准确的倾向分数估计（在大规模竞价中是难点）。

### 工程落地

**Q14: 生成式推荐/广告上线的关键工程挑战有哪些？**
A：
① **延迟**：自回归解码串行性（GR4AD 用动态 beam + KV cache 解决）
② **物品更新**：Semantic ID 码本随物品库更新需要重建（EGA-v2 用 PQ 变体支持增量更新）
③ **训练稳定性**：生成式模型比判别式更难训练（GPR 用两阶段：预训练→微调；GRAD 用 SFT→GRPO）
④ **评估指标**：生成式推荐的 offline 评估需要新指标（不能只用 AUC，需要覆盖率、多样性等）
⑤ **降级策略**：线上超时时自动降级为判别式模型（始终保留传统 pipeline 作为 fallback）

**Q15: 如何设计一个同时服务推荐和广告的统一基础模型？**
A：美团 MTFM 给出了参考：**共享 Transformer 底座 + 轻量 Adapter 场景分流**。底座学习通用的用户/物品表征，Adapter 适配推荐 vs 广告的不同目标（推荐优化相关性，广告优化 eCPM）。关键设计：①数据混合比例（按有效样本量 $\sqrt{N}$ 比例混合，避免大场景淹没小场景）；②梯度冲突处理（PCGrad 降低推荐和广告目标的梯度干扰）；③底座推理缓存（同一用户的底座输出跨推荐和广告复用，节约计算）。

**Q16: PROMISE 的 test-time scaling 能用在广告系统吗？**
A：理论上可以，实践上受延迟约束。PROMISE 的核心：生成 N 个候选 → PRM 评分 → 选最优。在广告精排阶段（从100→10个广告），如果延迟预算允许（<10ms），可以：①对每个广告生成多个出价/展示策略；②用 PRM-like 模型评估每个策略的长期价值；③选择最优策略上线。但广告的 SLA 比推荐更严，需要 GR4AD 级别的优化才能使 test-time scaling 在广告侧可行。

**Q17: BiGEL（图增强多行为）解决了什么问题，与纯序列模型有何区别？**
A：纯序列模型（HSTU/SASRec）把用户行为看作时序序列，忽略了**多行为类型之间的关系**（点击/购买/加购/分享）。BiGEL 用图结构建模多行为：每个行为节点连到物品节点，不同行为类型用不同边类型表示。图神经网络在图上传播信息，使点击行为能借鉴购买行为的信号（即使点击和购买的时间不相邻）。关键优势是捕捉非时序的跨行为关联；代价是图构建和 GNN 推理的工程复杂度更高。

**Q18: 广告系统引入 Agentic 范式的最大障碍是什么？**
A：**延迟**是最大障碍。LLM Agent 的规划/推理延迟通常在 100ms-1s，而广告投放 SLA 要求 <5ms，差距超过20倍。可行的渐进路线：①先用 Agent 离线制定策略（每小时/每天优化出价策略、创意选择策略）；②将离线策略编译为轻量查找表或小模型在线执行；③逐步将更多决策从离线迁移到在线（随着推理硬件进步）。最终形态：LLM Agent 作为"策略大脑"，传统模型作为"执行躯体"。

### 面试高频题

**Q19: 请描述一个工业级生成式广告系统的完整架构。**
A：参考 EGA-v2 + GR4AD + GRAD + UniROM 的组合：
- **离线层**：①Foundation Model 预训练（用户/广告通用表征）；②LLM 特征工厂（ELEC 风格，离线生成语义特征）；③Semantic ID 构建（RQ-VAE 将广告编码为 token）；④GRAD 竞价策略预训练
- **近线层（分钟级）**：①增量更新用户兴趣表征；②新广告的 IDProxy 代理 embedding 生成；③竞价策略 prompt 更新
- **在线层（毫秒级）**：①GR4AD 风格：自回归生成广告候选（beam search + KV cache）；②UniROM：单模型输出 CTR/CVR/LTV；③GRAD 出价策略：diffusion 采样最优出价；④最终 eCPM 排序
- **实验层**：A/B 流量分配 + 在线指标监控 + 降级切换

**Q20: 如果让你设计下一代推荐+广告统一平台，核心技术选型是什么？**
A：
- **统一表征层**：MTFM 风格的基础模型（Transformer 底座 + 场景 Adapter），统一学习用户和物品/广告的表征
- **生成引擎**：HSTU 的 Pointwise Aggregated Attention + Semantic ID，统一召回和排序
- **特征增强**：ELEC 离线 LLM 特征工厂（每小时批量更新）+ OneTrans 在线统一特征-序列建模
- **多任务架构**：MGOE 宏观专家图（任务数 20+）+ SMES 稀疏门控（样本级细粒度）
- **竞价优化**（广告专有）：GRAD 生成式自动出价 + Causal Bidding 反事实校正
- **测试时扩展**：PROMISE PRM 评分 + 动态 beam search（GR4AD 优化）
- **长期目标**：向 Agentic 架构迁移（LLM 策略大脑 + 传统模型执行躯体）
