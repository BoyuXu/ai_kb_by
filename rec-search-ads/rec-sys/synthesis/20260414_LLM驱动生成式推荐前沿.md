# LLM驱动生成式推荐前沿综合 (2026-04-14)

> 基于4篇论文/工具综合：OneLoc, GRC, ThinkRec, LLM4Rec Survey

## 1. 技术演进脉络

### 判别式LLM4Rec → 生成式LLM4Rec
- **DLLM4Rec**: LLM作为特征提取器/评分器，嵌入传统推荐流水线
- **GLLM4Rec**: LLM直接生成推荐结果，端到端生成式范式

### 2026年前沿方向
1. **地理感知生成式推荐** (OneLoc): 融合地理信息的semantic ID + 地理感知注意力
2. **自纠正生成式推荐** (GRC): Generation-Reflection-Correction三阶段解码
3. **思考增强推荐** (ThinkRec): 从快思考到慢思考，推理链注入

## 2. 核心技术对比

| 方法 | 核心创新 | 工业效果 | 复杂度 |
|------|---------|---------|--------|
| OneLoc | 地理语义ID + 双奖励RL | GMV +21%, 订单+17.89% | 高 |
| GRC | 反思-纠正解码 + GRPO | 基线+15.74%, 收入+1.79% | 中 |
| ThinkRec | 思考激活 + 专家融合 | 准确性+可解释性提升 | 中 |

## 3. 核心公式

### OneLoc双奖励RL
```
R_total = α·R_geo(distance) + β·R_gmv(conversion)
```

### GRC三阶段解码
```
Stage 1: y_draft = Generate(x)
Stage 2: errors = Reflect(y_draft, x)  # 多粒度反思信号
Stage 3: y_final = Correct(y_draft, errors)  # GRPO优化
```

### ThinkRec思考激活
```
z_think = KeywordSummarize(item_meta) → ReasoningTrace(z_think, user)
score = ExpertFusion(z_think, user_latent)  # 实例级动态权重
```

## 4. 工业实践要点

1. **地理信息是本地生活服务的第一特征**: OneLoc证明geo-aware ID比纯语义ID效果显著
2. **自纠正的收益-延迟平衡**: GRC的EGRS机制证明可在延迟约束内获得收益
3. **LLM推荐的两大挑战**: 实时性（推理延迟）和规模性（serving成本）

## 5. 面试考点

1. **生成式推荐vs判别式推荐**: 何时用生成式？→ 长尾item、冷启动、可解释性需求
2. **语义ID设计**: 如何为不同domain设计semantic ID？
3. **RL在推荐中的应用**: GRPO/DPO在推荐场景的特殊考虑
4. **自纠正机制**: 生成式系统如何保证输出质量？
