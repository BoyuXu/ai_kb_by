# 序列推荐中的高效注意力与状态空间模型 — 20260411

> 综合论文 | 领域：rec-sys / 序列推荐效率 | 创建：20260411
> 覆盖论文：FuXi-Linear, BlossomRec, SIGMA, M2Rec
> 核心命题：**Transformer 的 O(n^2) 注意力已成为序列推荐的瓶颈，线性注意力和状态空间模型(SSM/Mamba)是两条主流替代路径**

---

## 一、技术演进脉络

```
序列推荐基础: SASRec (Self-Attention) → BERT4Rec (Bidirectional)
  → 问题: O(n^2) 注意力限制长序列（>500 tokens）

路径1: 稀疏注意力
  → BlossomRec: 块级长短期稀疏注意力 + 门控融合

路径2: 线性注意力
  → LinRec → FuXi-Linear: 时序保持通道 + 线性位置通道, O(n) 复杂度

路径3: 状态空间模型 (Mamba)
  → MambaRec (单向) → SIGMA: 双向 PF-Mamba + DS Gate + FE-GRU
  → M2Rec: 多尺度 Mamba + FFT频域 + LLM语义 + 自适应门控
```

## 二、核心技术对比

| 方法 | 架构类型 | 复杂度 | 核心创新 | 关键指标 |
|------|---------|--------|----------|---------|
| FuXi-Linear | 线性注意力 | O(n) | Temporal Retention Channel + Linear Positional Channel | prefill 10x加速, decode 21x加速, power-law scaling |
| BlossomRec | 稀疏注意力 | O(n*k), k<<n | 长期+短期双稀疏pattern + 学习门控融合 | 内存显著减少, 性能持平或更优 |
| SIGMA | Mamba (SSM) | O(n) | PF-Mamba双向 + DS Gate + FE-GRU短期 | 5个数据集SOTA |
| M2Rec | Mamba + FFT + LLM | O(n) | 多尺度: Mamba(时序) + FFT(频域) + LLM(语义) + 自适应门控 | HR@10 +3.2%, 推理比Transformer快20% |

## 三、核心公式

### FuXi-Linear: Temporal Retention Channel
$$\text{TRC}(Q, K, V, \Delta t) = \sum_{j \leq i} \phi(Q_i) \cdot \phi(K_j)^T \cdot V_j \cdot \gamma(\Delta t_{ij})$$

其中 $\phi(\cdot)$ 是特征映射函数实现线性化，$\gamma(\Delta t_{ij})$ 是周期性时间衰减函数，独立于语义通道计算，避免时序-语义信号串扰。

### FuXi-Linear: Linear Positional Channel
$$\text{LPC}(x) = \sum_{j \leq i} k_{\text{pos}(i, j) \cdot V_j$$

$k_{\text{pos}(i, j)$ 是可学习位置核函数，在 O(n) 复杂度内编码位置关系。

### BlossomRec: 长短期稀疏注意力融合
$$\text{Output} = \sigma(g) \odot \text{Attn}_{\text{long-term}(x) + (1 - \sigma(g)) \odot \text{Attn}_{\text{short-term}(x)$$

其中 $\text{Attn}_{\text{long-term}}$ 使用全局采样稀疏 pattern，$\text{Attn}_{\text{short-term}}$ 使用局部窗口稀疏 pattern，$g$ 为可学习门控参数。

### SIGMA: Partially Flipped Mamba (PF-Mamba)
$$h_{\text{bi}} = \text{DS-Gate}(h_{\text{forward}}, h_{\text{backward}})$$
$$h_{\text{forward}} = \text{Mamba}(x_1, x_2, ..., x_n)$$
$$h_{\text{backward}} = \text{Mamba}(x_n, x_{n-1}, ..., x_1)$$

DS Gate (Dense Selective Gate) 是 input-sensitive 的，根据输入动态调整前向/后向权重。

### SIGMA: Feature Extract GRU
$$h_{\text{short}} = \text{GRU}(x_{t-w:t})$$

专门捕捉短期依赖（窗口 w 内），弥补 Mamba 状态估计不稳定导致的短期模式遗漏。

### M2Rec: 多尺度融合
$$h_{\text{final}} = \alpha \cdot h_{\text{Mamba}} + \beta \cdot h_{\text{FFT}} + \gamma \cdot h_{\text{LLM}}$$
$$[\alpha, \beta, \gamma] = \text{softmax}(\text{MLP}([h_{\text{Mamba}}; h_{\text{FFT}}; h_{\text{LLM}}]))$$

FFT 通道：$h_{\text{FFT}} = \text{IFFT}(\text{Filter}(\text{FFT}(x)))$，在频域分离周期性趋势和噪声。

## 四、工业实践经验

### 1. 线性注意力的时序-语义解耦是关键（FuXi-Linear）
- 直接将线性注意力用于序列推荐效果不佳，因为时序衰减和语义相关性耦合
- Temporal Retention Channel 独立处理时间信号，避免串扰
- 在千级 token 长度下观察到 power-law scaling，证明线性模型也有 scaling law

### 2. 稀疏注意力的长短期分离思路（BlossomRec）
- 用户兴趣天然分为长期偏好（品类、风格）和短期意图（当前会话）
- 全局稀疏捕捉长期，局部窗口捕捉短期，门控动态融合
- 适配不同长度序列：短序列短期主导，长序列长期重要

### 3. Mamba的单向性是序列推荐的核心障碍（SIGMA）
- 原始 Mamba 是单向的，但推荐中双向上下文很重要（如 BERT4Rec）
- PF-Mamba 的"部分翻转"比完全双向更高效
- 短期依赖需要专门模块（FE-GRU），Mamba 的遗忘机制对短期模式不友好

### 4. 多模态多尺度融合是 Mamba 的扩展方向（M2Rec）
- 单一 Mamba 对周期性行为建模不足，FFT 补充频域视角
- 稀疏交互数据需要 LLM embedding 补充语义信息
- 自适应门控比固定权重融合效果好 2-3%

### 5. 方法选择指南
| 场景 | 推荐方法 | 理由 |
|------|---------|------|
| 序列 < 200 | SASRec (标准 Attention) | O(n^2) 可接受，效果最好 |
| 序列 200-2000 | BlossomRec 或 FuXi-Linear | 稀疏/线性注意力平衡效率和效果 |
| 序列 > 2000 | FuXi-Linear | 线性复杂度 + power-law scaling |
| 需要双向上下文 | SIGMA | PF-Mamba 双向建模 |
| 交互稀疏 + 需要频域 | M2Rec | 多尺度融合补充信息 |

## 五、面试高频考点

**Q：为什么标准 Transformer 不适合超长序列推荐？**
A：Self-attention O(n^2) 复杂度使得 n>1000 时计算和内存不可接受。推荐场景用户行为序列动辄上千甚至上万，需要亚二次复杂度的替代方案。

**Q：线性注意力的核心原理是什么？**
A：将 softmax(QK^T)V 分解为 phi(Q)(phi(K)^T V)，利用结合律将复杂度从 O(n^2d) 降到 O(nd^2)。当 d << n 时（推荐场景通常如此），效率提升巨大。FuXi-Linear 进一步引入时序保持通道避免时序信号丢失。

**Q：Mamba 相比 Transformer 的优劣？**
A：优势：O(n) 复杂度，适合超长序列；隐式压缩历史信息，推理时 O(1) 内存。劣势：(1) 天然单向，需要 PF-Mamba 等改造实现双向；(2) 状态估计不稳定导致短期依赖捕捉差；(3) 缺乏显式的注意力权重，可解释性弱。

**Q：SIGMA 的 DS Gate 和普通门控有什么区别？**
A：普通门控权重固定或仅依赖位置，DS Gate 是 input-sensitive 的——根据当前输入动态调整前向/后向 Mamba 的融合权重。某些 position 前向信息更重要（如购买序列的因果性），某些需要后向（如浏览序列的上下文补充）。

**Q：M2Rec 为什么要引入 FFT？**
A：用户行为有周期性（如每周末购物、每天通勤听播客），纯时域模型难以显式捕捉这种周期模式。FFT 将序列转换到频域，通过可学习滤波器分离周期性趋势和随机噪声，再通过 IFFT 转回时域。这是时间序列分析的经典思路在推荐中的应用。

**Q：如何选择线性注意力 vs Mamba vs 稀疏注意力？**
A：看序列长度和任务特性：(1) 短序列（<500）用稀疏注意力即可；(2) 长序列 + 强时序性用 FuXi-Linear；(3) 需要双向建模用 SIGMA；(4) 数据稀疏+多模态用 M2Rec。工业实践中，线性注意力的确定性延迟是优势（Mamba 的 scan 操作在某些硬件上优化不充分）。

## 六、论文知识卡片

### Paper 7: FuXi-Linear — 线性注意力序列推荐
**问题定义：** 现有线性注意力应用于序列推荐面临三个问题：(1) 时序信号与语义信号耦合干扰，(2) 线性框架中位置信息不足，(3) 仅关注短序列和浅层架构。
**核心方法：** 双通道设计：Temporal Retention Channel 独立计算周期性时间注意力权重，Linear Positional Channel 通过可学习核函数编码位置信息，两通道均保持 O(n) 线性复杂度。
**关键创新：** 首次在序列推荐的线性注意力中实现时序-语义解耦，且在千级 token 尺度上观察到 power-law scaling property。
**实验亮点：** 推荐质量超越 SOTA，prefill 加速 10x，decode 加速 21x。代码已开源。

### Paper 8: BlossomRec — 块级稀疏注意力序列推荐
**问题定义：** Transformer 序列推荐中，随着交互历史增长，标准注意力的计算和内存开销呈二次增长。
**核心方法：** 将用户兴趣分为长期和短期两类，分别设计稀疏注意力 pattern：长期使用全局采样稀疏（捕捉跨时间段的偏好），短期使用局部窗口稀疏（捕捉当前会话意图）。通过可学习门控融合两路输出。
**关键创新：** 将推荐领域的长短期兴趣建模思想与稀疏注意力机制结合，在不同长度序列上保持稳定性能。
**实验亮点：** 4 个公开数据集上，集成到 SOTA Transformer 模型后，性能持平或超越，内存显著减少。

### Paper 9: SIGMA — 选择性门控Mamba序列推荐
**问题定义：** Mamba (SSM) 直接用于序列推荐面临两个核心挑战：(1) 单向性限制了上下文建模，(2) 状态估计不稳定导致短期模式捕捉差。
**核心方法：** 三组件设计：(1) Partially Flipped Mamba (PF-Mamba) 构建双向架构，(2) Dense Selective Gate (DS Gate) input-sensitive 地优化方向权重，(3) Feature Extract GRU (FE-GRU) 专门捕捉短期依赖。
**关键创新：** PF-Mamba 的"部分翻转"策略比完全双向 Mamba 更高效，DS Gate 实现动态方向权重分配。AAAI 2025 接收。
**实验亮点：** 5 个真实数据集上超越所有现有模型，代码开源。

### Paper 10: M2Rec — 多尺度Mamba序列推荐
**问题定义：** 现有 Mamba 推荐模型缺乏周期性行为捕捉、语义信息丰富和多模态融合能力。
**核心方法：** 三模块融合：(1) FFT-Enhanced Mamba 在频域显式建模周期性模式，(2) LLM text embedding 补充 item 描述语义，(3) Adaptive Gating 动态平衡时序/频域/语义三路特征。
**关键创新：** 首次将 FFT 频域分析引入 Mamba 推荐框架，通过可学习滤波器分离趋势和噪声。LLM embedding 在稀疏交互场景下提供冷启动能力。
**实验亮点：** HR@10 比现有 Mamba 模型提升 3.2%，推理比 Transformer 快 20%。
