## 📐 核心公式与原理

### 1. 矩阵分解
$$\hat{r}_{ui} = p_u^T q_i$$
- 用户和物品的隐向量内积

### 2. BPR 损失
$$L_{BPR} = -\sum_{(u,i,j)} \ln \sigma(\hat{r}_{ui} - \hat{r}_{uj})$$
- 正样本得分 > 负样本得分

### 3. 序列推荐
$$P(i_{t+1} | i_1, ..., i_t) = \text{softmax}(h_t^T E)$$
- 基于历史序列预测下一次交互

## 🎯 核心洞察（5条）

1. **特征工程是推荐系统 ROI 最高的环节**：在工业实践中，一个好的特征带来的 AUC 提升（0.5-2%）通常超过模型架构改进（0.1-0.5%）
2. **Embedding 是现代推荐的基石**：所有离散特征（用户 ID、物品 ID、类目、城市...）都通过 Embedding Table 转化为稠密向量，Embedding 占模型参数量 90%+
3. **特征交叉从手工走向自动**：FM 自动二阶交叉 → DCN 显式高阶交叉 → AutoFIS 自动搜索有效交叉 → Transformer 隐式全阶交叉
4. **实时特征是线上提升的关键**：用户最近 10 分钟的行为（实时 CTR、品类点击数、会话深度）对排序效果的提升通常大于更复杂的模型
5. **特征服务架构决定上限**：特征的存储（Redis/特征平台）、更新（流式/批量）、一致性（训练/推理特征对齐）是工程核心挑战

## 📈 技术演进脉络

```
手工特征 + LR（2010-2014）
  → Embedding + FM 二阶交叉（2014-2016）
    → Deep + Wide/Cross 深度交叉（2016-2018）
      → 序列特征 + Attention（DIN, 2018-2020）
        → 实时特征 + 流式更新（2020-2022）
          → 多模态特征（CLIP/BERT 编码内容, 2022-2024）
            → LLM 语义特征（2024-2026）
```

**关键转折点**：
- **Embedding 化（2014）**：离散 ID 特征不再 one-hot，Embedding 让模型学习语义关系
- **实时特征流水线（2020）**：Flink/Kafka 流式计算用户实时行为特征，秒级更新
- **多模态特征（2022）**：商品图片/视频用预训练视觉模型编码，丰富了内容侧信号

## 🎓 面试考点（6条）

### Q1: 推荐系统常用的特征分类？
**30秒答案**：①用户侧：用户 ID、年龄、性别、历史行为序列、实时行为统计；②物品侧：物品 ID、类目、标签、内容特征（图文/视频）、统计特征（CTR/CVR）；③交叉特征：用户×物品类目偏好、用户×时段活跃度；④上下文特征：时间、位置、设备、网络环境。
**追问方向**：哪类特征对效果影响最大？答：用户行为序列 > 交叉统计特征 > 基础画像特征。

### Q2: Embedding 维度怎么选？
**30秒答案**：常用经验：高频特征（用户 ID）128 维，中频特征（类目）64 维，低频特征（城市）16-32 维。理论：维度 ≈ 6×(类目数)^0.25（经验公式）。
**追问方向**：Embedding 太大内存放不下怎么办？答：①混合维度（高频大维度、低频小维度）；②哈希碰撞（Hash Embedding）；③量化压缩（INT8 Embedding）。

### Q3: 特征交叉的主要方法？
**30秒答案**：①FM：自动二阶交叉 `<v_i, v_j>`；②DCN-V2：显式高阶交叉层 `x_0 ⊙ (Wx + b) + x`；③FiBiNet：SENet + Bilinear 交叉；④CAN：Co-Action Network 动态交叉。
**追问方向**：为什么 FM 只做二阶？答：高阶 FM 参数量指数增长，用 DNN 隐式学习高阶更实际。

### Q4: 实时特征怎么做？
**30秒答案**：①数据源：用户点击/购买/浏览事件流；②计算：Flink 流式聚合（最近 10 分钟 CTR、最近 1 小时品类分布）；③存储：写入 Redis/特征缓存；④推理：模型在线读取 Redis 拼接特征。
**追问方向**：实时特征的延迟是多少？答：端到端通常 1-5 秒（事件产生到特征可用），对大多数场景够用。

### Q5: 训练和推理的特征一致性怎么保证？
**30秒答案**：①统一特征定义：用 Feature Store（如 Feast）管理特征 schema；②Point-in-time Join：训练数据使用"事件发生时刻"的特征快照，而非最新特征；③线上线下一致性检查：定期校验训练特征 vs 推理特征的分布差异。
**追问方向**：不一致会导致什么后果？答：模型在训练时"开了上帝视角"（用未来信息），上线效果会大幅下降。

### Q6: 多模态特征怎么融入推荐模型？
**30秒答案**：商品图片用预训练 ViT/CLIP 编码为 768 维向量，降维（MLP 768→64）后拼接到商品 Embedding；文本用 BERT 编码标题/描述。多模态特征对冷启动物品提升最大（无行为数据时唯一信号）。
**追问方向**：多模态特征会和 ID Embedding 冲突吗？答：可能。常用做法是 gate 机制动态融合，行为数据充足时降低多模态权重。

---


### Q7: 推荐系统的实时性如何保证？
**30秒答案**：①用户特征实时更新（Flink 流处理）；②模型增量更新（FTRL/天级重训）；③索引实时更新（新物品上架）；④特征缓存+预计算降低延迟。

### Q8: 推荐系统的 position bias 怎么处理？
**30秒答案**：训练时：①加 position feature 推理时固定；②IPW 加权；③PAL 分解 P(click)=P(examine)×P(relevant)。推理时：设置固定 position 或用 PAL 只取 P(relevant)。

### Q9: 工业推荐系统和学术研究的差距？
**30秒答案**：①规模（亿级 vs 百万级）；②指标（商业指标 vs AUC/NDCG）；③延迟（<100ms vs 不关心）；④迭代（A/B 测试 vs 离线评估）；⑤工程（特征系统/模型服务 vs 单机实验）。

### Q10: 推荐系统面试中设计题怎么答？
**30秒答案**：按层回答：①明确场景和指标→②召回策略（多路）→③排序模型（DIN/多目标）→④重排（多样性）→⑤在线实验（A/B）→⑥工程架构（特征/模型/日志）。
