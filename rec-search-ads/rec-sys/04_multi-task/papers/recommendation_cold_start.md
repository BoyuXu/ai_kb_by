# 推荐系统冷启动（Cold Start）

> 来源：工程实践 | 日期：20260317

## 问题定义

冷启动指系统对**新用户**或**新物品**缺乏历史交互数据，无法利用协同过滤信号的问题。分三类：
- **用户冷启动**：新注册用户无行为历史
- **物品冷启动**：新发布内容/商品无曝光记录
- **系统冷启动**：平台初建，整体数据稀疏

## 核心方法与创新点

### 用户冷启动方案

1. **基于人口属性的规则/统计推荐**
   - 用年龄、性别、地区等初始画像做热门推荐，作为冷启动 baseline
   - 快速积累初始行为（前 3~5 次交互）后切换个性化模型

2. **显式兴趣收集（Onboarding）**
   - 注册时让用户选择感兴趣的类目/标签
   - Meta-Learning（MAML/LFT）：从少量交互快速 adapt 到新用户

3. **跨域迁移**
   - 利用用户在其他场景（搜索、App 其他模块）的行为初始化召回

### 物品冷启动方案

1. **内容特征为主**
   - 新物品无 ID embedding，用内容特征（文本、图像）生成内容向量
   - 与 Content-Based 双塔模型结合，利用语义相似度召回

2. **Warm-up Embedding**
   - 用物品的 side information（类目、作者、标签）平均同类物品的 ID embedding 作为初始化
   - EGES（Enhanced Graph Embedding with Side information）：图上传播 side info

3. **流量强制探索**
   - 新物品强制分配一定流量进行 explore（UCB/Thompson Sampling）
   - 快速积累曝光点击后进入正常排序流程

4. **DropoutNet / 输入扰动**
   - 训练时随机 dropout ID 特征，迫使模型学习内容特征的表达
   - 推理时新物品仅用内容特征，已见物品用 ID + 内容

5. **Meta-Learning 方案（用于物品）**
   - 将每个新物品视为 meta-task，用少量正样本快速初始化其 embedding

## 实验结论

- DropoutNet 在新物品前 24h 的 CTR 提升约 20~30%，相比仅用内容特征基线
- EGES 利用图传播，新物品 embedding 质量在首次曝光后 1h 内接近成熟物品
- Meta-Learning 在短视频场景新物品 Recall@10 提升约 15%

## 工程落地要点

1. **物品年龄分层**：按物品发布时长分层，<1h, 1~24h, >24h 分别用不同策略
2. **内容特征预计算**：新物品入库时立即触发特征抽取（图文 embedding），保证召回及时性
3. **流量配额**：新物品探索流量通常占总请求 5~10%，独立流量池避免影响主流推荐
4. **Embedding 热更新**：新物品有了足够曝光后，触发增量训练更新其 ID embedding
5. **A/B 测试注意**：冷启动改进通常在新物品子集上评估，全局指标涨幅会被稀释

## 常见考点

- **Q: 物品冷启动为什么不直接随机初始化 ID embedding？**
  A: 随机初始化需要大量曝光才能收敛，新物品的曝光机会本就少，随机初始化会导致模型打分不准，影响新物品的分发。用内容/类目信息初始化可以提供语义先验。

- **Q: DropoutNet 的核心思想是什么？**
  A: 通过训练时随机 dropout ID embedding，让模型学会在没有 ID 信息时也能依赖内容特征做出合理预测，使得模型在推理时对新物品（无 ID embedding）有更好的泛化能力。

- **Q: 用户冷启动与物品冷启动的主要区别？**
  A: 用户冷启动更多依赖显式收集（onboarding）和快速积累初始交互；物品冷启动更多依赖内容特征和流量探索，因为物品特征通常比用户特征更容易获取（有内容本身）。

## 模型架构详解

### 共享层
- **底座网络**：共享 Embedding 层 + 底部 DNN 提取通用特征表示
- **专家网络**：MoE（Mixture of Experts）或 PLE（Progressive Layered Extraction）分离任务特有/共享知识
- **门控机制**：Softmax 门控决定各专家对不同任务的贡献权重

### 任务特有层
- **Tower 网络**：每个任务独立的 DNN Tower + 输出头
- **任务关系建模**：显式建模任务间的依赖关系（如 CTR→CVR 的因果链）
- **损失函数**：加权多任务损失 = Σ wᵢ × Lᵢ，权重可固定或动态调整

### 训练策略
- **动态权重**：GradNorm / Uncertainty Weighting 自适应调整任务权重
- **梯度冲突处理**：PCGrad / CAGrad 处理任务间的梯度方向冲突
- **渐进训练**：先训练简单任务（CTR），再加入复杂任务（CVR/LTV）

## 面试深度追问

- **Q: 多任务学习中的负迁移（Negative Transfer）是什么？**
  A: 不同任务的梯度方向冲突，联合训练反而比单独训练效果差。例如 CTR 任务偏好点击频繁的热门物品，而多样性任务偏好长尾物品。

- **Q: MMoE 和 PLE 的区别？**
  A: MMoE 所有专家对所有任务共享，用门控选择；PLE 区分共享专家和任务特有专家，并通过渐进层次避免浅层信息污染。PLE 在任务差异大时优于 MMoE。

- **Q: 如何判断两个任务是否适合联合训练？**
  A: 1) 任务相关性分析（目标正相关/互补）；2) 数据重叠度（共享用户/物品越多越好）；3) 梯度方向分析（余弦相似度 > 0 说明不冲突）；4) A/B Test 验证。

- **Q: LLM 增强推荐的典型方案？**
  A: 1) 特征增强：LLM 提取文本/知识特征作为推荐模型输入；2) 知识蒸馏：LLM 作为 Teacher 指导推荐模型学习；3) 端到端：LLM 直接输出推荐结果。
