## 📐 核心公式与原理

### 📐 1. 双塔相似度与 Sampled Softmax 训练目标推导

**相似度计算：**

$$
s(u, v) = \frac{\mathbf{u}^\top \mathbf{v}}{\|\mathbf{u}\| \cdot \|\mathbf{v}\|} \quad \text{（余弦相似度）} \quad \text{或} \quad s(u, v) = \mathbf{u}^\top \mathbf{v} \quad \text{（内积）}
$$

其中 $\mathbf{u} = f_U(x_u; \theta_U) \in \mathbb{R}^d$，$\mathbf{v} = f_V(x_v; \theta_V) \in \mathbb{R}^d$。

**训练目标：Sampled Softmax（全量 Item 集合上的 softmax 近似）**

全量 softmax 分母需遍历所有 $|\mathcal{V}|$（数百万）个物品，不可行。Sampled Softmax 对每个正样本只采 $K$ 个负样本：

$$
\mathcal{L}_{\text{SS}} = -\frac{1}{|\mathcal{B}|}\sum_{(u,v^+)\in\mathcal{B}} \log \frac{e^{s(u, v^+)/\tau}}{e^{s(u, v^+)/\tau} + \sum_{j=1}^{K} e^{s(u, v^-_j)/\tau}}
$$

**推导步骤：**

1. **正确分布下的全量 Softmax**：理想目标是最大化 $P(v^+ | u) = \frac{e^{s(u,v^+)/\tau}}{\sum_{v' \in \mathcal{V}} e^{s(u,v')/\tau}}$，NLL 为负对数似然

2. **采样近似**：按物品流行度分布 $q(v)$ 采负样本，引入对数修正（Correction for sampling bias）：

$$
\mathcal{L}_{\text{corrected}} = -\log \frac{e^{s(u, v^+) - \log q(v^+)}}{e^{s(u, v^+) - \log q(v^+)} + \sum_{j=1}^{K} e^{s(u, v^-_j) - \log q(v^-_j)}}
$$

减去 $\log q(v)$ 是因为频繁出现的物品（大 $q(v)$）会被过多采样为负样本，需要降低其惩罚力度；反之稀有正样本（小 $q(v^+)$）需要增大奖励。

3. **In-batch Negative**：用同一 batch 内其他用户的正样本作为负样本，$K = B - 1$（batch size - 1）。此时需要加 Sampling Bias Correction：$s_{\text{corrected}}(u, v_j^-) = s(u, v_j^-) - \log(p_j)$，$p_j$ 为第 $j$ 个物品在 batch 中出现的频率估计。

**符号说明：**

| 符号 | 含义 |
|------|------|
| $\mathbf{u} \in \mathbb{R}^d$ | 用户塔输出的用户向量 |
| $\mathbf{v}^+ \in \mathbb{R}^d$ | 正样本（用户交互过的）物品向量 |
| $\mathbf{v}^-_j \in \mathbb{R}^d$ | 第 $j$ 个负样本物品向量 |
| $\tau$ | 温度系数（通常 0.05–0.2），控制分布尖锐程度 |
| $K$ | 每个正样本对应的负样本数量 |
| $q(v)$ | 负采样分布（通常为流行度的 3/4 次方）|
| $\log q(v)$ | Sampling Bias Correction 项 |

**直观理解：** Sampled Softmax 相当于把"在全校选最好朋友"变成"在 K 个随机同学里选最好朋友"——只要 K 够大且负样本分布合理，近似效果很好。温度 $\tau$ 越小，分布越尖锐，模型被迫更精准区分正负样本。

---

### 📐 2. 硬负样本挖掘（Hard Negative Mining）

在 In-batch 负样本基础上，引入 Hard Negatives：

$$
\mathcal{L}_{\text{HNM}} = -\log \frac{e^{s(u, v^+)/\tau}}{e^{s(u, v^+)/\tau} + \underbrace{\sum_{j \in \mathcal{B}\setminus\{+\}} e^{s(u, v_j)/\tau}}_{\text{in-batch easy negatives}} + \underbrace{\sum_{k=1}^{K_h} e^{s(u, v^{h-}_k)/\tau}}_{\text{hard negatives}}}
$$

**Hard Negative 的选取**：从 ANN 检索 Top-M（如 M=500），排除已交互的，从 Top-500 中随机采 $K_h$ 个。这些是"模型认为相关但实际上没有互动"的高难度负样本，迫使模型学到更精细的边界。

**符号说明：**
- $v^{h-}_k$：第 $k$ 个 Hard Negative 物品
- $K_h$：Hard Negative 数量（通常 5–10，不宜太多否则训练不稳定）
- Top-M：模型当前认为与用户最相似的 M 个物品集合

### 3. 推荐系统漏斗（工程约束）

$$
\text{全量 } |\mathcal{V}| \xrightarrow{\text{ANN 召回}} 10^3 \xrightarrow{\text{粗排}} 10^2 \xrightarrow{\text{精排}} 10^1 \xrightarrow{\text{重排}} \text{展示}
$$

ANN 召回延迟目标：$t_{\text{recall}} < 10\text{ms}$（HNSW 索引，$|\mathcal{V}|=10^8$ 规模）

## 🌟 一句话解释

双塔模型把用户和物品分别编码成向量，**离线建好物品索引，在线只算用户向量然后做近似最近邻检索**，实现亿级物料毫秒级召回。

## ⚙️ 核心机制

```
【离线阶段】
Item Features → Item Tower(DNN) → Item Vector v ∈ ℝ^d
                                        │
                               存入 Faiss/Milvus 向量库

【在线阶段】
User Features → User Tower(DNN) → User Vector u ∈ ℝ^d
                                        │
                               ANN 检索（HNSW/IVF-PQ）
                                        │
                               Top-K 候选物品

【相似度】score = u · v （内积）或 cos(u,v)
```

**训练目标：Sampled Softmax + Hard Negative Mining**
```
L = -log [ exp(u·v⁺/τ) / (exp(u·v⁺/τ) + Σⱼ exp(u·vⱼ⁻/τ)) ]

vⱼ⁻ = in-batch random negatives + hard negatives（被召回但未点击）
```

## 🏭 工业落地

- **YouTube DNN (2016)**：最早将 DNN 用于视频召回，均值池化聚合历史行为
- **Facebook EBR**：in-batch + hard negatives，大规模部署经验
- **微信看一看**：多路召回融合，双塔 + 协同过滤 + 实时序列
- **快手 U2I2I**：用户→物品→物品两跳扩展，解决用户偏好探索

**工程关键点：**
1. **负样本偏差修正**：热门物品在 in-batch negative 中被采样过多，需减去 log p(v)
2. **用户向量实时性**：流式特征平台（Flink）保持行为序列 <1 分钟延迟
3. **物品向量更新**：通常每小时/每天重建索引，需版本对齐
4. **召回路数**：3~5 路并行（双塔 + CF + 内容 + 图），取并集送精排

## 🎯 常见考点

**Q1（基础）：为什么双塔不能引入用户-物品交叉特征？**
> 因为需要离线预算所有物品向量建索引，引入交叉特征后物品向量依赖用户，无法提前计算，在线需要对亿级物料做用户-物品联合推理，延迟不可接受。

**Q2（中等）：In-batch negative 存在什么问题？怎么解决？**
> 两个问题：①热门物品采样概率高，引入曝光偏差；②批次内可能存在假负样本（用户其实喜欢但没点击）。解决：①sampling bias correction：score -= log p(v)；②hard negative mining 混合难负例；③false negative filtering 过滤已交互物品。

**Q3（高难）：双塔模型如何处理用户多兴趣问题？**
> 单向量用户表示无法覆盖多样化兴趣。MIND 用 Capsule Network 生成多个用户兴趣向量，每个向量代表一个兴趣簇；检索时对每个兴趣向量分别召回 Top-K，再合并。代价是索引查询次数 × 兴趣数，可以控制在 3~5 个向量。

### Q2: 算法面试如何展现系统性思维？
**30秒答案**：①先说全局架构再说细节；②主动分析 trade-off；③提及工程约束（延迟/资源）；④讨论 A/B 测试验证；⑤对比多种方案优劣。

### Q3: 面试中遇到不会的问题怎么办？
**30秒答案**：①诚实说不了解具体细节；②从已知相关知识推导思路；③说明学习路径（"我会从 XX 论文入手了解"）。比胡编强 100 倍。

### Q4: 简历中项目经历怎么写？
**30秒答案**：①每个项目 3-5 行；②突出方法创新点和业务效果；③用数字量化（AUC/CTR/时长提升 X%）；④技术关键词匹配 JD；⑤按相关度排序而非时间顺序。

### Q5: 如何准备系统设计面试？
**30秒答案**：①准备推荐/搜索/广告各一个完整系统设计；②每个系统能说清召回→排序→重排全链路；③准备 scalability 方案（如何从百万到亿级）；④准备 failure mode 和降级方案。

### Q6: 八股文和实际项目经验如何结合？
**30秒答案**：八股文提供理论框架，项目经验证明落地能力。面试时：先用八股文回答「是什么/为什么」，再用项目经验回答「怎么做/效果如何」。纯八股文没有竞争力。

### Q7: 面试中如何展示 leadership？
**30秒答案**：①描述自己在项目中的角色和贡献；②说明如何推动跨团队协作；③展示主动发现问题并推动解决的案例；④分享技术方案选型的决策过程。

### Q8: 被问到不会的论文怎么办？
**30秒答案**：①说清楚自己了解的相关工作；②从论文标题推断可能的方法（如 xxx for recommendation 可能是把 xxx 技术迁移到推荐）；③承认不了解但表达学习意愿。

### Q9: 算法岗面试的常见流程？
**30秒答案**：①简历筛选→②一面（算法基础+项目）→③二面（系统设计+深度追问）→④三面（部门 leader，考察思维+潜力）→⑤HR 面→Offer。每轮约 45-60 分钟。
