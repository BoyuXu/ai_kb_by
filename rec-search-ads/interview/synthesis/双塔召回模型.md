## 📐 核心公式与原理

### 1. 推荐系统漏斗

$$
\text{全量} \xrightarrow{召回} 10^3 \xrightarrow{粗排} 10^2 \xrightarrow{精排} 10^1 \xrightarrow{重排} \text{展示}
$$

- 逐层过滤，平衡效果和效率

### 2. CTR 预估

$$
pCTR = \sigma(f_{DNN}(x_{user}, x_{item}, x_{context}))
$$

- 排序核心：预估用户点击概率

### 3. 在线评估

$$
\Delta metric = \bar{X}_{treatment} - \bar{X}_{control}
$$

- A/B 测试量化策略效果

## 🌟 一句话解释

双塔模型把用户和物品分别编码成向量，**离线建好物品索引，在线只算用户向量然后做近似最近邻检索**，实现亿级物料毫秒级召回。

---

## 🎭 生活类比

图书馆选书：
- **传统精排**：每次给你一本书都让你和作者面对面聊再决定要不要（计算成本极高）
- **双塔召回**：把所有书的"特征标签"提前打好印在书脊上（Item Tower 离线），你进门时把你的口味告诉图书管理员（User Tower 实时），管理员用你的口味向量在书架上快速扫一遍，找出最匹配的100本候选

## ⚙️ 核心机制

```
【离线阶段】
Item Features → Item Tower(DNN) → Item Vector v ∈ ℝ^d
                                        │
                               存入 Faiss/Milvus 向量库

【在线阶段】
User Features → User Tower(DNN) → User Vector u ∈ ℝ^d
                                        │
                               ANN 检索（HNSW/IVF-PQ）
                                        │
                               Top-K 候选物品

【相似度】score = u · v （内积）或 cos(u,v)
```

**训练目标：Sampled Softmax + Hard Negative Mining**
```
L = -log [ exp(u·v⁺/τ) / (exp(u·v⁺/τ) + Σⱼ exp(u·vⱼ⁻/τ)) ]

vⱼ⁻ = in-batch random negatives + hard negatives（被召回但未点击）
```

## 🏭 工业落地

- **YouTube DNN (2016)**：最早将 DNN 用于视频召回，均值池化聚合历史行为
- **Facebook EBR**：in-batch + hard negatives，大规模部署经验
- **微信看一看**：多路召回融合，双塔 + 协同过滤 + 实时序列
- **快手 U2I2I**：用户→物品→物品两跳扩展，解决用户偏好探索

**工程关键点：**
1. **负样本偏差修正**：热门物品在 in-batch negative 中被采样过多，需减去 log p(v)
2. **用户向量实时性**：流式特征平台（Flink）保持行为序列 <1 分钟延迟
3. **物品向量更新**：通常每小时/每天重建索引，需版本对齐
4. **召回路数**：3~5 路并行（双塔 + CF + 内容 + 图），取并集送精排

## 🎯 面试考点

**Q1（基础）：为什么双塔不能引入用户-物品交叉特征？**
> 因为需要离线预算所有物品向量建索引，引入交叉特征后物品向量依赖用户，无法提前计算，在线需要对亿级物料做用户-物品联合推理，延迟不可接受。

**Q2（中等）：In-batch negative 存在什么问题？怎么解决？**
> 两个问题：①热门物品采样概率高，引入曝光偏差；②批次内可能存在假负样本（用户其实喜欢但没点击）。解决：①sampling bias correction：score -= log p(v)；②hard negative mining 混合难负例；③false negative filtering 过滤已交互物品。

**Q3（高难）：双塔模型如何处理用户多兴趣问题？**
> 单向量用户表示无法覆盖多样化兴趣。MIND 用 Capsule Network 生成多个用户兴趣向量，每个向量代表一个兴趣簇；检索时对每个兴趣向量分别召回 Top-K，再合并。代价是索引查询次数 × 兴趣数，可以控制在 3~5 个向量。

**30秒答案**：①诚实说不了解具体细节；②从已知相关知识推导思路；③说明学习路径（"我会从 XX 论文入手了解"）。比胡编强 100 倍。

### Q4: 简历中项目经历怎么写？
**30秒答案**：①每个项目 3-5 行；②突出方法创新点和业务效果；③用数字量化（AUC/CTR/时长提升 X%）；④技术关键词匹配 JD；⑤按相关度排序而非时间顺序。

### Q5: 如何准备系统设计面试？
**30秒答案**：①准备推荐/搜索/广告各一个完整系统设计；②每个系统能说清召回→排序→重排全链路；③准备 scalability 方案（如何从百万到亿级）；④准备 failure mode 和降级方案。

### Q6: 八股文和实际项目经验如何结合？
**30秒答案**：八股文提供理论框架，项目经验证明落地能力。面试时：先用八股文回答「是什么/为什么」，再用项目经验回答「怎么做/效果如何」。纯八股文没有竞争力。

### Q7: 面试中如何展示 leadership？
**30秒答案**：①描述自己在项目中的角色和贡献；②说明如何推动跨团队协作；③展示主动发现问题并推动解决的案例；④分享技术方案选型的决策过程。

### Q9: 算法岗面试的常见流程？
**30秒答案**：①简历筛选→②一面（算法基础+项目）→③二面（系统设计+深度追问）→④三面（部门 leader，考察思维+潜力）→⑤HR 面→Offer。每轮约 45-60 分钟。
