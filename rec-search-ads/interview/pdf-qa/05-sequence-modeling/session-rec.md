# Session 推荐 / 长短期兴趣分离 / 多模态序列建模

## 1. Session-based 推荐

### 1.1 什么是 Session 推荐

Session 推荐：基于用户当前会话内的行为序列预测下一个交互物品，通常不依赖用户长期历史。

应用场景：
- 匿名用户（未登录、无 user_id）
- 新用户冷启动（注册初期无历史）
- 强 session 性场景（搜索、旅游规划、即时购物）

### 1.2 GRU4Rec

最早将 RNN 应用于 session 推荐的模型。

```
输入: session 内点击序列 [item_1, item_2, ..., item_t]
模型: GRU 逐步编码，h_t = GRU(e_t, h_{t-1})
输出: h_t → softmax/采样 → 预测下一项

关键设计:
  - session-parallel mini-batch: 不同 session 的相同位置组成 batch
  - BPR loss: pairwise 排序损失，正样本分数高于负样本
  - TOP1 loss: 负样本分数的 sigmoid 均值 + 正则项
```

局限：
- GRU 难以捕捉长距离依赖（但 session 通常较短，影响有限）
- 只建模当前 session，忽略跨 session 信息
- 训练速度慢（顺序计算）

### 1.3 NARM (Neural Attentive Recommendation Machine)

在 GRU4Rec 基础上引入注意力：
```
GRU 编码 → 隐状态序列 [h_1, ..., h_t]
全局兴趣: h_t (最后一个隐状态)
局部兴趣: attention(h_t, [h_1, ..., h_t]) → 加权求和

最终表示 = concat(全局兴趣, 局部兴趣)
```

改进：全局兴趣捕捉主要意图，局部兴趣关注最相关的历史行为。

### 1.4 SR-GNN (Session-based Recommendation with GNN)

将 session 序列建模为有向图：
```
session: [A, B, C, B, D]
图节点: A, B, C, D
有向边: A→B, B→C, C→B, B→D

GNN 消息传递 → 节点表示更新
Readout: attention-based 聚合 → session 表示
```

优势：
- 捕捉 session 内物品间的复杂转移关系（非线性序列）
- 处理重复访问（B 出现两次，图中只有一个节点）
- GNN 消息传递可以利用高阶邻居信息

### 1.5 Session 推荐的 Transformer 方案

直接用 SASRec 风格的因果 Transformer 编码 session：
- 优势：并行计算、长距离依赖
- session 通常较短（5-30 个行为），标准 Transformer 完全可行
- 无需特殊的长序列优化

## 2. 长短期兴趣分离

### 2.1 为什么要分离？

用户兴趣有不同的时间尺度：
- 长期兴趣：稳定偏好（如喜欢科幻电影、偏好Nike品牌）
- 短期兴趣：当前意图（如正在挑选生日礼物、最近在装修）
- 瞬时兴趣：即时需求（如刚搜索了「雨伞」因为下雨了）

单一模型难以同时建模所有尺度——长期兴趣需要全量历史，短期兴趣需要近期序列，建模方式不同。

### 2.2 多粒度兴趣建模方案

```
长期兴趣:
  全量历史行为 → 预训练用户 Embedding (或统计特征)
  更新频率: 天级/周级离线更新
  模型: DeepFM/矩阵分解/大规模 Embedding 表

短期会话兴趣:
  最近一个 session（或最近 N 个行为）→ Transformer/GRU 编码
  更新频率: 实时/近实时
  模型: SASRec/DIEN

瞬时兴趣:
  最近 3-5 次行为 → 简单加权/GRU
  更新频率: 实时
  模型: 轻量级 RNN 或加权平均
```

### 2.3 融合策略

拼接融合：
```
user_repr = concat(long_term_emb, short_term_emb, instant_emb)
→ MLP → final_repr
```

门控融合：
```
gate = sigmoid(W * [long_term, short_term, instant, context])
user_repr = gate * short_term + (1-gate) * long_term
```

注意力融合：
```
候选物品作为 Query，[长期, 短期, 瞬时] 作为 Key/Value
动态决定当前候选物品应更多关注哪个尺度的兴趣
```

### 2.4 实践经验

- 短期兴趣对 CTR 影响最大（用户当前意图最直接）
- 长期兴趣对多样性和探索性推荐更重要
- 瞬时兴趣容易包含噪声（误点击），需要去噪
- 不同场景权重不同：电商重短期，音乐/视频重长期

## 3. 多模态序列建模

### 3.1 行为序列 + 文本信息融合

传统序列模型只用物品 ID embedding，忽略了物品的内容信息（标题、描述、类目文本等）。

交叉注意力双塔融合方案：
```
物品文本编码器:
  轻量级 BERT → 为每个物品生成语义嵌入 t_i

用户序列编码器:
  输入: [t_1, t_2, ..., t_n]  (文本嵌入替代 ID 嵌入)
  模型: SASRec/Transformer → 用户序列表示 u

交叉注意力融合:
  Query = u (用户序列表示)
  Key/Value = 候选物品的文本嵌入
  → 深度语义匹配
```

### 3.2 ID Embedding vs 内容 Embedding

ID Embedding：
- 协同过滤信号强（大量用户行为训练）
- 冷启动无能为力（新物品无 ID embedding）
- 需要大量训练数据

内容 Embedding（文本/图片）：
- 泛化性好（新物品有文本描述即可）
- 跨域迁移能力强
- 语义信息丰富但缺乏协同过滤信号

最佳实践：两者结合
```
item_repr = W_id * id_emb + W_content * content_emb
```

### 3.3 行为序列 + 图像信息

```
物品图像 → CNN/ViT → 视觉嵌入 v_i
物品文本 → BERT → 文本嵌入 t_i
物品 ID → Embedding → id_emb_i

多模态物品表示: item_i = fusion(id_emb_i, t_i, v_i)
用户序列: [item_1, ..., item_n] → Transformer → 用户表示
```

融合方式：
- 早期融合：先拼接多模态表示，再做序列建模
- 晚期融合：分别做序列建模，最后拼接/注意力融合
- 推荐早期融合（让 Transformer 直接在多模态空间建模交互）

### 3.4 行为类型融合

用户行为不只是点击，还包括多种类型：
```
行为类型: 浏览、点击、收藏、加购、购买、搜索
行为权重: 浏览(0.1) < 点击(0.3) < 收藏(0.5) < 加购(0.7) < 购买(1.0)
```

建模方式：
```
输入 = item_emb + behavior_type_emb + position_emb + time_emb
```

behavior_type_emb 让模型区分不同行为的意义：
- 「浏览了但没点击」= 轻度兴趣或无兴趣
- 「收藏了」= 有兴趣但暂不购买
- 「加购了」= 强购买意图

## 4. 序列数据预处理

### 4.1 数据清洗

- 过滤极短停留（< 2 秒的曝光/点击 → 可能是误触）
- 去除重复点击（短时间内连续点击同一物品 → 保留一次）
- 过滤机器人/爬虫行为（行为频率异常高）

### 4.2 Session 切分

常用规则：
- 时间间隔 > 30 分钟 → 新 session
- 跨天 → 新 session
- 行为类型切换（如从浏览切到搜索）→ 新 session（可选）

### 4.3 序列长度处理

截断：
- 固定最大长度 K，保留最近 K 个行为
- 典型值：K = 50-200

填充：
- 短序列用 PAD token 填充到固定长度
- 注意力计算时 mask 掉 PAD 位置

变长处理：
- 用 pack_padded_sequence 避免 PAD 位置的无效计算
- 或用注意力 mask 直接处理

### 4.4 负采样

训练时需要负样本（用户未交互的物品）：
- 随机采样：从全物品集随机采样（最简单）
- 流行度采样：热门物品更容易被采为负样本（更难的负样本）
- 曝光未点击：从实际曝光但未点击的物品中采样（最接近真实分布）

## 5. 噪声处理策略

### 5.1 数据层面

- 时间衰减加权：近期行为权重高，远期行为权重低
- 行为类型加权：购买 > 收藏 > 点赞 > 浏览
- 异常检测：基于行为频率/时间分布过滤异常用户

### 5.2 模型层面

- 注意力机制本身可以降权不重要的行为
- 噪声感知注意力：辅助网络预测每个行为是噪声的概率，降权噪声行为
- 对比学习去噪：对序列做数据增强（随机 drop/mask），让模型学习鲁棒表示

### 5.3 损失层面

- 课程学习：先用干净样本训练，逐步引入噪声样本
- 对抗训练：在 embedding 上注入微小扰动，提升鲁棒性
- 加权损失：高质量交互（购买）的 loss 权重更高

## 6. 面试高频问题

Q: Session 推荐和序列推荐有什么区别？
A: Session 推荐只基于当前会话内的行为，不使用用户长期历史，适合匿名用户和冷启动。序列推荐使用用户全部历史行为，需要 user_id。Session 推荐是序列推荐的子问题。

Q: 如何处理用户的多层次兴趣？
A: 分三个粒度分别建模——瞬时兴趣（最近几次点击，GRU/加权平均）、短期会话兴趣（当前 session，Transformer）、长期稳定兴趣（全部历史，预训练 Embedding）。用门控或注意力机制动态融合，让候选物品决定应激活哪个层次的兴趣。

Q: 多模态信息怎么融入序列建模？
A: 推荐早期融合方案：用文本/图像编码器将各模态映射到统一空间，拼接或加权求和得到多模态物品表示，作为序列模型的输入。这样 Transformer 可以直接在多模态空间学习行为间的交互。

Q: Item2Vec 和 Word2Vec 的关系？
A: Item2Vec 直接将 Word2Vec 的 Skip-gram 应用于用户行为序列——每个物品视为一个「词」，用户的点击序列视为「句子」。在滑动窗口内共现的物品学到相近的向量表示。核心思想完全相同：用分布式表示捕捉共现关系。

Q: 行为序列中如何处理不同类型的行为（浏览/点击/购买）？
A: (1) 加入 behavior_type_embedding 区分行为含义；(2) 不同行为赋予不同权重（购买 > 收藏 > 点击 > 浏览）；(3) 辅助任务监督不同行为（如购买行为对应更高的 loss 权重）；(4) 多通道建模，不同行为走不同序列编码器。

Q: GRU4Rec 的 session-parallel mini-batch 是什么？
A: 传统 mini-batch 是从不同样本中采样。GRU4Rec 将多个 session 对齐到相同时间步，每个 batch 包含不同 session 的第 t 步。这样 GRU 可以并行处理多个 session 的同一位置，同时保持每个 session 内的顺序性。Session 结束时替换为新 session。
