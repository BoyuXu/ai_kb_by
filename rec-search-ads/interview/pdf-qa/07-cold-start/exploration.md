# 探索与利用平衡策略（Exploration & Exploitation）

## 1. 核心问题

推荐系统面临的探索-利用困境：
- 利用（Exploitation）：推荐已知用户喜欢的内容，最大化短期收益
- 探索（Exploration）：推荐不确定的内容，收集信息以发现新的用户兴趣
- 纯利用 -> 信息茧房、用户兴趣固化、长尾物品饥饿
- 纯探索 -> 短期体验差、用户流失
- 目标：在长期累积收益最大化的框架下自动平衡

## 2. 经典探索策略对比

### 2.1 epsilon-greedy
- 机制：以 1-epsilon 概率选最优，epsilon 概率完全随机
- epsilon 设计：
  - 固定值（如 0.1）
  - 时间衰减：epsilon_t = epsilon_0 / sqrt(t)
  - 分场景调整：新用户用更大 epsilon
- 优点：实现简单，计算开销极低
- 缺点：随机探索效率低，不利用不确定性信息

### 2.2 Thompson Sampling
- 机制：为每个物品维护奖励的后验概率分布，从后验采样决策
- Bernoulli 场景：Beta(alpha, beta) 后验
  - 点击：alpha += 1
  - 未点击：beta += 1
  - 采样 theta ~ Beta(alpha, beta)，选最大 theta 的物品
- 优点：
  - 概率匹配方式自然平衡探索-利用
  - 不确定性大的物品被探索的概率更高
  - 收敛后自动减少探索
- 缺点：需要维护每个物品的后验参数

### 2.3 UCB（Upper Confidence Bound）
- 机制：选择 argmax_a [均值估计 + 置信上界]
- 公式：a* = argmax_a [Q(a) + c * sqrt(ln(t) / N(a))]
- 「乐观面对不确定性」：未充分探索的物品获得更大上界
- 优点：确定性策略，理论后悔界好
- 缺点：对非平稳环境适应性差

### 2.4 三者对比

```
策略            探索效率    计算复杂度    适用场景             是否利用不确定性
epsilon-greedy  低          O(1)        快速原型/基线        否
UCB             中          O(1)        平稳/理论要求好      是（置信区间）
Thompson        高          O(1)        非平稳/工业系统      是（后验分布）
```

## 3. 上下文探索方法

### 3.1 LinUCB
- 将物品奖励建模为用户上下文特征的线性函数
- 为每个臂维护：精度矩阵 B_a, 特征累积 f_a
- 选择：a* = argmax_a [theta_a^T * x + alpha * sqrt(x^T * B_a^{-1} * x)]
- 适合冷启动：即使物品无历史反馈，也能通过上下文特征估计
- 工业应用：雅虎新闻推荐的经典案例

### 3.2 Neural Bandit
- 用深度神经网络替代线性模型捕捉非线性关系
- 不确定性估计方法：
  - MC Dropout：训练时 Dropout，推理时多次采样估计方差
  - Deep Ensemble：训练多个独立网络，用预测分歧度量不确定性
  - Bayesian Neural Network：显式建模权重的后验分布

### 3.3 上下文 Thompson Sampling
- 贝叶斯线性回归建模 r_a ~ N(x^T * theta_a, sigma^2)
- 后验更新：theta_a ~ N(mu_a, v^2 * B_a^{-1})
- 采样参数 -> 计算期望奖励 -> 选最大

## 4. 深度学习驱动的探索

### 4.1 深度贝叶斯集成
- 训练 K 个独立初始化的神经网络
- 预测方差 = 各网络预测的方差（认知不确定性的近似）
- 高方差物品更值得探索
- 工程友好：比 BNN 实现简单，可并行推理

### 4.2 Bootstrapped DQN
- 维护 K 个 Q 网络（共享底层特征提取，独立头部）
- 训练时每个网络用 bootstrapped 子集训练
- 推理时随机选一个头部决策
- 不同头部的预测差异自然驱动探索

### 4.3 元学习探索（Meta-Learning for Exploration）
- 思想：学习一个通用的探索策略，能快速适应新用户/新场景
- MAML 框架：
  - 外循环学习好的初始化参数
  - 内循环在少量交互后快速适应
- 适合冷启动探索：几步交互就能切换到合适的探索策略

## 5. 冷启动场景的探索策略

### 5.1 新用户冷启动探索
- 问题：新用户无历史数据，无法准确估计兴趣
- 策略组合：
  - 利用注册信息和上下文特征（年龄、性别、设备）做初始推荐
  - LinUCB/Thompson Sampling 主动探索兴趣
  - 设置探索队列：在推荐结果中混入多样化新物品
  - 跨域迁移：利用其他业务场景的行为预测兴趣
  - 热门/趋势性召回作为保底策略

### 5.2 新物品冷启动探索
- 问题：新物品无交互数据，传统协同过滤失效
- 策略组合：
  - 基于内容的召回：利用物品元数据（品类、标签、文本描述、图像特征）
  - 知识图谱：通过实体关系推理相似物品
  - 种子用户扩散：先推给活跃/广泛兴趣用户收集初始反馈
  - MAB 动态分配新物品曝光量
  - 物品属性图嵌入（Node2Vec/GNN）获得初始表示

### 5.3 冷启动探索评估
- 短期指标：新用户/新物品的点击率、转化率
- 长期指标：新用户留存率、新物品生命周期曝光量
- A/B 测试：对比不同冷启动策略对长期生态的影响
- 需要专门的冷启动评估集，不能只看全局指标

## 6. 动态环境适应

### 6.1 衰减因子方法
- 问题：用户兴趣和物品流行度随时间变化，静态策略失效
- 方法：在参数更新中引入衰减因子 gamma（0到1之间）
  - 新Alpha = gamma * 旧Alpha + 当前反馈
  - 新Beta = gamma * 旧Beta + (1 - 当前反馈)
- gamma = 1 时退化为标准 Thompson Sampling（记住所有历史）
- gamma < 1 时历史信息指数衰减，更快适应变化

### 6.2 滑动窗口方法
- 只用最近 W 次交互的数据更新模型
- 滑动窗口 UCB：在 W 大小窗口内计算均值和置信上界
- 窗口大小 W 权衡稳定性（大窗口）和适应性（小窗口）
- 实践：可用元学习或超参优化技术自动调 W

### 6.3 变化点检测
- 监控数据分布变化，检测到剧变时临时重置参数或提高探索率
- 适合应对突发新闻/热门事件导致的兴趣突变

## 7. 在线学习与实时反馈

### 7.1 在线学习框架
- 用户每次交互后实时更新模型/统计量
- 适合 Bandit 类算法（参数更新轻量）
- 深度模型：增量训练或 mini-batch 更新

### 7.2 实时反馈整合
- Kafka 收集行为日志 -> Flink 流处理 -> 实时特征/统计更新
- 挑战：反馈延迟（购买行为可能滞后点击数小时）
- 解决：分层反馈——即时反馈（点击）快速更新，延迟反馈（转化）定期修正

## 8. 探索策略评估指标

### 8.1 短期指标
- 探索物品的点击率/转化率
- 探索流量的即时体验损失

### 8.2 长期指标
- 用户留存率提升
- 推荐结果多样性（品类覆盖率）
- 惊喜度/新颖性指标
- 长尾物品渗透率

### 8.3 生态指标
- 内容基尼系数（曝光公平性）
- 新物品从上架到被充分探索的时间
- 创作者留存率

## 9. 工程挑战

### 9.1 延迟与性能
- 探索计算不能增加推荐链路的延迟
- Thompson Sampling 采样开销极低，工业友好
- 集成方法需要多个模型推理，需优化批处理

### 9.2 可扩展性
- 百万级物品不能为每个物品单独维护 Bandit 参数
- 解决：物品分组/聚类 Bandit，或上下文 Bandit 共享参数
- 分层探索：先探索类别，再探索类别内物品

### 9.3 A/B 测试集成
- 探索策略需要通过 A/B 测试验证对长期指标的影响
- 实验周期需足够长（至少 4 周）以捕捉长期效应
- 关注不同用户群体的差异化影响

### 9.4 安全机制
- 探索不能推荐违规/低质内容
- 设置探索比例上限（如总流量的 5-10%）
- 探索物品需通过基本质量筛选
- 监控探索带来的用户体验降级告警
