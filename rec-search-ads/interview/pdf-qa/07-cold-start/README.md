# Ch10 冷启动问题与解决方案

## 1. 问题定义与分类

冷启动 = 新用户或新物品缺乏交互数据，无法有效建模

用户冷启动：新注册用户，无历史行为数据
物品冷启动：新上架物品，无用户交互数据
系统冷启动：整个推荐系统初期，用户和物品数据都极少

## 2. 基础解决方案

### 2.1 基于规则的方法

热门推荐：按全局点击率/转化率排序
- 简单有效，作为兜底策略
- 问题：无个性化，加剧马太效应

随机推荐：从物品库中随机抽样
- 增加多样性和覆盖率
- 配合 Bandit 策略快速收集反馈

基于属性的推荐：利用用户注册信息（年龄/性别/地域）匹配
- 实现简单，有一定个性化
- 依赖用户填写信息的完整性

### 2.2 三阶段冷启动框架

阶段一（注册时）：利用注册信息 + 引导选择（兴趣标签选择页）
阶段二（少量行为后）：Bandit 快速探索 + 初步画像
阶段三（行为积累后）：逐步过渡到协同过滤 / 深度模型

关键设计：阶段间的平滑过渡
- 方法：基于行为数量设置权重 w
- 最终分 = w * 个性化分 + (1-w) * 冷启动分
- w 随行为数增加而单调递增

## 3. 迁移学习方案

### 3.1 跨域推荐

核心思想：从数据丰富的源域迁移知识到目标域

映射函数方法：
- 学习用户/物品在源域和目标域嵌入空间之间的映射
- f: V_source -> V_target
- 适合两个域有重叠用户但物品不同的场景

对抗域适应：
- 用对抗网络学习域无关的用户表示
- 最小化域判别器的准确率 + 最大化推荐任务的准确率
- 适合源域和目标域差异较大的场景

### 3.2 多任务学习

共享底层表示 + 任务特定的上层：
- 硬参数共享：直接共享底层网络参数
- 软参数共享：各任务有独立底层，通过正则化约束参数相似
- MMoE（Multi-gate Mixture-of-Experts）：多个专家网络 + 任务特定的门控

冷启动场景应用：将冷启动推荐和普通推荐作为两个任务联合训练

## 4. Bandit 算法

### 4.1 LinUCB（上下文 Bandit）

核心公式：
a_t = argmax(x_a^T * theta_a + alpha * sqrt(x_a^T * A_a^{-1} * x_a))

- 前半部分：线性回报估计
- 后半部分：不确定性奖励（探索项）
- A_a 是协方差矩阵，随数据积累，不确定性逐渐缩小

冷启动应用：
- 对新用户：利用注册特征作为上下文 x，快速学习偏好
- 对新物品：将新物品视为新的臂，通过探索获取初始反馈
- 优势：显式平衡探索与利用

### 4.2 Contextual Bandit vs Full RL

Contextual Bandit：即时奖励，无状态转移，计算简单
Full RL：考虑长期累积奖励和状态转移，但样本效率低
冷启动推荐通常用 Contextual Bandit 足够

## 5. 主动学习

核心思想：选择最有信息量的样本进行标注/交互

### 5.1 采样策略

不确定性采样：选择模型最不确定的物品推荐给用户
- 信息增益最大化
- 风险：可能推荐低质量物品伤害体验

多样性采样：选择覆盖不同兴趣方向的物品
- 确保探索的广度
- 基于聚类的方法：从不同聚类中心各选一个

混合策略：不确定性 * 多样性 * 质量分的加权组合

### 5.2 应用场景

新用户引导：精心设计几个问题/推荐快速定位兴趣
冷启动物品：主动推给可能提供有效反馈的用户群

## 6. 元学习（Meta-Learning）

### 6.1 MAML（Model-Agnostic Meta-Learning）

核心思想：学习一个好的初始化参数，使得新任务只需少量数据和几步梯度更新即可适应

双循环优化：
- 外循环：更新元参数 theta
- 内循环：对每个任务 T_i，从 theta 出发，用 T_i 的 support set 做几步梯度下降得到 theta_i'
- 外循环损失：在各 T_i 的 query set 上评估 theta_i' 的效果

冷启动应用：
- 每个用户视为一个"任务"
- 元学习得到的 theta 对所有用户都是较好的起点
- 新用户只需几次交互即可快速个性化

### 6.2 优缺点

优势：
- 理论上最优的"少样本适应"框架
- 模型无关，可与各种推荐模型结合

挑战：
- 二阶梯度计算开销大
- 任务定义（如何构造 support/query set）需要仔细设计
- 实际效果依赖任务间的相似性

## 7. 自监督学习

### 7.1 对比学习（Contrastive Learning）

思路：在无标签数据上学习有区分力的表示

正样本构造：
- 数据增强：随机裁剪/掩码用户行为序列
- 跨视图：同一物品的标题和图片互为正样本

负样本构造：
- 随机采样其他用户/物品
- 难负样本挖掘（batch 内相似度最高的非正样本）

损失函数：InfoNCE Loss
L = -log(exp(sim(z_i, z_j)/tau) / sum(exp(sim(z_i, z_k)/tau)))

冷启动价值：预训练阶段不依赖交互标签，可利用所有物品的内容信息

### 7.2 掩码物品预测

类似 BERT 的预训练：
- 随机掩码行为序列中的物品
- 训练模型根据上下文预测被掩码的物品
- 学到的物品表示可直接用于冷启动推荐

## 8. GAN 用于冷启动

生成对抗网络在冷启动中的应用：
- 生成器：根据物品属性/少量交互生成"模拟用户反馈"
- 判别器：区分真实交互和生成的交互
- 通过对抗训练，生成器学会为新物品生成合理的用户画像

## 9. 评估方法

离线评估：
- Leave-one-out：按时间留最后一个交互做测试
- 时序切分：严格按时间划分训练/验证/测试
- 冷启动专用评估：只在新用户/新物品上计算指标

指标选择：
- 召回阶段：Recall@K, Hit Rate@K
- 排序阶段：NDCG@K, MRR
- 冷启动特有：首 N 次交互后的推荐准确率（学习曲线）

## 10. 面试高频问题

Q: 新用户冷启动和新物品冷启动哪个更难？
A: 通常新用户更难。新物品至少有标题、图片、类别等内容特征可用，通过 NLP/CV 技术可以生成初始嵌入。新用户往往只有注册信息，且注册信息的信息量有限。

Q: 预训练词嵌入如何帮助冷启动？
A: 对新物品：将标题/描述输入预训练的 Word2Vec/BERT，得到文本向量作为物品初始表示，通过内容相似度匹配已有物品。对新用户：将注册信息/首次搜索词转为嵌入向量作为初始用户画像。

Q: MAML 和 fine-tuning 有什么本质区别？
A: Fine-tuning 的预训练目标是在源数据上表现好；MAML 的元训练目标是"快速适应新任务"——它优化的不是当前参数的表现，而是"从当前参数出发，几步更新后在新任务上的表现"。

Q: 冷启动阶段如何平衡探索和利用？
A: 三种策略：
1. Epsilon-greedy：简单但不自适应
2. UCB/LinUCB：理论保证好，适合特征丰富场景
3. Thompson Sampling：自适应探索，工业界首选
关键：随着交互增多，逐步减少探索比例

Q: 知识图谱如何帮助冷启动？
A: 通过实体链接将新物品/新用户的文本信息映射到知识图谱实体，再通过图谱中的关系进行多跳推理，发现潜在关联。例如新用户填写"喜欢科幻电影" -> 链接到"科幻"实体 -> 关联导演/演员 -> 推荐其作品。
