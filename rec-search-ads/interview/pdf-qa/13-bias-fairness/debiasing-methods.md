# 推荐系统去偏方法：IPW、因果推断、Doubly Robust、对抗训练

## 1. 逆倾向加权（IPW / IPS）

### 1.1 核心思想
观测数据不是随机采样的 → 直接在观测数据上训练会有偏 → 用倾向得分加权纠偏。

```
倾向得分（Propensity Score）：
  e(x) = P(O=1 | x) = 样本被观测到的概率

IPS 加权损失：
  L_IPS = (1/N) * sum_i [loss(y_i, y_hat_i) / e(x_i)]

直觉：
  - 高概率被观测的样本（如热门物品在好位置）→ 权重低（不过分依赖）
  - 低概率被观测的样本（如长尾物品在差位置）→ 权重高（珍贵信号）
```

### 1.2 倾向得分估计方法
```
方法一：基于位置的估计（Position Bias）
  e(position=k) = P(examine | position=k)
  通过随机化实验或 EM 算法估计

方法二：基于物品的估计（Popularity Bias）
  e(item) = 物品被展示的频率 / 总展示次数
  或用逻辑回归建模 P(展示 | 物品特征)

方法三：基于用户-物品的估计（Selection Bias）
  e(user, item) = P(展示 | user, item)
  用倾向得分模型（LR/小 DNN）预测
  特征：用户活跃度、物品流行度、历史交互模式
```

### 1.3 方差问题与改进
```
问题：当 e(x) 很小时，1/e(x) 极大 → 方差爆炸

改进一：截断 IPS（Clipped IPS / CIPS）
  w_i = min(1/e(x_i), M)
  M 为截断阈值，通常取经验值（如 50 或 100）
  效果：引入少量偏差，大幅降低方差

改进二：自归一化 IPS（SNIPS）
  w_i_normalized = w_i / sum_j(w_j)
  归一化后权重和为1，方差更稳定
  理论上仍一致，但有限样本下方差更低

改进三：加权截断（Weight Clipping with Smooth Decay）
  w_i = (1/e(x_i))^gamma，gamma < 1（如 0.5）
  平滑压缩权重范围

对比表：
  方法      偏差    方差    实现难度
  原始IPS   无偏    高      低
  CIPS      有偏    中      低
  SNIPS     渐近无偏 低     低
  DR        低      低      中
```

## 2. 双重稳健估计（Doubly Robust, DR）

### 2.1 核心思想
结合 IPS 和直接拟合（填补模型）的优点：只要两个模型中有一个正确，估计就是无偏的。

```
DR 估计器公式：
  V_DR = (1/N) * sum_i [
    f_hat(x_i)  +  (O_i / e(x_i)) * (y_i - f_hat(x_i))
  ]

其中：
  f_hat(x_i) = 填补模型的预测（直接拟合）
  e(x_i) = 倾向得分
  O_i = 是否被观测到（0或1）
  y_i = 实际观测值

分解理解：
  = 填补模型预测 + IPS加权的残差校正
  当填补模型完美时：残差=0，DR退化为直接拟合
  当倾向得分完美时：DR退化为IPS估计
```

### 2.2 在推荐系统中的应用
```
场景：离线评估新推荐策略的效果

实现步骤：
1. 用历史数据训练填补模型 f_hat：预测 E[reward | user, item]
2. 估计旧策略的倾向得分 e(user, item)
3. 对每条历史记录计算 DR 估计值
4. 聚合得到新策略的期望效果

代码框架：
  for each (user, item, reward) in log_data:
      imputed = f_hat.predict(user, item)          # 填补
      propensity = e_model.predict(user, item)     # 倾向得分
      new_prob = new_policy.prob(item | user)       # 新策略选择该物品的概率
      old_prob = old_policy.prob(item | user)       # 旧策略概率

      is_ratio = new_prob / old_prob                # 重要性采样比
      dr_value = imputed + is_ratio / propensity * (reward - imputed)
```

### 2.3 MRDR（More Robust Doubly Robust）
```
改进点：优化填补模型的训练目标
- 标准 DR：填补模型用 MSE 训练
- MRDR：填补模型的训练目标直接最小化 DR 估计器的方差
- 效果：进一步降低方差，实际效果显著优于标准 DR
```

## 3. 因果推断去偏

### 3.1 因果图建模
```
推荐系统的因果图（典型）：

  流行度 --→ 曝光 --→ 点击
     \                 ↑
      \-------→ 从众效应

  用户兴趣 --------→ 点击

  位置 --------→ 检视 --→ 点击

关键混淆因素：
- 流行度：同时影响"物品被推荐"和"用户点击"
- 位置：同时影响"用户是否看到"和"点击概率"
- 用户活跃度：活跃用户数据多 + 活跃用户点击率高
```

### 3.2 干预（do-calculus）
```
观测分布 vs 干预分布：
  P(click | item) = 观测条件概率（含混淆因素影响）
  P(click | do(item)) = 干预因果效应（去除混淆因素）

如何从观测数据估计干预效应？
  后门准则：控制所有混淆变量 Z
  P(click | do(item)) = sum_z P(click | item, z) * P(z)

推荐系统中的应用：
  控制流行度 Z=popularity：
  P(click | do(item)) = sum_pop P(click | item, pop) * P(pop)
  → 消除"因为热门所以被推荐所以被点击"的虚假相关
```

### 3.3 反事实推理
```
核心问题：如果推荐了不同的物品，用户会怎么反应？

Individual Treatment Effect (ITE)：
  tau(user) = Y(item=A) - Y(item=B)
  对同一用户，推荐 A vs 推荐 B 的因果效应

估计方法：
1. 匹配法：找到相似用户的不同推荐结果对比
2. 双重机器学习（DML）：
   - 第一步：用机器学习去除混淆因素
   - 第二步：在残差上做因果估计
3. 工具变量法：找到只影响推荐但不直接影响点击的变量（如排序算法版本）
```

### 3.4 因果 Embedding
```
DICE 方法（Disentangling Interest and Conformity）：
- 将物品 Embedding 分为两部分：
  - 兴趣 Embedding：捕捉用户真正的偏好信号
  - 从众 Embedding：捕捉流行度/从众效应
- 训练时两部分共同预测点击
- 推理时只用兴趣 Embedding → 天然去除流行度偏差

CausE 方法（Causal Embedding）：
- 在均匀随机曝光数据（无偏）和正常日志数据（有偏）上联合训练
- 利用无偏数据校准有偏模型
- 正则化约束两组 Embedding 的差异
```

## 4. 对抗训练去偏

### 4.1 架构设计
```
推荐模型（生成器 G）
    |
    v
用户表示向量 h
    |
    +--→ 主任务头：预测 P(click | user, item)
    |
    +--→ 判别器 D：预测 P(sensitive_attribute | h)

训练目标（对抗博弈）：
  min_G max_D  L_main(G) - lambda * L_adv(D)

含义：
  G 想学到好的用户表示来预测点击
  同时让 D 无法从表示中识别敏感属性（如用户群体）
  → 用户表示中去除了敏感属性信息
  → 推荐结果不依赖敏感属性 → 公平性
```

### 4.2 去偏应用场景
```
场景一：去除流行度偏差
  敏感属性 = 物品流行度
  目标：物品表示中不含流行度信息
  效果：模型不因物品热门而给高分

场景二：去除用户群体偏差（公平性）
  敏感属性 = 用户性别/年龄/种族
  目标：推荐结果不因用户群体而有差异
  应用：招聘推荐、信贷推荐等高敏感场景

场景三：去除位置偏差
  敏感属性 = 展示位置
  目标：物品表示中不含位置信息
  效果：排序分数只反映物品质量
```

### 4.3 训练技巧
```
1. 梯度反转层（GRL）：在判别器和生成器之间加 GRL
   前向传播：正常传递
   反向传播：梯度取反
   效果：等价于对抗训练，但更稳定

2. lambda 调度：
   训练初期 lambda 小（先学好主任务）
   逐步增大 lambda（逐步增强去偏约束）
   避免一开始就过度约束导致主任务性能崩溃

3. 判别器能力控制：
   判别器太强 → 生成器丢失过多有用信息
   判别器太弱 → 去偏效果不够
   实践：判别器用 2-3 层小 MLP
```

## 5. 多任务学习去偏

### 5.1 方法设计
```
将倾向得分估计作为辅助任务：

共享层
  |
  +--→ 主任务塔：预测 P(click | user, item)
  |
  +--→ 辅助任务塔：预测 P(observe | user, item)（倾向得分）

联合训练：
  L = L_main + alpha * L_propensity

优势：
  - 共享底层表示 → 倾向得分估计更准确
  - 端到端训练 → 倾向得分自然融入主任务学习
  - 不需要额外的倾向得分模型
```

### 5.2 与 ESMM 的类比
```
ESMM：P(purchase) = P(click) * P(purchase | click)
去偏：P(click) = P(observe) * P(click | observe)

相似结构：都是将目标分解为可估计的子问题
区别：ESMM 解决样本选择偏差（只在点击样本上训练 CVR），
      去偏多任务解决曝光偏差（只在曝光样本上训练 CTR）
```

## 6. 离线策略评估（OPE）中的去偏

### 6.1 IPS 估计器
```
评估新策略 pi_new 的期望奖励（用旧策略 pi_old 收集的数据）：

V_IPS(pi_new) = (1/N) * sum_i [pi_new(a_i|s_i) / pi_old(a_i|s_i)] * r_i

问题：
  当 pi_new 和 pi_old 差异大时，重要性比可能极端
  → 高方差，估计不可靠

改进：加权截断 + 自归一化
```

### 6.2 DR 估计器在 OPE 中的应用
```
V_DR(pi_new) = (1/N) * sum_i [
    E_pi_new[f_hat(s_i, a)]
    + [pi_new(a_i|s_i) / pi_old(a_i|s_i)] * (r_i - f_hat(s_i, a_i))
]

优势：方差远低于 IPS，且仍保持一致性
实践中是工业界离线评估的首选方法
```

### 6.3 MAGIC 估计器
```
多步 IS 与模型估计的加权组合：
- 短期用 IS 估计（方差低、偏差低）
- 长期用模型估计（方差低但偏差可能大）
- 自适应加权组合
适用于长序列交互场景（如会话推荐）
```

## 7. 公平性约束方法

### 7.1 预处理方法
```
在数据层面消除偏差：
- 重采样：对少数群体过采样 / 多数群体欠采样
- 特征变换：去除与敏感属性相关的特征分量
- 数据增强：为少数群体生成合成样本

优点：不修改模型
缺点：可能损失有用信息
```

### 7.2 过程中约束
```
在模型训练中加入公平性约束：
- 正则化项：L = L_main + lambda * Fairness_Penalty
- 约束优化：max Accuracy s.t. GroupParity < epsilon
- 拉格朗日对偶：将约束转为拉格朗日乘子自动平衡

Fairness_Penalty 示例：
  Statistical Parity: |P(Y=1|A=0) - P(Y=1|A=1)|
  Equal Opportunity: |P(Y=1|A=0,Y_true=1) - P(Y=1|A=1,Y_true=1)|
```

### 7.3 后处理方法
```
在排序结果上重排以满足公平性：
- 比例约束：各群体在推荐列表中的比例满足要求
- 曝光公平：各群体物品的曝光量与质量成正比
- 轮转机制：保证各群体物品轮流获得好位置

实现：
  ranked_list = fairness_rerank(scored_list, constraints)

  贪心算法：每步选满足约束的最高分物品
  LP 松弛：将整数规划松弛为线性规划求解

优点：模型无关，灵活可控
缺点：可能显著降低排序质量
```

## 8. 面试高频问题

```
Q: IPS 和 DR 的区别？什么时候用哪个？
A: IPS 只需估计倾向得分，但方差高。DR 结合 IPS 和填补模型，
   方差更低且双重稳健。实践中首选 DR，因为它对模型误差更鲁棒。

Q: 因果推断在推荐去偏中的具体落地路径？
A: 1) 画因果图识别混淆因素 2) 用后门准则或前门准则确定调整集
   3) 估计干预效应替代观测效应 4) 实际落地常用 DICE 式解耦方法，
   将混淆因素的影响在 Embedding 层面分离。

Q: 对抗训练去偏会损失推荐准确率吗？
A: 通常会有轻微损失（1-3% AUC），因为去除了部分与敏感属性相关的
   有效信号。关键是调节 lambda 找到准确率-公平性的最佳权衡点。

Q: 如何评估去偏方法是否有效？
A: 1) 离线：比较去偏前后的 NDCG 和公平性指标
   2) 在线：A/B 测试观察各群体的体验指标
   3) 反事实评估：用 DR 估计器评估去偏策略的效果
   4) 关注是否在去偏的同时保持了推荐质量

Q: 多种去偏方法可以叠加使用吗？
A: 可以，且推荐叠加使用。典型组合：
   模型层：因果 Embedding 去除流行度混淆
   训练层：IPS 加权纠正选择偏差
   排序层：位置去偏（PAL）
   重排层：公平性约束后处理
```
