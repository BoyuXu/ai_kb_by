# 推荐系统偏差类型：识别、量化与影响

## 1. 位置偏差（Position Bias）

### 1.1 现象与成因
- 用户浏览行为遵循自上而下模式（F 型扫描），靠前位置物品获得不成比例的曝光和点击
- 实验数据：首位 CTR 通常是第 10 位的 5-10 倍，但物品质量差距远没有这么大
- 本质：点击 = f(相关性) * f(位置)，位置因素混入了点击信号

### 1.2 数学建模
```
点击概率的分解假设（Examination Hypothesis）：
P(click | item, position) = P(examine | position) * P(click | examine, item)

其中：
- P(examine | position) = 位置的检视概率（与物品无关）
- P(click | examine, item) = 用户看到后真正的兴趣概率（与位置无关）

估计检视概率的方法：
1. 随机化实验：在部分流量中随机打乱排序，直接统计各位置点击率
2. EM 算法：将检视行为视为隐变量，迭代估计
3. 回归方法：在 CTR 模型中加入位置特征，学习位置系数
```

### 1.3 去偏方法
```
方法一：位置特征建模
- 训练时：模型输入包含位置特征 position_id
- 预测时：将 position_id 设为默认值（如全部设为位置1），消除位置影响
- 优点：简单易实现
- 缺点：假设位置效应对所有物品一样，实际上热门物品受位置影响更小

方法二：逆倾向加权（IPS）
- 训练时对每个样本加权：w_i = 1 / P(examine | position_i)
- 被放在差位置但仍被点击的样本获得更大权重
- 高位样本降权，低位样本升权

方法三：PAL（Position-Aware Learning）
- 训练一个 position tower 专门建模位置效应
- 主塔建模物品相关性
- 最终分数 = 主塔分数 * position tower（训练时）
- 推理时去掉 position tower
```

### 1.4 量化指标
```
位置偏差强度 = sum_k [CTR(position=k) / CTR(position=1)] 的偏离度
理想情况（无偏差）：各位置 CTR 仅由物品质量决定
实际测量：做随机实验，计算各位置的归一化 CTR 曲线斜率
```

## 2. 选择偏差（Selection Bias）

### 2.1 现象与成因
- 训练数据仅来自系统选中展示的物品，未展示物品无反馈信号
- 数据缺失机制是 Missing Not At Random (MNAR)
- 系统越倾向推荐某类物品 → 该类物品数据越多 → 模型越偏向该类 → 正反馈循环

### 2.2 与曝光偏差的区别
```
选择偏差：系统决定展示什么 → 观测数据有偏
曝光偏差：用户是否看到 → 负样本定义有偏

两者关系：
选择偏差 ⊃ 曝光偏差
选择偏差覆盖从系统到用户的全链路，曝光偏差特指用户端

例子：
- 选择偏差：系统只推荐过运动鞋，不知道用户对皮鞋的偏好
- 曝光偏差：系统推荐了10个物品，用户只看到前5个，后5个的"未点击"不等于"不喜欢"
```

### 2.3 去偏方法
```
方法一：随机化曝光实验
- 在 5-10% 流量中随机展示物品，收集无偏数据
- 用无偏数据训练/校准模型
- 代价：短期用户体验下降

方法二：IPS 加权
- 估计每个物品被系统选中展示的概率 P(O=1 | user, item)
- 训练时对每个样本加权 w = 1 / P(O=1 | user, item)
- 稀有物品获得更大权重

方法三：数据填补（Imputation）
- 用模型为未观测到的 (user, item) 对估算反馈
- 填补模型通常用简单模型（如矩阵分解）
- 结合 IPS 形成双重稳健估计器
```

## 3. 曝光偏差（Exposure Bias）

### 3.1 核心问题
- 推荐系统的负样本定义：用户未交互的物品 = 负样本？
- 真相：用户根本没看到该物品，"未点击" ≠ "不喜欢"
- 后果：大量假负例 → 模型学到的是"什么被展示过"而非"用户真正不喜欢什么"

### 3.2 影响分析
```
假负例的危害：
1. 稀有/长尾物品几乎不被展示 → 全变成负样本 → 模型更不推荐 → 恶性循环
2. 冷启动物品天然"负例率"高 → 新物品难以出头
3. 模型对"未曝光物品"的评估完全不可靠

量化方式：
- 曝光覆盖率 = 被展示过的 (user, item) 对 / 所有可能的对
- 典型值：< 0.1%，即 99.9% 的交互空间完全未被观测
```

### 3.3 解决方案
```
方法一：曝光感知模型
- 引入隐变量 E（用户是否看到物品）
- P(click) = P(E=1) * P(click | E=1)
- EM 算法或变分推断估计 P(E=1)

方法二：加权负采样
- 计算物品被曝光的概率 P(exposure | item)
- 负采样时按 P(exposure) 加权：曝光多但未点击 → 更可靠的负样本
- 未曝光物品降低负样本权重或不作为负样本

方法三：对比学习视角
- 正样本：点击/交互的物品
- 可信负样本：曝光但未交互的物品
- 不确定样本：未曝光物品 → 不直接参与对比学习
```

## 4. 流行度偏差（Popularity Bias）

### 4.1 多阶段传播机制
```
阶段1 - 数据层面：
  热门物品天然获得更多交互数据
  → 马太效应：数据量 ∝ 流行度

阶段2 - 模型层面：
  更多数据 → 模型对热门物品预测更准确
  → 热门物品得分系统性偏高

阶段3 - 展示层面：
  高分物品更多曝光 → 更多交互数据
  → 正反馈循环闭合

量化指标：
- 长尾覆盖率：推荐结果中长尾物品占比
- 基尼系数：物品曝光分布的不均匀度（0=完全均匀，1=完全集中）
- 流行度分布偏斜度：推荐列表中物品流行度的分布 vs 理想均匀分布
```

### 4.2 去偏方法
```
方法一：逆流行度加权（IPW for Popularity）
- w_item = 1 / popularity(item)^alpha
- alpha 控制去偏强度，通常 0.5-1.0
- 对热门物品降权，长尾物品升权

方法二：因果推断
- 构建因果图：流行度 → 曝光 → 点击 ← 用户兴趣
- 流行度是混淆因素，通过 do-calculus 剥离：
  P(click | do(item)) vs P(click | item)
- 前者去除了流行度的混淆效应

方法三：后处理调整
- 精排后对分数做流行度惩罚：score_adjusted = score - beta * log(popularity)
- 重排阶段强制加入长尾物品配额
- 优点：不影响模型训练，灵活可控

方法四：DICE（Disentangling Interest and Conformity）
- 将用户行为解耦为兴趣驱动和从众驱动
- 分别用两个 Embedding 表示
- 推理时只用兴趣 Embedding，去除从众效应
```

## 5. 其他偏差类型

### 5.1 信息茧房（Filter Bubble）
```
成因：系统持续推荐用户已有偏好的内容 → 用户视野越来越窄
检测：用户兴趣向量的方差随时间递减、推荐结果的品类集中度上升
打破方式：
- 探索性推荐注入（epsilon-greedy / Thompson Sampling）
- 跨领域推荐（用户在 A 领域的行为迁移到 B 领域）
- 多样性约束重排
```

### 5.2 幸存者偏差
```
成因：只分析成功案例（被点击的物品），忽略失败案例（未被展示/被展示但未点击）
推荐系统中的体现：
- 只优化已有用户的体验，忽略流失用户的反馈
- 只分析活跃物品的特征，忽略被淘汰物品的信息
```

### 5.3 锚定偏差
```
首个展示物品会影响用户对后续物品的评价标准
在推荐列表中：第一个物品的质量影响用户对整个列表的感知
应对：首位物品选择策略需要特别优化
```

## 6. 面试高频问题

```
Q: 位置偏差和曝光偏差有什么区别？
A: 位置偏差是"看到了但因为位置不好而不点击"，曝光偏差是"根本没看到"。
   位置偏差影响已曝光物品的反馈质量，曝光偏差影响负样本的可靠性。

Q: 如何判断推荐系统是否存在严重的流行度偏差？
A: 1) 计算推荐列表的基尼系数 vs 理想值
   2) 对比推荐结果中的流行度分布 vs 用户真实偏好分布
   3) 检查长尾物品的覆盖率是否持续下降
   4) 观察新上架物品的冷启动成功率

Q: IPS 方法的核心局限是什么？如何改进？
A: 核心局限是高方差——当倾向得分很小时，权重极大导致估计不稳定。
   改进：截断 IPS（设权重上限 M）、自归一化 IPS（权重归一化）、
   双重稳健估计器（结合 IPS 和直接拟合的优势）。

Q: 偏差之间会相互叠加吗？
A: 会。位置偏差 + 流行度偏差叠加：热门物品放在好位置 → 获得更多点击 →
   更高分 → 继续放好位置。选择偏差 + 曝光偏差叠加：系统偏向选择热门 →
   用户只能看到热门 → 长尾物品完全无反馈。多种偏差叠加使得反馈循环更强，
   必须在多个阶段同时去偏。
```
