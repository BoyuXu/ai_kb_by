## 1. 推荐系统中的偏差类型

### 1.1 位置偏差（Position Bias）
- 定义：用户更倾向于点击列表中靠前位置的物品，与物品本身质量无关
- 产生原因：用户浏览行为的自上而下模式，导致高位物品获得不成比例的曝光和点击
- 影响：模型会错误地将「被放在好位置」等同于「用户真正喜欢」
- 解决思路：
  - 位置特征建模：训练时加入位置特征，预测时将位置特征设为无偏值
  - 逆倾向加权（IPS）：对每个样本按其被展示在该位置的概率加权
  - 因果推断：通过干预（do-calculus）剥离位置对点击的因果效应

### 1.2 选择偏差（Selection Bias）
- 定义：训练数据仅来自被推荐系统选中展示的物品，未展示物品无反馈
- 本质：Missing Not At Random（MNAR），数据缺失不是随机的
- 影响：模型只学到了「被推荐过的物品空间」中的模式，对未曾推荐过的物品无法准确评估
- 解决思路：
  - 随机化曝光实验：在部分流量中随机展示物品收集无偏数据
  - IPS 加权：对每个样本按被选中展示的概率加权
  - 数据填补（Imputation）：为未观测到的样本估算反馈

### 1.3 曝光偏差（Exposure Bias）
- 定义：用户只能与被推荐的物品交互，对未曝光物品的「不点击」不代表「不喜欢」
- 与选择偏差的区别：曝光偏差更强调用户根本没看到该物品
- 影响：将「未曝光=负样本」会引入大量假负例，损害模型泛化能力
- 解决思路：
  - 曝光感知模型：将用户是否看到物品建模为隐变量
  - 加权负采样：根据物品被曝光的概率调整负样本权重

### 1.4 流行度偏差（Popularity Bias）
- 定义：热门物品获得更多曝光和交互，形成正反馈循环，挤压长尾物品
- 多阶段传播：
  - 数据层面：热门物品样本量天然更大
  - 模型层面：模型在热门物品上预测更准确，进一步偏向推荐热门
  - 用户层面：用户看到的多为热门内容，反馈数据进一步偏斜
- 影响：信息茧房、长尾物品生态恶化、内容多样性下降
- 解决思路：
  - 逆流行度加权：对热门物品降权、长尾物品升权
  - 因果推断消除流行度混淆因素
  - 重排阶段加入多样性约束

## 2. 去偏方法

### 2.1 逆倾向得分加权（IPS, Inverse Propensity Scoring）
核心思想：对每个样本按其被观测到的概率的倒数加权，纠正非随机采样导致的偏差

计算步骤：
1. 估计倾向得分 P(O=1|x)：即样本被观测到的概率
2. 对损失函数中每个样本乘以权重 w = 1/P(O=1|x)
3. 加权后的损失函数近似无偏估计

```
# IPS 加权损失函数
L_IPS = sum(w_i * loss(y_i, y_hat_i)) / sum(w_i)
# w_i = 1 / propensity_score_i
```

问题与改进：
- 高方差：当 P(O=1|x) 很小时，权重极大导致方差爆炸
- 截断 IPS（Clipped IPS）：对权重设置上限 M，w_i = min(1/P(O=1|x), M)
- 自归一化 IPS（SNIPS）：权重归一化以降低方差

### 2.2 双重稳健估计（Doubly Robust Estimation）
- 思想：结合 IPS 和直接拟合（填补模型）的优势
- 只要倾向得分模型或填补模型其中一个正确，估计就是无偏的
- 公式：DR = imputation + IPS * (actual - imputation)
- 优势：比单独使用 IPS 或填补更鲁棒
- 实际中两个模型都不完美时仍能显著降低偏差

### 2.3 多任务学习去偏
- 将去偏目标作为辅助任务，与主推荐任务联合训练
- 例如：主任务预测点击率 + 辅助任务预测倾向得分
- 共享底层表示学习更鲁棒的特征

### 2.4 因果推断去偏
- 构建因果图，识别混淆因素
- 通过 do-calculus 估计干预效应而非观测效应
- 典型应用：消除流行度偏差中的混淆路径

## 3. 公平性概念与度量

### 3.1 群体公平性（Group Fairness / Statistical Parity）
- 定义：不同群体（如性别、年龄组）获得推荐的概率应相等
- 度量：P(推荐|群体A) = P(推荐|群体B)
- 适用场景：招聘推荐、信贷推荐等高敏感领域

### 3.2 个体公平性（Individual Fairness）
- 定义：相似的用户应获得相似的推荐
- 度量：需要定义用户间的「相似性」度量
- 挑战：如何定义「相似」本身可能包含偏见

### 3.3 机会均等（Equalized Opportunity）
- 定义：在真正感兴趣的用户中，不同群体被推荐的概率应相等
- 度量：P(推荐|感兴趣, 群体A) = P(推荐|感兴趣, 群体B)
- 比统计平等更合理：允许整体推荐率不同，只要求对「该推荐」的用户公平

### 3.4 曝光公平（Exposure Fairness）
- 定义：供给侧（物品/创作者）获得的曝光应与其质量/价值成正比
- 度量：基尼系数、物品曝光分布的熵
- 目的：防止流量过度集中，保障生态健康

## 4. 公平性约束方法

### 4.1 对抗训练（Adversarial Training）
- 架构：推荐模型 + 判别器
- 推荐模型学习预测用户偏好，同时对抗判别器（判别器试图从推荐结果中识别敏感属性）
- 目标：推荐结果无法被判别器区分出所属群体
- 训练：min-max 博弈，类似 GAN

### 4.2 多目标优化
- 将公平性作为优化目标之一
- 约束优化：在公平性约束下最大化推荐准确率
- 帕累托优化：寻找准确率-公平性的帕累托前沿

### 4.3 后处理方法
- 在排序结果上重排，强制满足公平性约束
- 优点：不需要修改模型
- 缺点：可能损害推荐质量

## 5. 公平性评估实验设计

### 5.1 离线评估
- 按敏感属性分组计算各组的推荐指标（AUC/NDCG）
- 计算组间指标差异
- 分析曝光分布的公平性（基尼系数等）

### 5.2 在线AB测试
- 分群对比不同群体的体验指标
- 关注核心指标是否在所有群体中都有提升
- 警惕辛普森悖论：整体提升可能掩盖某些群体的体验下降

### 5.3 公平性-准确率权衡分析
- 绘制公平性约束强度 vs 推荐准确率的帕累托曲线
- 通过业务判断选择合适的权衡点
- 不同业务场景的容忍度不同（内容推荐 vs 金融推荐）

## 6. 反馈循环与偏差放大

关键认知：偏差不是静态的，推荐系统会放大已有偏差
- 模型推荐偏向热门 -> 热门物品获更多反馈 -> 模型更偏向热门
- 打破方式：
  - 探索策略注入（epsilon-greedy、Thompson Sampling）
  - 定期随机化曝光收集无偏数据
  - 因果模型剥离混淆因素
  - 监控系统持续跟踪偏差指标
