# 基础推荐模型原理与实现

## 2.1 协同过滤

### 核心思想
利用用户-物品交互矩阵，基于"相似"进行推荐，无需物品内容信息。

### UserCF vs ItemCF

| 维度 | UserCF | ItemCF |
|------|--------|--------|
| 核心假设 | 相似用户喜欢相似物品 | 用户喜欢与历史偏好相似的物品 |
| 适用场景 | 用户少、物品多（新闻） | 物品少、用户多（电商） |
| 实时性 | 能捕捉用户群体热点 | 更个性化，推荐稳定 |
| 计算瓶颈 | 用户相似度矩阵 | 物品相似度矩阵 |
| 冷启动 | 新物品可通过活跃用户快速传播 | 新用户可通过少量行为获得推荐 |

### 相似度计算
- 余弦相似度：`sim(u,v) = dot(r_u, r_v) / (|r_u| * |r_v|)`
- Jaccard：`|A ∩ B| / |A ∪ B|`，仅考虑是否交互，忽略评分
- Pearson相关：减去均值的余弦相似度，消除评分尺度差异
- 改进：IUF（Inverse User Frequency）降低活跃用户的权重（类似IDF思想）

### 稀疏性问题
用户-物品矩阵通常 99%+ 是空的。缓解方法：
- 引入隐式反馈（浏览、停留时长）扩充交互数据
- 矩阵分解降维
- 引入内容特征做混合推荐

## 2.2 矩阵分解

### FunkSVD
将用户-物品评分矩阵 R 分解为两个低秩矩阵：`R ≈ P * Q^T`，其中 P 是用户隐向量矩阵（m x k），Q 是物品隐向量矩阵（n x k）。

预测公式：`r_hat(u,i) = p_u^T * q_i + b_u + b_i + mu`
- `mu`：全局均值
- `b_u`、`b_i`：用户/物品偏置项
- `p_u`、`q_i`：用户/物品隐向量

### 优化方法
SGD更新规则（以用户隐向量为例）：
```
e_ui = r_ui - r_hat(u,i)
p_u <- p_u + lr * (e_ui * q_i - lambda * p_u)
q_i <- q_i + lr * (e_ui * p_u - lambda * q_i)
```

ALS（交替最小二乘）：固定一个矩阵解另一个，每步有解析解，适合并行化和隐式反馈。

### L2 正则化
损失函数：`L = sum((r_ui - p_u^T * q_i)^2) + lambda * (|p_u|^2 + |q_i|^2)`
- lambda 过大：欠拟合，所有隐向量趋向零
- lambda 过小：过拟合，在训练集上完美但泛化差

### 冷启动处理
- 新用户/物品无交互数据，无法学到隐向量
- 解决：引入 side information（用户画像、物品属性）作为隐向量的先验
- 或采用混合方法（MF + 内容特征）

## 2.3 协同过滤的挑战

### 冷启动问题
三种类型：
1. 新用户冷启动：无历史行为。方案：引导性问卷、热门推荐、基于注册信息的内容推荐
2. 新物品冷启动：无交互数据。方案：内容特征推荐、Bandit探索策略
3. 系统冷启动：交互数据极少。方案：迁移学习、基于内容的方法过渡

### 数据稀疏性
- 隐式反馈扩充：将浏览、搜索、停留等行为作为弱正样本
- 负采样策略：随机负采样、曝光未点击作为负样本、hard negative mining
- 图方法：通过多跳传播缓解稀疏

### 混合推荐
- 加权融合：CF分数 + 内容分数的加权和
- 级联式：CF粗排 -> 内容精排
- 特征融合：将CF结果作为特征输入统一模型

## 2.4 神经协同过滤（NeuMF）

### 架构设计
NeuMF = GMF + MLP 的融合模型：
- GMF（Generalized Matrix Factorization）：`phi_GMF = p_u * q_i`（逐元素乘积），捕获线性交互
- MLP：`phi_MLP = MLP(concat(p_u, q_i))`，捕获非线性交互
- 融合：`y = sigma(h^T * concat(phi_GMF, phi_MLP))`

GMF和MLP使用独立的Embedding，避免相互干扰。

### 预训练策略
1. 分别预训练GMF和MLP
2. 用预训练参数初始化NeuMF的对应部分
3. 端到端微调（学习率需降低）

### 损失函数
- 显式反馈：MSE / MAE
- 隐式反馈：Binary Cross-Entropy 或 BPR Loss
- BPR Loss：`L = -sum(log(sigma(r_hat(u,i) - r_hat(u,j))))`，其中 i 是正样本，j 是负样本

### 面试关键点
- NeuMF解决了传统MF只能建模线性交互的局限
- Embedding维度是关键超参：太小表达力不足，太大过拟合
- 工业界更常用 two-tower 结构（用户塔 + 物品塔），因为物品侧可离线计算、在线 ANN 检索

## 2.5 图神经网络推荐

### 为什么用 GNN
用户-物品交互天然构成二部图。GNN能通过消息传递聚合多跳邻居信息，捕获高阶协同信号（如"喜欢A和B的用户也喜欢C"）。

### LightGCN
对标准GCN的简化：去掉特征变换矩阵和非线性激活，只保留邻域聚合。

消息传递：
```
e_u^(k+1) = sum(1/sqrt(|N_u|*|N_i|) * e_i^(k))  对 i in N_u
e_i^(k+1) = sum(1/sqrt(|N_i|*|N_u|) * e_u^(k))  对 u in N_i
```

最终表示：各层表示的加权和 `e_u = sum(alpha_k * e_u^(k))`

简化的原因：推荐系统中节点特征就是 ID Embedding，特征变换和非线性激活反而引入噪声，降低性能。

### GraphSAGE
归纳式学习，支持新节点：
1. 邻居采样（固定每层采样数量，如 [25, 10]）
2. 聚合：Mean/LSTM/Pooling
3. 拼接自身表示和邻居聚合结果，经MLP更新

优势：不需要全图训练，可mini-batch训练，扩展性好。

### BPR 损失
推荐场景最常用的 pairwise 排序损失：
```
L_BPR = -sum(log(sigma(score(u,i) - score(u,j))))
```
正样本 i 是用户交互过的物品，负样本 j 是随机采样的未交互物品。

### 面试关键点
- GNN层数不宜过深（通常2-3层），过深会导致过平滑（所有节点表示趋同）
- 负采样策略影响很大：均匀采样 vs 热度采样 vs hard negative
- 工业落地挑战：图规模大（亿级节点/边）、实时更新图结构、推理延迟
