# 机器学习基础与模型评估

## 3.1 基础评估指标

### AUC-ROC
ROC曲线横轴是假阳性率（FPR），纵轴是真阳性率（TPR/召回率）。AUC是ROC曲线下面积，取值 [0.5, 1]。

AUC的概率含义：随机取一个正样本和一个负样本，模型给正样本打分高于负样本的概率。

在推荐系统中的意义：衡量排序模型将用户感兴趣的物品排在不感兴趣物品前面的能力。

AUC的局限性：
- 宏观指标，不关注 Top-K 位置质量（推荐更关心 NDCG/MAP/Recall@K）
- 对正负样本分布变化不敏感
- 无法直接衡量业务收益
- 多目标优化时（CTR + 停留时长）不够全面

### 曝光偏差对评估的影响
传统离线指标（Precision/Recall）假设"未交互 = 不喜欢"，但实际中未交互可能只是未曝光。这导致：
- 高估推荐热门/安全物品的模型
- 扼杀探索性算法（善于挖掘长尾但离线指标差）

改进方法：
- 逆概率加权（IPW）：按曝光概率的倒数加权样本
- 对AUC的曝光偏差不敏感变体（如平均间隔损失）
- 反事实评估框架
- 在线 A/B 测试 + 用户调研

### Precision 与 Recall
- Precision = TP / (TP + FP)：推荐的物品中用户真正喜欢的比例（"推荐得准不准"）
- Recall = TP / (TP + FN)：用户喜欢的物品中被成功推荐的比例（"推荐得全不全"）

两者的权衡关系：
- 高Precision → 保守推荐，只推高置信度物品 → Recall低，长尾物品被遗漏
- 高Recall → 激进推荐，大量推荐 → Precision低，干扰多

F1 = 2 * P * R / (P + R) 是两者的调和平均。

不同阶段侧重不同：召回阶段重Recall，排序阶段重Precision。

### 多目标评估指标设计
同时优化CTR、停留时长、互动率等多目标时：
1. 归一化：Min-Max 或 Z-score 统一量纲
2. 综合得分：`S = w1*Norm(CTR) + w2*Norm(停留时长) + w3*Norm(互动率)`
3. 乘法模型/一票否决：`S = CTR * log(停留时长)`，底线目标设硬性门槛
4. 最终必须结合线上 A/B 实验验证

## 3.2 离线与在线评估方法

### 离线 vs 在线评估

| 维度 | 离线评估 | 在线评估 |
|------|---------|---------|
| 数据 | 历史静态数据集 | 真实用户实时反馈 |
| 目标 | 快速筛选、低成本迭代 | 验证真实业务影响 |
| 指标 | AUC/GAUC/NDCG/Recall | CTR/转化率/留存率/停留时长 |
| 局限 | 曝光偏差、无法模拟全链路 | 成本高、周期长 |

离线指标好但线上差的常见原因：曝光偏差、无法模拟召回/排序/重排全链路、数据分布偏移。

### 评估偏差类型

位置偏差：用户倾向点击排名靠前的物品，无论实际相关性。
- 离线：位置偏差的日志直接计算指标会高估排在头部的模型
- 在线：影响 A/B 测试中不同排序策略效果的公正比较
- 缓解：逆概率加权、无偏估计器、随机化流量桶

选择偏差：系统只能观测到已推荐物品的反馈。
- 导致马太效应和信息茧房
- 缓解：Bandit探索策略、双重稳健估计

### CTR上升但停留时长下降的分析
可能原因：标题党/低质内容泛滥、推荐多样性降低、排序模型对沉浸式指标学习不足、探索不足导致信息茧房。

分析流程：
1. 数据细分下钻：按用户画像、内容类型、请求上下文切片
2. 关联指标分析：转化率、负反馈率、用户留存率
3. 归因分析：用户点击后的后续行为序列
4. 因果推断：Interleaving测试、微调模型目标权重

### A/B 测试设计
分流策略要点：
- 哈希分流保证随机性和均匀性，推荐以用户ID为分流主体
- 社交推荐需考虑网络效应（社交圈分流）
- 从小流量（5%）开始，逐步放大

样本量计算：
- 确定显著性水平（通常0.05）、统计功效（0.8-0.9）、最小可检测效应（MDE）
- 样本量与MDE的平方成反比：效应越小需要越多样本
- 方差缩减技术（如CUPED）可提升实验灵敏度

可靠性保障：实验周期至少覆盖一个完整业务周期、关注新奇效应、SRM检查、多重检验校正（Bonferroni）。

## 3.3 过拟合与欠拟合控制

### 过拟合 vs 欠拟合
- 过拟合：训练集好、测试集差。模型过度学习了噪声和局部特征
- 欠拟合：训练集和测试集都差。模型复杂度不够或训练不充分

推荐系统中的典型表现：
- 过拟合：离线AUC很高但线上CTR不提升；模型偏向推荐高频物品
- 欠拟合：推荐结果趋同化严重；离线指标本身就低

### Early Stopping
在训练过程中监控验证集性能，当连续 patience 个 epoch 验证损失不再下降时停止训练，回滚到验证损失最低时的模型参数。

本质是通过控制训练时间间接控制模型复杂度（一种隐式正则化）。

实践要点：
- 验证集要有代表性且分布与测试集接近
- 推荐系统中监控指标可能不是损失函数，而是GAUC等业务指标
- 现代优化器（如AdamW）含权重衰减，与早停结合使用需综合考虑

### 正则化方法

L1 正则化（Lasso）：
- 惩罚项：`lambda * sum(|w_i|)`
- 产生稀疏解（部分权重变为0），等效自动特征选择
- 推荐系统中用于筛选关键特征，简化模型

L2 正则化（Ridge）：
- 惩罚项：`lambda * sum(w_i^2)`
- 限制权重幅度但不产生零权重
- 使模型对噪声不敏感，输出更平滑
- 广泛用于线性模型、MF、深度模型各层

Dropout：
- 训练时随机丢弃神经元（概率 p，通常 0.1-0.5）
- 阻止神经元间的共适应关系，强制学习更鲁棒特征
- 推荐系统中防止对特定用户ID/物品ID的过度拟合

### K折交叉验证
将数据分为K份，轮流用一份做验证、其余做训练，取K次评估的平均值。

推荐系统的特殊考虑：
- 必须按时间划分（不能随机划分，否则数据穿越）
- 按用户划分保证每个用户的数据只在训练或验证中
- 数据量极大时可用留出法（K=2或3）代替
- 交叉验证用于模型选择和超参调优，最终需线上A/B验证

### 过拟合诊断与优化方案（电商场景）

诊断流程：
1. 验证集性能：训练后期是否下降或远差于训练集
2. 线上线下一致性：流量特征分布 vs 训练/验证集
3. 特征分析：是否有数据泄露、高维稀疏特征过多
4. 模型与数据量匹配度：复杂模型 + 少量数据 = 高风险

优化方案（分层）：
- 数据层：增加多样性样本、特征筛选去共线性
- 训练层：L2正则化、Dropout、严格早停
- 结构层：降低模型复杂度、集成学习（Bagging天然抗过拟合）
- 流程层：严格交叉验证、独立测试集

### 多任务学习中的动态正则化
不同任务过拟合风险不同。策略：基于任务不确定性（同方差不确定性）动态调整各任务的正则化强度。

具体做法：
- 损失波动大/训练难度高的任务 → 分配较弱的正则化
- 损失稳定/容易拟合的任务 → 分配较强的正则化
- 通过学习权重参数实现自动平衡
- 可结合梯度范数监控调整

## 3.4 超参数优化策略

### 网格搜索 vs 随机搜索

| 维度 | 网格搜索 | 随机搜索 |
|------|---------|---------|
| 原理 | 穷举所有候选组合 | 从分布中随机采样 |
| 优势 | 系统性强，不遗漏 | 计算效率更高 |
| 劣势 | 维度灾难，指数增长 | 随机性导致不稳定 |
| 适用 | 少量超参、小搜索空间 | 超参多、关键超参少 |

当目标函数仅依赖少数几个关键超参时，随机搜索效率远高于网格搜索。

### 贝叶斯优化
核心思想：将超参优化视为黑盒函数优化，用概率代理模型（通常是高斯过程）建模超参与性能的关系。

工作流程：
1. 先验分布（高斯过程）作为代理模型
2. 采集函数（期望改进 EI）平衡探索与利用，推荐下一个最有潜力的超参组合
3. 评估并更新代理模型
4. 重复

优势：样本效率高（更少试验次数找到更优解）、智能引导搜索方向。

推荐系统中常调优的超参：学习率、Embedding维度、正则化系数、网络层数与节点数、Dropout率、优化器参数。

### 分布式并行超参搜索
- 网格/随机搜索：天然可并行，各组合独立评估（Spark分发到各executor）
- 贝叶斯优化：序列依赖性导致并行困难，可用异步策略（每个worker完成后立即获取新建议）
- 多保真度优化（如SHA/Hyperband）：低资源快速淘汰劣质候选，资源集中于有潜力的配置

### 在线自适应超参优化
- 数据分布漂移导致最优超参变化
- 方案：基于元学习的动态贝叶斯优化（从历史调优任务学习初始化先验）
- Bandit方法：将不同超参值作为"臂"，根据线上反馈实时调整

### 多目标超参搜索
目标可能冲突（CTR vs 多样性），需要帕累托优化：
1. 定义搜索空间和多个评估目标
2. 加权综合得分：`S = w1*Norm(目标A) + w2*Norm(目标B)`
3. 贝叶斯优化迭代寻找帕累托前沿
4. 分布式并行加速评估

### 超参数 vs 模型参数
- 超参数：训练前设定，控制训练过程（学习率、batch size、正则化系数、Embedding维度、网络层数、损失函数选择、优化器参数）
- 模型参数：训练过程中自动学习（权重、偏置、Embedding向量）

## 3.5 因果推断与偏差分析

### 曝光偏差（Exposure Bias）
用户只对系统曝光的物品产生反馈，未曝光物品无交互记录。

对模型训练的影响：
- 学到"因为被曝光所以被点击"的虚假模式
- 反馈循环/马太效应：热门物品获得更多曝光和交互 → 模型更倾向推荐热门 → 长尾物品更难获得曝光
- 模型拟合的是有偏的观测数据分布，而非用户真实兴趣分布

对离线评估的影响：
- 测试集同样有曝光偏差，离线指标可能因推荐结果与历史曝光重合而虚高
- 离线指标与线上A/B结果之间存在鸿沟

缓解方法：
- 训练阶段：逆倾向加权（IPW），按曝光概率倒数加权样本
- 评估阶段：无偏离线评估器（如ESMM全空间多任务模型）
- 系统层面：引入Bandit探索机制，主动曝光长尾/新物品

### 线上好但离线差的因果分析
核心原因：离线评估依赖的日志数据存在严重偏差，无法准确估计因果效应。

具体分析：
1. 曝光偏差：离线数据缺失"如果推荐了其他物品用户会怎样"的反事实信息
2. 混杂效应：用户行为受物品热度、个人偏好等混杂因素影响，离线评估错误归因
3. 数据分布偏移：新策略改变了用户行为模式，离线数据无法覆盖

验证方法：
- 反事实评估：用逆倾向加权在离线模拟随机实验
- 数据切片分析：按用户群体/物品群体对比离线与线上指标
- 增量评估/Interleaving测试：小流量直接观察因果影响

### 倾向得分匹配（PSM）
解决选择偏差的经典方法。将观察性数据"伪随机化"：

步骤：
1. 定义实验组（曝光/推荐的物品-用户对）和对照组
2. 估计倾向得分：用逻辑回归/GBDT预测每个样本被曝光的概率
3. 匹配：为实验组的每个样本在对照组中找倾向得分最接近的样本（最近邻/卡尺/分层匹配）
4. 评估匹配质量：检验匹配后各特征分布是否平衡
5. 效应估计：在平衡数据集上计算因果效应

局限性：只能控制可观测混淆变量；丢弃大量未匹配样本；倾向得分模型本身若有偏，评估结果也有偏。

### 多任务学习中的混淆因素识别
多任务间存在共享混淆因素（如用户购买意愿同时影响点击和购买）。

因果推断方法：
1. 构建因果图识别混淆结构
2. 双重机器学习：两阶段建模，第一阶段估计倾向得分/混淆因素，第二阶段估计因果效应
3. 结构化多任务因果模型：共享混淆因素编码器 + 任务特定因果效应编码器 + 任务输出头
4. 工具变量方法：找到影响干预但不直接影响结果的变量

### 反事实评估（Counterfactual Evaluation）
基于因果推断的离线评估方法。核心思想：不仅看事实（当前策略下的行为），还估计反事实（如果换了推荐内容用户会怎样）。

实现方法：逆倾向得分加权对观测数据进行加权，模拟所有物品都有平等机会被展示的评估环境。

主要挑战：倾向得分估计的准确性、无混淆假设难以完全满足。是离线评估的有力补充，但不能完全替代在线A/B测试。

### 传统Precision/Recall的局限性
三大偏差导致传统指标失真：
- 曝光偏差：数据并非随机缺失，模型"迎合"历史曝光策略
- 选择偏差：用户倾向点击预期喜欢的物品，指标无法区分物品热度 vs 推荐精准性
- 从众偏差：用户更易点击热门物品，导致马太效应

必须结合在线A/B测试、反事实评估、多维度指标（多样性/新颖性/覆盖率/公平性）才能全面评估。

### 无偏评估框架设计
消除曝光偏差和位置偏差：
1. 随机化实验（随机化流量桶）：部分流量随机展示物品，收集无偏反馈。成本高但提供黄金标准
2. 反事实评估 + IPW：为每条记录分配 1/倾向得分 的权重，统计上"弥补"未曝光物品。成本低但依赖倾向得分准确性
3. 双重稳健估计：结合直接建模和IPW，只要其中一个正确，估计就是无偏的

## 3.6 泛化能力与鲁棒性

### 数据分布突变时的应对策略
当用户行为模式突变（如疫情、节日）：

数据与特征层：
- 实时数据监控（CTR/停留时长异常检测）
- 加速实时特征接入（Flink处理最近几分钟行为序列）
- 对抗性特征去相关，减少对可能失效的稳定特征的依赖

模型层：
- 在线学习（FTRL等）：流式更新权重，快速适应新分布
- 持续学习/元学习（MAML）：训练阶段就学会快速适应新任务
- 领域自适应：最小化分布差异或使用对抗训练

系统架构层：
- 模型集成系统：稳定基模型 + 快速适应模型，门控网络动态权衡
- 快速A/B测试平台：新策略小流量快速验证

### 对抗性测试
评估模型鲁棒性的关键手段：
1. 安全性评估：模拟恶意用户（刷单/点击）操纵推荐结果
2. 探索与利用平衡：注入不确定性/噪声，测试冷启动效果
3. 公平性与去偏：构造针对特定用户群体的对抗样本，检测偏见
4. 泛化能力：用对抗样本验证模型在分布轻微变化时的稳定性

测试案例示例（电商排序模型鲁棒性）：
- 对历史只买科技图书的用户，在行为序列中插入少量女装/美妆噪声点击
- 鲁棒模型应不被少数异常信号干扰，仍推荐科技类
- 脆弱模型会"带偏"，开始推荐大量女装

### 泛化能力的重要性
泛化能力 = 模型在未见过的新数据上仍有效的能力。在推荐系统中尤其关键：
- 用户兴趣动态变化，不能只复制历史行为
- 稀疏性和长尾特性：大多数用户/物品的交互数据极少
- 线上环境与离线训练环境存在差异（分布偏移、数据延迟）
- 过拟合导致信息茧房，损害多样性和长期用户留存
