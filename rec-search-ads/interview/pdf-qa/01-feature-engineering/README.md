# 数据预处理与特征工程

## 1.1 数据清洗与缺失值处理

缺失值的本质是信息缺失，处理策略取决于缺失机制：
- MCAR（完全随机缺失）：可直接删除或均值填充
- MAR（随机缺失）：用相关特征建模预测填充
- MNAR（非随机缺失）：缺失本身是信号，需增加"是否缺失"指示特征

推荐系统常用处理方法：
- 连续特征：中位数填充（抗异常值）、分组统计填充（按用户/物品类别）
- 类别特征：单独设"unknown"类别，让模型自行学习语义
- 序列特征：前向填充 / 插值
- 高缺失率特征（>80%）：考虑直接丢弃或仅保留"是否缺失"

异常值检测：箱线法（IQR）、Z-score、业务规则（如负价格、未来时间戳）。处理方式包括截断（winsorize）、分箱、log变换。

## 1.2 特征编码

### One-Hot 编码
将类别特征转为 0/1 向量。适用于低基数（<50）无序类别。高基数时维度爆炸，需配合特征哈希或分桶。

### Label 编码
将类别映射为整数。适用于有序类别（如评分等级），或树模型（GBDT/XGBoost天然处理离散分裂）。不适用于线性模型（会引入虚假序关系）。

### Target 编码
用目标变量的条件统计值替换类别值。例如：某城市的平均CTR作为该城市的编码值。
- 必须做正则化：加性平滑（贝叶斯收缩）、留一法、K折交叉
- 不正则化会严重过拟合（低频类别的统计值噪声大）

### 特征哈希（Feature Hashing）
将高基数类别映射到固定维度的哈希桶。优点：固定内存、支持在线学习。缺点：哈希冲突导致信息损失。

### 对比选择

| 场景 | 推荐编码 |
|------|---------|
| 低基数+线性模型 | One-Hot |
| 高基数+树模型 | Label / Target |
| 高基数+深度模型 | Embedding（端到端学习） |
| 超高基数+在线学习 | 特征哈希 |

## 1.3 特征交叉

核心思想：单特征无法表达的非线性关系，通过组合捕获。例如"性别x年龄段"可区分不同人群偏好。

### 手动交叉
- 笛卡尔积：`city x category`，适用于低基数组合
- 数值x类别：对数值特征按类别分组做统计（如每个城市的平均消费）

### 自动交叉
- FM/FFM：隐向量内积自动学习二阶交叉
- DCN（Deep & Cross Network）：Cross层显式学习高阶交叉
- AutoInt/FiBiNET：注意力机制自动选择重要交叉

实践要点：
- 交叉后特征空间指数增长，必须配合正则化或特征选择
- 深度模型中通常用 Embedding + MLP 隐式交叉，显式交叉作为补充

## 1.4 序列特征提取

用户行为序列（点击、购买、搜索历史）是推荐系统最重要的特征来源之一。

### 滑动窗口统计
- 在不同时间窗口（1h/1d/7d/30d）内统计点击数、购买数、品类分布
- 多窗口对比可反映兴趣趋势（近期 vs 长期）

### 序列建模方法
- Pooling（Mean/Max/Attention）：将变长序列压缩为固定维度向量
- GRU/LSTM：捕获时序依赖，但推理开销大
- Transformer/Self-Attention：并行计算，位置编码引入时序信息（如SASRec）
- DIN（Deep Interest Network）：用目标物品做 Query，对历史行为做 Attention 加权

### 面试关键点
- 序列截断长度的选择：太短丢信息，太长噪声多且计算贵
- 时间衰减：近期行为权重更高，可用指数衰减或时间位置编码
- 多行为类型融合：点击/收藏/购买是不同强度信号，需区分建模

## 1.5 统计聚合特征

### CTR 类统计特征
- 用户历史CTR、物品历史CTR、交叉CTR（用户x品类）
- 必须做平滑：`CTR_smooth = (clicks + alpha * global_ctr) / (impressions + alpha)`
- 必须严格防穿越：只用T时刻之前的数据计算T时刻的特征

### 时间衰减统计
- 指数衰减加权：`score = sum(value_i * exp(-lambda * delta_t_i))`
- 近期行为影响更大，适合捕捉短期兴趣漂移

### 转化率/留存率特征
- 类似CTR计算方式，但目标事件不同
- 转化漏斗各环节的转化率都可作为特征

### 实践要点
- 统计窗口选择影响巨大：短窗口敏感但波动大，长窗口稳定但滞后
- 低频实体的统计特征噪声极大，必须做贝叶斯平滑
- 实时特征 vs 离线特征：实时统计能捕捉突发热点，但工程复杂度高

## 1.6 Embedding 在特征工程中的应用

### Word2Vec / Item2Vec
- 将用户行为序列类比为"句子"，物品类比为"词"
- Skip-gram / CBOW 在行为序列上训练，学到物品的低维稠密表示
- 核心假设：共现 = 相似（在同一session中出现的物品语义接近）

### Graph Embedding
- DeepWalk/Node2Vec：在用户-物品交互图上随机游走生成序列，再用Word2Vec训练
- GraphSAGE：聚合邻居特征，归纳式学习（支持新节点）
- PinSAGE：在Pinterest十亿级图上的工业实践，采样+聚合+负采样训练

### 多模态 Embedding
- 文本：BERT/Sentence-BERT 提取标题/描述的语义表示
- 图像：ResNet/ViT 提取视觉特征
- 多模态融合：拼接、门控融合、对比学习（CLIP思路）

### 在线学习 Embedding
- 用户兴趣实时变化，Embedding需持续更新
- 增量训练：只在新数据上微调，而非全量重训
- 特征服务：Redis/Cassandra缓存最新Embedding，实时特征注入

## 1.7 大规模特征工程 Pipeline

### 离线 vs 实时特征

| 维度 | 离线特征 | 实时特征 |
|------|---------|---------|
| 时效性 | T+1（小时/天级） | 秒级/分钟级 |
| 计算框架 | Spark/Hive | Flink/Kafka Streams |
| 典型特征 | 用户画像、物品统计 | 实时CTR、近N次点击 |
| 存储 | HDFS/Hive | Redis/Cassandra |

### 特征监控与漂移检测
- PSI（Population Stability Index）：衡量特征分布在时间上的稳定性。PSI > 0.25 视为显著漂移
- KS检验：检测两个分布是否显著不同
- 缺失率/零值率监控：突变通常意味着上游数据管道问题

### 特征存储（Feature Store）
- Feast/Tecton/自建方案
- 核心能力：特征注册、版本管理、point-in-time join（防穿越）、在线/离线一致性

### 特征选择与降维
- 过滤法：互信息、卡方检验、方差阈值
- 包装法：递归特征消除（RFE）
- 嵌入法：L1正则化、树模型特征重要性
- 降维：PCA（线性）、AutoEncoder（非线性）

### 面试关键点
- 特征穿越是最严重的工程错误：用了未来信息导致离线指标虚高
- 在线/离线一致性：训练用离线特征，推理用在线特征，不一致会导致 training-serving skew
- 特征上线流程：开发 -> 离线验证 -> 小流量AB -> 全量
