# 图嵌入技术详解

## 核心思想

图嵌入将图结构数据中的节点（或边）映射为低维稠密向量，目标是在向量空间中保留原始图的结构特性：一阶相似性（直接连接）和高阶相似性（邻居社区的相似性）。

推荐系统中，用户和物品天然构成交互二部图，图嵌入能捕获序列模型无法捕获的高阶关联和全局结构信息。

---

## 方法详细对比

### DeepWalk

```
核心思想: 随机游走 + Skip-gram
流程:
  1. 在图上从每个节点出发，进行多次均匀随机游走
  2. 每次等概率选择一个邻居作为下一步
  3. 生成的节点序列视为"句子"
  4. 用 Skip-gram 模型训练节点 Embedding

特点:
  - 均匀随机游走，无法控制探索方向
  - 简单高效，适合同质图
  - 隐式假设图是无向、无权的
  - 主要捕获社区结构(同质性)
```

### Node2Vec

```
核心改进: 引入 p/q 参数控制有偏随机游走

参数 p（返回参数 / Return Parameter）:
  - 控制回到上一节点的概率
  - p 小 → 倾向回到上一节点 → 局部探索 → BFS 行为
  - p 大 → 不倾向回头 → 向外探索

参数 q（进出参数 / In-Out Parameter）:
  - 控制向远离源节点方向前进的概率
  - q 小 → 倾向远离源节点 → DFS 行为 → 探索结构等价性
  - q 大 → 倾向留在源节点附近 → BFS 行为 → 探索同质性

转移概率公式（从节点 t 经 v 到 x）:
  alpha(t, x) = 1/p    if d(t,x) = 0  (x 就是 t，即回头)
  alpha(t, x) = 1      if d(t,x) = 1  (x 是 t 的邻居)
  alpha(t, x) = 1/q    if d(t,x) = 2  (x 远离 t)

  P(x|v) = alpha(t,x) * w(v,x) / Z
```

### LINE (Large-scale Information Network Embedding)

```
核心思想: 显式优化一阶和二阶相似性

一阶相似性 (直接连接的节点):
  - 优化目标: 最小化联合概率分布和经验分布的 KL 散度
  - p1(v_i, v_j) = sigmoid(u_i^T * u_j)
  - 直接连接的节点 Embedding 应接近

二阶相似性 (共享邻居的节点):
  - 有相似邻居分布的节点应有相似表示
  - p2(v_j | v_i) = exp(u'_j * u_i) / sum_k exp(u'_k * u_i)
  - 优化条件概率分布

训练: 分别训练一阶和二阶，最后拼接
优势: 可扩展到百万节点级图
局限: 只考虑一阶和二阶，缺少更高阶结构信息
```

### GraphSAGE (SAmple and aggreGatE)

```
核心思想: 采样邻居 + 聚合特征，归纳式学习

框架:
  1. 采样: 对每个节点，每层采样固定 K 个邻居
  2. 聚合: 用聚合函数整合邻居特征
  3. 更新: 拼接自身特征和邻居聚合特征，通过非线性变换

聚合函数选择:
  Mean:    AGG = mean(h_u, u in N(v))
  LSTM:    AGG = LSTM(random_permutation(h_u))
  Pool:    AGG = max(sigma(W * h_u + b))

关键优势:
  - 归纳式学习: 训练好的聚合函数可为未见过的新节点生成 Embedding
  - 解决冷启动: 新物品只需有邻居连接即可生成向量
  - 扩展性好: 邻居采样控制计算量

每层采样数量: 通常 K=10-25，2-3层聚合
```

### Metapath2Vec

```
核心思想: 为异构图设计，用元路径指导随机游走

异构图: 包含多种类型节点和多种类型边
  例: 用户-物品-品牌-品类

元路径 (Metapath):
  定义: 节点类型的序列，如 用户-物品-品牌-物品-用户
  作用: 指导随机游走只沿指定类型序列进行
  不同元路径捕获不同语义关系:
    U-I-U: 用户通过共同购买物品的关联
    I-B-I: 物品通过同品牌的关联
    U-I-C-I-U: 用户通过同品类物品的关联

训练:
  1. 按元路径进行随机游走，生成节点序列
  2. 用 Skip-gram + 异构负采样训练

优势: 能有效融合异构信息，捕获不同类型关系
局限: 元路径需要领域知识人工设计
```

### LightGCN

```
核心思想: 简化 GCN，只保留邻居聚合

标准 GCN: 聚合 + 特征变换 + 非线性激活
LightGCN: 只保留加权平均聚合

更新规则:
  e_u^(k+1) = sum_{i in N(u)} (1/sqrt(|N(u)|*|N(i)|)) * e_i^(k)
  e_i^(k+1) = sum_{u in N(i)} (1/sqrt(|N(i)|*|N(u)|)) * e_u^(k)

最终表示: 各层加权平均
  e_u = sum_{k=0}^{K} alpha_k * e_u^(k)
  alpha_k 可均匀或可学习

优势:
  - 去掉特征变换和非线性后，推荐任务性能反而提升
  - 计算量大幅减少（数量级加速）
  - 参数量少，过拟合风险低
```

## 方法全景对比

```
方法          输入图类型    核心技术          能否处理新节点  计算复杂度
──────────────────────────────────────────────────────────────────────
DeepWalk     同质无权图    均匀随机游走       否(直推式)     中等
Node2Vec     同质有权图    有偏随机游走       否(直推式)     中等
LINE         同质有权图    一阶+二阶显式优化  否(直推式)     较低
GraphSAGE    同质/异构     邻居采样+聚合      是(归纳式)     可控
Metapath2Vec 异构图        元路径游走         否(直推式)     中等
LightGCN     二部图        简化GCN聚合        否(直推式)     较低
```

```
方法          适用场景                        冷启动能力
──────────────────────────────────────────────────────────────────────
DeepWalk     简单社交网络、小规模图           无
Node2Vec     需要同时捕获局部和全局结构       无
LINE         大规模网络嵌入                   无
GraphSAGE    动态图、冷启动严重的推荐场景     强（归纳学习）
Metapath2Vec 异构信息丰富（电商知识图谱）     弱
LightGCN     用户-物品推荐二部图              无
```

## 推荐系统中的图构建

### 节点类型

```
核心节点: 用户、物品
属性节点: 品牌、品类、店铺、标签
内容节点: 关键词、主题、演员、导演
上下文节点: 时间戳(工作日/周末)、地理位置(商场/住宅区)
```

### 边的设计

```
边类型          权重设计                    信号含义
──────────────────────────────────────────────────────────
用户-物品       行为类型加权                交互强度
(点击/购买/收藏) 点击=1,收藏=3,购买=5
                + 时间衰减 exp(-lambda*dt)

用户-用户       社交关系(关注/好友)         同质性假设
               相似用户连接                 兴趣传播

物品-物品       共现频次                    互补/替代关系
               同品牌/品类                  属性关联

物品-属性       belong_to 关系              属性标注
```

### 异构图的信息融合优势

```
1. 间接关联传播:
   用户A → 购买物品X → 属于品牌B → 物品Y
   → 用户A的Embedding间接受品牌B影响
   → 能学到更细粒度的品牌偏好

2. 冷启动增强:
   新文章 → 包含关键词K → 已有文章Z → 被用户C喜欢
   → 新文章通过关键词节点获得初始Embedding

3. 上下文感知:
   用户 → 工作日 → 科技文章
   用户 → 周末 → 娱乐视频
   → 捕获动态和上下文相关的偏好
```

## 大规模图嵌入工程优化

```
挑战                          解决方案
──────────────────────────────────────────────────────────────
邻接表/矩阵内存超限            图分区(METIS) + 分布式存储
随机游走序列生成慢              多线程/分布式并行生成
Softmax 计算 O(|V|)            负采样 → O(K)
GNN 全图邻居聚合不可行          GraphSAGE 固定采样 K 个邻居
训练效率低                      数据并行 + 参数服务器
                               PyTorch Geometric / DGL + GPU
动态图频繁变化                  增量学习 / 时序图神经网络
```

## 面试高频问题

```
Q1: DeepWalk 和 Node2Vec 的核心区别？
A: DeepWalk 用均匀随机游走，Node2Vec 引入 p/q 参数控制有偏游走。
   p 控制回头概率(BFS)，q 控制远离概率(DFS)。
   Node2Vec 能同时捕获同质性和结构等价性。

Q2: GraphSAGE 为什么能处理冷启动？
A: GraphSAGE 学习的是聚合函数（参数化的），不是节点 Embedding。
   新节点只要有邻居连接，就能通过聚合函数从邻居特征计算出 Embedding。
   这是归纳式学习 vs 直推式学习的本质区别。

Q3: 元路径需要人工设计，有什么替代方案？
A: 1. 注意力机制自动学习元路径权重(HAN)
   2. 图注意力网络(GAT)为不同邻居边分配不同权重
   3. 异构图 Transformer 自动发现重要关系

Q4: 为什么 LightGCN 去掉非线性后效果反而更好？
A: 推荐场景中用户-物品交互图的特征主要是 ID Embedding（本身随机初始化）。
   对这类特征做非线性变换没有额外信息增益，反而引入噪声和过拟合。
   简化后参数少、训练稳定、泛化更好。

Q5: 图嵌入 vs 序列 Embedding 怎么选？
A: 看数据特点：
   - 有明确时序行为序列 → Item2Vec / SASRec
   - 有丰富关系网络(社交/知识图谱) → 图嵌入
   - 实践中常融合：Item2Vec + GraphSAGE 拼接作为最终向量
```
