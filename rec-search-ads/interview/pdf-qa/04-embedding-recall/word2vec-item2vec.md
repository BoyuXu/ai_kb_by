# Word2Vec 原理与 Item2Vec 应用

## 核心思想

Word2Vec 基于分布式假设：一个词的语义由其上下文决定。通过训练浅层神经网络，将离散词映射到低维稠密向量空间，使语义相似的词在向量空间中距离更近。

Item2Vec 是 Word2Vec 在推荐系统中的直接迁移：用户行为序列类比句子，物品类比单词，通过物品的共现关系学习物品的向量表示。

---

## Skip-gram vs CBOW 详解

```
Skip-gram:
  输入: 中心词 w_t
  输出: 预测上下文词 w_{t-c}, ..., w_{t+c}
  目标: max sum_{-c<=j<=c, j!=0} log P(w_{t+j} | w_t)
  特点: 每个中心词生成多个训练样本

CBOW (Continuous Bag of Words):
  输入: 上下文词的向量平均 (1/2c) * sum(v_{t+j})
  输出: 预测中心词 w_t
  目标: max log P(w_t | context)
  特点: 上下文聚合后预测一次
```

### 对比

```
                Skip-gram            CBOW
──────────────────────────────────────────────────
训练速度        较慢(样本多)         较快(聚合后预测)
低频词表现      好(每个词都做中心词)  差(被上下文淹没)
数据稀疏场景    更适合               不适合
推荐系统首选    是                   否
计算复杂度      O(C * |V|)           O(|V|)
```

### 推荐系统更倾向 Skip-gram 的三个原因

1. 稀有物品表现更好：长尾物品出现频率低，Skip-gram 每次训练一个(中心词, 上下文词)对，低频物品也有多次被训练的机会
2. 行为序列通常较短（比自然语言句子短），Skip-gram 能从有限共现关系中提取更多有效训练样本
3. Skip-gram 的训练目标（让具有相似上下文的物品靠近）天然契合物品相似度发现

## 训练加速技术

### 负采样 (Negative Sampling)

```
原始 Softmax:
  P(w_o | w_i) = exp(v'_o * v_i) / sum_j exp(v'_j * v_i)
  计算复杂度: O(|V|)，词表/物品库上亿时不可行

负采样优化:
  对每个正样本 (w_i, w_o)，随机采 K 个负样本
  目标函数简化为:
  L = log sigma(v'_o * v_i) + sum_{k=1}^{K} E[log sigma(-v'_k * v_i)]

  采样分布: P(w) = f(w)^{3/4} / sum f(w)^{3/4}
  3/4次幂: 平滑频率分布，提升低频词被采中概率

  计算复杂度: O(K)，K 通常取 5-20
```

### 层次 Softmax (Hierarchical Softmax)

```
用哈夫曼树替代全量 Softmax:
  - 高频词靠近根节点，路径短
  - 低频词路径长
  - 每个节点做二分类
  计算复杂度: O(log|V|)

实践选择:
  - 词表/物品库较小 → 层次 Softmax
  - 词表/物品库很大 + 低频词重要 → 负采样（推荐系统首选）
```

## Item2Vec 行为序列构建

### 四种序列构建方法

```
方法              原理                      优点              缺点
──────────────────────────────────────────────────────────────────────
会话序列          30min超时切分会话          关联性强/噪声少   依赖会话划分准确性
                  会话内物品按时间排列                        忽略长期兴趣

全历史序列        用户所有行为按时间拼接      捕获长期+周期兴趣  含大量噪声物品
                                                            兴趣漂移导致序列
                                                            内部语义不一致

滑动窗口          固定大小窗口在行为流上滑动  缓解数据稀疏      大量重叠序列/冗余
                  窗口内物品构成序列          平衡短期和长期    窗口大小是超参

业务逻辑序列      加购->下单->支付路径        学习转化导向的    覆盖行为有限
                  同品类浏览路径              Embedding
```

### 数据清洗要点

```
构建序列前必须做:
1. 过滤低频物品（出现次数<阈值），否则 Embedding 质量差
2. 过滤非活跃用户（行为太少），否则序列无意义
3. 噪声行为处理：
   - 停留时长 < 阈值 → 过滤（无意点击）
   - 交互深度加权：点击=1, 收藏=2, 加购=3, 购买=5
4. 时效性控制：过老的行为可能已不代表当前兴趣
```

## Item2Vec 的应用场景

```
1. 相似物品推荐 (I2I):
   - 计算物品 Embedding 的余弦相似度
   - KNN 找 Top-K 相似物品
   - 应用：商品详情页"猜你喜欢"

2. 向量召回 (U2I):
   - 用户向量 = 近期交互物品 Embedding 的平均池化
   - 用用户向量在物品向量库中 ANN 检索 Top-K
   - 从亿级物品库筛选到百/千级候选集

3. 排序特征:
   - 用户-物品 Embedding 余弦相似度作为精排模型输入特征
   - 物品-物品相似度作为上下文特征
```

## Item2Vec 的局限性与演进

```
局限性                     影响                      解决方向
────────────────────────────────────────────────────────────────
仅用 ID 信息               冷启动差                  EGES(图+属性)
                                                    多模态 Embedding

忽略序列顺序               无法建模兴趣演化          BERT4Rec, SASRec
(Skip-gram 无序)                                    GRU4Rec

静态 Embedding             不随用户实时兴趣变化       在线学习
                                                    流式 Skip-gram

只捕获共现关系             无法利用图结构高阶关系     Node2Vec, GraphSAGE
(一阶相似性)
```

### Item2Vec vs 矩阵分解

```
Item2Vec 和 SVD++ 在数学形式上有等价性:
  - 都在隐式地分解物品共现矩阵
  - Item2Vec 基于局部序列的共现（局部窗口）
  - SVD 基于全局共现矩阵

差异: Item2Vec 通过序列的灵活性，能更好处理:
  - 物品出现的位置关系
  - 灵活的正样本定义（窗口/会话）
  - 大规模在线增量训练
```

## 面试高频问题

```
Q1: Word2Vec 的分布式假设是什么？
A: 一个词的语义由其上下文决定。相似上下文中出现的词，语义也相似。

Q2: 为什么推荐系统更用 Skip-gram？
A: 三点：低频物品表现好、短序列提取更多样本、训练目标契合相似物品发现。

Q3: 负采样的采样分布为什么用 3/4 次幂？
A: 平滑频率分布。纯频率采样高频词被采太多，均匀采样又忽略频率信息。
   3/4 次幂是经验最优的折中。

Q4: Item2Vec 的冷启动怎么解决？
A: 四种方案：
   1. 利用侧信息(类别/品牌/文本)通过 Embedding 编码初始化
   2. 图神经网络(GNN)从邻居聚合信息
   3. 元学习(MAML)学习好的初始化参数
   4. 非 Embedding 补充通道(热门/规则召回)

Q5: Item2Vec 和协同过滤的关系？
A: Item2Vec 本质是基于行为序列的协同过滤的 Embedding 化。
   序列中共现的物品 = 协同过滤的共现信号。
   优势是产出稠密向量，支持 ANN 高效检索。
```

## 工程实践要点

```
1. Embedding 维度选择: 64-256 维，物品库越大维度可越高
2. 窗口大小: 电商推荐通常 5-10，视频推荐可更大
3. 负采样数 K: 通常 5-20，物品库越大 K 可适当增大
4. 训练频率: 天级全量重训 + 小时级增量更新
5. 向量归一化: 用余弦相似度时要做 L2 归一化
6. 评估指标: 离线用相似度相关性/召回率，线上用 AB 测试看 CTR/CVR
```
