# 双塔模型架构与工业实践

## 核心概念

双塔模型（Two-Tower / Dual-Encoder）是向量召回阶段的经典架构。用户和物品通过两个独立的深度神经网络分别编码到同一向量空间，通过向量相似度（内积/余弦）进行匹配。

核心优势：用户塔和物品塔解耦，物品向量可离线预计算并构建 ANN 索引，在线只需实时计算用户向量，实现毫秒级召回。

---

## 架构详解

```
用户塔 (User Tower)                物品塔 (Item Tower)
──────────────────                ──────────────────
用户ID Embedding                  物品ID Embedding
+ 用户画像特征                    + 物品属性特征
+ 行为序列特征                    + 内容特征(文本/图像)
+ 上下文特征(时间/位置)
    |                                 |
    v                                 v
  MLP / Transformer               MLP / Transformer
    |                                 |
    v                                 v
  用户向量 u (d维)                 物品向量 v (d维)
    |                                 |
    +-------- score = u^T * v --------+
```

### 典型代表：YouTube DNN

```
用户塔输入:
  - 用户观看历史的视频 Embedding 平均池化
  - 搜索历史的 token Embedding 平均池化
  - 用户画像: 地理位置、设备、性别、年龄
  - 上下文: 距上次观看时间

通过多层 ReLU 全连接层 → 用户向量
物品塔: 视频 Embedding (训练阶段用 Softmax 分类)
在线服务: 用户向量 ANN 检索最近邻视频
```

## 训练技巧

### 损失函数选择

```
1. Softmax + 交叉熵 (最常用):
   L = -log( exp(u^T * v+ / tau) / sum_j exp(u^T * v_j / tau) )

   分母包含: 1个正样本 + batch内其他样本作为负样本(in-batch negatives)

2. BPR (Bayesian Personalized Ranking):
   L = -log sigma(u^T * v+ - u^T * v-)
   成对排序，只要正样本得分高于负样本

3. Triplet Loss:
   L = max(0, d(u, v+) - d(u, v-) + margin)
   三元组，显式控制正负样本间距
```

### 温度系数 (Temperature)

```
score = u^T * v / tau

tau 的作用:
  - tau 大 → 分布平滑 → 更关注整体排序 → 训练更稳定
  - tau 小 → 分布尖锐 → 更关注 Top 区分度 → 但容易过拟合

实践:
  - 通常 tau = 0.05 ~ 0.1
  - 可设为可学习参数
  - 训练初期用大 tau 稳定训练，后期调小提升区分度
```

### 负样本构造策略

```
策略                  原理                        效果
──────────────────────────────────────────────────────────────
随机负采样            从全局物品池随机采样          简单但大量"易负例"
                                                  模型区分度不足

In-batch Negatives    同 batch 内其他正样本的       高效利用计算
                      物品作为负样本                但受 batch 大小限制

基于曝光的负采样      曝光但未点击的物品            更接近真实场景
                                                  但有曝光偏差

困难负样本挖掘        选择模型当前得分高但          大幅提升模型能力
(Hard Negative)       实际不相关的物品              训练不稳定风险

混合策略(工业首选)   随机(50%) + 困难(30%)       兼顾稳定性和区分度
                      + In-batch(20%)
```

### 困难负样本挖掘方法

```
1. 离线挖掘:
   - 用上一版本模型对全量物品打分
   - 取得分高但非正样本的物品作为困难负例
   - 每个 epoch 更新一次

2. 在线挖掘:
   - 当前 batch 中得分最高的负样本
   - 需要 gradient detach 避免训练崩溃

3. 基于 ANN 的挖掘:
   - 在 ANN 索引中检索 Top-K 近邻
   - 非正样本的近邻作为困难负例

注意: 困难负样本比例不宜过高(30%以内)，否则训练不稳定
```

## 双塔模型的局限与改进

### 核心局限：交互信息损失

```
问题: 用户和物品分别独立编码，无法建模交叉特征
  例: "用户A对品牌X的偏好" 这种交叉信号无法在双塔中表达

改进方向:
1. 后期交互 (Late Interaction):
   - 输出多个向量而非单一向量
   - 用 MaxSim 等方式计算相似度
   - 代表: ColBERT (NLP检索)

2. 交叉特征注入:
   - 在用户塔中注入目标物品的部分特征(如品类)
   - 限制: 在线服务时需要枚举品类

3. 多向量表示:
   - 用户用多个兴趣向量表示(如 MIND 模型的胶囊网络)
   - 对每个兴趣向量分别检索再合并
```

### 与其他召回模型的关系

```
模型              本质                    优势              局限
──────────────────────────────────────────────────────────────
双塔 DSSM         双编码器+内积           高效/可ANN检索    无交叉特征
YouTube DNN       深度双塔                融合丰富特征      内积空间限制
MIND              多兴趣双塔              多兴趣表达        向量数量选择
TDM               树索引+模型             精确建模交互      树结构维护复杂
```

## 工业实践

### 在线服务架构

```
离线:
  1. 训练双塔模型
  2. 物品塔对全量物品编码 → 物品向量集合
  3. 构建 ANN 索引 (HNSW/IVF-PQ)
  4. 天级更新索引，发布到线上

在线:
  1. 获取用户实时特征
  2. 用户塔计算用户向量 (< 5ms)
  3. ANN 检索 Top-K 物品 (< 5ms)
  4. 融合其他召回通道结果
  5. 送入粗排/精排
```

### 在线更新策略

```
基于增量学习的双塔在线更新:

1. 实时样本构造:
   - 监听用户实时行为流(Kafka/Flink)
   - 新交互 → 构造(用户,物品)正样本对
   - 从物品池随机采样负样本

2. 增量模型更新:
   - 在线梯度下降(SGD/Adam)
   - 模型参数增量式更新，非全量重训
   - 关键: 学习率要小，防止灾难性遗忘

3. 向量服务同步:
   - 更新后的用户/物品 Embedding 实时同步到向量数据库
   - 新物品直接加入 ANN 索引(增量更新)

4. 闭环反馈:
   新行为 → 更新样本 → 模型更新 → 向量更新 → 影响下次召回
```

### 多路召回融合

```
向量召回在线上通常是多路召回之一:

召回通道            特点                    互补价值
──────────────────────────────────────────────────────
双塔向量召回        个性化、语义匹配         主通道
热门召回            覆盖热点内容             补充流行趋势
协同过滤召回        利用行为相似性           捕获社群偏好
标签/分类召回       基于用户兴趣标签         确保品类多样性
新品召回            规则触发新物品曝光       解决冷启动

融合方式:
  - 各通道独立召回 Top-K → 并集 → 去重 → 送入粗排
  - 各通道比例可通过在线学习或 AB 测试优化
```

## 面试高频问题

```
Q1: 双塔模型为什么适合召回阶段？
A: 用户塔和物品塔解耦，物品向量可以离线预计算并建 ANN 索引。
   在线只需计算用户向量 + ANN 检索，可在毫秒级从亿级物品库
   筛选出千级候选。精排模型(交叉特征)做不到这个效率。

Q2: In-batch Negatives 有什么问题？
A: 热门物品在 batch 中出现频率高，导致被过多地当作负样本，
   产生"热门惩罚"效应。解决方案: 根据物品频率做 logQ 校正。

Q3: 双塔模型如何处理用户兴趣多样性？
A: 1. MIND模型: 用胶囊网络/聚类提取多个兴趣向量
   2. 对每个兴趣向量分别 ANN 检索
   3. 合并去重后送入粗排
   4. 也可在单向量基础上融合 MMR 多样性策略

Q4: 温度系数 tau 怎么调？
A: tau 控制 Softmax 分布的尖锐程度。
   tau 太大 → 分布太平均，模型学不到区分度
   tau 太小 → 分布太尖锐，训练不稳定，过拟合
   通常从 0.1 开始，在 0.05-0.2 范围网格搜索。
   也可设为可学习参数。

Q5: 向量召回效果下降怎么排查？
A: 五步排查法:
   1. 服务层: 检查延迟/错误率/超时
   2. 索引层: 检查版本/参数(topK/nprobe)
   3. 模型层: 离线评估指标是否下降
   4. 数据层: 特征漂移/缺失/分布变化
   5. 实验层: AB分流逻辑/指标计算
```
