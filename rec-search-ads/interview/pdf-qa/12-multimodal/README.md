## 核心概念

多模态推荐的价值：利用物品的多种信息（图像、文本、视频、音频）丰富表征维度，弥补纯 ID 协同过滤的信息稀疏问题。

三大核心收益：
- 提升准确性：多维信息互补，理解物品本质更深入
- 解决冷启动：新物品无交互数据，但有丰富的内容特征
- 增强可解释性：可以通过模态注意力解释推荐理由

## 特征提取

### 文本特征
- Word2Vec/GloVe：词级别静态嵌入，简单高效
- BERT/Sentence-BERT：上下文感知嵌入，取 [CLS] 或均值池化，768 维
- 预训练模型直接提取 or 在推荐任务上微调

### 图像特征
- ResNet/VGG：去掉分类层，取全局平均池化后的特征，512/2048 维
- Vision Transformer (ViT)：Patch 分块 + 自注意力，捕捉全局结构
- 预训练权重（ImageNet）作为起点

### 视频特征
- 3D CNN (C3D, SlowFast)：捕捉时空特征
- 双流网络：空间流(RGB帧) + 时间流(光流)
- 抽取关键帧 + 图像模型，降低计算量

### 音频特征
- 梅尔频谱 + CNN
- 适用于音乐/视频推荐

### 特征归一化
- 各模态特征 L2 归一化到单位球面
- 防止某个模态因范数大而主导融合

## 融合策略

### 早期融合（数据级）
- 在特征层面直接拼接各模态特征向量
- 优点：简单，模态间充分交互
- 缺点：要求特征维度对齐、同步可用，噪声易干扰

### 晚期融合（决策级）
- 各模态独立建模出预测分数，最后加权/投票/元学习器整合
- 优点：模块化，易扩展，模态缺失时屏蔽即可
- 缺点：丢失早期细粒度交互信息

### 中期融合（模型级）
- 在模型中间层进行跨模态交互（当前主流）
- 典型技术：
  - 特征拼接 + 投影：最基础
  - 双线性融合：外积捕捉交互，但计算量大
  - 跨模态注意力：一个模态动态聚焦另一个模态的相关部分
  - 专用融合网络：多层感知机/Transformer 学习融合函数
- 优点：平衡了交互深度和灵活性

### 混合融合
- 不同阶段用不同策略：召回用晚期融合保证效率，排序用中期融合追求精度
- 工业界最常见

## 跨模态注意力机制

核心思路：让一个模态动态地关注另一个模态中与之最相关的部分。

### 实现方式
- Query 来自模态 A，Key/Value 来自模态 B
- attention = softmax(Q_A * K_B^T / sqrt(d)) * V_B
- 双向：同时计算 A->B 和 B->A
- 输出：图像增强的文本特征 + 文本增强的图像特征

### 优点
- 动态适应性：根据用户和上下文自动调整模态权重
- 可解释性：注意力权重可视化展示模型关注了哪些区域/词
- 模态缺失鲁棒：缺失模态的注意力权重自动趋近零

### 缺点
- 计算成本高，增加推理延迟
- 需要充分的多模态训练数据

## 语义鸿沟与模态对齐

### 问题定义
不同模态的特征空间分布不一致（如图像特征和文本特征在不同语义空间），直接融合效果差。

### 对齐方法

#### 对比学习（核心方法）
- 正样本对：同一物品的图像-文本对
- 负样本对：不同物品的图像-文本对
- InfoNCE 损失：拉近正对，推远负对
- 结果：图像和文本特征映射到共享语义空间

#### 跨模态投影
- 线性投影层将各模态映射到同一维度空间
- 简单但可能损失模态特有信息

#### 跨模态 Transformer
- ViLBERT / LXMERT：双流编码器 + 交叉注意力层
- 单流结构：图像 patch + 文本 token 拼接输入统一 Transformer
- 在深层语义层面实现对齐与融合

## CLIP/BLIP 在推荐中的应用

### CLIP 核心
- 图像编码器 + 文本编码器，对比学习训练
- 4 亿图文对预训练，通用语义理解能力强
- 输出：图像和文本在同一空间的 Embedding

### 推荐应用
- 多模态召回：CLIP Embedding -> FAISS/HNSW 向量检索
- 排序特征：CLIP 相似度作为排序模型的输入特征
- 冷启动：新物品通过 CLIP 直接获得高质量 Embedding
- 零样本推荐：无需微调即可计算用户查询与物品的相似度

### 挑战
- CLIP 在公开数据集训练，语义空间与推荐业务有鸿沟
- 解决：用业务数据微调 CLIP，或加一层适配网络

## 模态缺失处理

### 问题
现实中大量物品模态不完整（无图片、无描述文本），直接丢弃或填零都不理想。

### 方案一：模态缺失指示符 + 默认向量
- 每个模态引入二值指示特征（0/1 标记是否存在）
- 缺失时用可学习的默认向量替代
- 模型显式学习缺失模式

### 方案二：共享潜空间生成式建模
- 所有模态编码到共享潜空间
- 训练时加入跨模态重构损失（如用文本潜表示重构图像）
- 推理时缺失模态从已有模态在潜空间中生成
- 优点：生成的表示捕捉了模态间的语义关联

### 方案三：模态门控网络
- 门控机制根据数据质量动态调整各模态权重
- 低质量/缺失模态自动获得低权重
- 类似 MMoE 中 Gate 的作用

### 方案四：训练时随机模态丢弃
- Dropout 思路：训练时随机遮蔽某些模态
- 迫使模型学习对模态缺失的鲁棒性

## 序列建模 + 多模态

### 多模态序列兴趣网络
核心：将用户行为序列中每个物品用多模态 Embedding 表示。

架构：
1. 商品编码层：图像(ResNet/ViT) + 文本(BERT) -> 融合得商品多模态嵌入
2. 序列建模层：Transformer 编码器处理多模态嵌入序列
3. 兴趣提取层：目标注意力(Target Attention)，用候选商品查询序列
4. 预测层：用户兴趣向量 + 候选商品向量 -> 点击概率

### 双通道序列模型
- 文本通道：搜索词/评论序列 -> BERT -> 文本序列
- 图像通道：商品图/视频封面序列 -> ViT -> 视觉序列
- 跨模态注意力对齐后拼接，输入共享 Transformer 编码器

### 时间衰减
- 注意力权重中引入时间衰减掩码
- 近期行为权重更高
- 处理兴趣漂移

## 工业实践要点

### 系统架构
- 离线：多模态预训练模型提取 Embedding -> 存入向量数据库
- 在线召回：FAISS/HNSW 向量检索
- 在线排序：将多模态 Embedding 作为特征输入排序模型
- 推理效率：MobileNet/DistilBERT 替代重型模型

### 实时多模态特征管道
- Kafka 接收新物品/用户行为
- Flink 调用轻量 GPU 推理提取特征
- 写入 Feature Store / 向量数据库

### 效果评估
- 消融实验：逐个去除模态验证贡献
- 注意力权重可视化：哪个模态在什么场景权重高
- 严格 AB 测试：不能只看离线指标
- 分场景：服饰类视觉权重大 vs 书籍类文本权重大

### 效率与实时性权衡
- 离线预计算物品多模态向量（静态部分）
- 在线仅计算用户向量 + 相似度
- 复杂融合模型通过知识蒸馏压缩到轻量学生模型

## 前沿趋势

- 大模型统一提取：多模态大模型直接生成融合表征，简化手动融合设计
- 生成式推荐：利用扩散模型/LLM 生成推荐及解释
- GNN + 多模态：用户-物品交互图 + 物品多模态属性图联合建模
- 自监督预训练：对比学习作为预训练任务，下游推荐微调

## 面试高频考点

1. 三种融合策略对比：早期(简单但噪声)、晚期(模块化但丢交互)、中期(主流)
2. 跨模态注意力怎么做：Q 来自一个模态，KV 来自另一个模态
3. 模态缺失如何处理：默认向量+指示符 / 共享潜空间生成 / 门控网络
4. CLIP 在推荐中的用法：多模态召回 + 排序特征 + 冷启动
5. 语义鸿沟如何解决：对比学习拉齐 + 跨模态投影 + 业务数据微调
6. 多模态序列模型设计：多模态嵌入 -> 序列编码(Transformer) -> 目标注意力
7. 效率问题：离线预计算 + 知识蒸馏 + 轻量提取器
