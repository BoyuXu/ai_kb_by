# 在线学习与模型更新

## 1. 在线学习 vs 批量学习

### 1.1 核心区别

```
              在线学习                    批量学习
数据使用      逐样本/小批量增量更新         积累全量数据后训练
更新频率      秒级/分钟级                  天级/周级
实时性        极高，快速捕捉兴趣变化       延迟高，无法反馈最新行为
资源消耗      增量更新开销小               全量训练计算量大
模型稳定性    可能不稳定，受噪声影响       训练充分，模型稳定
架构复杂度    极高（流处理+特征+模型服务）  成熟稳定
数据漂移      天生适应，持续跟随分布变化    滞后响应，等下次训练周期
```

### 1.2 工业界混合策略（Nearline Learning）

纯在线或纯批量都不理想，工业界主流是混合方案：
```
离线全量训练（每天/每周）：
  → 用全部历史数据训练强基准模型
  → 保证模型全局准确性

在线增量更新（分钟/小时级）：
  → 在基准模型上做快速微调
  → 捕捉实时兴趣变化和热点趋势

定期基线重置：
  → 将离线模型作为新的在线学习起点
  → 防止增量更新积累的偏差
```

## 2. FTRL 算法详解

### 2.1 核心思想

FTRL（Follow The Regularized Leader）是工业界在线学习的事实标准。

```
核心：每步优化 = 累积损失 + 正则项

权重更新公式：
  w_{t+1} = argmin_w { sum_{s=1}^{t} g_s * w + R(w) }

其中 R(w) = lambda1 * ||w||_1 + lambda2/2 * ||w||_2^2
```

### 2.2 FTRL 的优势

1. 高效处理稀疏数据：
   - 推荐系统特征空间极其庞大且稀疏
   - FTRL 的 L1 正则产生稀疏解，大量不重要特征权重压缩为零
   - 极大减少模型体积，提升推理速度

2. 支持大规模特征自动筛选：
   - 训练过程同步完成特征选择
   - 无需人工特征工程

3. 适合在线学习场景：
   - 逐样本或逐批次更新，无需全量重训
   - 快速捕捉用户实时行为反馈

4. 内存效率高：
   - 仅需存储累积梯度等少量中间状态
   - 不需要历史数据集

### 2.3 关键参数

```
alpha, beta：per-coordinate 学习率调度参数
  学习率 = alpha / (beta + sqrt(sum_of_squared_gradients))
  频繁出现的特征学习率小（稳定），罕见特征学习率大（快速学习）

lambda1：L1 正则强度
  越大 → 越稀疏 → 特征更少但模型更小
  推荐系统中通常设较大值以获得稀疏性

lambda2：L2 正则强度
  防止权重过大，提升泛化能力
```

### 2.4 FTRL vs 其他在线优化器

```
OGD（Online Gradient Descent）：
  简单梯度下降，稀疏性差

FOBOS：精度好但稀疏性不如 FTRL
RDA：稀疏性好但精度不如 FTRL
FTRL：结合 FOBOS 的精度和 RDA 的稀疏性

结论：FTRL 是精度和稀疏性的最佳折中
```

### 2.5 FTRL 的局限

- 通常是线性或浅层模型，表达能力有限
- 难以捕捉复杂的非线性特征交互
- 在深度学习主导的排序模型中，直接应用受限
- 解决方向：将 FTRL 思想融入深度模型训练（稀疏权重优化器）

### 2.6 FTRL 适用场景

```
适合 FTRL + 实时特征：
  新闻资讯、信息流推荐（时效性强）
  大规模离散特征场景（高维稀疏）
  对模型稳定性和响应延迟要求极高

适合 Online Deep Learning + 实时特征：
  电商、短视频推荐（用户-物品交互丰富）
  业务复杂度高，需要深度网络建模
  有足够工程资源保障稳定运行
```

## 3. 模型漂移（Model Drift）

### 3.1 漂移类型

```
数据漂移（Data Drift / Covariate Shift）：
  输入特征分布变化
  例：节假日用户行为模式变化、新品类涌入

概念漂移（Concept Drift）：
  输入到标签的映射关系变化
  例：用户口味变化，同样的特征对应不同的点击倾向

标签漂移（Label Shift）：
  正负样本比例变化
  例：大促期间转化率整体上升
```

### 3.2 检测方法

```
PSI（Population Stability Index）：
  量化特征分布偏移
  PSI = sum((actual% - expected%) * ln(actual% / expected%))
  PSI < 0.1 → 无显著偏移
  PSI > 0.25 → 严重偏移，需要重训

KS 检验：统计检验两个分布是否相同
AUC/Loss 趋势监控：在线指标持续下降 = 漂移信号
预测分布监控：CTR 预估值的分布偏移
```

### 3.3 应对策略

```
自动触发重训：
  PSI 超阈值 → 自动触发全量训练 pipeline
  AUC 下降超过 X% → 告警 + 自动重训

在线学习跟随：
  增量更新提高模型跟随速度
  实时统计量代替离线统计量

模型集成切换：
  维护候选模型池（不同时期训练）
  动态选择与当前数据分布最匹配的模型
  Bandit 算法 + A/B 测试自动选优

模型回滚机制：
  新模型出现异常 → 自动切回稳定旧版本
  需要完善的版本管理和监控告警
```

## 4. 增量训练与全量重训

### 4.1 增量训练

```
方法：只用新数据更新模型参数
优势：速度快，资源消耗小
风险：
  灾难性遗忘（Catastrophic Forgetting）：
    新数据覆盖旧知识
    导致模型在旧分布上性能下降

缓解方案：
  EWC（弹性权重巩固）：
    约束重要参数不要变化太大
    对 Fisher 信息矩阵对角元素大的参数施加更强正则

  经验回放（Experience Replay）：
    混合少量旧数据一起训练
    防止完全遗忘历史模式

  知识蒸馏：
    用旧模型的输出作为软标签约束新模型
```

### 4.2 全量重训

```
方法：用全部历史数据重新训练
优势：准确，不存在遗忘问题
缺点：耗时长，资源消耗大
频率：通常按天/周执行
作用：作为增量训练的基线校准
```

### 4.3 模型热加载

```
目标：新模型就位后不重启服务直接替换
实现方案：
  双 buffer 切换：内存中维护两份模型，原子切换指针
  版本管理：每个模型有唯一版本号
  灰度发布：新模型先接 5% 流量验证
  自动回滚：在线指标异常时自动切回旧版本
```

## 5. 在线多任务学习

### 5.1 多任务模型架构

```
MMoE（Multi-gate Mixture-of-Experts）：
  多个专家网络共享信息
  门控网络为每个任务灵活选择专家组合
  适合任务间相关性中等的场景

PLE（Progressive Layered Extraction）：
  任务专属专家 + 共享专家分离
  引入门控网络处理任务间冲突
  在目标相关性不高的场景下优于 MMoE
```

### 5.2 在线多任务的挑战

```
多目标权重动态调整：
  简单静态损失加权无法应对数据分布变化
  方案：
    - 帕累托最优原则自动调整
    - 根据线上 A/B 测试指标动态更新权重
    - 不确定性加权（任务 loss 方差大 → 降权）

微批量更新策略：
  不在每次请求后更新，每几分钟收集一批数据
  平衡实时性和稳定性
  避免单条异常样本导致模型震荡
```

## 6. 在线学习的工程挑战

### 6.1 参数服务器架构

```
分布式在线学习的核心组件：
  Worker 节点：计算梯度
  参数服务器：聚合梯度并更新参数

关键设计：
  分布式的实时参数更新
  一致性哈希保证参数分片
  异步更新提升吞吐
  需控制学习率避免震荡
```

### 6.2 样本拼接

```
实时样本构建流程：
  用户行为 → 等待 label 窗口 → 拼接行为时刻的特征快照

关键细节：
  label 窗口：如 30 分钟内无转化视为负样本
  特征回放：必须用行为发生时的特征（非当前特征）
  延迟标签处理：有些转化延迟数小时（如购买后才确认）
```

### 6.3 数据安全与对抗

```
在线学习容易受攻击：
  恶意用户注入特定行为数据 → 模型被污染
  异常流量冲击 → 模型参数剧烈波动

防御机制：
  数据清洗和异常检测（过滤异常行为模式）
  稳健设计（梯度裁剪、学习率衰减）
  EE 策略注入探索流量（不完全信任模型）
  模型回滚机制（发现异常快速恢复）
```

## 7. 探索与利用（E&E）

### 7.1 核心矛盾

```
利用（Exploit）：推荐已知高收益物品 → 短期最优
探索（Explore）：尝试不确定物品 → 发现新机会

纯利用 = 信息茧房 + 无法发现新兴趣
纯探索 = 用户体验差 + 效率低
```

### 7.2 在线学习中的 E&E

```
Epsilon-Greedy：概率 epsilon 随机，简单但不智能

UCB：mean + c*sqrt(ln(N)/n_i)，被选次数少的物品自然获得探索机会

Thompson Sampling：
  从后验分布采样决策
  自然平衡：不确定物品方差大 → 有概率采到高值 → 被探索
  工业界最广泛使用

LinUCB / 上下文 Bandit：
  考虑用户和物品的上下文特征
  reward = x^T * theta + alpha * sqrt(x^T * A^{-1} * x)
  个性化探索：不同用户的探索方向不同
```

### 7.3 在线学习 + E&E 的结合

```
实时特征 → 在线学习模型预测 → 加入探索奖励 → 排序决策
                                     ↓
                          用户反馈 → 更新模型 → 闭环

探索策略融入模型层面：
  在预测分数中注入不确定性估计
  集成学习或贝叶斯神经网络估计预测方差
  对不确定性高的物品给予更多曝光机会
```

## 8. 监控与评估体系

### 8.1 在线模型监控指标

```
模型指标：AUC、LogLoss、校准度（predicted vs actual CTR）
特征指标：特征值分布、特征缺失率、PSI
系统指标：推理延迟 P99、QPS、模型版本
业务指标：CTR/CVR/留存的实时趋势
```

### 8.2 告警与自动响应

```
分级告警：
  P0：AUC 下降 > 3% → 自动回滚模型
  P1：PSI > 0.25 → 触发重训 pipeline
  P2：P99 延迟 > 200ms → 告警 + 排查
```

## 9. 面试高频问题速查

Q: FTRL 为什么能产生稀疏解？
A: L1 正则在原点处的次梯度截断效应。当累积梯度不足以克服 L1 惩罚时，权重被直接截断为零。

Q: 在线学习 vs 批量学习如何选？
A: 工业界通常混合使用。批量全量训练（天/周级）保证全局准确性 + 在线增量更新（分钟级）捕捉实时变化。

Q: 模型漂移如何检测？
A: PSI 监控特征分布偏移 + AUC/Loss 趋势图自动告警 + 预测分布监控。

Q: 如何防止在线学习模型不稳定？
A: 梯度裁剪 + 学习率衰减 + 微批量更新（非单样本）+ 定期全量重训校准 + 异常数据清洗 + 模型回滚机制。

Q: Thompson Sampling vs UCB？
A: TS 更灵活，支持批量场景和复杂后验，实践效果通常更优；UCB 确定性强、理论保证好。工业界首选 TS。
