# Ch5 推荐系统基础架构

## 核心概念总览

本章覆盖推荐系统全链路架构：多路召回、精排模型、重排策略、多目标优化、系统评估。

---

## 1. 推荐系统整体流程

```
用户请求
   ↓
召回层（Recall）：百万级 → 数千级
   多路召回：协同过滤、向量召回、热门、标签、图谱
   ↓
粗排层（Pre-Ranking）：数千级 → 数百级
   轻量模型快速打分（双塔、小 DNN）
   ↓
精排层（Ranking）：数百级 → 数十级
   复杂模型精确打分（DIN/DIEN/多任务）
   ↓
重排层（Re-Ranking）：数十级 → 最终展示
   多样性、业务规则、运营插入
```

## 2. 召回策略

### 协同过滤召回

```
User-CF：找相似用户 → 推荐其喜欢的物品
  - 适用：社交属性强、用户行为丰富
  - 缺点：用户多时相似度矩阵大

Item-CF：找相似物品 → 推荐用户历史的相似品
  - 适用：物品变化慢、长尾挖掘
  - 工业界更常用（物品数 < 用户数）

相似度计算：余弦相似度、Jaccard、皮尔逊相关
```

### Embedding 向量召回

```
双塔模型：
  User Tower: 用户特征 → 用户向量
  Item Tower: 物品特征 → 物品向量
  训练：内积/余弦 → 拉近正样本、推远负样本
  线上：ANN 检索（HNSW/IVF）→ Top-K 候选

优点：泛化能力强，覆盖长尾
缺点：双塔交互能力弱（无交叉特征）
```

### 多路召回融合

- 各路独立产出候选集
- 融合策略：并集去重 → 截断 → 统一进入粗排
- 多样性保证：各路配额控制

## 3. 排序模型演进

### 经典模型谱系

```
LR → GBDT → LR+GBDT → FM → FFM
  → DNN → Wide&Deep → DeepFM → DCN
    → DIN → DIEN → 多任务（MMoE/PLE）
```

### Learning-to-Rank 三范式

```
Pointwise：
  - 每个样本独立打分（回归/分类）
  - 简单，但忽略样本间相对关系
  - 例：LR、DNN 预测 CTR

Pairwise：
  - 学习样本对的偏序关系
  - 损失：BPR Loss, Hinge Loss
  - 例：RankSVM, LambdaRank

Listwise：
  - 直接优化整个列表的排序指标
  - 损失：ListNet（概率分布 KL 散度）、ApproxNDCG
  - 效果最好，计算最贵
```

## 4. 多任务学习

### Shared-Bottom

```
所有任务共享底层网络 → 各任务独立塔
优点：简单，参数少
缺点：任务冲突时互相干扰（负迁移）
```

### MMoE（Multi-gate Mixture of Experts）

```
多个 Expert 网络 + 每个任务独立 Gate
Gate(x) = Softmax(W_g * x)  → 各 Expert 加权
优点：任务自适应选择 Expert，减少负迁移
缺点：Expert 可能坍缩（都选同一个）
```

### PLE（Progressive Layered Extraction）

```
在 MMoE 基础上区分：
  - 任务特有 Expert
  - 跨任务共享 Expert
多层渐进式提取，逐层加深任务交互
效果 > MMoE，但参数量更大
```

### 多任务训练技巧

- 损失加权：手动调权 / 不确定性加权（Uncertainty Weighting）
- GradNorm：动态平衡各任务梯度大小
- 梯度冲突处理：PCGrad（投影冲突梯度）

## 5. 重排策略

### 规则重排

- 去重（同作者/同品类连续最多N个）
- 运营位插入（广告、活动）
- 过滤（违规、已购、黑名单）

### 多样性重排

```
MMR（Maximal Marginal Relevance）：
  score = lambda * relevance - (1-lambda) * max_sim(已选集合)
  贪心逐个选择，平衡相关性和多样性

DPP（Determinantal Point Process）：
  建模集合的联合概率 P(S) ~ det(L_S)
  L_S = diag(q) * S * diag(q)
  q = 物品质量分, S = 物品间相似度矩阵
  能同时优化质量和多样性
```

### 实时重排

- 基于 Flink 的实时上下文感知
- 根据用户实时反馈调整排序（如连续划过某类 → 降权）

## 6. 召回与排序的目标一致性

### 问题

- 召回用简单模型（双塔/CF），排序用复杂模型
- 两阶段优化目标不一致 → 召回的好物品可能被漏掉

### 解决方案

```
知识蒸馏：
  Teacher = 精排模型 → 生成 soft label
  Student = 召回模型 → 学习 Teacher 的打分分布
  缩小两阶段的目标 gap

统一优化：
  - 召回模型也用精排的 label（如 CTR）训练
  - 粗排模型逐步逼近精排复杂度
```

## 7. 系统评估体系

### 分阶段指标

```
召回层：
  - 召回率（Recall@K）：真实正样本被召回的比例
  - 覆盖率：能召回的物品占比
  - 多样性：候选集的类目/作者分布

排序层：
  - AUC / GAUC（分组 AUC）
  - NDCG@K
  - 校准度（预测 CTR vs 实际 CTR）

重排层：
  - 多样性指标（ILS, Entropy）
  - 业务约束满足率
```

### 漏斗分析

```
曝光量 → 点击率 → 深度交互率 → 转化率
每层转化率 = 该层输出 / 该层输入
定位问题层级：哪一层的转化率下降？
```

### A/B 测试

- 流量分桶：用户ID哈希 → 实验组/对照组
- 显著性检验：t 检验 / 置信区间
- 关键指标：CTR、停留时长、次日留存、GMV
- 注意：实验时长足够、避免 novelty effect
