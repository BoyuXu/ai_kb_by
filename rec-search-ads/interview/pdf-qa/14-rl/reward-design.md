# 强化学习推荐的奖励设计与离线评估

## 1. 奖励函数设计

### 1.1 即时奖励设计
```
基础奖励信号：
  点击：r = +1
  购买：r = +5
  收藏/关注：r = +3
  停留时长：r = +0.01 * seconds（归一化到 [0, 2]）
  快速划过：r = -0.5（停留 < 2s）
  不感兴趣反馈：r = -1
  举报：r = -5

加权组合：
  r = w1*click + w2*purchase + w3*dwell + w4*negative_feedback
  权重由业务目标决定（优化 GMV → w2 大，优化留存 → w3 大）
```

### 1.2 稀疏奖励问题
```
现象：
  大部分推荐交互无正反馈（CTR 通常 5-15%）
  购买等深度行为更稀疏（CVR < 1%）
  → 大量轨迹奖励为 0，策略梯度方向不明确

解决方案一：奖励塑形（Reward Shaping）
  添加中间奖励引导学习方向：
  r_shaped = r_original + F(s, a, s')
  F 必须满足势函数条件：F(s,a,s') = gamma*Phi(s') - Phi(s)
  → 保证最优策略不变

  推荐中的实践：
  - 用户浏览了推荐物品的详情页 → 小正奖励
  - 用户在推荐物品上的停留时长 → 连续奖励
  - 用户后续行为与推荐物品相关 → 延迟中间奖励

解决方案二：内在奖励（Intrinsic Reward）
  r_total = r_extrinsic + beta * r_intrinsic

  好奇心驱动：r_intrinsic = 状态转移预测误差
  - 训练一个前向模型预测 s' = f(s, a)
  - 预测越不准 → 内在奖励越大 → 鼓励探索未知
  - 推荐中：对用户反应预测不确定的物品给予奖励

  覆盖度奖励：r_intrinsic = 推荐物品的新颖度
  - 物品在用户历史中出现频率越低 → 奖励越高
  - 鼓励系统探索用户的潜在兴趣

解决方案三：分层奖励
  短期奖励（即时反馈）+ 长期奖励（留存/LTV）
  r = alpha * r_short + (1-alpha) * r_long
  alpha 随训练进程调整：先学短期信号 → 逐步加入长期信号
```

### 1.3 延迟奖励问题
```
现象：
  推荐 → 用户当天不购买 → 3天后购买 → 归因困难
  留存率只有在多天后才能观测

挑战：
  - 信用分配（Credit Assignment）：哪次推荐对最终购买贡献最大？
  - 时间跨度大：gamma^30 ≈ 0.04（gamma=0.9），30步前的推荐几乎无贡献

解决方案一：n-step Return
  不用完整的蒙特卡洛回报，用 n 步 TD：
  G_t^n = sum_{k=0}^{n-1} gamma^k * r_{t+k} + gamma^n * V(s_{t+n})
  n 控制短期/长期的权衡

解决方案二：辅助奖励信号
  用可快速观测的代理指标替代延迟奖励：
  - 用次日登录概率估计留存率
  - 用用户当天活跃度估计 LTV
  - 训练一个奖励预测模型 r_pred = f(trajectory)

解决方案三：Hindsight Credit Assignment
  轨迹结束后回溯分配延迟奖励：
  - 用注意力机制识别轨迹中的关键动作
  - 将最终奖励按注意力权重分配给各步
```

## 2. 多目标奖励设计

### 2.1 加权组合
```
最简方案：
  r = w1*CTR + w2*CVR + w3*diversity + w4*novelty - w5*负反馈

问题：
  - 权重需要人工调节，依赖经验
  - 不同指标量级不同，需要归一化
  - 权重固定无法适应不同用户/场景

实践技巧：
  1. 各指标先归一化到 [0, 1]
  2. 初始权重用网格搜索确定
  3. 线上 A/B 测试微调权重
```

### 2.2 约束优化
```
将次要目标作为约束而非加入奖励：
  max E[sum gamma^t * r_click]
  s.t. diversity(recommendations) >= threshold_d
       fairness(exposure) >= threshold_f

实现方式：
  拉格朗日方法：
  L = E[sum gamma^t * r_click] + lambda_d * (diversity - threshold_d)
                                + lambda_f * (fairness - threshold_f)
  lambda 自动调节（约束不满足时增大，满足时减小）

优势：主目标明确，约束可解释
缺点：约束阈值需要业务判断
```

### 2.3 帕累托多目标优化
```
同时优化多个目标，找帕累托前沿：
  不存在单一最优解，而是一组互不支配的解

方法一：多目标进化算法（MOEA）
  - 群体中维护多个解
  - 通过交叉/变异产生新解
  - 非支配排序筛选帕累托最优
  - 缺点：计算量大

方法二：线性标量化扫描
  - 遍历不同权重组合 w = [w1, w2, ...]
  - 每组权重训练一个策略
  - 所有策略的指标构成帕累托前沿估计

方法三：条件策略（Conditioned Policy）
  - 将目标权重 w 作为策略的输入
  - pi(a | s, w) = 一个模型适应所有权重
  - 推理时按需求设定 w，灵活切换目标偏好
```

## 3. 探索-利用权衡

### 3.1 探索策略
```
epsilon-Greedy 探索：
  以概率 epsilon 随机推荐，1-epsilon 按策略推荐
  简单但探索效率低，不区分探索价值高低

Boltzmann 探索：
  P(a|s) = exp(Q(s,a)/tau) / sum_a' exp(Q(s,a')/tau)
  温度 tau 控制随机性：tau 大 → 均匀探索，tau 小 → 利用
  优势：倾向探索 Q 值不确定但可能高的动作

UCB 式探索：
  a = argmax_a [Q(s,a) + c * uncertainty(s,a)]
  uncertainty 可用蒙特卡洛 Dropout / 深度集成估计

噪声网络（NoisyNet）：
  在网络权重中加入参数化噪声
  网络自己学习何时/如何探索
  无需额外探索机制
```

### 3.2 安全探索
```
推荐系统中的安全约束：

约束一：推荐质量下限
  不推荐明显低质量的物品（评分低、违规、与用户完全无关）
  实现：探索前过滤不满足基线质量的物品

约束二：探索比例控制
  限制每次推荐中探索物品的比例（如 ≤ 20%）
  核心位置（首位/头图）不做探索

约束三：策略偏移约束
  KL(pi_new || pi_old) < delta
  新策略不能偏离旧策略太远（PPO 的核心思想）
  防止一次性推荐大量不相关物品

约束四：熔断机制
  监控探索组的核心指标（CTR/留存）
  降幅超阈值 → 自动停止探索 → 回退到安全策略
```

## 4. 离线策略评估（OPE）

### 4.1 为什么需要 OPE？
```
问题：训练了新的 RL 策略，如何不上线就知道效果？
在线 A/B 测试成本高：
  - 探索低质量策略会损害用户体验
  - 实验周期长（1-2 周）
  - 实验流量有限

OPE 的作用：
  - 用历史日志快速评估新策略
  - 筛选有潜力的策略候选
  - 减少无效 A/B 测试次数
```

### 4.2 评估方法

#### IPS 估计器
```
V_IPS(pi_new) = (1/N) * sum_i [pi_new(a_i|s_i) / pi_old(a_i|s_i)] * r_i

重要性比 = 新策略选该动作的概率 / 旧策略选该动作的概率

性质：
  - 无偏估计
  - 方差高（重要性比可能极大或极小）

改进：
  - 截断 IPS：限制重要性比上限
  - 自归一化 IPS：权重归一化到和为 1
```

#### Direct Method（直接拟合）
```
训练一个奖励模型 r_hat(s, a) 预测奖励
V_DM(pi_new) = (1/N) * sum_i E_a~pi_new [r_hat(s_i, a)]

优点：方差低
缺点：有偏（依赖奖励模型的质量）
当奖励模型不准确时，估计可能完全错误
```

#### DR 估计器（双重稳健）
```
V_DR = (1/N) * sum_i [
    E_a~pi_new[r_hat(s_i, a)]
    + (pi_new(a_i|s_i) / pi_old(a_i|s_i)) * (r_i - r_hat(s_i, a_i))
]

= Direct Method + IPS 加权的残差校正

性质：
  - 只要 IPS 或 DM 其中一个正确 → 无偏
  - 方差远低于 IPS
  - 实践中首选方法
```

#### MAGIC 估计器
```
适用于多步 MDP 场景：

短期（前 H 步）：用多步 IPS 估计（方差可控）
长期（H 步之后）：用模型估计（避免累积 IS 比的方差爆炸）

V_MAGIC = sum_{t=0}^{H} [cumulative_IS_t * r_t]
          + cumulative_IS_H * V_model(s_H)

H 自适应选择：最小化估计的 MSE
适合长会话推荐场景
```

### 4.3 OPE 的局限与实践建议
```
局限：
  1. 新旧策略差异太大时，所有 OPE 方法都不可靠
  2. 无法评估新策略对用户状态转移的影响
  3. 计算倾向得分本身可能不准确

实践建议：
  1. 多种 OPE 方法交叉验证，结果一致才可信
  2. OPE 用于粗筛，最终决策仍需 A/B 测试
  3. 定期用 A/B 测试结果校准 OPE 估计器
  4. 控制策略偏移：新策略与旧策略的 KL 散度不超过阈值
```

## 5. 工业实践中的奖励设计

### 5.1 短视频推荐
```
奖励信号：
  - 完播率（最重要）：看完 > 50% 给 +1
  - 互动行为：点赞 +2，评论 +3，分享 +5
  - 停留时长：连续变量，需截断防止刷视频行为
  - 负反馈：快速划过 -0.5，长按不感兴趣 -1

长期信号：
  - 次日留存：用户第二天回来 +10
  - 7日活跃度：用户一周内活跃天数的变化

多目标权衡：
  短期完播率 vs 长期留存
  极端做法（纯优化完播率）→ 信息茧房 → 留存下降
  需要加入探索/多样性奖励
```

### 5.2 电商推荐
```
奖励信号：
  - 点击：+1
  - 加购：+3
  - 购买：+5 * 商品价格归一化
  - 退货：-10（延迟负反馈）

长期信号：
  - 用户复购率
  - 用户 LTV 变化
  - 品类探索广度（用户购买品类多样性）

特殊考虑：
  - 价格因素：推荐贵的物品 GMV 高但转化率低
  - 季节性：节假日权重调整
  - 促销干扰：促销期间的购买不完全反映推荐效果
```

### 5.3 信息流推荐
```
奖励信号：
  - 阅读时长：核心指标
  - 阅读完成度：读完比例
  - 互动：点赞、收藏、评论
  - 负反馈：不感兴趣、举报

内容生态奖励：
  - 优质内容加分：文章质量得分 * 交互奖励
  - 创作者激励：推荐新创作者的内容 + 额外奖励
  - 多样性：连续推荐同类内容 → 递减奖励
```

## 6. 面试高频问题

```
Q: 推荐系统中如何解决稀疏奖励问题？
A: 三个层次：1) 奖励塑形，添加中间奖励信号引导学习
   2) 内在奖励（好奇心驱动），鼓励探索未知物品
   3) 分层奖励，先用密集的短期信号学习，再加入稀疏的长期信号。

Q: OPE 的 IPS 估计器和 DR 估计器有什么区别？
A: IPS 无偏但高方差（重要性比极端时不稳定），DR 结合 IPS 和
   填补模型，只要其中一个正确就无偏，方差远低于 IPS。
   实践中首选 DR。

Q: 如何设计奖励函数来平衡短期收益和长期价值？
A: 1) 加权组合：r = alpha*r_short + (1-alpha)*r_long
   2) 约束优化：max r_short s.t. 留存率 >= 阈值
   3) 课程学习：先优化短期信号，逐步加入长期信号
   4) 用代理指标（如次日登录概率）近似长期奖励。

Q: 多目标奖励的权重如何确定？
A: 1) 离线网格搜索初始权重 2) 线上 A/B 测试微调
   3) 条件策略方法：将权重作为策略输入，一个模型支持多种权重
   4) 拉格朗日方法：将次要目标转为约束，自动调节拉格朗日乘子。

Q: 好奇心驱动探索在推荐中的风险？
A: 可能推荐过于小众/低质的物品损害用户体验。
   需要：1) 限制内在奖励权重 2) 设置质量下限
   3) 与外在奖励平衡 4) 监控探索组的用户指标。
```
