# 预训练语言模型在推荐中的深度应用

## 1. BERT 特征提取全流程

### 1.1 标准流程

```
原始文本 → 清洗+分词 → BERT Tokenizer([CLS]/[SEP]) → BERT前向传播
  → 特征提取([CLS]向量 / mean pooling) → 降维(PCA/AE, 768→64/128)
  → 与用户画像、统计特征拼接 → 输入排序模型
```

### 1.2 特征提取策略对比

```
策略              适用场景          特点
[CLS] 向量        句子级语义匹配    BERT 预训练目标直接优化此位置
Mean Pooling      标题级特征表示    平均所有 token 输出，信息更全面
Max Pooling       关键词强调        取每维最大值，突出显著语义
加权池化(TF-IDF)  长文本           重要词权重更大，减少噪声
```

### 1.3 降维的必要性

768 维 Embedding 直接用于推荐系统会导致：
- 排序模型中文本特征淹没其他低维特征（用户画像/统计特征）
- 在线推理存储开销大
- 解决：PCA 降至 64/128 维，或训练自编码器非线性压缩

## 2. BERT 模型优化（工业部署）

### 2.1 知识蒸馏

核心思想：大模型（教师）指导小模型（学生）学习

```
教师 BERT-base (110M参数)
    ↓ 蒸馏损失 = alpha * KL(student_logits, teacher_logits)
                + beta * MSE(student_hidden, teacher_hidden)
学生 TinyBERT (14M参数) / MiniLM (22M参数)
```

关键技术：
- Logits 蒸馏：学生模仿教师的输出分布
- 中间层蒸馏：学生模仿教师的注意力矩阵和隐藏层
- 任务特定蒸馏：先通用蒸馏再任务蒸馏，两阶段效果最好

### 2.2 模型量化

```
方法         精度损失   加速比   适用场景
动态量化     低         2x      推理时即时量化，无需校准数据
静态量化     中低       2-3x    预先用校准集确定量化参数
量化感知训练 最低       2-4x    训练时模拟量化噪声，精度最优
```

- FP32 → FP16：几乎无损，显存减半
- FP32 → INT8：精度微损，体积减 4x，吞吐提升 2-3x
- 推荐系统中常用 FP16 + 动态量化组合

### 2.3 模型剪枝

- 结构化剪枝：移除整个注意力头/隐藏层，硬件友好，加速明显
- 非结构化剪枝：将不重要权重置零，需稀疏计算支持
- 在推荐场景中，结构化剪枝更实用（BERT 12层 → 6层，效果损失 <3%）

### 2.4 推理引擎

```
引擎            特点                        适用
TensorRT       NVIDIA GPU 深度优化          GPU 推理首选
ONNX Runtime   跨平台，CPU/GPU 均支持       混合部署
TorchScript    PyTorch 原生，部署简单        快速上线
```

工程实践：模型缓存 + 请求批处理(dynamic batching) 可提高 GPU 利用率 3-5x

## 3. 双塔召回模型中的 PLM

### 3.1 架构设计

```
用户塔                          物品塔
用户侧文本 → PLM → v_user      物品标题/描述 → PLM → v_item
                    ↓                              ↓
              score = cosine(v_user, v_item)
```

### 3.2 用户塔文本来源

方案一（聚合式）：
- 收集用户历史交互物品的文本嵌入
- 通过注意力/加权平均聚合为用户向量
- 优点：信息丰富；缺点：计算开销大

方案二（直接式）：
- 编码用户自填信息（兴趣标签、搜索词）
- 优点：反映即时意图；缺点：依赖用户主动输入

### 3.3 训练策略

损失函数：
```
InfoNCE Loss = -log(exp(sim(u,i+)/tau) / sum(exp(sim(u,ij)/tau)))
```

负采样策略至关重要：
- 随机负采样：简单但信号弱
- 曝光未点击：中等难度负样本
- 对抗采样/难负例挖掘：最强信号但易过拟合
- 最佳实践：混合策略，不同比例组合

### 3.4 微调策略

- 领域自适应预训练(DAP)：在推荐领域文本上继续 MLM 预训练
- 端到端微调：PLM 与推荐损失联合优化
- 渐进式解冻：先冻结 PLM 训练顶层 → 逐步解冻底层
- 实际效果：DAP + 端到端微调 > 直接端到端 > 仅特征提取

## 4. BERT 序列建模

### 4.1 将用户浏览序列输入 BERT

```
用户行为序列: [商品A标题, 商品B标题, ..., 商品N标题]
    ↓
每个标题 → BERT → [CLS]向量 → 标题Embedding序列
    ↓
标题Embedding序列 → Transformer编码器 → 用户兴趣表示
    ↓
用于召回(向量检索) / 排序(用户侧特征)
```

### 4.2 工程挑战与解决

- BERT 推理开销大 → 知识蒸馏到 TinyBERT
- 长序列处理 → 截断最近 N 个 + 滑动窗口
- 超长序列 → Transformer-XL / Longformer（稀疏注意力）
- 实时性 → 预计算商品 Embedding + 增量更新用户序列表示

## 5. BERT 在推荐中的四类应用总结

```
应用类型          输入              输出              场景
物品侧特征提取    商品标题/描述      物品Embedding     向量召回、精排特征
用户侧兴趣建模    用户评论/搜索序列  用户兴趣向量      长期兴趣捕捉
语义匹配          用户历史+候选文本  匹配度分数        双塔召回
序列推荐          物品文本序列       下一个物品预测    序列推荐
```

## 6. 排序阶段的特征融合

### 6.1 三种融合策略

早期融合：
- 所有特征（文本BERT向量 + 用户画像 + 统计）拼接成统一向量
- 输入深度网络端到端训练
- 优点：自动学习特征交互
- 缺点：文本高维特征可能淹没其他低维特征
- 适用：特征维度差异不大时

中期融合（效果通常最好）：
- 双塔/多通道结构，各通道处理不同模态
- 交互方式：交叉注意力、外积、Concatenate+MLP
- 优点：灵活性高，可针对每种模态定制编码器
- 缺点：架构复杂，调参成本高

晚期融合：
- 各子模型独立打分，最后加权平均或 stacking
- 优点：实现简单，模块化程度高
- 缺点：无法捕捉细粒度跨模态交互
- 适用：快速上线，或各模态独立性强时

### 6.2 共同挑战

- 特征尺度不一致：BERT 768维 vs 用户ID Embedding 32维 → 需投影/归一化
- 实时性：预计算 + 缓存文本 Embedding，在线只做轻量特征拼接
- 噪声处理：文本特征可能引入噪声（差评、无关描述），需过滤

## 7. 序列推荐模型：SASRec vs BERT4Rec

### 7.1 SASRec (Self-Attentive Sequential Recommendation)

- 单向 Transformer：只看左侧历史，类似 GPT
- 训练目标：预测序列中下一个物品
- 注意力掩码：causal mask，防止信息泄露
- 优点：推理高效，因果性强
- 适用：实时推荐（只需历史信息）

### 7.2 BERT4Rec

- 双向 Transformer：看全部上下文，类似 BERT
- 训练目标：Masked Item Prediction（随机遮盖序列中的物品并预测）
- 优点：双向上下文信息更丰富，效果通常优于 SASRec
- 缺点：推理时需要特殊处理（用 [MASK] 占位预测下一个）
- 适用：离线计算或批量推荐

### 7.3 对比总结

```
维度         SASRec          BERT4Rec
方向性       单向(左到右)     双向
训练目标     Next Item        Masked Item
推理效率     高(自回归)       中(需[MASK])
上下文       仅历史           全局
冷启动       较弱             较好(双向信息)
```

## 面试高频问题

Q: BERT 在推荐系统中的最大瓶颈？
A: 推理延迟。BERT-base 110M参数，在线需毫秒级响应。优先级：(1)离线预计算缓存Embedding (2)蒸馏到TinyBERT/MiniLM (3)量化INT8 (4)TensorRT/ONNX推理优化 (5)请求批处理。

Q: 双塔模型用 PLM 的关键坑？
A: 负采样策略决定效果上限。纯随机负采样学到的只是粗粒度语义，必须加入曝光未点击的难负例。另外用户塔和物品塔的 PLM 是否共享需要实验，共享减参数但可能损失表达力。

Q: SASRec vs BERT4Rec 怎么选？
A: 在线实时推荐选 SASRec（单向，因果推理高效）；离线计算或对效果要求高选 BERT4Rec（双向上下文更丰富）。工业界 SASRec 更常用因为部署简单。

Q: 早期/中期/晚期融合怎么选？
A: 召回阶段用简单拼接或晚期融合（保效率），排序阶段用中期融合（注意力交叉，效果最好）。工程资源有限从晚期融合起步，逐步升级。
