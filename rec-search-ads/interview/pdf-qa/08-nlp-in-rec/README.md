## 1. 基础文本特征提取

### 1.1 TF-IDF

原理：
- TF（词频）= 词在文档中出现次数 / 文档总词数
- IDF（逆文档频率）= log(总文档数 / 包含该词的文档数)
- TF-IDF = TF * IDF
- 核心思想：在当前文档高频、在整个语料中低频的词更有区分力

在推荐系统中的应用：
- 召回阶段：计算商品 TF-IDF 向量间的余弦相似度，用于基于内容的召回
- 排序阶段：TF-IDF 特征作为排序模型的输入特征之一
- 用户画像：从用户评论中提取关键词，构建兴趣标签

局限性：
- 词袋模型，无法捕捉序列信息和语义关系
- "苹果好吃"和"好吃苹果"的 TF-IDF 表示完全相同
- 高维稀疏，维度 = 词汇表大小

### 1.2 N-gram 与 PMI

N-gram：捕捉局部序列信息（bigram/trigram）
PMI（逐点互信息）：衡量两个词的共现程度
- PMI(x,y) = log(P(x,y) / (P(x)*P(y)))
- PMI > 0 表示正相关（如"机器"和"学习"）

### 1.3 多语言文本处理

跨境电商场景的挑战：
- 中文用 jieba 分词，英文做小写化 + 词形还原
- 构建统一的多语言停用词表
- 分别构建各语言的 TF-IDF 矩阵，避免字符集混淆
- 特征融合：将中英文 TF-IDF 向量拼接

### 1.4 实时推荐中的瓶颈

传统文本方法的性能问题：
- 高维稀疏向量计算效率低
- 在线分词和特征提取延迟不可控
- 解决方案：预计算 + 缓存 + 降维（如保留 top-N TF-IDF 特征）

## 2. 词嵌入技术

### 2.1 Word2Vec

核心思想：通过浅层神经网络，从上下文共现中学习词的稠密向量表示
分布式假设：出现在相似上下文中的词具有相似语义

两种架构：
- Skip-gram：用中心词预测上下文词，适合小语料 + 低频词
- CBOW：用上下文词预测中心词，训练更快

训练技巧：
- 负采样（Negative Sampling）：每个正样本搭配 K 个负样本
- 层次 Softmax：用 Huffman 树加速 softmax 计算

在推荐系统中的应用：
1. 物品 Embedding：将标题/描述通过 Word2Vec 转化为物品向量，用于语义相似度计算
2. 用户兴趣建模：将用户历史评论的词向量聚合（平均/TF-IDF 加权）为用户兴趣向量
3. 缓解冷启动：新物品可通过文本直接生成初始 Embedding

Item2Vec：
- 将 Skip-gram 直接应用于用户行为序列（物品 = 词，行为序列 = 句子）
- 纯粹从共现行为中学习物品表示，不依赖内容信息
- 启示：算法的核心思想比其原始应用领域更重要

### 2.2 GloVe

原理：
- 基于全局词-词共现矩阵
- 训练目标：使学习到的词向量的点积尽可能接近共现次数的对数值
- 融合了矩阵分解和局部上下文窗口的优点

Word2Vec vs GloVe 对比：
- Word2Vec：基于局部上下文，擅长捕捉句法和语义相似性，处理复杂模式（如"苹果手机"vs"安卓手机"）
- GloVe：基于全局统计，擅长捕捉词间全局相关性（如"咖啡"和"咖啡杯"的高频共现）
- Word2Vec 对低频词/新物品更友好（只需局部上下文）
- GloVe 对长尾物品表示质量差（全局统计量不足）

### 2.3 电商场景的统一向量表示

方案设计（以商品为例）：
1. 文本向量 V_text：将标题/描述通过 Word2Vec 转化并池化
2. 类别向量 V_cat：类别作为可学习嵌入
3. 品牌向量 V_brand：品牌作为可学习嵌入
4. 融合方式：
   - 向量拼接：V_item = [V_text; V_cat; V_brand]
   - 加权求和或注意力机制融合
5. 应用：召回阶段做向量检索（FAISS/HNSW），排序阶段作为特征输入

## 3. 预训练语言模型应用

### 3.1 BERT 特征提取流程

完整流程：
1. 文本预处理：清洗 + 分词（中文尤其重要）
2. Tokenization：BERT Tokenizer 编码，添加 [CLS]/[SEP]
3. 模型前向传播：输入预训练 BERT
4. 特征提取：
   - [CLS] 向量：句子级语义表示
   - 所有词向量的均值/最大池化：标题级表示
5. 后处理降维：PCA 或自编码器，从 768 维降到 64/128 维
6. 特征集成：与用户画像、统计特征等拼接输入排序模型

### 3.2 BERT 模型优化（工业部署）

知识蒸馏：
- 大模型（教师）指导小模型（学生）学习
- 学生模型：TinyBERT、MiniLM
- 损失：模仿教师的 logits + 中间层特征

模型量化：
- 将 FP32 权重转为 FP16/INT8
- 动态量化：推理时即时量化
- 静态量化：预先校准量化参数
- 可减少 2-4x 模型体积，显著提升吞吐量

模型剪枝：
- 结构化剪枝：移除整个注意力头或隐藏层
- 非结构化剪枝：将不重要的权重置零（需要稀疏计算支持）

推理引擎：
- TensorRT（NVIDIA GPU 优化）
- ONNX Runtime（跨平台）
- 模型缓存 + 请求批处理提高 GPU 利用率

### 3.3 双塔召回模型中的 PLM

架构：
- 用户塔：PLM 编码用户侧文本 -> v_user
- 物品塔：PLM 编码物品标题/描述 -> v_item
- 匹配度：score = <v_user, v_item> 或 cosine(v_user, v_item)

用户塔文本来源：
- 方案一：聚合用户历史交互物品的文本嵌入（注意力/加权平均）
- 方案二：直接编码用户自填信息（兴趣标签、搜索词）

训练：
- 对比学习损失（InfoNCE）或三元组损失
- 负采样策略至关重要：随机 + 基于曝光未点击 + 对抗采样

微调策略：
- 领域自适应预训练：在推荐领域文本上继续预训练
- 端到端微调：将 PLM 与推荐任务目标一致

### 3.4 BERT 用于序列建模

将用户浏览的商品标题序列输入 BERT：
1. 序列构建：将商品标题按时间排列
2. 单项文本编码：每个标题通过 BERT 获取 [CLS] 向量
3. 序列建模：将标题嵌入序列输入 Transformer 编码器
4. 微调适配：在推荐任务数据上端到端微调
5. 应用：召回阶段做向量检索，排序阶段做用户侧特征

工程挑战：
- BERT 推理开销大，需蒸馏/量化
- 长序列处理：截断 + 滑动窗口
- 可用 Transformer-XL、Longformer 处理超长序列

### 3.5 BERT 在推荐中的四类应用

1. 物品侧深度文本特征提取：生成高质量物品 Embedding
2. 用户侧兴趣建模：编码用户评论/搜索序列，捕捉长期兴趣
3. 用户-物品语义匹配：拼接用户历史和候选物品文本，直接预测匹配度
4. 序列推荐：将物品文本序列作为输入，预测下一个可能感兴趣的物品

### 3.6 排序阶段的特征融合

文本特征（BERT 向量）与非文本特征的融合策略：

早期融合：
- 将所有特征（文本 + 用户画像 + 统计）拼接成统一向量
- 输入深度网络端到端训练
- 优点：自动学习特征交互
- 缺点：文本维度过高可能淹没其他特征

中期融合：
- 双塔或多通道结构，各通道处理不同模态
- 通过交叉注意力、外积、Concatenate+MLP 进行特征交互
- 灵活性高，效果通常最好

晚期融合：
- 各子模型独立打分，最后加权平均或 stacking
- 实现简单，模块化程度高
- 缺点：无法捕捉细粒度特征交互

共同挑战：
- 特征分布尺度不一致（需归一化/投影）
- 实时性与性能的平衡（预计算 + 缓存文本 Embedding）
- 数据稀疏和噪声处理

## 4. 高级语义特征提取

### 4.1 主题模型

LDA（Latent Dirichlet Allocation）：
- 从用户评论中提取主题分布
- 无需标注，可解释性强
- 用于用户画像构建：用户 = 主题分布向量

神经主题模型：
- 基于 VAE 的主题模型
- 更好地处理短文本和上下文语义
- 输出的主题分布可直接作为推荐特征

应用：
- 将主题分布与传统画像拼接，增强用户表示
- 按时间窗口构建主题兴趣演化序列
- 在重排层引入主题多样性正则项（MMR 算法）

### 4.2 情感分析

从用户评论中提取情感特征：
1. 方法一：BERT 微调做情感分类，取 softmax 概率作为连续特征
2. 方法二：取 [CLS] 隐藏层输出作为语义嵌入（含情感/主题等信息）
3. 池化策略：对多条评论做平均/最大池化，生成固定维度表示

融入推荐模型：
- 召回阶段：物品评论情感向量与物品 ID 嵌入拼接，用于向量检索
- 排序阶段：情感特征作为排序模型的额外输入维度

注意事项：
- 不同领域情感表达差异大（电商 vs 社交），需领域自适应微调
- 显式情感分数和隐式深度嵌入各有优势，可结合使用

### 4.3 其他文本信息源

除标题和描述外：
1. 用户搜索查询：反映即时意图，最强的兴趣信号
2. 用户评论和发布内容：长期兴趣、价值观、品味
3. 众包评论（物品全部评论）：物品质量、优缺点、隐式属性
4. 客服对话记录：用户痛点、未满足需求
5. 分类标签/运营标签：强语义约束，确保推荐准确性

## 5. 多模态信息融合

### 5.1 文本+图像融合方法

三种主要策略：

1. 拼接+全连接（早期融合）：
   - 文本向量和图像向量直接拼接
   - 输入全连接层学习交互
   - 简单高效，适合基线方案
   - 缺点：粗糙，难以捕捉细粒度交互

2. 双线性池化（中期融合）：
   - 计算文本和图像特征向量的外积
   - 捕捉每个维度间的乘积关系
   - 适合模态间精细对齐的任务（如时尚搭配）
   - 缺点：参数量巨大，需低秩分解优化

3. 交叉注意力（模型级融合）：
   - 一个模态的特征做 Query，另一个做 Key/Value
   - 动态非对称加权，聚焦最相关的跨模态信息
   - 效果最好，适合模态信息不对等的场景
   - 缺点：计算开销大

### 5.2 CLIP 在推荐中的应用

CLIP 特性：通过对比学习将图像和文本映射到统一语义空间

应用方式：
1. 作为特征提取器：生成高质量、对齐的多模态嵌入向量
2. 跨模态检索：在召回阶段实现"以文搜图"/"以图搜文"
3. 端到端微调：引入用户交互数据优化推荐任务

优势：
- 强语义理解力，理解物品内容和属性
- 天然跨模态对齐，无需人工标注
- 缓解冷启动：只需内容信息即可生成表示
- 零样本识别能力

局限：
- 领域差异（通用 vs 垂直领域）
- 缺少用户个性化偏好建模
- 计算开销大（实时推理 vs 离线特征提取的权衡）
- 可能存在模态偏见

### 5.3 语义鸿沟问题

场景：商品标题为"时尚连衣裙"，图像显示休闲风格

解决方案：
1. 对比学习对齐：构建正负样本对，用 InfoNCE Loss 拉近同一物品的文本和图像表示
2. 跨模态注意力交互：文本关注图像中的关键区域（如"时尚"关注设计细节）
3. 知识图谱辅助：将商品链接到实体和概念，缓解表面矛盾
4. 分层融合：召回用晚期融合（分别计算相似度再加权），排序用中期融合
5. 用户行为监督：用户点击/购买行为作为跨模态对齐的弱监督信号

### 5.4 评估多模态融合效果

实验设计：
- 对照组 A：仅文本特征
- 实验组 B：文本 + 图像融合
- 消融组 C：仅图像特征
- 通过 A vs B 评估融合增益，A vs C 和 B vs C 评估各模态贡献

指标：CTR、CVR、停留时长（线上），AUC、NDCG（离线）
注意排除其他干扰因素（流量随机性、特征一致性等）

## 6. 知识图谱增强推荐

### 6.1 知识图谱基础

知识图谱 = 实体 + 属性 + 关系 组成的语义网络

在推荐中的三大作用：
1. 缓解冷启动：通过语义关联弥补交互数据不足
2. 增强多样性：通过关联路径发现跳出信息茧房的推荐
3. 提升可解释性：生成"因为您喜欢 A -> 导演 D -> 推荐 B"的解释

### 6.2 两种融合方式

基于嵌入的方法：
- TransE/TransR 等将实体和关系映射到低维向量空间
- 图神经网络（GraphSAGE/KGAT）聚合多跳邻居信息
- 生成丰富的物品语义画像，用于向量化召回或排序

基于路径的方法：
- 挖掘用户到物品的元路径（如 用户->电影->导演->电影）
- PathRanking/GNN 建模路径可靠性
- 提供直观的语义解释

### 6.3 实体链接

将文本中提到的实体映射到知识图谱中的唯一节点

价值：
- 语义统一：将"苹果手机"/"iPhone"/"苹果牌手机"映射到同一实体
- 丰富特征：引入实体的结构化属性和关系
- 支持图推理：基于链接后的统一实体进行路径发现

挑战：
- 召回挑战：系统无法识别所有别名
- 消歧挑战："苹果"可能指公司/水果/电影
- 解决方案：深度学习实体链接模型 + 高质量别名词典 + 用户行为上下文辅助消歧

### 6.4 知识图谱动态更新

实时数据管道：
- Kafka/Pulsar 捕获新增实体和关系
- Flink/Spark Streaming 实时处理（NLP 实体识别 + 链接）
- 双缓冲机制：在线版本 + 待更新版本，低延迟切换

推荐模型适配：
- 新实体：通过 Embedding 模型生成初始向量 + 图游走补充邻居信息
- 排序模型：增量学习/在线学习持续微调
- 实时特征：用户与新实体的交互作为实时特征输入

## 7. 面试高频问题

Q: 为什么推荐系统中 TF-IDF 仍然有用？
A: TF-IDF 简单高效、无需训练、可解释性强。在实际系统中常作为多路召回的一路（与 Embedding 方法互补），也常作为排序模型的辅助特征。虽然无法捕捉语序和深层语义，但在关键词匹配场景仍有不可替代的价值。

Q: BERT 在推荐系统中的最大瓶颈是什么？
A: 推理延迟。BERT-base 有 110M 参数，在线服务需要毫秒级响应。解决方案按优先级：(1) 离线预计算并缓存文本 Embedding (2) 知识蒸馏到 TinyBERT/MiniLM (3) 量化到 INT8 (4) TensorRT/ONNX 推理优化 (5) 请求批处理提高 GPU 利用率。

Q: 早期/中期/晚期融合怎么选？
A: 看业务需求。召回阶段用简单的拼接或晚期融合（保证效率），排序阶段用中期融合（注意力交叉，效果最好）。工程资源有限时从晚期融合开始，逐步升级。

Q: 如何评估文本特征对推荐的实际贡献？
A: 设计消融实验：(A) 无文本 (B) 有文本 (C) 仅文本。通过 A vs B 量化文本增益，B vs C 量化其他特征的独立贡献。关键是保持其他变量一致，并做统计显著性检验。

Q: 知识图谱 vs 协同过滤，各自优势？
A: 协同过滤依赖交互数据，在数据充足时效果好但受冷启动限制。知识图谱引入外部结构化知识，天然缓解冷启动和数据稀疏，且提供可解释性。最佳实践是将两者结合：协同过滤信号 + 知识图谱特征共同输入排序模型。
