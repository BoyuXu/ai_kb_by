# A/B 实验设计原则与分桶策略

## 1. A/B 测试基本原理

A/B 测试是随机对照实验（RCT），核心机制：
- 对照组（Control）：保持原有策略，提供效果基线
- 实验组（Treatment）：应用新策略，与基线对比
- 随机分流：用户无偏分配到各组，保证可比性
- 单一变量原则：仅改变待测策略，其他条件一致

推荐系统中 A/B 测试的必要性：
1. 强主观性和场景依赖：离线指标好不代表线上效果好
2. 算法迭代闭环：提供因果性证据而非相关性猜测
3. 风险控制：小流量验证，避免全量上线的负面影响
4. 量化业务价值：直接证明新算法对 CTR/GMV/留存的提升

标准流程：
```
1. 提出假设 → 明确预期改进
2. 实验设计 → 选定主指标+护栏指标，计算样本量
3. 流量分配 → 随机分流（通常用 userID hash）
4. 实验运行 → 线上同时运行 A/B 版本，收集数据
5. 统计分析 → t 检验计算 p 值，判断显著性
6. 结果决策 → 主指标显著提升 + 护栏不恶化 → 上线
```

## 2. 样本量估算

影响样本量的四个核心参数：

```
显著性水平 alpha（通常 0.05）：犯第一类错误的概率上限
  → alpha 越小，所需样本越大

统计功效 1-beta（通常 0.80 或 0.90）：检测到真实差异的概率
  → 功效越高，所需样本越大

基线转化率 p0：当前指标水平
  → 越接近 50%，方差越大，样本越多

最小可检测效应 MDE：业务关心的最小提升幅度
  → MDE 越小，所需样本越大（与 MDE^2 成反比）
```

样本量公式（双侧检验，连续指标）：
```
n = (Z_{alpha/2} + Z_{beta})^2 * 2 * sigma^2 / delta^2

其中 delta = MDE * mu（绝对效应），sigma 为标准差
```

实践要点：
- 检测 1% 提升需要的样本是 2% 提升的 4 倍
- 高方差指标（如 GMV）需要更大样本量
- 可用 CUPED 降方差，减少 50%+ 所需样本量
- 实验周期 = 所需样本量 / 每日可分流用户数
- 必须覆盖完整业务周期（至少含一个完整周末）

## 3. 辛普森悖论

定义：分组数据合并后，整体趋势与各子组趋势完全相反。

推荐系统中的典型场景：
- 实验组偶然分到更多高活跃用户 → 整体指标好，但算法本身无提升
- 不同平台（iOS/Android）的用户比例不同 → 混淆平台效应

根因：存在未控制的混杂变量，影响了分组结构。

识别和避免方法：
1. 科学设计分组：确保随机分流，在关键维度（活跃度/新老/地域）分层抽样
2. 分层分析：不能只看整体，必须按用户群体、品类、时间段分别检查
3. 监控混杂变量：实验前检查分组在关键特征上的均衡性
4. 建立稳健评估体系：关注分位数指标和分群留存曲线

## 4. 多重检验校正

问题：同时检验 N 个指标时，假阳性率膨胀。
```
N 个独立检验时，至少 1 个假阳性的概率：
P = 1 - (1 - 0.05)^N

N=10 时，P = 40%；N=20 时，P = 64%
```

校正方法对比：

Bonferroni 校正：
- 方法：alpha' = alpha / N
- 特点：最保守，严格控制家族错误率（FWER）
- 适用：模型上线前的最终验证，不能容忍假阳性

BH-FDR（Benjamini-Hochberg）：
- 方法：将 p 值排序后，找最大的 i 使得 P(i) <= (i/K) * q
- 特点：控制错误发现率而非 FWER，更灵活
- 适用：探索性实验，大量策略筛选阶段

实践建议：
- 预先定义 1-2 个主指标（OEC）作为决策依据
- 其余为辅助观察指标，不作为上线决策依据
- 分层实验或网联化实验架构隔离实验间干扰

## 5. 新奇效应与 Peeking Problem

新奇效应（Novelty Effect）：
- 现象：新策略上线初期效果显著，后期衰减
- 原因：用户对新界面/新推荐的探索行为
- 应对方案：
  - 实验持续 >= 2 周，等新奇感消退
  - A-A-B 设计：先 A-A 阶段确认基线，再切到 B
  - 分别分析新老用户的效果差异
  - 观察指标随时间的趋势曲线

Peeking Problem：
- 现象：反复查看中间结果，"看到显著就停"
- 危害：严重膨胀假阳性率
- 解决：
  - 预设实验时长，到期才看结果
  - 使用序贯检验（Always Valid p-value）
  - 设置固定的分析时间点

## 6. 分桶策略与流量管理

分流粒度选择：
- 用户级（userID hash）：最常用，一个用户始终在同一组
- 设备级（deviceID）：未登录场景
- 请求级：适合短期效果评估，但同一用户体验不一致

分层实验框架（Google 方案）：
```
流量层1（召回层实验）：实验 A1/A2/A3 互斥
流量层2（排序层实验）：实验 B1/B2 互斥
流量层3（UI 层实验）：实验 C1/C2 互斥

关键：不同层正交组合，同层互斥
同一用户可同时参与 A1 + B2 + C1
要求不同层的实验效果独立
```

A/A 测试：
- 目的：验证实验系统本身的可靠性
- 两组用户使用完全相同策略
- 预期：p 值均匀分布，显著性比例约等于 alpha
- 作用：检验分流随机性、指标计算管道、建立基线方差

## 7. 网络效应实验

场景：社交/共享经济中，用户间存在互相影响。
问题：违反 SUTVA（稳定单元处理值假设）。

解决方案：
- 集群随机化（Cluster Randomization）：按社群/地域整体分组
- Switchback 实验：时间维度交替，减少空间溢出
- 图分割法：利用社交图谱找低连通度的切分点

## 8. CUPED 方差缩减

原理：利用实验前数据作为协变量，回归掉与处理无关的方差。
```
y_cuped = y - theta * (x_pre - E[x_pre])

theta = Cov(y, x_pre) / Var(x_pre)
```

效果：通常可减少 50%+ 的方差，等效于减少 50% 所需样本量。

适用条件：
- 实验前后指标有较高相关性
- 用户历史行为数据可获取
- 不适用于全新功能（无历史基线）

## 9. 自适应实验（MAB）

与传统 A/B 测试对比：
```
A/B 测试：固定分流比 → 实验结束后评估 → 统计严谨
MAB：动态调整分流比 → 边实验边优化 → 效率更高
```

三种经典 MAB 算法：

Epsilon-Greedy：
- 以 epsilon 概率随机探索，1-epsilon 选最优
- 简单但不自适应，探索效率低

UCB（Upper Confidence Bound）：
- 选择 argmax(mu_i + c * sqrt(ln(t) / n_i))
- 确定性算法，理论 regret bound 好

Thompson Sampling：
- 为每个臂维护 Beta(alpha, beta) 后验分布
- 每次从后验采样，选最大值
- 自然平衡探索与利用，工业界首选

汤普森采样实验设计示例：
```
1. 参数空间定义：如排序权重候选值 [0.1, 0.3, 0.5, 0.7, 0.9]
2. 初始化：Beta(1,1) 均匀先验
3. 采样与决策：为每个臂采样，选最大值服务本次请求
4. 反馈更新：有点击 alpha+=1，无点击 beta+=1
5. 衰减机制：定期乘以衰减因子，让模型"忘记"旧数据
```

MAB vs A/B 适用场景：
- A/B 适合最终决策（需要严格统计检验）
- MAB 适合持续优化、冷启动探索、参数调优
- 工业实践：MAB 探索 → 找到优势策略 → A/B 测试确认

## 10. 面试高频问题速查

Q: p 值是什么含义？
A: 在零假设（无差异）为真时，观测到当前或更极端结果的概率。不是"策略有效的概率"。

Q: 为什么不能"多看几天直到显著"？
A: Peeking Problem，反复检验会膨胀假阳性率。解决：预设实验时长 + 序贯检验。

Q: A/B 测试和 MAB 的区别？
A: A/B 固定分流、实验后评估、统计严谨；MAB 动态分流、边实验边优化、效率高。A/B 适合最终决策，MAB 适合持续优化。

Q: 如何处理实验组和对照组样本量不均衡？
A: 降低统计功效、引入选择偏差、触发辛普森悖论。预防：样本量计算器 + 分桶逻辑用哈希保证均匀 + 实时监控样本分布。
