# GNN 经典模型与大规模训练

## 1. 图卷积网络基础

### 1.1 核心机制

消息传递(Message Passing) + 邻域聚合(Neighborhood Aggregation)：
```
每一层: 节点从邻居收集信息 → 聚合 → 更新自身表示
L层 GNN = 捕捉L跳邻居的信息
```

用户-物品交互天然构成二部图，GNN 通过消息传递捕捉高阶协同信号：
```
A喜欢X, B也喜欢X和Y → GNN 传播后 → A可能喜欢Y
```

### 1.2 邻接矩阵归一化

```
对称归一化: A_norm = D^(-1/2) * A * D^(-1/2)
其中 D 是度矩阵

作用: 防止高度数节点主导信息传播
直觉: 每条边的消息按两端节点度数的几何平均加权
```

## 2. 经典模型详解

### 2.1 GCN (Graph Convolutional Network)

```
H^(l+1) = sigma(A_norm * H^(l) * W^(l))

A_norm: 归一化邻接矩阵(含自环)
H^(l): 第l层节点特征矩阵
W^(l): 可学习权重矩阵
sigma: 非线性激活(ReLU)
```

- 需要全图矩阵运算，内存开销大
- 适合小规模图（< 10万节点）
- 推荐场景中常作为基础组件

### 2.2 NGCF (Neural Graph Collaborative Filtering)

```
h_u^(l) = sigma(W1 * h_u^(l-1)
  + sum_{i in N(u)} 1/sqrt(|N(u)|*|N(i)|)
    * (W1 * h_i^(l-1) + W2 * (h_u^(l-1) odot h_i^(l-1))))
```

核心创新：
- 在用户-物品二部图上做多层图卷积
- 聚合时引入非线性变换和特征交互项 (h_u odot h_i)
- 显式捕捉高阶连通性

缺点：
- 过多非线性变换和特征交互引入噪声
- 被更简单的 LightGCN 超越

### 2.3 LightGCN

```
h_u^(l) = sum_{i in N(u)} 1/sqrt(|N(u)|*|N(i)|) * h_i^(l-1)
h_i^(l) = sum_{u in N(i)} 1/sqrt(|N(i)|*|N(u)|) * h_u^(l-1)

最终表示 = sum_{l=0}^{L} alpha_l * h^(l)  (层组合)
```

核心简化：
- 去除非线性激活函数
- 去除特征变换矩阵 W
- 只保留邻域聚合 + 对称归一化

为什么效果更好：
- 推荐是 ID-based 任务（不是内容理解），不需要复杂特征变换
- 非线性变换可能破坏 collaborative signal
- 层组合(layer combination)才是关键：融合不同跳数的信息

工业界广泛使用的基线，简单且强大。

### 2.4 GraphSAGE

```
1. 邻居采样: 每层随机采固定数量邻居(如10-25个)
2. 聚合: h_N^(l) = AGGREGATE({h_v^(l-1), v in Sample(N(u))})
3. 更新: h_u^(l) = sigma(W * [h_u^(l-1); h_N^(l)])
```

聚合函数选择：
```
函数     特点                    适用
Mean     简单，信息有损          一般场景
LSTM     有序，表达力强          有顺序关系的图
Pooling  先变换再取max，非线性    需要突出关键邻居
```

核心创新：
- 归纳式学习(Inductive)：可处理新节点
- mini-batch 训练可行（不需全图）
- 适合大规模动态图

### 2.5 GAT (Graph Attention Network)

```
alpha_ij = softmax(LeakyReLU(a^T * [W*h_i; W*h_j]))
h_i^(l) = sigma(sum_j alpha_ij * W * h_j^(l-1))
```

- 注意力权重自适应学习邻居重要性
- 多头注意力增强稳定性
- 适合邻居异质性大的场景（社交网络中不同好友影响不同）

### 2.6 模型对比总结

```
模型        核心特点          适用规模    推荐效果
GCN        全图卷积           小图        基线
NGCF       非线性+交互        中图        被LightGCN超越
LightGCN   极简聚合           中大图      强基线
GraphSAGE  采样+归纳学习      大图        适合新节点
GAT        注意力加权         中图        社交推荐首选
```

## 3. 大规模 GNN 采样策略

核心问题：全图训练内存爆炸 + 邻居数指数增长(L层采样, 每层K个 → K^L)

### 3.1 节点采样 (GraphSAGE)

```
每层为每个节点随机采样固定数量邻居
如: L=2, K=10 → 每个目标节点只需 10+100=110 个邻居
```
- 优点：控制计算量，支持 mini-batch
- 缺点：引入方差，采样邻居可能不具代表性
- 适合：百万级节点

### 3.2 层采样 (FastGCN)

```
每层独立采样一批节点作为该层代表
采样概率正比于节点度数(重要性采样)
```
- 优点：每层计算量固定，效率高
- 缺点：层间节点不一致，可能断开消息传递路径
- 适合：需要严格控制计算资源

### 3.3 子图采样 (Cluster-GCN)

```
METIS图分区 → 大图切成多个子图(cluster)
每次在一个cluster内做完整GNN训练
改进: 随机组合多个cluster训练
```
- 优点：子图内消息传递完整，无信息损失
- 缺点：跨子图边被忽略，丢失重要跨区连接
- 适合：亿级节点

### 3.4 对比总结

```
策略       核心思想         优点              缺点              规模
节点采样   每层采邻居       灵活mini-batch     方差大            百万
层采样     每层采节点       计算量固定         可能断路          百万
子图采样   图分区+子图训练  子图内完整传递     丢跨区边          亿级
```

## 4. 分布式 GNN 训练

### 4.1 节点分区

- 节点分配到不同机器，每台持有部分节点和边
- 跨分区边需通信（Halo 节点 = 远程节点的本地副本）
- 适合节点特征大、边少的场景

### 4.2 边分区

- 边分配到不同机器
- 同一节点可能出现在多台机器上（需同步梯度/表示）
- 适合边密集场景

### 4.3 通信瓶颈与解决

主要瓶颈：Halo 节点同步
- 每层 GNN 计算前需拉取远程节点的最新表示
- 解决：增大分区减少切边数 + 异步更新 + 压缩通信

## 5. 工业实践

### 5.1 两阶段架构

```
离线: GNN 在全图上训练 → 生成用户/物品 Embedding → 写入向量数据库
在线: 用户请求 → 取用户Embedding → ANN检索相似物品 → 返回推荐列表
```

GNN 不直接用于在线推理（太慢），而是定期（每天/每小时）更新 Embedding。

### 5.2 增量更新

- 新用户/物品通过邻居 Embedding 聚合快速生成表示
- GraphSAGE 天然支持归纳式推理
- 增量图更新：仅对新增/变化的节点重新计算 Embedding

## 6. GNN 过平滑问题

### 6.1 问题描述

层数增多 → 所有节点表示趋同 → 区分能力下降
原因：多层聚合等价于图上的低通滤波，高频信号被抹平

### 6.2 解决方案

```
方法           思路                    效果
残差连接       h^(l) = f(h^(l-1)) + h^(l-1)     保留原始信息
层归一化       每层后做归一化                      稳定训练
JKNet          拼接各层输出做最终表示               保留多尺度信息
DropEdge       训练时随机删边                      减缓过平滑
层数控制       推荐场景一般2-3层就够               简单有效
```

实践经验：推荐场景中 2-3 层 GNN 效果最好，更深反而降低。

## 7. 模型压缩与加速

### 7.1 知识蒸馏（最常用）

```
大GNN教师模型 → 小MLP学生模型
学生学习: 教师的Embedding 或 预测分布
在线部署用学生模型: 速度快几个数量级
```

### 7.2 剪枝

- 结构化剪枝：去除整个注意力头或 Expert
- 图剪枝：移除不重要的边，减少消息传递量
- 非结构化剪枝：将小权重置零

### 7.3 量化

- FP32 → INT8/FP16
- 量化感知训练(QAT) 比训练后量化(PTQ) 精度损失更小

## 面试高频问题

Q: LightGCN 为什么去掉非线性效果更好？
A: 推荐是 ID-based 任务，核心是学习 user/item ID Embedding 的协同信号。非线性变换（本为特征提取设计）反而会破坏这些协同信号。简单的归一化聚合已经足够。

Q: 三种采样策略对比？
A: 节点采样(GraphSAGE)灵活但方差大；层采样(FastGCN)效率高但可能断路；子图采样(Cluster-GCN)传递完整但丢跨区边。选择取决于图规模和计算资源。

Q: GNN 如何解决冷启动？
A: 通过内容特征和图结构为新节点聚合邻居信息。GraphSAGE 天然支持归纳推理：新节点只需连接到已有节点，通过邻居聚合即可生成表示。

Q: 分布式 GNN 的通信瓶颈？
A: Halo 节点同步。每层计算前需拉取远程节点最新表示。解决：增大分区减少切边、异步更新容忍短暂不一致、梯度压缩减少通信量。

Q: 过平滑怎么办？
A: 推荐场景控制在2-3层就够。如果需要更深层：残差连接保留原始信息 + JKNet 拼接各层输出 + DropEdge 训练时随机删边。
