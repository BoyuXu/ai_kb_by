## 核心概念

GNN 在推荐中的价值：用户-物品交互天然构成二部图，GNN 通过消息传递捕捉高阶协同信号（A 喜欢 X，B 也喜欢 X 和 Y，则 A 可能喜欢 Y）。

核心机制：消息传递(Message Passing) + 邻域聚合(Neighborhood Aggregation)
- 每一层：节点从邻居收集信息，更新自身表示
- L 层 GNN = 捕捉 L 跳邻居的信息

## 经典模型

### NGCF (Neural Graph Collaborative Filtering)
- 在用户-物品二部图上做多层图卷积
- 每层聚合邻居信息时引入非线性变换和交互项
- h_u^(l) = sigma(W1 * h_u^(l-1) + sum_{i in N(u)} 1/sqrt(|N(u)||N(i)|) * (W1 * h_i^(l-1) + W2 * (h_u^(l-1) odot h_i^(l-1))))
- 显式捕捉高阶连通性
- 缺点：过多非线性变换和特征交互可能引入噪声

### LightGCN
- 简化 NGCF：去除非线性激活和特征变换矩阵
- 只保留邻域聚合操作：h_u^(l) = sum_{i in N(u)} 1/sqrt(|N(u)||N(i)|) * h_i^(l-1)
- 最终表示 = 各层输出的加权求和（层组合）
- 效果反而优于 NGCF，说明推荐场景中简单聚合就够了
- 工业界广泛使用的基线

### GraphSAGE
- 核心创新：归纳式学习（Inductive），可处理新节点
- 邻居采样：每层随机采固定数量邻居（如 10-25 个）
- 聚合函数：Mean / LSTM / Pooling
- 训练时不需全图，mini-batch 可行
- 适合大规模动态图

## 大规模 GNN 采样策略

核心问题：全图训练内存爆炸 + 邻居爆炸（L 层采样，邻居数指数增长）。

### 节点采样 (GraphSAGE)
- 每层为每个节点随机采样固定数量邻居
- 优点：控制计算量，支持 mini-batch
- 缺点：引入方差，采样到的邻居可能不具代表性

### 层采样 (FastGCN)
- 每层独立采样一批节点作为该层的代表
- 用重要性采样降低方差：采样概率正比于节点度数
- 优点：每层计算量固定，效率高
- 缺点：层间节点不一致，可能断开消息传递路径

### 子图采样 (Cluster-GCN)
- 先用图分区算法（如 METIS）将大图切成子图
- 每次在一个子图（cluster）内做完整 GNN 训练
- 优点：子图内消息传递完整，无信息损失
- 缺点：跨子图边被忽略，可能丢失重要连接
- 改进：随机组合多个 cluster 训练

### 对比总结
- 小图：全图训练（GCN/LightGCN）
- 中图（百万节点）：GraphSAGE 节点采样
- 大图（亿级）：Cluster-GCN 或分布式训练

## 社交网络 + GNN

### 异构图建模
- 多种节点类型：用户、物品
- 多种边类型：用户-物品交互、用户-用户社交关系
- 不同关系使用不同的消息传递函数或变换矩阵

### 社交信息融合策略
- 注意力机制区分社交邻居的影响力权重
- 社交关系 != 兴趣一致：并非所有好友都有相似偏好
- GAT 学习哪些社交连接对推荐有用

### DiffNet / DiffNet++
- 在社交网络和兴趣网络上分别做信息传播
- 将社交影响力建模为递归扩散过程
- 融合两路信号得到最终用户表示

## 动态图神经网络

用户兴趣和社交关系随时间演化，静态图无法捕捉。

### 时序编码
- 为每条边附加时间戳
- 时间编码函数：将时间差映射为向量（类似位置编码）
- 消息传递时考虑时间衰减

### 记忆网络
- 每个节点维护一个记忆状态（如 GRU/LSTM 隐状态）
- 新交互发生时更新相关节点的记忆
- TGAT, TGN 等模型

### 时间衰减注意力
- 注意力权重考虑时间距离：越近的交互权重越大
- 结合 GAT + 时间衰减函数

## 分布式 GNN 训练

### 节点分区
- 将节点分配到不同机器，每台机器持有一部分节点和边
- 跨分区边需要通信（Halo 节点）
- 适合节点特征大、边少的场景

### 边分区
- 将边分配到不同机器
- 同一节点可能出现在多台机器上（需同步）
- 适合边密集的场景

### 工业实践
- 两阶段架构：离线 GNN 生成 Embedding -> 在线 ANN 检索
- GNN 不直接用于在线推理（太慢），而是定期更新 Embedding
- 增量更新：新用户/物品通过邻居 Embedding 聚合快速生成表示

## GNN 的公平性与偏差

### 偏差来源
- 流行度偏差放大：高度数节点聚合更多信息，表示更丰富，导致马太效应
- 结构偏差：图结构中的同质性（Homophily）使 GNN 倾向推荐相似群体偏好的物品
- 数据偏差继承：交互数据本身的偏差通过图传播被放大

### 去偏方法
- 对抗训练：引入判别器，使 Embedding 无法区分敏感属性
- 注意力重加权：降低偏差连接的聚合权重
- 公平性约束：在损失函数中加入公平性正则项
- 逆倾向加权：对低曝光物品的交互赋予更高权重

## GNN 可解释性

### GAT 注意力可视化
- 直接展示哪些邻居对最终推荐影响最大
- 自然的解释：推荐 Y 是因为你喜欢的 X 与 Y 有强关联

### GNNExplainer
- 为每个预测找到最重要的子图和特征
- 优化一个子图掩码，使保留的子图能复现原始预测
- 输出：关键节点 + 关键边 + 关键特征

### 代理模型
- 用简单可解释模型（决策树）拟合 GNN 的局部行为
- 牺牲保真度换取可解释性

## 模型压缩与加速

### 剪枝
- 结构化剪枝：去除整个注意力头或 Expert
- 非结构化剪枝：将小权重置零
- 图剪枝：移除不重要的边减少消息传递量

### 量化
- FP32 -> INT8/FP16，减少存储和计算
- 量化感知训练(QAT)比训练后量化(PTQ)精度损失更小

### 知识蒸馏
- 大 GNN 教师模型 -> 小 MLP 学生模型
- 学生模型学习教师的 Embedding 或预测分布
- 在线部署用学生模型，速度快几个数量级

## 面试高频考点

1. LightGCN 为什么去掉非线性效果更好：推荐是 ID-based，不需要复杂特征变换
2. 三种采样策略对比：节点(方差大)、层(可能断路)、子图(丢跨区边)
3. 动态图建模思路：时间编码 + 记忆网络 + 衰减注意力
4. GNN 如何解决冷启动：通过内容特征和图结构为新节点聚合邻居信息
5. 分布式 GNN 的通信瓶颈：Halo 节点同步，解决方案是增大分区减少切边
6. GNN 的公平性：流行度放大 + 同质性偏差，用对抗训练/公平约束缓解
