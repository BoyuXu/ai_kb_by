# 社交推荐与 GNN 进阶话题

## 1. 异构图建模

### 1.1 推荐系统中的异构图

```
节点类型: 用户(U), 物品(I), 类目(C), 品牌(B), ...
边类型:
  U--点击-->I    U--购买-->I    U--关注-->U
  I--属于-->C    I--品牌-->B    U--评论-->I
```

每种边类型使用不同的消息传递函数或变换矩阵：
```
h_u^(l) = AGG(
  W_click * AGG_click({h_i : i in N_click(u)}),
  W_social * AGG_social({h_v : v in N_social(u)}),
  W_buy * AGG_buy({h_j : j in N_buy(u)})
)
```

### 1.2 异构图注意力网络 (HAN)

```
层级注意力:
  1) 节点级注意力: 同一元路径下不同邻居的权重
  2) 语义级注意力: 不同元路径(关系类型)的权重

元路径示例:
  U-I-U: 用户-物品-用户 (行为相似用户)
  U-I-C-I: 用户-物品-类目-物品 (同类目扩展)
  U-U-I: 用户-好友-物品 (社交推荐)
```

## 2. 社交信息融合

### 2.1 核心洞察

社交关系 ≠ 兴趣一致：并非所有好友都有相似偏好
- 同事：工作关系，兴趣可能完全不同
- 家人：年龄差大，偏好差异明显
- 兴趣好友：共同爱好，推荐价值高

因此需要注意力机制区分社交邻居的影响力权重。

### 2.2 DiffNet / DiffNet++

DiffNet：
```
社交网络:  h_u^social = sum_{v in Friends(u)} alpha_v * h_v
兴趣网络:  h_u^interest = sum_{i in Items(u)} beta_i * h_i
融合:      h_u = f(h_u^social, h_u^interest)
```

核心思想：
- 在社交网络和兴趣网络上分别做信息传播
- 将社交影响力建模为递归扩散过程
- 每一层模拟一跳社交影响的传播

DiffNet++改进：
- 引入用户-物品交互图的信息传播
- 双向扩散：用户影响好友，好友的行为也反向影响用户
- 门控机制控制社交信号和兴趣信号的融合比例

### 2.3 社交推荐中的注意力设计

```
对于用户u的社交邻居v:
alpha_uv = softmax(
  LeakyReLU(a^T * [W_u * h_u; W_v * h_v; e_uv])
)

e_uv: 社交关系强度特征(互动频率、认识时长等)
```

GAT 学习哪些社交连接对推荐有用：
- 高权重：兴趣相似的好友
- 低权重：仅有社交关系但偏好不同的好友

## 3. 动态图神经网络

### 3.1 为什么需要动态图

用户兴趣和社交关系随时间演化：
- 用户可能从学生变成职场人，兴趣转变
- 社交关系有周期性（工作日 vs 周末互动模式不同）
- 静态图无法捕捉这些变化

### 3.2 时序编码

```
为每条边附加时间戳 t
时间编码: phi(delta_t) = [cos(w1*delta_t), sin(w1*delta_t), ..., cos(wd*delta_t), sin(wd*delta_t)]
类似Transformer的位置编码

消息传递时: msg = W * [h_v; phi(t_now - t_edge)]
```

### 3.3 记忆网络

```
每个节点维护记忆状态 s_u (GRU/LSTM隐状态)
新交互发生: s_u = GRU(s_u, msg_interaction)
                   ↑旧记忆  ↑新交互信息
```

代表模型：
- TGAT (Temporal Graph Attention Network)：时序注意力
- TGN (Temporal Graph Network)：记忆模块 + 图注意力

### 3.4 时间衰减注意力

```
alpha_ij = softmax(
  score(h_i, h_j) * decay(t_now - t_edge)
)

decay(delta_t) = exp(-lambda * delta_t)
```

越近的交互权重越大，远期交互逐渐衰减。

## 4. GNN 的公平性与偏差

### 4.1 偏差来源

流行度偏差放大：
- 高度数节点聚合更多信息 → 表示更丰富 → 被更多推荐 → 马太效应
- L层GNN后，热门物品与更多用户"连通"，冷门物品被边缘化

结构偏差：
- 图结构中的同质性(Homophily)：相似用户倾向互相连接
- GNN 传播后放大这种同质性 → 推荐结果向主流群体偏好倾斜

数据偏差继承：
- 交互数据本身的偏差（位置、曝光）通过图传播被放大

### 4.2 去偏方法

```
方法          思路                          实现
对抗训练      Embedding不可区分敏感属性      判别器猜性别/年龄，GNN让它猜不准
注意力重加权  降低偏差连接的聚合权重          低频物品邻居给更高注意力
公平性约束    损失函数加公平正则项            L = L_rec + lambda * L_fair
逆倾向加权    低曝光物品交互赋更高权重        weight = 1/P(exposure)
```

### 4.3 流行度去偏实践

```
训练时: 对长尾物品的正样本加权 w_i = (max_degree/degree_i)^beta
推理时: score_debiased = score_raw - alpha * log(popularity)
评估时: 分群评估（热门/温和/冷门物品分别计算指标）
```

## 5. GNN 可解释性

### 5.1 GAT 注意力可视化

- 直接展示哪些邻居对最终推荐影响最大
- 自然的解释："推荐Y是因为你喜欢的X与Y有强关联"
- 局限：注意力权重不等于因果关系

### 5.2 GNNExplainer

```
为每个预测找最重要的子图和特征:
1. 初始化子图掩码 M (边掩码)
2. 优化: min -MI(Y, (G_s, X_s))
   即找到使预测结果最大化保留的最小子图
3. 输出: 关键节点 + 关键边 + 关键特征
```

优势：模型无关，可应用于任何 GNN
缺点：每个预测需要独立优化，推理开销大

### 5.3 基于路径的解释

```
用户U --购买--> 物品A --同导演--> 物品B
解释: "推荐B因为您购买过同导演的A"
```

- 在知识图谱上枚举用户到推荐物品的路径
- 用可靠性模型评分各路径
- 选择最可靠且最短的路径作为解释

## 6. GNN + LLM 融合（前沿方向）

### 6.1 融合架构

```
方式1: LLM as Feature Extractor
  物品标题 → LLM → 文本Embedding → 作为GNN节点初始特征 → GNN聚合

方式2: GNN as Structure Encoder
  GNN学图结构 → 结构Embedding → 与LLM语义Embedding拼接 → 下游任务

方式3: 联合训练 (成本极高)
  LLM + GNN 端到端，共享梯度
  实际用LoRA冻结LLM大部分参数
```

### 6.2 协同训练策略

三阶段分阶段预热+LoRA微调：
```
阶段1: 独立预热
  - 通用语料预训练LLM（或用开源底座）
  - 业务数据独立训练GNN至收敛

阶段2: 特征交互对齐
  - 冻结LLM大部分参数，仅用适配器层提取语义表征
  - 将LLM语义特征注入已预热的GNN，第二轮微调
  - 目标：让GNN学会利用LLM的高质量语义信息

阶段3: 轻量级端到端协同微调
  - 用LoRA对LLM进行参数高效微调
  - 损失 = 推荐任务损失(交叉熵) + 可选语言建模辅助损失
  - 辅助损失防止LLM语义能力灾难性遗忘
```

### 6.3 核心挑战

- 训练目标不一致：LLM追求语言建模，GNN追求推荐优化
- 计算资源冲突：LLM参数巨大，与GNN联合训练内存压力大
- 数据形态差异：LLM处理文本序列，GNN处理图结构

## 7. GNN 冷启动解决方案

### 7.1 新用户冷启动

```
策略1: 注册信息编码
  年龄/性别/位置 → 特征向量 → 查找相似用户群 → 聚合群体Embedding

策略2: 首次行为快速更新
  第一次点击/搜索 → 连接到物品节点 → 1-hop聚合生成初始用户表示
  后续每次行为增量更新

策略3: 内容特征泛化
  用户搜索词 → BERT编码 → 与相似搜索词的老用户匹配
```

### 7.2 新物品冷启动

```
策略1: GraphSAGE归纳推理
  新物品连接到已有类目/品牌节点 → 聚合邻居生成初始表示

策略2: 内容Embedding
  标题+描述 → BERT/LLM → 内容向量 → 与行为Embedding空间对齐(投影)

策略3: 元学习
  学习"如何从少量交互快速学习"的元知识
  few-shot适配：5-10次交互后接近正常物品的推荐效果
```

## 面试高频问题

Q: 社交关系如何正确融入推荐？
A: 核心在于区分有效和无效的社交信号。用注意力机制学习每个好友对推荐的贡献权重，而不是平等对待所有好友。DiffNet 在社交网络和兴趣网络上分别传播再融合是典型方案。

Q: 动态图建模的核心思路？
A: 三个组件：(1)时间编码将时间差映射为向量 (2)记忆网络(GRU)维护节点状态随交互更新 (3)衰减注意力让近期交互权重更大。代表模型 TGN。

Q: GNN 的公平性问题怎么解决？
A: 流行度放大是核心问题。训练时用逆倾向加权提升长尾物品权重；推理时用流行度惩罚项校正分数；评估时分群考察热门/冷门物品的独立指标。

Q: GNN + LLM 融合的实际落地方式？
A: 最实际的是 LLM 离线生成文本Embedding → 作为GNN节点初始特征。端到端联合训练用LoRA控制成本，三阶段流程（独立预热 → 特征对齐 → 轻量联合微调）降低训练难度。

Q: GNN 冷启动 vs 协同过滤冷启动？
A: GNN天然优势：新节点只需连接到已有节点（类目/品牌），通过邻居聚合即可生成初始表示（GraphSAGE归纳推理）。协同过滤必须有足够交互数据才能工作。
