## L1 概念卡（What）
> 目标：30秒内说清楚，带类比，不用公式

---

### AB-L1-001
**Q：什么是 Auto Bidding？和手动出价的本质区别？**

**A（30秒版）：**
Auto Bidding 是让平台 AI 代替广告主做出价决策。
- **手动出价**：你盯着数据报表，每周调一次价，拍脑袋定个数字
- **Auto Bidding**：你只告诉平台"我每个转化最多花50元"，AI 根据这次流量的质量自动决定出多少

**记忆锚点：** 就像打滴滴——你不用管走哪条路、哪个司机便宜，你只说"送我去机场，预计费用 XX 元"，平台搞定一切。

---

### AB-L1-002
**Q：什么是 eCPM？为什么用 eCPM 排序而不是直接用出价？**

**A（30秒版）：**
eCPM = 每千次展示的期望收益 = `bid × pCTR × quality_score × 1000`

如果只按出价排序，一个出价 10 元但 CTR 只有 0.1% 的广告，会排在出价 5 元但 CTR 2% 的广告前面——但后者给平台带来更多收入。eCPM 把"出价"和"点击概率"合并，让真正有价值的广告胜出。

**记忆锚点：** eCPM 是"性价比排名"，不是"出价排名"。

---

### AB-L1-003
**Q：oCPC 和 oCPA 有什么区别？**

**A（30秒版）：**
- **oCPC**（Optimized CPC）：广告主以点击为计费单位，但平台内部优化 CPA（转化成本）
- **oCPA**（Optimized CPA）：广告主直接以转化为目标出价，平台全权负责出价

简单说：oCPC 是"我出钱买点击，但你帮我优化转化"；oCPA 是"我只管目标转化成本，其他都交给你"。oCPA 更彻底的自动化。

**记忆锚点：** C = Click（点击），A = Action（转化行为）。

---

### AB-L1-004
**Q：PID Pacing 是用来解决什么问题的？**

**A（30秒版）：**
广告预算如果不加控制，会在流量多（但未必质量好）的时间段瞬间烧完，导致后半天没钱买高质量流量。

PID Pacing 是一个自动控制系统，让实际花钱的节奏跟着"理想曲线"走——比如全天预算1万元，上午10点应该花了4000元，如果只花了3000，PID 就自动提高出价；如果花了5000，就降低出价。

**记忆锚点：** 就像驾驶员踩油门/刹车保持匀速，而不是一脚油门踩到底。

---

### AB-L1-005
**Q：什么是 Bid Shading？在什么场景下使用？**

**A（30秒版）：**
Bid Shading（出价压缩）是程序化广告（DSP）中的策略：在"一价拍卖"中，你出多少就付多少，所以聪明的做法是压低出价，只出"刚好能赢"的价格，而不是出你的真实价值。

公式：`shaded_bid = true_value × (1 - shading_factor)`

**使用场景：** OpenRTB 从 2nd price 切换到 1st price 之后，所有 DSP 都需要 Bid Shading，否则会过度付费。

**记忆锚点：** 二价拍卖可以说真话，一价拍卖要"留一手"。

---

### AB-L1-006
**Q：为什么 RL 要把出价建模成 MDP（序列决策问题）？**

**A（30秒版）：**
一天内有百万次竞价机会，每次出价决策都会影响预算余量，进而影响后续出价空间。这是一个**前后关联的序列问题**，不是独立的单次决策。

举例：上午竞争激烈，是否多花钱？如果多花了，下午预算不够；如果少花，错过好流量。RL 的 MDP 能把"当前决策对未来的影响"建模进去，这是 oCPC 这类单步决策模型做不到的。

**记忆锚点：** oCPC 是"一次性选择题"，RL 是"下棋——每步都影响局势"。

---

### AB-L1-007
**Q：MARL 出价和单 Agent RL 出价有什么区别？**

**A（30秒版）：**
单 Agent RL 把其他广告主的行为当作"固定的环境背景"，但实际上竞争对手会根据你的行为调整策略。MARL（多智能体RL）把每个广告主都当成独立的决策者，建模彼此的博弈关系。

**记忆锚点：** 单 Agent RL 是"对着假人练拳"，MARL 是"真人对打"。

---

### AB-L1-008
**Q：Auto Bidding 系统里的"Bid Simulator"是什么，有什么用？**

**A（30秒版）：**
Bid Simulator（出价模拟器）是一个离线仿真环境，用历史竞价数据重播（log replay）来模拟真实拍卖场景。RL Agent 在模拟器中训练，而不是直接在线上试错——因为线上试错代价极高（出价 0.01 元可能拿不到任何流量）。

**记忆锚点：** 就像飞行模拟器，飞行员先在模拟器里练，不行了重来，不会真的坠机。

## L2 原理卡（How）
> 目标：说清楚核心机制，能写出公式，每张有"一句话记忆"

---

### AB-L2-001
**Q：oCPA 的最优出价公式是怎么来的？**

**A（原理版）：**

每次展示的期望收益：
```
V_i = CPA_target × pCTR_i × pCVR_i
```

在二价拍卖中，出价只要等于期望收益就能最大化盈余：
```
bid*_i = CPA_target × pCTR_i × pCVR_i
```

引入预算调节系数 k（PID 输出）：
```
bid_i = k × CPA_target × pCVR_i
```

（其中 k 由 PID Pacing 控制器实时调整，k>1 提高出价消耗预算，k<1 降低出价节省预算）

**一句话记忆：** 出价 = 目标成本 × 转化概率 × 预算松紧系数

---

### AB-L2-002
**Q：KKT 最优出价是怎么推导的？结论是什么？**

**A（原理版）：**

**问题：** 在预算 B 内，最大化总转化价值

$$
\max \sum_i v_i \cdot w_i(b_i) \quad \text{s.t.} \quad \sum_i m_i \cdot w_i(b_i) \leq B
$$

**拉格朗日函数：**
$$
\mathcal{L} = \sum_i v_i w_i(b_i) - \lambda(\sum_i m_i w_i(b_i) - B)
$$

**对 b_i 求导 = 0：**
$$
w_i'(b_i)(v_i - \lambda m_i) = 0
$$

**结论（因为 w'_i > 0）：**
$$
b^*_i = \frac{v_i}{\lambda}
$$

λ 是"预算影子价格"，反映预算紧张程度。用二分搜索找满足预算约束的 λ。

**一句话记忆：** 最优出价 = 价值 ÷ 预算紧张程度（λ越大预算越紧，出价越保守）

---

### AB-L2-003
**Q：PID 三个分量各自做什么？为什么需要 Anti-windup？**

**A（原理版）：**

```
e(t) = 目标消耗 - 实际消耗

P（比例）= Kp × e(t)         → 快速响应当前偏差，但单靠P会有稳态误差
I（积分）= Ki × ∑e(τ)Δτ     → 消除历史累积误差，但可能超调
D（微分）= Kd × Δe/Δt       → 预测趋势，抑制震荡，对噪声敏感

bid_multiplier = 1 + P + I + D
```

**Anti-windup 必要性：**
当预算耗尽后，实际消耗 = 0，误差 e(t) 持续为正，积分项 I 无限累积（windup）。
一旦预算恢复，I 的累积值会导致剧烈超调。
解决：当出价达到上下限时，停止积分项更新。

**一句话记忆：** P 管现在，I 管历史，D 管未来；Anti-windup 防止历史账单在不该时候结清。

---

### AB-L2-004
**Q：RL 出价的 MDP 建模：state/action/reward 各包含什么？**

**A（原理版）：**

**State（状态）：**
```
s_t = {
    # 当前竞价机会特征
    pCTR, pCVR, ad_quality, user_features,
    
    # 预算状态
    budget_remaining_ratio,   # 剩余预算/总预算
    time_remaining_ratio,     # 剩余时间/总时间
    expected_budget_ratio,    # 当前应消耗比 = time_used/total_time
    
    # 累计效果
    actual_CPA_so_far,        # 当前实际CPA
    conversions_so_far,       # 已积累转化数
    
    # 市场状态
    recent_win_rate,          # 近期赢率
    recent_market_price       # 近期市场均价
}
```

**Action（动作）：**
```
# 方案A：连续出价乘数（DDPG）
a_t = λ ∈ [0.1, 3.0]，最终出价 = λ × base_bid

# 方案B：离散档位（DQN）
a_t ∈ {0.5x, 0.7x, 1.0x, 1.5x, 2.0x}

# 方案C：分层（粗粒度campaign + 细粒度impression）
```

**Reward（奖励）：**
```
r_t = v_t × w_t - cost_t - penalty_t

# v_t：赢得展示的转化价值（pCTR × pCVR × CPA_target）
# cost_t：实际付款（清结价）
# penalty_t：预算约束惩罚项（超预算则 penalty 大）

# 延迟转化处理：用即时 pCVR 估计，转化到达后修正
```

**一句话记忆：** State = 当前+预算+市场，Action = 出价倍率，Reward = 价值-成本-惩罚。

### AB-L2-005
**Q：DQN / PPO / DDPG 在出价中各有什么优劣？怎么选？**

**A（原理版）：**

| 算法 | 动作空间 | 训练方式 | 广告优势 | 广告劣势 |
|------|---------|---------|---------|---------|
| **DQN** | 离散 | Off-policy | 样本效率高，稳定 | 离散化损失精度 |
| **PPO** | 连续/离散 | On-policy | 稳定，clip防止策略崩溃 | 样本效率低 |
| **DDPG** | 连续 | Off-policy | 精细连续出价 | 容易不稳定 |
| **TD3/SAC** | 连续 | Off-policy | DDPG改进版，更稳 | 实现复杂 |

**选型原则：**
- 离散出价档位 → DQN
- 连续出价乘数 + 稳定优先 → PPO
- 连续 + 样本效率优先 → SAC/TD3

**一句话记忆：** DQN 选档位、PPO 求稳定、DDPG 连续精细、SAC 两者兼顾。

---

### AB-L2-006
**Q：ESMM 是什么？为什么能解决 CVR 训练的样本空间偏差？**

**A（原理版）：**

**问题：** CVR 模型在点击样本上训练，但出价决策需要在所有展示（impression）上预测 CVR——训练空间和推理空间不一致，产生 Sample Selection Bias。

**ESMM（Entire Space Multi-Task Model）的解法：**
```
不直接训练 pCVR，而是训练：
  pCTR：在全展示空间（无偏）
  pCTR × pCVR：在全展示空间（无偏）

推导出：
  pCVR = (pCTR × pCVR) / pCTR

Loss = CrossEntropy(pCTR, click_label) + CrossEntropy(pCTCVR, conversion_label)
```

两个 Task 共享底层 User/Item Embedding，通过多任务学习间接在全展示空间学到无偏 CVR。

**一句话记忆：** ESMM 不直接预测 CVR，通过联合学习 CTR 和 CTR×CVR 反推出全空间无偏 CVR。

---

### AB-L2-007
**Q：Bid Landscape 是什么？怎么用于出价优化？**

**A（原理版）：**

Bid Landscape 是"赢率函数"：对于某次竞价机会，出价 b 时，赢得该次展示的概率 w(b)。

**建模方式：**
```
# 历史竞价数据 → 拟合赢率曲线
# 常用模型：Log-logistic / Gamma 分布
w(b) = P(market_price ≤ b) ≈ 1 - exp(-b/μ)  # 指数分布简化版

# 利用赢率函数：
期望价值 = v × w(b) - 期望成本
最优出价 = argmax_b [v × w(b) - E[cost | win, b]]
```

**在 RL 中的作用：**
RL Agent 可以利用 Bid Landscape 进行 off-policy 评估，估计"如果出价 b，期望能赢得什么样的流量"，辅助 Reward 计算和策略评估。

**一句话记忆：** Bid Landscape = 出价-赢率曲线，告诉你"出多少钱能买到多少流量"。

---

### AB-L2-008
**Q：RL 出价如何处理非平稳环境（市场价格随时间变化）？**

**A（原理版）：**

**问题根源：** RL 策略在历史数据上训练，但上线后市场竞争格局已变（对手也在升级策略，节假日流量特征完全不同）。

**主要解法：**
1. **在线持续更新（Online RL）：** 每天/每小时用最新数据 fine-tune 策略，但需控制更新步长防止策略崩溃
2. **将时间信息加入 State：** 一天中的时间段、是否节假日、近期市场均价 → 上下文感知策略
3. **元学习（Meta-RL）：** 训练一个能"快速适应新环境"的元策略，few-shot 迁移
4. **域随机化：** 训练时随机扰动市场价格模型，提升策略的鲁棒性
5. **Sim-to-Real Gap 最小化：** 持续校准 Bid Simulator，使其与真实市场保持同步

**一句话记忆：** 应对非平稳：要么持续更新，要么把"时间"作为 state 教 Agent 认识时间规律。

## L3 决策卡（Why）
> 目标：说清楚"为什么选这个"，含对比分析，每张有"决策口诀"

---

### AB-L3-001
**Q：为什么用 RL 而不是用模型预测出价（oCPC）？oCPC 的核心局限是什么？**

**A（决策框架）：**

**oCPC 的两个致命假设：**
1. **静态假设：** 每次出价独立，不考虑预算动态变化对未来的影响
2. **线性假设：** 出价 = 目标CPA × CVR，假设流量质量均匀，不对流量价值差异化定价

**RL 解决的核心问题：**
- **序列性：** 把预算分配建模为跨时间的序列决策，早期的出价影响晚期的预算空间
- **全局最优：** oCPC 每次独立最优，RL 追求整个 Campaign 周期的累积最优
- **动态自适应：** State 包含实时预算状态和市场信号，策略随环境变化调整

**什么时候 oCPC 足够用？**
- 预算充足、Campaign 短期（单日）、流量质量较均匀 → oCPC 简单可靠
- 预算有限、长周期 Campaign、需要精细分配 → RL 更好

**决策口诀：** oCPC 管单次最优，RL 管全局最优；oCPC 假设静态，RL 处理动态。

---

### AB-L3-002
**Q：GSP 和 VCG 作为竞价机制各有什么优劣？工业界为什么主要用 GSP？**

**A（决策框架）：**

| 维度 | GSP | VCG |
|------|-----|-----|
| 激励相容 | ❌（非占优策略）| ✅（诚实出价最优）|
| 社会福利 | 次优 | 最优 |
| 计算复杂度 | O(n log n) | O(2^n)（需枚举）|
| 平台收入 | 通常更高 | 通常更低 |
| 广告主理解 | 直观易懂 | 难以理解定价逻辑 |

**工业界选 GSP 的原因（Edelman 2007实证）：**
1. 计算简单，毫秒级完成
2. 平台收入更高（广告主有出价竞争动机）
3. 广告主能看懂账单，投诉少
4. 历史惯例：Google AdWords 从2002年起用 GSP，生态已形成

**决策口诀：** 理论选 VCG，工程选 GSP；VCG 诚实但复杂，GSP 简单但博弈。

---

### AB-L3-003
**Q：RL 出价应该用 On-policy（PPO）还是 Off-policy（DQN/SAC）？**

**A（决策框架）：**

**On-policy（PPO）：**
- 每批数据只用一次，用完丢弃
- 策略分布 ≈ 数据分布，训练稳定
- 缺点：样本效率低，每次都要重新采集数据
- **广告场景适合度：** 有实时数据流时合适（每小时更新策略）

**Off-policy（DQN/SAC）：**
- 数据存入 Replay Buffer 反复利用，样本效率高
- 训练数据和当前策略分布可能不一致（distribution shift）
- 需要 Importance Sampling 修正（增加复杂度）
- **广告场景适合度：** 离线训练、历史日志数据量大时合适

**实践中的折中：**
字节 Auto Bidding 使用"对偶方法提供基准 + PPO 微调"：
- 对偶方法提供稳定可解释的出价基准（预算不会炸）
- PPO 在此基础上学习微调量（控制更新幅度，防策略崩溃）

**决策口诀：** 有实时数据流 → PPO；历史数据丰富 → SAC；求稳 → 对偶方法 + RL 微调。

---

### AB-L3-004
**Q：Bid Shading 适用于什么场景？为什么二价拍卖不需要 Bid Shading？**

**A（决策框架）：**

**二价拍卖（Vickrey）的特性：**
出最高价但只付第二高价 → 诚实出价（出真实价值 v）是占优策略。
即使你出 100 元赢了，只付第二名的 80 元。没有动机压低出价。

**一价拍卖的问题：**
出最高价且付你出的价格 → 如果出 100 元赢了，就真的付 100 元。
策略性：只要出"比第二名高一点点"就能赢，没必要出真实价值。

**Bid Shading 的公式：**
```
shaded_bid = v × (1 - optimal_shading_factor)
shading_factor ≈ 1 - E[second_price | win] / v
```

**工业背景：**
2019 年 Google AdManager 宣布从二价切换到一价，行业随之都需要实现 Bid Shading。

**决策口诀：** 二价说真话，一价留余地；一价市场必须 Bid Shading，否则多付冤枉钱。

---

### AB-L3-005
**Q：Auto Bidding 系统应该做 Campaign 级出价还是 Impression 级出价？**

**A（决策框架）：**

**Campaign 级（粗粒度）：**
- 每个 Campaign 一个出价乘数，全局一致
- 低延迟（乘数已算好），工程简单
- 无法区分同一 Campaign 内不同流量的质量差异

**Impression 级（细粒度）：**
- 每次展示机会独立决策出价
- 能充分利用流量差异，理论上最优
- 需要实时推理（<10ms），工程复杂

**字节 Auto Bidding 的分层方案（最优实践）：**
```
Campaign 级（RL）：决定预算节奏 + 全局乘数 λ
     ↓
Impression 级（规则/小模型）：在 λ 基础上微调
     bid_final = λ × base_bid × impression_quality_score
```

**决策口诀：** Campaign 管方向，Impression 管精度；分层设计兼顾效果和延迟。

## L4 场景卡（Scene）
> 目标：真实业务场景，用结构化方式回答

---

### AB-L4-001
**场景：** 广告主反馈"设了100元/转化目标，实际跑出140元/转化，超了40%"

**Q：分析原因，给出排查步骤和解决方案。**

**A（结构化回答）：**

**第一步：分析可能原因（三类）**

**① 模型问题（CVR 高估）**
- 模型预测 pCVR 偏高 → 出价偏高 → 赢了很多"看似高质量但实际转化差"的流量
- 检验：对比 pCVR 预测值 vs 实际 CVR，看是否系统性偏高

**② 竞争环境问题（市场价格上涨）**
- 同行出价大幅提升（节假日/大促），PID Pacing 来不及反应
- 检验：查近期市场均价（market price）趋势，是否有突增

**③ 预算分配问题（集中消耗）**
- 预算在高竞争时段（晚8-10点）过度集中消耗，高价买了差流量
- 检验：按时段拆分 CPA，看是否某时段 CPA 远高于均值

**第二步：排查步骤**
```
1. 拉分时段 CPA 报表 → 定位问题时段
2. 看 CVR 预测准确率（AUC/校准度）→ 判断是否模型问题
3. 看近30天市场均价变化 → 判断是否竞争加剧
4. 看 Pacing 曲线是否平滑 → 判断是否预算堆积
```

**第三步：解决方案**
- 模型问题：重新校准 CVR 模型（Calibration Layer），或降低置信度系数
- 竞争问题：调低 PID K_p，减缓对市场价格上涨的追随速度
- 预算问题：设置时段预算上限（dayparting），或加强 PID 积分项

---

### AB-L4-002
**场景：** 你是某平台广告算法工程师，需要从0设计一个 RL 出价系统

**Q：请描述你的设计方案，包括 MDP 建模、训练方案、上线策略。**

**A（结构化回答）：**

**MDP 建模**
```
State: [pCTR, pCVR, 剩余预算比, 时间进度比, 
        实际CPA/目标CPA, 近期赢率, 近期市场均价]
Action: 出价乘数 λ ∈ [0.1, 3.0]（连续）
Reward: pCVR × CPA_target - cost - max(0, over_budget_penalty)
```

**训练方案（三阶段）**
```
Phase 1：离线预训练（Bid Simulator）
  → 用6个月历史竞价日志 log replay
  → PPO 训练至收敛，得到基础策略 π_0

Phase 2：Shadow Mode 验证
  → 线上真实流量，RL 策略并行计算出价（不实际生效）
  → 比较 RL 出价 vs 现有 oCPA 出价的 Campaign 效果
  → 调整超参，确认无异常出价

Phase 3：A/B 上线
  → 小流量（5%）灰度
  → 监控 CPA 是否在目标 ±20% 以内
  → 逐步扩量至100%
```

**工程保护**
- 出价硬上下限：0.01元 ~ 50元（防止模型异常）
- CPA 超标熔断：当日 CPA > 目标 × 1.5，自动降级到 oCPA
- 模型更新频率：每天增量更新，周度全量重训

### AB-L4-003
**场景：** 你的 RL 出价系统上线后 CPA 达标，但 GMV（广告成交额）比 oCPA 低了5%

**Q：为什么会出现这种情况？如何优化？**

**A（结构化回答）：**

**根本原因分析：**
RL 策略把 CPA 控制得很好，但可能过度"节省"预算——在高价值时段出价保守，错过了高质量但价格稍贵的流量。CPA 控制好 ≠ GMV 最大化。

**两种 Reward 目标的矛盾：**
```
CPA 控制 → 最小化成本/转化
GMV 最大化 → 在预算内最大化转化总量
```
这两者有时是冲突的：高 GMV 需要在高竞争时段出高价抢流量，这会推高单次 CPA。

**解决方案：**
1. **调整 Reward 函数：** 引入转化量的奖励，而不只是 CPA 惩罚
   ```
   reward = conversion_value × w_gmv - λ_cpa × max(0, CPA - target_CPA)
   ```
2. **多约束优化：** 把 CPA 控制作为约束，GMV 作为目标（对偶方法）
3. **分时段策略：** 高流量时段（晚8-10点）允许 CPA 适当超标换取 GMV

### AB-L4-004
**场景：** 公司从二价竞价切换到一价竞价（类似 Google 2019年的变化）

**Q：对 Auto Bidding 系统有哪些影响？需要做哪些改动？**

**A（结构化回答）：**

**核心影响：**
- 二价：赢了付第二名的价，诚实出价最优
- 一价：赢了付你出的价，需要压价（Bid Shading）

**必要改动：**
1. **增加 Bid Shading 模块：**
   ```
   shaded_bid = raw_bid × (1 - shading_factor)
   shading_factor 由 Bid Landscape 估算：历史赢价/出价 的比值
   ```
2. **重新训练 RL 模型：**
   一价和二价的 cost 函数完全不同，原有 RL 模型的 Reward 计算无效，需用新规则重新训练

3. **更新 Bid Simulator：**
   模拟器的 payment 逻辑从二价改为一价，确保训练数据分布一致

4. **监控调整期异常：**
   切换初期出价策略混乱，需重点监控 CPA 波动，设置熔断机制

## L5 陷阱卡（Trap）
> 目标：反直觉结论、面试坑、边界条件

---

### AB-L5-001
**Q（陷阱）：RL 出价训练时，能不能直接用线上真实竞价日志做 Experience Replay？**

**A：**
**不能，原因是 Counterfactual Bias（反事实偏差）问题：**

线上日志只记录了"实际出价 b 赢得的展示"，没有记录"如果出价是 b' 会发生什么"。用这样的数据训练的策略，会系统性地低估低出价动作的价值——因为低出价很少赢，日志里的样本极少。

**正确做法：**
1. 用 Bid Simulator（log replay）模拟反事实场景
2. Importance Sampling 修正 off-policy 数据的分布偏差
3. 收集 Exploration 数据（主动尝试不同出价）补充训练数据

**面试金句：** "直接用线上日志训练 RL 出价，等于只从成功者身上学经验，会系统性忽视低价出价的潜力。"

---

### AB-L5-002
**Q（陷阱）：CVR 预测 AUC 提升了 0.5%，Auto Bidding 的实际 GMV 一定会提升吗？**

**A：**
**不一定，甚至可能下降：**

CVR 模型 AUC 提升说明排序能力变强，但对 Auto Bidding 的影响取决于"校准度（Calibration）"：
- 如果 AUC 提升是靠预测值整体偏移实现的（高估或低估），虽然排序准了，但绝对值偏差会导致出价系统性偏高或偏低
- Auto Bidding 依赖的是 CVR 的**绝对值**（出价 = target_CPA × pCVR），AUC 只衡量相对排序

**正确的评估指标：**
- 对 Auto Bidding 而言，CVR 模型的**校准曲线**（predicted vs actual）比 AUC 更重要
- 应额外关注 log-loss 和 calibration error

**面试金句：** "AUC 好不等于校准好；Auto Bidding 用 CVR 的绝对值出价，校准差会直接导致系统性出价偏差。"

---

### AB-L5-003
**Q（陷阱）：PID Pacing 的积分项 K_i 越大越好吗？能消除更多偏差？**

**A：**
**K_i 过大会导致超调震荡：**

积分项的作用是消除"稳态误差"（持续的消耗不足/过度），但如果 K_i 太大：
1. 对历史误差反应过于激烈
2. 一旦预算消耗超标，K_i 会大幅降价，导致接下来流量买不够，形成震荡
3. 节假日等特殊时段，误差积累量大，K_i 大会触发剧烈调价

**Anti-windup 也解决不了 K_i 过大的问题**（Anti-windup 只防止积分饱和，不防止 K_i 本身过大）。

**实践经验：** 广告 PID 的典型参数 K_p=0.5, K_i=0.1, K_d=0.05，积分项通常比比例项小5-10倍。

**面试金句：** "积分项大消偏差，但大过头会震荡；广告 Pacing 的 K_i 通常是 K_p 的 1/5 到 1/10。"

---

### AB-L5-004
**Q（陷阱）：RL 出价 Agent 在训练时 Reward 一直在上涨，说明策略在持续变好吗？**

**A：**
**不一定，可能是策略学会了"刷 Reward"而非真正优化广告效果：**

常见陷阱：
1. **Reward Hacking：** Agent 发现某些边界情况下（如预算快耗尽时）刷一堆低质量转化来提高 Reward，而实际业务价值很低
2. **Simulator Bias：** Bid Simulator 与真实环境的差距，导致策略在模拟器里 Reward 高，上线后效果差
3. **Mode Collapse：** 策略收敛到一种特定的出价模式，短期 Reward 高但泛化差

**正确的评估方式：**
- 在 Shadow Mode 下用真实数据验证，而不只看 Simulator Reward
- 关注业务指标：CPA/ROI/GMV，而不只是 Reward 曲线
- 设计 Reward 时要与真实业务目标严格对齐（Reward Shaping 谨慎使用）

**面试金句：** "Simulator Reward 上涨 ≠ 上线效果好；RL 出价必须经过 Shadow Mode 验证，Simulator 和真实环境的 Gap 会被策略利用。"

---

### AB-L5-005
**Q（陷阱）：MARL 出价中，如果多个广告主都升级了 RL 策略，整个市场会更好吗？**

**A：**
**不一定，可能导致"军备竞赛"和市场价格整体上涨：**

博弈论分析：
- 如果所有人都用 RL 优化出价，每个人都会更激进地竞价高质量流量
- 整个市场的均衡价格（清结价）可能上涨，平台受益，但广告主的 ROI 集体下降
- 这类似"囚徒困境"：个体理性 → 集体不理性

**实证观察：**
RTB 市场引入 Auto Bidding 后，行业 CPM 普遍上涨（2018-2022），广告主 ROI 整体压缩。

**面试金句：** "RL 出价对单个广告主有利，但如果所有人都用 RL，市场竞争加剧，行业整体 CPM 上涨，个体 ROI 优势消失——这是 MARL 的纳什均衡问题。"

---

## 快速总览（面试前5分钟看这个）

```
Auto Bidding 核心公式：
  bid* = v / λ   （KKT最优，λ=预算影子价格）
  bid  = k × CPA_target × pCVR   （oCPA实用公式）
  eCPM = bid × pCTR × quality   （排序公式）

演进口诀：
  规则 → 模型预测 → 优化方法 → RL → MARL
  手动 → 单步最优 → 约束优化 → 全局最优 → 博弈均衡

RL MDP三要素：
  State = opportunity + budget + market
  Action = bid_multiplier λ
  Reward = value - cost - budget_penalty

三大挑战：
  非平稳 → 在线更新 + 时间入state
  竞争对手 → 均场近似 + MARL
  奖励延迟 → 即时CVR代理 + 事后修正
```
