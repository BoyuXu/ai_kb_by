## L1 概念卡（What）

### 【L1-01】什么是CTR预估？

**定义**：CTR（Click-Through Rate）预估是预测用户在看到某个广告/商品后点击它的概率，输出 ∈ (0,1) 的值。

**核心公式**：
```
CTR = 点击次数 / 曝光次数
```

**在系统中的位置**：
```
召回（Recall）→ 粗排（Prerank）→ 精排（Rank，CTR预估）→ 重排（Rerank）→ 展示
```

**为什么重要**：
- 广告：出价 × CTR = eCPM，决定展示机会
- 推荐：精排核心分数，直接决定用户体验
- 搜索：相关性 + CTR = 综合排序分

**本质**：二分类问题，预测 P(click=1 | user, item, context)

---

### 【L1-02】逻辑回归（LR）

**定义**：最简单的CTR模型，对特征做线性组合后通过Sigmoid激活。

**公式**：
```
P(y=1|x) = σ(w^T x + b) = 1 / (1 + exp(-(w^T x + b)))
```

**特点**：
- ✅ 简单、快速、可解释
- ✅ 支持大规模稀疏特征（ID特征One-Hot）
- ❌ 特征交叉需要手工构造，无法自动学习
- ❌ 线性模型，表达能力受限

**适用场景**：特征维度极高、需要强可解释性、计算资源受限

---

### 【L1-03】因子分解机（FM）

**定义**：Factorization Machine，通过隐向量点积自动学习二阶特征交叉，解决稀疏特征下交叉项参数估计问题。

**核心思想**：每个特征对应一个k维隐向量，两个特征的交叉权重由对应隐向量的内积表示。

**解决的LR痛点**：稀疏数据中 x_i·x_j 通常同时为0，导致交叉项 w_ij 无法训练；FM将其分解为 <v_i, v_j>，借助其他样本学习 v_i 和 v_j。

---

### 【L1-04】Wide & Deep

**定义**：Google 2016年提出的双路架构，Wide路负责记忆（Memorization），Deep路负责泛化（Generalization）。

```
输入特征
   ├── Wide侧（线性LR）：手工交叉特征 → 记忆历史规律
   └── Deep侧（DNN）：  Embedding → MLP → 自动泛化
         ↓
   联合输出：σ(y_wide + y_deep)
```

**典型应用**：Google Play App推荐

---

### 【L1-05】DeepFM

**定义**：华为2017年提出，用FM替换Wide&Deep的Wide侧，实现端到端自动特征交叉，无需手工特征工程。

```
输入特征（共享Embedding）
   ├── FM侧：一阶 + 二阶自动交叉
   └── Deep侧：MLP高阶隐式交叉
         ↓
   联合输出：σ(y_FM + y_Deep)
```

**优势**：FM和Deep共享Embedding，参数高效；无需手工交叉特征。

---

### 【L1-06】深度兴趣网络（DIN）

**定义**：阿里巴巴2018年提出，对用户历史行为序列引入注意力机制，根据目标广告动态聚合用户兴趣。

**核心洞察**：用户兴趣是多样化的，对某商品的点击概率不应平等地受所有历史行为影响，而应聚焦与目标商品相关的历史行为。

**例子**：用户买过手机、书、鞋子，看到手机壳广告时，应该更关注"买手机"这条历史行为。

---

### 【L1-07】深度兴趣演化网络（DIEN）

**定义**：阿里巴巴2019年提出，在DIN基础上增加兴趣演化建模，用GRU捕捉用户兴趣随时间的动态变化。

**两层结构**：
- 兴趣提取层（Interest Extractor）：GRU建模行为序列，提取兴趣状态
- 兴趣演化层（Interest Evolving）：AUGRU（带注意力的GRU），与目标相关地演化兴趣

**相比DIN的进步**：DIN只静态聚合历史，DIEN捕捉兴趣的时序变化规律。

---

### 【L1-08】行为序列Transformer（BST）

**定义**：阿里巴巴2019年提出，将Transformer应用于用户行为序列建模，捕捉序列内部的复杂交互关系。

**与DIN的对比**：
| 维度 | DIN | BST |
|------|-----|-----|
| 序列建模 | 注意力聚合（并行） | Transformer（自注意力+位置编码） |
| 捕捉能力 | 目标与历史的关联 | 序列内部item间的交互关系 |
| 计算复杂度 | O(n) | O(n²) |

---

### 【L1-09】DCN / DCN-V2（深度交叉网络）

**定义**：Google提出的显式高阶特征交叉网络。

- **DCN（2017）**：Cross Network显式进行k阶特征交叉，Cross层数=交叉阶数
- **DCN-V2（2021）**：用矩阵替换向量乘法，提升交叉容量；支持Stacked和Parallel两种结构

**公式（Cross层）**：
```
x_{l+1} = x_0 · x_l^T · w_l + b_l + x_l
```
每一层在上一层基础上增加x_0的交叉，l层Cross=l+1阶交叉。

---

### 【L1-10】多任务学习：MMoE / PLE / ESMM

**MMoE（Google 2018）**：多个Expert网络 + 每个Task独立门控，任务自适应选择Expert组合，解决任务冲突问题。

**PLE（腾讯 2020）**：在MMoE基础上区分共享Expert和任务专属Expert，进一步减少负迁移。

**ESMM（阿里 2018）**：同时预测CTR和CVR，利用 pCVR = pCTR × pCTCVR 约束，在全空间（曝光空间）训练CVR，解决样本选择偏差和稀疏问题。

## L2 原理卡（How）

### 【L2-01】FM二阶交叉：O(n²) → O(nk) 推导

**原始二阶交叉**（O(n²)k）：
```
φ_FM = Σ_{i=1}^{n} Σ_{j=i+1}^{n} <v_i, v_j> · x_i · x_j
```
n个特征，需要计算 C(n,2) = n(n-1)/2 个交叉项，每项内积需要O(k)，总计 O(n²k)。

**优化推导**（利用平方差公式）：

```
2 · Σ_{i<j} <v_i, v_j> · x_i · x_j

= (Σ_i v_i · x_i)² - Σ_i (v_i · x_i)²

= Σ_f [ (Σ_i v_{i,f} · x_i)² - Σ_i (v_{i,f} · x_i)² ]
```

对每个维度 f ∈ [1,k]：
- `(Σ_i v_{i,f} · x_i)²`：先求和再平方，O(n)
- `Σ_i (v_{i,f} · x_i)²`：逐项平方后求和，O(n)

**总复杂度**：O(nk) ✅，从 O(n²k) 降到 O(nk)！

**完整FM公式**：
```
ŷ(x) = w_0 + Σ_i w_i·x_i + (1/2) Σ_f [(Σ_i v_{i,f}·x_i)² - Σ_i (v_{i,f}·x_i)²]
       ────────  ───────────   ────────────────────────────────────────────────────
       偏置项    一阶项             二阶交叉项（O(nk)复杂度）
```

### 【L2-02】DeepFM 结构详解

**架构图**：
```
输入：[field_1, field_2, ..., field_m]（稀疏ID特征）
          │
    ┌─────▼────────────────────────────┐
    │    共享 Embedding Layer           │
    │  v_1  v_2  v_3  ...  v_m        │
    └──┬────────────────────────────┬──┘
       │                            │
  ┌────▼────┐                  ┌────▼────┐
  │  FM部分  │                  │ Deep部分 │
  │ 一阶项   │                  │ MLP网络  │
  │ + 二阶   │                  │ (3层FC)  │
  │  交叉   │                  │          │
  └────┬────┘                  └────┬────┘
       │                            │
       └────────────┬───────────────┘
                    ▼
             Sigmoid输出层
               P(click)
```

**关键特点**：
1. FM和Deep共享同一套Embedding，参数高效
2. FM完成低阶（一阶+二阶）显式交叉
3. DNN完成高阶隐式交叉
4. 无需任何手工特征工程

---

### 【L2-03】DIN 注意力机制

**核心公式**：

用户兴趣表示 = 历史行为的加权求和：
```
e_u = Σ_{i∈历史行为} α_i · e_i

其中：α_i = Attention(e_i, e_a)
```

注意力分数计算：
```
α_i = softmax(g(e_i, e_a))

g(e_i, e_a) = MLP([e_i, e_a, e_i - e_a, e_i ⊙ e_a])
              ↑                ↑              ↑
         拼接向量        差向量（距离感）  元素积（相似度）
```

其中：
- `e_i`：历史行为item的Embedding
- `e_a`：目标广告（候选item）的Embedding
- `⊙`：元素乘积（Hadamard Product）

**激活函数**：DIN使用 Dice（Data-adaptive activation）代替ReLU，根据数据分布自适应调整激活点。

---

### 【L2-04】DIEN 兴趣演化机制

**两阶段架构**：

**第一阶段：兴趣提取层（Interest Extractor Layer）**
```
行为序列：b_1 → b_2 → b_3 → ... → b_T
              ↓       ↓       ↓
            GRU  → GRU  → GRU
              ↓       ↓       ↓
            h_1     h_2     h_T   ← 兴趣状态（用辅助损失监督）

辅助损失：用 h_t 预测 b_{t+1}，强制GRU学习有意义的兴趣表示
```

**第二阶段：兴趣演化层（Interest Evolving Layer）**
```
使用 AUGRU（Attention-based GRU）：
  h'_t = (1 - ũ_t) ⊙ h'_{t-1} + ũ_t ⊙ h̃'_t

  ũ_t = α_t · u_t   ← 用目标广告的注意力分数调制更新门

  α_t = softmax(e_b_t · W · e_a)  ← 第t步行为与目标广告的相关性
```

**效果**：只让与目标相关的兴趣状态"流过"，过滤无关的历史行为影响。

---

### 【L2-05】DCN-V2 特征交叉层

**Cross Network V2（矩阵版本）**：
```
x_{l+1} = x_0 ⊙ (W_l · x_l + b_l) + x_l

对比DCN v1：
x_{l+1} = x_0 · (x_l^T · w_l) + b_l + x_l
           ↑向量外积→标量再乘           ↑向量外积→矩阵乘法（V2）
```

**两种部署模式**：
```
Stacked模式：[Cross Network] → [DNN]  （串联）
Parallel模式：[Cross Network] ┐
              [DNN]           ├→ Concat → Output
                              ┘ （并联，推荐，效果更好）
```

---

### 【L2-06】Wide & Deep 联合训练

**损失函数**：
```
L = -1/N Σ [ y·log(P(y=1|x)) + (1-y)·log(1-P(y=1|x)) ]

P(y=1|x) = σ(y_Wide + y_Deep)
          = σ(w^T [x, φ(x)] + b^T a^{(l_f)} + b_f)
              ─────────────   ───────────────────────
                Wide侧              Deep侧
```

**训练技巧**：
- Wide侧用 FTRL（Follow-The-Regularized-Leader）优化，适合稀疏特征
- Deep侧用 Adam 优化，适合稠密向量
- 两侧梯度同时回传，端到端联合训练

---

### 【L2-07】MMoE 门控机制

**公式**：
```
任务k的输出：
y_k = h^k(f^k(x))

f^k(x) = Σ_{i=1}^{n} g_i^k(x) · E_i(x)   ← 专家加权组合

g^k(x) = Softmax(W_{gk} · x)               ← 门控权重，n维向量
```

其中：
- `E_i(x)`：第i个Expert网络的输出（共n个Expert）
- `g_i^k(x)`：任务k为第i个Expert分配的权重
- 每个任务有独立的门控网络 `W_{gk}`

**多任务损失**：
```
L_total = Σ_k λ_k · L_k
```
各任务损失加权求和，λ_k 是任务权重超参。

---

### 【L2-08】Embedding技术原理

**为什么需要Embedding**：
- ID特征（用户ID、商品ID）One-Hot编码后维度极高（百万级），稀疏
- Embedding将其映射到低维稠密空间（16~256维），捕捉语义相似性

**训练方式**：
```
稀疏ID特征 → Embedding Lookup（查表）→ 稠密向量
                    ↑
            梯度回传更新Embedding
```

**Embedding压缩技术**：
- **量化（Quantization）**：FP32 → INT8，降低存储4倍
- **哈希冲突（Hash Trick）**：多个ID共享Embedding
- **分解（Factorization）**：大Embedding拆为两个小矩阵 A×B
- **稀疏更新**：只更新batch内出现的Embedding行

---

### 【L2-09】ESMM：全空间CVR建模

**问题背景**：
```
传统CVR训练空间：点击样本
实际应用空间：曝光样本（全空间）
→ 样本选择偏差（SSB：Sample Selection Bias）

点击量远少于曝光量 → CVR标签极度稀疏
```

**ESMM核心设计**：
```
pCTCVR = pCTR × pCVR

↓

同时在曝光空间训练：
L = L_CTR + L_CTCVR
  = CrossEntropy(pCTR, y_CTR) + CrossEntropy(pCTCVR, y_CTCVR)

CTR模型（曝光空间）: 标签丰富
CTCVR模型（曝光空间）: 标签相对丰富（比点击空间）
pCVR = pCTCVR / pCTR：间接推导，无需在点击空间直接训练
```

---

### 【L2-10】评估指标计算

**AUC（Area Under ROC Curve）**：
```
AUC = P(score(正样本) > score(负样本))

= Σ_{正样本i} Σ_{负样本j} 1[score_i > score_j] / (|正| × |负|)
```

**LogLoss（交叉熵）**：
```
LogLoss = -1/N Σ_i [y_i · log(p_i) + (1-y_i) · log(1-p_i)]
```
衡量预测概率的准确性（校准度）。

**GAUC（Group AUC）**：
```
GAUC = Σ_u (impression_u / Σ impression) × AUC_u
```
按用户分组计算AUC再加权平均，消除用户个体差异对全局AUC的干扰。

## L3 决策卡（Why）

### 【L3-01】为什么LR → FM？（特征交叉稀疏问题）

**LR的局限**：
```
问题场景：
- 特征A="男性"，特征B="游戏广告"
- 手工交叉 A×B 会有效果
- 但训练数据中，同时出现A=1且B=1的样本很少
- 导致 w_{AB} 无法可靠估计
```

**FM的解法**：
```
w_{AB} = <v_A, v_B>

v_A：在所有"男性"相关的样本中学习
v_B：在所有"游戏广告"相关的样本中学习
→ 借助其他样本辅助学习，解决稀疏问题
```

**结论**：FM通过隐向量分解，在稀疏数据下实现了可靠的二阶特征交叉学习。

---

### 【L3-02】为什么需要Deep（DNN）？

**FM/显式交叉的局限**：
1. FM只能学二阶交叉，高阶交叉需要指数级参数
2. 显式交叉需要人工设计交叉组合，无法穷举
3. 某些特征关系无法用乘积形式表达

**DNN的优势**：
```
通过多层非线性变换，隐式学习任意阶高阶特征交叉
MLP理论上是通用函数近似器（Universal Function Approximator）

特征 → Embedding → Dense(512) → Dense(256) → Dense(128) → Output
                    ───────────────────────────────────────
                              隐式高阶交叉
```

**结论**：显式（FM/DCN）+ 隐式（DNN）互补，是现代CTR模型的标准范式。

---

### 【L3-03】为什么需要序列建模？

**Embedding平均聚合的问题**：
```
用户行为序列：[手机, 充电器, 手机壳, 书, 鞋子]
目标广告：手机保护膜

简单平均：所有物品等权，书和鞋子干扰了手机类的信号
DIN注意力：手机、充电器、手机壳权重高 → 聚焦相关兴趣
DIEN：兴趣会随时间演化，最近兴趣更重要
BST：物品之间的共现关系（手机→充电器→手机壳是同一购物链路）
```

**序列建模的价值**：
1. **相关性过滤**（DIN）：目标感知注意力
2. **时序建模**（DIEN）：兴趣随时间变化
3. **序列内交互**（BST）：行为间的依赖关系

---

### 【L3-04】为什么选LogLoss而不是AUC作为主要训练损失？

**AUC的问题**：AUC只关注排序，不关注预测概率的绝对值，AUC无法直接作为损失函数优化（不可微）。

**LogLoss的优势**：
```
1. 可微，可直接梯度下降优化
2. 惩罚置信度错误的预测（概率输出需准确）
3. 校准性好的模型LogLoss低
4. 对应最大似然估计，有理论保证
```

**实际做法**：
- **训练**：最小化LogLoss（Binary Cross-Entropy）
- **离线评估**：同时关注AUC（排序能力）+ LogLoss（校准度）
- **在线评估**：CTR、CVR、GMV等业务指标

---

### 【L3-05】为什么多任务学习有效？

**单任务训练的问题**：
```
CTR模型：标签丰富，但只优化点击
CVR模型：标签稀疏（只有点击样本才有转化标签）
GMV模型：标签更稀疏（只有转化才有GMV）
```

**多任务学习的收益**：
1. **共享表示**：用户特征的基础Embedding被多个任务共同优化
2. **数据增强**：CTR的丰富标签帮助CVR学习用户表示
3. **正则化效果**：防止单任务过拟合
4. **联合优化**：ESMM用CTR约束CVR，解决样本选择偏差

**引用**：阿里ESMM实验显示，相比单独训练CVR，多任务联合训练AUC提升显著。

---

### 【L3-06】为什么要做样本构造优化？

**曝光偏差（Exposure Bias）**：
```
训练样本只来自曝光过的item（模型已决定展示的）
→ 未曝光的item永远没有训练信号
→ 模型偏向已有偏好，新item难以突破（马太效应）
```

**位置偏差（Position Bias）**：
```
用户倾向于点击位置靠前的item，即使质量不如后面的
→ 前位的点击率天然高，不代表质量好
→ 模型误学位置效应而非真实偏好
```

**解决方案**：
- 曝光偏差：探索（ε-greedy、Thompson Sampling）+ 纠偏
- 位置偏差：训练时加入位置特征，推理时统一设为固定值（如位置1）

## L4 场景卡（Scene）

### 【L4-01】场景：新广告冷启动，如何初始化CTR？

**问题**：新广告没有历史展示数据，无法估计真实CTR，模型预测不准。

**解决方案**（多策略组合）：

**1. 基于内容特征初始化**：
```
利用广告的文字、图片、类目等内容特征
通过已有广告的相似性（embedding相似度）推断初始CTR
新广告CTR ≈ 内容相似广告的历史CTR加权平均
```

**2. 贝叶斯先验（Beta-Binomial）**：
```
CTR后验估计 = (点击数 + α) / (曝光数 + α + β)

α, β：来自同类广告的先验参数
冷启动阶段：主要由先验决定；曝光足够后：数据说话
```

**3. 广告主历史表现**：
```
同一广告主的历史广告CTR均值作为初始参考
```

**4. 强制探索**：
```
新广告加入探索池，按UCB/Thompson Sampling保证一定曝光量
快速积累真实数据，逃离冷启动状态
```

**5. 迁移学习**：
```
预训练大模型（图文理解）→ fine-tune到CTR预估
利用内容理解弥补行为数据不足
```

### 【L4-02】场景：CTR模型AUC高，但GMV反而下降？

**问题诊断**：AUC高说明排序能力强，但GMV下降说明优化目标与业务目标不对齐。

**常见原因**：

**1. 优化了点击，忽视了转化**：
```
CTR高 ≠ CVR高
模型把标题党、低价诱导的广告排在前面
用户点击后发现实际质量差 → 不购买 → GMV低
解决：引入CVR、GMV多目标优化（ESMM/MMoE）
```

**2. 样本选择偏差**：
```
训练集是历史曝光样本，新模型提升了长尾商品的曝光
但长尾商品的转化率低 → GMV下降
```

**3. 展示商品的价格分布改变**：
```
模型偏向高CTR但低客单价商品
每次点击的GMV贡献减少
解决：在排序分中加入 价格 × CVR 的GMV预估
```

**4. 用户体验下降（长期效果）**：
```
过度优化短期CTR，损害用户长期满意度
用户减少打开频率，整体GMV下降
```

### 【L4-03】场景：模型离线AUC好，但线上效果差（AUC-OE Gap）

**Gap产生的原因**：

**1. 训练-服务分布偏移**：
```
训练数据：历史行为（T-1时刻）
线上数据：实时行为（T时刻）
用户兴趣实时变化，但模型无法感知
解决：实时特征（近1小时行为序列）+ 在线学习（增量更新）
```

**2. 特征穿越（Feature Leakage）**：
```
训练时用了"未来信息"作为特征
例如：用item_id的7日点击率，但训练时该特征包含了标签时刻之后的数据
线上预测时该特征只能用历史数据 → 训练/线上不一致
```

**3. 线上特征缺失**：
```
离线有的特征，线上缺失或有延迟
例如：用户画像更新有延迟，实时特征获取失败
```

**4. 系统链路不一致**：
```
离线：全量候选集
线上：召回层已过滤，候选集分布不同
```

### 【L4-04】场景：如何处理样本的位置偏差？

**问题**：用户在列表第1位的点击率天然高于第10位，即使物品质量相同。

**量化位置偏差**：
```
P(click | user, item, pos) = P(observe | pos) × P(click | user, item, observed)
                               ────────────────   ─────────────────────────────────
                                  位置偏差因子              真实偏好
```

**解决方案1：位置特征 + 推理时固定**：
```
训练：将位置（pos=1,2,...,N）作为特征输入模型
推理：将所有候选的position特征统一设为1（或去掉）
→ 模型学习了位置对CTR的影响，推理时消除位置差异
```

**解决方案2：双塔去偏**：
```
user_tower(user, item) × position_tower(pos)  ← 乘法解耦
推理时只用 user_tower 的输出
```

**解决方案3：逆倾向加权（IPW）**：
```
Loss = Σ_i (1/P(observe|pos_i)) × CrossEntropy(y_i, p_i)
→ 位置靠后、被观察概率低的样本，赋予更高权重
```

### 【L4-05】场景：在线学习 vs 全量重训，如何选择？

**在线学习（Online Learning / FTRL）**：
```
适用场景：
✅ 对新鲜度要求高（直播、突发热点、大促）
✅ 数据量大，全量训练代价高
✅ 用户兴趣迁移快

代价：
❌ 模型遗忘历史（Catastrophic Forgetting）
❌ 样本顺序影响训练，需要careful设计
❌ 系统复杂度高（实时特征流、模型热更新）
```

**全量重训（Offline Batch）**：
```
适用场景：
✅ 模型结构复杂（DIN/BST等）
✅ 数据分布相对稳定
✅ 对一致性要求高

代价：
❌ 训练周期长（几小时），对实时变化响应慢
```

**工业实践**：两者结合
```
全量训练（每天/周）：学习稳定的用户偏好
+ 增量学习（每小时/分钟）：捕捉实时兴趣变化
```

## L5 陷阱卡（Trap）

### 【L5-01】陷阱：FM 和 FFM 的区别是什么？

**常见错误回答**：说FFM只是"更多参数的FM"。

**正确区别**：

| 维度 | FM | FFM |
|------|-----|-----|
| 隐向量数 | 每个特征 1个 v_i | 每个特征 F个（F=域数量）|
| 交叉计算 | `<v_i, v_j>` | `<v_i^{f_j}, v_j^{f_i}>` |
| 参数量 | n×k | n×F×k |
| 核心思想 | 统一视角的二阶交叉 | 考虑域信息，i看j用j的域向量 |

**FFM的语义**：
```
特征i与"广告类"特征交叉时：使用 v_i^{field=AD}
特征i与"用户类"特征交叉时：使用 v_i^{field=USER}
→ 同一特征对不同域的特征有不同的"视角"
```

**工业实践**：FFM效果略好但参数量是FM的F倍，内存和计算开销大，大规模应用较少。DeepFM基本取代了FFM。

---

### 【L5-02】陷阱：为什么用GAUC而不是AUC？

**常见错误**：认为AUC越高模型越好，用全局AUC评估CTR模型。

**全局AUC的问题**：
```
场景：用户A活跃（日均点击50次），用户B不活跃（日均点击1次）
全局AUC主要由用户A的行为决定
→ AUC衡量的是"区分用户A的能力"，而非对所有用户均匀公平的排序能力

跨用户比较：用户A的点击率本身就比用户B高
→ 模型把A的所有行为排在B的行为之上，AUC虚高
→ 但对用户B内部的排序质量不关心
```

**GAUC（Group AUC）**：
```
GAUC = Σ_u (w_u × AUC_u)

w_u = impression_u / Σ impression   ← 按曝光量加权

先在每个用户内部计算AUC（用户内排序），再加权平均
→ 避免活跃用户"霸占"评估结果
→ 更真实反映模型对每个用户的个性化排序能力
```

**经验**：GAUC提升1%通常对应线上CTR提升约0.1~0.5%（因业务而异）。

---

### 【L5-03】陷阱：CTR预测值偏低/偏高是什么问题？如何校准？

**问题**：模型AUC很高（排序能力好），但预测的绝对概率值偏低（如真实CTR=5%，模型预测均值=2%）。

**后果**：
```
广告出价公式：bid = value × pCTR
pCTR偏低 → bid偏低 → 广告竞争力下降 → 收入损失
```

**偏差来源**：
1. **负样本下采样**：工业场景下负样本远多于正样本，常下采样1:10或1:100，导致正样本比例虚高，预测概率偏高（注意：是偏高！）
2. **样本选择偏差**：训练只在曝光空间，预测概率可能整体偏高

**负采样后的校准**：
```
采样率：r（保留 r 比例的负样本）
校正公式：p_calibrated = p / (p + (1-p)/r)

直觉：模型学到的是高正例率分布，需要还原真实分布的概率
```

**Platt Scaling（简单有效）**：
```
用少量数据拟合：p_calibrated = σ(a × logit(p) + b)
a, b 通过最大似然估计
```

### 【L5-04】陷阱：位置特征训练和推理不一致会怎样？

**常见错误**：在推理时仍然使用真实位置，或者完全不加入位置特征。

**陷阱1：推理时使用真实位置（循环依赖）**：
```
排序结果决定了位置 → 位置特征又影响了排序分 → 循环！
实际上推理时还不知道最终排在第几位
```

**陷阱2：不加位置特征**：
```
模型强行把位置效应学进其他特征里
→ 某些item特征与位置产生强相关，无法分离
→ 模型对位置不公平
```

**正确做法**：
```
训练：position作为特征输入（让模型学到位置的影响）
推理：position=1（假设所有候选都在第一位，公平比较）
     OR position=0（用0/pad向量表示无位置信息）
```

**延伸**：去偏模型（Propensity-based Debias）更优雅：
```
P(click) = P(observe|pos) × P(like|user, item)
训练时解耦两个因子，推理时只用P(like)
```

### 【L5-05】陷阱：特征穿越（Feature Leakage）如何发生？

**定义**：训练时使用了包含"未来信息"的特征，导致离线AUC虚高，线上效果差。

**典型场景**：
```
样本：用户u在时刻t点击了item i（标签=1）

错误特征：item_i的7日点击总数（统计截止时间=T，包含了t之后的点击）
→ 热门item在t之后点击量大，该特征值高
→ 模型学到了"高点击量的item更容易被点击"
→ 看似有道理，实则用了未来数据

线上使用：只能用t-7到t的数据，分布与训练不同
```

**正确做法**：
```
特征计算时间窗口必须严格限定在标签时刻t之前
样本：(user, item, label_time=t)
特征：item_click_7d = count(click on item_i, from t-7d to t-1d)
                                                          ↑ 不包含t本身
```

**检测方法**：
```
1. 去掉该特征，看AUC变化（如果AUC剧烈下降，可能有穿越）
2. 训练集AUC远高于验证集AUC（过拟合信号）
3. 特征重要性排序中某个统计特征异常靠前
```
