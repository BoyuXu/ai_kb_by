# Hierarchy Enhanced Policy Optimization for Ad Ranking
> 来源：arxiv/2312.xxxxx | 领域：ads | 学习日期：20260326

## 问题定义
广告排序的强化学习优化面临：
- 奖励稀疏：用户购买/转化信号 T+1/T+3 才能获得，延迟大
- 动作空间巨大：候选广告数量百万级，直接 RL 不可行
- 多层级目标：平台收益（RPM）、广告主 ROI、用户体验需要平衡
- 探索-利用权衡：探索可能损害短期指标

## 核心方法与创新点
**HEPO（Hierarchy Enhanced Policy Optimization）**：分层策略优化。

**分层策略架构：**
```
高层策略 π_H：负责长期目标（用户留存、广告主 ROI）
  ↓ 给低层策略提供「子目标」g_t
低层策略 π_L：负责短期决策（当前时间步的广告排序）
  ↓ 根据子目标和即时奖励做动作

H-Level: g_t = π_H(s_t; θ_H)          # 子目标：如"偏向转化广告"
L-Level: a_t = π_L(s_t, g_t; θ_L)     # 排序动作
```

**分层奖励设计：**
```
低层即时奖励：r_L = CTR_reward + λ·CVR_reward
高层延迟奖励：R_H = RPM_T + α·advertiser_ROI_T + β·user_retention_T
```

**策略优化（PPO框架）：**
```
L_H = E[min(ratio·Â_H, clip(ratio, 1±ε)·Â_H)] - λ·KL(π_H||π_H_old)
L_L = E[min(ratio·Â_L, clip(ratio, 1±ε)·Â_L)]

ratio = π(a|s) / π_old(a|s)  # 新旧策略比值
```

**约束优化（广告主 ROI 保护）：**
```
maximize: E[R_H]
s.t.     E[ROI_advertiser] ≥ target_ROI
         # 用 Lagrangian Relaxation 转化为无约束问题
```

## 实验结论
- 某大型广告平台模拟环境：
  - RPM +3.1%，广告主 ROI +1.5%（vs 标准 PPO）
  - 用户留存（次日活跃率）：+0.5%（vs 贪婪最大化 RPM）
- 收敛速度：比 flat PPO 快 2.1x（分层结构提供更稳定的梯度）

## 工程落地要点
1. **离线模拟环境**：构建用户行为模拟器（基于历史日志），安全探索
2. **策略部署**：RL 策略作为排序分数的修正项（残差），不完全替换现有模型
3. **安全约束**：设置 RPM 下限保护（线上 RPM 下降超过阈值自动回滚）
4. **延迟奖励处理**：CVR/GMV 奖励 T+1 到达，用奖励缓存异步反馈
5. **探索控制**：ε-greedy 探索仅对少量流量（1-5%）开启

## 面试考点
**Q1: 分层 RL 相比单层 RL 在广告排序中的优势？**
A: 高层策略处理长期目标（广告主 ROI、用户留存），低层策略处理短期排序决策，时间尺度分离。这解决了长期奖励信号稀疏（延迟大）和短期决策频繁之间的矛盾，类似于人类的习惯（低层）和目标（高层）分离。

**Q2: 广告排序 RL 如何构建奖励函数？**
A: 多目标加权：r = w1·CTR + w2·CVR + w3·广告主ROI - w4·用户不满（如负反馈）。关键：各目标权重需要与业务 KPI 对齐，通过 AB 实验验证权重合理性。

**Q3: PPO 的 Clip 机制有什么作用？**
A: 限制策略更新幅度：当新旧策略差异过大（ratio 超出 [1-ε, 1+ε]），截断梯度。防止单步更新过大导致策略崩溃，增强训练稳定性。广告场景中 ε 通常设 0.1-0.2。

**Q4: 广告系统中 RL 探索与利用的平衡策略？**
A: ①Thompson Sampling：为每个广告维护 Beta 分布，按置信度探索 ②UCB：上置信界奖励，鼓励不确定性高的广告 ③Contextual Bandit：探索局限于轻量策略，不影响主模型 ④流量分桶：1-5% 流量专门用于探索。

**Q5: 如何确保 RL 广告系统满足广告主 ROI 约束？**
A: 约束 MDP + Lagrangian Relaxation：将 ROI 约束转化为惩罚项加入奖励函数：r' = r_revenue - λ·max(0, ROI_target - ROI_actual)，λ 通过对偶梯度上升动态更新，保证约束满足。
