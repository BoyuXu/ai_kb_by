## 二、拍卖机制深度解析

### 2.1 First-Price Auction（一价拍卖）

**规则：** 出价最高者获胜，并支付自己的出价金额。

```
示例：
  DSP A 出价 $5.00
  DSP B 出价 $3.50
  DSP C 出价 $2.00

  结果：DSP A 获胜，支付 $5.00（自己的出价）

核心特点：
  ✅ 简单直观，媒体收益最大化
  ❌ 出价即最终价格，广告主需要"猜测竞对出价"来制定策略
  ❌ 信息不对称加剧，出价过高→亏损，出价过低→失标

引出 Bid Shading（出价压缩）策略：
  由于不知道对手出价，DSP 会主动压低出价
  目标：在"刚好赢过第二名"的价格附近出价
  
  理想出价 ≈ E[第二名出价 | 自己赢拍的条件期望]
```

#### Bid Shading 算法框架

```
Bid Shading 策略：

1. 历史数据建模
   - 收集历史同类流量的竞拍结算价分布
   - 拟合竞对出价的概率分布 F(b)
   
2. 最优出价计算
   目标：最大化期望利润
   
   期望利润 = (真实价值v - 出价b) × Pr(b获胜)
            = (v - b) × F(b)
   
   对b求导令其为0：
   F(b) - (v - b) × f(b) = 0
   最优出价 b* 满足：(v - b*) = F(b*) / f(b*)（逆风险率）

3. 实践中的近似方法
   Step 1：预测当前流量的"竞争强度"（pWin曲线）
   Step 2：从历史Win Price分布取 ≈ 50-70th percentile 作为出价
   Step 3：根据广告主ROI目标迭代调整
   Step 4：在线实验验证

4. 在线学习更新
   每次竞拍后收到 Win Price 或 Loss 信号
   更新出价模型（Contextual Bandit 框架）
```

### 2.2 Second-Price Auction（Vickrey 拍卖）

**规则：** 出价最高者获胜，但支付第二名出价 + $0.01（最小竞价增量）。

```
示例：
  DSP A 出价 $5.00  ← 获胜
  DSP B 出价 $3.50  ← 第二名
  DSP C 出价 $2.00

  结果：DSP A 获胜，支付 $3.51（第二名出价+$0.01）
  
  对 DSP A 来说：
    如果真实价值 v = $4.50
    净收益 = $4.50 - $3.51 = $0.99（正收益 ✅）
```

#### 为什么诚实出价是占优策略（数学证明）

**定理：** 在二价拍卖中，诚实出价（b = v）是每位竞拍者的弱占优策略。

```
证明：

设：
  v = 广告主对此次展示的真实价值
  b = 广告主的出价（待选择）
  m = 其他所有竞拍者中出价最高者的出价（即第二高价格）

分析：广告主选择出价 b 时的期望支付

情形1：诚实出价 b = v

  若 v > m（真实价值高过竞对）：
    赢拍，净收益 = v - m > 0  ✅

  若 v < m（真实价值低于竞对）：
    落败，净收益 = 0          ✅

  若 v = m：
    平局（通常随机决定），净收益 ≈ 0  ✅

情形2：虚报高价 b > v（策略性加价）

  子情形 2a：b > m > v（虚报高价反而赢了）
    赢拍，净收益 = v - m < 0  ❌（亏损！因为v<m）

  子情形 2b：b > v > m（结果与诚实出价相同）
    赢拍，净收益 = v - m > 0  → 无改善，结果同诚实出价

  子情形 2c：m > b > v（虚报也没用）
    落败，净收益 = 0          → 同诚实出价

情形3：虚报低价 b < v（策略性压价）

  子情形 3a：b < m < v（错过本可赢的竞拍）
    落败，净收益 = 0          ❌（本可赢 v - m > 0）

  子情形 3b：b < v < m（结果同诚实出价）
    落败，净收益 = 0          → 无改善

  子情形 3c：m < b < v（结果与诚实出价相同）
    赢拍，净收益 = v - m > 0  → 同诚实出价

结论：
  - 虚报高价可能导致亏损（情形2a），不能改善情形
  - 虚报低价可能错过正收益机会（情形3a），不能改善情形
  - 诚实出价在任意情况下结果都 ≥ 非诚实出价

∴ b = v 是弱占优策略（Weakly Dominant Strategy）
  （不管竞对如何出价，诚实出价都不会更差）
                                              □ 证毕
```

### 2.3 GSP（Generalized Second Price / 广义二价拍卖）

**背景：** 搜索广告有多个广告位（位置1/2/3），需要同时分配多个位置。GSP 是针对多广告位场景的拍卖机制。

#### GSP 基本规则

```
场景：3个广告位，4个竞拍者

广告位点击率（CTR）：
  位置1：α₁ = 5%（最显眼，点击最多）
  位置2：α₂ = 3%
  位置3：α₃ = 1%

竞拍者出价（CPC出价）：
  广告A：$3.00
  广告B：$2.50
  广告C：$2.00
  广告D：$1.00

按出价排序分配：
  位置1 → 广告A，实际CPC = 广告B出价 + $0.01 = $2.51
  位置2 → 广告B，实际CPC = 广告C出价 + $0.01 = $2.01
  位置3 → 广告C，实际CPC = 广告D出价 + $0.01 = $1.01

广告主期望收益（假设A真实价值=$4，B=$3，C=$2.5）：
  A：(4 - 2.51) × 5% = 0.0745/展示
  B：(3 - 2.01) × 3% = 0.0297/展示
  C：(2.5 - 1.01) × 1% = 0.0149/展示
```

#### 为什么工业界偏爱 GSP

```
GSP vs VCG 对比：

┌──────────────────────┬──────────────────────────────┐
│       GSP            │           VCG                │
├──────────────────────┼──────────────────────────────┤
│ 计算复杂度：O(n log n)│ 计算复杂度：O(n³)（较高）      │
│ 规则直观（排价即排名）│ 规则复杂（社会外部性定价）     │
│ 实现简单              │ 实现复杂                     │
│ 不是诚实机制          │ 激励相容（诚实出价是均衡）     │
│ 纳什均衡下近似有效     │ 社会福利理论最大化             │
│ Google、百度均使用    │ 工业界几乎不用                │
└──────────────────────┴──────────────────────────────┘

GSP 的核心优势：
1. 计算效率：排序+差价计费，实时竞拍中每秒处理百万次请求
2. 透明性：广告主容易理解"我出价多少，排在第几，花多少钱"
3. 稳定性：存在纯策略纳什均衡（Locally Envy-Free Equilibrium）
4. 收益接近：在均衡状态下，GSP 收益与 VCG 接近

GSP 的问题：
  不是激励相容（Incentive Compatible）
  广告主有动机策略性出价（非诚实出价）
  例：广告A知道B出价$2.50，可以出$2.51而不是真实价值$4.00
```

### 2.4 VCG 机制（Vickrey-Clarke-Groves）

**原理：** 每个参与者支付的价格等于"自己的参与对其他人造成的损失"（外部性定价）。

#### VCG 定价公式推导

```
设定：
  n 个广告位，位置 i 的点击率为 α_i（α₁ > α₂ > ... > α_n）
  m 个竞拍者，竞拍者 j 的真实价值为 v_j
  按真实价值排序：v₁ ≥ v₂ ≥ ... ≥ v_m

VCG 分配规则：
  社会福利最大化 → 按价值从高到低分配位置
  竞拍者 j（分配位置 j）的贡献：v_j × α_j

VCG 定价：
  竞拍者 j 支付的价格 = Σ_{k>j} (α_k - α_{k+1}) × v_{k+1}
  
  其中 α_{n+1} = 0（没有位置n+1）

数值示例（3位置，4竞拍者）：

  α₁=5%, α₂=3%, α₃=1%, α₄=0%（排不上）
  v₁=$4, v₂=$3, v₃=$2.5, v₄=$1

  竞拍者1（位置1）的VCG价格：
    p₁ = (α₁-α₂)×v₂ + (α₂-α₃)×v₃ + (α₃-α₄)×v₄
       = (5%-3%)×3 + (3%-1%)×2.5 + (1%-0%)×1
       = 0.06 + 0.05 + 0.01 = $0.12（每次展示）
    等效CPC = $0.12 / 5% = $2.40

  竞拍者2（位置2）的VCG价格：
    p₂ = (α₂-α₃)×v₃ + (α₃-α₄)×v₄
       = (3%-1%)×2.5 + (1%-0%)×1
       = 0.05 + 0.01 = $0.06（每次展示）
    等效CPC = $0.06 / 3% = $2.00

  竞拍者3（位置3）的VCG价格：
    p₃ = (α₃-α₄)×v₄
       = (1%-0%)×1 = $0.01（每次展示）
    等效CPC = $0.01 / 1% = $1.00

VCG 的优美性质：
  ✅ 激励相容：诚实出价是每个参与者的最优策略
  ✅ 个体理性：赢家不会亏损（v_j ≥ p_j）
  ✅ 社会福利最大化：价值高的广告占据最好位置
  
VCG 为什么工业界极少使用：
  ❌ 计算复杂：需要遍历所有可能分配，复杂度指数级
  ❌ 不透明：定价公式难以向广告主解释
  ❌ 收入不稳定：VCG 可能比 GSP 收入低（无法独立提高单个位置收益）
  ❌ 抗合谋弱：合谋广告主可以操控结果
  ❌ 预算约束处理困难：VCG 假设无预算约束
```

## 三、Reserve Price（底价）机制

### 3.1 平台为什么设底价

```
底价（Reserve Price / Floor Price）是媒体/SSP 设置的最低可接受出价。

设底价的原因：

1. 保护媒体收益
   无底价时：最高出价 $0.01 也能赢拍
   有底价后：至少 $0.50 才能展示，保护单次展示收益下限

2. 过滤低质量广告主
   低出价往往对应低质量广告（效果差、相关性弱）
   底价自然筛除低质量需求，提升用户体验

3. 战略性媒体保护
   若流量价值高于市场认知，底价帮助媒体避免低价卖出
   例：热门新闻稿件流量价值高，底价避免被低价抢走

4. 对抗买家合谋
   多个 DSP 合谋压低出价时，底价保护媒体不被操控

5. 媒体议价筹码
   对强势广告主：底价是谈判基准
```

### 3.2 软底价 vs 硬底价

```
┌──────────────────────────────────────────────────────────────┐
│                   底价类型对比                                │
├────────────────┬─────────────────────────────────────────────┤
│   硬底价        │   软底价（Soft Floor）                       │
│  Hard Floor    │                                              │
├────────────────┼─────────────────────────────────────────────┤
│ 定义：          │ 定义：                                       │
│ 绝对最低价格    │ 拍卖的参考最低价，可灵活调整                  │
│ 低于此价拒绝   │ 低于软底价但高于某阈值时仍可成交              │
├────────────────┼─────────────────────────────────────────────┤
│ 机制：          │ 机制：                                       │
│ 出价 < 硬底价  │ 场景1：最高出价 > 软底价 → 按二价结算         │
│ → 直接拒绝     │ 场景2：最高出价在软底价和硬底价之间           │
│                │        → 最高出价者赢，支付自己的出价         │
│                │ 场景3：所有出价 < 硬底价 → 拒绝展示           │
├────────────────┼─────────────────────────────────────────────┤
│ 适用：          │ 适用：                                       │
│ 高质量媒体     │ 希望最大化填充率同时保护收益的媒体            │
│ 强势内容       │                                              │
├────────────────┼─────────────────────────────────────────────┤
│ 风险：          │ 风险：                                       │
│ 填充率下降     │ 结算机制复杂，DSP 策略难以针对优化            │
│ 变现收益减少   │ 可能引发 DSP 不信任（规则不透明）             │
└────────────────┴─────────────────────────────────────────────┘

数值示例（软底价=$1.00，硬底价=$0.50）：

  DSP A出价$2.00, DSP B出价$1.50：
    → 最高价$2.00 > 软底价$1.00
    → 正常二价结算：A胜，支付$1.51（B出价+$0.01）

  DSP A出价$0.80, DSP B出价$0.00：
    → 最高价$0.80，在软底价($1.00)和硬底价($0.50)之间
    → A胜，支付自己的出价$0.80（不是$0.01）

  所有出价 < $0.50（硬底价）：
    → 拒绝展示，流量空耗（Unfilled Impression）
```

### 3.3 Dynamic Reserve Price（动态底价）

```
静态底价的问题：
  同一媒体的不同用户/时段/场景，流量价值差异巨大
  高价值流量卖便宜 → 媒体损失
  底价设太高 → 低质量流量填充率低

动态底价（DRP）：根据特征动态计算最优底价

特征维度：
  用户维度：
    - 历史转化率（高CVR用户 → 高底价）
    - 消费能力（高净值用户 → 高底价）
    - 活跃度（高活跃 → 高底价）
    
  上下文维度：
    - 广告位置（首屏 → 高底价）
    - 内容相关性（强相关场景 → 高底价）
    - 时段（黄金时段 → 高底价）
    
  历史数据维度：
    - 该类流量历史成交eCPM分位数
    - 同类流量最近N次拍卖中位价

动态底价计算框架：
  ┌─────────────────────────────────────────────┐
  │        动态底价 = Base Floor × 调整因子       │
  │                                             │
  │  调整因子 = f(用户价值分, 场景分, 历史成交分)  │
  │                                             │
  │  实践方法：                                  │
  │  1. 历史成交eCPM第20-30百分位作为底价         │
  │     （保证大多数流量仍能成交）                │
  │  2. 机器学习预测最优底价                      │
  │     目标：max(底价 × 成交概率)               │
  │  3. Thompson Sampling 探索最优底价            │
  └─────────────────────────────────────────────┘

动态底价的优化目标（媒体侧）：

  传统：最大化成交率（Fillrate）
  → 底价设很低，成交多但单价低

  动态底价：最大化期望eCPM
  期望eCPM = 成交价格 × Pr(成交 | 价格)
            = P × (1 - F(P))   // F(P)=以价格P未能成交的概率

  最优底价 P* 满足：
  dE/dP = (1 - F(P)) - P × f(P) = 0
  即：P* = (1 - F(P*)) / f(P*)  （逆风险率公式，同Bid Shading）
```

## 四、从二价到一价的行业变迁

### 4.1 2019年 Google 宣布切换一价的背景

```
时间线：
  2019年3月  → Google ADX 宣布切换为统一一价拍卖（Unified First-Price Auction）
  2019年9月  → 全面切换完成

切换前的问题（促成切换的原因）：

1. 规则复杂性困扰 DSP
   Google 之前采用"带底价的二价"（即软底价机制）
   规则：出价高于软底价 → 二价结算
         出价低于软底价但高于硬底价 → 一价结算
   DSP 反馈：规则不透明，难以制定一致的出价策略

2. Header Bidding 的竞争压力
   Header Bidding（见下节）让 DSP 同时参与多个ADX的竞价
   各家ADX规则不同（有一价有二价），DSP面临极高策略复杂度
   统一为一价后，DSP 策略可以标准化

3. 媒体收益被压低
   二价机制下，赢家只需付第二名+0.01
   聪明的 DSP 通过 Bid Shading 进一步压低出价
   导致媒体实际收益远低于广告主真实愿意支付的价格

4. 行业去复杂化趋势
   越来越多的 SSP（如 OpenX、Index Exchange）早已切换一价
   Google 选择跟进，统一行业标准
```

### 4.2 Header Bidding 的兴起与影响

```
背景：
  传统"瀑布流"（Waterfall）模式：
  媒体按优先级依次询问ADX A → B → C
  优先级高的ADX先出价，低优先级ADX错过高价机会
  问题：ADX优先级是人工设定的，不是动态竞争，媒体收益次优

Header Bidding（头部竞价）：
  思路：让所有 DSP / ADX 同时并行竞价，取最高出价
  
  技术实现：
  在网页 <head> 标签中提前加载竞价代码
  页面加载时，所有 ADX 同时发起竞价（并行而非串行）
  收集所有出价后，再统一发给 Google DFP（广告服务器）决策

  ┌──────────────────────────────────────────────┐
  │          Header Bidding 架构                  │
  │                                              │
  │  用户浏览器                                  │
  │  ┌─────────────────────────────────┐         │
  │  │  <head>                         │         │
  │  │    Prebid.js 加载               │         │
  │  │    ↓ 并行请求                   │         │
  │  │    ├── AppNexus → 出价 $2.50    │         │
  │  │    ├── Rubicon  → 出价 $2.00    │         │
  │  │    ├── OpenX    → 出价 $3.00    │         │
  │  │    └── Google   → 出价 $2.80    │         │
  │  │    所有出价汇聚 → Google DFP    │         │
  │  │    DFP 选出 $3.00（OpenX胜）   │         │
  │  │  </head>                        │         │
  │  └─────────────────────────────────┘         │
  └──────────────────────────────────────────────┘

Header Bidding 的影响：
  ✅ 媒体收益提升 15-30%（真正的竞争最大化）
  ✅ DSP 机会均等，不再依赖优先级排序
  ✅ 推动行业从二价切换一价（因各 ADX 本已是一价）
  ❌ 页面加载时间增加 100-300ms（性能成本）
  ❌ 服务端 Header Bidding（Server-Side）缓解了性能问题
  ❌ 用户设备带宽/CPU 消耗增加
```

### 4.3 Bid Shading：一价市场下的出价策略

```
问题背景：
  二价市场：出价$5.00，赢了只需付$3.51（省了$1.49）
  一价市场：出价$5.00，赢了需付$5.00（每分都花出去）
  → 广告主在一价市场中有强烈的出价压缩动机

Bid Shading 核心逻辑：

  目标：找到 b* 使期望利润最大
  
  期望利润 E[π(b)] = (v - b) × Pr(win | b)
                   = (v - b) × F(b)
  
  其中 F(b) = 以出价b赢拍的概率（竞对出价分布的CDF）
  
  最优出价条件：
  dE/db = 0
  -(F(b)) + (v-b)f(b) = 0
  v - b = F(b) / f(b)
  
  b* = v - F(b*) / f(b*)

实操中的 Bid Shading 方法：

方法1：历史分布法
  收集历史同类流量的 Clearing Price（结算价）分布
  以第50-70百分位作为出价（预期赢拍概率50-70%）

方法2：在线学习 + Contextual Bandit
  特征：流量特征（设备/时段/地域/广告位大小）
  动作：出价区间（$0.5~$5.0，离散化100个档位）
  奖励：赢拍时 = 转化价值 - 支付价格；输了 = 0
  更新：Thompson Sampling 或 UCB

方法3：Win Rate Curve 拟合
  分段统计：在出价区间 [b, b+Δ] 内的历史赢拍率
  拟合平滑曲线 → 给定目标赢拍率，反查最优出价

Bid Shading 对市场的影响：
  媒体视角：有效eCPM被压低（广告主压价）
  DSP视角：节省预算，提升ROI
  行业均衡：最终结算价大约是真实价值的60-80%（取决于竞争程度）
```

**考点2：证明 Vickrey 二价拍卖中诚实出价是占优策略（推导题）**

```
核心证明思路（必须记住以下三个情形）：

设真实价值 v，最高竞对出价 m

情形1：诚实出价 b=v
  v>m → 赢拍，净收益 v-m > 0 ✅
  v<m → 落败，净收益 = 0 ✅

情形2：虚报高价 b>v
  若 b>m>v → 赢拍，净收益 v-m < 0（亏损！❌）
  若 b>v>m → 同情形1无改善
  若 m>b>v → 同情形1落败

情形3：虚报低价 b<v
  若 b<m<v → 落败（错过正收益！❌）
  其他子情形 → 同情形1无改善

结论：偏离诚实出价要么结果更差，要么无改变
     → b=v 是弱占优策略
```

**考点3：GSP 与 VCG 机制的区别，为什么工业界选 GSP？**

```
三个维度区分：

1. 定价规则
   GSP：每个赢家付"下一名出价+$0.01"（排位差价）
   VCG：每个赢家付"自己的参与对他人造成的外部性损失"

2. 激励特性
   GSP：非激励相容，存在策略性出价动机
        纳什均衡下出价低于真实价值
   VCG：激励相容，诚实出价是占优策略

3. 工业界选 GSP 的原因
   计算效率：GSP O(n log n)，VCG O(n³)
   透明度：广告主可以直觉理解GSP定价，VCG公式难解释
   稳定性：GSP 存在局部无嫉妒均衡（Locally Envy-Free）
   历史路径：搜索广告早期就用GSP，切换成本极高
```

**考点5（推导题）：动态底价的最优化目标是什么？推导最优底价公式。**

```
题目：媒体希望通过设置底价最大化期望收益，推导最优底价 P*

设：
  竞拍者出价分布的 CDF 为 F(P)（出价低于P的概率）
  媒体设置底价 P
  
分析：
  若所有出价 < P → 无人成交，媒体收益 = 0
  若最高出价 ≥ P → 媒体获得最高出价（一价）或第二高（二价）
  
  简化为一价拍卖场景：
  媒体期望收益 E[R(P)] = P × Pr(最高出价 ≥ P)
                       = P × (1 - F(P))

  对 P 求导令其为0：
  dE/dP = (1 - F(P)) + P × (-f(P)) = 0
  1 - F(P) = P × f(P)
  P* = (1 - F(P*)) / f(P*)

  这正是"逆风险率"（Inverse Hazard Rate），也叫 Mills Ratio
  
  直觉理解：
    提高底价 ΔP → 收益增加 ΔP × Pr(成交)
    提高底价 ΔP → 成交概率下降 f(P)×ΔP
    两者相等时为最优底价

应用举例：
  若历史出价服从均匀分布 U[0, 1]：
    F(P) = P，f(P) = 1
    P* = (1-P*)/1 → P* = 0.5（设在中间最优）
  
  若历史出价服从指数分布 Exp(λ)：
    F(P) = 1-e^{-λP}，f(P) = λe^{-λP}
    P* = (1-(1-e^{-λP*})) / (λe^{-λP*})
       = e^{-λP*} / (λe^{-λP*}) = 1/λ（即均值）
```
