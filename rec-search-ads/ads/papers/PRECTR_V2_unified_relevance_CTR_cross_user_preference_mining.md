# PRECTR-V2: Unified Relevance-CTR Framework with Cross-User Preference Mining

> 来源：arXiv 2025 | 领域：ads | 学习日期：20260404

## 问题定义

广告排序中，**相关性（Relevance）** 和 **CTR** 是两个独立但相互关联的优化目标：
- 相关性：广告内容与查询/场景的语义匹配
- CTR：用户点击概率（包含相关性 + 用户偏好 + 广告质量）

分开训练忽略了两者的相互增强关系；简单合并又会让 CTR 信号淹没相关性信号。

## 核心方法与创新点

**PRECTR-V2** 的核心创新：**跨用户偏好挖掘（Cross-User Preference Mining）**

1. **相关性增强 CTR 框架**：
   - 相关性分数作为 CTR 模型的辅助输入（而非独立并行）
   - 相关性 embedding + CTR 特征 → 统一 Transformer
   
$$\text{CTR}(u, a) = f([e_u, e_a, s_{\text{rel}}(q, a)])$$

2. **跨用户偏好挖掘**：
   - 找到「与当前用户偏好相似的其他用户」对同一广告的反应
   - 跨用户协同信号增强当前用户决策
   
$$h_u^{\text{cross}} = \text{Attention}(Q_u, K_{\mathcal{U}_{sim}}, V_{\mathcal{U}_{sim}})$$

其中 $\mathcal{U}_{sim}$ 是偏好相似用户集合

3. **双塔相关性预训练**：
   - 先在大规模 Query-Ad 对上预训练相关性模型
   - 冻结相关性 Encoder，只微调 CTR 层
   - 避免 CTR 信号破坏语义表示质量

4. **软相关性标签**：
   - 相关性标签不仅用人工标注，还用用户停留时间/转化率推断
   - 多源相关性信号融合

## 实验结论

- 搜索广告 CTR AUC: **+1.6‰**
- 相关性 NDCG: **+3.8%** vs 独立相关性模型
- 跨用户挖掘单独贡献: **+0.7‰** CTR AUC
- 用户冷启动场景（新用户）提升最显著: **+2.4‰**

## 工程落地要点

- 相似用户检索：离线构建用户群（聚类），在线实时获取群中代表用户的行为统计
- 相关性预训练数据：人工标注 + 用户行为弱监督（2-8 比例混合）
- 跨用户 Attention 候选集大小 ≤ 10（防止延迟）

## 面试考点

1. **Q**: 相关性和 CTR 为什么要联合建模？  
   **A**: 高相关但低 CTR 的广告（优质但无吸引力）和低相关高 CTR（诱导点击但低转化）都是问题；联合建模在相关性约束下优化 CTR，避免两者脱节。

2. **Q**: 跨用户偏好挖掘的工程挑战？  
   **A**: 相似用户实时检索（延迟）、相似用户定义（个性化 vs 人口统计 vs 行为）、隐私合规（不能直接用他人行为数据）。

3. **Q**: 冷启动用户为什么从跨用户挖掘获益最多？  
   **A**: 冷启动用户自身历史少，个人偏好信号弱；跨用户信息提供了替代信号（相似人群的行为统计），信息增益最大。
