# RTBAgent: A LLM-based Agent System for Real-Time Bidding

> 来源：https://arxiv.org/abs/2502.00792 | 领域：ads | 学习日期：20260420 | WWW Companion 2025

## 问题定义

RTB（实时竞价）中，传统方法（深度学习/RL）面临两个核心挑战：
1. **线上线下分布偏移**：离线训练环境与在线竞价环境存在显著差异
2. **竞价环境快速波动**：竞争对手策略、流量分布、出价水平随时变化

传统方法一旦部署就是"固定策略"，缺乏对市场变化的实时适应能力。

## 核心方法与创新点

**首个基于 LLM 的 RTB Agent 系统**，核心架构：

### 1. 四工具模块（Tool Use）

| 工具 | 功能 | 输入 | 输出 |
|------|------|------|------|
| CTR 预估模型 | 预测广告点击率 | 用户-广告特征 | pCTR |
| 专家策略知识库 | 提供出价公式/规则 | 当前市场状态 | 推荐出价策略 |
| 市场分析器 | 分析竞价环境 | 历史竞价数据 | 市场信号摘要 |
| 预算计算器 | 计算剩余预算 | 消耗记录 | 预算状态 |

### 2. 三记忆模块（Memory）

| 记忆类型 | 内容 | 更新频率 |
|----------|------|----------|
| 交易记录（Transaction Memory） | 历史竞价-结果对 | 每次竞价 |
| 决策记录（Decision Memory） | LLM 的推理过程 | 每次决策 |
| 日反思（Daily Reflection） | 全天策略复盘总结 | 每日 |

### 3. 两步决策过程

```
Step 1（策略层）：LLM 分析市场状态 + 记忆 → 确定出价策略方向
Step 2（执行层）：基于策略方向 + CTR 预估 + 预算状态 → 具体出价金额
```

## 关键性能

- 相比传统竞价算法：**Win Rate +11.3%, ROI +8.7%**
- 在 100 万+ 竞价请求的真实广告平台数据上验证

## 核心 Insight

1. **LLM Agent 的价值在于"适应性"而非"精度"** —— 传统模型精度更高，但 RTBAgent 在市场剧变时能快速调整策略，传统模型需要重新训练
2. **Daily Reflection 是 Agent 自我改进的关键** —— 类似人类复盘，LLM 总结全天得失并更新策略偏好
3. **工具使用让 LLM 不需要"懂" CTR 预估** —— LLM 负责高层决策，精确计算交给专用工具

## 与同类工作对比

| 维度 | RTBAgent | GRAD (Diffusion) | KBD (双过程) | RL 出价 |
|------|----------|-------------------|-------------|---------|
| 核心范式 | LLM Agent | 生成式预训练 | 知识+RL | 强化学习 |
| 适应性 | 强（实时推理） | 中（预训练迁移） | 中（知识注入） | 弱（固定策略） |
| 推理延迟 | 高（LLM 推理） | 中（Diffusion 10-20步） | 低 | 极低 |
| 可解释性 | 高（LLM 推理链） | 低 | 中 | 低 |
| 适用规模 | 中（延迟限制） | 大（批处理） | 大 | 大 |

## 面试考点

- Q: RTBAgent 的延迟问题如何解决？
  > 论文方案是两步决策：策略层可以异步/低频运行（分钟级），执行层结合策略方向做快速计算。但对于毫秒级 RTB，纯 LLM 在线推理仍不现实，更适合辅助策略制定而非实时出价。
- Q: LLM Agent 出价 vs RL 出价的根本区别？
  > RL 是"学习最优映射 $s \to a$"，LLM Agent 是"理解当前情境后推理决策"。RL 更高效但脆弱（分布偏移），Agent 更鲁棒但慢。未来趋势是 Agent 做策略层 + RL/规则做执行层。
