# Creative Generation with LLM for Ad Recommendation
> 来源：工业论文 | 领域：广告系统 | 学习日期：20260327

## 问题定义
广告创意（文案、图片）的质量直接影响CTR，但：
1. **人工创作瓶颈**：人工设计师无法为海量广告（数百万条）逐一设计个性化创意
2. **千人千面需求**：不同用户对广告表达方式有不同偏好
3. **合规风险**：自动生成的创意可能包含违规内容（虚假宣传、夸大效果）

**目标**：使用LLM自动生成个性化广告创意，RLHF对齐点击率和合规性要求，实现CTR+9%。

## 核心方法与创新点

### 1. 个性化文案生成
输入条件：
```
广告主提供的产品信息 + 用户画像 + 投放场景
→ LLM → 个性化广告文案
```

用户画像包含：年龄段、兴趣标签、购买力水平、语言风格偏好（严肃/活泼/专业）。

### 2. RLHF双目标对齐
**奖励函数设计**：

$$
r(text) = \alpha \cdot \text{CTR}_{\text{Reward}}(text) + \beta \cdot \text{Compliance}_{\text{Reward}}(text)
$$

- **CTR奖励**：用离线CTR预测模型评估文案的预期点击率
- **合规奖励**：用文本分类器识别违规内容（-1表示违规，0-1表示合规程度）

### 3. 合规Reward Model训练
使用标注数据训练合规分类器：
- 正样本：审核通过的历史广告文案
- 负样本：被拒绝的违规广告文案
Binary cross-entropy训练，输出合规概率。

### 4. 多轮优化
```
Step 1: 监督预训练（用优质历史文案）
Step 2: 奖励模型训练（CTR + 合规分类器）
Step 3: PPO强化学习（最大化双目标奖励）
Step 4: 人工审核兜底（高风险内容二次确认）
```

### 5. 创意多样性
同一广告生成N个候选文案（N=5-10），在线系统进行多臂赌博机（MAB）选择最优版本。

## 实验结论
- **CTR提升**：+9%（个性化文案 vs 统一文案）
- **合规通过率**：98.5%（RLHF后，未经RLHF为85%）
- **创作效率**：人工创作时间减少80%
- MAB探索：新文案上线后24小时内识别出最优版本

## 工程落地要点
1. **文案长度控制**：不同广告位字数限制不同（30字/80字/200字），需要在生成时约束长度
2. **品牌声调一致性**：奢侈品广告需要高端文案，大众消费品需要亲民文案，通过system prompt控制
3. **A/B测试框架**：每个广告的多个文案版本需要独立的流量分配和效果追踪
4. **实时生成 vs 离线缓存**：高频广告使用离线预生成+缓存，低频广告实时生成
5. **人工审核接口**：保留人工审核通道，对自动通过率低的类目（金融、医疗）强制人审

## 面试考点
Q1: 广告文案生成中CTR奖励和合规奖励会冲突吗？如何处理？
A: 确实存在冲突：某些吸引眼球的夸大表述（"绝对有效！"）CTR高但合规分低。处理方式：(1)设置合规硬约束：合规分<0.8的文案直接丢弃，不论CTR多高；(2)惩罚项：合规奖励为负时（违规），直接给予大惩罚（-10），优先级高于CTR奖励；(3)分阶段优化：先用合规奖励过滤，在合规候选集中再优化CTR。

Q2: 如何评估生成文案的个性化程度？
A: (1)点击率分层分析：按用户群体分层（年龄/兴趣/地域），个性化文案应该让不同群体的CTR都高于统一文案；(2)文本多样性指标：计算不同用户群体收到的文案的语义相似度（越低越个性化）；(3)人工盲测：给审核人员一批文案和用户画像，评估文案与用户的匹配程度；(4)用户反馈：收集"这条广告与我相关"的用户反馈。

Q3: LLM生成广告文案的"幻觉"风险如何控制？
A: (1)事实约束：广告文案不允许添加产品信息页面没有的属性（如"获得诺贝尔奖"），用RAG将产品信息注入生成过程；(2)事实验证：用独立的fact-checking模型验证文案中的数字、功效描述；(3)广告主审核：生成后先发给广告主确认，再投放；(4)合规拦截：专门的合规分类器过滤高风险描述（"治疗"、"最有效"等）。
