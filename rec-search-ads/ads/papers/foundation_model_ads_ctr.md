# Foundation Model for Ads: Scaling CTR Prediction with Pre-training

> 来源：arXiv 2503.15793 | 领域：ads | 学习日期：20260402

## 问题定义

广告 CTR 预估模型通常从零开始训练，无法利用跨场景、跨平台的通用知识。借鉴 NLP/CV 领域的基础模型范式，本文探索广告基础模型——通过大规模预训练学习通用的用户-广告交互模式，再微调适配具体场景。

## 核心方法与创新点

1. **大规模预训练**：在多平台、多场景的广告数据上进行自监督预训练，学习通用的特征表征
2. **多模态输入**：融合广告文本、图片、用户行为序列和上下文特征的多模态表征
3. **场景自适应微调**：设计轻量级 Adapter 层适配不同广告场景（搜索广告、信息流广告、开屏广告）
4. **Scaling Law 验证**：验证了预训练数据量和模型参数量与下游 CTR 性能的幂律关系

## 实验结论

- 预训练模型在 5 个下游广告场景上平均 AUC 提升 1.5%
- 新场景冷启动效率提升 60%，减少 70% 的标注数据需求
- Scaling Law 预测准确度 > 95%

## 工程落地要点

- **预训练数据**：需要跨场景数据的统一处理和隐私合规
- **微调成本**：Adapter 微调远低于全量微调，适合多场景部署
- **模型更新**：基础模型定期全量更新，Adapter 高频增量更新

## 常见考点

1. **Q：广告基础模型和 LLM 基础模型的区别？**
   A：广告模型以稀疏特征和行为序列为主要输入，预训练目标是点击/转化预测而非语言建模。
2. **Q：为什么需要场景自适应？**
   A：不同广告场景的用户意图和行为模式差异大（搜索意图明确 vs 信息流随意浏览），统一模型需要适配。

## 模型架构详解

### 输入表示
- **用户侧**：用户画像特征（年龄/性别/地域）+ 行为序列 Embedding（Target Attention 或 Transformer）
- **物品侧**：物品 ID Embedding + 属性特征（类目/品牌/价格区间）+ 内容特征（标题 BERT Embedding）
- **上下文**：时间/设备/页面位置/请求上下文

### 核心网络
- 特征交叉层：显式（DCN Cross Layer）或隐式（DNN）的高阶特征交叉
- 序列建模：Multi-Head Self-Attention 编码用户行为序列
- 多任务头：CTR/CVR/时长等多目标共享底座 + 独立 Tower

### 训练优化
- 损失函数：Binary Cross-Entropy + 辅助任务正则
- 采样策略：全量曝光样本或降采样 + 校准
- 分布式训练：参数服务器（PS）模式处理超大规模稀疏特征

## 面试深度追问

- **Q: 大规模 CTR 模型的在线 serving 架构？**
  A: 特征计算（特征服务）→ 模型推理（GPU/CPU 集群）→ 校准（Platt Scaling）→ 融合排序。关键优化：特征缓存、模型并行、量化推理。

- **Q: 如何处理样本延迟反馈（delayed feedback）？**
  A: 1) 等待足够归因窗口再训练；2) 重要性采样修正延迟 bias；3) 实时正样本 + 延迟负样本的分离处理。

- **Q: 特征工程 vs 端到端学习的权衡？**
  A: 工业界通常是 "特征工程 + 深度模型" 的混合方案。手工特征提供先验知识和可解释性，深度模型捕获高阶交互。纯端到端在冷启动和长尾上效果不稳定。

- **Q: 如何做离线-在线一致性校验？**
  A: 1) 特征一致性：离线和在线使用相同的特征 pipeline；2) 预测一致性：同一批样本离线和在线预测分数差异 < 0.1%；3) 监控指标：GAUC/LogLoss 日级趋势对比。
