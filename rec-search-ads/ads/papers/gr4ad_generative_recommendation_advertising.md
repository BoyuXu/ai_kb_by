# GR4AD: Generative Recommendation for Large-Scale Advertising

> 来源：arXiv 2602.22732 | 领域：ads | 学习日期：20260402

## 问题定义

大规模广告系统的生成式推荐面临实时性挑战：传统生成式模型的自回归解码速度难以满足毫秒级的在线投放要求。GR4AD 提出支持毫秒级多候选生成的实时生成式广告推荐方案。

## 核心方法与创新点

1. **动态束搜索**：自适应调整 beam width 和搜索深度，在质量和速度间动态平衡
2. **预计算缓存**：高频用户的部分解码状态预计算并缓存，减少在线计算量
3. **多候选并行生成**：利用 GPU 并行性同时生成多个候选广告序列
4. **Serving 优化**：量化推理 + KV Cache 复用 + 投机解码的组合优化

## 实验结论

- 在线推理延迟降低到 5ms 以内（P99），满足广告投放 SLA
- 相比贪心解码，动态束搜索在保持延迟约束下 CTR 提升 1.3%
- 缓存命中率 > 85%，有效减少重复计算

## 工程落地要点

- **缓存管理**：用户级缓存需要考虑缓存失效策略（行为更新后失效）
- **降级策略**：当延迟超过阈值时自动降级为贪心解码
- **AB 测试**：需要对比不同 beam width 和缓存策略的线上效果

## 常见考点

1. **Q：生成式广告推荐的实时性挑战？**
   A：自回归解码逐 token 生成，延迟随序列长度线性增长。广告投放要求毫秒级响应，需要大量工程优化。
2. **Q：动态束搜索 vs 固定束搜索的优势？**
   A：固定 beam width 在简单查询上浪费计算，在复杂查询上不够充分。动态调整可以根据当前解码难度分配计算资源。

## 模型架构详解

### 输入表示
- **用户侧**：用户画像特征（年龄/性别/地域）+ 行为序列 Embedding（Target Attention 或 Transformer）
- **物品侧**：物品 ID Embedding + 属性特征（类目/品牌/价格区间）+ 内容特征（标题 BERT Embedding）
- **上下文**：时间/设备/页面位置/请求上下文

### 核心网络
- 特征交叉层：显式（DCN Cross Layer）或隐式（DNN）的高阶特征交叉
- 序列建模：Multi-Head Self-Attention 编码用户行为序列
- 多任务头：CTR/CVR/时长等多目标共享底座 + 独立 Tower

### 训练优化
- 损失函数：Binary Cross-Entropy + 辅助任务正则
- 采样策略：全量曝光样本或降采样 + 校准
- 分布式训练：参数服务器（PS）模式处理超大规模稀疏特征

## 面试深度追问

- **Q: 大规模 CTR 模型的在线 serving 架构？**
  A: 特征计算（特征服务）→ 模型推理（GPU/CPU 集群）→ 校准（Platt Scaling）→ 融合排序。关键优化：特征缓存、模型并行、量化推理。

- **Q: 如何处理样本延迟反馈（delayed feedback）？**
  A: 1) 等待足够归因窗口再训练；2) 重要性采样修正延迟 bias；3) 实时正样本 + 延迟负样本的分离处理。

- **Q: 特征工程 vs 端到端学习的权衡？**
  A: 工业界通常是 "特征工程 + 深度模型" 的混合方案。手工特征提供先验知识和可解释性，深度模型捕获高阶交互。纯端到端在冷启动和长尾上效果不稳定。

- **Q: 如何做离线-在线一致性校验？**
  A: 1) 特征一致性：离线和在线使用相同的特征 pipeline；2) 预测一致性：同一批样本离线和在线预测分数差异 < 0.1%；3) 监控指标：GAUC/LogLoss 日级趋势对比。
