# QARM: Quantitative Alignment Multi-Modal Recommendation at Kuaishou
> 来源：https://arxiv.org/abs/2411.11739 | 领域：ads | 学习日期：20260401

**出处**：快手（Kuaishou）推荐算法团队  
**作者**：Xinchen Luo, Jiangxia Cao, Tianyu Sun 等（快手 + 阿里）  
**关键词**：多模态推荐、量化对齐、视觉特征、CTR预估

## 问题定义

在工业推荐系统中，多模态信息（图片、视频、文字）对提升 CTR/CVR 预测有显著价值。然而，将多模态大模型引入推荐系统面临两大核心挑战：

### 挑战1：表示不匹配（Representation Unmatching）

**现象**：多模态预训练模型（CLIP、ViT等）的表示空间与推荐模型的特征空间存在本质差异：
- 多模态模型用 NLP/CV 任务（图文对齐、图像分类）训练
- 推荐模型用用户-物品交互（点击、购买）训练
- 两者的优化目标不同，表示空间难以直接兼容

**后果**：直接将多模态 Embedding 输入推荐模型，特征适配差，效果提升有限甚至下降。

### 挑战2：表示不可更新（Representation Unlearning）

**现象**：工业系统中，多模态特征通常**离线预计算并缓存**：
- 多模态 Embedding 存储在特征库（Feature Store）中
- 推荐模型训练时，多模态特征作为固定输入，梯度不回传到多模态模型
- 导致多模态表示无法通过推荐任务进一步优化

**后果**：多模态特征是"死"的特征，不能随推荐系统学习而演进，性能上限低。

## 核心方法与创新点

### QARM：量化多模态对齐框架

QARM 的核心思想：**通过量化（Quantization）将多模态连续特征离散化，使其可训练且与推荐目标对齐**。

**为什么量化能解决这两个问题？**
- 量化码本（Codebook）可以在推荐任务监督下端到端更新，解决"不可更新"问题
- 量化后的离散表示天然适配推荐系统的 ID 特征体系，解决"不匹配"问题

### 量化对齐机制

**向量量化（Vector Quantization, VQ）**：

给定多模态特征 $z \in \mathbb{R}^d$，量化到最近的码本向量：

$$
z_q = \text{VQ}(z) = e_k, \quad k = \arg\min_j \|z - e_j\|^2
$$

其中 $e_j$ 是码本（Codebook）中的第 $j$ 个向量，共 $K$ 个。

**对齐目标**：
1. **重建损失**：量化向量尽量保留原始多模态信息

$$
\mathcal{L}_{recon} = \|z - z_q\|^2
$$

2. **推荐对齐损失**：量化码本在推荐任务监督下优化

$$
\mathcal{L}_{rec} = \text{CTR}}_{\text{{\text{loss}}}(\text{predict}(\text{user}, z_q), \text{label})
$$

3. **联合优化**：

$$
\mathcal{L}_{total} = \mathcal{L}_{rec} + \alpha \cdot \mathcal{L}_{recon}
$$

**直通估计器（Straight-Through Estimator）**：
量化操作不可导，使用直通估计器将梯度直接传回编码器：

$$
\frac{\partial \mathcal{L}}{\partial z} \approx \frac{\partial \mathcal{L}}{\partial z_q}
$$

### 面向不同下游模型的定制化表示

QARM 的量化框架是**下游任务感知**的：
- 同一个物品的多模态特征，对不同推荐模型（CTR/CVR/排序）训练不同码本
- 每个下游模型有专属的量化码本，最大化与各自任务的对齐
- 通过量化，多模态特征变成"可训练 ID"，融入推荐模型的 Embedding 体系

### 与级联范式的对比

**传统级联范式（Cascading Paradigm）**：
```
多模态预训练 → 固定Embedding → 推荐模型输入
（两阶段，不可端到端）
```

**QARM 范式**：
```
多模态预训练 → 量化对齐 → 可更新离散码本 → 推荐模型
（端到端优化，量化码本参与梯度更新）
```

## 实验结论

**快手大规模真实系统（工业实验）**：
- 在快手短视频推荐系统（日活亿级用户）上验证
- CTR 提升：相比固定多模态特征基线显著提升
- AUC 提升：多个下游任务均有改善
- 在线 A/B 实验：核心推荐指标（完播率、互动率）提升

**消融实验**：
- 有 QARM 量化 vs 直接使用多模态 Embedding：QARM 显著更好
- 联合损失 vs 仅推荐损失：联合优化保留更多多模态信息

**码本大小分析**：
- K=256 ~ 4096 区间内，性能随码本大小先增后稳定
- 工业部署中选 K=1024 作为性价比最优点

## 工程落地要点

1. **离线量化预计算**：量化后的码本 ID 可以离线预计算存储，在线只需查表，推理零额外延迟
2. **码本更新策略**：使用 Exponential Moving Average（EMA）更新码本，训练更稳定
3. **码本坍塌（Codebook Collapse）**：部分码本向量可能从不被使用，需要重置（Random Restart 策略）
4. **多粒度量化**：可以对不同尺度的特征（全局语义/局部细节）使用不同码本
5. **存储效率**：量化后用整数 ID 存储（vs 原始浮点向量），显存/磁盘节省 4-8x
6. **与 ID 特征融合**：量化多模态 ID 与传统行为 ID 特征通过 Embedding 层统一处理

## 常见考点

**Q1: 为什么多模态特征直接输入推荐模型效果不好？**
A: 两个核心原因：（1）表示不匹配：多模态模型用 NLP/CV 任务训练，特征空间与推荐任务不对齐；（2）梯度断层：缓存的固定 Embedding 无法通过推荐任务梯度优化，多模态模型无法学到推荐相关的表示。

**Q2: 向量量化（VQ）在推荐系统中的优势是什么？**
A: （1）将连续多模态特征转换为离散 ID，无缝融入推荐系统的 Embedding 体系；（2）码本参数可以在推荐任务监督下更新；（3）离散表示存储效率高（整数 vs 浮点向量）；（4）天然的正则化效果，避免 Embedding 坍塌。

**Q3: QARM 如何实现端到端训练？量化操作不可导怎么解决？**
A: 使用直通估计器（Straight-Through Estimator）：前向传播时使用量化值，反向传播时梯度直接"穿过"量化操作，当作恒等映射处理（即 $\partial z_q / \partial z \approx 1$）。这是 VQ-VAE 等工作的标准做法。

**Q4: 什么是码本坍塌（Codebook Collapse）？如何处理？**
A: 部分码本向量从不被任何样本选中（死码），导致有效码本容量下降。解决方案：（1）Random Restart：将长期未使用的码本向量随机重置为当前批次的某个输入向量；（2）Commitment Loss 增大，鼓励编码器使用更多不同的码本向量；（3）Product Quantization：将特征分组量化，每组独立码本。
