# Causal Bidding: Counterfactual Reasoning for Value-based Bid Optimization

> 来源：arXiv 2502.14123 | 领域：ads | 学习日期：20260402

## 问题定义

广告出价优化中存在严重的观测偏差：历史出价数据受到当时策略的影响，直接在偏差数据上训练会导致策略陷入局部最优。Causal Bidding 引入因果推断和反事实推理框架来消除竞价偏差。

## 核心方法与创新点

1. **因果图建模**：将竞价过程建模为因果图（DAG），识别出价→曝光→点击→转化的因果链
2. **反事实推理**：估计"如果出价为 x，该广告的预期收益是多少"——即使实际出价不是 x
3. **倾向评分加权**：用 IPW（Inverse Propensity Weighting）校正历史出价策略的选择偏差
4. **Double-Robust 估计**：结合模型预测和 IPW 的双重鲁棒方法，提升反事实估计的稳定性

## 实验结论

- 反事实出价优化使广告主 ROI 提升 11.3%
- 偏差校正后的 OPE（Off-Policy Evaluation）与在线效果的相关性从 0.62 提升到 0.89
- 在长尾广告主上效果尤为显著（ROI 提升 18%）

## 工程落地要点

- **因果图验证**：需要领域专家确认因果假设的合理性
- **倾向分数估计**：大规模竞价中精确估计倾向分数是难点
- **线上-线下一致性**：反事实估计结果需要线上验证

## 常见考点

1. **Q：竞价中的选择偏差是什么？**
   A：历史数据中只能观测到实际出价下的结果，无法观测"如果出价不同会怎样"，直接用偏差数据训练会学到次优策略。
2. **Q：IPW 的缺点和 Double-Robust 的优势？**
   A：IPW 在倾向分数极端时方差很大；Double-Robust 在模型或 IPW 其中一个正确时即可无偏，更鲁棒。

## 出价策略架构详解

### 状态表示
- **市场状态**：竞争强度（eCPM 分布）、流量模式（时段特征）、价格水平
- **广告状态**：剩余预算比例、累计 ROI、已消耗时间比例
- **约束条件**：预算约束、ROI 约束、频次约束

### 策略网络
- 输入：状态向量（市场 + 广告 + 约束）
- 网络：多层 MLP 或 Transformer，输出出价动作
- 约束处理：拉格朗日乘子法或投影梯度法

### 训练与部署
- 离线训练：基于历史竞价日志的 Offline RL（BCQ/CQL/TD3+BC）
- 在线微调：安全策略改进（trust region / PPO）
- 安全网：出价上下限 + 预算告警 + ROI 兜底

## 面试深度追问

- **Q: Offline RL 在出价中的应用和挑战？**
  A: 挑战：分布外（OOD）动作估值不准确导致策略过于激进。解决方案：BCQ（限制动作在数据分布内）、CQL（惩罚 OOD 动作的 Q 值）、One-step RL。

- **Q: 如何处理多广告主同时竞价的博弈均衡？**
  A: 1) 均场博弈（Mean Field Game）近似多智能体交互；2) 各广告主独立优化但考虑市场价格反馈；3) 平台层面做 Pacing 协调避免流量过度竞争。

- **Q: 实时出价（RTB）的系统架构？**
  A: DSP 收到 Ad Exchange 的 bid request → 特征提取 → CTR/CVR 预估 → 出价决策 → 返回 bid response。全链路 < 100ms，出价决策 < 10ms。

- **Q: 预算 Pacing 的经典方法？**
  A: 1) Throttling：按概率丢弃竞价请求，控制消耗速率；2) Bid Modification：调整出价倍数平滑消耗；3) PID 控制：基于目标-实际消耗偏差的比例-积分-微分控制。
