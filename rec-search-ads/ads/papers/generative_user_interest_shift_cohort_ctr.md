# Generative Modeling of User Interest Shift via Cohort-based Intent Learning for CTR

> 来源：https://arxiv.org/abs/2601.18251 | 领域：计算广告 | 学习日期：20260331

## 问题定义

用户兴趣会随时间漂移（interest shift），传统CTR模型假设兴趣静态，无法捕获突发兴趣变化。

## 核心方法与创新点

1. **群组意图学习**：将用户分组（cohort），学习群组级别的兴趣漂移模式
2. **VAE生成式建模**：

$$
z_t \sim q_\phi(z_t | z_{t-1}, x_t), \quad \hat{x}_{t+1} = p_\theta(x_{t+1} | z_t)
$$

3. **时间感知Cohort划分**：基于行为模式的动态用户分组
4. **兴趣转移矩阵**：建模不同兴趣类别间的转移概率

## 实验结论

在电商和新闻推荐数据上，interest shift明显场景（大促期间）CTR预测AUC提升0.8-1.2%。

## 工程落地要点

- Cohort划分可周期性更新（日级）
- VAE推理增加一次前向传播，延迟增加<5ms
- 群组级建模减少用户级噪声
- 适合有明显季节性/事件性业务

## 常见考点

1. **为什么群组而非个体？** 个体数据稀疏，群组聚合模式更清晰
2. **VAE在兴趣建模的优势？** 生成连续的兴趣演化轨迹
3. **检测interest shift？** 监控点击分布的KL散度变化

## 模型架构详解

### 输入表示
- **用户侧**：用户画像特征（年龄/性别/地域）+ 行为序列 Embedding（Target Attention 或 Transformer）
- **物品侧**：物品 ID Embedding + 属性特征（类目/品牌/价格区间）+ 内容特征（标题 BERT Embedding）
- **上下文**：时间/设备/页面位置/请求上下文

### 核心网络
- 特征交叉层：显式（DCN Cross Layer）或隐式（DNN）的高阶特征交叉
- 序列建模：Multi-Head Self-Attention 编码用户行为序列
- 多任务头：CTR/CVR/时长等多目标共享底座 + 独立 Tower

### 训练优化
- 损失函数：Binary Cross-Entropy + 辅助任务正则
- 采样策略：全量曝光样本或降采样 + 校准
- 分布式训练：参数服务器（PS）模式处理超大规模稀疏特征

## 面试深度追问

- **Q: 大规模 CTR 模型的在线 serving 架构？**
  A: 特征计算（特征服务）→ 模型推理（GPU/CPU 集群）→ 校准（Platt Scaling）→ 融合排序。关键优化：特征缓存、模型并行、量化推理。

- **Q: 如何处理样本延迟反馈（delayed feedback）？**
  A: 1) 等待足够归因窗口再训练；2) 重要性采样修正延迟 bias；3) 实时正样本 + 延迟负样本的分离处理。

- **Q: 特征工程 vs 端到端学习的权衡？**
  A: 工业界通常是 "特征工程 + 深度模型" 的混合方案。手工特征提供先验知识和可解释性，深度模型捕获高阶交互。纯端到端在冷启动和长尾上效果不稳定。

- **Q: 如何做离线-在线一致性校验？**
  A: 1) 特征一致性：离线和在线使用相同的特征 pipeline；2) 预测一致性：同一批样本离线和在线预测分数差异 < 0.1%；3) 监控指标：GAUC/LogLoss 日级趋势对比。
