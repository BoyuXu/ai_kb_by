## 核心知识体系

### 1. 广告系统全链路

```
广告主投放 → 广告库（亿级）
                ↓
         [召回层] 双塔/GR/ANN → Top-K 候选（万级）
                ↓
         [粗排层] 轻量级 CTR → Top-K 候选（千级）
                ↓
         [精排层] 重量级 CTR/CVR → Top-K（百级）
                ↓
         [重排层] 多目标融合 / 多样性 / 业务约束
                ↓
         [竞价] eCPM = CTR × bid（VCG/GSP）
                ↓
              广告展示
```

### 2. CTR 预估关键技术

#### 特征工程
- **用户特征**：人口属性、设备、历史行为（短/长期）
- **广告特征**：创意、类目、出价、素材质量分
- **上下文特征**：时段、位置、页面类型、query（搜索广告）
- **交叉特征**：user × ad 交叉（多路召回特征、历史共现）

#### 主流架构
| 架构 | 代表模型 | 特点 |
|------|---------|------|
| Wide & Deep | Google W&D | 记忆 + 泛化 |
| 特征交叉 | DCN-V2, FiBiNet | 高阶特征交叉 |
| 序列建模 | DIN, SIM | 用户兴趣提取 |
| 多任务 | ESMM, MMOE, PLE | CTR+CVR 联合训练 |
| Transformer | BST, CADET | 上下文感知 |

### 3. 多目标优化

```
eCPM = P(click) × P(convert|click) × bid
        ↑CTR模型     ↑CVR模型

常见多目标：点击率 + 转化率 + 时长 + 价值
优化方法：MMOE / PLE / 约束优化（MOEF）
```

### 4. 冷启动问题

- 新广告无历史数据 → ID embedding 随机初始化
- 解法：① Content-based embedding ② Meta-Learning ③ IDProxy（LLM proxy embedding）

## 2026-03-12 更新

### 本次新增内容

**1. IDProxy - 小红书多模态 LLM 解决广告冷启动**
- 来源：arXiv 2026-03-02，小红书工业论文
- 核心：用多模态 LLM 生成 proxy embedding 替代随机初始化 ID embedding
- 工程价值：冷启动阶段 CTR +5~8%，已在小红书线上部署
- 详细笔记：`../papers/IDProxy_Cold_Start_CTR_Prediction_for_Ads_and_Recommendat.md`

**2. Generative Recommendation for Large-Scale Advertising**
- 来源：arXiv 2026-03-04，头部广告平台工业论文
- 核心：GR 召回在亿级广告库上的工程化，RQ 量化 + eCPM 约束 beam search
- 工程价值：广告 RPM +2.3%，冷启动广告曝光 +30%
- 详细笔记：`../papers/Generative_Recommendation_for_Large_Scale_Advertising.md`

**3. CADET - Decoder-Only Transformer 广告 CTR**
- 来源：arXiv 2026-02-11
- 核心：causal attention 建模广告展示上下文序列，AUC +0.5~1.2%
- 详细笔记：`../papers/CADET_Context_Conditioned_Ads_CTR_Prediction_With_a_Decod.md`

**4. AdNanny - 一个推理 LLM 搞定所有离线广告任务（2026-02-01）**
- 微软工作：用单一推理 LLM 统一处理：广告相关性标注、出价建议、受众扩展等
- 工程价值：大幅降低广告系统的模型维护成本，但目前仍是离线场景

**5. ML-DCN @ Pinterest - Masked Low-Rank DCN**
- Pinterest 广告 CTR 模型：在 DCN-V2 基础上加入低秩分解和 Masked 训练
- 可扩展性更强，Pinterest 亿级广告规模验证

### 关键趋势（2026 Q1）

1. **生成式召回（GR）正在进入工业广告系统**：从学术研究到头部平台部署
2. **LLM 赋能广告冷启动**：proxy embedding、content understanding 两条路线并行
3. **上下文感知 CTR**：Transformer 上下文建模成主流，Decoder-Only 架构开始用于广告
