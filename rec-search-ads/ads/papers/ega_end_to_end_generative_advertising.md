# EGA: End-to-End Generative Framework for Industrial Advertising

> 来源：arXiv 2505.17549 | 领域：ads | 学习日期：20260402

## 问题定义

传统广告系统采用级联架构（召回→粗排→精排→重排），各阶段独立优化导致全局次优，且信息在阶段间传递时存在损失。EGA 提出首个统一建模用户兴趣、POI（兴趣点）和创意的端到端生成式广告框架。

## 核心方法与创新点

1. **端到端生成式架构**：用一个自回归 Transformer 统一广告系统的多个阶段，直接从用户特征生成最优广告+创意组合
2. **三维建模**：同时建模用户兴趣（who）、POI/场景（where/when）和广告创意（what）的三元关系
3. **Semantic ID 编码**：广告和创意用层次化 Semantic ID 表示，支持生成式检索
4. **在线 Serving 优化**：通过 beam search 和缓存机制实现毫秒级在线推理

## 实验结论

- 相比级联系统，广告 RPM（千次展示收入）提升 4.5%
- 端到端优化消除了阶段间的信息损失，CTR 提升 2.1%
- 创意与用户的匹配度提升 18%

## 工程落地要点

- **渐进式替换**：建议从精排开始逐步替换级联系统，而非一步到位
- **Serving 延迟**：端到端模型的推理延迟需要严格控制在广告投放 SLA 内
- **可解释性**：生成过程的中间状态可以提供推荐理由

## 面试考点

1. **Q：端到端 vs 级联架构的 trade-off？**
   A：端到端避免信息损失和局部最优，但模型复杂度高、调试困难。级联架构模块化便于独立优化和排查问题。
2. **Q：为什么要统一建模用户、POI 和创意？**
   A：三者存在强关联——同一用户在不同场景对同一广告的不同创意有不同反馈，独立建模无法捕捉这种交互。
