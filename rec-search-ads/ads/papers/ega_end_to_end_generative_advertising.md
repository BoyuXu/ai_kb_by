# EGA: End-to-End Generative Framework for Industrial Advertising

> 来源：arXiv 2505.17549 | 领域：ads | 学习日期：20260402

## 问题定义

传统广告系统采用级联架构（召回→粗排→精排→重排），各阶段独立优化导致全局次优，且信息在阶段间传递时存在损失。EGA 提出首个统一建模用户兴趣、POI（兴趣点）和创意的端到端生成式广告框架。

## 核心方法与创新点

1. **端到端生成式架构**：用一个自回归 Transformer 统一广告系统的多个阶段，直接从用户特征生成最优广告+创意组合
2. **三维建模**：同时建模用户兴趣（who）、POI/场景（where/when）和广告创意（what）的三元关系
3. **Semantic ID 编码**：广告和创意用层次化 Semantic ID 表示，支持生成式检索
4. **在线 Serving 优化**：通过 beam search 和缓存机制实现毫秒级在线推理

## 实验结论

- 相比级联系统，广告 RPM（千次展示收入）提升 4.5%
- 端到端优化消除了阶段间的信息损失，CTR 提升 2.1%
- 创意与用户的匹配度提升 18%

## 工程落地要点

- **渐进式替换**：建议从精排开始逐步替换级联系统，而非一步到位
- **Serving 延迟**：端到端模型的推理延迟需要严格控制在广告投放 SLA 内
- **可解释性**：生成过程的中间状态可以提供推荐理由

## 常见考点

1. **Q：端到端 vs 级联架构的 trade-off？**
   A：端到端避免信息损失和局部最优，但模型复杂度高、调试困难。级联架构模块化便于独立优化和排查问题。
2. **Q：为什么要统一建模用户、POI 和创意？**
   A：三者存在强关联——同一用户在不同场景对同一广告的不同创意有不同反馈，独立建模无法捕捉这种交互。

## 模型架构详解

### 输入表示
- **用户侧**：用户画像特征（年龄/性别/地域）+ 行为序列 Embedding（Target Attention 或 Transformer）
- **物品侧**：物品 ID Embedding + 属性特征（类目/品牌/价格区间）+ 内容特征（标题 BERT Embedding）
- **上下文**：时间/设备/页面位置/请求上下文

### 核心网络
- 特征交叉层：显式（DCN Cross Layer）或隐式（DNN）的高阶特征交叉
- 序列建模：Multi-Head Self-Attention 编码用户行为序列
- 多任务头：CTR/CVR/时长等多目标共享底座 + 独立 Tower

### 训练优化
- 损失函数：Binary Cross-Entropy + 辅助任务正则
- 采样策略：全量曝光样本或降采样 + 校准
- 分布式训练：参数服务器（PS）模式处理超大规模稀疏特征

## 面试深度追问

- **Q: 大规模 CTR 模型的在线 serving 架构？**
  A: 特征计算（特征服务）→ 模型推理（GPU/CPU 集群）→ 校准（Platt Scaling）→ 融合排序。关键优化：特征缓存、模型并行、量化推理。

- **Q: 如何处理样本延迟反馈（delayed feedback）？**
  A: 1) 等待足够归因窗口再训练；2) 重要性采样修正延迟 bias；3) 实时正样本 + 延迟负样本的分离处理。

- **Q: 特征工程 vs 端到端学习的权衡？**
  A: 工业界通常是 "特征工程 + 深度模型" 的混合方案。手工特征提供先验知识和可解释性，深度模型捕获高阶交互。纯端到端在冷启动和长尾上效果不稳定。

- **Q: 如何做离线-在线一致性校验？**
  A: 1) 特征一致性：离线和在线使用相同的特征 pipeline；2) 预测一致性：同一批样本离线和在线预测分数差异 < 0.1%；3) 监控指标：GAUC/LogLoss 日级趋势对比。
