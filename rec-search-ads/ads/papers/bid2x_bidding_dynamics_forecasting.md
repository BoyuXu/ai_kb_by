# Bid2X: Revealing Dynamics of Bidding Environment in Online Advertising

> 来源：https://arxiv.org/abs/2510.23410 | 领域：计算广告 | 学习日期：20260331

## 问题定义

在线广告竞价环境高度动态，广告主难以预测竞价环境变化，导致预算浪费和ROI下降。

## 核心方法与创新点

1. **竞价环境预测**：基于历史竞价数据预测未来竞争强度和市场价格分布
2. **时序建模**：Transformer编码竞价时序信号
3. **多粒度预测**：从campaign级到keyword级的层级预测
4. **不确定性估计**：输出竞价价格的置信区间

$$
\hat{p}_{t+1} = \text{Decoder}(\text{Encoder}([p_{t-w}, ..., p_t], [c_{t-w}, ..., c_t]))
$$

## 实验结论

Market price预测MAPE降低23%，基于预测的自动出价策略使ROI提升15%。

## 工程落地要点

- 预测模型需实时特征（当前竞争者数、实时竞价价格）
- 可与自动出价系统集成
- 多粒度预测支持不同层级预算分配
- 不确定性估计帮助风险控制

## 常见考点

1. **竞价环境预测核心挑战？** 非平稳性、竞争者策略不可观测
2. **处理竞争者行为不确定性？** 概率建模+集成学习
3. **预测结果如何用于出价？** 预测市场价格→调整出价→控制预算

## 出价策略架构详解

### 状态表示
- **市场状态**：竞争强度（eCPM 分布）、流量模式（时段特征）、价格水平
- **广告状态**：剩余预算比例、累计 ROI、已消耗时间比例
- **约束条件**：预算约束、ROI 约束、频次约束

### 策略网络
- 输入：状态向量（市场 + 广告 + 约束）
- 网络：多层 MLP 或 Transformer，输出出价动作
- 约束处理：拉格朗日乘子法或投影梯度法

### 训练与部署
- 离线训练：基于历史竞价日志的 Offline RL（BCQ/CQL/TD3+BC）
- 在线微调：安全策略改进（trust region / PPO）
- 安全网：出价上下限 + 预算告警 + ROI 兜底

## 面试深度追问

- **Q: Offline RL 在出价中的应用和挑战？**
  A: 挑战：分布外（OOD）动作估值不准确导致策略过于激进。解决方案：BCQ（限制动作在数据分布内）、CQL（惩罚 OOD 动作的 Q 值）、One-step RL。

- **Q: 如何处理多广告主同时竞价的博弈均衡？**
  A: 1) 均场博弈（Mean Field Game）近似多智能体交互；2) 各广告主独立优化但考虑市场价格反馈；3) 平台层面做 Pacing 协调避免流量过度竞争。

- **Q: 实时出价（RTB）的系统架构？**
  A: DSP 收到 Ad Exchange 的 bid request → 特征提取 → CTR/CVR 预估 → 出价决策 → 返回 bid response。全链路 < 100ms，出价决策 < 10ms。

- **Q: 预算 Pacing 的经典方法？**
  A: 1) Throttling：按概率丢弃竞价请求，控制消耗速率；2) Bid Modification：调整出价倍数平滑消耗；3) PID 控制：基于目标-实际消耗偏差的比例-积分-微分控制。
