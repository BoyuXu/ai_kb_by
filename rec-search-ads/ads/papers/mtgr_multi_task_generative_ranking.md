# MTGR: Multi-Task Generative Ranking for Industrial-Scale Advertising

> 来源：arXiv 2505.12663 (美团) | 领域：ads | 学习日期：20260402

## 问题定义

美团广告系统面临多目标优化挑战：需要同时优化 CTR、CVR、GMV 等多个目标，且不同目标间存在冲突。传统多任务模型（MMoE、PLE）在生成式推荐范式下的适配性不足。MTGR 提出基于生成式架构的多任务排序框架。

## 核心方法与创新点

1. **生成式多任务架构**：在 HSTU 生成式推荐基础上引入多任务输出头，统一序列建模与多目标预估
2. **DLRM 交叉特征融合**：证明生成式推荐仍需要传统特征交叉（Deep & Cross），不能仅依赖序列建模
3. **任务间注意力**：设计 cross-task attention 模块让不同任务的表征互相增强
4. **Pareto 最优搜索**：使用多目标优化算法自动搜索 Pareto 前沿上的最优任务权重

## 实验结论

- 美团广告排序系统 GMV 提升 2.1%，CTR 提升 0.8%，CVR 提升 1.5%
- 生成式架构 + DLRM 特征交叉的组合优于纯生成式或纯 DLRM
- Pareto 搜索比人工调权效率提升 10 倍

## 工程落地要点

- **特征工程不可省**：即使用了生成式架构，传统特征交叉仍然是重要的性能来源
- **多目标权重**：需要根据业务目标（收入导向 vs 用户体验）动态调整
- **在线评估**：多目标系统需要综合指标体系评估

## 常见考点

1. **Q：为什么生成式推荐还需要 DLRM 特征交叉？**
   A：生成式模型擅长序列模式捕捉，但对静态特征间的交叉（如用户年龄×物品类目）建模能力有限。两者互补。
2. **Q：Pareto 最优在多目标优化中的作用？**
   A：Pareto 前沿是所有不能被其他解同时优于的解的集合。业务方可以在 Pareto 前沿上选择最符合当前目标的权重方案。

## 模型架构详解

### 输入表示
- **用户侧**：用户画像特征（年龄/性别/地域）+ 行为序列 Embedding（Target Attention 或 Transformer）
- **物品侧**：物品 ID Embedding + 属性特征（类目/品牌/价格区间）+ 内容特征（标题 BERT Embedding）
- **上下文**：时间/设备/页面位置/请求上下文

### 核心网络
- 特征交叉层：显式（DCN Cross Layer）或隐式（DNN）的高阶特征交叉
- 序列建模：Multi-Head Self-Attention 编码用户行为序列
- 多任务头：CTR/CVR/时长等多目标共享底座 + 独立 Tower

### 训练优化
- 损失函数：Binary Cross-Entropy + 辅助任务正则
- 采样策略：全量曝光样本或降采样 + 校准
- 分布式训练：参数服务器（PS）模式处理超大规模稀疏特征

## 面试深度追问

- **Q: 大规模 CTR 模型的在线 serving 架构？**
  A: 特征计算（特征服务）→ 模型推理（GPU/CPU 集群）→ 校准（Platt Scaling）→ 融合排序。关键优化：特征缓存、模型并行、量化推理。

- **Q: 如何处理样本延迟反馈（delayed feedback）？**
  A: 1) 等待足够归因窗口再训练；2) 重要性采样修正延迟 bias；3) 实时正样本 + 延迟负样本的分离处理。

- **Q: 特征工程 vs 端到端学习的权衡？**
  A: 工业界通常是 "特征工程 + 深度模型" 的混合方案。手工特征提供先验知识和可解释性，深度模型捕获高阶交互。纯端到端在冷启动和长尾上效果不稳定。

- **Q: 如何做离线-在线一致性校验？**
  A: 1) 特征一致性：离线和在线使用相同的特征 pipeline；2) 预测一致性：同一批样本离线和在线预测分数差异 < 0.1%；3) 监控指标：GAUC/LogLoss 日级趋势对比。
