# Project 1：重排层设计与实现基础

> 项目类型：深度技术指南 | 日期：20260324 | 领域：ads ranking

## 导言

重排层（Reranking Layer）是**现代广告系统中最被忽视但最有潜力的优化点**。

典型的排序 Pipeline：
```
[召回] → [粗排] → [精排] → [重排] ← 这一层通常被简化处理
```

大多数公司在精排（Fine Ranking）上投入大量资源，建设了复杂的 CTR/CVR 预估模型。但在重排层上，往往只用简单的**贪心启发式**：
- 循环法则（Round-Robin）：广告和内容交错
- 距离惩罚：避免相似广告相邻
- 频率限制：同一广告主不超过 N 次

**问题**：这些启发式无法学习，效果有天花板。

**本项目目标**：
1. 理解重排层的核心问题和算法
2. 学习从启发式 → 学习排序的演进
3. 掌握约束优化排序的工程实现
4. 探索用户反馈融入的方法

## Part A：重排层的问题定义

### A1. 输入与输出

```
【输入】
- 精排模型输出：Top-N 候选（例如 Top-50）
  每个广告有：ID、预估 CTR、CVR、品牌安全分、广告主 ID、话题、创意特征等

- 用户上下文：
  用户 ID、设备、位置、时间、最近 K 个展示历史
  
- 系统约束：
  多样性下限、广告频率上限、RPM 目标、用户体验评分下限

【输出】
- 最终排序：Top-K 个广告（通常 K < N，例如 20 个）
- 排序顺序：Ad_1, Ad_2, ..., Ad_K
- 排序原因（可选）：每个广告的排序分数、选择原因

【优化目标】
maximize: Σ Score(Ad_i)
subject to: 满足所有约束条件
```

### A2. 为什么需要重排层？（精排还不够吗）

**现象**：精排后的排序往往存在以下问题

| 问题 | 表现 | 原因 |
|-----|------|------|
| **头部拥挤** | Top-3 都是电商广告 | 精排只看 eCPM，电商出价高 |
| **话题重复** | 连续 5 个美妆广告 | 同类话题 CTR 高，容易堆积 |
| **用户疲劳** | 用户看到同一品牌 3 次 | 频率限制在精排没有 |
| **品质下降** | 低质广告突然靠前 | 精排模型欠拟合、泛化差 |
| **冷启动虚拟** | 新广告排名靠后 | 没有历史数据，模型无法预估 |

**为什么精排无法解决？**
- 精排只看单个广告的绝对分数，不看**相对位置和上下文**
- 精排模型通常离线训练，反映的是过去的分布，不是**实时用户偏好**
- 精排模型的优化目标（最大化 CTR）与实际目标（最大化 RPM 的同时保证体验）不完全对齐

**重排层的作用**：
```
精排 → 绝对评分（这个广告有多好）
重排 → 相对评分 + 约束（在给定上下文和约束下，该展示这个广告吗）
```

## Part B：重排算法详解

### B1. 启发式重排（Heuristic Reranking）

#### B1.1 循环法则（Round-Robin Reranking）

**核心思想**：交替展示不同类型的内容

```
精排结果：[电商广告, 电商广告, 美妆广告, 电商广告, ...]
           分数：     0.95      0.92       0.88       0.86

循环法则应用：
- 定义内容类型：广告 vs 内容
- 广告-内容-广告-内容交错

结果：[电商广告, [内容], 美妆广告, [内容], 电商广告, ...]
```

**参数**：
- `ad_ratio = 0.3`：广告占比 30%
- `content_ratio = 0.7`：内容占比 70%

**优点**：
- 简单快速，O(n) 复杂度
- 直观有效

**缺点**：
- 不灵活，无法学习最优比例
- 不同用户群体可能需要不同 ad_ratio
- 没有考虑特定广告的质量差异

#### B1.2 距离惩罚（Position Bias Correction）

**核心思想**：避免相似或重复的广告相邻

```
定义相似度（例如基于话题）：
sim(ad_i, ad_j) = cosine(topic_embedding_i, topic_embedding_j)

贪心选择：
1. 从精排 Top-N 中选择第 1 个（最高分数）
2. 选择第 2 个：max Score(ad) - λ × sim(ad, ad_1)
3. 选择第 3 个：max Score(ad) - λ × max(sim(ad, ad_1), sim(ad, ad_2))
...

λ 是惩罚强度参数
```

**示例**：
```
候选：[电商1:0.95, 电商2:0.92, 美妆:0.88, 服饰:0.85]

λ=0.1
- 第 1 位：电商1 (0.95)
- 第 2 位：美妆 (0.88 - 0.1×0.9 = 0.79) vs 电商2 (0.92 - 0.1×0.95 = 0.81)
          选电商2（0.81 > 0.79）？不对！应该选美妆避免重复
  
λ=0.5（强惩罚）
- 第 1 位：电商1 (0.95)
- 第 2 位：美妆 (0.88 - 0.5×0.9 = 0.43) vs 电商2 (0.92 - 0.5×0.95 = 0.44)
          选电商2（0.44 > 0.43）

λ=1.0（极强惩罚）
- 第 1 位：电商1 (0.95)
- 第 2 位：美妆 (0.88 - 1.0×0.9 = -0.02) vs 电商2 (0.92 - 1.0×0.95 = -0.03)
          选美妆！强制多样性
```

**关键参数调优**：
- λ 越大 → 多样性越强 → 分数牺牲越大
- λ 太小 → 多样性效果差
- λ 需要 A/B 测试找到最优值

#### B1.3 频率限制（Frequency Capping）

**核心思想**：限制用户在 T 时间内看到同一广告主的次数

```
定义：
freq_cap = {
  "same_advertiser": max_freq=3,  // 同一广告主最多 3 次
  "same_brand": max_freq=5,        // 同一品牌最多 5 次
  "time_window": 24h              // 在 24 小时内
}

实现（基于用户历史）：
user_history = [Ad_A (12:00), Ad_B (13:30), Ad_A (15:00), ...]
advertisers = {"A": 2, "B": 1}

新候选出现时：
if advertisers[advertiser_id] >= freq_cap["same_advertiser"]:
    不展示这个广告
else:
    可以展示
```

**关键问题**：
- 如何定义"时间窗口"？（T=24h? 7天？）
- 如何定义"同一广告主"？（整个账户 or 特定活动？）
- 频率限制是**硬约束还是软约束**？
  - 硬约束：必须遵守，可能导致填充不足
  - 软约束：违反时降低分数

### B2. 基于约束的贪心重排（Constrained Greedy Reranking）

**核心思想**：逐步选择候选，每次选择满足约束的最高分广告

```
【算法伪代码】

selected = []
remaining = candidates  // Top-N 候选

for i in 1 to K:  // 选 K 个
    for ad in remaining:
        if check_constraints(ad, selected):  // 检查所有约束
            if ad.score == max(remaining.score):
                selected.append(ad)
                remaining.remove(ad)
                break
                
return selected
```

**约束函数示例**：
```python
def check_constraints(candidate, selected_so_far, user_context):
    # 约束 1：频率限制
    freq = count_advertiser(candidate.advertiser_id, selected_so_far)
    if freq >= freq_cap:
        return False
    
    # 约束 2：多样性
    diversity = compute_diversity(candidate, selected_so_far)
    if diversity < min_diversity_threshold:
        return False
    
    # 约束 3：用户体验评分
    if candidate.quality_score < quality_threshold:
        return False
    
    # 约束 4：隐藏用户之前看过的广告
    if candidate.id in user_context["viewed_ads"]:
        return False
    
    return True
```

**时间复杂度**：O(K × N)，其中 K 是输出大小，N 是输入大小
- 通常 K=20, N=50 → 1000 次约束检查，<1ms

**优点**：
- 直观，容易添加新约束
- 线上推理快

**缺点**：
- 贪心不一定最优（可能被局部最优陷阱困住）
- 约束之间的冲突如何解决？（例如多样性 vs 高分数）

### B3. 动态规划重排（DP for Reranking）

**动机**：贪心可能不是最优，用 DP 找全局最优

```
【问题定义】
给定 N 个候选和 K 个位置，选择 K 个候选排列，最大化总分数
同时满足约束

【DP 状态】
dp[i][j][state] = 使用前 i 个候选，选了 j 个，当前约束状态为 state 时的最大分数

【状态转移】
dp[i][j][state] = max(
    dp[i-1][j][state],  // 不选候选 i
    dp[i-1][j-1][new_state] + score[i]  // 选候选 i
)

其中 new_state 是选了候选 i 后的新约束状态
```

**实际应用**：
- 如果约束数量少（<3），DP 可行
- 如果约束数量多（>10），DP 的状态空间爆炸，不实用

## Part C：从启发式到学习排序的演进

### C1. 为什么启发式不够

**问题案例**：
```
启发式规则：循环法则 ad_ratio=0.3

情况 1：用户是电商爱好者
  → ad_ratio=0.3 太少，应该 0.5
  
情况 2：用户对广告厌烦
  → ad_ratio=0.3 太多，应该 0.1
  
情况 3：黄金时段（高转化率时间）
  → ad_ratio=0.3 太少，应该 0.6
```

**根本问题**：
- 启发式是**全局固定**的，无法根据上下文调整
- 启发式**无法学习**从过去展示的效果

### C2. Learning To Rank for Reranking

**思路**：用 LTR（学习排序）替代启发式

**训练数据**：
```
【离线数据】
- 每个展示的排序结果
- 该排序下用户的点击/停留时间/转化
- 上下文特征（用户、时间、候选集）

【目标**
学习一个排序函数，最大化用户反馈指标（例如 NDCG）
```

**LTR 模型适配重排**：
```
输入特征：
- 候选广告的初始得分（来自精排）
- 广告之间的相似度（为了捕捉多样性）
- 用户历史（近期看过哪些类别）
- 上下文（时间、位置、设备）

输出：
每个候选的"重排分数"（与初始分数不同）

比较：
精排分数：0.95, 0.92, 0.88, 0.85
重排分数：0.65, 0.72, 0.88, 0.55
          （抬升多样性广告，降低重复广告）
```

**算法选择**：
- **LambdaMART**：直接优化 NDCG，最适合重排
- **ListNet**：优化 Cross-Entropy，适合小列表重排
- **RankNet**：基础 Pairwise，已过时

### C3. 强化学习重排（Reinforcement Learning for Reranking）

**关键创新**：用用户的**实时反馈**而不仅是离线标签

```
【MDP 定义】
状态 S_t = 
  - 已展示的广告列表
  - 用户交互历史（点击、停留时间）
  - 当前候选池

动作 A_t = 选择下一个广告展示

奖励 R_t =
  - +1 用户点击
  - +0.5 用户停留 > 3秒
  - +0.1 用户滑过（至少看到了）
  - 0 用户跳过

目标：
maximize E[Σ γ^t R_t]
```

**实现方案**：
1. **Contextual Bandit**（简化）：不考虑长期影响，每次只决策下一个广告
2. **Policy Gradient（A3C）**：用神经网络学习策略，考虑长期影响
3. **Q-Learning**：离线学习价值函数

**工业应用场景**：
- 推荐系统的重排（头部平台如 TikTok、YouTube）
- 广告系统中的出价优化（与重排可结合）

## Part D：多样性度量与优化

### D1. 多样性指标体系

```
【话题多样性】
topic_diversity = (不同话题数) / (总广告数)
示例：20 个广告中有 12 个不同话题 → 0.6

【覆盖度】
coverage = (不同话题数) / (全部话题数)
示例：系统有 50 个话题，展示了 12 个 → 0.24

【新颖性】
novelty_score = 1 - (用户最近 K 个展示中该话题的出现频率)
示例：用户最近看过 3 个电商广告，再展示电商
      novelty = 1 - 3/10 = 0.7（相对新颖）

【豆瓣相似度】(Diverse Ranking, DR)
diversiy = Σ (1 - sim(ad_i, ad_j)) for all pairs
sim = cosine similarity
示例：任意两个广告尽可能不同
```

### D2. 多样性与相关性的权衡

**经典权衡**：
```
目标函数 = w1 × relevance + w2 × diversity
relevance = initial_score（精排模型的分数）
diversity = 多样性指标

w1 + w2 = 1，通常 w1=0.7, w2=0.3
```

**问题**：
- w1 和 w2 的权重如何设定？
- 不同时刻、不同用户可能需要不同权重

**解决方案**：
- **A/B 测试**逐步调优
- **学习权重**（用元学习自动调适）
- **约束化**：relevance ≥ 阈值，多样性最大化

### D3. 帕累托最优重排

**思想**：不寻找"最优"权重，而是找**帕累托前沿**上的若干解

```
【帕累托最优的定义】
如果无法在不降低某个目标的情况下提升另一个目标，
则称为帕累托最优

【在重排中的应用】
寻找多个重排方案，形成帕累托前沿：

     高多样性
        ^
        |  * (多样性=0.8, 分数=0.7)
        |    \
        |     * (多样性=0.7, 分数=0.75)
        |      \
        |       * (多样性=0.5, 分数=0.80)
        |        \
        +--*-------> 高分数
        
选择哪个？根据**当前目标权重**动态选择
```

**实现**：
- MGDA（Multiple Gradient Descent）
- Pareto-based MultiObjective Evolutionary Algorithm
- 线性规划：最大化 score s.t. diversity ≥ threshold
