# GNOLR: Generalized Nested Ordered Logistic Regression for Multi-Task Learning
> 来源：https://arxiv.org/abs/2501.12948 | 领域：ads | 日期：20260323

## 问题定义
广告系统中用户行为存在自然序关系（曝光→点击→加购→购买），传统多任务学习（MTL）忽略这种有序结构。GNOLR提出广义嵌套有序逻辑回归，利用行为的序关系提升多任务建模效果。

## 核心方法与创新点
- 有序回归框架：将CTR/CVR等多任务建模为有序逻辑回归
- 嵌套约束：P(购买)≤P(加购)≤P(点击)，用嵌套结构强制约束
- 广义化：支持任意有序任务层级，不限于固定序列
- 联合损失：有序约束损失 + 任务特定损失的联合优化

## 实验结论
在电商广告数据集，GNOLR相比MMoE等MTL基线，CVR AUC提升约0.4%，CTCVR（转化率）提升约0.3%；有序约束确保预测概率的单调性，减少约85%的概率违序样本。

## 工程落地要点
- 有序约束可以作为正则项加入损失，无需修改模型架构
- 嵌套概率需要在推理时后处理，确保输出概率满足约束
- 不同电商场景的行为序列不同（如直播购物：观看→互动→下单），需要定制

## 常见考点
1. **Q: 广告多任务学习（MTL）的常见方案？** A: MMoE（多门混合专家）、PLE（渐进分层提取）、ESMM（完整空间多任务）
2. **Q: ESMM如何解决CVR的样本选择偏差？** A: 在全曝光空间建模CTCVR=CTR×CVR，间接得到CVR的全空间估计
3. **Q: 为什么CVR预估比CTR困难？** A: 正样本少（转化稀疏）、样本选择偏差（只有点击才有CVR标签）
4. **Q: 有序逻辑回归和普通逻辑回归的区别？** A: 有序LR建模累积概率P(Y≤k)，利用有序结构减少参数，提升泛化
5. **Q: 多任务学习中的负迁移（negative transfer）如何处理？** A: 任务相关性分析、专家网络隔离（MMoE）、梯度裁剪和任务权重平衡
