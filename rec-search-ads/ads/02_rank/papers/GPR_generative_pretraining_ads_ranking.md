# GPR: Generative Pre-Training for Ads Ranking
> 来源：工业论文 | 领域：广告系统 | 学习日期：20260327

## 问题定义
广告排序系统包含多种子任务：CTR预测、CVR预测、相关性评估、新广告冷启动打分等。
传统方案：每个子任务独立训练，导致：
1. **知识孤岛**：不同任务学到的广告表示不能互相借鉴
2. **冷启动弱**：新广告没有历史数据，独立模型无法打分
3. **维护成本高**：N个子任务 = N套模型 + N套serving

**目标**：用生成式预训练范式统一多个广告排序子任务，并解决新广告冷启动问题。

## 核心方法与创新点

### 1. 广告排序任务的统一序列格式
将所有子任务转化为序列生成：
```
[广告内容特征] [用户特征] [上下文特征] → [任务标识] → [预测输出]
```

任务标识（Task Token）：`[CTR]`、`[CVR]`、`[RELEVANCE]`、`[COLDSTART]`

### 2. 多任务预训练目标

$$
\mathcal{L}_{pretrain} = \mathcal{L}_{MLM} + \mathcal{L}_{NSP} + \mathcal{L}_{task\_specific}
$$

- MLM：掩码广告属性预测（学习广告语义表示）
- NSP：下一个广告预测（学习广告序列模式）
- 任务特定：每个任务的监督损失

### 3. 提示微调（Prompt Tuning）
不同子任务通过不同的prompt适配，主干模型固定：

$$
h_{task} = \text{LLM}(\text{prompt}}_{\text{{task}} + x_{ad} + x_{user})
$$

减少子任务微调的参数量，支持快速新增任务。

### 4. 冷启动机制
新广告没有历史交互，利用广告内容特征（文本、图片）作为输入：
- 预训练时大量学习内容-行为对（内容特征 → 历史CTR）
- 推理时新广告只有内容特征，通过预训练的内容理解能力估计CTR

## 实验结论
- **整体AUC**：+1.2%（相比独立子任务模型）
- **新广告冷启动效果**：+30%（Recall@20，冷启动阶段）
- **模型维护成本**：减少60%（由N个模型压缩为1个）
- Prompt Tuning微调效率：新增任务只需要微调<1%参数

## 工程落地要点
1. **统一Serving**：单一模型serving多任务，需要良好的任务路由机制
2. **冷启动过渡**：随着新广告积累历史数据，逐渐增加行为特征比例，减少纯内容依赖
3. **Prompt设计**：不同子任务的prompt需要精心设计，避免任务歧义
4. **更新频率**：广告内容变化快，预训练模型需要定期（周/月）更新
5. **容量规划**：统一大模型比多个小模型的计算资源需求更高，需要模型蒸馏

## 常见考点
Q1: 广告排序的预训练与NLP预训练（BERT/GPT）有何异同？
A: 相同：都用自监督信号（MLM等）在大规模数据上预训练，再fine-tune到下游任务。不同：(1)输入格式不同：广告系统输入是结构化特征（类目ID、价格等）+文本，而非纯文本；(2)预训练信号：广告系统可以用用户行为（点击、转化）作为额外监督信号；(3)任务多样性：广告系统下游任务更多样（竞价、创意、排序等）；(4)实时性要求：广告模型需要更频繁更新。

Q2: Prompt Tuning与Fine-tuning对广告排序各有什么优缺点？
A: Fine-tuning：更新主干模型所有参数，任务适应性更强，但每个任务存储一个完整模型副本（存储成本高）。Prompt Tuning：只优化少量Prompt参数（<1%），多任务共享一个主干模型，存储效率高；但需要模型足够大（>10B参数）才能有效响应prompt，小模型上效果差。广告场景建议：主干用Prompt Tuning，关键任务（CTR）用LoRA补充微调。

Q3: 如何衡量冷启动广告排序的效果？
A: (1)新广告的首日Recall@K：新广告上线第1天内，系统能否为合适的用户召回并展示；(2)Bootstrap时间：新广告从上线到获得稳定的个性化排名需要多少次曝光/点击；(3)对比老广告：同类目新广告vs老广告的CTR比值（好的冷启动方案应该让两者差距<20%）；(4)广告主流失率：因为冷启动效果差而放弃投放的广告主比例。

## 模型架构详解

### 输入层
- **用户特征**：用户画像（年龄/性别/地域）、行为序列（点击/购买历史）、实时上下文（时间/设备/位置）
- **广告特征**：广告 ID Embedding、广告主类目、创意特征（标题/图片 Embedding）
- **上下文特征**：页面位置、请求时段、竞价上下文

### 特征交叉层
- 低阶交叉：FM/FFM 风格的二阶特征交叉
- 高阶交叉：Deep 网络或 Cross Network 捕获高阶非线性

### 预测层
- 多目标输出头：CTR Tower + CVR Tower（或联合 CTCVR）
- 校准层：Platt Scaling 或 Isotonic Regression 确保预测概率准确

### 训练策略
- 样本构建：曝光日志 + 点击/转化事件 Join
- 负采样：全量负样本或按比例降采样 + 权重修正
- 优化器：Adam / AdaGrad with learning rate warmup

## 与相关工作对比

| 维度 | 本文方法 | 传统方法 | 优势 |
|------|---------|---------|------|
| 特征交叉 | 自动化/高阶 | 手工/低阶 | 减少特征工程，捕获复杂模式 |
| 多任务学习 | 联合优化 | 独立模型 | 共享知识，缓解数据稀疏 |
| 在线适应 | 增量更新 | 全量重训 | 更快响应分布漂移 |
| 样本效率 | 全空间/去偏 | 有偏子集 | 更准确的概率估计 |

## 面试深度追问

- **Q: 这个方法如何处理特征稀疏问题？**
  A: 通过预训练 Embedding + 特征交叉网络，将稀疏 ID 特征映射到稠密空间。对于冷启动特征，可借助 side information（类目、文本描述）进行特征迁移。

- **Q: 在线服务的延迟优化方案？**
  A: 1) 模型蒸馏/量化减少参数量；2) 特征缓存避免重复计算；3) 异步特征获取 + 超时降级；4) GPU/TensorRT 推理加速。

- **Q: 如何评估模型的线上效果？**
  A: 离线指标（AUC/GAUC/LogLoss）+ 在线 A/B Test（CTR/CVR/RPM/GMV）。注意 GAUC 按用户分组计算 AUC 的均值，更贴近线上排序效果。

- **Q: 训练样本的时效性如何保证？**
  A: 1) 实时特征流（Flink）+ 小时级增量训练；2) 样本时间窗口控制（通常7~14天）；3) 特征 serving 与训练使用相同 pipeline 避免 train-serve skew。
