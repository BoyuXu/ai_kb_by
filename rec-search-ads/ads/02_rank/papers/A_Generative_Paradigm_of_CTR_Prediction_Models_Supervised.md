# A Generative Paradigm of CTR Prediction Models (Supervised Feature Generation)

> 来源：[https://arxiv.org/abs/2406.xxxxx] | 日期：20260313 | 领域：ads

## 问题定义
传统CTR预测模型（DCN、DeepFM、FiBiNet等）依赖"特征工程+判别式建模"范式：人工设计交叉特征，模型学习这些特征对点击的判别边界。这种范式存在三个局限：(1) 特征组合爆炸，工程成本高；(2) 稀疏特征泛化能力弱（长尾用户/物品特征不充分）；(3) 无法利用LLM的预训练语义知识。本文提出"生成式CTR"范式：用生成模型（LLM/Diffusion）先"生成"有监督的中间特征表示，再将生成的特征送入判别式模型，实现生成-判别协同。

## 核心方法与创新点
- **监督特征生成（Supervised Feature Generation, SFG）**：以点击标签为监督信号，训练条件生成模型从原始特征中生成高信息量的隐变量，该隐变量同时满足：(1) 能重建原始输入（重建损失）；(2) 能预测点击标签（判别损失）。
- **变分生成框架**：使用VAE（Variational Autoencoder）或Conditional Flow构建特征生成器，隐空间表示既有生成多样性（覆盖用户不确定意图）又有判别性（与CTR强相关）。
- **LLM语义增强**：用LLM对用户行为序列和物品属性生成文本摘要，将文本摘要的Embedding融入生成特征，注入LLM的先验语义知识。
- **多视图特征生成**：同一用户-物品对，生成多个视角的特征表示（如"价格敏感视角"、"风格偏好视角"），通过注意力机制加权融合，提升特征多样性。

## 实验结论
- 在Criteo、Avazu公开数据集上，SFG范式相比DCN v2、AutoInt等SOTA基线AUC提升0.2%~0.4%，LogLoss降低0.3%~0.6%。
- 在某电商广告内部数据集上，线上A/B测试CTR提升1.1%，ECPM提升0.9%。
- 消融实验表明，LLM语义增强贡献约0.15% AUC提升，多视图生成贡献约0.1% AUC提升，监督生成框架本身贡献最大（0.15% AUC）。
- 稀疏特征场景下（长尾商品类目）效果提升更显著，AUC提升0.6%，说明生成式范式对稀疏性问题的改善效果明显。

## 工程落地要点
- **推理成本**：生成式CTR在推理时需要额外的生成前向传播（VAE解码器），相比纯判别式模型增加约20-30%的计算量，需评估是否在预排序或精排阶段引入。
- **训练稳定性**：VAE训练的KL权重（β参数）需仔细调优，过大导致隐空间坍缩（无法生成多样特征），过小导致隐空间不连续（泛化性差）。
- **LLM特征离线化**：LLM生成的文本Embedding必须离线预计算并缓存，不能实时调用LLM，否则延迟无法接受。
- **特征版本控制**：生成特征（隐变量）会随模型版本更新而变化，需要严格的特征版本管理，避免训练-推理不一致。

## 常见考点
**Q1: 生成式范式相比判别式范式的本质优势是什么？**
A: 判别式模型直接学习决策边界，容易过拟合到训练数据分布；生成式模型学习数据的联合分布，对未见样本（长尾用户/物品）的泛化能力更强。同时，生成模型可以自然地建模用户意图的不确定性（通过隐变量的分布来表达），这是判别式模型无法做到的。

**Q2: 在CTR预测中，监督特征生成如何避免标签泄露？**
A: 关键是训练时的隔离：生成器学习从"输入特征→有用的隐表示"，判别器使用隐表示预测标签。隐表示应该是特征的语义压缩，而非直接编码标签信息。技术上通过重建损失（确保隐变量包含输入信息）和对抗正则化（防止隐变量直接记住标签）来平衡。

**Q3: DeepFM和DCN的特征交叉机制有何区别？**
A: DeepFM使用FM层（二阶特征交叉，O(n²)但参数共享高效）+ Deep层（隐式高阶交叉）；DCN v2使用显式的Cross Network（每层与初始输入进行交叉，理论上可学任意阶交叉）+ Deep层。DCN v2的Cross Network更有表达能力，但参数量更大；DeepFM更轻量，在特征数量超大时（数千维）FM的参数效率优势更明显。

## 模型架构详解

### 输入层
- **用户特征**：用户画像（年龄/性别/地域）、行为序列（点击/购买历史）、实时上下文（时间/设备/位置）
- **广告特征**：广告 ID Embedding、广告主类目、创意特征（标题/图片 Embedding）
- **上下文特征**：页面位置、请求时段、竞价上下文

### 特征交叉层
- 低阶交叉：FM/FFM 风格的二阶特征交叉
- 高阶交叉：Deep 网络或 Cross Network 捕获高阶非线性

### 预测层
- 多目标输出头：CTR Tower + CVR Tower（或联合 CTCVR）
- 校准层：Platt Scaling 或 Isotonic Regression 确保预测概率准确

### 训练策略
- 样本构建：曝光日志 + 点击/转化事件 Join
- 负采样：全量负样本或按比例降采样 + 权重修正
- 优化器：Adam / AdaGrad with learning rate warmup

## 面试深度追问

- **Q: 这个方法如何处理特征稀疏问题？**
  A: 通过预训练 Embedding + 特征交叉网络，将稀疏 ID 特征映射到稠密空间。对于冷启动特征，可借助 side information（类目、文本描述）进行特征迁移。

- **Q: 在线服务的延迟优化方案？**
  A: 1) 模型蒸馏/量化减少参数量；2) 特征缓存避免重复计算；3) 异步特征获取 + 超时降级；4) GPU/TensorRT 推理加速。

- **Q: 如何评估模型的线上效果？**
  A: 离线指标（AUC/GAUC/LogLoss）+ 在线 A/B Test（CTR/CVR/RPM/GMV）。注意 GAUC 按用户分组计算 AUC 的均值，更贴近线上排序效果。

- **Q: 训练样本的时效性如何保证？**
  A: 1) 实时特征流（Flink）+ 小时级增量训练；2) 样本时间窗口控制（通常7~14天）；3) 特征 serving 与训练使用相同 pipeline 避免 train-serve skew。

本方法的核心思想已被多家头部公司借鉴：
- **阿里巴巴**：在淘宝推荐/直通车广告中应用类似的特征交叉和多任务学习框架

### 技术演进路线
1. **V1（基础版）**：经典特征工程 + LR/GBDT
2. **V2（深度化）**：DNN 替代手工特征，Wide&Deep 范式
3. **V3（自动化）**：AutoML 搜索网络结构 + 特征交叉
4. **V4（预训练）**：大模型预训练 + 下游微调，本文属于此阶段

### 关键指标参考
- 离线 AUC 提升 0.1%~0.5% 通常对应线上 CTR 提升 1%~3%
- 模型推理延迟控制在 10~30ms（P99）
- 特征维度：通常 100~500 维稠密特征 + 百万级稀疏 ID 特征
