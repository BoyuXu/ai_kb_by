# Counterfactual Learning for Unbiased Ad Ranking in Industrial Search Systems

> 来源：arxiv | 日期：20260321 | 领域：ads

## 问题定义

在工业搜索广告系统中，模型训练数据来自用户点击日志，存在严重的**选择偏差（Selection Bias）**和**位置偏差（Position Bias）**：

1. **位置偏差**：排名靠前的广告被点击概率高，模型错误学习"高位 = 高质量"，形成马太效应
2. **曝光偏差**：只有展示过的广告有点击标注，未展示的广告无反馈（Missing Not At Random）
3. **自我强化循环**：偏差模型的推荐结果成为下一轮训练数据，偏差不断放大

如果不纠偏，模型会越来越偏向已有高曝光的广告，新广告永远没有公平竞争的机会，且用户体验因"排名≠质量"而下降。

## 核心方法与创新点

1. **IPS（Inverse Propensity Score）加权**：
   - 估计每个广告在当前位置被点击的概率（倾向分 Propensity）
   - 训练时对样本加权：`w = 1 / P(click | position, ad_features)`
   - 纠偏后的估计量：`E[CTR_true] = E[click × w] = E[click / propensity]`

2. **倾向分估计**：
   - 用随机流量（A/B 的随机分配桶）学习无偏的点击率作为参考
   - 位置倾向模型：`P(click|pos) = f(pos)` 通过 EM 算法估计
   - 避免"直接用训练数据估计倾向分 → 循环依赖"

3. **自反事实（Counterfactual）排序学习**：
   - 核心问题：同一广告在不同位置的真实 CTR 是什么（反事实）？
   - 使用 DR（Doubly Robust）估计量，结合直接模型估计和 IPS 纠偏，降低方差：
   ```
   DR = Σᵢ [ŷᵢ + (yᵢ - ŷᵢ) / eᵢ]
   ```
   其中 ŷ 是直接估计，y 是观测点击，e 是倾向分

4. **工业级落地**：
   - 大规模系统中准确估计每个（广告, 位置）的倾向分代价高，用位置分桶（Top3/Mid/Bottom）简化
   - 倾向分裁剪（Clipping）：防止极端小倾向分导致权重爆炸

## 实验结论

- 在内部搜索广告平台：纠偏后广告排序 NDCG **+5.3%**，新广告（冷启动）点击率 **+21%**（排名公平性提升）
- 在随机对照实验中验证：IPS 加权的 CTR 估计比原始 CTR 更接近真实无偏 CTR（MSE 降低 **43%**）
- 整体广告收入：**+3.1%**（更公平的竞争使高质量新广告获得更多曝光，总盘子增大）

## 工程落地要点

1. **随机流量桶维护**：纠偏模型依赖无偏的随机流量数据，需要保留 1-5% 的随机展示流量（不损失收入前提下尽量小）
2. **倾向分的实时更新**：市场竞争和用户行为变化会改变位置倾向，倾向分模型需要定期更新（至少每周）
3. **权重裁剪**：IPS 权重可能极端大（倾向分接近 0），需要设置上界（如 max_weight=10），平衡偏差-方差权衡
4. **纠偏 vs 不纠偏的联合监控**：纠偏可能降低短期点击率（模型不再倾向高位广告），但提升长期质量，需要双重 KPI 体系

## 常见考点

- Q: 什么是位置偏差？如何在推荐/广告系统中纠偏？
  A: 位置偏差指靠前位置的物品被点击概率高于靠后位置，即使质量相同。纠偏方法：(1) IPS 加权：除以展示倾向分；(2) DLA（Deep Learning Approach）：用辅助网络学习位置偏差；(3) 随机化：随机打乱部分结果收集无偏数据；(4) Regression EM：用 EM 算法估计倾向分。

- Q: IPS 估计量的缺陷是什么？DR 如何改进？
  A: IPS 的问题：(1) 高方差：倾向分小时权重大，样本方差剧增；(2) 倾向分估计误差会导致 IPS 有偏。DR（Doubly Robust）改进：结合直接模型估计（低方差但有偏）和 IPS（无偏但高方差），只要两者之一正确，DR 估计就是无偏的，且方差通常更低。

- Q: 反事实学习在工业推荐/广告中的主要应用场景？
  A: (1) 纠偏训练（本文）；(2) 离线评估：评估没有展示过的策略效果（Off-Policy Evaluation）；(3) 因果推断：估计广告对购买的真实因果效应（而非相关性）；(4) 反事实数据增强：模拟未观测到的用户行为。

## 模型架构详解

### 输入层
- **用户特征**：用户画像（年龄/性别/地域）、行为序列（点击/购买历史）、实时上下文（时间/设备/位置）
- **广告特征**：广告 ID Embedding、广告主类目、创意特征（标题/图片 Embedding）
- **上下文特征**：页面位置、请求时段、竞价上下文

### 特征交叉层
- 低阶交叉：FM/FFM 风格的二阶特征交叉
- 高阶交叉：Deep 网络或 Cross Network 捕获高阶非线性

### 预测层
- 多目标输出头：CTR Tower + CVR Tower（或联合 CTCVR）
- 校准层：Platt Scaling 或 Isotonic Regression 确保预测概率准确

### 训练策略
- 样本构建：曝光日志 + 点击/转化事件 Join
- 负采样：全量负样本或按比例降采样 + 权重修正
- 优化器：Adam / AdaGrad with learning rate warmup


## 与相关工作对比

| 维度 | 本文方法 | 传统方法 | 优势 |
|------|---------|---------|------|
| 特征交叉 | 自动化/高阶 | 手工/低阶 | 减少特征工程，捕获复杂模式 |
| 多任务学习 | 联合优化 | 独立模型 | 共享知识，缓解数据稀疏 |
| 在线适应 | 增量更新 | 全量重训 | 更快响应分布漂移 |
| 样本效率 | 全空间/去偏 | 有偏子集 | 更准确的概率估计 |


## 面试深度追问

- **Q: 这个方法如何处理特征稀疏问题？**
  A: 通过预训练 Embedding + 特征交叉网络，将稀疏 ID 特征映射到稠密空间。对于冷启动特征，可借助 side information（类目、文本描述）进行特征迁移。

- **Q: 在线服务的延迟优化方案？**
  A: 1) 模型蒸馏/量化减少参数量；2) 特征缓存避免重复计算；3) 异步特征获取 + 超时降级；4) GPU/TensorRT 推理加速。

- **Q: 如何评估模型的线上效果？**
  A: 离线指标（AUC/GAUC/LogLoss）+ 在线 A/B Test（CTR/CVR/RPM/GMV）。注意 GAUC 按用户分组计算 AUC 的均值，更贴近线上排序效果。

- **Q: 训练样本的时效性如何保证？**
  A: 1) 实时特征流（Flink）+ 小时级增量训练；2) 样本时间窗口控制（通常7~14天）；3) 特征 serving 与训练使用相同 pipeline 避免 train-serve skew。
