# EGA-V2：工业广告的端到端生成框架

> 来源：EGA-V2: An End-to-end Generative Framework for Industrial Advertising | 日期：20260318 | 领域：ads

## 问题定义

传统工业广告系统是多阶段流水线（召回→粗排→精排→出价→创意生成），各阶段独立优化，存在目标不一致问题：精排模型优化 CTR，但最终目标是广告主 ROI 和平台 Revenue 的联合最大化。EGA-V2 探索**端到端生成式框架**：用统一的生成模型直接从用户请求生成最优广告投放方案（包括选广告主、选创意、定出价）。

## 核心方法与创新点

1. **统一序列生成**：将广告投放决策建模为 token 序列生成任务：
   - 输入：用户 context（query、历史、画像）+ 广告主候选列表
   - 输出：有序的（广告主, 创意, 出价因子）三元组序列
   - 用自回归 Transformer 生成，每步生成一个广告投放单元

2. **多目标对齐**：生成模型的训练目标同时包含：
   - 平台 Revenue 最大化（RLHF 风格的奖励模型）
   - 广告主 ROI 约束满足（约束奖励惩罚项）
   - 用户体验保护（广告密度、相关性）

3. **候选压缩**：用层级量化（VQ-VAE）将广告候选 embedding 离散化，减少生成时的词表大小，提升解码速度。

4. **拍卖机制内化**：将 GSP 拍卖的竞争逻辑编码进训练数据格式，模型隐式学习竞价博弈。

## 实验结论

- 在某大型广告平台的离线实验，相比多阶段基线 Revenue +4.2%，广告主 ROI 达标率 +6.1%。
- 用户体验指标（CTR、停留时长）无显著下降（< 0.5%）。
- 端到端推理延迟比多阶段串行 pipeline 低约 20ms（消除了阶段间数据传输开销）。

## 工程落地要点

- **候选规模限制**：自回归生成对候选规模敏感，实际需配合前置召回将候选压缩到 100-500 量级。
- **在线服务架构**：生成模型需要 GPU 推理，对延迟要求严格的场景需要流式解码 + early stopping。
- **安全回退**：端到端模型在 OOD 情况下可能生成异常方案，需要规则层兜底（出价上限、广告类目过滤）。
- **模型更新频率**：广告市场动态变化（竞争格局、季节性），生成模型需要每日或每周增量更新。

## 常见考点

**Q: 广告系统为什么难以做端到端优化？**
- 各阶段规模差异巨大（召回百万→精排百条），统一模型难以兼顾；多目标（平台/广告主/用户）难以联合优化；延迟约束严格。

**Q: 生成式广告和判别式广告的本质区别？**
- 判别式：给定候选打分排序（预测 CTR/CVR）；生成式：直接生成投放方案，不受候选集限制，可探索新的广告组合。

**Q: RLHF 如何应用到广告优化？**
- 用历史广告效果数据训练奖励模型（Revenue、ROI、用户体验综合打分），用 PPO 等算法微调生成模型，使其输出的广告方案奖励最大化。

## 模型架构详解

### 输入层
- **用户特征**：用户画像（年龄/性别/地域）、行为序列（点击/购买历史）、实时上下文（时间/设备/位置）
- **广告特征**：广告 ID Embedding、广告主类目、创意特征（标题/图片 Embedding）
- **上下文特征**：页面位置、请求时段、竞价上下文

### 特征交叉层
- 低阶交叉：FM/FFM 风格的二阶特征交叉
- 高阶交叉：Deep 网络或 Cross Network 捕获高阶非线性

### 预测层
- 多目标输出头：CTR Tower + CVR Tower（或联合 CTCVR）
- 校准层：Platt Scaling 或 Isotonic Regression 确保预测概率准确

### 训练策略
- 样本构建：曝光日志 + 点击/转化事件 Join
- 负采样：全量负样本或按比例降采样 + 权重修正
- 优化器：Adam / AdaGrad with learning rate warmup


## 与相关工作对比

| 维度 | 本文方法 | 传统方法 | 优势 |
|------|---------|---------|------|
| 特征交叉 | 自动化/高阶 | 手工/低阶 | 减少特征工程，捕获复杂模式 |
| 多任务学习 | 联合优化 | 独立模型 | 共享知识，缓解数据稀疏 |
| 在线适应 | 增量更新 | 全量重训 | 更快响应分布漂移 |
| 样本效率 | 全空间/去偏 | 有偏子集 | 更准确的概率估计 |


## 面试深度追问

- **Q: 这个方法如何处理特征稀疏问题？**
  A: 通过预训练 Embedding + 特征交叉网络，将稀疏 ID 特征映射到稠密空间。对于冷启动特征，可借助 side information（类目、文本描述）进行特征迁移。

- **Q: 在线服务的延迟优化方案？**
  A: 1) 模型蒸馏/量化减少参数量；2) 特征缓存避免重复计算；3) 异步特征获取 + 超时降级；4) GPU/TensorRT 推理加速。

- **Q: 如何评估模型的线上效果？**
  A: 离线指标（AUC/GAUC/LogLoss）+ 在线 A/B Test（CTR/CVR/RPM/GMV）。注意 GAUC 按用户分组计算 AUC 的均值，更贴近线上排序效果。

- **Q: 训练样本的时效性如何保证？**
  A: 1) 实时特征流（Flink）+ 小时级增量训练；2) 样本时间窗口控制（通常7~14天）；3) 特征 serving 与训练使用相同 pipeline 避免 train-serve skew。


## 工业界应用案例

本方法的核心思想已被多家头部公司借鉴：
- **阿里巴巴**：在淘宝推荐/直通车广告中应用类似的特征交叉和多任务学习框架
- **字节跳动**：在抖音广告系统中结合了类似的建模思路，配合大规模分布式训练
- **美团**：在搜索广告场景中验证了该方法对长尾商品效果的改善

### 技术演进路线
1. **V1（基础版）**：经典特征工程 + LR/GBDT
2. **V2（深度化）**：DNN 替代手工特征，Wide&Deep 范式
3. **V3（自动化）**：AutoML 搜索网络结构 + 特征交叉
4. **V4（预训练）**：大模型预训练 + 下游微调，本文属于此阶段

### 关键指标参考
- 离线 AUC 提升 0.1%~0.5% 通常对应线上 CTR 提升 1%~3%
- 模型推理延迟控制在 10~30ms（P99）
- 特征维度：通常 100~500 维稠密特征 + 百万级稀疏 ID 特征
