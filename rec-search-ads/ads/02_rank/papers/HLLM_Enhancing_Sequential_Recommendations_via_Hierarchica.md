# HLLM: Enhancing Sequential Recommendations via Hierarchical Large Language Models
> 来源：https://arxiv.org/abs/2409.12740 | 领域：ads | 日期：20260323

## 问题定义
如何利用大语言模型（LLM）增强序列推荐系统。HLLM提出层级LLM框架：小LLM理解物品语义，大LLM建模用户长期偏好，两级层级协同实现高效序列推荐。

## 核心方法与创新点
- 层级架构：物品级LLM（小，处理物品文本）+ 用户级LLM（大，建模用户历史）
- 两级蒸馏：用户级LLM的知识蒸馏到物品级LLM，实现知识传递
- 长序列处理：用户级LLM专门优化处理长达1000+步的行为序列
- 效率设计：物品级LLM结果缓存，减少重复计算

## 实验结论
在Amazon商品推荐benchmark，HLLM相比SASRec等序列推荐基线提升约10% NDCG；相比单一LLM推荐，在相同计算预算下效果提升约5%；层级设计使推理速度提升约3x。

## 工程落地要点
- 物品级LLM的结果可以预计算并缓存（每物品一次），大幅降低在线计算
- 用户序列长度超过512时需要截断策略（最近的行为 vs 最重要的行为）
- 需要构建物品文本描述数据库，支持物品级LLM的输入

## 常见考点
1. **Q: 为什么用层级LLM而不是单一LLM？** A: 物品语义和用户偏好是不同粒度的问题，层级分工效率更高
2. **Q: 用户行为序列的长度处理策略？** A: 截断（取最近N个）、压缩（用稀疏注意力/聚类）、层级压缩（HLLM的做法）
3. **Q: LLM用于推荐时如何处理物品ID与文本的对应？** A: 维护ID-to-text映射表，查找物品描述作为LLM输入
4. **Q: 序列推荐模型（SASRec/BERT4Rec/GRU4Rec）的差异？** A: SASRec：单向注意力；BERT4Rec：双向注意力+MLM；GRU4Rec：循环网络
5. **Q: LLM推荐系统中的数据隐私挑战？** A: 用户历史行为不能直接传给外部API；需要本地部署或特征级别的隐私保护
