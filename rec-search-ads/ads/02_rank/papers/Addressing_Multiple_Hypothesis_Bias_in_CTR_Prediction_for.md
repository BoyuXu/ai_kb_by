# Addressing Multiple Hypothesis Bias in CTR Prediction for Ad Selection
> 来源：工业界论文（Meta/Google内部研究）| 日期：20260319

## 问题定义
广告CTR预测中存在"多重假设偏差"（Multiple Hypothesis Bias）：同一用户-广告对可能经过多次模型迭代、多个实验桶、多种特征版本被预测，这些多次预测的最高值会被选出进行展示，形成乐观偏差（Optimistic Bias）。类似统计学中的多重比较问题，导致实际CTR系统性高估，进而影响出价和收益优化。

## 核心方法与创新点
1. **偏差来源建模**：将多重假设问题形式化为order statistics问题，k次独立预测取最大值的期望比真实CTR高E[max(X₁,...,Xₖ)] - E[X]
2. **校正因子估计**：通过历史实验数据回归拟合偏差校正系数，对不同竞价位置、候选数量学习不同校正量
3. **贝叶斯校正框架**：用贝叶斯方法建模CTR不确定性，将多重假设偏差融入后验更新，输出校正后的CTR分布
4. **在线去偏训练**：在模型训练时引入反事实样本，用未展示广告的负反馈信号校正训练分布
5. **双模型架构**：一个模型预测展示概率（propensity），另一个预测无偏CTR，组合输出去偏预测

## 实验结论
- 校正后线上CTR预测偏差从+12%降至+2%以内
- 竞价效率提升，广告主ROI改善约3%-5%
- 在高候选数（>1000条广告）场景下去偏效果最显著

## 工程落地要点
- 在召回→精排漏斗中，候选数量越多偏差越大，需对不同层分别校正
- 校正系数需随候选集分布变化动态更新（A/B实验版本不同时候选分布不同）
- 注意区分多重假设偏差与位置偏差（Position Bias），两者叠加需分开建模

## 常见考点
**Q: CTR预测中常见的偏差有哪些？**
A: （1）位置偏差（靠前展示CTR虚高）；（2）选择偏差（只有展示样本才有标签）；（3）曝光偏差（热门物品曝光多）；（4）多重假设偏差（本文）；（5）确认偏差（模型预测影响展示，形成反馈循环）。

**Q: 如何用IPS（Inverse Propensity Score）解决选择偏差？**
A: 对训练样本按展示概率（propensity）的倒数加权，使训练分布接近随机展示分布，公式：Loss = Σ (1/p_i) * L(y_i, ŷ_i)。

**Q: 多重比较问题在A/B实验中如何控制？**
A: Bonferroni校正（α/n）、FDR控制（Benjamini-Hochberg方法）、预注册实验假设、减少同时运行的实验数量。

## 模型架构详解

### 输入层
- **用户特征**：用户画像（年龄/性别/地域）、行为序列（点击/购买历史）、实时上下文（时间/设备/位置）
- **广告特征**：广告 ID Embedding、广告主类目、创意特征（标题/图片 Embedding）
- **上下文特征**：页面位置、请求时段、竞价上下文

### 特征交叉层
- 低阶交叉：FM/FFM 风格的二阶特征交叉
- 高阶交叉：Deep 网络或 Cross Network 捕获高阶非线性

### 预测层
- 多目标输出头：CTR Tower + CVR Tower（或联合 CTCVR）
- 校准层：Platt Scaling 或 Isotonic Regression 确保预测概率准确

### 训练策略
- 样本构建：曝光日志 + 点击/转化事件 Join
- 负采样：全量负样本或按比例降采样 + 权重修正
- 优化器：Adam / AdaGrad with learning rate warmup

## 面试深度追问

- **Q: 这个方法如何处理特征稀疏问题？**
  A: 通过预训练 Embedding + 特征交叉网络，将稀疏 ID 特征映射到稠密空间。对于冷启动特征，可借助 side information（类目、文本描述）进行特征迁移。

- **Q: 在线服务的延迟优化方案？**
  A: 1) 模型蒸馏/量化减少参数量；2) 特征缓存避免重复计算；3) 异步特征获取 + 超时降级；4) GPU/TensorRT 推理加速。

- **Q: 如何评估模型的线上效果？**
  A: 离线指标（AUC/GAUC/LogLoss）+ 在线 A/B Test（CTR/CVR/RPM/GMV）。注意 GAUC 按用户分组计算 AUC 的均值，更贴近线上排序效果。

- **Q: 训练样本的时效性如何保证？**
  A: 1) 实时特征流（Flink）+ 小时级增量训练；2) 样本时间窗口控制（通常7~14天）；3) 特征 serving 与训练使用相同 pipeline 避免 train-serve skew。


## 工业界应用案例

本方法的核心思想已被多家头部公司借鉴：
- **阿里巴巴**：在淘宝推荐/直通车广告中应用类似的特征交叉和多任务学习框架
- **字节跳动**：在抖音广告系统中结合了类似的建模思路，配合大规模分布式训练
- **美团**：在搜索广告场景中验证了该方法对长尾商品效果的改善

### 技术演进路线
1. **V1（基础版）**：经典特征工程 + LR/GBDT
2. **V2（深度化）**：DNN 替代手工特征，Wide&Deep 范式
3. **V3（自动化）**：AutoML 搜索网络结构 + 特征交叉
4. **V4（预训练）**：大模型预训练 + 下游微调，本文属于此阶段

### 关键指标参考
- 离线 AUC 提升 0.1%~0.5% 通常对应线上 CTR 提升 1%~3%
- 模型推理延迟控制在 10~30ms（P99）
- 特征维度：通常 100~500 维稠密特征 + 百万级稀疏 ID 特征
