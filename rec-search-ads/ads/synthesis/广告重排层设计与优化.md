## 📐 核心公式与原理

### 1. 重排的优化目标

**Single Objective（简单版）**
$$\text{maximize} \sum_{i=1}^{K} \text{Score}(\text{Ad}_i | \text{context})$$

**Multi-Objective（实际工业版）**
$$\text{maximize} \quad \alpha \cdot \text{Revenue} + \beta \cdot \text{UserExp} + \gamma \cdot \text{Diversity}$$
$$\text{subject to} \quad \text{Constraints}(\text{frequency, frequency, diversity...})$$

### 2. 多样性度量：Shannon Entropy

$$H(\text{Diversity}) = -\sum_{c=1}^{C} p_c \log p_c$$

其中 $p_c$ 为话题 $c$ 在当前 Feed 中的占比。范围：$[0, \log C]$。

### 3. 位置衰减函数（Position Bias）

$$\text{Value}(i) = \text{CTR}(i) \times \text{bid}(i) \times \text{position\_discount}(i)$$

其中位置衰减因子通常为：$\text{discount}(i) = 1/\log(i+1)$ 或 $e^{-\lambda i}$

### 4. 约束满足度量

**频率约束**：同一广告主最多 N 个
$$\sum_{i: \text{advertiser}(i) = a} x_i \leq N \quad \forall a$$

**相似度约束**：避免连续相似广告
$$\text{similarity}(i, i+1) < \tau \quad \forall i$$

## 🎯 核心洞察

### 1. **精排 vs 重排的本质区别**
- **精排（Fine Ranking）**：绝对评分——这个广告有多好？（0-1 分）
- **重排（Reranking）**：相对评分——在给定上下文和约束下，该展示这个广告吗？
- 精排追求点击率最大化；重排追求用户留存和长期价值最大化

### 2. **重排的三类主流算法**
| 方法 | 复杂度 | 效果上界 | 工业应用 |
|-----|-------|---------|---------|
| **贪心启发式**（Round-Robin、距离惩罚） | O(n) | ~70% | 被广泛应用但有天花板 |
| **约束优化**（Integer Linear Programming, 动态规划） | O(n³) ~ O(2ⁿ) | ~90% | 中等规模候选（Top-100） |
| **学习排序**（LambdaMART on reranking list）| O(n log n) | ~95% | 大规模工业部署 |

### 3. **多样性和收入的帕累托关系**
- 用户刷 Feed，连续看 3 个同类广告 → **点击率 ↓ 40%**
- 优化多样性 → **用户留存 ↑** + **RPM 保持甚至 ↑**
- 这不是 trade-off，而是**双赢**

### 4. **重排层的四大约束来源**
| 约束类型 | 例子 | 处理方式 |
|---------|------|---------|
| **广告主约束** | 同一品牌最多 2 个 | 计数器 |
| **话题约束** | 电商不超过 40% | Entropy 限制 |
| **质量约束** | 低质广告排名下限 | 排名位置限制 |
| **用户约束** | 避免曾看过的广告 | 历史匹配过滤 |

### 5. **从离线到在线的闭环**
- **离线**：用历史数据训练排序模型（LambdaMART）
- **在线**：实时用户反馈调整权重，A/B Test 验证
- **反馈闭环**：点击/停留 → 更新特征 → 影响下一轮排序

## 📊 重排问题的完整定义

### 输入（Input）

```
【精排结果】
- Top-N 候选（例如 Top-50）
- 每个广告的特征：ID、CTR、CVR、bid、topic、advertiser、quality_score
- 用户上下文：user_id、历史、时间、设备

【系统约束】
- 多样性下限：某话题最少 X%
- 频率上限：同广告主最多 N 个
- 质量下限：排名前 K 中低质广告不超过 M 个
- RPM 目标：维持 >baseline RPM
```

### 输出（Output）

```
【最终排序】
- Top-K 排列顺序：[Ad_1, Ad_2, ..., Ad_K]
- K 通常 < N（例如 N=50 → K=20）
- 每个广告的排序分数、约束满足情况
```

### 问题类型

| 类型 | 难度 | 算法 |
|-----|------|------|
| **无约束** | ⭐ | 直接按 eCPM 排序 |
| **单约束**（多样性 OR 频率） | ⭐⭐ | 贪心 + 启发式 |
| **多约束** | ⭐⭐⭐ | ILP / DP / 学习排序 |
| **多目标** | ⭐⭐⭐⭐ | Pareto 优化 / RL |

## 🔧 重排算法分类

### A. 启发式重排（2010-2020）

**典型做法**：
```
1. 按 eCPM 排序得到初始排列
2. 循环遍历，逐个检查约束
3. 如果违反约束（如连续相同话题），则交换位置
4. 重复直到所有约束满足
```

**优点**：简单、快速（O(n²) 最坏情况）  
**缺点**：
- 无法学习，效果有天花板
- 启发式规则需手工调试
- 新约束需新规则

**工业案例**：早期 Google/Facebook 的混排层

---

### B. 约束优化重排（2015-2023）

**方法 1：Integer Linear Programming (ILP)**

```
maximize:  Σ x_i × score_i

subject to:
  - Σ_{i: advertiser(i)=a} x_i ≤ N_a  (频率约束)
  - Σ_{i: topic(i)=t} x_i / K ≥ p_t    (多样性约束)
  - x_i ∈ {0, 1}                       (二值决策)
  
求解：分支定界、整数规划求解器（Gurobi）
```

**缺点**：计算复杂度 O(2ⁿ)，N > 200 时不可行

---

**方法 2：动态规划（DP）**

```
定义状态：DP[i][state] = 前 i 个广告、约束满足状态 state 的最大收益
转移：对第 i+1 个广告，决策"选或不选"，更新约束状态
时间复杂度：O(n × |state_space|)，state 指数增长但可剪枝
```

**适用**：Top-100 候选，约束维度 < 5 个

---

### C. 学习重排（2020-2026）

**核心思想**：用梯度增强树（LambdaMART）直接优化重排指标

```
【训练】
- 离线标注：展示 [Ad_1, ..., Ad_K] → 用户点击情况
- 目标函数：NDCG（排序指标）或自定义 utility function
- 训练集：历史展示和点击数据

【推理】
- 输入：精排 Top-50 + 用户上下文
- 输出：学到的重排顺序
- 成本：~10ms
```

**优势**：
- 直接优化业务指标，效果最好（+2-5% RPM）
- 自动学习约束，无需手工规则
- 可灵活加入新特征

**挑战**：
- 标注成本（需要展示记录）
- 离线-在线偏差（训练集分布 vs 线上真实分布）
- 冷启动新话题时特征稀疏

```python
    """贪心启发式：快速满足约束"""
    ranking = []
    state = ConstraintState()
    
    while len(ranking) < K:
        # 从剩余候选中选择分数最高 + 不违反约束的广告
        best_ad = max([ad for ad in candidates 
                       if state.can_add(ad)],
                      key=lambda ad: score(ad))
        ranking.append(best_ad)
        state.add(best_ad)
        
    return ranking
```

## 🔗 与其他模块的关系

```
广告系统整体架构：

【召回】（候选生成）
  ↓
【精排】（单广告评分）
  ↓
【重排】（全局优化） ← 本文重点
  ↓
【混排】（与内容混合展示）
  ↓
【用户】（点击/转化反馈）
```

- 重排的输入依赖精排的质量
- 重排的输出影响混排效果（多样性约束）
- 反馈闭环用于训练新的重排模型
