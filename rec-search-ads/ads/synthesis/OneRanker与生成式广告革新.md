# OneRanker 与生成式广告：从判别到生成的范式跃迁

> 创建：2026-04-15 | 领域：广告系统 / 推荐系统 | 类型：技术综述

---

## 引言：广告系统正在经历什么？

2024-2026 年间，工业广告系统正在经历一场静悄悄的范式革命：**从判别式（Discriminative）到生成式（Generative）的全面迁移**。

传统广告系统是一条精心设计的四级流水线——召回、粗排、精排、重排——每个阶段有独立的模型、独立的优化目标、独立的工程团队。这种架构已经服务了十余年，但其固有的 **Stage Gap**（阶段鸿沟）问题正日益成为系统效率的天花板。与此同时，LLM 和扩散模型的成熟催生了生成式方法在广告全链路的渗透——从排序到出价，从创意制作到用户理解。

本文从两个代表性方向展开：**OneRanker（统一生成-排序）** 和 **生成式广告创意**，勾勒这场变革的技术轮廓。

---

## 第一部分：OneRanker — 消灭 Stage Gap 的统一范式

### 1.1 问题：四级流水线的结构性困境

传统广告排序系统的四阶段架构（召回 → 粗排 → 精排 → 重排）存在三个根本性问题：

1. **目标不对齐**：召回优化 Recall（"多找一些相关的"），精排优化 AUC（"从中挑最好的"），两个目标各自为战。更深层的冲突是——用户兴趣建模（CTR/CVR）与商业价值优化（eCPM、GMV）本质上是两个不同的优化空间。

2. **信息截断**：每个阶段的截断（10万→1千→100→30）不可逆地丢失了被过滤的候选。精排永远看不到被召回截掉的"潜在好物"，系统的天花板在第一阶段就被锁定。

3. **表示不一致**：生成（召回）和排序使用不同的模型架构和训练范式，二者的 embedding 空间天然不对齐，generate-then-rank 架构存在冗余和误差传播。

### 1.2 OneRanker 的三项核心创新

OneRanker（腾讯微信视频号广告系统，2026）提出了"单一模型统一生成与排序"的解决方案，其创新点可以用三个关键词概括：

#### 创新一：价值感知的多任务解耦（Value-Aware Multi-Task Decoupling）

传统方案要么用一个 tower 同时学兴趣和价值（耦合），要么用完全独立的模型（割裂）。OneRanker 设计了可学习的 **任务 Token 序列**：

$$T = [t_{i1}, ..., t_{im}, t_v]$$

其中 $t_{i1}...t_{im}$ 是兴趣 Token，建模用户多兴趣分布；$t_v$ 是价值 Token，专门学习商业目标（eCPM）。所有 Token 共享底层用户表示，但通过独立预测头解耦输出。**因果掩码**强制了任务排序先验：曝光 → 点击 → 转化 → 价值，让信息单向流动而非相互干扰。

**为什么重要？** 这解决了广告系统最核心的矛盾——用户体验和商业变现的目标冲突——不再是粗暴地用一个 $\alpha$ 加权，而是通过结构化的 Token 设计让模型自适应地学习二者的关系。

#### 创新二：粗到细的目标感知（Coarse-to-Fine Target Awareness）

生成阶段最大的短板是"target-agnostic"：用户表示在生成过程中固定不变，无法动态适应候选物品。OneRanker 设计了两级感知：

- **粗粒度**：通过 K-means 聚类构造"Fake Item Tokens"——它们不是真实物品，而是物品语义空间的"锚点"。将这些锚点拼接到任务 Token 中，让生成阶段就能感知物品的全局分布特征。
- **细粒度**：R-Decoder Block 对生成阶段的输出做 cross-attention，排序任务 Token 动态融合生成阶段已选出的候选物品信息。

**直观理解**：粗粒度就像一个导购先告诉你"今天男装、3C、运动三个品类最热"（全局分布），细粒度则是你已经选了几件衣服后，系统根据你的已选推荐搭配（上下文感知）。

#### 创新三：输入输出双侧一致性（Dual-Side Consistency）

生成阶段和排序阶段的表示空间如果不对齐，信息传递就会失真。OneRanker 从两个方向保证一致性：

- **输入侧**：KV Pass-Through，排序 Decoder 直接复用生成阶段的 Key/Value，物理上共享计算结果
- **输出侧**：Distribution Consistency Loss（KL 散度），确保生成分支和排序分支的偏好分布一致

联合损失函数：

$$\mathcal{L} = \mathcal{L}_{MTP} + \mathcal{L}_{rank} + \mathcal{L}_{DC}$$

### 1.3 效果与工程经验

| 指标 | 提升 | 说明 |
|------|------|------|
| HR@1 | +44.7% | 离线，相比 GPR（华为生成式统一排序） |
| HR@10 | +14.2% | 离线 |
| GMV-Normal | +1.34% | 微信视频号在线 A/B |

工程洞察：
- 训练时间约为独立模型的 1.5x，但减少了两套模型的维护成本
- 在线 serving 需支持生成/排序两种推理模式的统一服务
- 排序模式下候选集作为 context 注入，需限制 ≤200，否则延迟过高
- 建议渐进式迁移：先用排序任务初始化，再加入生成任务微调

### 1.4 OneRanker 在生成式广告版图中的位置

OneRanker 并非孤例。它属于一个更大的趋势——**统一模型取代级联流水线**：

| 模型 | 公司 | 统一范围 | 关键差异 |
|------|------|---------|---------|
| OneRanker | 腾讯 | 生成+排序 | 价值感知解耦 + 双侧一致性 |
| GPR | 华为 | 四阶段全链路 | 层级混合 Decoder（粗→细→交互） |
| EGA-v2 | — | 端到端广告 | Semantic ID（RQ-VAE） |
| MTGR | 美团 | 12+ 场景 | 场景 Prompt + 推测解码 |
| UniROM | — | 统一排序 | One-Model for all ranking |

从 OneRanker 到 GPR 再到 EGA-v2，这条路径的底层逻辑是清晰的：**用一个足够强大的生成模型，替代多个独立的判别模型，以端到端的方式联合优化全链路目标**。

## 第二部分：生成式广告创意 — AI 接管设计师的工作

如果说 OneRanker 是"选什么广告给用户看"这个问题的革新，那生成式创意则是"广告长什么样"这个问题的革命。

### 2.1 从 DCO 到 AIGC：四代创意优化

| 代际 | 时期 | 核心方法 | 局限 |
|------|------|---------|------|
| 1.0 模板时代 | 2015-2019 | 人工设计 + A/B 测试 | 成本高、迭代慢 |
| 2.0 DCO 时代 | 2019-2023 | 动态创意组合 + MAB/Thompson Sampling | 组合空间有限，素材仍需人工制作 |
| 3.0 CTR 导向生成 | 2023-2025 | Diffusion Model + CTR Reward | "好看"不等于"高CTR"的 gap 缩小 |
| 4.0 多 Agent 全自动 | 2025- | MLLM Agent 协作 + 组件化输出 | 品牌一致性仍有差距 |

### 2.2 关键革新点

#### 革新一：CTR-Driven 图像生成（CAIG, WWW 2025）

核心思想：将文生图模型的优化目标从"好看"（美学）对齐到"高点击"（CTR）。

三阶段流程：
1. **电商知识预训练**：在 120 万多模态样本上训练 MLLM 理解商品图片
2. **CTR 奖励模型**：将 CTR 预测转化为图片对的二分类比较——给定两张图，判断哪张 CTR 更高
3. **产品中心偏好优化（PCPO）**：构造对比对——正例是与原始商品匹配的描述，负例是与其他商品混淆的描述。这比标准 DPO 更精准地约束了"生成的图片必须忠于产品本身"

关键数据：
- 在线 A/B：整体 CTR +7.4%，美妆类目高达 +9.5%
- 每张素材成本 ~$0.05 vs 人工设计 $50-200，降低 3 个数量级
- 奖励模型配对准确率 58.6%（GPT-4V/Claude 3.5 仅 ~50%，接近随机）

**为什么重要？** 这是第一次在工业规模上证明了"CTR 可以作为图像生成的直接优化信号"。传统的 CLIP Score 或美学评分都是间接代理，而 CTR Reward Model 直接对接业务目标。

#### 革新二：多 Agent 协作设计（BannerAgency, EMNLP 2025）

BannerAgency 的创新不在单一技术，而在**系统设计范式**——把广告 Banner 的设计过程拆解为四个专业化 Agent 的协作流水线：

```
Strategist → Background Designer → Foreground Designer → Developer
品牌分析        背景图生成           布局规划           代码渲染
```

关键突破：
- **组件化输出**：输出 SVG/Figma 格式，广告主可以直接编辑文字、颜色、布局，而非不可修改的像素图——修改时间减少 ~60%
- **无需训练**：纯提示工程 + 预训练 MLLM，随基座模型升级自动提升
- **多尺寸自适应**：一次设计自动生成移动端、PC端、各种广告位的适配版本

成本对比：每张 Banner $0.09-$1.21 vs 人工设计师 $50-$150。

#### 革新三：组合级创意优化（GenCO, 2025）

GenCO 解决了一个被忽视的问题：**创意元素的组合效应**。

标题 A 单独测效果好，图片 B 单独测效果好，但 A+B 组合未必好——颜色冲突、信息重复、视觉干扰都可能让 1+1 < 2。传统方法要么忽略组合效应（独立评估每个元素），要么穷举搜索（标题 × 图片 × 亮点的组合空间呈指数增长）。

GenCO 的方案：
1. **生成式模型**快速产出多样创意组合候选集
2. **多实例学习（MIL）奖励模型**将组合级反馈（点击/不点击）归因到单个创意元素——用弱监督信号学强信号

在线效果：CTR +3.88%，RPM +3.26%。

#### 革新四：个性化创意生成（HLLM-Creator, 字节跳动, 2025）

HLLM-Creator 推动了创意生成的最后一公里——**千人千面**。不同用户看到的广告创意不同，且创意内容必须与商品事实一致（幻觉控制）。

技术方案：
- CoT 驱动的数据构造：先推理"为什么这个用户会点这条广告"，再反向构造创意
- 层级 LLM 建模用户兴趣和商品属性的交互
- 事实一致性检验：RAG 检索商品属性作为 grounding，后处理校验价格/规格/品牌名

## 第三部分：技术趋势总结

### 趋势一：生成取代判别

从 OneRanker 到 GPR 到 EGA-v2，广告系统的核心模块正逐步从判别式（"这个广告点击概率是多少？"）切换为生成式（"直接生成用户最可能点击的广告序列"）。这不仅仅是模型架构的变化，更是优化目标的根本转变——从"评分+排序"到"端到端生成"。

核心公式对比：

$$\text{判别式：} \hat{y} = f(user, item) \in [0,1]$$

$$\text{生成式：} P(item_1, ..., item_k | user) = \prod_{i=1}^{k} P(item_i | item_{<i}, user)$$

### 趋势二：基础模型统一多场景

美团 MTFM 验证了推荐基础模型的 scaling law（1B→10B，AUC 持续提升），新场景接入时间从数周缩短到 3 天。Meta GEM 用 LLM 启发的架构将训练效率提升 4x。**一个基础模型服务全平台**正在成为现实。

### 趋势三：创意从人工制作到 AI 全自动

AIGC 在广告创意领域的渗透速度超出预期：
- 文案生成已接近人工水平（GPT-4 级别）
- 图片生成可用但需审核（CTR 导向生成缩小了美学和效果的 gap）
- 视频生成进入早期应用阶段

关键约束仍在品牌一致性（~15% 差距）和内容安全审核。

### 趋势四：LLM 渗透从离线到在线

LLM 在广告系统中的集成路径日益清晰：
1. **离线特征增强**：LLM 提取文本/多模态特征，蒸馏到轻量模型（ELEC: <2ms 延迟）
2. **数据质量提升**：ICL-Bandit 用 LLM 做高质量标注，准确率 +6.3%
3. **冷启动破解**：IDProxy 用多模态 LLM 为新物品生成代理 embedding
4. **创意生成**：BannerAgency、CAIG 直接用 MLLM 生成广告素材

## 面试/汇报怎么讲

### 30 秒版本

> "2024-2026 年广告系统最大的变化是从判别到生成的范式迁移。以 OneRanker 为代表，用一个统一的生成模型取代四阶段级联流水线，消除阶段间的目标不对齐和信息截断。在创意侧，CTR 导向的图像生成和多 Agent 协作设计让 AI 接管了从素材制作到组合优选的全流程，成本降低三个数量级，CTR 提升 7-10%。"

### 3 分钟版本

> "我重点介绍两个方向。
>
> **第一是排序侧的 OneRanker**。传统四阶段流水线有三个固有缺陷：目标不对齐、信息截断、表示不一致。OneRanker 用统一的 Transformer 同时做生成（召回）和排序，三个核心设计——Value-Aware Multi-Task Decoupling 解耦兴趣和商业价值目标、Coarse-to-Fine Target Awareness 让生成阶段也能感知候选物品、Dual-Side Consistency 确保两个分支的表示对齐。在微信视频号广告上线 GMV +1.34%。
>
> **第二是创意侧的生成式革新**。CTR-Driven Image Generation（WWW 2025）首次证明可以用 CTR 作为图像生成的直接优化信号，线上 CTR +7.4%。BannerAgency（EMNLP 2025）用四个 MLLM Agent 协作完成 Banner 设计，输出可编辑的 SVG/Figma 组件。GenCO 用多实例学习解决创意组合评估问题。字节 HLLM-Creator 实现了千人千面的个性化创意。整体趋势是 AI 正在接管广告创意的全生命周期——从制作到优选到个性化分发。"
