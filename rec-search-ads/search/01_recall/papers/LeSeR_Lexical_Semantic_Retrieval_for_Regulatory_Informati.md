# LeSeR: Lexical-Semantic Retrieval for Regulatory Information Retrieval

> 来源：https://aclanthology.org/2025.regnlp-1.6.pdf | 日期：20260320 | 领域：search

## 问题定义

监管文档（Regulatory Documents）对组织确保合规性构成重大挑战，原因包括：
- 文档复杂且不断变化
- 需要自动化信息检索、监管对比和合规验证流程
- 传统检索方法难以处理专业术语和领域特定表达
- 纯语义检索或纯词汇检索在监管领域均存在局限性

**核心问题**：如何在监管领域实现高质量的信息检索，以支持问答（Q/A）和合规检查任务？

## 核心方法与创新点

### 1. LeSeR 框架（Lexical reranking of Semantic Retrieval）

LeSeR 是一种创新的混合检索方法，结合了密集语义检索（Dense Semantic Retrieval）和经典词汇重排序（Lexical Reranking）：

**两阶段架构**：
- **第一阶段 - 语义检索**：使用微调后的嵌入模型（如BGE）生成密集向量，通过FAISS进行近似最近邻搜索，检索Top-20候选段落
- **第二阶段 - 词汇重排序**：使用BM25对语义检索结果进行重排序，结合语义分数和词汇分数进行加权聚合

**关键创新**：
1. **严格模块化**：与传统混合检索同时融合两种分数不同，LeSeR将两个阶段解耦，语义检索负责高召回率，词汇重排序负责精确度
2. **加权聚合**：通过加权组合密集语义分数和BM25稀疏分数，生成最终排序

### 2. 模型选择

实验了多种开源嵌入模型：
- Stella
- BGE（最终选用）
- CDE
- Mpnet

### 3. 检索流程

```
查询 → 密集向量编码 → FAISS检索Top-20 → BM25重排序 → Top-10结果 → LLM生成答案
```

## 实验结论

### 数据集
- COLING 2025 RegNLP RIRAG挑战赛
- 监管信息检索和答案生成任务

### 主要结果
- **Recall@10**: 0.8201
- **MAP@10**: 0.6655
- BGE + LeSeR 配合 Qwen2.5 7B 在召回率和答案质量上表现最佳

### 关键发现
1. 混合方法显著优于纯词汇或纯语义模型
2. 领域微调对监管文本检索至关重要
3. 两阶段解耦设计在监管领域表现优于传统融合方法

## 工程落地要点

### 1. 部署架构
```
用户查询 → 嵌入模型(BGE) → 向量检索(FAISS) → BM25重排序 → 结果输出
```

### 2. 关键配置
- **FAISS索引**：建议使用IVF或HNSW索引平衡速度和精度
- **BM25参数**：需针对监管语料调优k1和b参数
- **加权系数**：语义分数和词汇分数的权重需在开发集上调优

### 3. 性能优化
- 嵌入模型可离线计算并缓存文档向量
- BM25检索可利用倒排索引加速
- 建议对高频查询实施缓存策略

### 4. 适用场景
- 法律法规检索
- 合规文档问答
- 金融监管信息查询
- 医疗监管条款检索

## 常见考点

**Q1：LeSeR与传统混合检索的主要区别是什么？**
A：LeSeR采用严格的两阶段解耦设计：第一阶段用密集检索获取高召回候选集，第二阶段用BM25对候选集重排序。传统混合检索通常在同一阶段融合两种分数。LeSeR的模块化设计更适合监管领域，可针对每个阶段独立优化。

**Q2：为什么监管领域需要结合词汇和语义检索？**
A：监管文本具有三个特点：(1) 专业术语精确性要求高，适合词汇匹配；(2) 同义词和表述变化多，需要语义理解；(3) 合规要求严格，不能遗漏关键条款。纯词汇检索可能错过同义表述，纯语义检索可能忽略关键术语匹配。

**Q3：BM25在重排序阶段的作用是什么？**
A：BM25在重排序阶段发挥"精排"作用：(1) 利用精确术语匹配校正语义检索的误差；(2) 对专业术语赋予更高权重；(3) 提供可解释的相关性分数；(4) 弥补嵌入模型在罕见术语上的不足。

**Q4：LeSeR方法在工程部署时的瓶颈在哪里？**
A：主要瓶颈包括：(1) 双路检索增加延迟（向量检索+BM25重排序）；(2) 需要维护两套索引（向量索引和倒排索引）；(3) 加权参数需要领域数据调优；(4) 长文档的嵌入计算开销大。优化建议包括预计算文档向量、使用近似检索、实施多级缓存等。
