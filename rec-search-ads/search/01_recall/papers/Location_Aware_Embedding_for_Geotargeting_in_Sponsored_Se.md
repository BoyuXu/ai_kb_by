# Location Aware Embedding for Geotargeting in Sponsored Search Advertising

> 来源：arxiv | 日期：20260321 | 领域：search

## 问题定义

搜索广告的地理定向（Geotargeting）是广告主指定广告只对特定地区用户展示的功能。现有系统大多用粗粒度地理区域（城市/省份）匹配，存在以下问题：

1. **粗粒度定向精度低**：同一城市内商圈差异巨大（三里屯 vs 五环外郊区），用户需求和消费能力差异显著
2. **地理特征孤岛**：地理信息通常作为独立类别特征处理，无法捕获地点间的语义相似性（如"望京"和"国贸"同属CBD商圈）
3. **稀疏性问题**：细粒度地理位置（如街道级）数据稀疏，难以学习有效表示

本文提出 Location-Aware Embedding，将地理位置编码为稠密语义 embedding，并与搜索查询和广告特征深度融合。

## 核心方法与创新点

1. **层次化地理编码**：
   - 地理位置按多粒度层次编码：国家 → 省/州 → 城市 → 区 → 街道 → 坐标（GPS）
   - 每层独立 embedding，通过注意力机制融合：
   ```
   loc_emb = Attention([country_emb, city_emb, district_emb, street_emb])
   ```

2. **地理-语义融合**：
   - 引入 POI（Point of Interest）信息：周边商圈类型、消费密度、人口特征
   - 用 GNN（图神经网络）建模地点之间的地理邻近关系
   - "同商圈"的位置在 embedding 空间中距离近

3. **实时位置感知 CTR 预估**：
   - 将用户实时位置 embedding 与查询 embedding、广告 embedding 三向融合
   - 考虑位置-意图关联："在机场搜索酒店" vs "在市中心搜索酒店"需求不同

4. **GPS 坐标编码**：
   - 使用正余弦位置编码（类似 Transformer 位置编码）将经纬度编码为连续向量
   - 避免将坐标直接离散化（粒度固定）或忽略绝对坐标

## 实验结论

- 地理定向广告 CTR **+11.3%**，CVR **+8.7%**（相比城市级粗粒度定向）
- 细粒度位置（街道级）的广告主 ROI **+15.2%**（相关性更高）
- 冷启动新区域（新开业商圈）：基于 POI 语义迁移，效果比纯稀疏 ID 好 **+28%**

## 工程落地要点

1. **用户位置获取**：GPS 定位需用户授权（隐私），降低覆盖率；可用 IP 定位（精度差）或 WiFi/基站定位作为补充
2. **位置数据新鲜度**：POI 数据（商场、餐厅等）变化频繁，需要定期更新地图数据，防止 embedding 过时
3. **实时 vs 离线**：用户实时位置需要在线推理，但 POI embedding 可以离线预计算，只在线查表
4. **隐私合规**：精细位置数据是个人隐私，需要用差分隐私（Differential Privacy）或模糊化（Geohash 降精度）处理

## 常见考点

- Q: 搜索广告中地理位置信息如何影响 CTR 预估？
  A: 位置信息影响用户意图（在餐厅附近搜"外卖"和在家搜的意图不同）、消费能力（商圈级别）和供给（周边有哪些门店）。高效建模需要捕获这些语义关联，而非简单的城市 ID。

- Q: 如何处理地理位置信息的稀疏性？
  A: (1) 层次化共享：细粒度位置共享粗粒度的 embedding；(2) POI 特征迁移：基于周边商业环境的相似性泛化；(3) GNN：利用地理邻近关系传播信息；(4) Geohash 分桶：将连续坐标映射到固定精度的网格，平衡稀疏性和精度。

- Q: 广告系统中的地理定向（Geotargeting）有哪些技术实现方式？
  A: (1) 粗粒度：国家/城市级别，根据用户 IP 或注册城市匹配；(2) 半径定向：以某坐标为圆心 N 公里内；(3) 地图区域定向：广告主在地图上画多边形；(4) 兴趣地点定向（POI-based）：曾去过某类地点的用户；(5) 本文的语义地理 embedding 是最精细的方式。
