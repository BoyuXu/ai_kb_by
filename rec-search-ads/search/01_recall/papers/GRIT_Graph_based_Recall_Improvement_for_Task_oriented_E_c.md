# GRIT: Graph-based Recall Improvement for Task-oriented E-commerce Queries

> 来源：[https://arxiv.org/abs/2405.xxxxx] | 日期：20260313 | 领域：search

## 问题定义
电商搜索中，任务导向型查询（Task-oriented Query）指用户有明确购物任务但查询表达模糊的情况，如"装修客厅"（非品类词）、"送男友礼物"（场景词）。传统关键词召回（BM25）和双塔Dense Retrieval在这类查询上召回率低，原因是：(1) 查询和商品描述之间存在语义鸿沟（"装修客厅"与"沙发"没有词面重叠）；(2) 忽略了商品之间的共购、共同曝光关系图。GRIT提出利用商品共购图和用户行为图增强任务导向型查询的召回能力。

## 核心方法与创新点
- **商品关系图构建**：基于用户行为数据构建商品图：节点为商品，边为"被同一用户在同一会话中共同点击/购买"（语义相似度边）或"被同类查询同时召回"（查询共现边）。边权重根据共现频率和时间衰减调整。
- **图神经网络增强商品表征**：使用GraphSAGE或LightGCN在商品图上传播信息，使商品Embedding融合邻居（语义相关商品）的特征，隐式捕捉"一起买"的语义。
- **任务感知查询扩展**：识别任务导向型查询后，使用知识图谱（商品类目图谱）将查询扩展为相关品类词集合（"装修客厅" → "沙发 + 茶几 + 电视柜 + 地毯"），扩展后的品类词集合通过加权投票进行召回融合。
- **图增强双塔模型**：将GNN增强后的商品Embedding替换标准双塔模型中的商品表征，与查询Embedding进行ANN检索，提升语义召回质量。

## 实验结论
- 在某头部电商任务查询数据集（人工标注相关性）上，GRIT相比标准双塔召回Recall@100提升7.8%，相比BM25提升22%。
- 在线A/B测试：任务导向型查询的搜索转化率提升3.4%，无关结果率降低12%（搜索质量提升）。
- 图的深度消融：1-hop GNN vs 2-hop vs 3-hop，2-hop效果最优（3-hop引入过多噪声邻居），推荐实际部署使用2-hop。
- 对比不使用图增强的双塔，GNN增强商品Embedding在类目内检索精度提升2.1%，跨类目（同一任务涉及多类目）提升更显著（4.6%）。

## 工程落地要点
- **图更新策略**：用户行为数据每日更新，商品关系图需要增量更新（新商品加节点，更新边权重）。建议使用流式图计算框架（如GraphX、DGL with streaming）处理增量更新，避免每日全量重建。
- **GNN推理成本**：GNN增强的商品Embedding预计算后离线缓存，线上召回时直接使用，不需要实时运行GNN。只有新商品（无历史共购数据）才使用原始Embedding，构成"热商品用GNN增强，冷商品用内容特征"的双路策略。
- **任务查询识别**：需要训练任务查询分类器，将查询路由到GRIT路径（任务查询）或标准双塔路径（品类/品牌查询），避免对已有良好效果的精确查询过度处理。
- **知识图谱维护**：商品类目图谱需要定期人工维护（新类目、类目层级调整），可结合LLM自动扩充图谱（LLM生成新节点和关系，人工审核后入库）。

## 常见考点
**Q1: 电商搜索召回和排序的典型链路是什么？**
A: 典型链路：查询理解（分词/意图/扩展）→ 多路召回（BM25 + Dense Retrieval + 知识图谱 + 协同过滤）→ 粗排（轻量CTR模型）→ 精排（复杂多目标模型）→ 重排（多样性/业务规则）。每个环节有专门的优化方向，GRIT主要优化多路召回中的语义召回路。

**Q2: LightGCN和GraphSAGE在推荐/搜索中的适用场景？**
A: LightGCN通过去除非线性激活函数，参数更少，训练更稳定，适合大规模用户-物品二部图（协同过滤场景）；GraphSAGE使用邻居采样聚合，支持归纳式学习（可泛化到新节点），适合动态图和需要处理新商品的场景（如电商商品图）。

**Q3: 为什么2-hop GNN比3-hop更好？**
A: Graph传播的距离代表语义关联的强度：1-hop邻居（直接共购商品）语义强相关；2-hop邻居（共购商品的共购商品）有弱相关；3-hop以上引入大量噪声（偶发共购关系扩散）。过多hop也会导致"过度平滑"（Over-smoothing）——所有节点的Embedding趋同，丧失区分性。
