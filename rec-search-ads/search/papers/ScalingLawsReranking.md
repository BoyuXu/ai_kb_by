# 信息检索重排序的 Scaling Laws

> 来源：Scaling Laws for Reranking in Information Retrieval | 日期：20260318 | 领域：search

## 问题定义

在信息检索中，重排序（Reranking）模型（如 cross-encoder）的效果随模型规模（参数量）和训练数据量的增长如何变化？是否遵循类似 LLM 的 Scaling Laws？这个问题对工业界实践至关重要：**投入更大的重排序模型值得吗？数据收益边际递减在哪里？**

## 核心方法与创新点

1. **系统性实验设计**：在 MS-MARCO、BEIR 等标准数据集上，训练不同规模的 cross-encoder 重排序模型（BERT-tiny → BERT-base → BERT-large → T5-3B），控制变量（数据量固定 / 模型固定）。

2. **幂律拟合**：发现重排序效果（NDCG@10）与模型参数量 N 和训练 token 数 D 之间存在幂律关系：
   - `NDCG(N, D) ≈ A · N^α · D^β + C`
   - α ≈ 0.07-0.09（模型规模收益），β ≈ 0.04-0.06（数据规模收益）

3. **最优计算分配**：在固定 FLOPs 预算下，最优分配是**模型规模 > 数据规模**（比 LLM 预训练的 Chinchilla 结论不同），因为重排序任务复杂度高，更大模型的边际收益更大。

4. **检索质量影响**：第一阶段检索质量（召回率）对重排序效果有上限效应，重排序无法弥补召回阶段遗漏的相关文档。

## 实验结论

- 参数量从 110M (BERT-base) 增至 3B (T5-3B)，NDCG@10 稳定提升约 3-4 个点。
- 训练数据从 100K 增至 10M 查询-文档对，提升约 2-3 个点，但边际递减明显（从 1M 到 10M 仅提升约 0.8 点）。
- 幂律预测在 1B+ 参数范围内仍然成立，没有观察到明显的"涌现"现象。

## 工程落地要点

- **阶段性选择**：检索阶段追求 Recall（保证相关文档不遗漏），重排序阶段追求 Precision（相关文档靠前）；两阶段 scaling 策略不同。
- **延迟-效果 trade-off**：大模型重排序效果好但延迟高，实际工程常用蒸馏将大模型知识压缩到小模型（ColBERT、MonoT5 的 small 版本）。
- **数据质量 > 数据数量**：重排序训练数据需要高质量的相关性标注（BM25 难负例、人工标注）；堆砌低质量数据收益有限。
- **预算分配建议**：固定推理预算时，优先用于模型规模（更大的 cross-encoder），而非增加数据量。

## 面试考点

**Q: 为什么重排序用 cross-encoder 而不是 bi-encoder？**
- Cross-encoder 对 query-document 做完整交叉注意力，捕捉细粒度交互，精度高但速度慢（需对每个候选单独推理）；Bi-encoder 独立编码，只做 dot product，速度快但精度略低，适合第一阶段检索。

**Q: 重排序的 Scaling Law 和 LLM 预训练 Scaling Law 的区别？**
- LLM 预训练：Chinchilla 发现最优分配是模型和数据等比例增长；重排序：模型规模的边际收益更大，应偏向大模型。两者任务性质不同（生成 vs 判别）。

**Q: 如何在工业中部署大型重排序模型？**
- 知识蒸馏：大模型离线生成软标签，训练小模型；量化（INT8/FP16）：减少显存和计算量；batching 优化：将多候选打包推理；缓存：常见 query 的重排序结果缓存。
