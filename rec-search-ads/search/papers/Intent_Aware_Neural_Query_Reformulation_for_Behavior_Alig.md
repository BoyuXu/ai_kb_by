# Intent-Aware Neural Query Reformulation for Behavior-Aligned Product Search

> 来源：arxiv | 日期：20260321 | 领域：search

## 问题定义

电商产品搜索中，用户输入的查询（Query）往往存在以下问题：

1. **查询过于简短**：用户输入"手机"，真实需求可能是"5000元左右的拍照好的手机"
2. **查询不准确**：用户可能拼写错误、使用俗语/简称（"华为小折叠"而非"华为Pocket S"）
3. **意图模糊**：同一查询"苹果"在不同上下文可能指水果或 Apple 品牌

查询改写（Query Reformulation）通过将原始查询转化为更精准的查询来改善召回和排序。传统基于规则/词典的改写无法捕获语义意图。

## 核心方法与创新点

1. **行为对齐的意图建模**：
   - "行为对齐"（Behavior-Aligned）：改写后的查询应与用户的历史购买/点击行为中的物品描述对齐
   - 而非仅仅与搜索结果文本对齐（传统方法的局限）
   - 引入用户历史行为序列作为意图上下文

2. **Seq2Seq 查询改写模型**：
   - 输入：原始查询 + 用户最近 K 个点击/购买商品的标题
   - 输出：改写后的查询（扩展更多属性词）
   - 基于预训练 T5/BART 微调，训练数据来自用户成功购买的（查询, 商品标题）对

3. **意图分类辅助**：
   - 先对查询进行意图分类（品类意图、属性意图、品牌意图）
   - 不同意图类型使用不同的改写策略（品类扩展 vs 属性细化 vs 品牌纠错）

4. **多候选改写 + 在线排序**：
   - 生成 Top-K 候选改写
   - 用轻量模型对候选打分（召回率、与用户历史的一致性）
   - 选择最优改写用于后续检索

## 实验结论

- 相比原始查询直接检索：Recall@100 **+16.2%**，完美匹配率（搜索结果=用户意图商品）**+22%**
- 相比基于词共现的传统改写：NDCG@10 **+9.3%**
- 行为对齐的意图建模相比纯文本改写：对个性化查询的改写准确率 **+31%**（相同查询、不同用户历史 → 不同改写）

## 工程落地要点

1. **改写延迟**：查询改写在搜索链路最前端，延迟要求极严（<20ms），需要蒸馏为小模型或用 KV Cache 缓存高频查询的改写结果
2. **改写链路熔断**：模型故障时必须有 fallback（直接用原始查询），不能让改写模块成为单点故障
3. **个性化隐私**：使用用户历史行为做改写涉及个人数据，需要合规处理（联邦学习/差分隐私）
4. **AB 测试设计**：查询改写影响整个搜索链路，需要全链路（召回→排序）A/B 测试，而非只看改写层的准确率

## 面试考点

- Q: 电商搜索中查询改写有哪些常见策略？
  A: (1) 同义词扩展（手机 → 智能手机、移动电话）；(2) 纠错（ipone → iPhone）；(3) 属性扩展（短袖 → 夏季短袖 T 恤）；(4) 品牌规范化（华为小折叠 → 华为 Pocket S）；(5) 意图缩写展开（OOTD → 今日穿搭）。本文重点是基于用户行为的个性化属性扩展。

- Q: 查询理解（QU）包含哪些子任务？
  A: (1) 查询纠错（Spell Correction）；(2) 查询分类（Category Classification）；(3) 实体识别（品牌/属性/价格识别）；(4) 查询改写/扩展（Reformulation）；(5) 意图识别（导航式/信息式/交易式）。这些任务在工业系统中通常并行运行。

- Q: 如何评估查询改写的质量？
  A: (1) 离线：改写后的查询是否能检索到用户最终购买的商品（Recall）；(2) 在线：改写后 Session CTR、转化率、用户搜索放弃率；(3) 人工评估：改写是否保留了原始意图、是否引入了错误扩展。离线和在线指标经常不一致，最终以在线为准。
