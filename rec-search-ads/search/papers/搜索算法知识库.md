## 核心知识体系

### 1. 工业搜索系统架构

```
用户 Query
     ↓
[Query 理解] 分词 / 纠错 / 意图识别 / 扩展
     ↓
[召回层] 
├── 倒排索引（BM25/TF-IDF）← 精确匹配
├── 稠密向量检索（双塔模型 + Faiss ANN）← 语义匹配
└── 稀疏向量（SPLADE / TILDE）← 语义稀疏
     ↓
[粗排] 轻量级 relevance 模型
     ↓
[精排] Cross-Encoder（BERT 类）/ 重排模型
     ↓
搜索结果页（SERP）
```

### 2. 检索模型演进

| 方法 | 类型 | 优点 | 缺点 |
|------|------|------|------|
| BM25 | 稀疏 | 高效、可解释 | 词汇不匹配 |
| DPR | 稠密双塔 | 语义召回好 | 需要 ANN，训练成本高 |
| ColBERT | 多向量 | 精度高 | 存储大，延迟高 |
| SPLADE | 语义稀疏 | 兼顾效率和语义 | 训练复杂 |
| Cross-Encoder | 交互式 | 精度最高 | 延迟高，只能用于精排 |

### 3. 多向量重排（2026 趋势）

来自 arXiv 2026-01：**Multivector Reranking in the Era of Strong First-Stage Retrievers**
- 背景：当 first-stage retriever 已经很强时，多向量 reranker 的增益在下降
- 结论：需要更聪明的 reranking 策略，而不是简单堆叠多向量
- 意义：对搜索系统 reranking 模块的设计有指导意义

## 2026-03-12 更新

### 本次新增内容

**1. Farewell to Item IDs - 语义 Token 用于排序扩展（与搜索相关）**
- 语义 token 的思路同样适用于搜索结果排序：doc 用语义 token 表示，支持零样本推广
- 对跨域搜索（多品类、多语言）特别有价值

**2. Multivector Reranking 的局限性发现**
- First-stage retriever 越来越强（DPR → ColBERT → 更强的双塔）
- 多向量 reranker 的边际收益递减
- 工程启示：优先优化 retriever 质量，再考虑 reranker 的复杂化

**3. 搜索 + 推荐融合趋势**
- RGAlign-Rec (2026-02-13)：在电商 chatbot 中融合搜索意图和推荐逻辑
- "零查询推荐"（zero-query recommendation）：从用户行为预测意图，无需显式 query

### 工程实践建议

1. **Hybrid 检索**：BM25 + 稠密向量 ensemble，互补效果好
2. **Late Interaction**（如 ColBERT）：适合精排，不适合大规模召回
3. **Query 改写**：LLM query expansion 可以显著提升 recall@k
