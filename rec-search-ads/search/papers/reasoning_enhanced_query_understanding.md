# Reasoning-enhanced Query Understanding through Decomposition and Interpretation

> 来源：arXiv 2509.06544 | 领域：search | 学习日期：20260402

## 问题定义

复杂搜索查询（如多条件、隐含意图、比较型查询）的理解是检索系统的关键瓶颈。传统 query 理解方法依赖浅层特征匹配，难以处理需要推理的查询。本文通过查询分解和意图解释增强 query 理解。

## 核心方法与创新点

1. **查询分解**：将复杂查询分解为多个原子子查询，每个子查询对应一个独立的检索意图
2. **意图解释**：为每个子查询生成自然语言解释，明确用户的隐含需求
3. **推理链**：构建查询理解的 Chain-of-Thought 推理链，从表层文字到深层意图
4. **检索融合**：各子查询的检索结果通过学习到的融合策略合并

## 实验结论

- 在复杂查询数据集上 NDCG@10 提升 8.5%
- 推理链的可解释性评估中 82% 被标注者认为合理
- 对长尾查询的改善尤为显著（提升 15%）

## 工程落地要点

- **延迟控制**：推理链生成增加延迟，需要权衡质量与速度
- **查询分类**：先判断查询复杂度，只对复杂查询启用推理增强
- **缓存**：高频查询的分解结果可以缓存复用

## 面试考点

1. **Q：查询分解和查询扩展的区别？**
   A：查询扩展是添加同义词/相关词，不改变查询结构；查询分解是将复杂查询拆分为多个独立意图，需要理解查询的逻辑结构。
2. **Q：如何判断查询是否需要分解？**
   A：基于查询长度、是否含连词/比较词、预估复杂度分数来决定。简单查询直接检索效率更高。
