# Generative Query Expansion for E-Commerce Search at Scale
> 来源：https://arxiv.org/search/?query=generative+query+expansion+ecommerce+search&searchtype=all | 领域：search | 日期：20260323

## 问题定义
电商搜索中，用户查询往往短而模糊，词汇鸿沟（vocabulary gap）导致召回不足。生成式查询扩展（GQE）利用LLM生成相关词和同义词，扩展查询覆盖范围。

## 核心方法与创新点
- LLM驱动扩展：用LLM生成查询的相关词、同义词、上位词
- 电商场景适配：针对品牌、型号、属性等电商特有词汇的扩展策略
- 质量过滤：对LLM生成的扩展词进行相关性过滤，避免噪声引入
- 规模化部署：亿级查询的离线扩展与在线增量更新策略

## 实验结论
在大规模电商搜索（ASIN检索），生成式查询扩展将零结果率降低约40%；Recall@100提升约12%；精确率下降约3%（引入少量噪声），综合F1提升约8%。

## 工程落地要点
- 高频查询可以离线预扩展并缓存，低频查询实时扩展
- 扩展词的权重需要低于原始查询词，避免语义漂移
- 需要黑名单机制，防止敏感词或竞品品牌名被扩展进来

## 面试考点
1. **Q: 查询扩展（Query Expansion）的传统方法？** A: 伪相关反馈（PRF）、WordNet同义词、用户点击数据挖掘、KNN查询扩展
2. **Q: 生成式查询扩展相比传统方法的优势？** A: 语义理解更强、能处理新词/长尾词、生成多样化表达
3. **Q: 电商搜索的零结果率（Zero Results Rate）如何降低？** A: 查询扩展、查询改写、模糊匹配、同义词词典
4. **Q: 查询扩展引入噪声的风险如何控制？** A: 相关性过滤（分类器/embedding相似度）、权重降低、A/B实验验证
5. **Q: 大规模电商如何处理数十亿查询的扩展？** A: 频率分层（高频离线、低频在线）、增量更新、分布式批处理
