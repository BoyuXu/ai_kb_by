# On the Theoretical Limitations of Embedding-Based Retrieval

> 来源：arXiv 2508.21038 | 领域：search | 学习日期：20260402

## 问题定义

基于嵌入的检索（双塔模型）已成为工业级检索系统的标配，但其理论表达能力的局限性尚未被充分理解。本文从理论角度分析嵌入检索的表达能力边界，揭示其在哪些场景下注定失败。

## 核心方法与创新点

1. **表达能力分析**：证明固定维度的嵌入空间无法完美表达某些查询-文档相关性模式
2. **维度下界**：给出嵌入维度的理论下界——要精确表达 N 个文档的相关性，至少需要 O(log N) 维
3. **交互模式限制**：内积/余弦相似度只能表达线性可分的相关性模式，对非线性关系（如"与 A 和 B 都相关但不与 C 相关"）有固有限制
4. **改进方向**：提出多嵌入表示（一个实体用多个向量）和后期交互作为突破方案

## 实验结论

- 在合成数据上验证了理论预测：128 维嵌入在 100 万文档时出现明显的表达能力瓶颈
- 多嵌入表示将 Recall@100 从 85% 提升到 93%
- 后期交互（ColBERT 风格）几乎不受这些理论限制影响

## 工程落地要点

- **维度选择**：嵌入维度不是越大越好，存在收益递减。建议 256-768 维作为实践起点
- **多嵌入策略**：对复杂实体（如多主题文档），考虑使用多个嵌入向量
- **混合检索**：嵌入检索的限制可以通过稀疏检索补偿

## 常见考点

1. **Q：为什么单向量嵌入的表达力有限？**
   A：内积/余弦相似度本质是线性函数，只能划分线性可分的相关性模式。非线性相关性（如 XOR 类）无法精确表达。
2. **Q：多嵌入表示如何提升表达力？**
   A：用多个向量表示一个实体，查询与文档的相关性通过多个向量对的最大相似度计算，增加了非线性表达能力。
3. **Q：这些理论限制在实践中严重吗？**
   A：对大多数查询-文档对来说不严重，但对复杂查询和多主题文档可能造成显著的检索质量损失。
