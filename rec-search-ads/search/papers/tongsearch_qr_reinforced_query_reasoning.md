# TongSearch-QR: Reinforced Query Reasoning for Retrieval

> 来源：arXiv 2506.11603 | 领域：search | 学习日期：20260402

## 问题定义

在推理密集型检索任务（如多跳问答、复杂条件检索）中，直接用用户原始查询检索效果差。TongSearch-QR 提出用小规模 LLM（1.5B/7B）通过强化学习学习查询推理重写，在推理密集型检索任务上达到 GPT-4 级别效果。

## 核心方法与创新点

1. **查询推理重写**：LLM 先分析查询的推理需求，生成包含推理链的增强查询
2. **强化学习训练**：用检索效果（Recall/NDCG）作为奖励信号训练查询重写器
3. **小模型高性能**：1.5B/7B 参数模型通过 RL 训练后在推理检索上媲美 GPT-4
4. **即插即用**：重写器与检索器解耦，可以与任意检索系统组合使用

## 实验结论

- 7B 模型在 BRIGHT 基准上达到 GPT-4 效果的 95%
- 相比无推理重写，检索 Recall@100 提升 23%
- 1.5B 模型推理延迟仅 50ms，适合在线部署

## 工程落地要点

- **延迟预算**：查询重写增加 30-80ms 延迟，需要在延迟预算内
- **选择性启用**：简单查询不需要推理重写，可通过查询复杂度分类器控制
- **持续训练**：RL 奖励来自检索效果，可以持续在线优化

## 面试考点

1. **Q：为什么小模型能通过 RL 达到 GPT-4 级别？**
   A：查询重写是相对窄的任务，RL 的任务特定优化可以让小模型在这个特定能力上超越通用大模型。
2. **Q：RL 训练查询重写的奖励如何设计？**
   A：将重写后的查询送入检索系统，用检索结果与 ground truth 的 Recall/NDCG 作为奖励。
3. **Q：和传统查询扩展相比优势？**
   A：传统查询扩展添加关键词，TongSearch-QR 做推理式重写——理解查询意图后生成全新查询表达。
