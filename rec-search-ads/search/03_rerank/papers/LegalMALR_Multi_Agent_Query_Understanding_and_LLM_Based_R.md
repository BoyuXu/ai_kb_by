# LegalMALR: Multi-Agent Query Understanding and LLM-Based Reranking for Chinese Statute Retrieval
> 来源：https://arxiv.org/abs/2501.18500 | 日期：20260319

## 问题定义
中国法律条文检索面临特殊挑战：(1) 法律术语专业性强，普通用户查询和法律条文之间存在语义鸿沟；(2) 相似法条多，需精准排序；(3) 中文法律语言表达严谨但不同于日常用语。LegalMALR提出多智能体框架解决法律检索的查询理解和精排问题。

## 核心方法与创新点
1. **多智能体查询理解**：
   - **意图识别Agent**：判断用户查询的法律意图类型（合同、侵权、刑事等）
   - **查询扩展Agent**：生成法律专业术语的扩展查询
   - **实体抽取Agent**：识别查询中的法律实体（当事人、行为、后果）
   - 多Agent协作产生增强查询，提升召回质量

2. **LLM重排序**：
   - 用LLM（如GPT-4/Claude）对候选法条进行细粒度相关性评分
   - 提示词设计：包含案情背景、候选法条，要求LLM解释相关性
   - 考虑法条间的逻辑关系（特殊法优于一般法原则等）

3. **混合检索**：
   - BM25关键词检索 + 语义向量检索的融合
   - 法律专用嵌入模型（在中国法律语料库上fine-tuning）

## 实验结论
- 在中国法律检索数据集上，NDCG@10比基础BM25提升约25-35%
- 多智能体查询理解比单一查询扩展提升约8-12%
- LLM重排序比cross-encoder重排序提升约3-5%（但延迟更高）

## 工程落地要点
1. **法律数据预处理**：中国法律条文需结构化处理（条文编号、章节层级）
2. **延迟控制**：多Agent + LLM重排序延迟高，可对简单查询走快速通道
3. **知识更新**：法律修订频繁，需建立条文更新机制
4. **幻觉风险**：LLM可能生成错误的法条解释，需要来源验证

## 常见考点
Q1: 法律领域的语义检索比通用领域有哪些特殊挑战？
> (1) 术语专业性：法律术语与日常用语含义不同（如"善意"在法律和日常语境中的差异）；(2) 精确性要求：法律检索错误代价极高；(3) 逻辑关系：法条间有引用、例外、优先级关系；(4) 长文本：法条通常较长，需要段落级检索。

Q2: 多智能体系统在信息检索中的优势是什么？
> 不同Agent专注不同子任务，通过分工协作提升整体质量；Agent间可以相互验证和纠错；灵活扩展（新增Agent处理新类型任务）。缺点：延迟增加，需要协调开销，调试复杂。

Q3: BM25和语义向量检索各有什么优缺点？如何混合？
> BM25：精确关键词匹配，无需训练，速度快，对罕见词/专有名词友好；缺点：无法理解语义，同义词匹配差。向量检索：语义理解强，同义词/改写效果好；缺点：需要训练，罕见词/专业术语效果差。混合：Reciprocal Rank Fusion (RRF) 或加权融合两种结果，取长补短。
