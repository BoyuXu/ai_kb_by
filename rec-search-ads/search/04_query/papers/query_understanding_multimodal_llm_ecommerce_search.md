# Query Understanding with Multi-Modal LLMs for E-Commerce Search

> 来源：arxiv | 领域：search | 学习日期：20260328

## 问题定义

电商搜索中用户意图理解面临多模态信息利用不足的问题：
- 用户查询往往简短模糊（如"红色连衣裙"），缺乏上下文
- 商品图文描述不一致，纯文本模型无法充分利用图像特征
- 多模态 Query 理解需在低延迟约束下完成

目标：利用多模态大语言模型（MM-LLM）提升电商搜索的查询理解质量，包括意图分类、查询改写、属性提取。

## 核心方法与创新点

### 架构
- **多模态 Query 编码器**：将用户文本查询 + 搜索历史图片 + 点击商品图 送入 MM-LLM
- **Query 改写模块**：利用 LLM 将口语化/模糊查询改写为更具语义丰富性的表达
- **意图分类**：MM-LLM 输出购物意图标签（品类/品牌/属性），指导后续检索策略
- **知识蒸馏**：大 MM-LLM 作为 Teacher，蒸馏出小型 Student 模型用于线上推理

### 创新点
1. 首次将多模态信息（用户行为图像）引入电商 Query 理解链路
2. 视觉上下文增强 Query 语义：相同文字查询+不同图片历史 → 不同改写结果
3. 端到端训练而非 pipeline 堆叠，减少误差传播

## 实验结论

- 在内部电商搜索数据集上，多模态 Query 改写相比纯文本基线：
  - 点击率（CTR）提升 +3.2%
  - 意图分类准确率提升 +5.7%
- 结合用户历史图片的 Query 改写质量（BLEU/ROUGE）显著优于无图片版本
- 蒸馏后小模型（125M）达到大模型（7B）90% 以上的改写质量，延迟降低 15×

## 工程落地要点

1. **延迟控制**：MM-LLM 推理成本高，必须量化 + 蒸馏，线上只部署 Student 模型
2. **A/B 测试漏斗**：Query 改写 → 召回率变化 → CTR/GMV，全链路监控
3. **数据飞轮**：线上用户点击行为反馈（RLHF 信号）持续迭代改写质量
4. **冷启动**：新品类/新查询无历史图片时，回退到纯文本模式
5. **安全过滤**：LLM 改写需过滤敏感/违规词汇，不能直接透传给召回

## 常见考点

**Q1: 电商搜索 Query 理解的主要子任务有哪些？**
A: 意图分类（品类/属性/品牌）、查询改写/扩展、实体识别（颜色/尺码/材质）、查询分类（导航型/信息型/交易型）

**Q2: 多模态信息如何改善 Query 理解？**
A: 用户历史点击图片提供了 "视觉语义" 上下文，帮助消歧（如"苹果" → 品牌还是水果），提升意图分类和改写准确性

**Q3: LLM 改写查询有什么风险？**
A: 过度改写偏离原意、引入幻觉词汇、增加检索漂移、延迟过高；需要置信度阈值控制 + 对比实验

**Q4: 如何将 7B MM-LLM 蒸馏到可线上部署的小模型？**
A: 用大模型输出的 soft labels（改写文本概率分布）训练小模型，结合 token-level KL 散度 + task-specific loss
