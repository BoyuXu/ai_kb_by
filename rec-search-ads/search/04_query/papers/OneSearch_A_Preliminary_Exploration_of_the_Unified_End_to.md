# OneSearch: A Preliminary Exploration of the Unified End-to-End Generative Framework for E-commerce Search
> 来源：https://arxiv.org/abs/2509.03236 | 领域：search | 日期：20260323

## 问题定义
探索电商搜索的统一端到端生成式框架，将查询理解、商品召回、排序统一到单一生成式模型中，消除多阶段pipeline的信息损失。

## 核心方法与创新点
- 统一生成框架：单一模型完成搜索全链路
- 电商专有tokenization：商品ASIN/SKU的层级token设计
- 查询-商品联合建模：同一序列内的查询和商品token交互
- 工业探索：在大规模电商数据上的初步验证

## 实验结论
初步实验表明，统一生成框架在精确匹配查询上Recall@100提升约5%，语义查询提升约12%；端到端训练比分阶段训练减少约15%的错误传播；但latency比传统pipeline高约3x。

## 工程落地要点
- 电商SKU量极大（亿级），需要高效的商品token压缩和索引策略
- 搜索场景的精确匹配要求高，生成式框架需要保证不遗漏精确匹配结果
- 端到端系统的可解释性和调试难度更高，需要专门的监控体系

## 常见考点
1. **Q: 电商搜索和内容搜索的主要差异？** A: 电商需要精确属性匹配（尺码/颜色）、价格排序、库存约束、个性化购买意图
2. **Q: 统一搜索框架的核心优势？** A: 减少多阶段信息损失、联合优化、更好的语义理解
3. **Q: 亿级SKU的生成式召回可行性？** A: 需要层级token（类目→品牌→SKU），递归beam search，将搜索空间从O(N)降至O(log N)
4. **Q: 如何保证生成式搜索不遗漏精确匹配？** A: 约束解码（必须包含query关键词的商品）、混合召回（生成式+BM25）
5. **Q: 电商搜索的排序因素有哪些？** A: 相关性、点击率预估、转化率预估、价格、好评率、物流速度、个性化
