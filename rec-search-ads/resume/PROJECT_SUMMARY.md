# 项目总结：5 个可用于面试的广告/搜索/推荐系统案例

## 项目清单与 Elevator Pitch

### 项目1：广告 CTR 预估模型优化
📄 文件：`project_1_ctr_optimization.md`  
⏱️ 行数：~1000 行

**30 秒 Pitch：**  
> 我在某大型广告平台优化 CTR 预估模型，发现核心瓶颈不在模型复杂度，而在样本偏差。通过 ESMM 多任务学习和知识蒸馏，我把模型精度从 AUC 0.72 提升到 0.78，同时推理延迟从 20ms 压到 8ms。这个系统上线后，广告点击量增加 12%，直接带来月收入 +6000 万。

**核心贡献：**
- ESMM 多任务学习纠正样本偏差（CVR 预估 AUC 从 0.62 → 0.75）
- AutoFE 自动特征交叉 + 特征选择
- DeepFM 模型升级 + Bayesian 超参优化
- 知识蒸馏降低延迟（20ms → 8ms）
- AB 测试设计与灰度发布

**关键数据：**
- AUC：0.72 → 0.78（+8.3%）
- CTR：点击量 +12%
- 延迟：20ms → 8ms（-60%）
- 收入：月增 6000 万

**面试要点：** 样本偏差问题、ESMM 的数学推导、蒸馏如何保留精度、灰度发布的风险管理

### 项目2：多样性混排系统（Feed 优化）
📄 文件：`project_2_mixing_ranking.md`  
⏱️ 行数：~1200 行

**30 秒 Pitch：**  
> 我在某社交平台做 Feed 优化，发现一个反直觉的现象：单条点击率高的 Feed 反而让用户留存率下降。根本原因是同类广告堆积导致用户疲劳。我用 DPP（行列式点过程）来重新建模 Feed 混排问题，从'最大化单条 CTR'转变为'最大化列表多样性'。虽然单条 CTR 下降 2%，但用户留存提升 8%，最终 RPM 增加 5.9%。

**核心贡献：**
- 多样性度量设计（Entropy、Source Diversity、Content Diversity）
- 硬规则 → 启发式 → DPP → 多目标优化的演进
- DPP 数学原理与贪心采样算法
- 列表级别联合优化而非逐条优化
- 多目标优化求解帕累托前沿

**关键数据：**
- 多样性：Entropy 1.2 → 2.1（+75%）
- 用户留存：+8%（7 日留存 40% → 43%）
- 单条 CTR：-2%（因为把热门广告分散了）
- RPM：+5.9%（虽然单条 CTR 下降，但总展示数增加）
- 小广告主流量：+40%（从 5% → 7%）

**面试要点：** DPP 的数学原理、为什么列表优化比逐条优化更好、如何平衡 CTR 和多样性

### 项目3：搜索广告自动竞价系统
📄 文件：`project_3_bidding_system.md`  
⏱️ 行数：~1400 行

**30 秒 Pitch：**  
> 我在搜索广告平台设计了一个自动竞价系统。广告主之前需要手工调整出价，效率低，CPA 达标率只有 65%。我通过 ESMM 多任务学习来准确预估转化率，然后用对偶优化（Lagrangian）计算最优出价，再用 PID 控制器实时调整，让广告主只需设定'目标 CPA'。结果 CPA 达标率从 65% 提升到 92%，广告主转化量 +18%，平台收入 +7%。

**核心贡献：**
- CVR 延迟转化处理（7 天观察窗口）
- 对偶优化推导最优竞价公式：$bid = \beta \times CPA \times pCVR$
- PID 控制器动态调整系数 $\beta$
- Thomson Sampling 冷启动竞价
- 多目标优化与 KKT 条件

**关键数据：**
- CPA 达标率：65% → 92%（+27 pp）
- 广告主转化量：+18%
- 广告主成本：-12%（避免过度付费）
- 平台收入：+7%
- 竞价延迟：平均 45ms（RTB 需求 <100ms）

**面试要点：** 对偶优化的推导、PID 控制的参数调整、CVR 延迟转化的处理、冷启动与 Thomson Sampling

### 项目4：推荐系统冷启动优化
📄 文件：`project_4_coldstart_recommendation.md`  
⏱️ 行数：~1200 行

**30 秒 Pitch：**  
> 我优化了推荐系统的冷启动问题。新用户因为没有历史行为数据，系统推不了好的内容，导致 7 日留存率只有 28%。我用三个方案：（1）图神经网络学习用户属性之间的关系，（2）NLP+CV 自动理解新内容的特征，（3）Thompson Sampling 在推荐已验证内容和探索新内容之间平衡。结果新用户 Day 1 CTR 从 1.8% 提升到 2.9%，7 日留存率从 28% 提升到 35%（+25%）。

**核心贡献：**
- 用户相似度匹配 + 行为迁移
- GraphSAGE 图神经网络学习用户表示
- 元学习（Meta-Learning）快速适应新用户
- 内容理解：NLP + CV 特征提取
- Thompson Sampling 探索-利用平衡
- Contextual Bandit 框架联合优化

**关键数据：**
- Day 1 CTR：1.8% → 2.9%（+61%）
- 7 日留存率：28% → 35%（+25%，业界 top 水平）
- Day 30 留存：42% → 48%（+14%）
- 新用户 7 天交互：8.5 → 11.2 条（+32%）
- 内容多样性：+14%

**面试要点：** 冷启动的本质（分布偏移 vs 数据稀缺）、图神经网络在推荐中的应用、Thompson Sampling 的贝叶斯思想、Contextual Bandit 框架

### 项目5：CTR 模型蒸馏与在线服务优化
📄 文件：`project_5_model_distillation.md`  
⏱️ 行数：~1000 行

**30 秒 Pitch：**  
> 我优化了 CTR 预估模型的推理性能。原来的 DeepFM 模型虽然精度很高（AUC 0.78），但推理延迟 20ms，太慢了。我用三个技术：（1）知识蒸馏把模型从 50M 压到 8M，（2）INT8 量化进一步压缩和加速，（3）TensorRT 编译优化。最终推理延迟从 20ms 降到 6ms（70% 降低），模型大小从 450MB 降到 50MB，精度衰减控制在 <1%。GPU 成本月省 $1200。

**核心贡献：**
- 知识蒸馏原理与温度缩放
- 选择合适的 Student 模型（3 层 MLP）
- INT8 量化与量化感知训练（QAT）
- 量化校准数据的选择
- TensorRT 图优化与编译
- 批处理、缓存等系统级优化

**关键数据：**
- 推理延迟：20ms → 6ms（-70%）
- 模型大小：450MB → 50MB（-89%）
- AUC 衰减：0.783 → 0.773（-0.5%，可接受）
- 单机 QPS：2000 → 8000（+4x）
- GPU 成本：$2000/月 → $800/月（省 $1200）

**面试要点：** 知识蒸馏的温度参数、量化感知训练 vs 直接量化、TensorRT 优化原理、系统级优化的协同效应

## 面试讲述框架

### 通用 4 步讲述法

每个项目在面试中，用这个框架讲：

**Step 1：问题（Problem）** — 1 分钟
- 定量描述问题（具体的数据、指标）
- 问题对业务的影响（金钱、用户体验、竞争力）
- 之前的解决方案为什么不行（不要踩别人，说明问题的难度）

**Step 2：方案（Solution）** — 2 分钟
- 核心洞察（为什么这个方案能解决问题）
- 算法原理（关键公式、数学推导）
- 实现细节（不是算法论文，而是真实的工程细节）

**Step 3：结果（Results）** — 1 分钟
- 线上 AB 测试的具体数据
- 相比基线的改进（%）
- 对业务的影响（收入、用户、成本）

**Step 4：学到的（Lessons Learned）** — 30 秒
- 1-2 个关键洞察（不要太多）
- 如何应用到其他项目

## 常见面试追问与回答思路

### 项目 1 和 5 的追问

**Q: "ESMM 和知识蒸馏看起来都是把复杂问题简化。两者有什么区别？"**

A:  
ESMM 是**数据层**的方案，知识蒸馏是**模型层**的方案：
- ESMM：用多任务学习纠正数据本身的偏差（样本选择偏差）。根本上改变了学习的目标函数。
- 蒸馏：假设数据已经处理好，只是把知识从大模型转移到小模型。是模型压缩的技巧。

两者是正交的，可以同时用：好的数据（ESMM）+ 高效的模型（蒸馏）。

---

### 项目 2 的追问

**Q: "DPP 很复杂，为什么不直接用启发式的多样性惩罚？"**

A:  
启发式方法是贪心的，每次只看当前最高分的元素。DPP 虽然数学更复杂，但考虑全局最优：
- 贪心可能选择"热门 + 热门"（虽然单个分数都很高）
- DPP 会考虑"热门 + 新颖"（整体多样性更好）

但在线上，我们最终用的不是 DPP，而是**多目标优化**（显式权衡 CTR 和多样性）。原因是：
- DPP 理论优雅，但参数多，调优困难，延迟也高（25ms）
- 多目标参数少（只有 α），易调，延迟低（12ms），效果也很接近（多样性 2.05 vs DPP 的 2.1）

**教训**：理论上最好的 ≠ 工程上最优的。要考虑维护成本、延迟、可解释性。

---

### 项目 3 的追问

**Q: "对偶优化的公式 bid = β × CPA × pCVR 看起来很简单，为什么还需要 PID？"**

A:  
公式是理想情况。现实中有三个问题：

1. **pCVR 不准**：模型会犯错，导致竞价偏离
2. **环境在变化**：竞争对手的出价、用户偏好会变，静态 β 跟不上
3. **预算约束**：有时候达不到精确的目标 CPA，需要动态调整

PID 的作用是：
- 观察实际 CPA 与目标的差距
- 动态调整 β，使实际 CPA 逐渐逼近目标
- 不会剧烈波动（有 D 项防振荡）
- 自动适应环境变化

简单的 β 可能是 1.0 不动。有 PID 后，β 会实时变化（如 0.95、1.05、0.98...），逐渐收敛到最优值。

### 项目 4 的追问

**Q: "Thompson Sampling 和 ε-贪心有什么区别？为什么选前者？"**

A:  
两者都是探索-利用平衡，但思想不同：

**ε-贪心：**
```
以概率 ε 随机探索
以概率 1-ε 选择最好的
```
问题：
- ε 是固定的。即使积累了很多证据表明某个选项很好，仍然要随机探索
- 初期浪费太多在明显不好的选项上

**Thompson Sampling：**
```
对每个选项，基于观察数据维护一个 Beta 分布
采样一个"可能的转化率"
选择采样出来最高的选项
```
优点：
- 自适应：初期不确定性大，采样波动大，自然探索
- 后期数据多，分布变尖，自动聚焦到最优
- 理论有保障（Bayesian 最优）

在实验中，Thompson Sampling 的"学习曲线"比 ε-贪心 更平滑，能更快收敛到最优选项。

### 项目 5 的追问

**Q: "知识蒸馏里，温度参数的最优值怎么确定？"**

A:  
通常用网格搜索：T ∈ [1, 2, 3, 5, 10, 20]，在验证集上评估。

但有个经验法则：

```
如果 Base Model 的困难样本多（即模型常犯错）：
  → 用大的 T（如 10），让软标签很平滑
  → 这样 Student 能学到 Teacher 的"错误思考过程"

如果 Base Model 很准（几乎不犯错）：
  → 用小的 T（如 2-3），因为软标签信息量有限
  → 更应该依赖 hard label
```

另一个实用技巧是观察 **Student 和 Teacher 的输出分布**：
- 如果 Student 的输出聚集在 0.5（犹豫不决），说明 T 太大，学不到东西
- 如果 Student 和 Teacher 的输出差异很大，说明蒸馏没有起效，T 可能太小

最优的状态是：Student 的输出分布接近 Teacher（说明学到了知识）。

## 面试时的常见错误与避免

### ❌ 错误讲述方式

❌ **"我用了 DeepFM 模型，AUC 提升了 8%"**
- 太肤浅，没有展现思考深度
- 没有说明**为什么** DeepFM 更好
- 没有说明**代价**是什么（延迟、成本）

✅ **正确讲述方式：**
> "我们发现 CTR 模型的瓶颈不在精度，而在样本选择偏差。基础 LR 模型只能从点击样本学转化率，导致 CVR 预估系统性低估。我用 ESMM 多任务学习，在展示和点击两个层级同时建模，纠正了这个偏差。通过这个方案，CVR 的准确性从 AUC 0.62 提升到 0.75。虽然模型变复杂了（推理延迟从 10ms 到 20ms），但我用知识蒸馏压回到 8ms，精度基本保持。整体来看，新模型的商业价值（精度 + 速度 + 成本）都更好。"

### ❌ 夸大其词

❌ **"我从零开始设计了一个推荐系统"**（实际上是优化现有系统的一个模块）

✓ **实话实说：**
> "我在现有推荐系统的基础上，针对新用户冷启动问题进行了优化。具体是..."

## 最后建议

1. **深入理解，不要背诵**。面试官会追问，如果你讲不出算法细节，就暴露了。
2. **准备 3-5 个深度问题的答案**。比如 ESMM 的数学推导、DPP 的行列式为什么能衡量多样性、PID 怎么调参。
3. **有数据比没数据好**。用具体的数字说话（AUC 0.72 → 0.78，不是"提升很多"）。
4. **说明权衡和选择**。不是选最复杂的方案，而是选最合适的方案。项目2最后没用 DPP，而是多目标优化，要能解释为什么。
5. **联系实际**。不要只讲算法，要讲工程挑战（延迟、成本、稳定性）。
