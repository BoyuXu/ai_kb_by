## 四、面试深度要求

### 必须能手推的公式（高频考点）
1. **CTR向量加权**：$\tilde{v}_d = v_d \cdot \sqrt{\text{CTR}(d) / \bar{\text{CTR}}}$
2. **贝叶斯CTR平滑**：$\text{CTR\_smooth} = \frac{\text{click} + \alpha}{\text{impression} + \alpha + \beta}$
3. **BLEU公式**：$\text{BLEU} = BP \cdot \exp\left(\sum_{n=1}^{N} w_n \log p_n\right)$
4. **MMR公式**：$\text{MMR} = \arg\max_{d_i \in C \setminus S} [\lambda \text{sim}(d_i, q) - (1-\lambda) \max_{d_j \in S} \text{sim}(d_i, d_j)]$
5. **Shapley值**：$\phi_i(v) = \sum_{S \subseteq N \setminus \{i\}} \frac{|S|!(|N|-|S|-1)!}{|N|!} [v(S \cup \{i\}) - v(S)]$

### 必须能写的代码
1. 动态K检索的Python实现
2. CTR贝叶斯平滑
3. Self-BLEU计算
4. MMR算法实现

### 面试中可能被深挖的方向
- **系统设计题**：画出完整架构，说清楚每个组件的选型理由

## 六、项目背景与业务价值

### 为什么要做广告创意生成？
- **广告主痛点**：人工写文案效率低、创意枯竭、无法个性化
- **平台痛点**：低质量创意拉低整体CTR，影响广告收入
- **AI机会**：RAG+LLM 可以利用历史高效创意，生成有参考依据的新文案

### 系统核心价值量化
- 广告主创意制作时间：从2小时 → 10分钟（**12x提效**）
- CTR提升：高质量参考创意 + CoT推理 → 预期**+5~15%**
- 创意多样性：从人工5条 → 系统50条候选，覆盖更多用户群体

### 与竞品系统的差异化
| 特性 | 本系统 | 直接Prompt LLM | 纯模板系统 |
|-----|-------|----------------|----------|
| 历史创意参考 | ✅ RAG检索 | ❌ | ❌ |
| CTR效果保障 | ✅ 加权检索 | ❌ | 部分 |
| 诉求理解 | ✅ CoT拆解 | 部分 | ❌ |
| 多样性保证 | ✅ MMR+监控 | ❌ | ❌ |
| 失败可解释 | ✅ SHAP归因 | ❌ | ❌ |
