# 每日项目生成计划

## 概览
- **目标**：每天生成 10 个可用于简历/面试的项目案例
- **执行时间**：每天下午 15:00（Asia/Shanghai 时区）
- **执行方式**：Cron Job + MelonEggLearn 自动化
- **输出位置**：`~/Documents/ai-kb/resume/daily_projects_YYYYMMDD.md`
- **索引更新**：`~/Documents/ai-kb/resume/projects_inventory.json`

---

## 项目生成规则

### 项目分布（每日 10 个轮转）
- **Day 1**：广告系统 4 个 + 推荐系统 4 个 + 搜索系统 2 个
- **Day 2**：NLP/LLM 3 个 + 创新方向 3 个 + 跨域融合 4 个
- **Day 3**：深度学习模型优化 4 个 + 实时系统工程 3 个 + 数据处理 3 个
- **Day 4**：用户增长 3 个 + 商业化 3 个 + 平台治理 4 个
- **Day 5**：重复周期或创新新方向

### 项目案例结构（每个 300-400 字）
```
### 项目N：[项目名称]

**背景与痛点**：
- 公司/平台面临的核心问题
- 为什么这个问题重要（业务影响）

**算法方案**：
- 核心创新点（1-2 句话）
- 技术细节（公式 / 架构 / 框架）
- 与现有方案的区别

**效果数据**：
- 关键指标提升（具体数字）
- A/B 测试结果
- 副作用（如有）

**技术栈**：
- 框架 / 库 / 工具

**核心洞察**（最关键，1-2 句）：
- 这个项目最值得学习的地方
- 可以迁移到其他场景的通用知识
```

## 面试使用指南

### 如何选择讲哪个项目
1. **如果面试官来自广告团队**：优先讲广告系统项目（CTR、竞价、混排）
2. **如果面试官来自推荐团队**：优先讲推荐系统项目（冷启动、排序、多样性）
3. **如果面试官看重工程能力**：讲工程类项目（实时系统、模型优化）
4. **如果面试官看重创新**：讲创新方向项目（因果推断、GNN）

### 讲项目的故事框架
1. **背景**（30 秒）：公司 / 问题规模 / 为什么重要
2. **核心挑战**（30 秒）：为什么基础方案不行
3. **我的方案**（1 分钟）：关键创新点 + 技术细节
4. **效果**（30 秒）：数字结果 + 业务影响
5. **学到的**（1 分钟）：可迁移的洞察 + 如果重做会怎样改

**总计 3-4 分钟，让面试官有追问的空间。**
