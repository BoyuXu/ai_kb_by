# 冷启动问题：推荐/广告/搜索的统一视角

> 冷启动 = 没有历史数据时如何决策。三个域都有这个问题，但具体场景和解法不同。

## 为什么冷启动很难？

核心矛盾：**你需要数据才能做好决策，但你需要做决策才能收集数据**。

典型恶性循环：
- 新物品没有点击数据 → 召回/排序给低权重 → 曝光少 → 数据更少 → 永远冷启动

## 三域冷启动对比

| 维度 | 推荐（新物品/新用户）| 广告（新广告主/新创意）| 搜索（新文档）|
|------|---------------------|---------------------|------------|
| 冷启动程度 | 中（有品类/标签先验）| 重（无历史，无法出价）| 轻（有内容可以索引）|
| 代价 | 曝光浪费，用户体验差 | 广告主钱烧错，ROAS 差 | 新内容找不到 |
| 核心挑战 | 数据稀疏 | 出价不知道设多少 | 相关性未知 |

## 推荐：新物品冷启动

**方法1：内容特征替代行为特征**
- 冷启动期：用物品文字描述/图片的 Embedding 代替协同过滤向量
- 过渡期（获得 100+ 曝光后）：混合使用内容+行为特征
- 成熟期（1000+ 曝光）：纯行为特征

**方法2：类目先验**
- 新物品 → 所属类目的平均统计（平均 CTR、平均转化率）
- 做召回的初始分数：`score = alpha * 类目均值 + (1-alpha) * 个性化分`，alpha 随曝光量降低

**方法3：Bandit 探索**
- UCB（Upper Confidence Bound）：`score = 预测均值 + C * sqrt(log(t)/n)`
- 曝光少（n 小）时，不确定性项 C * sqrt(...) 大 → 强制探索

## 广告：新广告主冷启动

最核心问题：pCTR/pCVR 完全不知道，导致：
1. 出价太高 → 钱烧掉、ROAS 差
2. 出价太低 → 没有曝光、学不到数据

**解法：阶梯式探索预算**
- 平台给新广告主 100-500 元"探索预算"，以平台承担 bid 风险的方式获取初始数据
- 用这些数据估计 pCTR，然后切换到正常竞价

**解法：相似广告迁移**
- 找到同类目、同受众、同创意风格的历史广告 → 迁移其 pCTR 估计作为先验
- `pCTR_new = 0.3 * similar_ads_mean + 0.7 * model_prediction`（逐步减少先验权重）

## 搜索：新文档冷启动

搜索冷启动比推荐/广告轻，因为：
- 文档内容可以被索引（BM25 可以工作）
- 无需用户行为数据就能判断相关性

**仍有的挑战**：
- 权威性（Authoritativeness）：新文档没有引用/点击历史，PageRank 分很低
- 质量信号缺失：垃圾内容和高质量新内容外表相似

**解法**：
1. 沙盒期（7-14天）：降低权重，观察用户行为信号（点击、停留时长）
2. 内容质量快速评估：LLM 判断内容是否垃圾（语言质量、信息量）
3. 新鲜度加权：近期发布的文档在新闻/时效性 query 上给加分

## 通用原则：冷启动三板斧

1. **找先验**：类目均值、相似物品/广告/文档迁移
2. **限探索**：不要全量曝光新物品，用 Bandit/沙盒期控制风险
3. **快速迭代**：获得 100 个曝光后立即更新模型，不要等批量

## 面试：你会怎么解决新 App 上线后推荐系统无历史数据的问题？

> **答**：分三个阶段。第一阶段（0-1周）：纯内容特征，用物品文字/图片 Embedding 做相似度召回，不用行为数据；用热门内容兜底。第二阶段（1-4周）：积累了第一批用户的行为后，引入协同过滤信号，用 alpha 混合（从 alpha=0.9 内容逐渐降至 alpha=0.3）。第三阶段（1月+）：行为数据丰富，转为标准推荐算法，内容特征退为辅助。整个过程要监控用户留存，不能为了"填满推荐位"而降低质量损害第一印象。
