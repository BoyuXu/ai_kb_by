## 1. 经典信息检索

### 1.1 布尔检索模型

**核心概念**：最早的检索模型，文档和查询都表示为词项的布尔表达式（AND/OR/NOT）。匹配是精确的二值判断——要么匹配要么不匹配，无相关性排序。

**深度要求**：了解

**面试高频考点**：
- 布尔模型的局限性是什么？（无法排序、结果集太大或太小、不支持部分匹配）
- 实际系统中布尔检索用在哪里？（过滤阶段、结构化字段筛选）

### 1.2 倒排索引（Inverted Index）

**核心概念**：搜索引擎的核心数据结构。从词项到文档列表的映射（Posting List），每个 posting 记录文档ID、词频、位置信息等。支持快速的 term-level 查询。构建流程：文档 → 分词 → 建立 term→doc 映射 → 排序压缩。

**深度要求**：精通

**面试高频考点**：
- 倒排索引的构建流程？内存中怎么构建、磁盘上怎么合并？
- Posting List 的压缩算法：VByte、PForDelta、Roaring Bitmap
- 跳表（Skip List）在倒排索引中的作用？如何加速 AND 查询？
- 实时索引（Real-time Index）如何实现？增量索引 vs 全量索引
- 倒排索引 vs 正排索引的区别和各自用途

### 1.3 TF-IDF

**核心概念**：词频-逆文档频率，衡量一个词对文档的重要性。$\text{TF-IDF}(t,d) = \text{TF}(t,d) \times \log\frac{N}{\text{DF}(t)}$。TF 衡量词在文档中的频率，IDF 衡量词的全局稀缺性。高频但在所有文档都出现的词（如"的"）TF-IDF 低。

**深度要求**：熟悉

**面试高频考点**：
- TF-IDF 的数学公式和直觉解释
- TF 的不同变体：raw count、log normalization、augmented TF
- TF-IDF 的局限性（无法捕捉语义、无法建模词序）
- 与 BM25 的对比

### 1.4 BM25

**核心概念**：基于概率检索模型的评分函数，是 TF-IDF 的改进版本。核心公式：

$$\text{BM25}(q,d) = \sum_{t \in q} \text{IDF}(t) \cdot \frac{f(t,d) \cdot (k_1+1)}{f(t,d) + k_1 \cdot (1-b+b \cdot \frac{|d|}{\text{avgdl}})}$$

两个关键超参：$k_1$（控制词频饱和度，通常1.2-2.0）和 $b$（控制文档长度归一化，通常0.75）。

**深度要求**：精通

**面试高频考点**：
- BM25 公式推导和每个参数的含义
- $k_1$ 和 $b$ 的取值对结果的影响？$k_1 \to 0$ 退化成什么？
- BM25 相比 TF-IDF 的改进在哪里？（词频饱和、文档长度归一化）
- BM25F（字段加权版本）的原理
- BM25 在现代搜索系统中的角色（通常作为一路召回 baseline）

### 1.5 向量空间模型（VSM）

**核心概念**：将文档和查询都表示为高维向量空间中的向量，用余弦相似度衡量相关性。每个维度对应一个词项，权重通常用 TF-IDF。

**深度要求**：了解

**面试高频考点**：
- 余弦相似度 vs 欧氏距离 vs 内积的区别和适用场景
- VSM 的局限性（维度灾难、稀疏性、无法捕捉语义）

## 2. Query 理解

### 2.1 分词（Tokenization / Word Segmentation）

**核心概念**：将用户输入的查询字符串切分为有意义的词项。中文分词是搜索系统的第一步，直接影响召回质量。主流方法：基于词典（正向/逆向最大匹配）、基于统计（HMM/CRF）、深度学习（BERT-based NER+分词联合）。

**深度要求**：熟悉

**面试高频考点**：
- 中文分词的主要难点？（歧义切分、未登录词、新词发现）
- Jieba 分词的原理？（DAG + 动态规划 + HMM 新词）
- 搜索场景分词与 NLP 通用分词的区别？（搜索偏细粒度、支持同义词扩展）
- 分词错误对搜索的影响和兜底策略

### 2.2 Query 纠错（Query Correction）

**核心概念**：检测并修正用户查询中的拼写错误、拼音输入、同音字错误等。核心技术：编辑距离（Levenshtein Distance）、拼音纠错、N-gram 语言模型、Seq2Seq 纠错模型。

**深度要求**：熟悉

**面试高频考点**：
- 编辑距离算法的时间复杂度？如何优化？（BK-Tree、SymSpell）
- 拼音纠错的实现思路
- 纠错的 Precision/Recall 权衡（错纠 vs 漏纠哪个更严重？）

### 2.3 Query 改写（Query Rewriting）

**核心概念**：将用户原始 query 转化为更适合检索的形式。包括同义词替换、Query 扩展（QE）、Query 缩减、Query 模板化。现代方法：LLM-based query expansion（如 HyDE、Query2Doc）、强化学习改写。

**深度要求**：精通

**面试高频考点**：
- Query 改写的目的和常用方法？
- Pseudo Relevance Feedback (PRF) 的原理？RM3 公式？
- HyDE（Hypothetical Document Embedding）的思路？
- Query 扩展和 Query 缩减分别解决什么问题？
- 改写后如何保证语义不漂移？

### 2.4 意图识别（Intent Classification）

**核心概念**：判断用户查询的搜索意图类型。常见分类：导航型（找特定网站）、信息型（获取知识）、交易型（购买/下载）。电商场景更复杂：品类意图、品牌意图、属性筛选意图等。方法从规则 → 分类模型 → BERT → LLM。

**深度要求**：熟悉

**面试高频考点**：
- 搜索意图的三大类型及判断依据
- 电商搜索中意图识别如何影响排序策略？
- 多意图 query 如何处理？

### 2.5 命名实体识别（NER）

**核心概念**：从 query 中识别出有意义的实体（品牌、品类、属性值、地理位置等）。搜索 NER 与通用 NER 的区别：query 短、无上下文、实体类型与业务强相关。方法：BiLSTM-CRF → BERT-CRF → Prompt-based NER。

**深度要求**：熟悉

**面试高频考点**：
- CRF 在序列标注中的作用？为什么不直接用 Softmax？
- 搜索 NER 面临的特殊挑战？
- NER 结果如何用于改善搜索（实体链接、属性筛选、知识图谱查询）？

### 2.6 Query 理解的工程 Pipeline

**核心概念**：完整的 Query 理解流程是一个多阶段 pipeline：原始 query → 预处理（大小写/繁简/全半角）→ 分词 → 纠错 → NER → 意图识别 → 改写/扩展 → 同义词扩展 → 下游检索。每个模块的输出作为下一个模块的输入。

**深度要求**：熟悉

**面试高频考点**：
- Pipeline 中各模块的顺序为什么重要？
- 如何评估 Query 理解模块的整体质量？
- 线上 Query 理解的延时预算一般多少？如何优化？

## 3. 召回

### 3.1 稀疏召回

**核心概念**：基于词项匹配的召回方式，利用倒排索引实现。代表方法：BM25、TF-IDF。优点是可解释性强、无需训练、对精确匹配（品牌名、型号）效果好。进阶：学习型稀疏检索（SPLADE），用 BERT 生成稀疏向量，结合了语义理解和倒排索引的高效性。

**深度要求**：精通

**面试高频考点**：
- SPLADE 的原理？如何在倒排索引框架下引入语义？
- 稀疏检索的优缺点？什么场景下稀疏检索优于稠密检索？
- 词项权重学习（Learned Sparse Retrieval）的训练方法

### 3.2 稠密向量召回（Dense Retrieval）

**核心概念**：将 query 和文档编码为稠密向量，用向量相似度（内积/余弦）进行检索。代表模型：DPR（Dual Encoder + 对比学习）、E5、GTE、BGE 等。训练方式：对比学习 + 难负例挖掘（Hard Negative Mining）。

**深度要求**：精通

**面试高频考点**：
- Bi-Encoder vs Cross-Encoder 的区别？为什么召回阶段必须用 Bi-Encoder？
- 难负例挖掘（Hard Negative Mining）的方法和重要性
- 对比学习的 Loss 函数：InfoNCE 公式 $\mathcal{L} = -\log\frac{e^{s(q,d^+)/\tau}}{\sum_i e^{s(q,d_i)/\tau}}$
- In-batch Negatives 的原理和局限性
- 如何解决稠密检索的"词汇失配"问题？

### 3.3 近似最近邻检索（ANN）

**核心概念**：精确的最近邻搜索在高维空间代价太高（$O(N)$），ANN 以微小的精度损失换取数量级的加速。四大类方法：
- **基于树**：KD-Tree（低维有效，高维退化）
- **基于哈希**：LSH（局部敏感哈希，理论优美但实际不如其他方法）
- **基于量化**：PQ（Product Quantization）、OPQ、ScaNN
- **基于图**：HNSW（层次化可导航小世界图，当前最优之一）

**深度要求**：精通

**面试高频考点**：
- HNSW 的构建和搜索过程？时间复杂度？
- PQ（乘积量化）的原理？为什么要分段量化？
- IVF（倒排文件索引）+ PQ 的组合方案
- Faiss、ScaNN、Milvus 的对比和选型
- ANN 索引的 Recall@K vs QPS 权衡曲线
- 向量维度选择对 ANN 性能的影响

### 3.4 混合检索（Hybrid Retrieval）

**核心概念**：结合稀疏检索和稠密检索的优势。融合策略：RRF（Reciprocal Rank Fusion）、线性加权、学习型融合。RRF 公式：$\text{RRF}(d) = \sum_{r \in R} \frac{1}{k + r(d)}$，其中 $k$ 通常取 60。

**深度要求**：精通

**面试高频考点**：
- 为什么混合检索通常优于单一检索？（互补性：精确匹配 + 语义匹配）
- RRF 融合的原理和超参数选择
- 混合检索的工程实现方案（并行双路 vs 串联级联）
- 何时稀疏召回更优、何时稠密更优？

### 3.5 多路召回架构

**核心概念**：工业搜索系统通常部署多路召回通道：BM25、向量召回、个性化召回、热门召回、实体召回等，各路结果合并后送入排序阶段。核心问题：各路召回数量分配、去重、截断策略。

**深度要求**：精通

**面试高频考点**：
- 多路召回 vs 单路召回的优劣？
- 各路召回数量如何分配？
- 多路召回结果如何合并和去重？
- 新增一路召回如何评估其增量价值？

## 4. 排序（Learning to Rank）

### 4.1 Pointwise 方法

**核心概念**：将排序问题转化为回归或分类问题。每个文档独立预测相关性分数（或分类标签），按分数排序。代表：线性回归、Logistic Regression、单文档 GBDT。Loss：MSE（回归）或 Cross-Entropy（分类）。

**深度要求**：熟悉

**面试高频考点**：
- Pointwise 方法的根本缺陷？（忽略文档间的相对顺序，无法直接优化排序指标）
- 为什么实际工业界很多 CTR 预估本质上是 Pointwise？

### 4.2 Pairwise 方法

**核心概念**：将排序转化为文档对的偏序关系学习。给定 query $q$ 和文档对 $(d_i, d_j)$，学习 $d_i$ 是否应排在 $d_j$ 前面。代表：RankSVM、RankNet、LambdaRank。RankNet Loss：$\mathcal{L} = -\bar{P}_{ij}\log P_{ij} - (1-\bar{P}_{ij})\log(1-P_{ij})$。

**深度要求**：熟悉

**面试高频考点**：
- Pairwise 相比 Pointwise 的优势？
- RankNet 的梯度推导
- Pairwise 方法的计算复杂度问题（$O(n^2)$ 文档对）

### 4.3 Listwise 方法

**核心概念**：直接优化整个排序列表的质量。代表方法：
- **LambdaMART**：在 GBDT 框架上定义 Lambda 梯度，直接优化 NDCG。Lambda 梯度 = RankNet 梯度 $\times$ $|\Delta \text{NDCG}|$
- **ListNet**：用 Plackett-Luce 模型定义排列概率
- **SoftRank**：对排序指标做平滑近似

**深度要求**：精通

**面试高频考点**：
- LambdaMART 的 Lambda 梯度是什么？为什么乘以 $|\Delta \text{NDCG}|$？
- LambdaMART 为什么是工业界最常用的 LTR 方法？
- Listwise vs Pairwise vs Pointwise 的对比总结
- ApproxNDCG Loss 的原理

### 4.4 GBDT/XGBoost 在排序中的应用

**核心概念**：GBDT 家族（XGBoost、LightGBM、CatBoost）是传统排序模型的主力。优势：天然处理特征交叉、对特征工程友好、训练快、可解释性好。LambdaMART = Lambda梯度 + MART（Multiple Additive Regression Trees）。

**深度要求**：精通

**面试高频考点**：
- XGBoost 的目标函数和正则项？二阶泰勒展开的推导？
- XGBoost vs LightGBM 的区别？（Level-wise vs Leaf-wise、直方图加速）
- GBDT 在排序中的特征工程经验
- 树模型 vs 深度模型在排序中的对比

## 5. 语义匹配

### 5.1 DSSM（Deep Structured Semantic Model）

**核心概念**：微软提出的双塔语义匹配模型。Query 和 Doc 各自通过 DNN 编码为低维向量，用余弦相似度衡量匹配度。输入用 word hashing（字母 trigram）解决 OOV 问题。训练用点击数据 + 随机负采样。

**深度要求**：精通

**面试高频考点**：
- DSSM 的网络结构和训练方式
- Word Hashing 的原理和优势
- DSSM 的局限性（无法建模 query-doc 细粒度交互）
- DSSM → CDSSM → ARC-I/ARC-II 的演进

### 5.2 Bi-Encoder

**核心概念**：DSSM 的 BERT 时代升级版。Query 和 Doc 分别用预训练语言模型（BERT/RoBERTa）编码为向量，离线计算 Doc 向量并建索引，在线只需编码 Query 并做 ANN 检索。代表：DPR、Sentence-BERT、E5、GTE、BGE。

**深度要求**：精通

**面试高频考点**：
- Bi-Encoder 的训练策略：对比学习、难负例挖掘、知识蒸馏
- [CLS] Pooling vs Mean Pooling vs Last Token Pooling 的区别
- 如何提升 Bi-Encoder 的效果？（更好的负例、更大 batch、指令微调）

### 5.3 Cross-Encoder

**核心概念**：将 query 和 doc 拼接后一起输入 BERT，用 [CLS] 输出做相关性预测。能捕捉 query-doc 之间的细粒度词级交互，效果显著优于 Bi-Encoder，但无法预计算 doc 向量，只能用于重排阶段。

**深度要求**：精通

**面试高频考点**：
- Cross-Encoder 为什么效果好但不能用于召回？
- Cross-Encoder 的推理延时优化方案
- 如何用 Cross-Encoder 蒸馏 Bi-Encoder？

### 5.4 ColBERT（Late Interaction）

**核心概念**：Bi-Encoder 和 Cross-Encoder 的折中。Query 和 Doc 分别编码为 token-level 向量序列，用 MaxSim 操作计算相似度：$s(q,d) = \sum_{i} \max_{j} \mathbf{q}_i^\top \mathbf{d}_j$。保留了细粒度交互能力，同时 Doc 向量可以离线预计算。

**深度要求**：精通

**面试高频考点**：
- ColBERT 的 MaxSim 操作为什么有效？
- ColBERT 的存储开销问题及解决方案（ColBERTv2 的残差压缩）
- Late Interaction vs Early Interaction 的对比
- ColBERT 在工业界的部署经验

### 5.5 Poly-Encoder

**核心概念**：Facebook 提出的介于 Bi-Encoder 和 Cross-Encoder 之间的方案。Doc 端用 Bi-Encoder 编码，Query 端生成多个全局向量（Poly codes），通过注意力机制与 Doc 向量交互。效果接近 Cross-Encoder，速度接近 Bi-Encoder。

**深度要求**：了解

**面试高频考点**：
- Poly-Encoder 的 Poly codes 数量如何选择？
- 与 ColBERT 的对比

## 6. 深度学习排序模型

### 6.1 Wide & Deep / DeepFM

**核心概念**：Wide & Deep 结合线性模型（记忆）和 DNN（泛化）。DeepFM 用 FM 层替代 Wide 部分，自动学习二阶特征交叉。这些模型最初用于推荐 CTR 预估，在搜索排序中同样适用。

**深度要求**：熟悉

**面试高频考点**：
- Wide 部分和 Deep 部分分别学什么？
- FM 的原理和参数量计算
- DeepFM vs Wide&Deep 的区别

### 6.2 DCN / DCN-V2（Deep & Cross Network）

**核心概念**：用 Cross Network 显式建模高阶特征交叉。DCN-V2 引入混合专家 Cross 层：$\mathbf{x}_{l+1} = \mathbf{x}_0 \odot (\mathbf{W}_l \mathbf{x}_l + \mathbf{b}_l) + \mathbf{x}_l$，自动学习任意阶交叉特征。

**深度要求**：熟悉

**面试高频考点**：
- Cross Network 如何实现高阶特征交叉？
- DCN-V2 的 Mix 模式是什么？
- 与 AutoInt（Self-Attention 做特征交互）的对比

### 6.3 DIN / DIEN 在搜索中的应用

**核心概念**：
- **DIN**（Deep Interest Network）：用 Attention 机制从用户历史行为中提取与当前 query/item 相关的兴趣表示，而非简单 pooling。搜索场景中，根据当前 query 对用户历史搜索/点击做 attention 加权。
- **DIEN**（Deep Interest Evolution Network）：引入 GRU + AUGRU 建模兴趣演化过程，捕捉用户兴趣的时序变化。

**深度要求**：精通

**面试高频考点**：
- DIN 的 Attention 机制和普通 Attention 的区别？
- DIEN 的辅助 Loss 是什么？为什么需要？
- AUGRU 如何根据目标 item 控制兴趣演化？
- DIN/DIEN 在搜索 vs 推荐场景的差异
- [[concepts/attention_in_recsys]] 中 Attention 在搜广推的统一视角

### 6.4 BERT-based 排序

**核心概念**：直接用 BERT 作为排序模型，输入 [CLS] query [SEP] doc [SEP]，用 [CLS] 向量做相关性打分。MonoBERT、DuoBERT（pairwise 方式）。通过预训练+微调范式，大幅超越传统特征工程方法。

**深度要求**：精通

**面试高频考点**：

- 预训练目标对排序效果的影响（MLM、ICT、Representative Word Prediction）

### 6.5 多目标排序

**核心概念**：搜索排序需要同时优化多个目标：相关性、点击率、转化率、内容质量、时效性等。融合方式：加权求和、Pareto 最优、多塔多目标（MMOE/PLE）。最终排序分 = $\sum_i w_i \cdot f_i$。

## 7. 评估指标

### 7.1 离线评估指标

**核心概念**：

| 指标 | 公式 | 特点 |
|------|------|------|
| **Precision@K** | 前K个结果中相关文档占比 | 简单但不考虑排序位置 |
| **Recall@K** | 前K个结果覆盖了多少相关文档 | 衡量召回阶段常用 |
| **MAP** | $\text{MAP} = \frac{1}{\|Q\|}\sum_q \text{AP}(q)$, $\text{AP} = \frac{1}{R}\sum_{k=1}^{n} P(k) \cdot \text{rel}(k)$ | 二值相关性，考虑位置 |
| **MRR** | $\text{MRR} = \frac{1}{\|Q\|}\sum_q \frac{1}{\text{rank}_q}$ | 只关注第一个相关结果的位置 |
| **nDCG@K** | $\text{DCG} = \sum_{i=1}^{K}\frac{2^{rel_i}-1}{\log_2(i+1)}$, $\text{nDCG} = \frac{\text{DCG}}{\text{IDCG}}$ | 多级相关性，位置折扣 |
| **ERR** | $\text{ERR} = \sum_{r=1}^{n}\frac{1}{r}\prod_{i=1}^{r-1}(1-R_i) \cdot R_r$ | 级联浏览模型，考虑用户停止概率 |

**深度要求**：精通

**面试高频考点**：
- nDCG 的完整计算过程（手算题高频出现）
- MAP vs nDCG 的区别？什么场景用哪个？
- MRR 适用于什么场景？（问答、导航型搜索）
- 为什么不能只看 Precision/Recall？（忽略排序质量）

### 7.2 在线评估指标

**核心概念**：
- **CTR**（点击率）：$\frac{\text{clicks}}{\text{impressions}}$
- **Session 成功率**：用户在一个搜索 session 中找到满意结果的比例
- **Abandonment Rate**：搜索后无点击的比例（但注意 good abandonment）
- **Dwell Time**：用户停留时间，侧面反映结果质量
- **Reformulation Rate**：用户修改 query 重新搜索的比例（高→结果不满意）
- **pFound**：Yandex 提出的基于浏览模型的在线指标

**深度要求**：熟悉

**面试高频考点**：
- 在线指标和离线指标不一致怎么办？
- 位置偏差（Position Bias）如何影响 CTR？如何修正？
- A/B 测试的设计要点（流量分桶、最小样本量、显著性检验）
- Interleaving 实验 vs A/B 测试的优劣

## 8. 搜索系统架构

### 8.1 全链路架构

**核心概念**：典型搜索系统三段式架构：

```
用户查询 → Query理解 → 召回（多路） → 粗排 → 精排 → 重排 → 展示
                         ↑                              ↑
                    离线索引构建                    业务规则/多样性
```

各阶段的文档规模：召回（百万→万级）→ 粗排（万→千级）→ 精排（千→百级）→ 重排（百→十级）。

**深度要求**：精通

**面试高频考点**：
- 画一个完整的搜索系统架构图并解释每个模块
- 各阶段的模型复杂度和延时预算如何分配？
- 为什么需要粗排？直接精排不行吗？
- 召回数量如何确定？

### 8.2 索引构建

**核心概念**：索引构建是搜索的离线核心。流程：文档采集 → 解析清洗 → 分词 → 建倒排索引 → 建正排索引（用于排序特征获取）→ 建向量索引。索引格式：SSTable、Segment-based（类 Lucene）。

**深度要求**：熟悉

**面试高频考点**：
- 索引构建的 MapReduce 流程
- 索引压缩的常用方法
- 如何处理文档更新？（增量索引 + 定期全量重建）

### 8.3 实时索引

**核心概念**：支持文档的秒级/分钟级可搜索。核心思路：内存中维护一个小的实时索引，接收新文档写入；定期与磁盘上的主索引合并。挑战：一致性、内存管理、合并策略。参考 Elasticsearch 的 Segment 机制。

**深度要求**：熟悉

**面试高频考点**：
- Elasticsearch 的 Near Real-Time 搜索原理？Refresh vs Flush？
- 实时索引的一致性如何保证？
- 实时索引对搜索延时的影响

### 8.4 分布式检索

**核心概念**：大规模文档无法在单机存储和检索，需要分布式架构。两种分片策略：
- **文档分片（Document Partitioning）**：按文档 ID 分片，每个分片包含部分文档的完整索引
- **词项分片（Term Partitioning）**：按词项分片，每个分片包含部分词项的完整倒排列表

查询流程：Broker 接收查询 → 广播到各分片 → 各分片本地检索 → 结果合并排序。

**深度要求**：熟悉

**面试高频考点**：
- 文档分片 vs 词项分片的优缺点？工业界为什么普遍用文档分片？
- 分布式 Top-K 查询的正确性保证
- 分片数量如何确定？
- 副本（Replica）的作用和一致性问题

### 8.5 缓存策略

**核心概念**：搜索系统中缓存是降低延时的关键。多级缓存：
- **结果缓存**：缓存 query → 结果列表（命中率高但需处理时效性）
- **倒排列表缓存**：缓存热门 term 的 posting list
- **计算缓存**：缓存排序特征的中间计算结果

缓存淘汰策略：LRU、LFU、ARC。缓存一致性：TTL + 主动失效。

**深度要求**：了解

**面试高频考点**：
- 搜索结果缓存的 TTL 如何设置？
- 缓存命中率与结果新鲜度的权衡
- 个性化搜索如何影响缓存策略？

## 9. NLP 基础

### 9.1 词向量（Word Embeddings）

**核心概念**：将离散的词表示为连续的稠密向量。Word2Vec（CBOW/Skip-gram）、GloVe（全局共现矩阵分解）、FastText（子词级别）。这些静态词向量无法处理一词多义。

### 9.2 Transformer

**核心概念**：基于自注意力机制的序列建模架构，完全替代 RNN/LSTM。核心组件：Multi-Head Self-Attention + Feed-Forward Network + Layer Normalization + Residual Connection。注意力计算：$\text{Attention}(Q,K,V) = \text{softmax}(\frac{QK^\top}{\sqrt{d_k}})V$。

**深度要求**：精通

**面试高频考点**：

- Pre-Norm vs Post-Norm 的区别
- [[fundamentals/attention_transformer]] 和 [[fundamentals/transformer_evolution]]

### 9.3 BERT

**核心概念**：双向 Transformer 编码器，通过 MLM（Masked Language Model）和 NSP（Next Sentence Prediction）预训练。BERT 革新了 NLP，在搜索中的应用：语义匹配、排序、Query 理解、NER。

**深度要求**：精通

**面试高频考点**：

### 9.4 预训练语言模型在搜索中的应用

**核心概念**：预训练模型在搜索全链路的应用：

## 10. RAG / LLM 增强搜索

### 10.1 RAG 架构

**核心概念**：Retrieval-Augmented Generation，将检索结果注入 LLM 上下文以增强生成质量。基本流程：Query → 检索 Top-K 文档 → 拼接为 Prompt → LLM 生成答案。解决 LLM 幻觉和知识时效性问题。

### 10.2 Advanced RAG 技术

**核心概念**：
- **Query 改写**：HyDE（用 LLM 生成假设文档做检索）、Query2Doc、Step-Back Prompting
- **Chunk 策略**：固定窗口 / 语义分块 / 递归分块 / Late Chunking
- **重排**：LLM Reranker、Cross-Encoder Reranker
- **Multi-hop RAG**：多跳推理检索（IRCoT、Self-Ask）

**深度要求**：精通

### 10.3 GraphRAG / 知识图谱增强

**核心概念**：将知识图谱与 RAG 结合。方法：从文档中提取实体和关系构建知识图谱 → 检索时结合图结构进行子图检索 → 将子图信息注入 LLM 上下文。Microsoft GraphRAG 的 Community Detection + Summarization 方案。

### 11.2 长尾 Query

### 11.3 多模态搜索
